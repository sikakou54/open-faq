FR02: ユーザー管理(オーナー + メンバー)

この要件は、1 契約のアカウント利用者を、オーナー 1 名とプロジェクト単位の役割を持つメンバーで安全に運用できるようにすることを目的とし、以下について記載します。

  • オーナーとメンバーのユーザーモデルと権限の表し方
  • メンバーの招待と、本人によるアカウント有効化
  • 役割の変更・プロジェクトからの離脱・アカウントの無効化
  • メンバー数のアラートと、重複招待の防止

1. 業務要件

ID 観点 ステークホルダ 業務要件 関連 NFR
BR-037 ユーザーモデルと権限 オーナー / メンバー
  • オーナー：アカウント登録時に自動で決まり、全プロジェクトへの全権とオーナー専有機能を恒久的に保持する(1 契約 1 名、降格・譲渡なし)。メンバーが 0 人でも単独で契約運営を継続できる
  • メンバー：権限はプロジェクトごとの役割(プロジェクト管理者 / メンバー)で表し、割り当てられたプロジェクトのデータのみ操作できる
 NFR-303
BR-038 メンバー招待 オーナー / プロジェクト管理者 プロジェクト単位で、メールアドレスと当該プロジェクトでの役割を指定してメンバーを招待する。氏名は招待時に入力せず、本人が有効化時に入力する(個人情報保護の原則)。同じ人を複数プロジェクトに参加させる場合は各プロジェクトで個別に招待する。 —
BR-039 アカウント有効化 アカウント利用者 招待された本人が、招待リンク(有効期限 7 日)から有効化ページに進み、次を入力してアカウントを有効化する。有効化の完了時に役割の割当が有効になる。
  1. 氏名(表示名)
  2. 初回パスワード
  3. 利用規約への同意
  4. プライバシーポリシーへの同意
 —
BR-040 役割変更・離脱・編集 オーナー / プロジェクト管理者 当該プロジェクトのメンバーについて、役割の変更・「プロジェクトから外す」・メールアドレスの更新を行える。自分自身の降格・割当解除はできない。変更は監査記録し、本人へお知らせとメールで通知する。氏名は本人のみが変更できる。 —
BR-041 アカウント無効化・データ消去 システム 最後の有効なプロジェクト割当が解除された時に、アカウントを利用停止し、全ログインセッションと未使用招待を無効化する。90 日経過後にデータを消去する。同じメールアドレスで再招待された場合は別の新しい利用者として扱う。 —
BR-042 メンバー数アラート システム メンバー数に固定上限は設けず、上限への接近や急増を検知した時に、オーナーおよび同一契約内の全プロジェクト管理者へアラート通知する。 —
BR-043 重複招待の防止 システム 同一オーナー配下で、既存の有効・招待中アカウントと同一メールアドレスのメンバーを重複して招待できないようにする。別オーナー配下では別アカウントとして扱う。 —
BR-044 オーナー専有機能 オーナー 課金情報・支払方法・請求履歴・退会・規約再同意・契約設定、およびプロジェクトの作成・編集・削除はオーナー専有とし、いずれの役割にも付与しない。プロジェクト単位の利用上限・無料枠は、オーナーまたは当該プロジェクト管理者が変更できる。 —

2. 機能要件

ID 要件 関連業務要件 優先度
FR-015 オーナーおよびプロジェクト管理者は、自分が担当するプロジェクトでメンバーを招待できること。
  1. 招待はメールアドレスと当該プロジェクトでの役割(プロジェクト管理者 / メンバー)を指定して行う
  2. 招待時に氏名(表示名)は入力しない(個人情報保護の原則により、他人の氏名を管理者が任意に登録することを禁止する)。氏名は招待された本人が有効化時に入力する(FR-016d)
  3. メンバーの管理は常にプロジェクト単位で行う
  4. 同一の人を複数プロジェクトに参加させる場合は各プロジェクトで個別に招待する
  5. 同一オーナー配下でメールアドレスが一致する場合は同一の利用者として扱い、新たに参加するプロジェクト分の割当のみを追加する
 BR-038 P0
FR-015a オーナーはアカウント新規登録時に自動的に決まり、全プロジェクトに対する全権とオーナー専有機能を恒久的に保持すること。オーナーは契約境界に基づく全権を持ち、プロジェクト管理者の割当を別途付与されることなく自プロジェクトを操作できる。オーナーは役割変更の対象外とし、降格・譲渡は提供しない(MVP)。1 契約につきオーナーは 1 人とする BR-037 P0
FR-015b メンバーの権限はプロジェクトごとに付与する役割で表し、役割はプロジェクト管理者とメンバーの 2 種とすること。プロジェクト管理者は当該プロジェクトの FAQ 管理・未解決質問管理・ログ参照・メンバー管理(当該プロジェクト範囲の招待・離脱)を、メンバーは当該プロジェクトの FAQ 管理・未解決質問管理・ログ参照を行える。個別の権限フラグや権限テンプレートは提供しない BR-037 P0
FR-015c 契約単位の課金情報・支払方法・請求履歴・退会申請・規約再同意・契約設定、およびプロジェクトの作成・編集・削除はオーナー専有機能とし、いずれの役割でも付与できないこと。プロジェクト単位の利用上限・無料利用枠は、オーナーまたは当該プロジェクト管理者が変更できること BR-044 P0
FR-015d メンバーは割り当てられたプロジェクトに属するデータ(FAQ・要対応の質問・質問ログ等)のみにアクセスできること。プロジェクトから外す操作を行ったメンバーが他に有効なプロジェクト割当を持たない場合は、そのアカウントを利用できない状態にする。オーナーは割当の有無に関わらず全プロジェクトを閲覧・操作できる BR-037 P0
FR-015e プロジェクト作成時、オーナーは作成したプロジェクトに対して全権を持つこと。オーナーへ管理者割当を別途付与する処理は行わない。プロジェクト作成時に「管理者を指定」する操作は設けない。他者をプロジェクト管理者として加える場合は、当該プロジェクトのメンバー管理からプロジェクトユーザーを招待し、管理者として割り当てる BR-037 P0
FR-016 招待されたメンバーは招待リンクから有効化ページに進み、本人による氏名(表示名)入力・初回パスワード設定・利用規約およびプライバシーポリシーへの同意を行ってアカウントを有効化できること。招待リンクの有効期限は 7 日。期限切れの場合はオーナーまたは当該プロジェクト管理者が再送信する BR-039 P0
FR-016a 招待時に指定された役割の割当は、招待対象者が有効化(氏名入力・パスワード設定・規約同意)を完了した時点で有効になること BR-039 P0
FR-016b 招待リンクの再送信は、オーナーまたは当該プロジェクト管理者のみが行えること。再送信時は旧リンクを失効させ新リンクを発行する。招待中のメンバーを取りやめたい場合は、招待取消ではなくメンバー削除(FR-019)で対応する BR-038 P0
FR-016c 有効化前のメンバーは、招待時に指定された役割の割当を予約状態として保持するが、ログインおよび利用はできないこと(有効化完了時に初めて利用可能となる) BR-039 P0
FR-016d 招待されたメンバーは、有効化ページで次を入力して有効化を完了する。利用規約とプライバシーポリシーは個別の文書として版管理され、双方の最新版への同意が有効化の必須条件となる。氏名は本人のみが入力でき、招待元の管理者は事前に入力できない(個人情報保護の原則)。氏名は本人が個人設定で後から変更できる
  1. 氏名(表示名)
  2. 初回パスワード
  3. 利用規約への同意
  4. プライバシーポリシーへの同意
 BR-039 P0
FR-016e 有効化の完了時には、本人が入力した氏名・初回パスワード・規約同意(利用規約とプライバシーポリシーでそれぞれ 1 件、計 2 件)の記録と、招待時に指定されたプロジェクト・役割の割当有効化が、まとめて確実に行われること。どのプロジェクト・役割への招待であったかは招待リンクに紐づけて管理する BR-039 P0
FR-017 アカウント利用者は、現在開いているプロジェクトに割り当てられたメンバーの一覧と詳細を参照できること。詳細には種別(契約オーナー / プロジェクト管理者 / プロジェクトメンバー)、状態(利用中 / 招待中)、当該プロジェクトでの役割を含める。最終ログイン日時は表示しない。招待期限と再送の導線はメンバー招待・編集の操作に集約する。契約をまたいだメンバー一覧は提供しない BR-040 P0
FR-018 オーナーおよびプロジェクト管理者は、自分(オーナー)に紐づくメンバーの メールアドレスのみ を更新できること(氏名の更新は不可)。
  1. 氏名(表示名)は本人のみが有効化ページおよび個人設定で変更できる(個人情報保護の原則により、管理者は他人の氏名を編集できない)。編集時、氏名は閲覧のみとし、未入力時は「未設定」と表示する
  2. オーナー自身の氏名・メールの更新はオーナー本人のみが行える
  3. プロジェクト管理者が更新できる対象は、当該プロジェクトに割当のあるメンバーに限る
  4. 個人ごとの通知受信オプトアウト(重要度別の受信切替)は MVP では提供しない。プロジェクト関連通知は常時オンとし、当該プロジェクトの全メンバー(オーナー + 管理者 + メンバー)へ即時配信する(画面上の受信設定は設けない)
 BR-040 P0
FR-018a プロジェクト管理者(オーナーを含む)は、当該プロジェクトのメンバー 1 人の割当について、役割(プロジェクト管理者 / メンバー)の変更と「プロジェクトから外す」操作を行えること。他プロジェクトの割当には影響しない。自分自身の降格・割当解除はできない。他メンバーを管理者からメンバーへ降格する操作には、誤操作を防ぐ確認を必須とする BR-040 P0
FR-018b 役割変更・削除時には、変更前後のプロジェクトと役割の差分を記録(監査)し、当該メンバーへお知らせとメールで通知できること BR-040 P0
FR-018c オーナーは、メンバーを別プロジェクトへ追加で割り当てる場合、当該プロジェクトに切り替えて招待する。プロジェクト管理者は他プロジェクトへの割当はできず、自分が管理者であるプロジェクトの範囲でのみ招待・役割変更・削除を行える(範囲内の役割制約は FR-019 を正とする) BR-038 P0
FR-019 メンバーの利用終了は「プロジェクトから外す」操作に統合すること。対象が他プロジェクトに有効な割当を持たない場合、当該プロジェクトの割当解除に加えてアカウントを利用できない状態にし、ログインセッションと未使用の招待を無効化する。オーナー自身は対象外とし、契約をまたいだメンバー削除の画面・機能は提供しない BR-041 P0
FR-019a 「プロジェクトから外す」操作は当該プロジェクトの割当のみを解除する。対象が他プロジェクトに有効な割当を持つ場合は、利用者アカウントと他の割当を維持し、次回以降の利用可否判定に反映する。他に有効な割当がない場合は、アカウントが利用できなくなる旨を確認時に明示する BR-041 P0
FR-019b 最後の有効なプロジェクト割当を解除した場合は、全ログインセッションの無効化、未使用招待の無効化、および一定期間(90 日)経過後にデータが完全に消去される予定であることを確認時に明示する。利用停止後に同じメールアドレスで再招待された場合は、過去の削除済みデータとは別の新しい利用者として扱う BR-041 P0
FR-020 招待リンクには有効期限を設けること。有効期限: 7 日。期限切れ後は招待の再送信で新リンクが発行される BR-039 P0
FR-021 メンバー数に固定の上限を設けず、利用量のしきい値に基づいて上限への接近や急増を検知し、アカウント利用者(オーナー / メンバー)へアラート通知できること BR-042 P0
FR-021a メンバー数のしきい値接近・超過時のアラート通知先は、オーナーおよび同一契約内でいずれかのプロジェクトの管理者である全員とすること(最も重要度の高い通知の宛先と一致する) BR-042 P1
FR-021b 契約にメンバーが 1 人も登録されていない状態でも、オーナーが単独で契約運営を継続できること(オーナーは全機能を保持する) BR-037 P0
FR-021c 同一オーナーに紐づく既存の有効・招待中アカウントと同一メールアドレスでメンバーを重複して招待できないこと。別オーナー配下では別アカウントとして扱う BR-043 P0
