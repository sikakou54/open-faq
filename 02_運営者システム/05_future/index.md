## 3. 今回対象外との関係

| 対象外内容 | 理由 | 将来対応候補 |
|---|---|---|
| 運営者サブロール分割 | MVP は単一 `service_operator` ロールで職務分掌は内部統制レビューでカバー | FUT-01 |
| 4-eyes 原則の完全強制 | MVP は 3 操作をハードゲート化し、7 操作は承認ログのみ | FUT-03 |
| 24/365 オンコール体制 | MVP は営業時間 + 緊急時対応のみ(NFR-820) | FUT-02 |
| 顧客契約 SLA | MVP は内部 SLO のみ、GA + 6 ヶ月の実績をもとに契約 SLA を策定 | FUT-02 |
| 利用規約・プライバシーポリシー本文最終承認版 | MVP は暫定文言で限定公開対応 | FUT-02 |
| OSS ライセンス一覧・SBOM 整備 | MVP は手動レビュー、CI 自動生成は Post-MVP | FUT-02 |
| Runbook 追加(RB-009〜RB-018) | MVP は最小 Runbook セット、Future で起動条件・コマンド・通知・証跡・ポストモーテムテンプレート整備 | FUT-02 |
