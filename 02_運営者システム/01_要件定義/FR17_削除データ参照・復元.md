## 1. 機能概要

利用者から「誤って削除した」等の問い合わせを受けた際の救済手段として、顧客管理システムは論理削除済み(DB 上は残るが利用上は削除済み扱いとなった状態)データの参照と復元を行う。**復元の非対称性**(削除は利用者側で行えるが、復元は運営者側でのみ行える設計)を採用し、メインシステムの管理画面側にはこの機能を提供しない。

**本書側で正本管理する FR 範囲**: FR-200〜FR-212(削除データ参照・復元 + 管理者ユーザー運営者操作通知)。

## 3. 補足説明・統合参照

- FR-204 契約復元は §6.2.3 の 4-eyes 対象(承認ログ区分)。物理削除キュー投入後(NFR-704 の 30 日経過後)は復元不可。
- 復元後の状態(下書き / アーカイブ / 有効 / 利用中)は本書 §9.9 削除データ復元フローを参照。
- 連携 IF #4(復元実行)を通じてメイン側の論理削除データを復元する(本書 §11.2)。
- 復元通知は連携 IF #12(運営者操作通知)経由でメイン側の管理者ユーザーへ届く。
- 管理画面からの復元は提供しない(FR-040b: 「FAQ・プロジェクト・管理者ユーザー・お知らせ・契約の削除は管理画面から復元できないこと」を参照)。
