## 1. 機能概要

管理者向けお知らせ受信箱(Inbox)の一覧・詳細表示・既読/未読管理・種別フィルタ(請求確定 / 運営お知らせ / システム通知)・ヘッダ通知ベル・受信箱お知らせ生成(請求確定 / 運営お知らせ / システム通知)・退会時削除・ポーリング・監査ログ記録を扱う。本書は基本的にメインシステム要件と整合する参照側で、運営者側はお知らせを生成する(SCR-094 / FR-149 / FR-188)。

## 3. 補足説明・統合参照

- お知らせ種別は「請求確定 / 運営お知らせ / システム通知」の 3 区分(§3 用語定義)。
- お知らせ重要度は `low` / `normal` / `high` / `critical`(本書 [FR12_通知.md](FR12_通知.md) と整合)。`critical` は強制メール通知。
- お知らせ種別マッピング表(重要度・生成主体・主な発生イベント・オプトアウト可否・配信経路)は **メインシステム §12.5 が正本**(本書 §12.5 参照)。重複生成対策(同一契約・同一イベント種別の 60 分集約)もメインシステム §12.5 に従う。
- お知らせ受信箱保持期間は 1 年(NFR-705)。
