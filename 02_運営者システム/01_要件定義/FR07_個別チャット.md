## 1. 機能概要

未解決質問とは独立したチャット ID 単位の部屋作成・メール登録必須化・投稿・通知・再入室・上限・機密情報注意喚起を扱う。案件状態、FAQ 登録、部屋のクローズ状態は扱わない。

## 2. 収録 FR-NNN 一覧

| ID | 要件 | 優先度 |
|---|---|:---:|
| FR-080 | 未解決質問とは独立したチャット ID ごとに個別チャット部屋を作成できること | P0 |
| FR-081 | エンドユーザーを個別チャット部屋へ誘導できること | P0 |
| FR-082 | 個別チャット利用前にメールアドレス登録を必須とすること | P0 |
| FR-083 | エンドユーザーはチャット ID または通知メールから同じチャット部屋へ再入室できること | P0 |
| FR-084 | 再入室の際、本人性を確認できる仕組み(メール送信トークン、有効期限付きリンク)を設けること。**トークン有効期限は MVP 初期値 30 日**。期限切れ時はメール再送リンクから新規トークンを発行できる。同一メールアドレスで複数のチャット ID がある場合はチャット ID を識別できること | P0 |
| FR-085 | エンドユーザーはチャットにメッセージを投稿できること | P0 |
| FR-086 | 管理者ユーザーはチャットに返信できること | P0 |
| FR-087 | チャット投稿時に相手側へメール通知できること | P0 |
| FR-088 | 個別チャットの作成・投稿・既読・通知・再入室は未解決質問の状況を更新せず、チャット画面から FAQ 登録や未解決質問の状況変更を行えないこと | P0 |
| FR-089 | 個別チャットには業務上のオープン / クローズ状態、自動クローズ、手動クローズ、再オープン、利用者待ち判定を設けないこと | P1 |
| FR-090 | チャット投稿に上限文字数および投稿頻度制限を設定できること | P0 |
| FR-091 | 機密情報(パスワード、決済情報等)入力を控えるよう注意喚起すること | P0 |

## 3. 補足説明・統合参照

- 再入室トークンの暗号方式は NFR-305 を正本とし、オーナー派生鍵で鍵付きハッシュを計算する。
- チャット投稿時の PII 検出・マスキングは本書 [FR05_AIパラメータ設定.md](FR05_AIパラメータ設定.md) FR-060(c)(d) と整合。
- 運営者はチャット本文を平常時には閲覧できない(FR-223 / 本書 [FR18_運営者の責務.md](FR18_運営者の責務.md))。
