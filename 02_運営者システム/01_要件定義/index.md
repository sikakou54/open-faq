## 1. システム概要

本サービス「open-faq」は、Webサイト運営者(管理者)が登録した FAQ をもとに、サイト訪問者(エンドユーザー)からの問い合わせに自動応答する SaaS である。公開済み FAQ に記載された内容のみを根拠に AI が回答することで、ハルシネーションを抑えながら問い合わせ自動化を実現する。回答できなかった質問は人手による個別対応へ確実に引き継ぎ、その内容を再度 FAQ に反映するループを回す。

本サービスは次の 2 サブシステムから成り、本書は **顧客管理システム(運営者向け運用コンソール)** の要件を機能グループ別に分割して保持する。

| サブシステム | 主な対象 |
|---|---|
| メインシステム | 管理者ユーザー向け管理画面、およびエンドユーザー向け FAQ ウィジェット |
| 顧客管理システム(本書) | サービス運営者(`service_operator`)向けの運用コンソール |

文書の同期・表記統一・検証に関する保守ルールはリポジトリ直下の `CLAUDE.md` に置く。

## 2. 背景・目的

Web サイト運営者は、(a) 訪問者からの問い合わせ対応コストの増大、(b) 回答品質のばらつき、(c) 夜間・休日の対応不足といった課題を抱えている。生成 AI による自動応答は手軽に導入できる反面、ハルシネーション(社内に存在しない事実をもっともらしく回答する現象)のリスクがある。

本サービスは「**公開済み FAQ に記載された内容のみ**」を根拠に AI が回答する仕組みを提供することで、ハルシネーションを抑えつつ問い合わせ自動化を実現する。顧客管理システム(本書)は、サービス運営者が SaaS 全体の安定運用・不正利用対応・課金確認・監査対応・削除データ復元・AI 推論パラメータ管理・4-eyes 承認等の内部統制を担保するための運用コンソールを提供する。

## 3. スコープ

本書は **MVP 範囲のみ** を扱う。MVP で実現する要件、初期値、受入条件、運用制約だけを本文に置く。

| 情報種別 | 本書での扱い | 正本 |
|---|---|---|
| 業務・機能・非機能の WHAT | MVP の要求事項、優先度、受入条件のみを記載 | 本書 |
| 方式・状態遷移・画面項目・外部連携方式 | 要件 ID から参照し、本文に重複掲載しない | [../02_基本設計/index.md](../02_基本設計/index.md) |
| API / DDL / JSON Schema / Cron / Queue / 実装パラメータ | 要件本文には置かない | [../03_詳細設計/index.md](../03_詳細設計/index.md) |

対象範囲は、運営者コンソール上での (a) 削除データ参照・復元、(b) AI 推論パラメータ・レート制限(契約単位)・上限件数(プロジェクト単位)の上書き、(c) お知らせ作成・配信、(d) 課金 Webhook 一次受信・リプレイ、(e) 監査ログ閲覧・エクスポート、(f) 4-eyes 承認、(g) 運営者ロール管理・MFA、(h) 運営者活動ダッシュボード、を含む。

## 4. 利用者・関係者

| 利用者 | 説明 | 主な操作 |
|---|---|---|
| オーナー | 契約作成時に確定する利用者(1 名固定)。譲渡・降格・停止・削除不可 | 契約内全機能、メンバー管理、課金情報操作、退会申請、規約再同意の承諾 |
| メンバー | オーナーまたは該当プロジェクト管理者から招待された利用者 | `project_users.role` = `admin`(プロジェクト管理者)/ `member`(メンバー)の組合せで許可された範囲 |
| エンドユーザー | 管理者の Web サイト訪問者 | 質問入力、回答確認、解決可否回答、メール登録、個別チャット利用 |
| サービス運営者 | 本 SaaS 提供者(`service_operator` ロール) | 障害対応、不正利用対応、課金確認、監査対応、削除データ復元、4-eyes 承認 等 |

| ステークホルダー | 関心事 |
|---|---|
| プロダクトオーナー | サービス価値の最大化、開発投資対効果 |
| 管理者ユーザー(顧客) | 問い合わせ対応の自動化、運用負荷低減、個別対応の効率化 |
| エンドユーザー | 疑問の早期解決、個別問い合わせの容易さ |
| サービス運営者 | 安定稼働、不正利用対策、法令遵守 |
| 開発者 | 要件の明確さ、実装容易性 |
| QA | 検証可能性、受入条件の明確さ |
| 法務/プライバシー担当 | 規約・プライバシー法令への適合 |
| 経営層 | 売上、コスト、リスク |

権限マトリクスおよび 4-eyes 原則対象操作の詳細は §6.2 / §6.2.3(本書 [../01_要件定義書.md](../01_要件定義書.md))を参照する。
