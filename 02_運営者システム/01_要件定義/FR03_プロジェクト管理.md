## 1. 機能概要

1 契約内で FAQ・ウィジェット・ログ・チャットを分けて管理するための「プロジェクト」概念を扱う。MVP では作成・編集・削除・許可ドメイン設定・関連データの取扱選択・上限監視・アーカイブを提供する。プロジェクト数のハード上限引上げは運営者承認(本書 [FR23_4-eyes承認.md](FR23_4-eyes承認.md) §6.2.3 対象)で個別契約単位に上書き可能とする。

|---|---|:---:|
| FR-030 | 管理者はプロジェクトを作成、編集、削除できること | P0 |

| FR-034 | プロジェクト削除時の関連データ(FAQ、ログ、チャット)の取扱いを利用者が確認・選択できること | P0 |
| FR-035 | プロジェクト数の **ハード上限は NFR-113(プロジェクト 50 / 契約)に従う** ものとし、加えて利用状況を監視し急激な増加(不正利用兆候)を検知して管理者へ通知できること。ハード上限の引上げは運営者承認(§6.2.3 4-eyes 対象)で個別契約単位に上書き可能。検知ロジック、通知経路、バッチ失敗時の扱いは基本設計 / 詳細設計を正本とする | P1 |
| FR-036 | プロジェクトをアーカイブ状態にできること(削除せず利用停止) | P1 |

## 3. 補足説明・統合参照

- プロジェクト削除はメインシステムの管理画面で行い、運営者側からの直接削除は提供しない。誤削除時の復元は本書 [FR17_削除データ参照・復元.md](FR17_削除データ参照・復元.md) FR-202(復元後はアーカイブ状態)を参照。
- 急激な増加検知の運営者通知は連携 IF #8(監視メトリクス取得)経由で観測される(本書 §11.2)。
- 許可ドメインのウィジェット側検証は FR-151 / FR-173(本書 [FR13_ウィジェット.md](FR13_ウィジェット.md) / [FR15_セキュリティ.md](FR15_セキュリティ.md))を参照。
