## 1. 機能概要

メール通知の送信契機(チャット投稿、問い合わせ ID 発行、チャット終了、FAQ 登録完了、請求確定、運営お知らせ等)・再送・サプレス・配信状態管理・運営お知らせ配信機能(FR-149)を扱う。請求確定通知(FR-148)と運営お知らせ(FR-149)は受信箱お知らせ生成(本書 [FR16_お知らせ.md](FR16_お知らせ.md) FR-187 / FR-188)と同期実行される。

## 3. 補足説明・統合参照

- NFR-502(再送ポリシー: 最大 5 回・指数バックオフ・累計 24 時間打ち切り)を遵守する。永久失敗(配信不能確定 / バウンス確定)はサプレスリスト登録・再送中止。
- NFR-504(共通送信ドメインのレピュテーション保護)に従い、FR-144 のしきい値監視を必須とする。
- 通知重要度は `low` / `normal` / `high` / `critical` の 4 段階。`critical` は強制送信(規約改定・重大セキュリティ通知・サスペンション確定など)で、オーナー + 同一オーナースコープで `account_project_grants.role='admin'` を 1 件でも保持する全プロジェクト管理者へ配信(リポジトリ CLAUDE.md / 共有概念正本ルール参照)。
- 連携 IF #7(お知らせ生成)を通じて運営者作成のお知らせをメイン側配信機構へ連携する(本書 §11.2)。
- FR-149 の本文 XSS 対策(R-011)は無害化処理の二段階適用(永続化前・表示時)で行う。
