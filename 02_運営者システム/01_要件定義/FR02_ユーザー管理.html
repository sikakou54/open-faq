1. 機能概要

利用者(オーナー / メンバー)の招待・有効化・参照・更新・削除・上限到達アラート等を扱う。本書は参照側であり、ユーザー管理の機能要件正本はメインシステム §8.3。実装はメインシステムが担い、運営者側は契約単位の上限引上げ等の管理(本書 FR23_4-eyes承認.html §6.2.3 の対象)を担う。

新規 ID(FR-015a〜c / FR-016a〜c / FR-018a〜b / FR-021a〜c / FR-336〜338)はメインシステム要件定義書を参照する。

3. 補足説明・統合参照

  • プロジェクト別ロール(project_users.role = admin / member)とオーナーの 3 ロール体系の運用ルールは ../02_基本設計/04_権限設計.html(参照のみ。正本はメイン側)を経由してメイン側 04 を正本とする。
  • オーナー専有機能(プロジェクト作成・編集・削除、課金、退会、規約再同意の承諾、メンバーアカウント完全削除)はプロジェクト別ロールでは付与できない(§3 用語定義参照)。
  • FR-021 のハード上限の運営者上書きは FR23_4-eyes承認.html で扱う 4-eyes ハードゲート/承認ログ対象とは別軸(承認ログ対象)であり、対応チケット ID 入力と監査ログ記録が必須(FR-231 / FR-229 と整合)。
