## 1. 機能概要

管理者の登録・ログイン・パスワード再設定・退会・再認証・規約再同意までの、本 SaaS 上の利用者アカウントライフサイクル全般を扱う。MVP では再認証はパスワード再入力、運営者は FR-222 のとおり MFA 必須(本書 [FR18_運営者の責務.md](FR18_運営者の責務.md) を参照)。ロック・セッション・パスワード強度の数値はメインシステム正本と整合させる(本書 §8.2 は参照側)。

| FR-002 | 登録時に利用規約とプライバシーポリシーへの同意を取得できること | P0 |
| FR-003 | メールアドレス確認(本人確認メール)ができること | P0 |
| FR-004 | ログイン、ログアウト、パスワード再設定ができること | P0 |
| FR-005 | 重要操作(パスワード変更、退会、課金情報変更、管理者ユーザー登録・停止・削除、月次上限件数変更)前に本人確認(再認証)を求めること。再認証方式は **MVP ではパスワード再入力**。**運営者は FR-222 のとおり MVP から MFA 必須**。再認証の有効期間は **当該操作 1 回のみかつ 15 分以内**とし、実装方式は基本設計を正本とする(メインシステム FR-005 と整合) | P0 |
| FR-006 | パスワードに最低長・複雑性要件を適用できること(最低 12 文字、英大文字・小文字・数字・記号のうち 3 種類以上)。パスワードリセットリンクの有効期限は **1 時間**(メインシステム FR-006 と整合) | P0 |
| FR-007 | ログイン試行の失敗回数制限ができること。**5 回連続失敗** で **15 分間ロック**し、時間経過または管理者解除で復旧できること。ロック単位、通知経路、監査記録の方式は基本設計を正本とする(メインシステム FR-007 と整合) | P0 |
| FR-008 | セッションタイムアウトを以下のとおり実装すること: (a) **無操作タイムアウト 30 分**、(b) **絶対タイムアウト 12 時間**(メインシステム FR-008 と整合) | P0 |
| FR-009 | 退会できること(猶予期間付き) | P0 |
| FR-011 | 利用規約・プライバシーポリシー改定時に再同意を取得できること | P0 |

## 3. 補足説明・統合参照

- FR-005 の再認証ポリシー(15 分・操作 1 回限り)は運営者高権限操作にも適用される(本書 FR-222、[FR18_運営者の責務.md](FR18_運営者の責務.md) 参照)。
- パスワードハッシュ方式は NFR-304(本書 §10.3)を正本とし、運営者は一般利用者より強いパラメータを用いる。具体パラメータは詳細設計を正本とする。
- 退会時の契約状態遷移は §9.8 / 9.8.1(本書 [../01_要件定義書.md §9.8](../01_要件定義書.md))を正本とする。サスペンションフローはメインシステム §9.10 を正本とする。
- 規約再同意の承諾はオーナー専有機能であり、メンバーには付与できない(§3 用語定義「オーナー専有機能」を参照)。
