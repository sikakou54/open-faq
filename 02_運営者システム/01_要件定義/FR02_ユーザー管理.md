## 1. 機能概要

利用者(オーナー / メンバー)の招待・有効化・参照・更新・削除・上限到達アラート等を扱う。**本書は参照側であり、ユーザー管理の機能要件正本はメインシステム §8.3**。実装はメインシステムが担い、運営者側は契約単位の上限引上げ等の管理(本書 [FR23_4-eyes承認.md](FR23_4-eyes承認.md) §6.2.3 の対象)を担う。

新規 ID(FR-015a〜c / FR-016a〜c / FR-018a〜b / FR-021a〜c / FR-336〜338)はメインシステム要件定義書を参照する。

## 2. 収録 FR-NNN 一覧

| ID | 要件 | 優先度 |
|---|---|:---:|
| FR-015 | オーナーおよびメンバー権限「ユーザー管理」保持者は、メールアドレス、氏名、付与するメンバー権限フラグ(0〜5 個)を指定して同一契約へメンバーを招待できること(メインシステム FR-015 と整合) | P0 |
| FR-016 | 招待されたメンバーは招待リンクからアカウントを有効化できること。招待リンクの有効期限は **MVP 初期値 7 日**。期限切れの場合はオーナーまたは「ユーザー管理」保持者が再送信する(メインシステム FR-016 と整合) | P0 |
| FR-017 | 利用者(オーナー / メンバー)は同一契約内のオーナーおよびメンバーの一覧および詳細を参照できること(メインシステム FR-017 と整合) | P0 |
| FR-018 | オーナーおよび「ユーザー管理」保持者は、同一契約内のメンバーの氏名、メールアドレス、通知設定を更新できること(メインシステム FR-018 と整合) | P0 |
| FR-019 | オーナーおよび「ユーザー管理」保持者は、同一契約内のメンバーを削除できること。オーナーは削除の対象外とする(メインシステム FR-019 / FR-333 と整合)。MVP では一時的な利用停止 / 再開機能は提供しない | P0 |
| FR-020 | 招待リンクには有効期限を設けること | P0 |
| FR-021 | メンバー数の **ハード上限は NFR-113(管理者ユーザー 100 名/契約)に従う** ものとし、利用量しきい値(80%=80 名で警告、100%=登録拒否)に基づいて上限接近・超過を検知し、オーナーおよび「ユーザー管理」保持メンバーへアラート通知できること。ハード上限の引上げは運営者承認(§6.2.3 4-eyes 対象)で個別契約単位に上書き可能とすること(メインシステム FR-021 / FR-021a と整合) | P0 |

## 3. 補足説明・統合参照

- メンバー権限フラグ 5 種(`faq:manage` / `chat:respond` / `users:manage` / `project:manage` / `logs:view`)の運用ルールは [../02_基本設計/04_権限設計.md](../02_基本設計/04_権限設計.md) を正本とする。
- オーナー専有機能(課金、退会、規約再同意の承諾)はメンバーに付与できない(§3 用語定義参照)。
- FR-021 のハード上限の運営者上書きは [FR23_4-eyes承認.md](FR23_4-eyes承認.md) で扱う 4-eyes ハードゲート/承認ログ対象とは別軸(承認ログ対象)であり、対応チケット ID 入力と監査ログ記録が必須(FR-231 / FR-229 と整合)。
