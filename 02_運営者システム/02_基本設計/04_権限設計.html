権限設計書(運営者)

1. 文書概要

1.1 目的

運営者システムの権限モデルを一元化する。単一ロール service_operator + 4-eyes 対象 10 操作(ハードゲート 3 + 承認ログ 7)を中心に、画面別・API 別の操作可否を定義する。

1.2 対象範囲

  • 対象: 運営者ロール(service_operator)、4-eyes 対象 10 操作、画面別・API 別権限
  • 対象外: 認可判定フロー(09_認証認可設計書.html が正本)、利用者ロール(01_メインシステム/個別設計書群/05_権限設計書.html 参照)

2. ロール一覧

(§3.1 から移管)

ロールID 表示名 説明文(UI) 利用者 範囲
service_operator サービス運営者 契約状況の確認、復元、配信、監査などの運用作業を行えます。重要操作には再認証や4-eyes承認が必要です。 運営事業者 全契約横断(各種ガード適用)

2.1 単一ロール採用理由

MVP は単一 service_operator ロールで運用。運営者は SCR-090〜098 の運用機能を扱い、操作ごとの安全性は以下で担保:

  1. IP allowlist(認証・認可設計書 09 §3)
  2. MFA(09 §4)
  3. 再認証(5 分以内、09 §5)
  4. 4-eyes 承認(10 操作のみ、本書 §7)
  5. 監査ログ全件(セキュリティ設計書 10 §7)

7.3 ハードゲート KV フラグの管理

KV キー: feature:hard-gate:<action_code>

  • 対象 action コードは 実装定数(コード内 const) で管理
  • 許可操作: 緊急一時無効化のみ(true → false 一時切替)
  • 発動条件: セキュリティインシデント + メイン要件 §6.2.2 の条件成立時
  • フラグ変更操作は 4-eyes 承認ログ対象(feature.hard_gate.toggle、retention_class='5y')
  • 誤操作で MVP 重大操作(owner.physical_delete)が単独実行可能になるリスクを排除

7.4 自己承認禁止 / 自己却下禁止 / 取下げ規約

ルール 内容
自己承認禁止 DB CHECK: requested_by != approved_by + アプリ層チェック
自己却下禁止 DB CHECK: requested_by != rejected_by + アプリ層チェック
取下げ規約 申請者本人(requested_by)のみ可。別運営者の取下げは不可

詳細は 09_認証認可設計書.html §8 を正本とする。

8. 4-eyes 未承認・権限不足時の動作

ケース システム動作 画面表示 メッセージID 関連エラーID
4-eyes 未承認で操作実行 403 + 専用コード 「この操作には承認が必要です。承認待ち一覧から申請してください」 MSG-OP-4EYES-NOT-APPROVED E-OP-4EYES-REQUIRED
4-eyes ハードゲート KV false 403 + 即時拒否 「現在この操作は利用できません。運用責任者に有効化を依頼してください」 MSG-OP-4EYES-HARDGATE-OFF E-OP-4EYES-HARDGATE
自己承認試行 422 + 専用コード 「自分が申請した操作は承認できません。他の運営者に承認を依頼してください」 MSG-OP-4EYES-SELF E-OP-4EYES-SELF
自己却下試行 422 + 専用コード 「自分が申請した操作は却下できません。他の運営者に確認を依頼してください」 MSG-OP-4EYES-SELF-REJECT E-OP-4EYES-SELF
承認 TTL 切れ(72h) 410 + 専用コード 「承認の有効期限が切れました。内容を確認して再申請してください」 MSG-OP-4EYES-EXPIRED E-OP-4EYES-EXPIRED
再認証期限切れ(5 分以内) 403 + 再認証画面誘導 「再認証が必要です。パスワードを入力して操作を続けてください」 MSG-OP-REAUTH-REQUIRED E-OP-REAUTH-REQUIRED
IP allowlist 拒否 401(即時) (拒否画面なし、HTTP のみ) - E-OP-IP-DENIED
payload_hash 改ざん検知 409 + 専用コード 「申請内容が変更されている可能性があります。最新の内容を確認してから承認してください」 MSG-OP-4EYES-PAYLOAD-CHANGED E-OP-4EYES-PAYLOAD-CHANGED
ticket_id 必須項目欠落 400 + 入力欄エラー 「操作チケット ID を入力してください」 MSG-OP-4EYES-TICKET-REQUIRED E-OP-INPUT-TICKET-REQUIRED
