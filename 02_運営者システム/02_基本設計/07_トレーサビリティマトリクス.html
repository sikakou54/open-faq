トレーサビリティマトリクス(運営者)

1. 文書概要

1.1 目的

運営者側の要件 → 機能 → 画面 → API → DB → 権限 → エラー → メッセージ → テスト観点 の対応関係を一元化する。

1.2 対象範囲

  • 対象: 運営者側全要件(FR-220〜232 / FR-300〜310 / NFR-602 / NFR-602d / NFR-707 / NFR-808-810 等)、運営者機能(F-OP-*)、運営者画面(SCR-090〜098 / SCR-AUTH / SCR-AUTH-M1 / SCR-HOME / SCR-APPROVALS / M1 / M2)、運営者 API、運営者専用テーブル(19 個)
  • 対象外: 利用者側要件(01_メインシステム/個別設計書群/08_トレーサビリティマトリクス.html 参照)
