# 観点(用語・概念の一貫性)
1. 同じ用語が異なる意味で使われている箇所はないか。
2. 異なる用語で同じ概念を指している箇所はないか(例: 役割/立場/ロール、契約/プラン、利用枠/無料枠、所有者/オーナー/作成者 等)。
3. ロール(オーナー/メンバー/管理者/運営)、権限、ステータス、プラン、プロジェクト、契約・課金アカウントといった重要概念が、要件層と基本設計層で一貫しているか。ある層で廃止/改名された概念が別の層に旧称で残っていないか。
4. 同義に見えて実は区別すべき概念(例: 「プロジェクトのオーナー」と「課金アカウントの所有者」)が、文書上で混同され区別がつかない箇所はないか。

厳守:
- 提供ソースのみ。推測で補完しない。各指摘は、ぶれている**両方**の用語の出現箇所(相対パス + 実在 ID または見出し)を根拠に挙げる。片方しか挙げられないものは挙げない。
- HTML/パイプ表が空に見えても指摘にしない。
- 指摘が無ければ「指摘なし」と明記。
