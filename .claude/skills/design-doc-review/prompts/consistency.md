# レビュー観点(整合性)
1. 用語の不統一(同一概念が複数語、ロール/状態名/区分の揺れ、用語規約との矛盾)。
2. ID 体系の不統一(旧体系の識別子が新体系と混在)。
3. ステータス/コード値の不統一(同じ状態が文書ごとに別名・別値)。
4. ロール/権限呼称の不統一(権限・API 認可・画面で揃っているか)。
5. エラーコードとメッセージの対応齟齬(片方にあって片方に無い)。
6. 廃止仕様・古い前提・重複記載・相互矛盾の残存。
7. 要件に存在しない仕様が基本設計に追加されていないか。
8. 層間整合: 画面↔API の I/O、API↔テーブルのデータ項目、権限↔画面/API、エラー↔メッセージ、逆引きリストの重複。
