## 10. Obsidianレビュー記録保存ルール

NotebookLMレビューで得られた重要なコンテキストはObsidianに保存する。

### 保存対象（必須）

- レビュー観点 / NotebookLM投入プロンプト / レビュー結果
- 重要指摘 / 指摘への対応方針 / 指摘が発生した原因
- 指摘対応後の再レビュー結果 / 再発防止策
- 設計判断に影響するレビューコメント
- 成果物間の調整が必要なレビュー指摘
- 課題化した指摘とその解決履歴

### 保存先

```
99_obsidian_context/
└── 06_review_logs/
    ├── REVIEW_001_全体構成レビュー.md
    ├── REVIEW_002_UI_APIモジュールSQLDBレビュー.md
    ├── REVIEW_003_共通定義レビュー.md
    └── REVIEW_004_再レビュー.md
```

## 11. Obsidian課題管理ルール

NotebookLMレビューで発見された課題はすべてObsidianに保存する。

### 課題保存先

```
99_obsidian_context/
└── 04_issues_and_solutions/
    ├── INDEX_課題一覧.md
    ├── ISSUE_001_課題名.md
    └── ISSUE_002_課題名.md
```

### 課題メモ命名規則

```
ISSUE-XXX_課題名.md
```

例: `ISSUE-001_状態値の定義重複.md`

### 課題ステータス

| ステータス | 意味 |
|---|---|
| draft | 課題メモ作成中 |
| active | 対応中または判断待ち |
| resolved | 解決済み |
| archived | 対応不要または履歴保管 |

### 運用ルール

- 課題は必ずObsidianにMarkdownとして保存する
- 課題の解決方針が決まるまで、設計書本文へ確定仕様として反映しない
- 推測で仕様を確定しない
- 課題が解決した場合も削除せず、Obsidian上に履歴として残す
- 未解決課題は `INDEX_課題一覧.md` で一覧管理する
