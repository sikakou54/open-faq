# レビュー観点(落とし込み漏れ)
1. 業務要件→機能要件→業務ユースケース→基本設計 の連鎖に断絶がないか(対応する下流が無い上流)。
2. 基本設計の各層(画面/画面イベント/API/テーブル/システム処理/権限/エラー/メッセージ)が必要分そろっているか。
3. バッチ/定期処理・外部連携(メール送信・決済・Webhook・全文検索等)が基本設計で設計されているか。
4. 逆方向: 基本設計に存在するが要件に根拠が無い仕様(要件外の追加仕様)はないか。
5. 異常系・例外系・権限不足・入力不正・データ不整合・外部連携失敗・競合(楽観ロック)・リトライ・順序/冪等性 の考慮漏れ。
6. 非機能(運用/監査/ログ/セキュリティ/性能/可用性/データ保持・削除ライフサイクル)の考慮漏れ。
