1. システム概要

本サービス(FAQ AI ウィジェット SaaS、サービス名: open-faq)は、Web サイト管理者(管理者ユーザー)が登録した FAQ をもとに、サイト訪問者(エンドユーザー)からの問い合わせに自動応答する SaaS である。

観点 概要
提供形態 クラウドサービス(SaaS)
利用者 Web サイト運営事業者、その従業員、Web サイト訪問者
提供価値 FAQ の自動応答、未解決問い合わせの集約、問い合わせ先案内、FAQ 改善ループ
課金 完全従量課金制 + 月次無料枠(MVP)
設置形態 利用者サイトへのスクリプトタグ埋め込み(ウィジェット)
アカウント管理 自前アカウント管理
メール配信 メール配信プロバイダ経由

システム構成: 本サービスは管理者向け管理画面とエンドユーザー向け FAQ ウィジェットから成る。本書はその要件定義の正本である。

スコープ(MVP のみ): 本書は MVP で実現する範囲のみを記載する。方式・状態遷移・画面項目・外部連携方式は基本設計を、実装の詳細は詳細設計をそれぞれ正本とする。

2. 背景・目的

Web サイト運営事業者は、訪問者からの問い合わせ対応コストの増大、回答品質のばらつき、夜間・休日対応の限界という課題を抱えている。生成 AI による自動応答は手軽に導入できる反面、社内に存在しない事実を「もっともらしく」回答してしまう ハルシネーション のリスクがある。

本サービスは、この課題に対し「公開済み FAQ に記載された内容のみ」を根拠に AI が回答する仕組みを提供することで、ハルシネーションを抑えながら問い合わせ自動化を実現する。回答できなかった質問は問い合わせ ID を付与して未解決質問として内部管理し、ウィジェットでは確認済み連絡先を案内する。その内容を FAQ に反映するループを回すことで、運用とともに自動回答率を高める。

差別化ポイント: ハルシネーション抑制 / 未解決質問の確実な収集 / FAQ 登録進捗の明確化 / 運用と改善の循環 / 1 契約内マルチプロジェクト。

3. スコープ

本書は MVP で実現する範囲のみ を記載する。

対象 内容
管理画面 アカウント、オーナー / メンバー、管理者ユーザー、プロジェクト、FAQ、ウィジェット、質問ログ、未解決質問、課金、設定の管理
公開ウィジェット Web サイト訪問者が質問し、FAQ に基づく AI 回答または未解決時の連絡先案内を受ける
FAQ 改善機能 未解決質問から FAQ を下書き・公開し、登録状況を管理できる
通知機能 請求確定、運営お知らせ、重要な状態変更をメール・受信箱で通知する
運用機能 ログ確認、障害対応、データ保持、退会処理、データ削除(契約全データエクスポートは将来対応)

MVP の安全境界: AI が FAQ にない内容を推測回答しないこと、FAQ の公開は管理者が編集画面で内容を確認したうえで状態(下書き / 公開中 / 非公開)を選択して保存する明示操作であること(FR-045)。具体要件は FR-051 を正本とする。

4. 利用者・関係者

本サービスは立場の異なる利用者を想定する。「認可範囲」は、オーナー境界・プロジェクト別の役割・ユーザー種別の組み合わせで決定する。

利用者種別 説明
オーナー(オーナーアカウント) 本サービスを契約する利用者で、1 アカウント = 1 契約単位。全機能を恒久保持(全プロジェクトに対する管理者相当の権限)。降格・譲渡不可
メンバー(メンバーアカウント) オーナーまたは該当プロジェクト管理者から招待された利用者。プロジェクトごとにプロジェクト管理者またはメンバーの役割が付与される。プロジェクト管理者は該当プロジェクト範囲で FAQ 管理 / 未解決質問管理 / ログ参照 + ユーザー管理(招待・離脱)、メンバーは該当プロジェクト範囲で FAQ 管理 / 未解決質問管理 / ログ参照のみ
エンドユーザー 管理者の Web サイト訪問者。公開ウィジェットから FAQ 質問と回答の閲覧を行う

関係者(ステークホルダー): プロダクトオーナー、管理者ユーザー(顧客)、エンドユーザー、開発者、QA、法務 / プライバシー担当、経営層。
