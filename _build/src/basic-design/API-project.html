API-PRJ-001 プロジェクト一覧

基本情報

項目内容
API IDAPI-PRJ-001
API 名プロジェクト一覧
エンドポイント/projects
HTTP メソッドGET
認証Cookie
権限オーナー専有

処理概要

契約配下のプロジェクト一覧を返す。

処理 ID処理内容
P-01認証・認可(オーナー専有)を検証する
P-02契約配下の有効なプロジェクトを取得する(FAQ 件数・許可ドメイン等を付帯)
P-03ページング(カーソル)で整形して返す

API-PRJ-002 プロジェクト新規作成

基本情報

項目内容
API IDAPI-PRJ-002
API 名プロジェクト新規作成
エンドポイント/projects
HTTP メソッドPOST
認証Cookie + CSRF
権限オーナー専有(プロジェクト設定全般を含めオーナーのみ実行可)

処理概要

プロジェクトを作成し公開キーを発行する(オーナーは isOwner で当該プロジェクトの全権を持つ)。

処理 ID処理内容
P-01認証・認可(オーナー専有)を検証する
P-02入力値(プロジェクト名・許可ドメイン)を検証し、名称重複を判定する
P-03プロジェクトを作成しウィジェット公開キーを発行する
P-04作成結果(ID・公開キー)を返す

I/O

テーブルCRUD
M_PROJECTS◯◯——
M_ALLOWED_DOMAINS◯———

リクエストパラメータ

Query Parameters

{}

Path Parameters

{}

Request Body

{
  "name": "新規プロジェクト",
  "description": "説明文",
  "allowedDomains": ["example.com"]
}
項目型説明
namestringプロジェクト名(必須)
descriptionstringプロジェクトの説明文(任意)
allowedDomainsarrayウィジェット許可ドメインのリスト(必須)

オーナー固定化(FR-030a / FR-015e):

  • リクエストに initialAdmins フィールド(selfAsAdmin フラグや adminEmails[])は持たない。オーナーは isOwner bypass で当該プロジェクトの全権を持つため、admin 指定は不要(M_PRJ_USERS への自動付与も行わない)。
  • サーバー側処理: プロジェクト作成は M_PROJECTS 行と公開キーのみを作成し、M_PRJ_USERS への自動 admin 付与は行わない。オーナーは isOwner=true bypass で全権を持つ。critical 通知の宛先解決はオーナー(M_CONTRACT)+ 当該プロジェクトの M_PRJ_USERS.role='admin' かつ valid=1 を両マスタから合算して行う(08_認証・認可設計.html §5.5 正本)。
  • 他者をプロジェクト管理者として招待する操作は、プロジェクト作成後に当該プロジェクトの SCR-009 / SCR-009-001(プロジェクト WS)から POST /projects/{id}/members(API-011)経由で行う。

レスポンス(201)

{
  "id": "01HZ...",
  "widgetKey": "pk_live_abc123..."
}
項目型説明
idstring作成したプロジェクトの ID
widgetKeystring発行したウィジェット公開キー

エラー

HTTP ステータスエラーコード内容

409DUPLICATE_NAMEプロジェクト名重複
403E-AUTHZ-OWNER-ONLYオーナー以外は不可

API-PRJ-003 プロジェクト更新・削除

基本情報

項目内容
API IDAPI-PRJ-003
API 名プロジェクト更新・削除
エンドポイント/projects/{id}
HTTP メソッドPATCH / DELETE
認証Cookie + CSRF(削除は再認証必須、L3 = プロジェクト名タイプ確認 + パスワード再認証)
権限オーナー専有

処理概要

プロジェクトの部分更新 / 論理削除(FR-030b)。

処理 ID処理内容
P-01当該プロジェクトの全メンバー割当を論理削除する(オーナー自身の管理者割当を含む)
P-02他プロジェクトの有効な割当が 0 件かつオーナーでない利用者は、アカウントを論理削除し、全セッション失効と未使用招待トークンの失効を行う
P-03プロジェクトを削除状態(status='deleted'・論理削除)にする
P-04関連データ(許可ドメイン / FAQ / 未解決質問 / 質問ログ)へ論理削除を伝播する。匿名化モードの即時匿名化は行わず、90 日後の物理削除バッチで実施する
P-05監査ログ(プロジェクト論理削除・retention_class='general')を記録する

上記 1-5 はアプリ層のトランザクション内で実装する。論理削除データは updated_at 基準で 90 日経過後に物理削除バッチ(DD13)で物理削除される。物理削除時は ON DELETE CASCADE で関連行が連鎖物理削除される。

API-PRJ-004 ウィジェット鍵ローテーション

基本情報

項目内容
API IDAPI-PRJ-004
API 名ウィジェット鍵ローテーション
エンドポイント/projects/{id}/widget-key/rotate
HTTP メソッドPOST
認証再認証必須
権限オーナー専有

処理概要

公開キーを再発行する(旧キーは 24 時間猶予で失効)。

処理 ID処理内容
P-01認証・認可(オーナー専有)と再認証を検証する
P-02新しいウィジェット公開キーを発行して有効化する
P-03旧キーを 24 時間猶予で失効予約する
P-04新キーと失効予告を返す

I/O

テーブルCRUD
M_PROJECTS——◯—

リクエストパラメータ

Query Parameters

{}

Path Parameters

{ "id": "プロジェクト ID" }
項目型説明
idstring鍵をローテーションする対象プロジェクトの ID(必須・パス内で指定)

Request Body

{}

公開キーは無期限のため有効期限指定パラメータは持たない。

レスポンス(201)

{ "newKey": "pk_live_xyz789...", "deprecationWarning": "..." }
項目型説明
newKeystring再発行したウィジェット公開キー
deprecationWarningstring旧キーの 24 時間猶予失効予告メッセージ

(deprecationWarning は旧キーの 24 時間猶予失効予告)
