# ユースケース再編成 レビュー記録(パイロット)

本書は「ユースケース一覧の再編成」作業の **フェーズ1(レビュー)** と **フェーズ3(改善確認)** の記録である。
改善工程(フェーズ2)の唯一のインプットとして用いる。ポータル配信対象外(`_build/` 配下のツール領域)。

- 対象パイロット: SCR-001 / SCR-006 / SCR-006-001 + システム UC(UC-SYSTEM-001/002/003)。
- 前提: 画面イベント一覧は見直し済みで「1 イベント = 1 ユースケース」の最小単位。**本作業で画面イベント一覧(SCR-*.md §6)は変更しない**。
- 命名: 画面起点 `04_usecases/UC-<SCRID>.md`、UC サブ ID `UC-<SCRID>-EV<nn>`(各 EV と 1:1)。システム起点 `04_usecases/UC-SYSTEM-<nnn>.md`。

> [!NOTE]
> **トリアージ方針** 基本設計レベルの抽象度を超える指摘(DLQ/リトライ実装・冪等キー実装・監視ダッシュボード/Datadog・KV スキーマ等)は、本ユースケース文書のスコープ外として「参考」に降格する。シーケンス図には SQL・クラス/メソッド・ループ・フレームワーク固有処理を書かない(タスクの「記載しない内容」)。
