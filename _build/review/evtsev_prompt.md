# 背景(直近の構成変更)
このリポジトリでは次の変更を行いました。整合性をレビューしてください。
- 「画面イベント設計書(EVT-xxx.md)」と「システムイベント設計書(SEV-xxx.md)」の独立ページ・フォルダ・専用テンプレートを全削除した。
- EVT-xxx / SEV-xxx の識別子は、画面設計 SCR の「§6 画面イベント一覧」/ システム設計 SYS の「§6 システムイベント一覧」に、**非リンクのアンカー**(例 `<span id="EVT-001"></span>EVT-001`)として残した。リンクは除去済み。
- シーケンス設計 SEQ は、対応する画面イベントを**プレーンテキストの ID**(例 `EVT-001`、リンクなし)で参照する。
- API / エラー ERR / 権限 PERM および apis/errors/permissions の集約 index 表からは EVT 参照(列・行・見出し)を除去した。
- 設計ルール正本(CLAUDE.md)とテンプレート、各ナビ index、README をこの方針に合わせて更新した。

# レビュー観点(この変更に限定)
1. **取り残し**: いずれかのドキュメントに、EVT/SEV を「独立したページ/フォルダ/層/テンプレート」として説明・参照する記述が残っていないか(例:「画面イベント設計書」「システムイベント設計書」への導線、`02_screen_events` / `02_system_events` への言及、削除済み専用テンプレートへの参照)。
2. **参照切れ・宙ぶらりん**: EVT-xxx / SEV-xxx を参照しているのに、その ID の定義(SCR §6 / SYS §6 のアンカー)が見当たらない箇所はないか。逆に、文意が不自然に途切れている箇所はないか。
3. **方針の一貫性**: 「EVT-ID は SCR §6 の非リンクアンカー、SEV-ID は SYS §6 の非リンクアンカー、SEQ はプレーン ID で参照」という新方針が、設計ルール正本(CLAUDE.md の読み順・採番表・構成・相互参照表・テンプレ表)と各設計層で**矛盾なく**記述されているか。
4. **採番・系列の整合**: CLAUDE.md の採番ルール表に残した EVT-/SEV- の記述(「§6 にアンカーとして保持」)が、実際の SCR/SYS の §6 と一致しているか。SCR §6 にイベント ID 列(EV-xx)と EVT-ID 列が併存する点に不整合や説明漏れはないか。
5. **集約表の整合**: API↔UC↔TBL 対応表、エラー対応表、権限対応表から EVT 列を除いた結果、列数・見出し・説明文に不整合が残っていないか。
6. **上位→下位の反映漏れ**: この変更に伴い、業務ユースケース UC や要件層に元々 EVT/SEV への言及があった場合、それが適切に処理されているか(なお UC 本文には元来 EVT/SEV を書かない方針)。
