# レビュー観点(用語・ID・コードの統一/重複/矛盾)
1. 用語の不統一: 同一概念が複数語で書かれていないか(例: 「アカウント利用者/ウィジェット利用者」の区別、ロール名「オーナー/管理者/メンバー」、状態名(下書き/公開中/非公開, 確認待ち 等)、「プロジェクト」「契約」等)。CLAUDE.md の用語規約と矛盾する語はないか。
2. ID 体系の不統一: 旧体系の識別子(`E-…`, `API-XXX-NNN`, `MSG-SCR-…`, `IT-`/`EV-`/`E-` 等)が新体系(`ERR-###`/`EVT-###`/`MSG-###` 等)と混在していないか。
3. ステータス/コード値の不統一: 同じ状態・区分が文書ごとに異なる値・名称で書かれていないか。
4. ロール/権限呼称の不統一: 権限(PERM)・API 認可・画面で使うロール呼称が揃っているか。
5. エラーコード(ERR)とメッセージ(MSG)の対応に齟齬がないか(ERR にあるが MSG 無し、その逆)。
6. 廃止済み仕様・古い前提・重複記載・相互に矛盾する記載が残っていないか。
7. 要件定義に存在しない仕様が基本設計に追加されていないか(要件根拠のない設計)。
