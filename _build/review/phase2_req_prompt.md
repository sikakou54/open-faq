# レビュー対象

# レビュー観点(記載粒度・構成 / 要件定義層)
1. 業務要件(BR)・機能要件(FR)・非機能要件(NFR)・業務ユースケース(UC)の粒度が混在していないか(例: BR に FR レベルの細粒度=数値しきい値・手順番号・状態名・画面/UI 文言・FR とほぼ同文の機能記述、が混入。CLAUDE.md「BR と FR の記載粒度(重複を作らない)」参照)。
2. 要件定義としては細かすぎる内容(画面項目・UI 文言・実装手順・アルゴリズム)が混入していないか。
3. BR と FR の間で同一内容の重複記載がないか(BR=業務意図1〜2文、FR=システム機能の細粒度、という役割分担が守られているか)。
4. 要件が「基本設計の内容を前提にした後付け説明」になっていないか(What/Why ではなく How に寄りすぎ)。
5. UC が業務処理粒度(誰が・何の目的で・どの業務を)になっているか。操作粒度すぎ/umbrella すぎの UC がないか。
6. 上流(BR)→下流(FR/UC)へ自然に読め、構成・章立てが一貫しているか。
