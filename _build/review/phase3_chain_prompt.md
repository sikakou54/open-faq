# レビュー対象
要件定義(01_requirements / 02_usecases)から基本設計(03_screens_events / 04_backend / 05_sequences / 06_cross)への落とし込みの連鎖。判定基準は 00_rules。

# レビュー観点(落とし込み漏れ・トレース連鎖)
1. 業務要件(BR)→機能要件(FR)→業務ユースケース(UC)→基本設計 の連鎖に断絶がないか。具体的に、対応する下流が見当たらない上流(例: 基本設計に対応する画面/API/テーブル/システム処理が無い UC・FR)があれば挙げる。
2. 基本設計の各層が必要分そろっているか: 画面(SCR)・画面イベント(EVT)・API・テーブル(TBL)・システム処理(SYS)・システムイベント(SEV)・権限(PERM)・エラー(ERR)・メッセージ(MSG)。要件が示す機能に対し、いずれかの層が欠落している箇所はないか。
3. バッチ/定期処理・外部連携(メール送信・決済・Webhook 受信・全文検索等)について、要件にある機能が基本設計(SYS/SEV/API)で設計されているか。設計が見当たらない外部連携・バッチはないか。
4. 逆方向: 基本設計に存在するが、要件定義(BR/FR/UC)に根拠が見当たらない仕様(要件外の追加仕様)はないか。
5. 各 UC に対応するシーケンス(SEQ)が存在するか。SEQ・PERM・ERR・MSG の結線(対応UC/対応API 等)で空欄・未結線が目立つ箇所はないか。

# 出力形式(厳守)

| No | 重要度 | 起点要件 | 落とし込み先 | 現状 | 抜け漏れ内容 | 理由 | 修正方針 |

重要度=高/中/低。**指摘が無ければ「抜け漏れに関する指摘なし」と明記。** 根拠の無い推測指摘は挙げないこと(実在 ID を伴わない指摘は不可)。
