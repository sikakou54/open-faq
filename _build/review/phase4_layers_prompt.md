# レビュー観点(層間の相互整合)
1. 画面(SCR)とAPI の整合: 画面の入出力項目・画面イベント(EVT)の呼出 API が、対応する API の I/O(リクエスト/レスポンス項目)と整合しているか。画面が必要とするデータを API が返す/受けるか。
2. API とテーブル(TBL)の整合: API が読み書きするデータ項目が、利用テーブルのカラム定義と整合しているか。API が参照するのに利用テーブル節に無い、または逆の不整合はないか。
3. 権限(PERM)と 画面/API の整合: PERM のロール別可否が、各 API の認可欄・画面の表示制御と矛盾していないか。
4. エラー(ERR)と API/メッセージ(MSG)の整合: API のエラー表の各コードが ERR 定義と一致し、ユーザー提示が必要なものは MSG と結線されているか。
5. 同一の機能・画面・API・テーブル・項目について、ドキュメントごとに異なる説明・値になっていないか。
6. 要件(BR/FR)と基本設計の内容に、明確な矛盾(要件と逆の挙動、要件にない制約)がないか。
