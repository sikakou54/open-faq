1. 移管方針

個別チャット機能は MVP から除外し、Post-MVP の開発候補として本書へ移管する。MVP では AI が回答できなかった質問を管理用の問い合わせ ID 付きで T_INQUIRIES に登録し、ウィジェットへは確認済みプロジェクト連絡先メールを表示するところまでを提供する。問い合わせ ID はウィジェットに表示しない。

MVP では次の機能を実装しない。

  • エンドユーザーと管理者ユーザー間のメッセージ送受信
  • 個別チャット部屋・チャット ID の作成
  • チャットへの担当者割当・デフォルト担当者設定
  • エンドユーザー再入室トークン・再入室画面
  • チャット投稿通知
  • チャット部屋数の利用量計測・課金
  • chat_rooms / chat_messages / チャット用 end_users の本番マイグレーション

6. MVP との境界

MVP の未解決時フローは次のとおりとする。

  1. 質問ログを記録する。
  2. T_INQUIRIES に未解決質問を登録し、問い合わせ ID を発行する。
  3. ウィジェットに「回答を見つけられなかった」旨を表示し、管理用の問い合わせ ID は表示しない。
  4. 確認済みプロジェクト連絡先メールがある場合は問い合わせ先として表示する。
  5. FAQ 質問入力は継続可能とし、有人チャットへの切替は行わない。
