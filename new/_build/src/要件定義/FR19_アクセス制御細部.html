FR19: アクセス制御細部

この要件は、ロールとプロジェクト境界に基づくアクセス制御の細部を定め、権限のない画面・データへの到達を確実に拒否することを目的とし、以下について記載します。

  • 複数デバイスからの同時ログイン
  • オーナーの不可制約(退会・停止・削除・降格・譲渡)と、自分自身への操作の禁止
  • メンバーの 1 オーナー固定と、プロジェクト単位の役割選択
  • ダッシュボードの閲覧範囲と、役割変更後のセッション・割当反映
  • 最後の割当解除時のアカウント自動無効化と、権限外 UI の非表示

1. 業務要件

ID 観点 ステークホルダ 業務要件 関連 NFR
BR-162 同時ログイン アカウント利用者 / システム
  • アカウント利用者：同一アカウントで複数デバイスから同時にログインできる
  • システム：契約停止などの契機で当該アカウントの全セッションを強制ログアウトできる
 —
BR-163 オーナーの不可制約 システム オーナーに対する退会・停止・削除・降格・譲渡を一切受け付けない。1 契約 = 1 オーナーのため、オーナー不在(孤立)状態は発生しない。 —
BR-164 自己操作の禁止 アカウント利用者 / システム
  • アカウント利用者：自分自身の利用停止・削除・役割降格はできない
  • システム：これらの操作要求を拒否し、別の管理者またはオーナー経由での実行を促す
 —
BR-165 メンバーのオーナー固定 システム メンバーを 1 オーナー配下に固定する。同一メールアドレスでも別オーナー配下では別アカウントとして扱う。 —
BR-166 役割の付与単位 アカウント利用者 メンバーの役割をプロジェクトごとに「割当しない / メンバー / 管理者」から選んで付与する。契約横断のプリセット役割は持たない。 —
BR-167 ダッシュボード閲覧 アカウント利用者 / システム
  • アカウント利用者：オーナー・メンバーともダッシュボード(自分の通知・お知らせ受信箱)を閲覧できる
  • システム：ダッシュボード以外の画面では、ロールとプロジェクト境界の充足を要求する
 —
BR-168 役割・割当の反映タイミング システム 役割やプロジェクト割当の追加・剥奪では既存セッションを即時失効させず、次回の認可チェック時点で反映する。 —
BR-169 最後の割当解除時の無効化 システム メンバーの最後の有効なプロジェクト割当が解除された時に、対象アカウントを自動で無効化し、全セッションと未使用の招待を失効する。割当のないプロジェクトへの直アクセスは拒否する。 —
BR-170 権限外 UI の扱い アカウント利用者 / システム
  • アカウント利用者：現在の管理範囲と適用される役割をヘッダーおよびアカウントメニューで確認できる
  • システム：権限外の主ナビゲーションと操作は原則表示せず、権限のない画面への直アクセスは説明と戻り先を添えて拒否する
 —

  1. 退会
  2. 停止
  3. 削除
  4. 降格
  5. 譲渡
 BR-163 P0
FR-334 アカウント利用者は自分自身への次の操作を行えないこと。
  1. 自分自身の利用状態変更・削除はできない
  2. 自分が当該プロジェクトで管理者である最後の 1 人の場合、自身を管理者からメンバーへ降格できない(別の管理者またはオーナー経由で行う)
  3. 自分自身のアカウント削除は常に不可とし、オーナーが他人を経由して自身を削除する経路も設けない(他者の削除はオーナー本人の操作のみで行う)
 BR-164 P0
FR-335 メンバーを 1 オーナー配下に固定すること。同一メールアドレスでも別オーナー配下では別アカウントとして扱う BR-165 P1
FR-336 プロジェクト別の役割指定を、当該プロジェクトごとに「割当しない / メンバー / 管理者」の 3 つから選んで行うこと。契約横断のプリセット役割は持たない BR-166 P0
FR-337 ダッシュボード(自分の通知・お知らせ受信箱)を、オーナーおよびメンバーが閲覧できること。それ以外の画面は、それぞれのロールとプロジェクト境界の充足を要求する BR-167 P0
FR-338 プロジェクト別ロールおよびプロジェクト割当の追加・剥奪は、アクティブセッションを即時失効させず、次回の認可チェック時点で反映することで足りること。反映上限時間とキャッシュ方式は基本設計を正本とする BR-168 P1
FR-339 メンバーの最後の有効なプロジェクト割当を解除した場合の挙動を定めること。
  1. 対象アカウントを自動で無効化する
  2. 全セッションと未使用の招待を失効する
  3. 割当のないプロジェクトへの直アクセスは拒否する
  4. 契約横断のメンバー管理画面は提供しない
 BR-169 P0
FR-340 現在の管理範囲と適用ロールを、ヘッダーの「管理範囲を切り替え」およびアカウントメニューで確認できること。独立したロールバッジを全画面へ重複表示しない。複合ロールの場合は、現在のプロジェクトで適用されるロールを表示する BR-170 P0
FR-341 権限外の主ナビゲーションと操作の表示を制御すること。
  1. 権限外の主ナビゲーションと操作は原則として表示しない
  2. 共同作業上、存在を知らせる価値がある項目は読み取り専用で表示し、必要権限と依頼先を説明する
  3. 権限のない画面への直アクセスを拒否する際は、同等の説明と戻り先を提示する
 BR-170 P0
