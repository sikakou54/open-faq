FR06: 未解決質問登録

この要件は、FAQ で回答できなかった質問やウィジェット利用者が「解決しなかった」とした質問を未解決質問として記録し、アカウント利用者が対応状況を追跡できるようにすることを目的とし、以下について記載します。

  • 未解決質問が登録される 2 つの契機(回答不能・利用者の未解決選択)
  • 未解決質問に記録する情報(問い合わせ ID を含む)
  • 対応状況の 2 区分(対応中 / 対応済み)と手動切替
  • FAQ 操作と状況更新を連動させない非連動ルール

1. 業務要件

ID 観点 ステークホルダ 業務要件 関連 NFR
BR-071 未解決質問の登録契機 システム / ウィジェット利用者
  • システム：登録済み FAQ で回答できなかった質問を、未解決質問として登録する
  • ウィジェット利用者：回答に対して「解決しなかった」を選択した時に、その質問を未解決質問として登録させる
 —
BR-072 記録する情報 システム 未解決質問の登録時に、質問・未解決理由・発生日時・関連プロジェクト・関連質問ログを記録し、追跡用の問い合わせ ID を付与する。 —
BR-073 対応状況の管理 システム / アカウント利用者
  • システム：未解決質問の対応状況を「対応中 / 対応済み」の 2 区分で保持し、登録時は「対応中」とする
  • アカウント利用者：詳細画面の手動操作で「対応中 ↔︎ 対応済み」を切り替える(双方向・再オープン回数の制限なし)
 —
BR-074 FAQ 操作との非連動 システム FAQ の下書き保存・FAQ の公開成功が起きても、未解決質問の対応状況を変更しない。状況の変更は詳細画面からの手動操作のみとする。 —
BR-075 対応状況の確認 アカウント利用者 未解決質問の現在の対応状況(対応中 / 対応済み)を、一覧画面と詳細画面で確認できる。 —
