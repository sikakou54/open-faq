FR11: 通知

この要件は、請求確定・運営お知らせ・役割変更などの契機で、対象のアカウント利用者へメールと受信箱でお知らせを確実に届け、不達や悪用を防ぐことを目的とし、以下について記載します。

  • 通知契機(請求確定・運営お知らせ・運用イベント・役割割当変更)と配信先
  • 送信失敗時の再送と再送回数上限、配信状態の可視化
  • バウンス・苦情の検知と全契約横断での送信停止
  • 送信レート・バウンス率・苦情率の監視と、認証通知のオプトアウト不可

版数 v1.2更新 2026-06-16機能グループ 通知優先度 Must承認済

1. 業務要件

ID 観点 ステークホルダ 業務要件 関連 NFR
BR-103 通知の到達 システム 請求確定・運営お知らせなどの契機が発生した時に、対象のアカウント利用者へメールまたはお知らせ受信箱で通知を届ける。 NFR-501
BR-104 請求確定通知 システム 月次の請求が確定した時に、オーナーへ請求対象期間・請求金額・内訳・請求書 / 明細の確認導線を含むメールを送り、お知らせ受信箱にも「請求確定」のお知らせを生成する。同一契約・同一請求月で重複して請求しない。 —
BR-105 お知らせ配信 システム / アカウント利用者
  • システム：宛先範囲・件名・本文・送信予定日時を指定したお知らせ(メンテナンス予告・機能追加・規約改定・価格改定など)が配信された時に、対象のアカウント利用者へ届ける
  • アカウント利用者：種別ごとにオプトアウトの可否を設定する。ただし規約改定・重要セキュリティ通知は強制送信としオプトアウトできない
 —
BR-106 役割変更通知 システム メンバーのプロジェクト別の役割割当が追加・変更・剥奪された時に、当該メンバーへお知らせ受信箱とメールで通知する。 —
BR-107 システム通知の自動生成 システム 運用イベント(利用上限への接近・到達、通知失敗の急増、サスペンション、復元、規約改定、価格改定など)が発生した時に、「システム通知」のお知らせを自動生成する。 —
BR-108 送信失敗時の再送 システム 通知の送信に失敗した時に、上限を設けたうえで再送する。 NFR-501
BR-109 バウンス・苦情対応 システム バウンスや無効アドレス・苦情を検知した時に、当該宛先への送信を停止し、管理者へ通知する。送信停止リストは全契約横断で共有する。 —
BR-110 送信品質の監視 システム 契約 / プロジェクト単位で送信レート・バウンス率・苦情率を監視し、しきい値を超過した時に送信を抑制する。 —
BR-111 配信状態の可視化 アカウント利用者 通知の配信状態(送信待ち / 送信済み / 配信済み / 失敗 / バウンス / 苦情 / 停止)を確認できる。 —
BR-112 認証通知の強制送信 アカウント利用者 アカウント認証関連の通知(パスワード再設定等)はオプトアウトできない。 —
BR-113 なりすまし・スパム対策 システム ウィジェット利用者が入力した文字列を、メール件名・送信元情報など外部に露出する箇所へそのまま使わない。 —
