画面設計書

メインシステムの全 28 画面 + 公開ウィジェットを、4 ワークスペース別に整理した独立設計書です。 各画面名のリンクから、画面レイアウト(モックアップ)・画面項目定義・入出力・画面イベントを載せた個別ページを開けます(旧 SCR 設計書の体裁)。§4 では画面横断で「アクション → API → DB(CRUD)」を一覧できます。

版数 v2.1更新 2026-06-19画面数 28 + ウィジェット独立設計書

★
各画面のレイアウトは個別ページに画面名をクリックすると、その画面の 画面レイアウト・画面遷移図・画面項目定義・入出力一覧・画面イベント一覧を旧設計書と同じ構成で確認できます。

0.画面設計の方針#

SPA

SPA + スコープ別URL

URL を /auth //owner //projects/:id //account の 4 系列に分離。

2

2 ワークスペース

契約(オーナー専有)とプロジェクト(共同作業)+ 共通領域。サイドメニューは領域別に切替。

A11y

状態の3点伝達

状態は「色 + アイコン + テキスト」の 3 点セットで伝達(WCAG 2.1 AA)。

3.共通 UI 部品#

全画面が共通で使う主要部品(抜粋)。各画面の項目表はこれらを参照します。

部品用途・主な仕様
WorkspaceSwitcher契約 / プロジェクトの管理範囲切替。適用ロールを補助表示
StatusBadge状態を「色 + アイコン + テキスト」で表示(FR-390)
BulkActionBar1 件以上選択時に下部固定。FAQ は 1 操作上限 50 件
ConfirmDialog / ReauthConfirmModal重要操作の L2 タイプ確認 / L3 再認証(FR-005)
PaymentMethodBanner / UsageBar無料枠 80/100/125% の警告と支払方法登録誘導(FR-136)
EmptyState / LoadingSkeleton / Toast0 件・読込中・処理結果の標準表現

4.画面別 アクション → API → DB#

各画面のユーザー操作(画面イベント)が呼ぶ API と、その API が読み書きするテーブル(CRUD)です。各画面の詳細・レイアウトは見出しの「レイアウト →」から開けます。— は画面遷移のみ。
