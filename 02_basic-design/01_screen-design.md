# 画面設計書

**メインシステムの全 28 画面 + 公開ウィジェットを、4 ワークスペース別に整理した独立設計書です。** 各画面名のリンクから、**画面レイアウト(モックアップ)・画面項目定義・入出力・画面イベント**を載せた個別ページを開けます(旧 SCR 設計書の体裁)。§4 では画面横断で「アクション → API → DB(CRUD)」を一覧できます。

*版数 v2.1 ・ 更新 2026-06-19 ・ 画面数 28 + ウィジェット ・ 独立設計書*

> [!NOTE]
> **各画面のレイアウトは個別ページに** 画面名をクリックすると、その画面の **画面レイアウト**・画面遷移図・画面項目定義・入出力一覧・画面イベント一覧を旧設計書と同じ構成で確認できます。

## <span id="ui"></span>3.共通 UI 部品

全画面が共通で使う主要部品(抜粋)。各画面の項目表はこれらを参照します。

| 部品 | 用途・主な仕様 |
|----|----|
| `WorkspaceSwitcher` | 契約 / プロジェクトの管理範囲切替。適用ロールを補助表示 |
| `StatusBadge` | 状態を「色 + アイコン + テキスト」で表示(FR-390) |
| `BulkActionBar` | 1 件以上選択時に下部固定。FAQ は 1 操作上限 50 件 |
| `ConfirmDialog / ReauthConfirmModal` | 重要操作の L2 タイプ確認 / L3 再認証(FR-005) |
| `PaymentMethodBanner / UsageBar` | 無料枠 80/100/125% の警告と支払方法登録誘導(FR-136) |
| `EmptyState / LoadingSkeleton / Toast` | 0 件・読込中・処理結果の標準表現 |

## <span id="flow"></span>4.画面別 アクション → API → DB

各画面のユーザー操作(画面イベント)が呼ぶ API と、その API が読み書きするテーブル(CRUD)です。各画面の詳細・レイアウトは見出しの「レイアウト →」から開けます。`—` は画面遷移のみ。
