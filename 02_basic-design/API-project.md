# プロジェクト管理 API

> **このページは、プロジェクトの一覧・作成・更新・削除とウィジェット鍵ローテーションの API 契約を定義します。**
>
> **このページの API**
>
> - API-PRJ-001 プロジェクト一覧
> - API-PRJ-002 プロジェクト新規作成
> - API-PRJ-003 プロジェクト更新・削除
> - API-PRJ-004 ウィジェット鍵ローテーション

*版数 v1.6 ・ 更新 2026-06-17 ・ 承認済*

## <span id="API-PRJ-001"></span>API-PRJ-001 プロジェクト一覧

<span id="531-get-M_PROJECTS"></span>

### 基本情報

| 項目           | 内容             |
|----------------|------------------|
| API ID         | API-PRJ-001      |
| API 名         | プロジェクト一覧 |
| エンドポイント | `/projects`      |
| HTTP メソッド  | GET              |
| 認証           | Cookie           |
| 権限           | **オーナー専有** |

### 処理概要

契約配下のプロジェクト一覧を返す。

| 処理 ID | 処理内容 |
|----|----|
| P-01 | 認証・認可(オーナー専有)を検証する |
| P-02 | 契約配下の有効なプロジェクトを取得する(FAQ 件数・許可ドメイン等を付帯) |
| P-03 | ページング(カーソル)で整形して返す |

| テーブル            | C   | R   | U   | D   |
|---------------------|-----|-----|-----|-----|
| `M_PROJECTS`        | —   | ◯   | —   | —   |
| `M_FAQS`            | —   | ◯   | —   | —   |
| `M_ALLOWED_DOMAINS` | —   | ◯   | —   | —   |

### リクエストパラメータ

| 項目     | 型      | 説明                                             |
|----------|---------|--------------------------------------------------|
| `cursor` | string  | 次ページ取得用カーソル(任意・共通ページング仕様) |
| `limit`  | integer | 1 ページの取得件数(任意・共通ページング仕様)     |

```json

```

#### Request Body


### レスポンス(200)

```json
{
  "items": [
    {
      "id": "01HZ...",
      "name": "...",
      "status": "active|deleted",
      "createdAt": "...",
      "faqCount": 42,
      "widgetKey": "pk_live_...",
      "allowedDomains": ["example.com", "*.example.com"],
      "contactEmail": "contact@...",
      "contactEmailVerifiedAt": "..." | null
    }
  ],
  "nextCursor": null
}
```

| 項目 | 型 | 説明 |
|----|----|----|
| `items` | array\<object\> | 契約配下のプロジェクト一覧 |
| `　└ id` | string | プロジェクト ID |
| `　└ name` | string | プロジェクト名 |
| `　└ status` | enum | プロジェクト状態(`active` / `deleted`) |
| `　└ createdAt` | string (ISO 8601) | 作成日時 |
| `　└ faqCount` | integer | 当該プロジェクトの FAQ 件数 |
| `　└ widgetKey` | string | ウィジェット公開キー |
| `　└ allowedDomains` | array | 許可ドメインのリスト |
| `　└ contactEmail` | string | プロジェクト連絡先メールアドレス |
| `　└ contactEmailVerifiedAt` | string (ISO 8601) | 連絡先メール確認日時(未確認は `null`) |
| `nextCursor` | string | 次ページ取得用カーソル(末尾は `null`) |

## <span id="API-PRJ-002"></span>API-PRJ-002 プロジェクト新規作成

<span id="532-post-M_PROJECTS"></span>

### 基本情報

| 項目 | 内容 |
|----|----|
| API ID | API-PRJ-002 |
| API 名 | プロジェクト新規作成 |
| エンドポイント | `/projects` |
| HTTP メソッド | POST |
| 認証 | Cookie + CSRF |
| 権限 | **オーナー専有**(プロジェクト設定全般を含めオーナーのみ実行可) |

### 処理概要

プロジェクトを作成し公開キーを発行する(オーナーは isOwner で当該プロジェクトの全権を持つ)。

| 処理 ID | 処理内容                                                         |
|---------|------------------------------------------------------------------|
| P-01    | 認証・認可(オーナー専有)を検証する                               |
| P-02    | 入力値(プロジェクト名・許可ドメイン)を検証し、名称重複を判定する |
| P-03    | プロジェクトを作成しウィジェット公開キーを発行する               |
| P-04    | 作成時にオーナーを当該プロジェクトのメンバーとして `M_PRJ_USERS`(`valid=1`)に登録する(一覧表示・担当割当・通知宛先の網羅用。認可権威は引き続き isOwner) |
| P-05    | 作成結果(ID・公開キー)を返す                                     |

### I/O

| テーブル            | C   | R   | U   | D   |
|---------------------|-----|-----|-----|-----|
| `M_PROJECTS`        | ◯   | ◯   | —   | —   |
| `M_ALLOWED_DOMAINS` | ◯   | —   | —   | —   |
| `M_PRJ_USERS`       | ◯   | —   | —   | —   |

### リクエストパラメータ

#### Query Parameters

```json
{}
```

#### Path Parameters

```json
{}
```

#### Request Body

```json
{
  "name": "新規プロジェクト",
  "description": "説明文",
  "allowedDomains": ["example.com"]
}
```

| 項目             | 型     | 説明                                   |
|------------------|--------|----------------------------------------|
| `name`           | string | プロジェクト名(必須)                   |
| `description`    | string | プロジェクトの説明文(任意)             |
| `allowedDomains` | array  | ウィジェット許可ドメインのリスト(必須) |

オーナー固定化(FR-030a / FR-015e):

- リクエストにメンバー指定フィールドは持たない。オーナーは `isOwner` bypass で当該プロジェクトの全権を持つため、メンバー指定は不要。
- サーバー側処理: プロジェクト作成は `M_PROJECTS` 行と公開キーを作成し、加えてオーナーを当該プロジェクトのメンバーとして `M_PRJ_USERS`(`valid=1`)へ自動登録する(一覧表示・担当割当・通知宛先の網羅用。専有判定には用いず、認可権威は引き続き `isOwner=true` bypass)。critical 通知の宛先解決はオーナー(`M_CONTRACT`)+ 当該スコープの有効メンバー(`M_PRJ_USERS.valid=1`)を両マスタから合算し、正規化メールで重複排除して行う(08_認証・認可設計.html §5.5 正本)。
- 他のメンバーをプロジェクトへ招待する操作は、プロジェクト作成後に当該プロジェクトの SCR-009 / SCR-009-001(プロジェクト WS)から `POST /projects/{id}/members`(API-011)経由で行う。

### レスポンス(201)

```json
{
  "id": "01HZ...",
  "widgetKey": "pk_live_abc123..."
}
```

| 項目        | 型     | 説明                         |
|-------------|--------|------------------------------|
| `id`        | string | 作成したプロジェクトの ID    |
| `widgetKey` | string | 発行したウィジェット公開キー |

### エラー

| HTTP ステータス | エラーコード         | 内容                               |
|-----------------|----------------------|------------------------------------|
| 400             | `VALIDATION_ERROR`   | `name` / `allowedDomains` 必須違反 |
| 409             | `DUPLICATE_NAME`     | プロジェクト名重複                 |
| 403             | `E-AUTHZ-OWNER-ONLY` | オーナー以外は不可                 |

## <span id="API-PRJ-003"></span>API-PRJ-003 プロジェクト更新・削除

<span id="533-patch-projectsid--delete-projectsid"></span>

### 基本情報

| 項目 | 内容 |
|----|----|
| API ID | API-PRJ-003 |
| API 名 | プロジェクト更新・削除 |
| エンドポイント | `/projects/{id}` |
| HTTP メソッド | PATCH / DELETE |
| 認証 | Cookie + CSRF(削除は再認証必須、L3 = プロジェクト名タイプ確認 + パスワード再認証) |
| 権限 | **オーナー専有** |

### 処理概要

プロジェクトの部分更新 / 論理削除(FR-030b)。

| 処理 ID | 処理内容 |
|----|----|
| P-01 | 当該プロジェクトの全メンバー割当を論理削除する(オーナー自身の自動メンバー割当を含む) |
| P-02 | 他プロジェクトの有効な割当が 0 件かつオーナーでない利用者は、アカウントを論理削除し、全セッション失効と未使用招待トークンの失効を行う |
| P-03 | プロジェクトを削除状態(`status='deleted'`・論理削除)にする |
| P-04 | 関連データ(許可ドメイン / FAQ / 未解決質問 / 質問ログ)へ論理削除を伝播する。匿名化モードの即時匿名化は行わず、90 日後の物理削除バッチで実施する |
| P-05 | 監査ログ(プロジェクト論理削除・`retention_class='general'`)を記録する |

上記 1-5 はアプリ層のトランザクション内で実装する。論理削除データは `updated_at` 基準で 90 日経過後に物理削除バッチ(DD13)で物理削除される。物理削除時は `ON DELETE CASCADE` で関連行が連鎖物理削除される。

| `M_PROJECTS`        | —   | ◯   | ◯   | —   |
| `M_ALLOWED_DOMAINS` | —   | —   | ◯   | —   |
| `M_PRJ_USERS`       | —   | —   | ◯   | —   |
| `M_PRJ_USERS`       | —   | —   | ◯   | —   |
| `T_SESSIONS`        | —   | —   | ◯   | —   |
| `T_ACCESS_TOKENS`   | —   | —   | ◯   | —   |
| `M_FAQS`            | —   | —   | ◯   | —   |
| `H_QUESTION_LOGS`   | —   | —   | ◯   | —   |
| `T_INQUIRIES`       | —   | —   | ◯   | —   |
| `H_AUDIT_LOGS`      | ◯   | —   | —   | —   |

### リクエストパラメータ

#### Query Parameters

```json
{}
```

#### Path Parameters

```json
{ "id": "プロジェクト ID" }
```

| 項目 | 型     | 説明                                                   |
|------|--------|--------------------------------------------------------|
| `id` | string | 更新 / 削除の対象プロジェクトの ID(必須・パス内で指定) |

#### Request Body

```json
{
  "name": "...",
  "allowedDomains": ["..."],
  "contactEmail": "..."
}
```

| 項目 | 型 | 説明 |
|----|----|----|
| `name` | string | 更新後のプロジェクト名(PATCH 時・任意の部分更新項目) |
| `allowedDomains` | array | 更新後の許可ドメインのリスト(PATCH 時・任意の部分更新項目) |
| `contactEmail` | string | 更新後の連絡先メールアドレス(PATCH 時・任意の部分更新項目) |

上記 Request Body は PATCH(部分更新)時のもの。DELETE 時はボディなし。`DELETE /projects/{id}` 実行時の挙動(FR-030b、論理削除)。

### レスポンス(200)

PATCH は更新後のプロジェクト、DELETE は `{ "ok": true, "status": "deleted" }`。

**PATCH(更新後のプロジェクト)**

| 項目             | 型     | 説明                         |
|------------------|--------|------------------------------|
| `id`             | string | 更新したプロジェクトの ID    |
| `name`           | string | 更新後のプロジェクト名       |
| `allowedDomains` | array  | 更新後の許可ドメインのリスト |
| `contactEmail`   | string | 更新後の連絡先メールアドレス |

**DELETE(論理削除結果)**

| 項目     | 型      | 説明                                |
|----------|---------|-------------------------------------|
| `ok`     | boolean | 論理削除の成否                      |
| `status` | enum    | 削除後のプロジェクト状態(`deleted`) |

### エラー

| HTTP ステータス | エラーコード         | 内容                 |
|-----------------|----------------------|----------------------|
| 404             | `NOT_FOUND`          | オーナー境界違反偽装 |
| 403             | `E-AUTHZ-OWNER-ONLY` | オーナー以外は不可   |

## <span id="API-PRJ-004"></span>API-PRJ-004 ウィジェット鍵ローテーション

<span id="534-post-projectsidwidget-keyrotate"></span>

### 基本情報

| 項目           | 内容                               |
|----------------|------------------------------------|
| API ID         | API-PRJ-004                        |
| API 名         | ウィジェット鍵ローテーション       |
| エンドポイント | `/projects/{id}/widget-key/rotate` |
| HTTP メソッド  | POST                               |
| 認証           | 再認証必須                         |
| 権限           | オーナー専有                       |

### 処理概要

公開キーを再発行する(旧キーは 24 時間猶予で失効)。

| 処理 ID | 処理内容                                       |
|---------|------------------------------------------------|
| P-01    | 認証・認可(オーナー専有)と再認証を検証する     |
| P-02    | 新しいウィジェット公開キーを発行して有効化する |
| P-03    | 旧キーを 24 時間猶予で失効予約する             |
| P-04    | 新キーと失効予告を返す                         |

### I/O

| テーブル     | C   | R   | U   | D   |
|--------------|-----|-----|-----|-----|
| `M_PROJECTS` | —   | —   | ◯   | —   |

### リクエストパラメータ

#### Query Parameters

```json
{}
```

#### Path Parameters

```json
{ "id": "プロジェクト ID" }
```

| 項目 | 型 | 説明 |
|----|----|----|
| `id` | string | 鍵をローテーションする対象プロジェクトの ID(必須・パス内で指定) |

#### Request Body

```json
{}
```

公開キーは無期限のため有効期限指定パラメータは持たない。

### レスポンス(201)

```json
{ "newKey": "pk_live_xyz789...", "deprecationWarning": "..." }
```

| 項目                 | 型     | 説明                                   |
|----------------------|--------|----------------------------------------|
| `newKey`             | string | 再発行したウィジェット公開キー         |
| `deprecationWarning` | string | 旧キーの 24 時間猶予失効予告メッセージ |

(`deprecationWarning` は旧キーの 24 時間猶予失効予告)

### エラー

想定エラーは 05_エラー設計.html を正本とする。
