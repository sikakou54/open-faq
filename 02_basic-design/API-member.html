API-MBR-001 メンバー一覧

基本情報

項目内容
API IDAPI-MBR-001
API 名メンバー一覧
エンドポイント/projects/{id}/members
HTTP メソッドGET
認証Cookie
権限オーナー / 当該プロジェクト管理者

処理概要

指定プロジェクトのメンバー割当一覧を返す。

処理 ID処理内容
P-01認証・認可(オーナー / 当該プロジェクト管理者)を検証する
P-02指定プロジェクトの有効な割当を取得する
P-03ページング(カーソル)で整形して返す

API-MBR-002 メンバー招待

基本情報

項目内容
API IDAPI-MBR-002
API 名プロジェクト単発招待(SCR-009-001 招待モード正本)
エンドポイント/projects/{id}/members
HTTP メソッドPOST
認証Cookie + CSRF + 再認証必須
権限オーナー / 該当プロジェクトの admin ロール保持メンバー

既存のメンバーアカウントの場合は M_PRJ_USERS 行のみ追加(指定ロール)。新規メールアドレスの場合は M_PRJ_USERS を name=NULL / status='pending_activation' で作成 + M_PRJ_USERS を valid=0 で同時 INSERT + 招待メール送信(7 日有効、T_ACCESS_TOKENS.purpose='activation'、meta={"invitedProjectId": "...", "invitedRole": "..."} を JSON 保持)。氏名(displayName)は受け付けない(招待された本人が SCR-018 メンバーアカウント有効化ページで入力する。FR-016d / FR-016e)。

処理概要

該当プロジェクト 1 件に対する単発招待を行う。

処理 ID処理内容
P-01認証・認可(オーナー / 当該プロジェクト管理者)と再認証を検証する
P-02入力値(メール形式・ロール)を検証し、当該プロジェクトへの既存割当(重複)を判定する
P-03既存アカウントは指定ロールの割当を追加し、新規メールは未有効化アカウントと無効状態の割当を作成する
P-04新規メールには有効化トークンを発行し招待メールを送信する
P-05招待結果を返す

API-MBR-003 メンバーロール変更

基本情報

項目内容
API IDAPI-MBR-003
API 名プロジェクト単位のロール変更(SCR-009-001 編集モード正本)
エンドポイント/projects/{id}/members/{userId}
HTTP メソッドPATCH
認証Cookie + CSRF + 再認証必須
権限オーナー / 該当プロジェクトの admin ロール保持メンバー

処理概要

当該プロジェクトの M_PRJ_USERS.role を上書きする(他プロジェクトに影響なし)。

処理 ID処理内容
P-01認証・認可(オーナー / 当該プロジェクト管理者)と再認証を検証する
P-02対象割当の存在と入力ロール値を検証する
P-03最後の管理者の降格・自身の不正降格を判定し該当時は拒否する
P-04当該プロジェクトのロールを上書きする(他プロジェクトに影響しない)
P-05更新結果を返す

リクエストパラメータ

Query Parameters

{}

Path Parameters

{
  "id": "プロジェクト ID",
  "userId": "対象ユーザー ID"
}
項目型説明
idstring対象プロジェクトの ID(必須・パス内で指定)
userIdstringロールを変更する対象利用者の ID(必須・パス内で指定)

Request Body

{
  "role": "admin" | "member"
}
項目型説明
roleenum変更後のロール(必須・admin / member)
  • 同時に表示名(displayName)を更新する場合は別途 PATCH /members/{id} を呼び出す(プロジェクト範囲外の属性のため)
  • 「該当プロジェクトの最後の admin を member へ降格」する結果になる場合は 403 LAST_ADMIN_PROTECTED(E-AUTHZ-LAST-ADMIN-PROTECTED)で拒否
  • 自分自身が当該プロジェクトの最後の admin で、自身を member へ降格する場合は 403 SELF_MUTATION_FORBIDDEN
  • ロール変更後も当該プロジェクトに M_PRJ_USERS 行(割当)が維持される限り、normal / low 通知の配信対象に含まれる(配信先はトグル ON 時にプロジェクトの全メンバー)

レスポンス(200)

{ "userId": "...", "projectId": "...", "role": "admin|member", "updatedAt": "..." }
項目型説明
userIdstringロールを変更した対象利用者の ID
projectIdstring対象プロジェクトの ID
roleenum変更後のロール(admin / member)
updatedAtstring (ISO 8601)ロール更新日時

エラー

HTTP ステータスエラーコード内容
404NOT_FOUND対象が存在しない

403SELF_MUTATION_FORBIDDEN自身を不正に降格しようとした
400VALIDATION_ERROR入力値の検証エラー

API-MBR-004 プロジェクト割当解除

基本情報

項目内容
API IDAPI-MBR-004
API 名プロジェクト割当解除
エンドポイント/projects/{id}/members/{userId}
HTTP メソッドDELETE
認証Cookie + CSRF + 再認証必須
権限オーナー / 該当プロジェクトの admin ロール保持メンバー

対象が自分自身またはオーナーの場合は拒否する。

  • 対象ユーザーに他プロジェクトの有効割当が残る場合: M_PRJ_USERS、他プロジェクトの割当(M_PRJ_USERS)、セッション、招待トークンは維持する。
  • 他プロジェクトの有効割当が 0 件になる場合: 同一トランザクションで M_PRJ_USERS.valid=0、全セッション失効、未使用招待トークン失効を実行する。

処理概要

対象プロジェクトの M_PRJ_USERS.valid=0 を更新する。

処理 ID処理内容
P-01認証・認可(オーナー / 当該プロジェクト管理者)と再認証を検証する
P-02対象が自分自身またはオーナーの場合は拒否する
P-03当該プロジェクトの割当を論理削除する
P-04他プロジェクトの有効割当が 0 件になる場合はアカウントを論理削除し、全セッション失効と未使用招待トークン失効を行う
P-05解除結果(アカウント無効化の有無)を返す

API-MBR-005 招待メール再送

基本情報

項目内容
API IDAPI-MBR-005
API 名招待メール再送
エンドポイント/members/{id}/resend-invitation
HTTP メソッドPOST
認証Cookie + CSRF + 再認証必須
権限オーナー / 該当プロジェクトの admin 保持メンバー

旧 T_ACCESS_TOKENS.purpose='activation' を失効させ新規トークン(7 日有効、meta を同一の invitedProjectId / invitedRole で再発行)を発行。

処理概要

status='pending_activation' のメンバーへ招待メールを再送する。

処理 ID処理内容
P-01認証・認可(オーナー / 当該プロジェクト管理者)と再認証を検証する
P-02対象が未有効化(pending_activation)のメンバーであることを確認する(オーナー / 自身は対象外)
P-03旧有効化トークンを失効させ、同一の招待先情報で新規トークンを再発行する
P-04招待メールを再送し再発行結果を返す
