FR03: プロジェクト管理

この要件は、オーナー配下で FAQ・ウィジェット・ログを分けて運用する単位「プロジェクト」を、作成から削除まで安全に管理できるようにすることを目的とし、以下について記載します。

  • プロジェクトの作成・編集・削除と、作成時の管理者の決まり方
  • プロジェクト削除時のメンバー割当解除と関連データの取扱い
  • プロジェクト単位のウィジェット設定と公開キーの発行・再発行
  • 許可ドメインの設定と、確認済みプロジェクト連絡先メールの表示
  • プロジェクト数の監視と急増検知のアラート

版数 v1.3更新 2026-06-16機能グループ プロジェクト管理優先度 Must承認済

1. 業務要件

ID 観点 ステークホルダ 業務要件 関連 NFR
BR-045 プロジェクトのライフサイクル オーナー FAQ・ウィジェット・ログを分けて運用する単位「プロジェクト」を、必要なときに作成・編集・削除する。削除は、プロジェクト名の入力確認と本人確認(再認証)を必須とする重要操作として扱う。 —
BR-046 作成時の管理者 オーナー / システム
  • オーナー：作成したプロジェクトに対して常に全権を持つ。プロジェクト作成時に管理者を別途指定せず、他者をプロジェクト管理者に加える場合は当該プロジェクトのメンバー管理からプロジェクトユーザーを管理者として割り当てる
  • システム：プロジェクト作成時に作成者であるオーナーへ別途の管理者割当は行わない。オーナーは契約境界に基づく全権でプロジェクトを操作できる
 —
BR-047 削除時のメンバー割当・データ消去 システム プロジェクトを削除した時に、当該プロジェクトのメンバー割当を解除する。割当解除の結果、他に有効な割当を持たないメンバー(オーナーを除く)はアカウントを利用できない状態にし、他プロジェクトに有効な割当を持つメンバーは当該プロジェクトの割当のみ解除する。オーナーのアカウントは対象外とする。利用停止・解除されたデータは 90 日経過後に完全に消去する。 —
BR-048 ウィジェット設定と公開キー アカウント利用者 / システム
  • アカウント利用者：当該プロジェクトの管理者以上はウィジェット設定の編集と公開キーの再発行を行え、メンバーは閲覧と埋め込みコードのコピーのみ行える
  • システム：公開キーをプロジェクト単位で発行・ローテーションし(契約共通にしない)、再発行時は本人確認(再認証)と既存埋め込みコードが直ちに無効化される旨の確認を求める
 NFR-312
BR-049 プロジェクト単位のデータ分離 システム FAQ・質問ログ・未解決質問を、プロジェクトごとに分けて管理する。 NFR-302
BR-050 許可ドメイン アカウント利用者 ウィジェットの設置を許可する Web サイトのドメインを、プロジェクトごとに複数設定する。 NFR-312
BR-051 プロジェクト連絡先メール アカウント利用者 / システム
  • アカウント利用者：プロジェクトごとに連絡先メールアドレスを 1 件設定する
  • システム：確認メール(有効期限 24 時間)による本人確認を経て有効化し、確認完了前のアドレスは表示に用いない。ウィジェット利用者向けメールは常に共通ドメインの no-reply 送信で固定し、本アドレスは返信先に用いない
 —
BR-052 連絡先の公開表示 システム 公開ウィジェットの未解決時・質問受付制限時に、確認完了済みのプロジェクト連絡先メールをお問い合わせ先として表示する。未設定または未確認の場合は表示しない。 —
BR-053 削除時の関連データの確認 アカウント利用者 プロジェクト削除時に、関連データ(FAQ・質問ログ・未解決質問)の取扱いを確認・選択する。 —
BR-054 プロジェクト数の監視 システム プロジェクト数に固定上限は設けず、利用状況を監視して急激な増加を検知した時に、オーナーへ通知する。 —

2. 機能要件

ID 要件 関連業務要件 優先度
FR-030 オーナーはプロジェクトを作成・編集・削除できること。削除は、プロジェクト名の入力確認と本人確認(再認証)を必須とする重要操作として扱う BR-045 P0
FR-030a プロジェクト作成時には、作成者であるオーナー自身が当該プロジェクトに対して全権を持つこと。オーナーへ管理者割当を別途付与する処理は行わない(オーナーは契約境界に基づく全権でプロジェクトを操作する)。
  1. プロジェクト作成時に管理者を別途指定する操作は設けない
  2. 一般メンバーの追加や他者をプロジェクト管理者として加える操作は、当該プロジェクトのメンバー管理から行い、プロジェクト管理者はプロジェクトユーザーへの管理者割当で付与する
 BR-046 P0
FR-030b プロジェクト削除時のメンバー割当と関連データの取扱いを次のとおり定めること。
  1. 当該プロジェクトのメンバー割当を解除する
  2. 割当解除の結果、他に有効なプロジェクト割当を持たないメンバー(オーナーを除く)は、アカウントを利用できない状態にする
  3. 他プロジェクトに有効な割当を持つメンバーのアカウントは維持し、当該プロジェクトの割当のみを解除する
  4. オーナーのアカウントは本処理の対象外とする
  5. 利用停止・解除されたデータは一定期間(90 日)経過後に完全に消去する
 BR-047 P0
FR-031 プロジェクトごとにウィジェット設定(公開キー・埋め込みコード・見た目・プレビュー)を管理できること。
  1. 公開キーはプロジェクト単位で発行・ローテーションし、契約共通ではない
  2. 公開キーの再発行は当該プロジェクトの管理者以上に許可し、本人確認(再認証)と「既存の埋め込みコードが直ちに無効化される」旨の確認を必須とする
  3. メンバーは閲覧と埋め込みコードのコピーのみ行える
  4. ウィジェット設定はプロジェクト単位の作業領域に配置する
 BR-048 P0
FR-032 プロジェクトごとに FAQ、質問ログ、未解決質問を分けて管理できること BR-049 P0
FR-033 許可する Web サイトのドメインを複数設定できること BR-050 P0
FR-033a プロジェクトごとに連絡先メールアドレスを 1 件設定でき、確認メール(有効期限 24 時間)による本人確認を経て有効化できること。
  1. 確認完了前のアドレスはウィジェット表示に用いない
  2. ウィジェット利用者向けメールは常に共通ドメインの no-reply 送信で固定し、本アドレスは返信先には用いない
 BR-051 P0
FR-033c 公開ウィジェットにおいて、確認完了済みのプロジェクト連絡先メールを未解決時・質問受付制限時のお問い合わせ先として表示できること。未設定または未確認の場合はメールアドレスを表示しないこと BR-052 P0
FR-034 プロジェクト削除時の関連データ(FAQ、質問ログ、未解決質問)の取扱いをアカウント利用者が確認・選択できること BR-053 P0
FR-035 プロジェクト数に固定の上限を設けず、利用状況を監視し急激な増加を検知してオーナーへ通知できること BR-054 P1
