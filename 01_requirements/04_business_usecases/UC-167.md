# <span id="UC-167"></span>UC-167: 利用規約同意をチェック

> **利用規約同意チェックの状態を切り替え、両チェック充足に応じて続行ボタンの活性を更新する、クライアント内処理のみのユースケース。**

*主アクター オーナー / メンバー ・ ステータス ドラフト ・ 再構成 P2*

| 項目 | 内容 |
|---|---|
| 業務ユースケースID | UC-167 |
| 業務ユースケース名 | 利用規約同意をチェック |
| 対応要件ID | [FR-010](../02_FunctionalRequirement/01_account-fr.md#FR-010) |
| 主アクター | オーナー / メンバー |
| 目的 | 利用規約同意チェックの状態を切り替え、両チェック充足に応じて続行ボタンの活性を更新する、クライアント内処理のみのユースケース。 |

## 事前条件

利用規約が改定対象で同意チェック(IT-02)が表示されている

適用業務ルール: [RULE-006](../01_BusinessRequirement/08_rule.md#RULE-006)。

## 基本フロー

1. 利用者が利用規約同意(IT-02)をチェック / アンチェックする。
2. 画面は両チェックの充足状態を判定し、IT-04 の活性 / 非活性を更新する。

## 代替フロー

—(本イベントは単一の正常フロー。条件分岐は基本フローに含む)

## 例外フロー

- なし(画面内の状態更新のみ)。

## 事後条件

チェック状態を切り替え、両チェックの充足状態に応じて「同意して続行する」(IT-04)の活性 / 非活性を更新する
