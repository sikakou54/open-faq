FR09: 利用量・課金

この要件は、完全従量課金制 + 月次無料枠の課金モデルのもとで、利用量を正しく集計し、プロジェクト単位の上限・アラートと支払い状態に応じてウィジェットの質問受付を制御することを目的とし、以下について記載します。

  • 課金対象(質問数 / FAQ 件数 / AI 利用コスト)と MVP 無料枠・超過単価の初期値
  • プロジェクト単位の質問数上限・アラート閾値の設定と、上限到達時の質問受付停止
  • 支払方法未登録時の支払方法ゲート(契約は有効のまま質問受付のみ停止)
  • 支払い失敗時の猶予期間とサスペンションへの移行、復帰条件
  • 課金プロバイダからの通知(イベント)受信と再処理

版数 v2.1更新 2026-06-16機能グループ 利用量・課金優先度 Must承認済

1. 業務要件

ID 観点 ステークホルダ 業務要件 関連 NFR
BR-086 課金モデル システム 完全従量課金制 + 月次無料枠を採用し、JST の暦月を月次境界として無料枠を毎月リセットする。課金対象と MVP 初期値は次のとおり。
  1. 質問数:無料枠 1,000 件 / 月、超過単価 0.5 円 / 件(月次上限件数の設定対象)
  2. FAQ 件数:無料枠 100 件、超過単価 5 円 / 件 / 月(上限なし、計測と従量課金のみ継続)
  3. AI 利用コスト(原価):MVP はサービス側が吸収し、契約単位の社内可視化指標として扱う
無料枠は全課金対象でプロジェクト単位、月次上限件数は質問数のみプロジェクト単位で判定する。		 —
BR-087 利用量集計 システム 質問数・AI 回答数(参考値)・未解決質問数・AI 利用コスト・FAQ 件数を、課金対象の定義に従った粒度で集計し、課金対象外(失敗時)の区別を保持する。利用発生時のリアルタイム集計を行い、管理画面向けサマリは 5 分以内の遅延で反映する。 —
BR-088 上限・アラート設定 オーナー / プロジェクト管理者 プロジェクトごとに質問数の月次上限件数(オン / オフ)とアラート閾値を設定する。設定の優先順位は「プロジェクト設定 > デフォルト推奨値」とする。FAQ 件数は上限・アラートの対象としない。 —
BR-089 上限到達時の受付停止 システム プロジェクトの質問数上限がオンで利用が上限件数の 100% に到達した時点で、支払方法の有無に関わらず当該プロジェクトのウィジェット新規質問受付を停止する。契約は有効のまま、エンドユーザーへ安全な定型文を表示し入力・送信を無効化する。125% は集計遅延・誤差対策の最終ガードとして追加のアラート通知を行う。停止からの復帰は翌月 1 日のリセット・上限引き上げ・上限オフのいずれかで即時とする。 —
BR-090 上限アラート通知 システム プロジェクトごとに選択された質問数上限アラート閾値(25% / 50% / 80% / 90% / 100%)へ当月初回到達した時点で、オーナー + 当該プロジェクトの有効な管理者へメール送信する(同一メールアドレスは重複排除)。全閾値未選択時はアラート通知なしとする。 —
BR-091 支払方法ゲート システム 支払方法はクレジットカード払いとし、新規登録時点での登録は必須としない。無料枠判定はプロジェクト単位、支払方法は契約単位とする。
  1. いずれかのプロジェクトが無料枠の 80% に到達した時点で、利用状況・お知らせ・メールで事前告知する
  2. 支払方法未登録のまま無料枠を超過した時点で、当該プロジェクトのウィジェット新規質問受付のみを停止し、支払方法登録へ誘導する(契約サスペンションは行わず、管理画面は継続利用可能)
  3. 支払方法登録時点で即時再開する
 —
BR-092 支払い失敗時のサスペンション システム 決済失敗確定の通知を受信した時刻を起点に 7 日間の猶予期間を計測する。
  1. 猶予期間中も管理者ユーザーの管理画面ログイン・支払方法更新を許可する
  2. 猶予期間中の再決済成功の通知を受信した時点で即時に猶予を解除する
  3. 猶予期間経過後は契約をサスペンション中へ移行し、ウィジェット応答を機能停止の旨を示す応答へ切り替える
  4. サスペンション中は管理者ユーザーの管理画面アクセスを課金・退会のみに限定する
 —
BR-093 請求内訳の確認 オーナー 請求画面で支払方法・請求状態・請求履歴・契約全体の請求見込み・プロジェクト別の課金内訳を確認する。契約全体と各プロジェクトの利用量、およびプロジェクト単位の上限詳細を確認できる。利用契約の解約申請はオーナー専有とする。 —
BR-094 レート制限 システム システムは各機能に契約・種別別のレート制限を DDoS / Bot / 暴走対策として適用する。レート制限は契約単位で管理し、プロジェクト単位化(月次上限件数・無料枠)の対象外とする。 —
BR-095 課金プロバイダ通知の受信 システム システムは課金プロバイダからの通知(イベント)を受信し、送信元の正当性確認・重複処理防止・直近 30 日の受信履歴保持・処理失敗時の通知を行う。再処理の指示を受け付ける。 —

  1. 月次上限はオン / オフを切り替えできる
  2. 設定の優先順位は「プロジェクト設定(オーナー / 当該プロジェクト管理者)> デフォルト推奨値」とする
  3. プロジェクト単位の上限管理では質問数のみを対象とし、当月利用・今月の利用上限・利用量を簡潔に確認できる
  4. FAQ 件数は上限・アラートの対象とせず、管理者ユーザー向けの設定も提供しない

FR-122 プロジェクトごとの質問数上限がオンの場合、上限到達時にウィジェットの新規質問受付を停止できること。
  1. 質問数が当該プロジェクトの設定上限件数の 100% に到達した時点で、支払方法の有無に関わらず当該プロジェクトのウィジェット新規質問受付を停止する(契約は有効のまま、エンドユーザーにはプロジェクト連絡先メールへの問い合わせを促す安全な定型文をチャット内のシステム返信として表示し、質問入力と送信を無効化する)
  2. 125% は集計遅延・誤差対策の最終ガードとして追加のアラート通知を行う
  3. 管理者向けメールは FR-127 で選択された閾値に従い、100% が未選択の場合も停止処理自体は行う
  4. 上限オフ時は利用率の算出・上限アラート・上限到達による停止を行わない
  5. 停止からの復帰は、翌月 1 日のリセット、質問数上限の引き上げ、または上限オフのいずれかで即時とする
  6. FAQ 件数は本判定と月次件数による作成停止の対象外とする
  7. 支払方法未登録の契約のいずれかのプロジェクトが質問数無料枠を超過した場合も、FR-136 の支払方法ゲートにより同様に当該プロジェクトのウィジェット質問受付を停止する
 BR-089 P0

FR-124 支払い失敗や停止時の利用制限を行えること。
  1. 決済失敗確定の通知を受信した時刻を起点として 7 日間の猶予期間を計測する
  2. 猶予期間中も管理者ユーザーの管理画面ログイン・支払方法更新を許可する
  3. 猶予期間中の再決済成功(支払い成功の通知受信)時は即時に猶予を解除する
  4. 猶予期間経過後は契約をサスペンション中へ移行させ、ウィジェット応答を機能停止の旨を示す応答に切り替える
  5. サスペンション中も管理者ユーザーは管理画面の課金・退会のみアクセス可能とし、それ以外の機能は利用不可とする
 BR-092 P0
FR-125 プロジェクトごとの質問数上限と上限アラートの設定変更を、オーナー / 当該プロジェクト管理者が行えること。利用契約の解約申請はオーナーが行えること(解約はオーナー専有) BR-088, BR-093 P0
FR-126 質問数の選択アラート閾値への到達、100% 停止判定、125% 追加通知について、利用発生時のリアルタイム集計を行い、管理者ユーザー向けサマリ表示は 5 分以内の遅延で反映されること BR-087 P1
FR-127 プロジェクトごとに質問数上限に対する 25% / 50% / 80% / 90% / 100% のアラート閾値を複数選択でき、選択した各閾値へ当月初回到達した時点で、オーナー + 当該プロジェクトの有効な管理者へメール送信できること(同一メールアドレスは重複排除)。送信先は固定とし、全閾値未選択はアラート通知なしとする。FAQ 件数は上限アラート対象外とする BR-090 P0

FR-136 支払い方式はクレジットカード払いとし、支払方法ゲートを設けること。新規登録時点では支払方法の登録は必須としない(月次無料利用枠内は支払方法未登録でも利用可能)。無料利用枠判定はプロジェクト単位、支払方法は契約単位とする。

  3. 支払方法登録後は即時再開する
