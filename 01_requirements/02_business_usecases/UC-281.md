# <span id="UC-281"></span>UC-281: 表示要素の状態を区別して表示する

> **KPI・一覧・アラートが0件・未集計・集計中・取得失敗・期間外の状態を区別して表示できることを定義します。**

*主アクター プロジェクトメンバー ・ ステータス ドラフト ・ 再構成 P7後続第2段*

| 項目 | 内容 |
|---|---|
| 業務ユースケースID | UC-281 |
| 業務ユースケース名 | 表示要素の状態を区別して表示する |
| 対応要件ID | [FR-111](../01_specifications/FR-111.md#FR-111) ・ [BR-075](../01_specifications/BR-075.md#BR-075) |
| 主アクター | プロジェクトメンバー |
| 目的 | KPI・一覧・アラートが0件・未集計・集計中・取得失敗・期間外の状態を区別して表示できることを定義します。 |

## 事前条件

ダッシュボードの表示要素を描画している。

## 基本フロー

1. 各表示要素が現在の状態(0件 / 未集計 / 集計中 / 取得失敗 / 期間外)を判定する。
2. 状態ごとの文言と配色ルール(基本設計の画面設計を正本)に従って表示する。

## 例外フロー

—

## 事後条件

各表示要素が状態を区別して表示される。
