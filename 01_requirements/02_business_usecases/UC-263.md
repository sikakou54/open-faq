# <span id="UC-263"></span>UC-263: セキュリティ目的の契約単位レート制限を適用する

> **DDoS / Bot / 暴走対策として、各機能に契約・種別別のレート制限を契約単位で適用することを定義します。**

*主アクター システム(バッチ・通知・非同期) ・ ステータス ドラフト ・ 再構成 P7後続第2段*

| 項目 | 内容 |
|---|---|
| 業務ユースケースID | UC-263 |
| 業務ユースケース名 | セキュリティ目的の契約単位レート制限を適用する |
| 対応要件ID | [FR-095](../01_specifications/03_usage-fr.md#FR-095) ・ [BR-067](../01_specifications/03_usage-br.md#BR-067) |
| 主アクター | システム(バッチ・通知・非同期) |
| 目的 | DDoS / Bot / 暴走対策として、各機能に契約・種別別のレート制限を契約単位で適用することを定義します。 |

## 事前条件

各機能へのアクセスが発生している。

## 基本フロー

1. トリガー: 機能へのリクエストが発生する。
2. システムが契約・種別別のレート制限を契約単位で評価する。
3. 上限超過時はリクエストを抑制する。

## 代替フロー

本レート制限はプロジェクト単位化の対象外とする(月次上限件数・無料枠のみプロジェクト単位)。

## 例外フロー

—

## 事後条件

セキュリティ目的のレート制限が契約単位で適用される。
