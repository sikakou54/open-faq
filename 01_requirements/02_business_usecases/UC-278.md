# <span id="UC-278"></span>UC-278: 概要・利用状況画面の目的を冒頭1行で宣言する

> **概要画面と利用状況画面の冒頭に目的を1行で宣言することを定義します。**

*主アクター プロジェクトメンバー ・ ステータス ドラフト ・ 再構成 P7後続第2段*

| 項目 | 内容 |
|---|---|
| 業務ユースケースID | UC-278 |
| 業務ユースケース名 | 概要・利用状況画面の目的を冒頭1行で宣言する |
| 対応要件ID | [FR-107](../01_specifications/FR-107.md#FR-107) ・ [BR-075](../01_specifications/BR-075.md#BR-075) |
| 主アクター | プロジェクトメンバー |
| 目的 | 概要画面と利用状況画面の冒頭に目的を1行で宣言することを定義します。 |

## 事前条件

概要または利用状況画面を表示している。

## 基本フロー

1. 概要は選択中プロジェクトの利用状況把握を目的として冒頭1行で宣言する。
2. 利用状況は契約全体と各プロジェクトの当月利用状況把握を目的として冒頭1行で宣言する。

各画面の目的が冒頭で明示される。
