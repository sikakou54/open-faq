# <span id="UC-279"></span>UC-279: 未読件数を遷移時取得し過大負荷なく更新する

> **お知らせ未読件数を管理画面遷移時に取得し、滞在中も過大な負荷を生まない範囲で更新できることを定義します。**

*主アクター アカウント利用者 ・ ステータス ドラフト ・ 再構成 P7後続第2段*

| 項目 | 内容 |
|---|---|
| 業務ユースケースID | UC-279 |
| 業務ユースケース名 | 未読件数を遷移時取得し過大負荷なく更新する |
| 対応要件ID | [FR-166](../01_specifications/05_notification-fr.md#FR-166) ・ [BR-110](../01_specifications/05_notification-br.md#BR-110) |
| 主アクター | アカウント利用者 |
| 目的 | お知らせ未読件数を管理画面遷移時に取得し、滞在中も過大な負荷を生まない範囲で更新できることを定義します。 |

## 事前条件

管理画面を利用している。

## 基本フロー

1. 利用者が管理画面へ遷移する。
2. システムが未読件数を取得する。
3. システムが画面滞在中も過大な負荷を生まない範囲で未読件数を更新する。

## 事後条件

未読件数が遷移時に取得され、適度に更新される。
