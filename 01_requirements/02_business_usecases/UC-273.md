# <span id="UC-273"></span>UC-273: 通知配信状態を可視化する

> **送信待ち・送信済み・配信済み・失敗・バウンス・苦情・停止の通知配信状態を可視化できることを定義します。**

*主アクター プロジェクトメンバー ・ ステータス ドラフト ・ 再構成 P7後続第2段*

| 項目 | 内容 |
|---|---|
| 業務ユースケースID | UC-273 |
| 業務ユースケース名 | 通知配信状態を可視化する |
| 対応要件ID | [FR-119](../01_specifications/05_notification-fr.md#FR-119) ・ [BR-084](../01_specifications/05_notification-br.md#BR-084) |
| 主アクター | プロジェクトメンバー |
| 目的 | 送信待ち・送信済み・配信済み・失敗・バウンス・苦情・停止の通知配信状態を可視化できることを定義します。 |

## 事前条件

通知配信実績が蓄積されている。

## 基本フロー

1. 利用者が通知配信状態を表示する。
2. システムが各通知の配信状態(送信待ち / 送信済み / 配信済み / 失敗 / バウンス / 苦情 / 停止)を可視化する。

## 代替フロー

—

## 例外フロー

対象データが無い場合は0件状態を表示する。

## 事後条件

通知配信状態が可視化される。
