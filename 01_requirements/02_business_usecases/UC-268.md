# <span id="UC-268"></span>UC-268: 通知失敗・バウンス件数を確認する

> **通知の失敗件数・バウンス件数を確認できることを定義します。**

*主アクター プロジェクトメンバー ・ ステータス ドラフト ・ 再構成 P7後続第2段*

| 項目 | 内容 |
|---|---|
| 業務ユースケースID | UC-268 |
| 業務ユースケース名 | 通知失敗・バウンス件数を確認する |
| 対応要件ID | [FR-104](../01_specifications/FR-104.md#FR-104) ・ [BR-073](../01_specifications/BR-073.md#BR-073) |
| 主アクター | プロジェクトメンバー |
| 目的 | 通知の失敗件数・バウンス件数を確認できることを定義します。 |

## 事前条件

通知配信実績が蓄積されている。

## 基本フロー

1. 利用者がダッシュボードを開く。
2. システムが通知失敗・バウンス件数を表示する。

## 代替フロー

—

## 例外フロー

対象データが無い場合は0件状態を表示する。

## 事後条件

通知失敗・バウンス件数が確認できる。
