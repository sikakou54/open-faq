## <span id="1-システム概要"></span>1. システム概要

本サービス(FAQ AI ウィジェット SaaS、サービス名: open-faq)は、サイト運営者(アカウント利用者)が登録した FAQ をもとに、サイト訪問者(エンドユーザー)からの問い合わせに自動応答する SaaS である。

| 観点 | 概要 |
|----|----|
| 提供形態 | クラウドサービス(SaaS) |
| 利用者 | Web サイト運営事業者、その従業員、Web サイト訪問者 |
| 提供価値 | FAQ の自動応答、未解決問い合わせの集約、問い合わせ先案内、FAQ 改善ループ |
| 課金 | 完全従量課金制 + 月次無料枠(MVP) |
| 設置形態 | 利用者サイトへのスクリプトタグ埋め込み(ウィジェット) |
| アカウント管理 | 自前アカウント管理 |
| メール配信 | メール配信プロバイダ経由 |

**システム構成**: 本サービスは利用者向け管理画面とエンドユーザー向け FAQ ウィジェットから成る。本書はその要件定義の正本である。

**スコープ(MVP のみ)**: 本書は MVP で実現する範囲のみを記載する。実現の方式・状態遷移・画面項目・外部連携方式・実装の詳細といった実現手段ではなく、業務上・利用者上の要求(何を実現したいか)を定義する。

## <span id="2-背景目的"></span>2. 背景・目的

Web サイト運営事業者は、訪問者からの問い合わせ対応コストの増大、回答品質のばらつき、夜間・休日対応の限界という課題を抱えている。生成 AI による自動応答は手軽に導入できる反面、社内に存在しない事実を「もっともらしく」回答してしまう **ハルシネーション** のリスクがある。

本サービスは、この課題に対し「**公開済み FAQ に記載された内容のみ**」を根拠に AI が回答する仕組みを提供することで、ハルシネーションを抑えながら問い合わせ自動化を実現する。回答できなかった質問は問い合わせ ID を付与して未解決質問として内部管理し、ウィジェットでは確認済み連絡先を案内する。その内容を FAQ に反映するループを回すことで、運用とともに自動回答率を高める。

**差別化ポイント**: ハルシネーション抑制 / 未解決質問の確実な収集 / FAQ 登録進捗の明確化 / 運用と改善の循環 / 1 アカウントでのマルチプロジェクト運用(作成したプロジェクトと招待されて参加するプロジェクトを同じユーザーが扱える)。

## <span id="3-スコープ"></span>3. スコープ

本書は **MVP で実現する範囲のみ** を記載する。

| 対象 | 内容 |
|----|----|
| 管理画面 | アカウント、オーナー / メンバー、プロジェクト、FAQ、ウィジェット、質問ログ、未解決質問、課金、設定の管理 |
| 公開ウィジェット | Web サイト訪問者が質問し、FAQ に基づく AI 回答または未解決時の連絡先案内を受ける |
| FAQ 改善機能 | 未解決質問から FAQ を下書き・公開し、登録状況を管理できる |
| 通知機能 | 請求確定、運営お知らせ、重要な状態変更をメール・受信箱で通知する |
| 運用機能 | ログ確認、障害対応、データ保持、退会処理、データ削除(アカウント全データエクスポートは将来対応) |

**MVP の安全境界**: AI が FAQ にない内容を推測回答しないこと、FAQ の公開はメンバーが編集画面で内容を確認したうえで状態(下書き / 公開中 / 非公開)を選択して保存する明示操作であること([FR-049](02_functional_requirement/02_faq-ai-fr.md#FR-049))。具体要件は [FR-057](02_functional_requirement/02_faq-ai-fr.md#FR-057) を正本とする。

## <span id="4-利用者関係者"></span>4. 利用者・関係者

本サービスは立場の異なる利用者を想定する。立場(オーナー / メンバー)はプロジェクトごとに決まり、アカウント単位で固定されない。「認可範囲」は、プロジェクト所有(作成者)・プロジェクト割当・ユーザー種別(オーナー / メンバー)の組み合わせで決定する。主要用語の正式名称・別名・意味は [用語集](00_glossary.md) を参照する。

| 利用者種別 | 説明 |
|----|----|
| オーナー | そのプロジェクトを作成したユーザー。当該プロジェクトの管理責任と課金責任を持ち、当該プロジェクトに対して全権を持つ。立場はプロジェクトごとに決まるため、別のプロジェクトではメンバーになり得る |
| メンバー | 他者が作成したプロジェクトに招待されて参加し、許可された範囲で当該プロジェクトを利用する。当該プロジェクトで FAQ 管理 / 未解決質問管理 / ログ参照に加え、メンバー招待・編集・割当解除、上限 / アラート設定、お知らせ受信箱、ウィジェット設定を行える。割り当てられたメンバーは当該プロジェクト内で同一の権限を持ち、役割差は持たない。複数の異なるオーナーのプロジェクトに同時に参加でき、自らもプロジェクトを作成してオーナーになれる |
| エンドユーザー | サイト運営者(アカウント利用者)の Web サイト訪問者。公開ウィジェットから FAQ 質問と回答の閲覧を行う |

**関係者(ステークホルダー)**: プロダクトオーナー、アカウント利用者(顧客)、エンドユーザー、開発者、QA、法務 / プライバシー担当、経営層。
