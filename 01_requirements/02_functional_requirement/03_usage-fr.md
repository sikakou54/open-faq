# 利用量・課金・ダッシュボード・運用(機能要件)

> **このページは、利用量・課金・ダッシュボード・運用 に関する機能要件を定義します。**

*複数の個別要件を統合したカテゴリ別ページ。 ステータス ドラフト*

| ID | 名称 | 優先度 | 要件 |
|----|----|----|----|
| <span id="FR-086"></span>FR-086 | 利用量・AI 利用コストの集計 | P0 | 質問数、AI 回答数(参考値)、未解決質問数、AI 利用コスト(原価)、FAQ 件数を集計できること。集計粒度は課金対象の定義に従い、課金対象外(失敗時)の区別も保持すること。 |
| <span id="FR-087"></span>FR-087 | プロジェクト別の月次上限・アラート設定 | P0 | プロジェクトごとに質問数の月次上限件数と上限アラートを任意に設定できること。月次上限はオン/オフを切り替えでき、設定の優先順位はプロジェクト設定(オーナー/当該プロジェクトのメンバー)優先とする。 |
| <span id="FR-088"></span>FR-088 | 上限到達時のウィジェット受付停止 | P0 | プロジェクトごとの質問数上限がオンの場合、上限到達時にウィジェットの新規質問受付を停止できること。質問数が設定上限の100%に到達した時点で当該プロジェクトの受付を停止し、支払方法未登録でも無料枠超過時に同様に停止する。 |
| <span id="FR-089"></span>FR-089 | 請求情報の確認(支払方法・履歴・見込み) | P0 | オーナーは請求画面で、支払方法・請求状態・請求履歴・自分が作成したプロジェクト全体の請求見込み・プロジェクト別の課金内訳を確認できること。 |
| <span id="FR-090"></span>FR-090 | 支払い失敗や停止時の利用制限を行える | P0 | 支払い失敗や停止時の利用制限を行えること。決済失敗確定の通知受信を起点に7日間の猶予期間を計測し、猶予経過後はアカウントをサスペンション状態へ移行させる。 |
| <span id="FR-091"></span>FR-091 | 上限・アラート設定変更と解約申請 | P0 | プロジェクトごとの質問数上限と上限アラートの設定変更を、オーナー/当該プロジェクトのメンバーが行えること。プロジェクト削除はオーナー専有、アカウントの退会は本人専有とする。 |
| <span id="FR-092"></span>FR-092 | 利用量のリアルタイム集計と反映 | P1 | 質問数の閾値到達・100%停止判定・125%追加通知について利用発生時のリアルタイム集計を行い、アカウント利用者向けサマリ表示は5分以内の遅延で反映されること。 |
| <span id="FR-093"></span>FR-093 | 上限アラート閾値の通知 | P0 | プロジェクトごとに質問数上限に対する25%/50%/80%/90%/100%のアラート閾値を複数選択でき、選択した各閾値へ当月初回到達した時点でオーナーと当該プロジェクトの有効なメンバーへメール送信できること。 |
| <span id="FR-094"></span>FR-094 | オーナー・種別別のレート制限 | P0 | 各機能に対してオーナー・種別別のレート制限をDDoS/Bot/暴走対策として適用すること。本レート制限はオーナー単位で管理し、プロジェクト単位化の対象外とする。 |
| <span id="FR-095"></span>FR-095 | 支払い方式はクレジットカード払いとし、支払方法ゲートを設ける | P0 | 支払い方式はクレジットカード払いとし、支払方法ゲートを設けること。新規登録時点では支払方法の登録は必須とせず、無料利用枠超過時に当該プロジェクトのウィジェット受付を停止して請求画面へ誘導し、登録後は即時再開する。 |
| <span id="FR-096"></span>FR-096 | 猶予期間経過後のサスペンション移行 | P0 | 支払い失敗を検知した場合、猶予期間(7日)を経過後にアカウントをサスペンション状態へ移行すること。 |
| <span id="FR-097"></span>FR-097 | サスペンション中の機能制限 | P0 | サスペンション中は利用者ログインと請求情報更新のみ可能とし、その他の機能および新規ウィジェット応答は機能停止の旨を示す応答に切り替えること。 |
| <span id="FR-098"></span>FR-098 | 課金プロバイダ通知の受信処理 | P0 | 課金プロバイダからの通知(イベント)を受信し、送信元の正当性確認、重複処理防止、直近30日の受信履歴、再処理、処理失敗時の通知を備えること。 |
| <span id="FR-099"></span>FR-099 | 質問数・未解決数を確認 | P0 | 質問数・未解決数を確認できること。 |
| <span id="FR-100"></span>FR-100 | 要対応質問の対応状況確認 | P0 | 要対応の質問の状況を、アカウント利用者向けには「対応中/対応済み」で確認できること。 |
| <span id="FR-101"></span>FR-101 | よく聞かれる質問や未解決傾向を確認 | P1 | よく聞かれる質問や未解決傾向を確認できること。 |
| <span id="FR-102"></span>FR-102 | 期間絞り込み | P0 | 期間絞り込みができること。 |
| <span id="FR-103"></span>FR-103 | 通知失敗・バウンス件数を確認 | P0 | 通知失敗・バウンス件数を確認できること。 |
| <span id="FR-104"></span>FR-104 | ダッシュボードで My / Join を区分し立場に応じた範囲を表示 | P0 | ダッシュボードで自分が作成したプロジェクト(My)と参加しているプロジェクト(Join)を区分し、各プロジェクトに対する自分の立場(オーナー/メンバー)に応じた範囲を表示できること。 |
| <span id="FR-105"></span>FR-105 | ダッシュボードと各プロジェクト概要の間を、迷わず行き来 | P0 | ダッシュボードと各プロジェクト概要の間を迷わず行き来できること。ダッシュボードの各プロジェクトから概要へ1操作で遷移でき、概要から1操作でダッシュボードへ戻れる。 |
| <span id="FR-106"></span>FR-106 | 画面目的の冒頭宣言 | P0 | 概要画面とダッシュボードは冒頭に目的を1行で宣言すること。概要は選択中プロジェクトの利用状況の把握、ダッシュボードは関与プロジェクト(My/Join)の区分一覧と状況把握を目的とする。 |
| <span id="FR-107"></span>FR-107 | 集計時点を明示し、リアルタイム値との誤認を防ぐ | P0 | 集計時点を明示し、リアルタイム値との誤認を防ぐこと。最終更新時刻を表示する画面ではいつ時点の集計かを明示し、集計が遅延している場合は警告表示する。 |
| <span id="FR-108"></span>FR-108 | KPI カードからの一覧遷移 | P1 | 主要指標(KPI)カード(質問数/未解決数/公開FAQ件数等)はクリックで該当一覧画面へ遷移できること。0件・集計中・取得失敗の状態ではクリック不可とする。 |
| <span id="FR-109"></span>FR-109 | 比較値の表記を統一 | P1 | 比較値を表示する画面では前月比・前週比の表記を統一すること。概要のKPIカードは項目名と件数だけを表示し、前月比・コメント・ゲージは表示しない。 |
| <span id="FR-110"></span>FR-110 | 集計状態の区別表示 | P0 | KPI/一覧/アラートの各表示要素は、0件・未集計・集計中・取得失敗・期間外の状態を利用者がそれぞれ区別して認識できるよう表示できること。 |
| <span id="FR-111"></span>FR-111 | プロジェクト 0 件時の空状態表示 | P2 | 利用状況でプロジェクトが0件の場合は、複雑なセットアップチェックリストを表示せず、「プロジェクトがまだありません」と「新規プロジェクトを作成」する導線を持つ空状態を表示すること。 |
| <span id="FR-167"></span>FR-167 | FAQ 編集は同時編集時の衝突を検出 | P1 | FAQ編集は同時編集時の衝突を検出できること。衝突時は最新版の確認を促し、再読み込みの導線を提供すること。 |
| <span id="FR-168"></span>FR-168 | 下書き自動保存と離脱警告 | P2 | FAQ下書きは自動保存し、未保存の変更がある状態でページ離脱を試みた際は警告すること。 |
| <span id="FR-169"></span>FR-169 | 一括操作の 1 操作あたりの上限を設ける | P1 | 一括操作の1操作あたりの上限を設けること。お知らせの一括既読は最大100件、FAQの一括状態変更・一括削除は最大50件とする。 |
| <span id="FR-170"></span>FR-170 | 影響度に応じた確認レベルの割当 | P0 | 不可逆または影響範囲の大きい操作に、影響度に応じた確認レベルを必ず割り当てること。確認レベルは確認ダイアログ・対象名のタイプ確認・本人確認(再認証)に区分し、影響度が大きいほど高い確認レベルを割り当てる。 |
| <span id="FR-171"></span>FR-171 | 管理スコープの分離表示 | P1 | 利用者向け管理コンソールは、管理スコープ(アカウント範囲/プロジェクト範囲/認証フロー/共通領域)を画面から判別できるよう分離すること。 |
| <span id="FR-172"></span>FR-172 | 空状態の作成導線のみ表示 | P1 | 利用状況でプロジェクトが0件の場合は、説明文と「新規プロジェクトを作成」する導線だけを持つ空状態を表示すること。段階型のセットアップチェックリストは表示しない。 |
| <span id="FR-173"></span>FR-173 | アカウント項目の利用中プロジェクト数併記 | P2 | アカウントスコープの参照表示には、影響範囲を示す「利用中のプロジェクト: N」を併記すること。N=0の場合は「未使用」と表示し、月次利用上限・無料利用枠はプロジェクト単位のため対象外とする。 |

| <span id="FR-175"></span>FR-175 | 一覧では名称を主リンク、システム識別子をコピー可能な補助情報として表示 | P0 | 一覧では名称を主リンク、システム識別子をコピー可能な補助情報として表示すること。内部の英語値(状態コード等)はユーザー向け表示へ直接露出させず、日本語の業務語彙へ変換すること。 |
| <span id="FR-176"></span>FR-176 | 設定種別の分離と影響範囲の判別 | P0 | 個人設定・アカウント設定・プロジェクト編集を分離し、設定対象と影響範囲を画面タイトル・説明から判別できること。 |
| <span id="FR-177"></span>FR-177 | 画面見出しの簡潔化 | P1 | 画面見出しは画面名だけを表示し、組織名・プロジェクト名・サイト名を連結しないこと。削除・公開キー再発行等の対象確認や編集画面では誤操作防止に必要な対象名表示を維持すること。 |
