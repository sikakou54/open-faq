# 利用量・課金・ダッシュボード・運用(機能要件)

*複数の個別要件を統合したカテゴリ別ページ。各要件の優先度・トレースは各節を参照。 ステータス ドラフト*

## <span id="FR-087"></span>FR-087: 利用量・AI 利用コストの集計

> **この機能要件は「利用量・AI 利用コストの集計」を定義します。**

*種別 機能要件 ・ 機能グループ 利用量・課金 ・ 優先度 P0 ・ ステータス ドラフト*

### 要件

質問数、AI 回答数(参考値)、未解決質問数、AI 利用コスト(原価)、FAQ 件数を集計できること。集計粒度は課金対象の定義に従い、課金対象外(失敗時)の区別も保持すること

### シーケンス

```mermaid
sequenceDiagram
  autonumber
  actor U as ユーザー
  participant UI as UI
  participant S as サーバー
  Note over S: トリガ(定期集計)
  U->>UI: 利用状況画面を開く
  UI->>S: 集計データ要求
  S-->>UI: 利用量・コストの集計結果
  UI-->>U: ダッシュボード表示
```

## <span id="FR-088"></span>FR-088: プロジェクト別の月次上限・アラート設定

> **この機能要件は「プロジェクト別の月次上限・アラート設定」を定義します。**

*種別 機能要件 ・ 機能グループ 利用量・課金 ・ 優先度 P0 ・ ステータス ドラフト*

### 要件

プロジェクトごとに質問数の月次上限件数と上限アラートを任意に設定できること。
1. 月次上限はオン / オフを切り替えできる
2. 設定の優先順位は「プロジェクト設定(オーナー / 当該プロジェクトのメンバー)&gt; デフォルト推奨値」とする
3. プロジェクト単位の上限管理では質問数のみを対象とし、当月利用・今月の利用上限・利用量を簡潔に確認できる
4. FAQ 件数は上限・アラートの対象とせず、アカウント利用者向けの設定も提供しない

### シーケンス

```mermaid
sequenceDiagram
  autonumber
  actor U as ユーザー
  participant UI as UI
  participant S as サーバー
  U->>UI: 上限設定画面を開く
  UI->>S: 現在の上限設定要求
  S-->>UI: 上限・アラート設定値
  U->>UI: 上限件数とアラートを入力して保存
  UI->>S: 上限設定の更新要求
  S-->>UI: 保存結果
  UI-->>U: 設定完了を表示
```

## <span id="FR-089"></span>FR-089: 上限到達時のウィジェット受付停止

> **この機能要件は「上限到達時のウィジェット受付停止」を定義します。**

*種別 機能要件 ・ 機能グループ 利用量・課金 ・ 優先度 P0 ・ ステータス ドラフト*

### 要件

プロジェクトごとの質問数上限がオンの場合、上限到達時にウィジェットの新規質問受付を停止できること。
1. 質問数が当該プロジェクトの設定上限件数の **100%** に到達した時点で、支払方法の有無に関わらず当該プロジェクトのウィジェット新規質問受付を停止する(契約は有効のまま、エンドユーザーにはプロジェクト連絡先メールへの問い合わせを促す安全な定型文をチャット内のシステム返信として表示し、質問入力と送信を無効化する)
2. **125%** は集計遅延・誤差対策の最終ガードとして追加のアラート通知を行う
3. 管理者向けメールは [FR-094](#FR-094) で選択された閾値に従い、100% が未選択の場合も停止処理自体は行う
4. 上限オフ時は利用率の算出・上限アラート・上限到達による停止を行わない
5. 停止からの復帰は、翌月 1 日のリセット、質問数上限の引き上げ、または上限オフのいずれかで即時とする
6. FAQ 件数は本判定と月次件数による作成停止の対象外とする
7. 支払方法未登録の契約のいずれかのプロジェクトが質問数無料枠を超過した場合も、[FR-096](#FR-096) の支払方法ゲートにより同様に当該プロジェクトのウィジェット質問受付を停止する

### シーケンス

```mermaid
sequenceDiagram
  autonumber
  actor U as ユーザー
  participant UI as UI
  participant S as サーバー
  Note over S: トリガ(利用量集計で上限到達を検知)
  S->>S: 当該プロジェクトの新規質問受付を停止
  U->>UI: 利用状況画面を開く
  UI->>S: プロジェクト状態の要求
  S-->>UI: 受付停止中の旨と復帰条件
  UI-->>U: 停止状態を表示
```

## <span id="FR-091"></span>FR-091: 支払い失敗や停止時の利用制限を行える

> **この機能要件は「支払い失敗や停止時の利用制限を行える」を定義します。**

*種別 機能要件 ・ 機能グループ 利用量・課金 ・ 優先度 P0 ・ ステータス ドラフト*

### 要件

支払い失敗や停止時の利用制限を行えること。
1. 決済失敗確定の通知を受信した時刻を起点として **7 日間**の猶予期間を計測する
2. 猶予期間中もアカウント利用者の管理画面ログイン・支払方法更新を許可する
3. 猶予期間中の再決済成功(支払い成功の通知受信)時は即時に猶予を解除する
4. 猶予期間経過後は契約をサスペンション中へ移行させ、ウィジェット応答を機能停止の旨を示す応答に切り替える
5. サスペンション中もアカウント利用者は管理画面の課金・退会のみアクセス可能とし、それ以外の機能は利用不可とする

### シーケンス

```mermaid
sequenceDiagram
  autonumber
  actor U as ユーザー
  participant UI as UI
  participant S as サーバー
  Note over S: トリガ(決済失敗の課金通知)
  S->>S: 猶予期間を起算
  alt 猶予中に再決済成功
    Note over S: トリガ(支払い成功の課金通知)
    S->>S: 猶予を即時解除
  else 猶予経過
    S->>S: 契約をサスペンション中へ移行
  end
  U->>UI: 管理画面へアクセス
  UI->>S: 利用可否の確認要求
  S-->>UI: 利用制限状態
  UI-->>U: 制限状態を表示
```

## <span id="FR-096"></span>FR-096: 支払い方式はクレジットカード払いとし、支払方法ゲートを設ける

> **この機能要件は「支払い方式はクレジットカード払いとし、支払方法ゲートを設ける」を定義します。**

*種別 機能要件 ・ 機能グループ 利用量・課金 ・ 優先度 P0 ・ ステータス ドラフト*

### 要件

支払い方式はクレジットカード払いとし、支払方法ゲートを設けること。新規登録時点では支払方法の登録は必須としない(月次無料利用枠内は支払方法未登録でも利用可能)。無料利用枠判定はプロジェクト単位、支払方法は契約単位とする。
1. いずれかのプロジェクトが無料利用枠の **80%** に到達した場合、利用状況の該当プロジェクト・お知らせ・メールで事前告知する
2. 未登録のまま無料利用枠を超過した場合は当該プロジェクトのウィジェット新規質問受付のみを停止し、請求(支払方法登録)へ誘導する
3. 支払方法登録後は即時再開する

### シーケンス

```mermaid
sequenceDiagram
  autonumber
  actor U as ユーザー
  participant UI as UI
  participant S as サーバー
  Note over S: トリガ(無料枠超過を集計で検知)
  S->>S: 当該プロジェクトの新規質問受付を停止
  U->>UI: 請求画面で支払方法を登録
  UI->>S: 支払方法の登録要求
  S->>S: 受付を即時再開
  S-->>UI: 登録結果
  UI-->>U: 再開を表示
```

### 要件

画面見出しは画面名だけを表示し、組織名・プロジェクト名・サイト名を連結しないこと。現在の管理範囲名は管理範囲の切替表示に集約すること。ただし、削除・公開キー再発行等の対象確認文やデータ名称そのものを扱う編集画面では、誤操作防止に必要な対象名表示を維持すること

## <span id="FR-191"></span>FR-191: ダッシュボードのセットアップ進捗表示と KPI 表示の切替

> **この機能要件は「ダッシュボードのセットアップ進捗表示と KPI 表示の切替」を定義します。**

*種別 機能要件 ・ 機能グループ 管理ダッシュボード ・ 優先度 P1 ・ ステータス ドラフト*

### 要件

ダッシュボードは、ウィジェットを利用可能にするための初期設定(セットアップ)の完了状態に応じて表示を切り替えること。

1. 設定が未完了の間は、セットアップ進捗(プロジェクト作成・FAQ 登録・ウィジェット埋め込みコード配置の各ステップの完了 / 未完了と次アクション導線)を表示する。
2. 全ステップ完了後は、主要指標(質問数・未解決数・公開 FAQ 件数・利用率)を表示する。
3. セットアップ進捗は独立した画面・メニューとしては設けず、ダッシュボードの一表示パターンとする。
