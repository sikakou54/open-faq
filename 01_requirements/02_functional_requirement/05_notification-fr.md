# 通知・お知らせ(機能要件)

> **このページは、通知・お知らせ に関する機能要件を定義します。**

*複数の個別要件を統合したカテゴリ別ページ。 ステータス ドラフト*

| ID | 名称 | 優先度 | 要件 |
|----|----|----|----|
| <span id="FR-112"></span>FR-112 | 契機に応じた通知配信 | P0 | 請求確定・運営お知らせ・運用イベント・役割割当変更・認証関連など定められた契機で、対象アカウント利用者へメールまたは受信箱通知できること。 |
| <span id="FR-113"></span>FR-113 | 通知の送信失敗時に再送 | P0 | 通知の送信失敗時に再送できること。 |
| <span id="FR-114"></span>FR-114 | バウンス検知時の通知停止と関係者(オーナー/該当メンバー)への通知 | P0 | バウンスや無効アドレスを検知し、通知停止と関係者(オーナー/該当メンバー)への通知ができること。 |
| <span id="FR-115"></span>FR-115 | 認証関連通知のオプトアウト不可 | P1 | アカウント利用者のアカウント認証関連通知(パスワード再設定等)はオプトアウト不可とすること。 |
| <span id="FR-116"></span>FR-116 | 送信品質の監視と送信抑制 | P0 | プロジェクト単位で送信レート制限・バウンス率・苦情率を監視し、しきい値超過時に送信抑制ができること。 |
| <span id="FR-117"></span>FR-117 | 外部露出箇所の入力文字列の無害化 | P0 | ウィジェット利用者が入力した文字列を、メール件名・送信元情報など外部に露出する箇所にそのまま使わないこと(スパム埋め込み・なりすまし対策)。 |
| <span id="FR-118"></span>FR-118 | 通知配信状態の可視化 | P0 | 通知配信状態(送信待ち / 送信済み / 配信済み / 失敗 / バウンス / 苦情 / 停止)を可視化できること。 |
| <span id="FR-119"></span>FR-119 | 通知再送回数の上限を設ける | P0 | 通知再送回数の上限を設けること。 |
| <span id="FR-120"></span>FR-120 | 請求確定のオーナー宛メール通知 | P0 | 月次の請求が確定したタイミングで、オーナーにメール通知できること。通知には請求対象期間・金額・内訳・請求書確認導線を含め、同タイミングで受信箱にもお知らせを生成し、重複請求は行わない。 |
| <span id="FR-121"></span>FR-121 | 運営お知らせの配信 | P0 | 特定のプロジェクトまたは全プロジェクトに対して任意のお知らせをメール通知できること。宛先範囲・件名・本文・送信予定日時を指定でき、規約改定などの重要通知はオプトアウト不可とする。 |
| <span id="FR-122"></span>FR-122 | 運用イベントのシステム通知自動生成 | P1 | 運用イベント(FAQ利用上限80%/100%接近、AI利用上限到達、通知失敗急増、サスペンション、復元、規約改定、価格改定等)を契機にシステム通知のお知らせを自動生成できること。 |
| <span id="FR-123"></span>FR-123 | 配信停止リストの全プロジェクト横断共有 | P0 | バウンス・苦情検知時のメール配信停止リストは全プロジェクト横断で共有し、宛先別に再送停止すること(共通ドメイン使用に伴う対策)。 |
| <span id="FR-124"></span>FR-124 | 役割割当変更時のメンバー通知 | P0 | メンバーのプロジェクト別の役割割当が追加・変更・剥奪された場合、当該メンバーへお知らせ受信箱とメールで通知できること。 |
| <span id="FR-151"></span>FR-151 | お知らせ一覧の取得・表示 | P0 | アカウント利用者は自分宛のお知らせ一覧を取得・表示できること。 |
| <span id="FR-152"></span>FR-152 | お知らせ詳細の表示 | P0 | アカウント利用者はお知らせの詳細(本文・種別・重要度・配信日時)を表示できること。重要度は「低 / 通常 / 重要 / 最重要」の4段階とし、「最重要」は規約改定・重大セキュリティ通知・サスペンション確定時に限定する。 |
| <span id="FR-153"></span>FR-153 | お知らせの個別・一括既読化 | P0 | アカウント利用者はお知らせを個別および一括で既読化できること。 |
| <span id="FR-154"></span>FR-154 | お知らせの絞り込み | P1 | アカウント利用者はお知らせを種別(請求確定 / 運営お知らせ / システム通知)、既読状態、期間で絞り込みできること。 |
| <span id="FR-155"></span>FR-155 | 通知ベルの未読件数バッジ表示 | P0 | 管理画面ヘッダの通知ベルに、未読件数のバッジを表示できること(0件時は非表示)。 |
| <span id="FR-156"></span>FR-156 | 通知ベルの直近お知らせ表示 | P1 | 管理画面ヘッダの通知ベルから、直近10件のお知らせをドロップダウンで表示できること。 |
| <span id="FR-157"></span>FR-157 | お知らせ閲覧範囲の限定 | P0 | お知らせはアカウント利用者のみ閲覧可能とし、ウィジェット利用者は閲覧できないこと。 |
| <span id="FR-158"></span>FR-158 | 受信箱への請求確定お知らせ生成 | P0 | 月次請求の確定タイミングで、アカウント利用者の受信箱に請求確定のお知らせを生成できること。請求確定メール通知と同期して実施する。 |
| <span id="FR-159"></span>FR-159 | 受信箱への運営お知らせ生成 | P0 | お知らせを配信したタイミングで、対象アカウント利用者の受信箱にお知らせを生成できること。お知らせメール通知と同期して実施する。 |
| <span id="FR-160"></span>FR-160 | 運用イベントをシステム通知として受信箱に生成 | P1 | 運用イベント(上限接近・上限超過、通知配信失敗の急増、AI利用上限到達)をシステム通知として受信箱に生成できること。同一プロジェクト・同一イベント種別で60分以内の連続発火は1件に集約する。 |
| <span id="FR-161"></span>FR-161 | 受信箱の保持と退会時削除 | P0 | お知らせ受信箱はアカウント利用者ごとに保持し、アカウント無効化・退会時に削除されること。 |
| <span id="FR-162"></span>FR-162 | 未読件数の取得と更新 | P1 | お知らせの未読件数は管理画面遷移時に取得し、画面滞在中も過大な負荷を生まない範囲で更新できること。 |
| <span id="FR-163"></span>FR-163 | 一括既読操作の監査ログ記録 | P0 | 一括既読操作は監査ログに記録できること(操作者・対象件数・実行日時)。 |
