# 通知・お知らせ(機能要件)

> **このページは、通知・お知らせ に関する機能要件を定義します。**

*複数の個別要件を統合したカテゴリ別ページ。各要件の優先度・トレースは各節を参照。 ステータス ドラフト*

## <span id="FR-113"></span>FR-113: 契機に応じた通知配信

> **この機能要件は「契機に応じた通知配信」を定義します。**

*種別 機能要件 ・ 機能グループ 通知 ・ 優先度 P0 ・ ステータス ドラフト*

### 要件

次の契機で、対象となるアカウント利用者へメールまたは受信箱通知できること。配信契機は次の各種別を網羅する。
1. 請求確定(請求確定のオーナー宛メール通知、[FR-121](#FR-121))
2. 運営からのお知らせ(運営お知らせの配信、[FR-122](#FR-122))
3. 運用イベント(利用上限への接近・到達、通知失敗の急増、サスペンション、復元、規約改定、価格改定 等。運用イベントのシステム通知自動生成、[FR-123](#FR-123))
4. メンバーのプロジェクト別役割割当の変更(役割割当変更時のメンバー通知、[FR-125](#FR-125))
5. アカウント認証関連通知(パスワード再設定 等。認証関連通知のオプトアウト不可、[FR-116](#FR-116))

### シーケンス

```mermaid
sequenceDiagram
  autonumber
  actor U as ユーザー
  participant UI as UI
  participant S as サーバー
  Note over S: トリガ(例 請求確定/運営お知らせ)
  S->>S: 対象アカウント利用者を特定
  S->>UI: メール送信・受信箱通知
  UI-->>U: 通知を受領
```

## <span id="FR-121"></span>FR-121: 請求確定のオーナー宛メール通知

> **この機能要件は「請求確定のオーナー宛メール通知」を定義します。**

*種別 機能要件 ・ 機能グループ 通知 ・ 優先度 P0 ・ ステータス ドラフト*

### 要件

月次の請求が確定したタイミングで、オーナーにメール通知できること。
1. 通知には請求対象期間、請求金額、内訳(質問数・各種上限の超過分など)、請求書 / 明細の確認導線を含める
2. 同タイミングで対象オーナーのお知らせ受信箱に「請求確定」のお知らせを生成する
3. 同一契約・同一請求月での重複請求を行わない

### シーケンス

```mermaid
sequenceDiagram
  autonumber
  actor U as ユーザー
  participant UI as UI
  participant S as サーバー
  Note over S: トリガ(月次請求の確定)
  S->>S: 重複請求でないことを確認
  S->>UI: オーナーへメール通知
  S->>UI: 受信箱に請求確定のお知らせを生成
  UI-->>U: 通知を受領
```

## <span id="FR-122"></span>FR-122: 運営お知らせの配信

> **この機能要件は「運営お知らせの配信」を定義します。**

*種別 機能要件 ・ 機能グループ 通知 ・ 優先度 P0 ・ ステータス ドラフト*

### 要件

特定契約または全契約に対して任意のお知らせ(メンテナンス予告、機能追加、規約改定、価格改定など)をメール通知できること。
1. 宛先範囲(全契約 / 特定契約 / 特定ユーザー種別)、件名、本文、送信予定日時を指定できる
2. 受信者はアカウント利用者を対象とし、種別ごとにオプトアウトの可否を設定できる(規約改定・重要セキュリティ通知などは強制送信としオプトアウト不可)
3. 同タイミングで対象契約のアカウント利用者のお知らせ受信箱に「お知らせ」を生成する

### シーケンス

```mermaid
sequenceDiagram
  autonumber
  actor U as ユーザー
  participant UI as UI
  participant S as サーバー
  U->>UI: お知らせの宛先範囲・件名・本文を入力
  UI->>S: お知らせ配信を要求
  S-->>UI: 受付結果
  UI-->>U: 受付を表示
  Note over S: トリガ(送信予定日時)
  S->>UI: 対象アカウント利用者へメール通知・受信箱生成
  UI-->>U: 通知を受領
```

## <span id="FR-164"></span>FR-164: 運用イベントをシステム通知として受信箱に生成

> **この機能要件は「運用イベントをシステム通知として受信箱に生成」を定義します。**

*種別 機能要件 ・ 機能グループ お知らせ ・ 優先度 P1 ・ ステータス ドラフト*

### 要件

運用イベントをシステム通知として受信箱に生成できること。
1. 対象とする運用イベントは次のとおり。
    1. 上限接近(80%)・上限超過
    2. 通知配信失敗の急増
    3. AI 利用上限到達
2. 同一契約・同一イベント種別で 60 分以内に連続発火した場合は 1 件に集約する

### シーケンス

```mermaid
sequenceDiagram
  autonumber
  actor U as ユーザー
  participant UI as UI
  participant S as サーバー
  Note over S: トリガ(運用イベント発火)
  S->>S: 連続発火を集約判定
  S->>S: システム通知を受信箱に生成
  S->>UI: 受信箱に反映
  UI-->>U: 通知を受領
```

## <span id="FR-165"></span>FR-165: 受信箱の保持と退会時削除

> **この機能要件は「受信箱の保持と退会時削除」を定義します。**

*種別 機能要件 ・ 機能グループ お知らせ ・ 優先度 P0 ・ ステータス ドラフト*

### 要件

お知らせ受信箱はアカウント利用者ごとに保持し、アカウント無効化・退会時に削除されること

### シーケンス

```mermaid
sequenceDiagram
  autonumber
  actor U as ユーザー
  participant UI as UI
  participant S as サーバー
  Note over S: トリガ(アカウント無効化・退会)
  S->>S: 当該利用者の受信箱を削除
  S->>UI: 処理結果を反映
  UI-->>U: 完了を確認
```

## <span id="FR-166"></span>FR-166: 未読件数の取得と更新

> **この機能要件は「未読件数の取得と更新」を定義します。**

*種別 機能要件 ・ 機能グループ お知らせ ・ 優先度 P1 ・ ステータス ドラフト*

### 要件

お知らせの未読件数は管理画面遷移時に取得し、画面滞在中も過大な負荷を生まない範囲で更新できること

### シーケンス

```mermaid
sequenceDiagram
  autonumber
  actor U as ユーザー
  participant UI as UI
  participant S as サーバー
  U->>UI: 管理画面へ遷移
  UI->>S: 未読件数を要求
  S-->>UI: 未読件数
  UI-->>U: 未読件数を更新表示
```

## <span id="FR-167"></span>FR-167: 一括既読操作の監査ログ記録

> **この機能要件は「一括既読操作の監査ログ記録」を定義します。**

*種別 機能要件 ・ 機能グループ お知らせ ・ 優先度 P0 ・ ステータス ドラフト*

### 要件

一括既読操作は監査ログに記録できること(操作者・対象件数・実行日時)

### シーケンス

```mermaid
sequenceDiagram
  autonumber
  actor U as ユーザー
  participant UI as UI
  participant S as サーバー
  U->>UI: 一括既読を操作
  UI->>S: 一括既読化を要求
  S->>S: 操作者・対象件数・実行日時を監査ログに記録
  S-->>UI: 処理結果
  UI-->>U: 既読状態を反映
```
