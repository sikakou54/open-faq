# アカウント・ユーザー・アクセス制御(機能要件)

> **このページは、アカウント・ユーザー・アクセス制御 に関する機能要件を定義します。**

*複数の個別要件を統合したカテゴリ別ページ。 ステータス ドラフト*

| ID | 名称 | 優先度 | 要件 |
|----|----|----|----|
| <span id="FR-001"></span>FR-001 | メール・パスワードでのアカウント新規作成 | P0 | 利用者はメールアドレスとパスワードでアカウントを新規作成できること。アカウント作成は招待とは独立した経路で行う。 |
| <span id="FR-002"></span>FR-002 | 登録時の規約・ポリシー同意取得 | P0 | 登録時に利用規約とプライバシーポリシーへの同意を取得できること。 |
| <span id="FR-003"></span>FR-003 | メールアドレス確認(本人確認メール) | P0 | メールアドレス確認(本人確認メール)ができること。 |
| <span id="FR-004"></span>FR-004 | ログイン・ログアウト・パスワード再設定 | P0 | ログイン、ログアウト、パスワード再設定ができること。 |
| <span id="FR-005"></span>FR-005 | 重要操作の前に本人確認(再認証)を求める | P0 | 重要操作(パスワード変更・退会・課金情報変更・メンバー登録停止削除・月次上限件数変更)の前に本人確認(再認証)を求めること。再認証の有効範囲は当該操作 1 回かつ 15 分以内とする。 |
| <span id="FR-006"></span>FR-006 | パスワードに最低長・複雑性要件を適用 | P0 | パスワードに最低長・複雑性要件(最低 12 文字、英大文字・小文字・数字・記号のうち 3 種類以上)を適用できること。パスワードリセットリンクの有効期限は 1 時間とする。 |
| <span id="FR-007"></span>FR-007 | ログイン失敗回数の制限とロックアウト | P0 | ログイン試行の失敗回数制限ができること。5 回連続失敗で 15 分間ロックし、ロック発動時は該当本人へ通知して救済導線を提供する。 |
| <span id="FR-008"></span>FR-008 | セッションタイムアウトの設定 | P0 | セッションタイムアウトとして無操作タイムアウト 30 分・絶対タイムアウト 12 時間を設けること。セッション失効時は明示的なメッセージを表示し再ログインを誘導する。 |
| <span id="FR-009"></span>FR-009 | 退会できること(即時退会) | P0 | 退会はアカウント単位で本人の操作により即時に成立し、猶予期間は設けないこと(取り消し不可)。退会時は参加プロジェクトから離脱し自分が作成したプロジェクトは削除し、退会後も請求情報のみ閲覧できる。 |
| <span id="FR-010"></span>FR-010 | 利用規約・プライバシーポリシー改定時に再同意を取得 | P0 | 利用規約・プライバシーポリシー改定時に再同意を取得できること。発効 30 日前に予告し、同意期限(発効日 + 14 日)超過後は再同意の割込みと機能制限を行い、非同意の場合は退会の案内を行う。 |
| <span id="FR-011"></span>FR-011 | サスペンション時の挙動 | P0 | アカウントがサスペンションされた場合、配下の全セッションを速やかに無効化し、進行中の長時間処理は完了まで継続すること。再ログイン時はサスペンション中の機能制限(管理画面は課金・退会のみ許可、ウィジェット応答は停止中の旨に切替)に従う。 |
| <span id="FR-012"></span>FR-012 | プロジェクト単位のメンバー招待 | P0 | オーナーおよび当該プロジェクトのメンバーは、自分が割り当てられたプロジェクトでメンバーをメールアドレス指定で招待できること。招待は登録済みのユーザーのみを対象とし、表示名は招待された本人が有効化時に入力する。 |
| <span id="FR-013"></span>FR-013 | オーナーの全権と恒久保持 | P0 | オーナーはプロジェクトを作成した時に当該プロジェクトのオーナーとなり全権とオーナー専有機能を保持すること。オーナーは降格・譲渡の対象外で、1 プロジェクトにつきオーナーは 1 人(作成者)とする。 |
| <span id="FR-014"></span>FR-014 | メンバー権限のプロジェクト単位付与 | P0 | メンバーはプロジェクトごとの割当で表し、割り当てられたメンバーは当該プロジェクト内で同一の権限を持つこと(役割差なし)。個別の権限フラグや権限テンプレートは提供しない。 |
| <span id="FR-015"></span>FR-015 | オーナー専有機能とメンバー開放範囲 | P0 | 支払方法・退会・規約再同意はアカウント(本人)単位の操作とし、課金情報・請求履歴の確認およびプロジェクトの作成・編集・削除はオーナー専有機能とすること。プロジェクト単位の利用上限・無料利用枠はオーナーまたはメンバーが変更できる。 |
| <span id="FR-016"></span>FR-016 | メンバーのデータアクセス範囲 | P0 | メンバーは割り当てられたプロジェクトに属するデータのみにアクセスできること。プロジェクトから外す操作を行ったメンバーが他に有効な割当を持たない場合はアカウントを利用できない状態にする。 |
| <span id="FR-017"></span>FR-017 | プロジェクト作成時のオーナー全権 | P0 | プロジェクト作成時、オーナーは作成したプロジェクトに対して全権を持つこと。管理者割当を別途付与する処理は行わず、作成者であるオーナーを当該プロジェクトのメンバーとして自動追加する。プロジェクト作成時の構成要素には回答可否の関連度・信頼度しきい値(AI しきい値)を含め、これを必須項目として設定する(グローバル既定値などのフォールバック先は持たない)。 |
| <span id="FR-018"></span>FR-018 | 招待リンクからのアカウント有効化 | P0 | 招待されたメンバーは招待リンクから有効化ページに進み、表示名入力・初回パスワード設定・規約同意を行ってアカウントを有効化できること。招待リンクの有効期限は 7 日で、期限切れの場合はオーナーまたはメンバーが再送信する。 |
| <span id="FR-019"></span>FR-019 | 有効化完了時のプロジェクト割当の有効化 | P0 | 招待時に指定されたプロジェクト割当は、招待対象者が有効化(表示名入力・パスワード設定・規約同意)を完了した時点で有効になること。 |
| <span id="FR-020"></span>FR-020 | 招待リンクの再送信と旧リンク失効 | P0 | 招待リンクの再送信はオーナーまたは当該プロジェクトのメンバーのみが行えること。再送信時は旧リンクを失効させ新リンクを発行する。 |
| <span id="FR-021"></span>FR-021 | 有効化前の割当予約とログイン不可 | P0 | 有効化前のメンバーは招待時に指定されたプロジェクト割当を予約状態として保持するが、ログインおよび利用はできないこと(有効化完了時に初めて利用可能となる)。 |
| <span id="FR-022"></span>FR-022 | 有効化ページでの入力項目と同意 | P0 | 招待されたメンバーは有効化ページで表示名(表示名)・初回パスワード・利用規約への同意・プライバシーポリシーへの同意を入力して有効化を完了すること。表示名は本人のみが入力でき、後から個人設定で変更できる。 |
| <span id="FR-023"></span>FR-023 | 有効化完了時の記録と割当の一括処理 | P0 | 有効化の完了時には、表示名・初回パスワード・規約同意(2 件)の記録と招待時に指定されたプロジェクト割当の有効化がまとめて確実に行われること。 |
| <span id="FR-024"></span>FR-024 | メンバー一覧・詳細の参照 | P0 | アカウント利用者は現在開いているプロジェクトに割り当てられたメンバーの一覧と詳細(種別・状態)を参照できること。プロジェクトをまたいだメンバー一覧は提供しない。 |
| <span id="FR-025"></span>FR-025 | メンバーのメールアドレス更新 | P0 | オーナーおよび当該プロジェクトのメンバーは、当該プロジェクトに割り当てられたメンバーのメールアドレスのみを更新できること(表示名の更新は不可)。表示名は本人のみが有効化ページおよび個人設定で変更できる。 |
| <span id="FR-026"></span>FR-026 | メンバーのプロジェクト割当解除 | P0 | 当該プロジェクトのメンバー(オーナーを含む)は、当該プロジェクトのメンバー 1 人の割当について「プロジェクトから外す」操作を行えること。自分自身の割当解除はできず、他メンバーを外す操作には確認を必須とする。 |
| <span id="FR-027"></span>FR-027 | 割当変更の監査記録と通知 | P0 | 割当の追加・解除時には変更前後の差分を記録(監査)し、当該メンバーへお知らせとメールで通知できること。 |
| <span id="FR-028"></span>FR-028 | 別プロジェクトへの追加割当 | P0 | オーナーはメンバーを別プロジェクトへ追加で割り当てる場合、当該プロジェクトに切り替えて招待すること。メンバーは自分が割り当てられたプロジェクトの範囲でのみ招待・編集・削除を行える。 |
| <span id="FR-029"></span>FR-029 | メンバーの利用終了は「プロジェクトから外す」操作に統合 | P0 | メンバーの利用終了は「プロジェクトから外す」操作に統合すること。対象が他プロジェクトに有効な割当を持たない場合はアカウントを利用できない状態にしセッションと未使用の招待を無効化する。 |
| <span id="FR-030"></span>FR-030 | 「プロジェクトから外す」操作は当該プロジェクトの割当のみを解除する | P0 | 「プロジェクトから外す」操作は当該プロジェクトの割当のみを解除すること。他プロジェクトに有効な割当を持つ場合はアカウントと他の割当を維持し、他に有効な割当がない場合はアカウントが利用できなくなる旨を確認時に明示する。 |
| <span id="FR-031"></span>FR-031 | 最終割当解除時の無効化とデータ消去予告 | P0 | 最後の有効なプロジェクト割当を解除した場合は、全ログインセッションの無効化・未使用招待の無効化・90 日経過後のデータ完全消去予定を確認時に明示すること。 |
| <span id="FR-032"></span>FR-032 | 招待リンクには有効期限を設ける | P0 | 招待リンクには有効期限(7 日)を設けること。期限切れ後は招待の再送信で新リンクが発行される。 |
| <span id="FR-033"></span>FR-033 | メンバー数の上限非設定 | P0 | メンバー数に上限を設けない。上限チェック・急増検知およびそれに伴うアラート通知は行わない。 |
| <span id="FR-034"></span>FR-034 | プロジェクト運用通知の通知先 | P1 | プロジェクト運用に関するシステム通知の通知先は、当該プロジェクトのオーナーおよび当該プロジェクトに割り当てられた全メンバーとすること。 |
| <span id="FR-035"></span>FR-035 | オーナー単独でのプロジェクト運営継続 | P0 | 自分が作成したプロジェクトにメンバーが 1 人も登録されていない状態でも、オーナーが単独でプロジェクト運営を継続できること(オーナーは当該プロジェクトの全機能を保持する)。 |
| <span id="FR-036"></span>FR-036 | 同一メールでの重複招待の禁止 | P0 | 同一プロジェクトに既に割当のある(有効・招待中の)ユーザーと同一メールアドレスで重複して招待できないこと。メールアドレスは 1 アカウントに一意とする。 |
| <span id="FR-037"></span>FR-037 | オーナーはプロジェクトを作成・編集・削除 | P0 | オーナーはプロジェクトを作成・編集・削除できること。削除はプロジェクト名の入力確認と本人確認(再認証)を必須とする重要操作として扱う。 |
| <span id="FR-038"></span>FR-038 | 作成時のオーナー全権とメンバー自動追加 | P0 | プロジェクト作成時には作成者であるオーナーが当該プロジェクトに対して全権を持つこと。管理者を別途指定する操作は設けず、作成者であるオーナーを当該プロジェクトのメンバーとして自動追加する。プロジェクト作成時の構成要素には回答可否の関連度・信頼度しきい値(AI しきい値)を含め、これを必須項目として設定する(グローバル既定値などのフォールバック先は持たない)。 |
| <span id="FR-039"></span>FR-039 | プロジェクト削除時の割当・データ取扱い | P0 | プロジェクト削除時は当該プロジェクトのメンバー割当を解除し、他に有効な割当を持たないメンバーのアカウントを利用できない状態にすること。利用停止・解除されたデータは 90 日経過後に完全に消去する。 |
| <span id="FR-040"></span>FR-040 | プロジェクト単位のウィジェット設定管理 | P0 | プロジェクトごとにウィジェット設定(公開キー・埋め込みコード・見た目・プレビュー)を管理できること。公開キーの再発行は再認証と「既存の埋め込みコードが直ちに無効化される」旨の確認を必須とする。 |
| <span id="FR-041"></span>FR-041 | プロジェクトごとに FAQ、質問ログ、未解決質問を分けて管理 | P0 | プロジェクトごとに FAQ、質問ログ、未解決質問を分けて管理できること。 |
| <span id="FR-042"></span>FR-042 | 許可する Web サイトのドメインを複数設定 | P0 | 許可する Web サイトのドメインを複数設定できること。 |
| <span id="FR-043"></span>FR-043 | プロジェクト連絡先メールの設定と確認 | P0 | プロジェクトごとに連絡先メールアドレスを 1 件設定でき、確認メール(有効期限 24 時間)による本人確認を経て有効化できること。確認完了前のアドレスはウィジェット表示に用いない。 |
| <span id="FR-044"></span>FR-044 | ウィジェットでの連絡先メール表示 | P0 | 公開ウィジェットにおいて確認完了済みのプロジェクト連絡先メールを未解決時・質問受付制限時のお問い合わせ先として表示できること。未設定または未確認の場合はメールアドレスを表示しない。 |
| <span id="FR-045"></span>FR-045 | プロジェクト数の上限非設定 | P1 | プロジェクト数に上限を設けない。上限チェック・急増検知およびそれに伴う通知は行わない。 |
| <span id="FR-178"></span>FR-178 | 複数デバイス同時ログインと一括失効 | P0 | 同一アカウントの複数デバイス同時ログインを可能とすること。アカウント停止などの契機で全セッションの強制ログアウトを実行できる。 |
| <span id="FR-179"></span>FR-179 | オーナーに対する操作の禁止 | P0 | オーナーに対する退会・停止・削除・降格・譲渡の操作を不可とすること。1 プロジェクト = 1 オーナー(作成者)のためオーナー不在状態は構造的に発生せず、緊急時はサポート窓口経由で対応する。 |
| <span id="FR-180"></span>FR-180 | 自分自身への操作の禁止 | P0 | アカウント利用者は自分自身への利用状態変更・削除を行えないこと。オーナーが他人を経由して自身を削除する経路も設けない。 |
| <span id="FR-181"></span>FR-181 | メールアドレスのアカウント一意制約 | P1 | メールアドレスは 1 アカウントに一意とすること。1 つのメールアドレスにつき 1 つのアカウントのみを許し、同一ユーザーが複数プロジェクトに異なる立場で所属する場合はこの一意なアカウントを通じて表す。 |
| <span id="FR-182"></span>FR-182 | プロジェクト単位の割当選択 | P0 | プロジェクトごとのメンバー割当を「割当しない / メンバー」から選んで行うこと。プロジェクト内に役割差は持たず、アカウント横断のプリセット役割も持たない。 |
| <span id="FR-183"></span>FR-183 | ダッシュボード閲覧と画面別アクセス制御 | P0 | ダッシュボード(自分の通知・お知らせ受信箱)をオーナーおよびメンバーが閲覧できること。それ以外の画面はユーザー種別とプロジェクト境界の充足を要求する。 |
| <span id="FR-184"></span>FR-184 | 割当変更の次回認可時反映 | P1 | プロジェクト割当の追加・剥奪は、アクティブセッションを即時失効させず、次回の認可チェック時点で反映することで足りること。 |
| <span id="FR-185"></span>FR-185 | 最終割当解除時のアカウント無効化 | P0 | メンバーの最後の有効なプロジェクト割当を解除した場合は、対象アカウントを自動で無効化し、全セッションと未使用の招待を失効すること。割当のないプロジェクトへの直アクセスは拒否する。 |
| <span id="FR-186"></span>FR-186 | 現在のプロジェクトと立場の確認 | P0 | 現在開いているプロジェクトと自分の立場(オーナー / メンバー)を、ヘッダーおよびアカウントメニューで確認できること。独立したロールバッジを全画面へ重複表示しない。 |
| <span id="FR-187"></span>FR-187 | 権限外ナビゲーション・操作の表示制御 | P0 | 権限外の主ナビゲーションと操作は原則として表示しないこと。共同作業上存在を知らせる価値がある項目は読み取り専用で表示し、権限のない画面への直アクセスを拒否する際は説明と戻り先を提示する。 |
| <span id="FR-192"></span>FR-192 | 招待先メールの登録要否チェック | P0 | メンバー招待時、指定されたメールアドレスが登録済みアカウントに一致しない場合はエラーとし、招待を行わないこと(先にアカウント登録を促す)。 |
| <span id="FR-193"></span>FR-193 | 支払方法のユーザー単位の登録・更新 | P0 | 支払方法はアカウント(ユーザー)単位で 1 件登録・更新でき、本人が作成した全プロジェクトの請求に用いること。変更時は本人確認(再認証)を要する。 |
| <span id="FR-194"></span>FR-194 | Myプロジェクト / Joinプロジェクトの一覧 | P0 | ダッシュボードで、自分が作成したプロジェクト(My)と招待されて参加しているプロジェクト(Join)を区分して一覧できること。 |
| <span id="FR-195"></span>FR-195 | オーナー宛のプロジェクト別請求の確認 | P0 | 自分が作成した各プロジェクトの課金状況(プロジェクト別内訳)と合計を、オーナー宛にまとめて確認できること。 |
