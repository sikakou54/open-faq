## <span id="FR-001"></span>FR-001: メール・パスワードでのアカウント新規作成

> **この機能要件は「メール・パスワードでのアカウント新規作成」を定義します。**

*種別 機能要件 ・ 機能グループ アカウント管理 ・ 優先度 P0 ・ ステータス ドラフト*

## <span id="FR-005"></span>FR-005: 重要操作の前に本人確認(再認証)を求める

> **この機能要件は「重要操作の前に本人確認(再認証)を求める」を定義します。**

*種別 機能要件 ・ 機能グループ アカウント管理 ・ 優先度 P0 ・ ステータス ドラフト*

### 要件

重要操作の前に本人確認(再認証)を求めること。
1. 対象の重要操作は次のとおり。
    1. パスワード変更
    2. 退会
    3. 課金情報変更
    4. メンバー登録・停止・削除
    5. 月次上限件数変更
2. 再認証は本人による再度の認証で行う
3. 再認証の有効範囲は **当該操作 1 回かつ 15 分以内**

### シーケンス

```mermaid
sequenceDiagram
  autonumber
  actor U as ユーザー
  participant UI as UI
  participant S as サーバー
  U->>UI: 重要操作を実行
  UI->>S: 操作要求
  S-->>UI: 再認証を要求
  UI-->>U: 再認証を求める
  U->>UI: 本人確認情報を入力
  UI->>S: 再認証付きで操作要求
  alt 再認証成功
    S-->>UI: 操作を受理
    UI-->>U: 結果を表示
  else 再認証失敗
    S-->>UI: 拒否
    UI-->>U: エラーを表示
  end
```

## <span id="FR-006"></span>FR-006: パスワードに最低長・複雑性要件を適用

1. 最低 **12 文字**
2. 英大文字・小文字・数字・記号のうち **3 種類以上**

### シーケンス

```mermaid
sequenceDiagram
  autonumber
  actor U as ユーザー
  participant UI as UI
  participant S as サーバー
  U->>UI: パスワードを入力して送信
  UI->>S: パスワード設定要求
  alt 要件を満たす
    S-->>UI: 設定完了
    UI-->>U: 完了を表示
  else 要件を満たさない
    S-->>UI: 検証エラー
    UI-->>U: 修正を促す
  end
```

```mermaid

  U->>UI: 認証情報を入力して送信

    S-->>UI: 認証エラー
    UI-->>U: 失敗を表示

    S-->>UI: ロックと救済導線

  end
```

```mermaid
  UI-->>U: 退会の即時成立を案内
```

```mermaid
  UI->>S: 再同意状況の確認要求
  S-->>UI: 未同意のため再同意を要求
  UI-->>U: 再同意の割込みを表示
  U->>UI: 再同意して送信
```

## <span id="FR-011"></span>FR-011: サスペンション時の挙動

> **この機能要件は「サスペンション時の挙動」を定義します。**

*種別 機能要件 ・ 機能グループ アカウント管理 ・ 優先度 P0 ・ ステータス ドラフト*

### 要件

アカウントがサスペンションされた場合の挙動を定めること。
1. 配下のアカウント利用者の全セッションを速やかに無効化する
2. 進行中の長時間処理(大量インポート等)は**完了まで継続**し中断しない。完了後の次回操作で再認証を要求する
3. 再ログイン時、サスペンション状態の場合は停止時のアクセス制限ルール(下記)に従う

「停止時のアクセス制限ルール」とは、サスペンション中の機能制限([FR-098](03_usage-fr.md#FR-098))に従い、次のとおり利用を制限することをいう。
1. 管理画面の操作は、オーナーによる課金(請求情報・支払方法の更新)と退会のみを許可し、それ以外の操作は拒否する
2. ウィジェットの新規応答は行わず、機能停止の旨を示す応答に切り替える
3. 制限はサスペンション中(停止中)に適用する(退会済みのアカウントは [FR-140](06_security-fr.md#FR-140) の請求限定アクセスに従う)

### シーケンス

```mermaid
sequenceDiagram
  autonumber
  actor U as ユーザー
  participant UI as UI
  participant S as サーバー
  Note over S: サスペンションを契機に起動
  S->>S: 配下の全セッションを無効化
  U->>UI: 再ログインを試行
  UI->>S: ログイン要求
  S-->>UI: 停止時のアクセス制限を適用
  UI-->>U: 制限内容を表示
```

## <span id="FR-012"></span>FR-012: プロジェクト単位のメンバー招待

> **この機能要件は「プロジェクト単位のメンバー招待」を定義します。**

*種別 機能要件 ・ 機能グループ ユーザー管理(オーナー + メンバー) ・ 優先度 P0 ・ ステータス ドラフト*

### 要件

オーナーおよび当該プロジェクトのメンバーは、自分が割り当てられたプロジェクトでメンバーを招待できること。
1. 招待はメールアドレスを指定して行う(プロジェクト内の役割差はない)
2. 招待時に氏名(表示名)は入力しない(個人情報保護の原則により、他人の氏名を招待者が任意に登録することを禁止する)。氏名は招待された本人が有効化時に入力する([FR-022](#FR-022))
3. メンバーの管理は常にプロジェクト単位で行う
4. 同一の人を複数プロジェクトに参加させる場合は各プロジェクトで個別に招待する
5. 招待は登録済みのユーザーのみを対象とし、未登録(アカウントのない)メールアドレスは招待できない(先にアカウント登録を促す)。既に登録済みのユーザーには当該プロジェクト分の割当のみを追加する

### シーケンス

```mermaid
sequenceDiagram
  autonumber
  actor U as ユーザー
  participant UI as UI
  participant S as サーバー
  U->>UI: 招待先メールアドレスを入力して送信
  UI->>S: メンバー招待要求
  S-->>UI: 招待結果(招待送信/エラー)
  UI-->>U: 結果を表示
```

## <span id="FR-013"></span>FR-013: オーナーの全権と恒久保持

> **この機能要件は「オーナーの全権と恒久保持」を定義します。**

*種別 機能要件 ・ 機能グループ ユーザー管理(オーナー + メンバー) ・ 優先度 P0 ・ ステータス ドラフト*

### 要件

オーナーはプロジェクトを作成した時に当該プロジェクトのオーナーとなり、当該プロジェクトに対する全権とオーナー専有機能を保持すること。オーナーは作成者であることに基づく全権を持ち、別途の割当付与なしに自プロジェクトを操作できる(認可権威はオーナー権限)。加えて、作成した各プロジェクトに自身のメンバー行を自動保持する(一覧表示・担当割当・通知宛先の網羅用で、専有判定には用いない)。オーナーは降格・譲渡の対象外とする(MVP)。1 プロジェクトにつきオーナーは 1 人(作成者)とし、1 人のユーザーは複数プロジェクトのオーナーになれ、同時に他者が作成したプロジェクトのメンバーにもなれる

### シーケンス

```mermaid
sequenceDiagram
  autonumber
  actor U as ユーザー
  participant UI as UI
  participant S as サーバー
  U->>UI: プロジェクト作成を送信
  UI->>S: プロジェクト作成要求
  S->>S: プロジェクト作成者を当該プロジェクトのオーナーとして確定し全権を付与
  S-->>UI: 作成結果
  UI-->>U: オーナーとして利用開始
```

## <span id="FR-014"></span>FR-014: メンバー権限のプロジェクト単位付与

### 要件

メンバーはプロジェクトごとの割当で表し、割り当てられたメンバーは当該プロジェクト内で同一の権限を持つこと(役割差なし)。当該プロジェクトのメンバーは、当該プロジェクトの FAQ 管理・未解決質問管理・ログ参照・メンバー管理(当該プロジェクト範囲の招待・編集・割当解除)・上限 / アラート設定・お知らせ受信箱・ウィジェット設定を行える。個別の権限フラグや権限テンプレートは提供しない

### シーケンス

```mermaid
sequenceDiagram
  autonumber
  actor U as ユーザー
  participant UI as UI
  participant S as サーバー
  U->>UI: プロジェクト内の操作を実行
  UI->>S: 操作要求
  S->>S: 当該プロジェクトの割当を確認
  alt 割当あり
    S-->>UI: 操作を受理
    UI-->>U: 結果を表示
  else 割当なし
    S-->>UI: 拒否
    UI-->>U: 権限不足を表示
  end
```

## <span id="FR-016"></span>FR-016: メンバーのデータアクセス範囲

> **この機能要件は「メンバーのデータアクセス範囲」を定義します。**

*種別 機能要件 ・ 機能グループ ユーザー管理(オーナー + メンバー) ・ 優先度 P0 ・ ステータス ドラフト*

### 要件

メンバーは割り当てられたプロジェクトに属するデータ(FAQ・要対応の質問・質問ログ等)のみにアクセスできること。プロジェクトから外す操作を行ったメンバーが他に有効なプロジェクト割当を持たない場合は、そのアカウントを利用できない状態にする。オーナーは自分が作成したプロジェクトを割当の有無に関わらず閲覧・操作でき、他者が作成したプロジェクトには招待されてメンバーとなった範囲でのみアクセスできる

### シーケンス

```mermaid
sequenceDiagram
  autonumber
  actor U as ユーザー
  participant UI as UI
  participant S as サーバー
  U->>UI: プロジェクトのデータを参照
  UI->>S: データ取得要求
  S->>S: 割当の範囲を確認
  S-->>UI: 割当範囲のデータ(範囲外は拒否)
  UI-->>U: 結果を表示
```

## <span id="FR-017"></span>FR-017: プロジェクト作成時のオーナー全権

> **この機能要件は「プロジェクト作成時のオーナー全権」を定義します。**
