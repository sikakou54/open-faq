## <span id="FR-137"></span>FR-137: 退会の即時実行と厳格な確認

> **この機能要件は「退会の即時実行と厳格な確認」を定義します。**

*種別 機能要件 ・ 機能グループ プライバシー・データ管理 ・ 優先度 P0 ・ ステータス ドラフト*

### 要件

オーナーによる退会は即時に実行し、猶予期間は設けないこと。実行前に厳格な確認を必須とする。
1. 退会の影響(サービスの即時停止・運用データの削除・以後は請求情報の閲覧のみ可能になること)を明示する
2. 取り消しできない操作である旨を警告し、氏名(プロフィール名)の入力一致と本人確認(再認証)を必須とする
3. 確認の完了をもってアカウントを即時に退会済みへ移行する(申請を保留する猶予状態は持たない)

### シーケンス

```mermaid
sequenceDiagram
  autonumber
  actor U as ユーザー
  participant UI as UI
  participant S as サーバー
  U->>UI: 退会を開始
  UI-->>U: 退会の影響と不可逆である旨を提示
  U->>UI: 氏名(プロフィール名)を入力し本人確認に応じる
  UI->>S: 厳格な確認を伴う退会要求
  alt 確認成立
    S->>S: アカウントを即時に退会済みへ移行
    S-->>UI: 退会完了(以後は請求閲覧のみ)
  else 確認不成立
    S-->>UI: 退会を実行せず差し戻し
  end
  UI-->>U: 結果を表示
```

## <span id="FR-138"></span>FR-138: 退会時のデータ削除と請求データの保持

> **この機能要件は「退会時のデータ削除と請求データの保持」を定義します。**

*種別 機能要件 ・ 機能グループ プライバシー・データ管理 ・ 優先度 P0 ・ ステータス ドラフト*

### 要件

退会時に運用データを速やかに削除し、アカウント・請求データは法定保持期間にわたり保持すること。
1. 退会と同時に、FAQ・プロジェクト・許可ドメイン・質問ログ・未解決質問・利用量・通知・お知らせ受信箱などの運用データを速やかに削除する
2. アカウント・請求関連データ(アカウント・請求書・サブスクリプション・支払方法・監査記録)は法定保持期間(7 年、[RULE-022](../01_business_requirement/08_rule.md#RULE-022))にわたり保持し、保持期間の経過後に削除する
3. 削除は不可逆とし、削除内容を監査記録として残す

### シーケンス

```mermaid
sequenceDiagram
  autonumber
  actor U as ユーザー
  participant UI as UI
  participant S as サーバー
  U->>UI: 退会を実行
  UI->>S: 退会要求
  S->>S: 運用データを速やかに削除し、アカウント・請求データを保持
  S-->>UI: 退会完了・保持対象を案内
  Note over S: トリガ 保持期間(7 年)の経過
  S->>S: アカウント・請求データを削除し記録
  UI-->>U: 結果を表示
```

## <span id="FR-139"></span>FR-139: 利用規約・プライバシーポリシー改定時に再同意を取得

利用規約・プライバシーポリシー改定時に再同意を取得できること。
1. 利用規約とプライバシーポリシーは個別の文書として版管理し、それぞれ同意を取得する
2. 改定された文書のみを再同意の対象とする

    S-->>UI: 登録完了

    S-->>UI: 通常画面へ
  end

## <span id="FR-140"></span>FR-140: 退会後の請求限定アクセス

> **この機能要件は「退会後の請求限定アクセス」を定義します。**

*種別 機能要件 ・ 機能グループ プライバシー・データ管理 ・ 優先度 P0 ・ ステータス ドラフト*

### 要件

退会後も、アカウント・請求データの保持期間(7 年)中はオーナーがログインでき、請求情報の閲覧のみ行えること。
1. ログインと請求情報(請求履歴・請求書・請求状態等)の閲覧のみを許可する
2. FAQ・プロジェクト・ウィジェット・メンバー管理・設定変更などの操作および新規書込みはすべて不可とする(機能停止の旨を示す応答に切り替える)
3. アカウント・請求データの保持期間の経過後はログインできない

### シーケンス

```mermaid
sequenceDiagram
  autonumber
  actor U as ユーザー
  participant UI as UI
  participant S as サーバー
  U->>UI: 退会後にログインして操作
  UI->>S: 操作要求
  alt 請求情報の閲覧
    S-->>UI: 請求情報を返す
  else その他の操作
    S-->>UI: 退会済みのため機能停止として拒否
  end
  UI-->>U: 結果を表示
```

## <span id="FR-147"></span>FR-147: Bot 対策(レート制限・ロックアウト等)を必要に応じて実施

> **この機能要件は「Bot 対策(レート制限・ロックアウト等)を必要に応じて実施」を定義します。**

*種別 機能要件 ・ 機能グループ セキュリティ ・ 優先度 P1 ・ ステータス ドラフト*

### 要件

Bot・不正自動アクセス対策を、次の発火条件で実施できること。「必要に応じて」とは次の各契機をいう。
1. ログイン連続失敗時: 同一アカウントで連続してログインに失敗したとき、アカウントロックにより自動試行を抑止する(連続失敗のしきい値・ロックは [RULE-001](../01_business_requirement/08_rule.md#RULE-001))
2. 未認証の再送・送信時: 未認証状態で行われる送信操作(パスワード再設定メール・連絡先確認メールの再送、ウィジェットからの問い合わせ送信 等)で、レート制限により短時間の連続要求を抑止する
3. 不審なリクエスト検知時: 大量アクセス・未許可ドメイン等の不審なリクエスト(FR-148)を検知したとき、追加の制限・記録を行う

### シーケンス

```mermaid
sequenceDiagram
  autonumber
  actor U as ユーザー
  participant UI as UI
  participant S as サーバー
  U->>UI: 送信操作
  UI->>S: 要求
  alt 正常な要求
    S-->>UI: 処理を継続
  else しきい値超過・不審と判定
    S-->>UI: 要求を制限・記録(レート制限 / アカウントロック)
    UI-->>U: 時間をおいて再試行する旨を表示
  end
  UI-->>U: 結果を表示
```

## <span id="FR-150"></span>FR-150: ウィジェット公開キーのローテーション機能を利用者操作で提供

*種別 機能要件 ・ 機能グループ セキュリティ ・ 優先度 P0 ・ ステータス ドラフト*

### シーケンス

```mermaid
sequenceDiagram
  autonumber

  participant UI as UI
  participant S as サーバー
  U->>UI: キーのローテーションを指示
  UI->>S: 新キー発行を要求
  S-->>UI: 新キーを返し旧キーを失効予告
  UI-->>U: 新キーと猶予を表示
  Note over S: トリガ 猶予期間の経過
  S->>S: 旧キーを完全失効とし記録
```

## <span id="FR-192"></span>FR-192: 保持期間超過データの定期削除

> **この機能要件は「保持期間超過データの定期削除」を定義します。**

*種別 機能要件 ・ 機能グループ セキュリティ ・ 優先度 P0 ・ ステータス ドラフト*

### 要件

保持期間を超過した運用データを定期的に検出し、対象に応じた方式で削除すること。
1. 対象は質問ログ・未解決質問・通知ログ・お知らせ受信箱とする
2. 各データを定められた保持期間(1 年)の経過後に削除対象とする
3. 後から確定削除する仕組みを持つ対象(質問ログ・未解決質問)は削除予定として扱い、追記専用の対象(通知ログ・お知らせ受信箱)はその場で確定削除する
4. 削除を実行した結果を記録する

### シーケンス

```mermaid
sequenceDiagram
  autonumber
  participant SCH as スケジューラ
  participant S as サーバー
  SCH->>S: 定期起動(日次)
  S->>S: 保持期間を超過したデータを抽出
  alt 対象あり
    S->>S: 対象に応じた方式で削除し結果を記録
  else 対象なし
    S->>S: 何もせず正常終了
  end
```
