# FAQ・AI 回答・未解決質問・処理エラー(機能要件)

> **このページは、FAQ・AI 回答・未解決質問・処理エラー に関する機能要件を定義します。**

*複数の個別要件を統合したカテゴリ別ページ。 ステータス ドラフト*

| ID | 名称 | 優先度 | 要件 |
|----|----|----|----|
| <span id="FR-046"></span>FR-046 | FAQ の登録・編集・削除 | P0 | FAQ を登録、編集、削除できること。 |
| <span id="FR-047"></span>FR-047 | 管理画面からの削除の不可逆性 | P0 | FAQ・プロジェクト・メンバーユーザー・お知らせ・アカウントの削除は管理画面から復元できないこと。誤削除時はサポート窓口経由でのみ救済される。 |
| <span id="FR-048"></span>FR-048 | FAQ に質問と回答を登録 | P0 | FAQ に質問と回答を登録できること。 |
| <span id="FR-049"></span>FR-049 | FAQ を下書き、公開中、非公開の状態で管理 | P0 | FAQ を下書き、公開中、非公開の状態で管理できること。 |
| <span id="FR-050"></span>FR-050 | FAQ をカテゴリで整理 | P1 | FAQ をカテゴリで整理できること。 |
| <span id="FR-051"></span>FR-051 | FAQ の検索・並び替え・絞り込み | P1 | FAQ の検索、並び替え、絞り込みができること。 |
| <span id="FR-052"></span>FR-052 | 公開前の内容確認 | P0 | FAQ の公開前にアカウント利用者が内容を確認できること。 |
| <span id="FR-053"></span>FR-053 | FAQ 件数・文字数の上限管理 | P0 | FAQ 件数および 1 件あたりの文字数の上限は全プロジェクト共通の基準とし、極端に大きい場合は警告または登録拒否できること。件数上限は段階通知(80% / 100%)で警告し 120% で新規登録拒否、文字数超過は登録時にエラーとする。 |
| <span id="FR-054"></span>FR-054 | 同時編集の競合検出 | P1 | FAQ 更新時、同時編集による競合を検出できること。 |
| <span id="FR-055"></span>FR-055 | FAQ 登録元の保持 | P1 | FAQ の登録元(未解決質問からの登録か手動登録か)を保持できること。 |
| <span id="FR-056"></span>FR-056 | 公開中 FAQ のみを根拠とする回答 | P0 | ウィジェット利用者の質問に対し、公開中の登録済み FAQ のみを根拠として回答できること。 |
| <span id="FR-057"></span>FR-057 | 根拠なき独自生成の抑止 | P0 | FAQ に根拠がない内容を AI が独自に作成しないこと。 |
| <span id="FR-058"></span>FR-058 | FAQ 表現の整理にとどめる加工 | P0 | AI は FAQ の内容を要約・言い換え・整理できるが、新しい事実・数値・固有名詞・手順を追加しないこと。 |
| <span id="FR-059"></span>FR-059 | 参照 FAQ の記録と提示 | P0 | 回答に利用した FAQ を記録し、ウィジェット利用者にも参照 FAQ を提示できること。 |
| <span id="FR-060"></span>FR-060 | FAQ 矛盾時の未解決扱い | P0 | FAQ 同士に矛盾がある場合、断定回答せず未解決として扱えること。 |
| <span id="FR-061"></span>FR-061 | プロジェクト単位のしきい値調整 | P1 | 回答可否の判定に使う信頼度・関連度のしきい値を、グローバル既定値とプロジェクト設定値で管理できること。対象プロジェクトの設定値がある場合はそれを使用し、未登録の場合はグローバル既定値を使用する。設定単位はプロジェクト単位のみとし、設定値は保存と同時に有効化される。 |
| <span id="FR-062"></span>FR-062 | 問い合わせ ID 付与と回答不可案内 | P0 | FAQ 登録済みデータでは回答できなかった未解決時に、管理用の問い合わせ ID を付与して未解決質問を登録し、回答できなかった旨と確認済みプロジェクト連絡先メールを会話欄に表示すること。問い合わせ ID は未解決(回答不可)時に提示する識別子であり、サーバー内部エラー時に提示するエラー識別子(FR-084)とは発生文脈が異なる。 |
| <span id="FR-063"></span>FR-063 | 処理エラー時のエラー表示 | P0 | 処理エラーの場合は、未解決質問登録ではなくエラー表示を行えること。 |
| <span id="FR-064"></span>FR-064 | プロンプト注入耐性 | P0 | ウィジェット利用者の入力により AI の動作方針(FAQ 限定回答方針)が変更されないこと。代表的な攻撃パターンを含む回帰テストを AI モデル更新時およびプロンプト変更時に実行する。 |
| <span id="FR-065"></span>FR-065 | モデル切替時の品質回帰とロールバック | P1 | 利用する AI モデルや基盤の変更時に、動作確認・切替・品質回帰確認・必要時のロールバックを行えること。標準テストデータセットは FAQ × 想定質問のペア 50 組以上とする。 |
| <span id="FR-066"></span>FR-066 | 回答出力前の検査 | P1 | AI 回答の出力前に検査を行えること。参照 FAQ 外の固有名詞・数値・手順を検出した場合は未解決登録に倒し、個人情報を検出した場合は所定の形式でマスキングして返却し検出種別を記録する。 |
| <span id="FR-067"></span>FR-067 | 回答不可質問の未解決登録 | P0 | FAQ 登録済みデータでは回答できなかった質問を未解決質問として登録できること。 |
| <span id="FR-068"></span>FR-068 | 未解決申告による未解決登録 | P0 | ウィジェット利用者が「解決しなかった」を選択した場合、未解決質問として登録できること。 |
| <span id="FR-069"></span>FR-069 | 未解決質問の記録項目 | P0 | 未解決質問には、質問・未解決理由・発生日時・関連プロジェクト・関連質問ログを記録できること。 |
| <span id="FR-070"></span>FR-070 | 未解決質問への問い合わせ ID 付与 | P0 | 未解決質問には問い合わせ ID を付与できること。 |
| <span id="FR-071"></span>FR-071 | 状況の 2 区分管理 | P0 | 未解決質問の状況は「対応中 / 対応済み」の 2 区分とし、保持すること。 |
| <span id="FR-072"></span>FR-072 | 登録時の初期状況「対応中」 | P0 | 未解決質問の登録時に状況を「対応中」とすること。状況変更は詳細画面からの手動操作のみで行い、FAQ 下書き保存・FAQ 公開成功とは連動しない。 |
| <span id="FR-073"></span>FR-073 | 状況の一覧・詳細表示 | P0 | 未解決質問の現在の状況(対応中 / 対応済み)を一覧画面・詳細画面で確認できること。状況は詳細画面からの手動切替のみを表し、FAQ 操作から派生または更新しない。 |
| <span id="FR-074"></span>FR-074 | 状況変更の手動操作限定 | P0 | 未解決質問の状況変更は詳細画面からの手動操作のみとすること。対応中 ↔ 対応済み の双方向切替を許可し、再オープン回数に制限を設けない。 |
| <span id="FR-075"></span>FR-075 | 未解決質問からの FAQ 登録開始 | P0 | アカウント利用者は未解決質問から FAQ 登録を開始できること。 |
| <span id="FR-076"></span>FR-076 | 質問文の質問欄への初期反映 | P0 | 未解決質問の質問文を、新しい FAQ の質問欄へ初期反映できること。 |
| <span id="FR-077"></span>FR-077 | 回答の手動入力と自動反映の禁止 | P1 | FAQ 回答はアカウント利用者が入力・編集でき、未解決質問の質問文以外を回答欄へ自動取得または初期反映しないこと。 |
| <span id="FR-078"></span>FR-078 | 登録前の内容確認・編集 | P0 | FAQ 登録前に、アカウント利用者が質問・回答・公開状態を確認・編集できること。 |
| <span id="FR-079"></span>FR-079 | FAQ 操作と状況の非連動 | P0 | FAQ の下書き初回保存・公開は、未解決質問の状況を変更しないこと。未解決質問の状況は詳細画面からの手動操作のみで対応中 ↔ 対応済みを切替する。 |
| <span id="FR-080"></span>FR-080 | 登録先 FAQ の参照 | P1 | 未解決質問から登録先 FAQ を参照できること。 |
| <span id="FR-081"></span>FR-081 | 処理エラーの検知とエラー応答 | P0 | 処理エラー(通信障害・上流障害・入力不備・認可エラー等)を検知し、ウィジェット利用者向けにエラー表示と状況に応じた応答ができること。連絡先メールが設定済みであれば代替の問い合わせ先として案内し、AI 推論のタイムアウト・AI プロバイダ障害時は定型の案内文へフォールバックする。 |
| <span id="FR-082"></span>FR-082 | 処理エラーと未解決登録の区別 | P0 | 処理エラーは未解決登録分岐(FAQ なし・信頼度不足・FAQ 矛盾)と区別し、未解決質問として自動登録しないこと。 |
| <span id="FR-083"></span>FR-083 | 再試行案内の表示 | P0 | 自動再試行で回復しなかった場合に再試行案内を表示できること。ウィジェット利用者による再試行が妥当な場合に表示し、再試行操作の連打防止を備える。 |
| <span id="FR-084"></span>FR-084 | 内部エラーの運用記録と識別子提示 | P0 | サーバー内部起因のエラーは運用確認できるように記録すること。エラー識別子をウィジェット利用者にも提示し、サポート問い合わせで紐付け可能にする。エラー識別子はサーバー内部エラー時に提示する識別子であり、未解決(回答不可)時に提示する問い合わせ ID(FR-062)とは発生文脈が異なる(両者は排他的な場面で提示される)。 |
| <span id="FR-085"></span>FR-085 | エラー記録の機密を保護 | P0 | エラー記録の機密を保護すること。個人情報・認証トークン・パスワードハッシュ・カード情報を含めず、ウィジェット利用者の入力はそのまま全文保存せず最小限の記録にとどめる。 |
| <span id="FR-188"></span>FR-188 | AI 推論のタイムアウト上限 | P0 | AI 推論のタイムアウト上限は 8 秒とし、超過時は処理エラーとして扱うこと。 |
| <span id="FR-189"></span>FR-189 | しきい値変更の推論反映とフォールバック | P0 | AI しきい値の変更を短時間で推論動作へ反映できること。プロジェクト設定値を取得できない場合はグローバル既定値(信頼度 0.60 / 関連度 0.50)で動作を継続し、フォールバック中はアラート通知する。 |
| <span id="FR-196"></span>FR-196 | ウィジェット回答フィードバックの送信と必須化 | P1 | ウィジェット利用者は AI 回答(回答可能)に対して「役に立った(解決)/ 役に立たなかった(未解決)」のフィードバックを送信できること。フィードバックは次の質問送信の前提として必須とし、未選択のままでは次の質問を送信できない。 |
| <span id="FR-197"></span>FR-197 | 未解決フィードバック時の未解決質問記録 | P1 | 「役に立たなかった(未解決)」のフィードバックを受けた場合、当該質問を未解決質問として記録すること(理由は利用者による未解決)。連絡先が設定済みの場合は連絡先案内を示す。「役に立った(解決)」の場合は未解決質問を記録しない。 |
| <span id="FR-198"></span>FR-198 | 候補 FAQ の全文検索と関連度算定 | P0 | AI 回答時、公開中の FAQ を全文検索で候補抽出し、一致スコア(関連度)の上位所定件数を回答根拠候補とすること。関連度は候補 FAQ の検索一致度として算定し、回答可否判定(関連度・信頼度のしきい値)に用いる。 |
