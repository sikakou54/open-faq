# セキュリティ・プライバシー(業務要件)

> **このページは、セキュリティ・プライバシー に関する業務要件を定義します。**

*複数の業務要件を一覧化したカテゴリ別ページ。掲載要件は全件 種別 業務要件 ・ 優先度 P0 ・ ステータス ドラフト。トレースは下流(UC 等)からの逆引きで辿る。*

<td>プライバシー・データ管理</td>

<td>プライバシー・データ管理</td>
<td>利用規約・プライバシーポリシー改定時に、アカウント利用者へ改定内容を確実に把握させ法的同意を担保するため、改定された文書のみを対象に再同意を取得する。両文書は個別の文書として扱い、文書ごとに同意・版管理する方針とする。</td>
</tr>
<tr>
<td><span id="BR-098"></span>BR-098</td>
<td>データ保存リージョン</td>
<td>プライバシー・データ管理</td>
<td>契約データの保存リージョン(例:日本国内)を選択またはサービス側で明示する。</td>
</tr>
<tr>
<td><span id="BR-099"></span>BR-099</td>
<td>Cookie 同意取得</td>
<td>プライバシー・データ管理</td>
<td>ウィジェット利用者のプライバシーを尊重し法令を順守するため、Cookie 利用の同意を取得する。必須 Cookie 以外はオプトインとし、第三者トラッキング系は同意取得後にのみ有効化する方針とする。</td>
</tr>
<tr>
<td><span id="BR-100"></span>BR-100</td>
<td>機密データ保護</td>
<td>セキュリティ</td>
<td>情報漏洩から利用者の資産と信頼を守るため、通信を暗号化し、機密データ(パスワード、トークン、課金情報等)は暗号化または安全なハッシュで保存する方針とする。</td>
</tr>
<tr>
<td><span id="BR-101"></span>BR-101</td>
<td>アクセス制御・データ分離</td>
<td>セキュリティ</td>
<td>他契約のデータへ到達できないようにし契約間の情報漏洩を防ぐため、最小権限の原則に従い、オーナー境界でデータを分離し、ユーザー種別ごとに認可する方針とする。</td>
</tr>
<tr>
<td><span id="BR-102"></span>BR-102</td>
<td>濫用・過負荷防御</td>
<td>セキュリティ</td>
<td>サービスの可用性と公平な利用を守るため、API・ウィジェットへの濫用・過負荷を防御し、不審なリクエストを検知・記録する。</td>
</tr>
<tr>
<td><span id="BR-103"></span>BR-103</td>
<td>ウィジェット配信元制限</td>
<td>セキュリティ</td>
<td>公開キーの不正流用や成りすまし配信を防ぐため、許可した埋め込み元ドメイン以外からのウィジェット読み込みを拒否する方針とする。</td>
</tr>
<tr>
<td><span id="BR-104"></span>BR-104</td>
<td>操作監査</td>
<td>セキュリティ</td>
<td>不正・誤操作の事後追跡と説明責任を果たすため、アカウント利用者によるユーザー管理操作を、改ざん検知可能な形で監査ログとして保持する。</td>
</tr>
<tr>
<td><span id="BR-105"></span>BR-105</td>
<td>公開キー管理</td>
<td>セキュリティ</td>
<td>公開キー漏洩などの不正利用に備え、アカウント利用者が必要時にウィジェット公開キーをローテーション(再発行)できるようにする。埋め込み済みウィジェットの突然停止を避けるため、再発行時は旧キーを猶予付きで失効させ、猶予中は新旧両キーでの認証を許可する。公開キー自体には有効期限を設けない方針とする。</td>
</tr>
<tr>
<td><span id="BR-106"></span>BR-106</td>
<td>不正利用検知・自動ブロック</td>
<td>セキュリティ</td>
<td>サービスの安定稼働と健全な利用を守るため、不正利用(大量送信・Bot 疑い・規約違反)を検知して自動でブロックし、誤検知時の救済として管理者ユーザーへ即時通知し申し立て窓口を案内する。契約ごとの運用実態に合わせ、判定ルール・しきい値は契約別に上書きできる方針とする。</td>
</tr>
