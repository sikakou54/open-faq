# 利用量・課金・ダッシュボード・運用(業務要件)

> **このページは、利用量・課金・ダッシュボード・運用 に関する業務要件を定義します。**

*複数の業務要件を一覧化したカテゴリ別ページ。掲載要件は全件 種別 業務要件 ・ 優先度 P0 ・ ステータス ドラフト。トレースは下流(UC 等)からの逆引きで辿る。*

<td>利用量・課金</td>

</tr>

<td>利用量・課金</td>
<td>想定外の超過課金から契約者を守るため、質問数上限がオンのプロジェクトが上限に到達したら、支払方法の有無に関わらず当該プロジェクトのウィジェット新規質問受付を停止する。契約自体は有効のまま維持し、エンドユーザーには安全な定型文を示す方針とする。停止は翌月のリセット・上限引き上げ・上限オフのいずれかで即時に復帰させる。</td>
</tr>

<td>上限アラート通知</td>
<td>利用量・課金</td>
<td>上限到達前に契約者が利用量へ気づき対処できるようにするため、選択された質問数上限アラート閾値へ当月初回到達した時点で、オーナーと当該プロジェクトの有効なメンバーへメールで通知する。閾値を一つも選択しない場合はアラート通知なしを許容する。</td>
</tr>
<tr>
<td><span id="BR-064"></span>BR-064</td>
<td>支払方法ゲート</td>
<td>利用量・課金</td>
<td>無料枠内での試用開始を妨げず、かつ未払い利用を防ぐため、支払方法はクレジットカード払いとし、新規登録時点での登録は必須としない。無料枠判定はプロジェクト単位、支払方法は契約単位とする方針とする。無料枠超過が近づいた段階で事前告知し、支払方法未登録のまま超過した場合は当該プロジェクトのウィジェット新規質問受付のみを停止して支払方法登録へ誘導する(契約サスペンションは行わず管理画面は継続利用可能)。支払方法登録後は即時再開する。</td>
</tr>
<tr>

<td>支払い失敗時のサスペンション</td>
<td>利用量・課金</td>
<td>支払い回復の機会を与えつつ未払い利用の継続を防ぐため、決済失敗確定後は一定の猶予期間を設け、その間も管理者ユーザーが支払方法を更新できるようにする。猶予中に再決済が成功すれば即時に解除し、猶予経過後は契約をサスペンションへ移行してウィジェット応答を停止する。サスペンション中の管理画面アクセスは課金・退会のみに限定する方針とする。</td>
</tr>
<tr>
<td><span id="BR-066"></span>BR-066</td>
<td>請求内訳の確認</td>
<td>利用量・課金</td>
<td>オーナーが請求内容を把握し費用をコントロールできるようにするため、支払方法・請求状態・請求履歴・請求見込みやプロジェクト別の課金内訳など、契約全体と各プロジェクトの請求・利用量を確認できるようにする。利用契約の解約申請はオーナー専有とする。</td>
</tr>

<td>レート制限</td>
<td>利用量・課金</td>
<td>DDoS / Bot / 暴走による負荷や悪用からサービスを守るため、各機能にレート制限を適用する。レート制限は課金目的ではなくセキュリティ目的のため契約単位で管理し、月次上限件数・無料枠のようなプロジェクト単位化の対象外とする方針とする。</td>
</tr>

<td><span id="BR-068"></span>BR-068</td>
<td>課金プロバイダ通知の受信</td>
<td>利用量・課金</td>
<td>決済状態と契約状態の整合性を保つため、課金プロバイダからの通知を確実に受信して取り込む。なりすましや重複処理を防ぎ、処理失敗時は気づけるようにし、必要に応じて再処理できるようにする。</td>
</tr>

<td><span id="BR-069"></span>BR-069</td>
<td>利用状況の把握</td>
<td>管理ダッシュボード</td>
<td>アカウント利用者が日次運用の中で状況を素早く把握し対応案件を俯瞰できるよう、対象範囲の主要指標を一目で確認できるようにする。</td>
</tr>
<tr>
<td><span id="BR-070"></span>BR-070</td>
<td>要対応質問の状況確認</td>
<td>管理ダッシュボード</td>
<td>要対応の質問の対応漏れを防ぐため、各質問の現在の対応状況を確認できるようにする。</td>
</tr>
<tr>
<td><span id="BR-071"></span>BR-071</td>
<td>FAQ 改善の起点把握</td>
<td>管理ダッシュボード</td>
<td>FAQ の改善を継続的に進められるよう、よく聞かれる質問や未解決の傾向から改善ターゲット候補を見つけられるようにする。</td>
</tr>
<tr>
<td><span id="BR-072"></span>BR-072</td>
<td>期間絞り込み</td>
<td>管理ダッシュボード</td>
<td>知りたい期間の状況を把握できるよう、集計対象期間を指定して利用状況を絞り込めるようにする。</td>
</tr>
<tr>
<td><span id="BR-073"></span>BR-073</td>
<td>通知到達性の確認</td>
<td>管理ダッシュボード</td>
<td>重要な通知が届いていない事態に気づけるよう、通知の到達性の異常を確認できるようにする。</td>
</tr>
<tr>
<td><span id="BR-074"></span>BR-074</td>
<td>管理範囲の切替とユーザー種別別表示</td>
<td>管理ダッシュボード</td>
<td>権限に応じた範囲だけを安全に把握できるよう、ユーザー種別に応じて表示範囲を切り替える。<ul>
<li><strong>オーナー：</strong>契約全体の利用状況と各プロジェクトの概要を行き来できる。</li>
<li><strong>メンバー：</strong>参加しているプロジェクトのみを確認でき、契約全体の範囲は表示しない。</li>
</ul></td>
</tr>
<tr>
<td><span id="BR-075"></span>BR-075</td>
<td>表示の一貫性・誤認防止</td>
<td>管理ダッシュボード</td>
<td>利用者がダッシュボードの値を誤って解釈しないよう、各画面の目的・集計時点・比較値・各種データ状態の表示を一貫させ、誤認を防ぐ。</td>
</tr>
<tr>
<td><span id="BR-124"></span>BR-124</td>
<td>同時編集の保護</td>
<td>UX 細部・データ運用</td>
<td>複数のアカウント利用者が同じ FAQ を同時に編集した際の後勝ち上書きによる編集内容の喪失を防ぐ。</td>
</tr>
<tr>
<td><span id="BR-125"></span>BR-125</td>
<td>編集中のデータ保護</td>
<td>UX 細部・データ運用</td>
<td>不慮の離脱による入力喪失を防ぎ、編集作業を安心して行えるようにする。<ul>
<li><strong>システム：</strong>FAQ 下書きの編集中の内容を保護し、入力を失わせないようにする</li>
<li><strong>アカウント利用者：</strong>未保存の変更がある状態での離脱前に注意喚起を受けられる</li>
</ul></td>
</tr>
<tr>
<td><span id="BR-126"></span>BR-126</td>
<td>一括操作の上限</td>
<td>UX 細部・データ運用</td>
<td>大量データの一括処理による誤操作や過負荷を防ぐため、1 操作あたりの一括処理件数に上限を設ける。</td>
</tr>
<tr>
<td><span id="BR-127"></span>BR-127</td>
<td>影響度に応じた確認</td>
<td>UX 細部・データ運用</td>
<td>不可逆または影響範囲の大きい操作の誤実行を防ぐため、影響度に応じた確認を経て実行させる。<ul>
<li><strong>アカウント利用者：</strong>重大な操作の前に、影響度に応じた確認を経て実行する</li>
<li><strong>システム：</strong>各操作に影響度に応じた確認レベルを割り当て、誤操作を防ぐ</li>
</ul></td>
</tr>
<tr>
<td><span id="BR-128"></span>BR-128</td>
<td>管理範囲の判別</td>
<td>UX 細部・データ運用</td>
<td>管理コンソールでの設定ミスや影響範囲の取り違えを防ぐため、現在の管理スコープと設定対象の影響範囲を画面から判別できるようにする。</td>
</tr>
<tr>
<td><span id="BR-129"></span>BR-129</td>
<td>表示語彙の分かりやすさ</td>
<td>UX 細部・データ運用</td>
<td>利用者が画面の表示を直感的に理解できるよう、業務語彙を主体とした分かりやすい表示を提供する。<ul>
<li><strong>アカウント利用者：</strong>名称を手掛かりに目的の項目をたどり、システム識別子も必要に応じて利用できる</li>
<li><strong>システム：</strong>内部の英語値を画面へ直接露出させず、日本語の業務語彙へ変換して表示する</li>
</ul></td>
</tr>
<tr>
<td><span id="BR-130"></span>BR-130</td>
<td>初回着地時の案内</td>
<td>UX 細部・データ運用</td>
<td>初めて利用する利用者が最初の一歩で迷わないよう、空状態で次にすべきことを明確に示す。<ul>
<li><strong>アカウント利用者：</strong>プロジェクトが 1 件もない時に、次にすべきことを迷わず把握できる</li>
<li><strong>システム：</strong>簡潔な説明と作成導線に絞った空状態を示し、過剰な案内は行わない</li>
</ul></td>
</tr>
<tr>
<td><span id="BR-131"></span>BR-131</td>
<td>共通項目の影響把握</td>
<td>UX 細部・データ運用</td>
<td>契約スコープの共通項目を変更する際に影響範囲を見誤らないよう、その項目がどれだけのプロジェクトで利用されているかを確認できるようにする。</td>
</tr>
