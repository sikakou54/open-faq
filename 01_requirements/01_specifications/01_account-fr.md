## <span id="FR-005"></span>FR-005: 重要操作の前に本人確認(再認証)を求める

> **この機能要件は「重要操作の前に本人確認(再認証)を求める」を定義します。**

*種別 機能要件 ・ 機能グループ アカウント管理 ・ 優先度 P0 ・ ステータス ドラフト*

### 要件

重要操作の前に本人確認(再認証)を求めること。
1. 対象の重要操作は次のとおり。
    1. パスワード変更
    2. 退会
    3. 課金情報変更
    4. 管理者ユーザー登録・停止・削除
    5. 月次上限件数変更
2. 再認証は本人による再度の認証で行う
3. 再認証の有効範囲は **当該操作 1 回かつ 15 分以内**

### シーケンス

```mermaid
sequenceDiagram
  autonumber
  actor U as ユーザー
  participant UI as UI
  participant S as サーバー
  U->>UI: 重要操作を実行
  UI->>S: 操作要求
  S-->>UI: 再認証を要求
  UI-->>U: 再認証を求める
  U->>UI: 本人確認情報を入力
  UI->>S: 再認証付きで操作要求
  alt 再認証成功
    S-->>UI: 操作を受理
    UI-->>U: 結果を表示
  else 再認証失敗
    S-->>UI: 拒否
    UI-->>U: エラーを表示
  end
```

## <span id="FR-010"></span>FR-010: 利用規約・プライバシーポリシー改定時に再同意を取得

```mermaid
  UI->>S: 再同意状況の確認要求
  S-->>UI: 未同意のため再同意を要求
  UI-->>U: 再同意の割込みを表示
  U->>UI: 再同意して送信
```

## <span id="FR-011"></span>FR-011: 契約が停止された場合の挙動を定める

> **この機能要件は「契約が停止された場合の挙動を定める」を定義します。**

*種別 機能要件 ・ 機能グループ アカウント管理 ・ 優先度 P0 ・ ステータス ドラフト*

### 要件

契約が停止された場合の挙動を定めること。
1. 配下の管理者ユーザーの全セッションを速やかに無効化する
2. 進行中の長時間処理(大量インポート等)は**完了まで継続**し中断しない。完了後の次回操作で再認証を要求する
3. 再ログイン時、契約が停止状態の場合は停止時のアクセス制限ルールに従う

### シーケンス

```mermaid
sequenceDiagram
  autonumber
  actor U as ユーザー
  participant UI as UI
  participant S as サーバー
  Note over S: 契約停止を契機に起動
  S->>S: 配下の全セッションを無効化
  U->>UI: 再ログインを試行
  UI->>S: ログイン要求
  S-->>UI: 停止時のアクセス制限を適用
  UI-->>U: 制限内容を表示
```

## <span id="FR-012"></span>FR-012: オーナーおよび当該プロジェクトのメンバーは、自分が割り当てられたプロジェ

> **この機能要件は「オーナーおよび当該プロジェクトのメンバーは、自分が割り当てられたプロジェ」を定義します。**

*種別 機能要件 ・ 機能グループ ユーザー管理(オーナー + メンバー) ・ 優先度 P0 ・ ステータス ドラフト*

### 要件

オーナーおよび当該プロジェクトのメンバーは、自分が割り当てられたプロジェクトでメンバーを招待できること。
1. 招待はメールアドレスを指定して行う(プロジェクト内の役割差はない)
2. 招待時に氏名(表示名)は入力しない(個人情報保護の原則により、他人の氏名を招待者が任意に登録することを禁止する)。氏名は招待された本人が有効化時に入力する(FR-013d)
3. メンバーの管理は常にプロジェクト単位で行う
4. 同一の人を複数プロジェクトに参加させる場合は各プロジェクトで個別に招待する
5. 同一オーナー配下でメールアドレスが一致する場合は同一の利用者として扱い、新たに参加するプロジェクト分の割当のみを追加する

### シーケンス

```mermaid
sequenceDiagram
  autonumber
  actor U as ユーザー
  participant UI as UI
  participant S as サーバー
  U->>UI: 招待先メールアドレスを入力して送信
  UI->>S: メンバー招待要求
  S-->>UI: 招待結果(招待送信/エラー)
  UI-->>U: 結果を表示
```

## <span id="FR-013"></span>FR-013: オーナーはアカウント新規登録時に自動的に決まり、全プロジェクトに対する全

> **この機能要件は「オーナーはアカウント新規登録時に自動的に決まり、全プロジェクトに対する全」を定義します。**

*種別 機能要件 ・ 機能グループ ユーザー管理(オーナー + メンバー) ・ 優先度 P0 ・ ステータス ドラフト*

### 要件

オーナーはアカウント新規登録時に自動的に決まり、全プロジェクトに対する全権とオーナー専有機能を恒久的に保持すること。オーナーは契約境界に基づく全権を持ち、別途の割当付与なしに自プロジェクトを操作できる(認可権威はオーナー権限)。加えて、作成した各プロジェクトに自身のメンバー行を自動保持する(一覧表示・担当割当・通知宛先の網羅用で、専有判定には用いない)。オーナーは降格・譲渡の対象外とする(MVP)。1 契約につきオーナーは 1 人とする

### シーケンス

```mermaid
sequenceDiagram
  autonumber
  actor U as ユーザー
  participant UI as UI
  participant S as サーバー
  U->>UI: アカウント登録を送信
  UI->>S: 登録要求
  S->>S: 登録者をオーナーとして確定し全権を付与
  S-->>UI: 登録結果
  UI-->>U: オーナーとして利用開始
```

```mermaid
  U->>UI: 最後の有効な割当を解除する操作を実行
  UI->>S: 割当解除要求
  S->>S: 対象を無効化し全セッション・未使用招待を失効
  S-->>UI: 処理結果
  UI-->>U: 結果を表示
```

## <span id="FR-191"></span>FR-191: 権限外の主ナビゲーションと操作の表示を制御

> **この機能要件は「権限外の主ナビゲーションと操作の表示を制御」を定義します。**

*種別 機能要件 ・ 機能グループ アクセス制御細部 ・ 優先度 P0 ・ ステータス ドラフト*

### 要件

権限外の主ナビゲーションと操作の表示を制御すること。
1. 権限外の主ナビゲーションと操作は原則として表示しない
2. 共同作業上、存在を知らせる価値がある項目は読み取り専用で表示し、必要権限と依頼先を説明する
3. 権限のない画面への直アクセスを拒否する際は、同等の説明と戻り先を提示する

### シーケンス

```mermaid
sequenceDiagram
  autonumber
  actor U as ユーザー
  participant UI as UI
  participant S as サーバー
  U->>UI: 画面・操作にアクセス
  UI->>S: 表示可否の判定要求
  alt 権限あり
    S-->>UI: 操作を許可
    UI-->>U: 操作を表示
  else 権限なし
    S-->>UI: 非表示または説明と戻り先
    UI-->>U: 説明と戻り先を表示
  end
```

### 関連

| 優先度 | 対応業務UC |
|----|----|
| P0 | [UC-173](../02_business_usecases/UC-173.md#UC-173) |

P7 後続(第2段)で、関連業務要件(BR)を手掛かりに対応業務UC候補 UC-173 を機械導出した(要レビュー)。
