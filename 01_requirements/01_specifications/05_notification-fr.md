## <span id="FR-113"></span>FR-113: 請求確定、運営からのお知らせなどの契機で、対象となるアカウント利用者へメ

> **この機能要件は「請求確定、運営からのお知らせなどの契機で、対象となるアカウント利用者へメ」を定義します。**

*種別 機能要件 ・ 機能グループ 通知 ・ 優先度 P0 ・ ステータス ドラフト*

### 要件

請求確定、運営からのお知らせなどの契機で、対象となるアカウント利用者へメールまたは受信箱通知できること

### シーケンス

```mermaid
sequenceDiagram
  autonumber
  actor U as ユーザー
  participant UI as UI
  participant S as サーバー
  Note over S: トリガ(例 請求確定/運営お知らせ)
  S->>S: 対象アカウント利用者を特定
  S->>UI: メール送信・受信箱通知
  UI-->>U: 通知を受領
```

| 優先度 | 対応業務UC |

## <span id="FR-121"></span>FR-121: 月次の請求が確定したタイミングで、オーナーにメール通知

> **この機能要件は「月次の請求が確定したタイミングで、オーナーにメール通知」を定義します。**

*種別 機能要件 ・ 機能グループ 通知 ・ 優先度 P0 ・ ステータス ドラフト*

### 要件

月次の請求が確定したタイミングで、オーナーにメール通知できること。
1. 通知には請求対象期間、請求金額、内訳(質問数・各種上限の超過分など)、請求書 / 明細の確認導線を含める
2. 同タイミングで対象オーナーのお知らせ受信箱に「請求確定」のお知らせを生成する
3. 同一契約・同一請求月での重複請求を行わない

### シーケンス

```mermaid
sequenceDiagram
  autonumber
  actor U as ユーザー
  participant UI as UI
  participant S as サーバー
  Note over S: トリガ(月次請求の確定)
  S->>S: 重複請求でないことを確認
  S->>UI: オーナーへメール通知
  S->>UI: 受信箱に請求確定のお知らせを生成
  UI-->>U: 通知を受領
```

### 関連

| 優先度 | 対応業務UC |
|----|----|
| P0 | [UC-233](../02_business_usecases/UC-233.md#UC-233) |

## <span id="FR-122"></span>FR-122: 特定契約または全契約に対して任意のお知らせ(メンテナンス予告、機能追加、

> **この機能要件は「特定契約または全契約に対して任意のお知らせ(メンテナンス予告、機能追加、」を定義します。**

*種別 機能要件 ・ 機能グループ 通知 ・ 優先度 P0 ・ ステータス ドラフト*

### 要件

特定契約または全契約に対して任意のお知らせ(メンテナンス予告、機能追加、規約改定、価格改定など)をメール通知できること。
1. 宛先範囲(全契約 / 特定契約 / 特定ユーザー種別)、件名、本文、送信予定日時を指定できる
2. 受信者はアカウント利用者を対象とし、種別ごとにオプトアウトの可否を設定できる(規約改定・重要セキュリティ通知などは強制送信としオプトアウト不可)
3. 同タイミングで対象契約のアカウント利用者のお知らせ受信箱に「お知らせ」を生成する

### シーケンス

```mermaid
sequenceDiagram
  autonumber
  actor U as ユーザー
  participant UI as UI
  participant S as サーバー
  U->>UI: お知らせの宛先範囲・件名・本文を入力
  UI->>S: お知らせ配信を要求
  S-->>UI: 受付結果
  UI-->>U: 受付を表示
  Note over S: トリガ(送信予定日時)
  S->>UI: 対象アカウント利用者へメール通知・受信箱生成
  UI-->>U: 通知を受領
```

### 関連

| 優先度 | 対応業務UC |
|----|----|
| P0 | [UC-234](../02_business_usecases/UC-234.md#UC-234) |
