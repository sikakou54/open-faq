# 利用量・課金・ダッシュボード・運用(業務要件)

> **このページは、利用量・課金・ダッシュボード・運用 に関する業務要件を定義します。**

*複数の個別要件を統合したカテゴリ別ページ。各要件の優先度・トレースは各節を参照。 ステータス ドラフト*

## <span id="BR-059"></span>BR-059: 課金モデル

> **この業務要件は「課金モデル」を定義します。**

*種別 業務要件 ・ 機能グループ 利用量・課金 ・ 優先度 P0 ・ ステータス ドラフト*

### 要件

スモールスタートを促すため、完全従量課金制 + 月次無料枠を採用し、無料枠は毎月リセットする。課金対象は質問数(月次上限件数の設定対象)と FAQ 件数(上限なしで計測・従量のみ)とし、AI 利用コスト(原価)は MVP ではサービス側が吸収して契約単位の社内可視化指標として扱う。無料枠量・各単価・リセット規則は対応業務ルールを正本とする。

## <span id="BR-062"></span>BR-062: 上限到達時の受付停止

*種別 業務要件 ・ 機能グループ 利用量・課金 ・ 優先度 P0 ・ ステータス ドラフト*

想定外の超過課金から契約者を守るため、質問数上限がオンのプロジェクトが上限に到達したら、支払方法の有無に関わらず当該プロジェクトのウィジェット新規質問受付を停止する。契約自体は有効のまま維持し、エンドユーザーには安全な定型文を示す方針とする。停止は翌月のリセット・上限引き上げ・上限オフのいずれかで即時に復帰させる。停止・追加通知の到達率や最終ガードのしきい値は対応機能要件・業務ルールを正本とする。

## <span id="BR-064"></span>BR-064: 支払方法ゲート

> **この業務要件は「支払方法ゲート」を定義します。**

*種別 業務要件 ・ 機能グループ 利用量・課金 ・ 優先度 P0 ・ ステータス ドラフト*

### 要件

無料枠内での試用開始を妨げず、かつ未払い利用を防ぐため、支払方法はクレジットカード払いとし、新規登録時点での登録は必須としない。無料枠判定はプロジェクト単位、支払方法は契約単位とする方針とする。無料枠超過が近づいた段階で事前告知し、支払方法未登録のまま超過した場合は当該プロジェクトのウィジェット新規質問受付のみを停止して支払方法登録へ誘導する(契約サスペンションは行わず管理画面は継続利用可能)。支払方法登録後は即時再開する。告知の閾値・経路の詳細は対応機能要件・業務ルールを正本とする。

## <span id="BR-127"></span>BR-127: 影響度に応じた確認

*種別 業務要件 ・ 機能グループ UX 細部・データ運用 ・ 優先度 P0 ・ ステータス ドラフト*

### 要件

不可逆または影響範囲の大きい操作の誤実行を防ぐため、影響度に応じた確認を経て実行させる。
- **アカウント利用者：**重大な操作の前に、影響度に応じた確認を経て実行する
- **システム：**各操作に影響度に応じた確認レベルを割り当て、誤操作を防ぐ

対象操作・確認レベルの区分の詳細は対応機能要件および基本設計を正本とする。

## <span id="BR-128"></span>BR-128: 管理範囲の判別

*種別 業務要件 ・ 機能グループ UX 細部・データ運用 ・ 優先度 P0 ・ ステータス ドラフト*

### 要件

管理コンソールでの設定ミスや影響範囲の取り違えを防ぐため、現在の管理スコープと設定対象の影響範囲を画面から判別できるようにする。スコープ分離・画面構成の詳細は対応機能要件および基本設計を正本とする。

## <span id="BR-129"></span>BR-129: 表示語彙の分かりやすさ

> **この業務要件は「表示語彙の分かりやすさ」を定義します。**

*種別 業務要件 ・ 機能グループ UX 細部・データ運用 ・ 優先度 P0 ・ ステータス ドラフト*

### 要件

利用者が画面の表示を直感的に理解できるよう、業務語彙を主体とした分かりやすい表示を提供する。
- **アカウント利用者：**名称を手掛かりに目的の項目をたどり、システム識別子も必要に応じて利用できる
- **システム：**内部の英語値を画面へ直接露出させず、日本語の業務語彙へ変換して表示する

表示・変換の仕様詳細は対応機能要件を正本とする。

## <span id="BR-130"></span>BR-130: 初回着地時の案内

> **この業務要件は「初回着地時の案内」を定義します。**

*種別 業務要件 ・ 機能グループ UX 細部・データ運用 ・ 優先度 P0 ・ ステータス ドラフト*

### 要件

初めて利用する利用者が最初の一歩で迷わないよう、空状態で次にすべきことを明確に示す。
- **アカウント利用者：**プロジェクトが 1 件もない時に、次にすべきことを迷わず把握できる
- **システム：**簡潔な説明と作成導線に絞った空状態を示し、過剰な案内は行わない

空状態の表示仕様の詳細は対応機能要件を正本とする。
