## <span id="FR-139"></span>FR-139: 利用規約・プライバシーポリシー改定時に再同意を取得

> **この機能要件は「利用規約・プライバシーポリシー改定時に再同意を取得」を定義します。**

*種別 機能要件 ・ 機能グループ プライバシー・データ管理 ・ 優先度 P0 ・ ステータス ドラフト*

### 要件

利用規約・プライバシーポリシー改定時に再同意を取得できること。
1. 利用規約とプライバシーポリシーは個別の文書として版管理し、それぞれ同意を取得する
2. 改定された文書のみを再同意の対象とする
3. 閲覧画面も文書ごとに分離する

### シーケンス

```mermaid
sequenceDiagram
  autonumber
  actor U as ユーザー
  participant UI as UI
  participant S as サーバー
  U->>UI: ログイン
  UI->>S: 同意状態を照会
  alt 改定文書あり
    S-->>UI: 再同意対象の文書を返す
    UI-->>U: 再同意を要求
    U->>UI: 文書ごとに同意
    UI->>S: 同意を登録
    S-->>UI: 登録完了
  else 再同意不要
    S-->>UI: 通常画面へ
  end
  UI-->>U: 結果を表示
```

  end

## <span id="FR-152"></span>FR-152: ウィジェット公開キーのローテーション機能を管理者操作で提供

> **この機能要件は「ウィジェット公開キーのローテーション機能を管理者操作で提供」を定義します。**

*種別 機能要件 ・ 機能グループ セキュリティ ・ 優先度 P0 ・ ステータス ドラフト*

### 要件

ウィジェット公開キーのローテーション機能を管理者操作で提供すること。
1. 新キー発行と同時に、旧キーを **24 時間の猶予期間** 付きの失効予告状態とする
2. 猶予中は新旧両キーで認証を許可し、アクセスログを両キーで記録する(どちらの利用率が高いかを管理者が確認できる)
3. 猶予期間の経過後は、旧キーを完全失効とする

### シーケンス

```mermaid
sequenceDiagram
  autonumber
  actor U as ユーザー
  participant UI as UI
  participant S as サーバー
  U->>UI: キーのローテーションを指示
  UI->>S: 新キー発行を要求
  S-->>UI: 新キーを返し旧キーを失効予告
  UI-->>U: 新キーと猶予を表示
  Note over S: トリガ 猶予期間の経過
  S->>S: 旧キーを完全失効とし記録
```

## <span id="FR-154"></span>FR-154: 不正利用(大量送信・Bot 疑い・規約違反)の検知と自動ブロックを行う

> **この機能要件は「不正利用(大量送信・Bot 疑い・規約違反)の検知と自動ブロックを行う」を定義します。**

*種別 機能要件 ・ 機能グループ セキュリティ ・ 優先度 P1 ・ ステータス ドラフト*

### 要件

不正利用(大量送信・Bot 疑い・規約違反)の検知と自動ブロックを行うこと。
1. 自動検知はメインシステム側で行う
2. 判定ルール・しきい値は契約別に上書きできる(設定は他システムの操作を契機とする)
3. 自動ブロックはメインシステム側で実行する
4. 自動ブロック発火時は管理者ユーザーへ即時通知し、申し立て窓口を案内する

### シーケンス

```mermaid
sequenceDiagram
  autonumber
  actor U as ユーザー
  participant UI as UI
  participant S as サーバー
  U->>UI: アクセス
  UI->>S: 要求
  alt 正常利用
    S-->>UI: 正常応答
  else 不正利用を検知
    S->>S: 自動ブロックし記録
    S-->>UI: 拒否を返す
    Note over S: トリガ 管理者へ通知
  end
  UI-->>U: 結果を表示
```
