FR18: UX 細部・データ運用

この要件は、FAQ 編集時のデータ保護(同時編集の衝突検出・自動保存・改訂履歴)と、影響範囲の大きい操作の確認・画面構成の分かりやすさを通じて、アカウント利用者が安全かつ迷わず操作できるようにすることを目的とし、以下について記載します。

  • FAQ の同時編集衝突検出・下書き自動保存・改訂履歴(直近 50 件)と差分・ロールバック
  • 一括操作の上限(お知らせ 100 件 / FAQ 50 件)
  • 不可逆・影響大の操作に応じた確認レベルの割り当て
  • 管理スコープ(契約範囲 / プロジェクト範囲)の分離と、主ナビゲーション・画面見出しの整理
  • プロジェクト 0 件時の空状態と、契約共通項目の影響範囲表示

1. 業務要件

ID 観点 ステークホルダ 業務要件 関連 NFR
BR-152 同時編集の保護 システム 複数のアカウント利用者が同じ FAQ を同時に編集した時に、後勝ちによる上書きを防ぐため衝突を検出し、最新版の確認と再読み込みを促す。 —
BR-153 編集中のデータ保護 システム / アカウント利用者
  • システム：FAQ 下書きの編集中は内容を自動保存し、不慮の離脱でも入力を失わないようにする
  • アカウント利用者：未保存の変更がある状態でページを離れようとした時に、警告を受けてから離脱を判断できる
 —
BR-154 変更の追跡と復元 システム / アカウント利用者
  • システム：FAQ の本文変更と状態変更(下書き / 公開 / 非公開)を直近 50 件まで改訂履歴として保持する
  • アカウント利用者：改訂履歴の差分を確認し、必要な時に任意の版へロールバックできる
 —
BR-155 一括操作の上限 アカウント利用者 大量データをまとめて処理する時に、1 操作あたりお知らせの一括既読は最大 100 件、FAQ の一括状態変更は最大 50 件までとする。 —
BR-156 影響度に応じた確認 アカウント利用者 / システム
  • アカウント利用者：削除・公開キー再発行・退会・上限件数変更・規約再同意拒否などの不可逆または影響範囲の大きい操作を行う時に、影響度に応じた確認を経て実行する
  • システム：各操作に確認レベル(確認ダイアログ / 対象名のタイプ確認 / 本人確認(再認証))を割り当て、誤操作を防ぐ
 —
BR-157 管理範囲の判別 アカウント利用者 管理コンソールを操作する時に、現在の管理スコープ(契約範囲 / プロジェクト範囲)と設定対象の影響範囲を画面から判別できる。 —
BR-158 表示語彙の分かりやすさ アカウント利用者 / システム
  • アカウント利用者：一覧では名称を主リンクとしてたどり、システム識別子は必要に応じてコピーして利用できる
  • システム：内部の英語値(状態コード等)を画面へ直接露出させず、日本語の業務語彙へ変換して表示する
 —
BR-159 初回着地時の案内 アカウント利用者 / システム
  • アカウント利用者：プロジェクトが 1 件もない時に、次にすべきことを迷わず把握できる
  • システム：説明文と作成導線だけの空状態を表示し、段階型のセットアップチェックリストは表示しない
 —
BR-160 共通項目の影響把握 アカウント利用者 契約スコープの共通項目を参照する時に、影響範囲を示す「利用中のプロジェクト: N」を確認できる(N=0 は「未使用」と表示)。 —

2. 機能要件

ID 要件 関連業務要件 優先度
FR-320 FAQ 編集は同時編集時の衝突を検出できること。衝突時は最新版の確認を促し、再読み込みの導線を提供すること。衝突制御方式は基本設計を正本とする BR-152 P1
FR-321 FAQ 下書きは自動保存し、未保存の変更がある状態でページ離脱を試みた際は警告すること BR-153 P2
FR-322 FAQ 改訂履歴は本文変更および状態変更(下書き / 公開 / 非公開)を保持すること。直近 50 件 を保持し、差分表示および任意版へのロールバックを可能とする BR-154 P1
FR-323 一括操作の 1 操作あたりの上限を設けること。
  1. お知らせの一括既読は 最大 100 件
  2. FAQ の一括状態変更は 最大 50 件
 BR-155 P1
FR-324 不可逆または影響範囲の大きい操作に、影響度に応じた確認レベルを必ず割り当てること。
  1. 対象操作には、削除・公開キー再発行・退会・上限件数変更・規約再同意拒否などを含む
  2. 確認レベルは、確認ダイアログ・対象名のタイプ確認・本人確認(再認証)に区分する
  3. 各操作と確認レベルの対応の正本は基本設計の画面設計に置く
 BR-156 P0
FR-325 利用者向け管理コンソールは、管理スコープ(契約範囲 / プロジェクト範囲 / 認証フロー / 共通領域)を画面から判別できるよう分離すること。スコープと各画面の対応の正本は基本設計の画面設計に置く BR-157 P1
FR-326 利用状況でプロジェクトが 0 件の場合は、説明文と「新規プロジェクトを作成」する導線だけを持つ空状態を表示すること。段階型のセットアップチェックリストは表示しない BR-159 P1
FR-327 契約スコープの参照表示には、影響範囲を示す「利用中のプロジェクト: N」を併記すること。N=0 の場合は「未使用」と表示する。月次利用上限・無料利用枠はプロジェクト単位のため対象外とする BR-160 P2
FR-328 主ナビゲーションはユーザーの目的語を優先して構成すること。
  1. 契約範囲では「利用状況 / プロジェクト / 請求 / 設定」の 4 項目だけを用いる
  2. プロジェクト範囲では「概要 / 要対応の質問 / FAQ / ウィジェット / メンバー / 利用量と上限」を用いる
  3. 配置の例外は次のとおりとする
    1. プロジェクトの作成・編集・削除は契約範囲の「プロジェクト」へ置く
    2. 通知はヘッダーへ置く
    3. 法務文書はヘルプまたはフッターへ置く
 BR-157 P0
FR-329 一覧では名称を主リンク、システム識別子をコピー可能な補助情報として表示すること。内部の英語値(状態コード等)はユーザー向け表示へ直接露出させず、日本語の業務語彙へ変換すること BR-158 P0
FR-330 個人設定・契約設定・プロジェクト編集を分離し、設定対象と影響範囲を画面タイトル・説明から判別できること BR-157 P0
FR-346 画面見出しは画面名だけを表示し、契約名・プロジェクト名・サイト名を連結しないこと。現在の管理範囲名は管理範囲の切替表示に集約すること。ただし、削除・公開キー再発行等の対象確認文やデータ名称そのものを扱う編集画面では、誤操作防止に必要な対象名表示を維持すること BR-157 P1
未確定事項v1.0 リリース時点で全項目確定済み。現在オープンな確認事項はありません。
