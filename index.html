  3. 運営者システム ── 同様に 要件定義 → 基本設計 → 詳細設計
  4. 運用設計 / 将来対応 ── 各システムの運用・将来バックログを必要に応じて参照

更新・保守時の注意事項

  • 本ポータルは HTML が正本です。各ページの <article> 内を直接編集します。
  • サイドバー・検索・目次・パンくずは assets/js/portal.js が読み込み時に生成します(各 HTML には焼き込まれません)。
  • ページを追加したら assets/js/nav-data.js に 1 行追記し、python3 99_script/build-search-index.py で検索索引を再生成します。
  • 仕様変更後は bash 99_script/check-spec-sync.sh を実行してください。
