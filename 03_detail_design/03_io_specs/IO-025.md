> **この入出力設計書は「利用量と上限画面と利用量サマリ(プロジェクト)API・プロジェクト上限・アラート取得API の入力/出力項目・入力チェック・編集加工」を論理項目単位で定義します。**

## 1. 目的

本入出力設計は、オーナー / メンバーが当該プロジェクトの当月利用量と質問数の月次上限を一画面で確認する一連の入力/出力項目を、画面と API を突き合わせて論理項目単位で確定する。橋渡しする画面は [SCR-026](../../02_basic_design/01_frontend/01_screens/SCR-026.md#SCR-026) の初期表示(EVT-01)、API は [API-041](../../02_basic_design/02_backend/03_apis/API-041.md#API-041)(利用量サマリ)と [API-046](../../02_basic_design/02_backend/03_apis/API-046.md#API-046)(上限・アラート設定)である。

## 2. 対象範囲

本設計が対象とする画面・API・データと対象外を明示する。質問数上限・アラートの設定入力、集計ロジック自体の詳細は別文書へ委ねる。

| 区分 | 対象 |
|----|----|
| 対象画面 | [SCR-026](../../02_basic_design/01_frontend/01_screens/SCR-026.md#SCR-026) 利用量と上限(初期表示 EVT-01) |
| 対象API | [API-041](../../02_basic_design/02_backend/03_apis/API-041.md#API-041) 利用量サマリ(プロジェクト) ・ [API-046](../../02_basic_design/02_backend/03_apis/API-046.md#API-046) プロジェクト上限・アラート取得 |
| 対象データ | 対象プロジェクト識別子・当月質問数利用状況(使用数 / 上限有無 / 上限件数 / 利用率 / リセット日)・アラート閾値・参考課金額・空状態 |
| 対象外 | 「アラート設定」(EVT-02)押下後に開く質問数上限設定モーダルの入出力([SCR-027](../../02_basic_design/01_frontend/01_screens/SCR-027.md#SCR-027) の入出力)・FAQ 件数の利用状況(本画面は表示しない)・利用量集計ロジック自体 |

## 3. 入力項目一覧

初期表示画面から API へ渡す入力項目を論理項目単位で定義する。本画面は利用者が値を入力する項目を持たず(SCR-026 §5)、初期表示時にサーバーへ送出する識別情報のみを扱う。

| 項目ID | 項目名 | データ型 | 必須 | 初期値 | 入力制約 | 入力例 | 備考 |
|----|----|----|----|----|----|----|----|
| I-01 | セッション認証(ログイン中オーナー / メンバー) | — | ◯ | — | 有効なログインセッションであり、かつ当該プロジェクトへの参加・閲覧権限を持つこと | — | 認証ヘッダで送信。画面には表示しない([API-041](../../02_basic_design/02_backend/03_apis/API-041.md#API-041) `## リクエスト` 認証ヘッダ P-01 / [API-046](../../02_basic_design/02_backend/03_apis/API-046.md#API-046) `## リクエスト` 認証ヘッダ P-01) |
| I-02 | 対象プロジェクト識別子 | 文字列 | ◯ | — | 有効なプロジェクト識別子であり、かつ自身が参加・閲覧権限を持つこと | — | API-041 Query Parameters `projectId` / API-046 Path Parameters `id` |
| I-03 | 対象期間 | 列挙値 | — | `current_month`(当月固定) | 固定値 `current_month` のみ。画面に期間セレクトは設けない([API-041](../../02_basic_design/02_backend/03_apis/API-041.md#API-041) `## リクエスト` Query Parameters `period`) | `current_month` | API-041 `period`。画面からは送出せず既定値のまま呼び出す |
| I-04 | 集計の表示単位 | 列挙値 | — | `project`(プロジェクト単位固定) | 固定値 `project` のみ | `project` | API-041 `viewMode`。画面からは送出せず既定値のまま呼び出す |

## 4. 出力項目一覧

API レスポンス・画面へ返す出力項目を論理項目単位で定義する。API-046(上限・アラート取得)の `limitEnabled` を主分岐とし、上限 ON / OFF / 空状態により表示項目が分岐する。

| 項目ID | 項目名 | データ型 | 表示形式 | 表示条件 | 備考 |
|----|----|----|----|----|----|
| O-01 | 集計対象期間 | 文字列(ISO 8601) | 集計対象期間・最終更新に表示(SCR-026 §4 #2) | 通常時 | API-041 `period` / API-046 `period` |
| O-02 | 当月の質問数(使用数) | 整数 | 質問数サマリー(SCR-026 §4 #5)・消化率(SCR-026 §4 #7)の分子に表示 | 上限 ON 時 | API-046 `questions.used` |
| O-03 | 月次上限有効フラグ | 真偽値 | 表示分岐の判定に用いる(画面非表示) | 常時 | `false` のとき上限 OFF 説明(SCR-026 §4 #9)を表示。API-046 `questions.limitEnabled` |
| O-04 | 今月の利用上限件数 | 整数 | 質問数サマリー(SCR-026 §4 #5)の分母に表示 | 上限 ON 時 | 上限 OFF 時は `null`(画面非表示)。API-046 `questions.limit` |
| O-05 | 消化率 | 数値(%) | 消化率プログレスバー・N / M 件表示(SCR-026 §4 #7) | 上限 ON 時 | 上限 OFF 時は `null`。API-046 `questions.percentage` |
| O-06 | 状態バッジ | 列挙値 | 状態バッジに表示(SCR-026 §4 #6) | 上限 ON 時 | O-05(消化率)から画面側で判定([SCR-026](../../02_basic_design/01_frontend/01_screens/SCR-026.md#SCR-026) データパターン表)。API 出力項目ではない |
| O-07 | 月次リセット日 | 文字列(ISO 8601) | 月次リセット日に表示(SCR-026 §4 #8) | 上限 ON 時 | API-041 `questions.resetAt` |
| O-08 | アラート閾値配列 | 配列 | 画面非表示 | 上限 ON 時 | 設定済みアラート閾値(%)。本画面では表示せず「アラート設定」(EVT-02)遷移先で参照。API-046 `questions.alertThresholds` |
| O-09 | 参考課金額 | 整数(円) | 画面非表示 | 上限 ON 時 | 上限消化時の参考課金額。本画面では表示せず「アラート設定」(EVT-02)遷移先で参照。上限 OFF 時は `null`。API-046 `questions.yenEquivalent` |
| O-10 | 上限設定元 | 列挙値 | 画面非表示 | 上限 ON 時 | `owner`。API-046 `questions.source` |
| O-11 | 上限到達警告バナー表示要否 | 真偽値 | 上限到達警告バナー(SCR-026 §4 #4)の表示要否を判定 | 上限 ON かつ O-05(消化率)が 100% 以上のとき | O-03(`limitEnabled`)と O-05 から画面側で判定 |
| O-12 | 空状態表示要否 | 真偽値 | 空状態(SCR-026 §4 #10)の表示要否を判定 | 集計前 / API-041 または API-046 の取得失敗時 | いずれの API も取得できない場合、または集計未実施の場合に画面側で判定 |

## 5. 入力チェック仕様

各入力項目のチェック内容・実施主体(クライアント / サーバー)・エラー時参照を定義する。本画面は利用者入力を持たないため(SCR-026 §5)、チェックはすべてサーバー側の認証・認可・パラメータ検証である。

| 項目名 | チェック内容 | タイミング | エラー時メッセージ | 備考 |
|----|----|----|----|----|
| セッション認証(I-01) | セッション有効性チェック | サーバー | [ERR-033](../../02_basic_design/05_errors/ERR-033.md#ERR-033) | 全認証保護 API 共通([エラー設計 §3](../../02_basic_design/05_errors/index.md#authz-map))。期限切れ時は再ログインへ誘導し空状態(SCR-026 §4 #10)を表示しない |
| セッション認証(I-01) | プロジェクト参加・閲覧権限チェック(境界判定) | サーバー | [ERR-017](../../02_basic_design/05_errors/ERR-017.md#ERR-017) | 対象プロジェクトへの割当を持たない場合は境界判定で 404 偽装([エラー設計 §3](../../02_basic_design/05_errors/index.md#authz-map)) |
| 対象プロジェクト識別子(I-02) | 対象プロジェクトへのアクセス権チェック | サーバー | [ERR-017](../../02_basic_design/05_errors/ERR-017.md#ERR-017) | API-041 P-01 / API-046 P-01 |
| 対象期間(I-03) | 固定値チェック(`current_month` 以外を許容しない) | サーバー | [ERR-001](../../02_basic_design/05_errors/ERR-001.md#ERR-001) | API-041 は当月固定集計のみを提供する([API-041](../../02_basic_design/02_backend/03_apis/API-041.md#API-041) `## リクエスト` Query Parameters) |
| 集計の表示単位(I-04) | 固定値チェック(`project` 以外を許容しない) | サーバー | [ERR-001](../../02_basic_design/05_errors/ERR-001.md#ERR-001) | 本画面はプロジェクト単位のみを表示する([API-041](../../02_basic_design/02_backend/03_apis/API-041.md#API-041) `## リクエスト` Query Parameters) |

## 6. 編集・加工仕様

入力値の正規化・出力値の整形など編集加工を定義する。上限 ON / OFF・取得失敗の分岐判定は画面側で行う。

| 項目名 | 加工内容 | 加工タイミング | 備考 |
|----|----|----|----|
| 状態バッジ(O-06) | 消化率(O-05)から状態バッジの表示区分を判定 | 応答受信時(クライアント) | 判定区分は [SCR-026](../../02_basic_design/01_frontend/01_screens/SCR-026.md#SCR-026) データパターン表(80% 未満 / 80% 以上 100% 未満 / 100% 以上)を正本とする |
| 上限到達警告バナー表示要否(O-11) | O-03(`limitEnabled`)と O-05(消化率)から表示要否を判定 | 応答受信時(クライアント) | 上限 OFF 時は判定対象外(非表示) |
| 空状態表示要否(O-12) | API-041 / API-046 いずれかの取得失敗、または集計未実施を統合判定 | 応答受信時(クライアント) | 専用フラグ項目はなく、画面側が API 応答の成否から確定する |
| 参考課金額(O-09) | 「(上限件数)-(無料枠)=(課金対象件数)(¥(参考額) / 月)」形式へ整形 | 「アラート設定」遷移先表示時(クライアント) | 算出式・無料枠/超過単価の正本は [API-046](../../02_basic_design/02_backend/03_apis/API-046.md#API-046)・[システム仕様書 §2](../../02_basic_design/07_system-spec.md#2-課金利用量上限)。本画面では整形せず非表示のまま次画面へ引き継ぐ |

## 7. 後続工程への引き継ぎ事項

テスト設計・DB 物理設計・詳細ロジック設計へ引き継ぐ観点を挙げる。質問数上限・アラート設定自体の入出力は別文書へ委ねる。

- 上限 ON(消化率 80% 未満 / 80% 以上 100% 未満 / 100% 以上)・上限 OFF・空状態(集計前 / 取得失敗)の表示分岐(SCR-026 §7 EVT-01)をテスト設計でケース化する。
- API-041 と API-046 の初期表示時の呼び出し順・並行呼び出し可否、片方のみ失敗した場合の表示方式(空状態とするか部分表示とするか)は詳細ロジック設計へ引き継ぐ。
- 消化率境界値(0% / 79% / 80% / 99% / 100% / 100% 超)の表示分岐はテスト設計でパターン化する。
- 論理項目 ↔ 物理カラムの対応は [DBP-010](../07_db_physical/DBP-010.md#DBP-010) で確定する。
