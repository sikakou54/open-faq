# <span id="IO-005"></span>IO-005: メール確認 入出力

> **この入出力設計書は「メール確認画面と、メール確認 API・確認メール再送 API の入力/出力項目・入力チェック・編集加工」を論理項目単位で定義します。**

*種別 入出力設計 ・ ステータス ドラフト*

| 項目 | 値 |
|----|----|
| 入出力設計ID | IO-005 |
| 業務ユースケースID | [UC-002](../../01_requirements/04_business_usecases/UC-002.md#UC-002) ・ [UC-003](../../01_requirements/04_business_usecases/UC-003.md#UC-003) |
| 関連画面 | [SCR-018](../../02_basic_design/01_frontend/01_screens/SCR-018.md#SCR-018) |
| 関連 API | [API-006](../../02_basic_design/02_backend/03_apis/API-006.md#API-006) ・ [API-001](../../02_basic_design/02_backend/03_apis/API-001.md#API-001) |

## 1. 目的

本入出力設計は、未認証ユーザーが確認メールのリンクからメールアドレスの所有を証明し、送信済み・確認成功・確認失敗の各状態を画面に反映する一連の入力/出力項目を、画面と API を突き合わせて論理項目単位で確定する。橋渡しする画面は [SCR-018](../../02_basic_design/01_frontend/01_screens/SCR-018.md#SCR-018) の初期表示(EVT-01)・再送(EVT-02)、API は本人確認を行う [API-006](../../02_basic_design/02_backend/03_apis/API-006.md#API-006) と確認メールを再送する [API-001](../../02_basic_design/02_backend/03_apis/API-001.md#API-001) である。

## 2. 対象範囲

対象とする画面・API・データと、対象外を明示する。

| 区分 | 対象 |
|----|----|
| 対象画面 | [SCR-018](../../02_basic_design/01_frontend/01_screens/SCR-018.md#SCR-018) メール確認(初期表示 EVT-01・再送 EVT-02) |
| 対象API | [API-006](../../02_basic_design/02_backend/03_apis/API-006.md#API-006) メール確認 ・ [API-001](../../02_basic_design/02_backend/03_apis/API-001.md#API-001) 確認メール再送 |
| 対象データ | 確認メール送信先アドレス・確認トークン・確認結果(成功 / 期限切れ / 使用済み)・再送受付結果 |
| 対象外 | アカウント新規登録本体の入力([IO-003](IO-003.md#IO-003) 相当の別範囲・本ページは扱わない)・メールアドレス変更後の遷移先([SCR-002](../../02_basic_design/01_frontend/01_screens/SCR-002.md#SCR-002) の入出力) |

## 3. 入力項目一覧

画面から受け取り API へ渡す入力項目を論理項目単位で定義する。文字数上限などの具体値は基本設計へ ID 参照する。

| 項目ID | 項目名 | データ型 | 必須 | 初期値 | 入力制約 | 入力例 | 備考 |
|----|----|----|----|----|----|----|----|
| I-01 | 確認トークン | 文字列 | ◯ | — | URL パス末尾で受け取る有効な確認トークン | — | 確認メール内リンクの遷移時に URL から取得。画面には表示しない([API-006](../../02_basic_design/02_backend/03_apis/API-006.md#API-006) `## リクエスト` Path Parameters `token`) |
| I-02 | 再送対象メールアドレス | 文字列 | ◯ | 送信済み画面表示中のアドレス | RFC 5322 準拠・最大 254 文字 | user@example.com | SCR-018 §4 #2 の表示アドレスを再送要求時に送出。API リクエスト `email`([API-001](../../02_basic_design/02_backend/03_apis/API-001.md#API-001) Request Body: 確認メール再送) |

## 4. 出力項目一覧

API レスポンス・画面へ返す出力項目を論理項目単位で定義する。表示状態(送信済み / 確認成功 / 確認失敗)により出力項目が分岐する。

| 項目ID | 項目名 | データ型 | 表示形式 | 表示条件 | 備考 |
|----|----|----|----|----|----|
| O-01 | 送信先メールアドレス | 文字列 | label(SCR-018 §4 #2) | 送信済み状態 | 確認メールの送信先。登録・再送時に確定した値を表示 |
| O-02 | 状態タイムライン | 列挙値 | label(SCR-018 §4 #1) | 送信済み状態 | 登録手続きの進捗表示。取りうる値と意味は §列挙値 |
| O-03 | 確認対象アカウント識別子 | 文字列 | 画面非表示 | 確認成功状態 | 確認完了したアカウントの識別子。API `userId`([API-006](../../02_basic_design/02_backend/03_apis/API-006.md#API-006)) |
| O-04 | メール確認日時 | 文字列(ISO 8601) | 画面非表示 | 確認成功状態 | メールアドレスを確認済みとした日時。API `verifiedAt`([API-006](../../02_basic_design/02_backend/03_apis/API-006.md#API-006)) |
| O-05 | 確認成功案内 | 文字列 | alert(SCR-018 §4 #6) | 確認成功状態 | ログインへの案内文を表示 |
| O-06 | 確認失敗理由 | 列挙値 | alert(SCR-018 §4 #8・EM-01) | 確認失敗状態 | 期限切れ / 使用済み / トークン不正。取りうる値と意味は §列挙値 |
| O-07 | 再送受付結果 | 真偽値 | — | 再送要求後 | 受理時 `true`。API `ok`([API-001](../../02_basic_design/02_backend/03_apis/API-001.md#API-001)) |
| O-08 | 次回再送可能時刻 | 文字列(ISO 8601) | 「メールを再送する」(#4)の非活性解除条件 | 再送成功後、レート制限区間中 | サーバーから明示返却はせず、画面側は再送成功時刻に基づき算出([SCR-018](../../02_basic_design/01_frontend/01_screens/SCR-018.md#SCR-018) EVT-02) |

## 5. 入力チェック仕様

各入力項目のチェック内容・実施主体(クライアント / サーバー)・エラー時参照を定義する。文面は本ページに再掲せず ID 参照する。

| 項目名 | チェック内容 | タイミング | エラー時メッセージ | 備考 |
|----|----|----|----|----|
| 確認トークン(I-01) | 形式チェック | サーバー | [ERR-001](../../02_basic_design/05_errors/ERR-001.md#ERR-001) | トークン形式不正時 |
| 確認トークン(I-01) | 存在チェック | サーバー | [ERR-008](../../02_basic_design/05_errors/ERR-008.md#ERR-008) | トークンが存在しない場合 |
| 確認トークン(I-01) | 有効期限チェック([システム仕様書 §4](../../02_basic_design/07_system-spec.md#4-データ保持期間削除猶予) アカウントメール確認リンク有効期限) | サーバー | [ERR-006](../../02_basic_design/05_errors/ERR-006.md#ERR-006) / SCR-018 EM-01 | 期限切れ時は確認失敗状態(#8)へ |
| 確認トークン(I-01) | 使用済みチェック | サーバー | [ERR-007](../../02_basic_design/05_errors/ERR-007.md#ERR-007) / SCR-018 EM-01 | 使用済み時は確認失敗状態(#8)へ |
| 再送対象メールアドレス(I-02) | 形式チェック | サーバー | [ERR-001](../../02_basic_design/05_errors/ERR-001.md#ERR-001) | RFC 5322 準拠・最大 254 文字 |
| 再送要求(送信可否) | 再送間隔チェック([システム仕様書 §5](../../02_basic_design/07_system-spec.md#5-レート制限キャパシティ目安) 認証系レート初期既定) | サーバー | [ERR-009](../../02_basic_design/05_errors/ERR-009.md#ERR-009) / SCR-018 EM-03 | 「メールを再送する」(#4)を押下時に判定。違反時は再送を行わない |

## 6. 編集・加工仕様

入力値の正規化・出力値の整形など編集加工を定義する。

| 項目名 | 加工内容 | 加工タイミング | 備考 |
|----|----|----|----|
| 再送対象メールアドレス(I-02) | 前後空白の除去・小文字化 | 送信前(サーバー) | [IO-003](IO-003.md#IO-003) の登録時加工と同一方針 |

## 列挙値

出力の状態タイムライン(O-02)と確認失敗理由(O-06)の取りうる値と意味を示す。

| 項目 | 取りうる値 | 意味 |
|----|----|----|
| 状態タイムライン(O-02) | 送信済み | 確認メールを送信済みで、未確認の状態([状態モデル](../../02_basic_design/08_state-model.md)) |
| 確認失敗理由(O-06) | 期限切れ | 確認トークンの有効期限([システム仕様書 §4](../../02_basic_design/07_system-spec.md#4-データ保持期間削除猶予))を過ぎている |
| 確認失敗理由(O-06) | 使用済み | 確認トークンが既に一度使用されている |
| 確認失敗理由(O-06) | トークン不正 | 確認トークンの形式不正または存在しない([ERR-001](../../02_basic_design/05_errors/ERR-001.md#ERR-001) / [ERR-008](../../02_basic_design/05_errors/ERR-008.md#ERR-008)) |

> [!NOTE]
> **確認失敗理由(O-06)は画面上では期限切れ・使用済みを区別せず確認失敗状態(#8・EM-01)として一括案内する。** トークン不正(形式不正・存在しない)はメール内リンクの改変等の異常系であり通常操作では発生しない。状態タイムライン(O-02)の確認成功 / 確認失敗の表示区分は [SCR-018](../../02_basic_design/01_frontend/01_screens/SCR-018.md#SCR-018) §4 #6〜#9 の表示条件による。

## 7. 後続工程への引き継ぎ事項

テスト設計・DB 物理設計・詳細ロジック設計へ引き継ぐ観点を挙げる。

- 確認トークン(I-01)の期限切れ・使用済み・不正の各異常系はテスト設計でケース化する。境界値(有効期限直前 / 直後)を含める。
- 再送要求の間隔判定(§5)は境界値(間隔到達直前 / 直後)をテスト設計でパターン化する。
- 次回再送可能時刻(O-08)の画面側算出ロジックの実装詳細は詳細ロジック設計へ引き継ぐ。
- 論理項目 ↔ 物理カラムの対応は [DBP-001](../07_db_physical/DBP-001.md#DBP-001) で確定する。
