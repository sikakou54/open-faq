> **この入出力設計書は「メンバー一覧画面とメンバー一覧 API・プロジェクト割当解除 API の入力/出力項目・入力チェック・編集加工」を論理項目単位で定義します。**

## 1. 目的

本入出力設計は、オーナー / メンバーが当該プロジェクトのメンバー一覧を検索・絞り込みで確認し、行アクションメニューから割当解除を起動して結果を一覧へ反映する一連の入力/出力項目を、画面と API を突き合わせて論理項目単位で確定する。橋渡しする画面は [SCR-013](../../02_basic_design/01_frontend/01_screens/SCR-013.md#SCR-013) の初期表示・検索・招待状態フィルタ・行アクションメニュー(EVT-01〜EVT-03・EVT-07・EVT-10)、API は一覧取得の [API-020](../../02_basic_design/02_backend/03_apis/API-020.md#API-020)、割当解除の [API-023](../../02_basic_design/02_backend/03_apis/API-023.md#API-023) である。

## 2. 対象範囲

本設計が対象とする画面・API・データと対象外を明示する。招待・編集・招待メール再送、および割当解除確認モーダル([SCR-014](../../02_basic_design/01_frontend/01_screens/SCR-014.md#SCR-014))自体の入出力は別文書へ委ねる。

| 区分 | 対象 |
|----|----|
| 対象画面 | [SCR-013](../../02_basic_design/01_frontend/01_screens/SCR-013.md#SCR-013) メンバー(初期表示 EVT-01 / 検索 EVT-02 / 招待状態フィルタ EVT-03 / 行アクションメニュー EVT-07 / 「プロジェクトから外す」起動 EVT-10) |
| 対象API | [API-020](../../02_basic_design/02_backend/03_apis/API-020.md#API-020) メンバー一覧 ・ [API-023](../../02_basic_design/02_backend/03_apis/API-023.md#API-023) プロジェクト割当解除 |
| 対象データ | 検索キーワード・招待状態フィルタ・ページング条件、メンバー一覧の表示項目(表示名・メールアドレス・区分・ステータス・参加日)、割当解除の再認証・対象利用者・解除結果(アカウント無効化有無) |
| 対象外 | メンバー招待・編集・招待メール再送の入出力([SCR-014](../../02_basic_design/01_frontend/01_screens/SCR-014.md#SCR-014) EVT-01〜EVT-05・EVT-09)・割当解除確認ダイアログの表示項目自体(SCR-014 §4 #13〜#15)・再認証の入出力([API-005](../../02_basic_design/02_backend/03_apis/API-005.md#API-005)) |

## 4. 出力項目一覧

API レスポンス・画面へ返す出力項目を論理項目単位で定義する。立場(isOwner)・招待状態(status)により行内の操作可否が分岐する。

| 項目ID | 項目名 | データ型 | 表示形式 | 表示条件 | 備考 |
|----|----|----|----|----|----|
| O-01 | 利用者識別子 | 文字列 | 画面非表示 | メンバーが 1 件以上のとき | 行アクションメニューの操作対象解決に用いる。API `items[].id` |
| O-02 | メールアドレス | 文字列 | メールアドレス(SCR-013 §4 #6) | メンバーが 1 件以上のとき | API `items[].email` |
| O-03 | 利用者表示名 | 文字列 | 利用者表示名(SCR-013 §4 #5) | メンバーが 1 件以上のとき | 招待中(未有効化)は `null`。オーナー行・自分の行はリンクでなくテキスト表示(§6)。API `items[].name` |
| O-04 | アカウント状態 | 列挙値 | ステータス(SCR-013 §4 #8) | メンバーが 1 件以上のとき | `active`(有効)/ `pending_activation`(招待中)。取りうる値と意味は [状態モデル §3](../../02_basic_design/08_state-model.md#3-メンバー割当状態)。API `items[].status` |
| O-05 | このプロジェクトでの区分 | 列挙値 | このプロジェクトでの区分(SCR-013 §4 #7) | メンバーが 1 件以上のとき | `オーナー` / `メンバー`。O-06(isOwner)から導出(§6)。SCR-013 §4 データパターン #7 |
| O-06 | オーナー判定 | 真偽値 | 画面非表示 | メンバーが 1 件以上のとき | 当該プロジェクトのオーナー(作成者)と一致するかどうか。区分(O-05)・行内操作可否(§6)の導出元。API `items[].isOwner` |
| O-07 | 参加日 | 文字列(日時) | 参加日(SCR-013 §4 #9) | メンバーが 1 件以上のとき | 招待中(未有効化)は `null`。API `items[].joinedAt` |
| O-08 | 招待トークン有効期限 | 文字列(日時) | 画面非表示 | アカウント状態が `pending_activation` のとき | API `items[].invitationExpiresAt` |
| O-09 | 次ページ取得用カーソル | 文字列 | 画面非表示 | 常時 | 末尾ページは `null`。追加読込操作の要否判定に用いる。API `nextCursor` |
| O-10 | 割当解除対象利用者識別子 | 文字列 | 画面非表示 | 割当解除成功時 | API-023 レスポンス `userId` |
| O-11 | 割当解除対象プロジェクト識別子 | 文字列 | 画面非表示 | 割当解除成功時 | API-023 レスポンス `projectId` |
| O-12 | アカウント無効化有無 | 真偽値 | 画面非表示 | 割当解除成功時 | 他プロジェクトの有効割当が 0 件となり対象アカウントを論理削除したかどうか。API-023 レスポンス `accountDeactivated` |

## 5. 入力チェック仕様

各入力項目のチェック内容・実施主体(クライアント / サーバー)・エラー時参照を定義する。文面は本ページに再掲せず ID 参照する。

| 項目名 | チェック内容 | タイミング | エラー時メッセージ | 備考 |
|----|----|----|----|----|
| 検索キーワード(I-01) | 最大長チェック | クライアント | — | SCR-013 §4 #2 の入力欄制約(最大 100 文字)を超える入力は入力自体を受け付けない |
| 招待状態フィルタ(I-02) | 許容値(`すべて` / `有効` / `招待中`)チェック | クライアント | — | SCR-013 §4 #3 の選択操作は許容値のみを送出するため画面側での不正値入力は発生しない |
| 対象プロジェクト識別子(I-05) | 境界判定(所有境界 / 割当境界) | サーバー | [ERR-017](../../02_basic_design/05_errors/ERR-017.md#ERR-017) | 部外者・割当なしは 404 偽装([PERM-005](../../02_basic_design/04_permissions/PERM-005.md#PERM-005)) |
| 利用者セッション(I-06) | セッション有効性チェック | サーバー | [ERR-019](../../02_basic_design/05_errors/ERR-019.md#ERR-019) | 当該プロジェクトへの権限なしの場合はサーバーのみ判定(API-020 P-01 / API-023 P-01) |
| 割当解除対象利用者識別子(I-07) | 対象存在チェック | サーバー | [ERR-017](../../02_basic_design/05_errors/ERR-017.md#ERR-017) | 対象が当該プロジェクトの割当に存在しない場合(API-023 P-03) |
| 割当解除対象利用者識別子(I-07) | オーナー除外チェック | クライアント・サーバー | [ERR-021](../../02_basic_design/05_errors/ERR-021.md#ERR-021) | オーナー行では行アクションメニューの「プロジェクトから外す」自体を非表示にしクライアント側で操作を防止(SCR-013 EVT-07)。サーバーは API-023 P-02 で再判定 |
| 割当解除対象利用者識別子(I-07) | 自分自身除外チェック | クライアント・サーバー | [ERR-022](../../02_basic_design/05_errors/ERR-022.md#ERR-022) | ログイン中の自分の行では行アクションメニューの「プロジェクトから外す」自体を非表示にしクライアント側で操作を防止(SCR-013 EVT-07)。サーバーは API-023 P-02 で再判定 |
| 再認証トークン(I-08) | 再認証有効性チェック | サーバー | [ERR-013](../../02_basic_design/05_errors/ERR-013.md#ERR-013) | 未提示または無効時はサーバーのみ判定(API-023 P-00 / P-01) |

## 6. 編集・加工仕様

入力値の正規化・出力値の整形など編集加工を定義する。立場(isOwner)・招待状態(status)に応じて画面側の表示・操作導線を切り替える。

| 項目名 | 加工内容 | 加工タイミング | 備考 |
|----|----|----|----|
| 招待状態フィルタ(I-02) | 未指定時は `すべて` を補う | 送信前(クライアント) | SCR-013 既定表示は「すべて」(SCR-013 §4 データパターン #3) |
| オーナー判定(O-06) | `true` のとき区分(O-05)へ「オーナー」を、`false` のとき「メンバー」を割り当てる | 応答描画時(クライアント) | SCR-013 §4 データパターン #7 |
| 利用者表示名(O-03) | `null`(招待中)のときリンク化せずテキスト表示。オーナー行・ログイン中の自分の行も同様にテキスト表示 | 応答描画時(クライアント) | SCR-013 §4 #5 表示条件 / EVT-06 |
| 参加日(O-07) | `null`(招待中)のとき「—」を表示 | 応答描画時(クライアント) | SCR-013 §4 データパターン #9 |
| アカウント状態(O-04) | `pending_activation` の行に招待中行強調(SCR-013 §4 #11)を付す | 応答描画時(クライアント) | SCR-013 EVT-01 |
| 行アクションメニュー項目 | 「招待メールを再送」は O-04 が `pending_activation` の行のみ表示。「プロジェクトから外す」は O-06 が `true`(オーナー行)またはログイン中の自分の行では非表示 | 応答描画時(クライアント) | SCR-013 EVT-07 |
| メンバー一覧(O-01〜O-09) | 0 件時は空状態(SCR-013 §4 #12)へ切替、1 件以上時は件数表示(#4)・一覧(#5〜#10)へ切替 | 応答描画時(クライアント) | SCR-013 EVT-01 |

## 7. 後続工程への引き継ぎ事項

テスト設計・DB 物理設計・詳細ロジック設計へ引き継ぐ観点を挙げる。招待・編集・招待メール再送の詳細、割当解除確認ダイアログ自体の入出力は別文書へ委ねる。

- 検索キーワード(I-01)と招待状態フィルタ(I-02)の組み合わせ、0 件(空状態)・1 件以上(一覧表示)の各分岐はテスト設計でケース化する。
- 割当解除(API-023)のオーナー除外(ERR-021)・自分自身除外(ERR-022)・再認証未済(ERR-013)・対象不在(ERR-017)・権限なし(ERR-019)の各異常系は、クライアント側で操作導線自体を非表示にする場合とサーバー側でのみ判定する場合を区別してテスト設計でケース化する。
- アカウント無効化有無(O-12)が `true` となる条件(他プロジェクトの有効割当 0 件)の判定ロジックは [API-023](../../02_basic_design/02_backend/03_apis/API-023.md#API-023) P-04 を参照し、詳細ロジックは IPO 側で確定する。
- 論理項目 ↔ 物理カラムの対応は [DBP-005](../07_db_physical/DBP-005.md#DBP-005) で確定する。
