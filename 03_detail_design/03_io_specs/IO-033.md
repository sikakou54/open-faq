# <span id="IO-033"></span>IO-033: ウィジェット起動・埋め込み 入出力

> **この入出力設計書は「FAQ ウィジェットの起動画面とウィジェット起動 API の入力/出力項目・入力チェック・編集加工」を論理項目単位で定義します。**

*種別 入出力設計 ・ ステータス ドラフト*

| 項目 | 値 |
|----|----|
| 入出力設計ID | IO-033 |
| 業務ユースケースID | [UC-040](../../01_requirements/04_business_usecases/UC-040.md#UC-040) ・ [UC-042](../../01_requirements/04_business_usecases/UC-042.md#UC-042) ・ [UC-057](../../01_requirements/04_business_usecases/UC-057.md#UC-057) |

## 1. 目的

本入出力設計は、ウィジェット利用者が設置サイトでチャットを開いた際に、公開キーとオリジンを検証してウィジェットセッションを確立し、表示設定を反映して質問受付可能な状態にする一連の入力/出力項目を、画面と API を突き合わせて論理項目単位で確定する。橋渡しする画面は [SCR-030](../../02_basic_design/01_frontend/01_screens/SCR-030.md#SCR-030) の起動(EVT-02)、API は [API-037](../../02_basic_design/02_backend/03_apis/API-037.md#API-037) である。許可ドメイン照合の判定処理は [SYS-005](../../02_basic_design/02_backend/01_system/SYS-005.md#SYS-005) を正本とする。

## 2. 対象範囲

本設計が対象とする画面・API・データと対象外を明示する。質問送信・未解決登録・回答フィードバックは別 IO で扱う。

| 区分 | 対象 |
|----|----|
| 対象画面 | [SCR-030](../../02_basic_design/01_frontend/01_screens/SCR-030.md#SCR-030) FAQ ウィジェット(起動 EVT-02) |
| 対象API | [API-037](../../02_basic_design/02_backend/03_apis/API-037.md#API-037) ウィジェット起動 |
| 対象データ | 公開キー・オリジン、ウィジェットセッショントークン・表示設定(主色・表示位置・見出し・連絡先メール)の各出力項目 |
| 対象外 | 質問送信([IO-001](IO-001.md#IO-001))・未解決質問登録(API-039)・回答フィードバック送信(API-069)・許可ドメイン照合の判定ロジック([SYS-005](../../02_basic_design/02_backend/01_system/SYS-005.md#SYS-005))・公開キーローテーションの発行操作(SCR-011 側) |

## 3. 入力項目一覧

ウィジェット起動時に埋め込みスクリプトが保持する値から API へ渡す入力項目を論理項目単位で定義する。いずれも画面には表示しない(利用者の入力操作を伴わない)。

| 項目ID | 項目名 | データ型 | 必須 | 初期値 | 入力制約 | 入力例 | 備考 |
|----|----|----|----|----|----|----|----|
| I-01 | 公開キー | 文字列 | ◯ | — | 埋め込みコードに設定された公開キー([API-037](../../02_basic_design/02_backend/03_apis/API-037.md#API-037) `## リクエスト`) | `pk_live_abc123...` | 画面には表示しない。SCR-011 で発行された値を埋め込みコードが保持。API リクエスト `publicKey` |
| I-02 | オリジン | 文字列 | ◯ | — | ウィジェット読み込み元のオリジン([API-037](../../02_basic_design/02_backend/03_apis/API-037.md#API-037) `## リクエスト`) | `https://example.com` | 画面には表示しない。ブラウザが保持する現在ページのオリジンを送出。API リクエスト `origin` |

## 4. 出力項目一覧

API レスポンス・画面へ返す出力項目を論理項目単位で定義する。成功時はセッションと表示設定を、失敗時は SCR-030 EVT-07 のエラーメッセージ表示に用いる結果を返す。

| 項目ID | 項目名 | データ型 | 表示形式 | 表示条件 | 備考 |
|----|----|----|----|----|----|
| O-01 | ウィジェットセッショントークン | 文字列 | 画面非表示 | 起動成功時 | 以降のウィジェット質問送信([IO-001](IO-001.md#IO-001))・未解決質問登録・回答フィードバック送信の認証ヘッダに使用。API `sessionToken` |
| O-02 | セッション有効期限 | 数値 | 画面非表示 | 起動成功時 | セッショントークンの有効期限(秒)。API `expiresIn` |
| O-03 | 主色 | 文字列 | ウィジェットのヘッダー・アクセントカラーに反映(SCR-030 §4 #3) | 起動成功時 | プロジェクト設定の主色([TBL-004](../../02_basic_design/02_backend/04_database/TBL-004.md#TBL-004) `settings.headerColor`)。未設定時は既定値を適用。API `projectConfig.headerColor` |
| O-04 | 表示位置 | 列挙値 | ウィジェットの画面上の配置に反映 | 起動成功時 | プロジェクト設定の表示位置([TBL-004](../../02_basic_design/02_backend/04_database/TBL-004.md#TBL-004) `settings.placement`)。未設定時は既定値を適用。API `projectConfig.placement` |
| O-05 | ヘッダー見出し | 文字列 | ヘッダー(SCR-030 §4 #3)に表示 | 起動成功時 | プロジェクト設定の見出し([TBL-004](../../02_basic_design/02_backend/04_database/TBL-004.md#TBL-004) `settings.headerTitle`)。未設定時はウィジェット既定文言。API `projectConfig.headerTitle` |
| O-06 | 連絡先メール | 文字列 | 連絡先メール案内に使用(SCR-030 §4 #11。起動時点では非表示) | 起動成功時 | プロジェクトの問い合わせ先メール([TBL-004](../../02_basic_design/02_backend/04_database/TBL-004.md#TBL-004) `contact_email`)。未設定時は `null`。API `projectConfig.contactEmail` |
| O-07 | 起動エラー | 真偽値 | エラーメッセージ表示(SCR-030 §4 #13 / EM-03) | 起動失敗時 | 公開キー不正・許可ドメイン外・サスペンション中のいずれかで発生。画面はエラー種別を判別せず一律 EM-03 を表示する(SCR-030 EVT-02 失敗系) |

## 5. 入力チェック仕様

各入力項目のチェック内容・実施主体(クライアント / サーバー)・エラー時参照を定義する。いずれもサーバーのみで判定し、違反時は起動失敗としてウィジェットを動作させない。

| 項目名 | チェック内容 | タイミング | エラー時メッセージ | 備考 |
|----|----|----|----|----|
| 公開キー(I-01) | 公開キーの有効性検証(現行キー照合。不一致時は旧キーを猶予期限内で照合) | サーバー | [ERR-026](../../02_basic_design/05_errors/ERR-026.md#ERR-026) / SCR-030 EM-03 | 現行・旧キーいずれにも一致しない、または旧キーの猶予期限超過は失効として拒否([API-037](../../02_basic_design/02_backend/03_apis/API-037.md#API-037) P-01・P-02) |
| オリジン(I-02) | 許可ドメイン照合 | サーバー | [ERR-027](../../02_basic_design/05_errors/ERR-027.md#ERR-027) / SCR-030 EM-03 | 対象プロジェクトの許可ドメインに一致しない場合、または許可ドメイン未設定の場合は拒否([SYS-005](../../02_basic_design/02_backend/01_system/SYS-005.md#SYS-005) PR-02・PR-03・PR-05) |
| 対象プロジェクトの課金状態(起動可否) | サスペンション状態の確認 | サーバー | [ERR-004](../../02_basic_design/05_errors/ERR-004.md#ERR-004) / SCR-030 EM-03 | 対象プロジェクトの課金アカウントがサスペンション中([状態モデル §2](../../02_basic_design/08_state-model.md#2-課金アカウント状態) `suspended`)の場合は拒否([API-037](../../02_basic_design/02_backend/03_apis/API-037.md#API-037) P-04) |

## 6. 編集・加工仕様

入力値の正規化・出力値の整形など編集加工を定義する。

| 項目名 | 加工内容 | 加工タイミング | 備考 |
|----|----|----|----|
| 連絡先メール(O-06) | 未設定時は `null` を返す | 応答生成時(サーバー) | 起動時点では画面に表示せず、未解決・制限中の案内表示時にのみ用いる(SCR-030 §4 #11) |
| ヘッダー見出し(O-05) | 未設定時はウィジェット既定文言を適用 | 応答生成時(サーバー) | プロジェクト設定値が空文字の場合の既定値適用([TBL-004](../../02_basic_design/02_backend/04_database/TBL-004.md#TBL-004) `settings.headerTitle`) |
| 主色(O-03) | 未設定時は既定値を適用 | 応答生成時(サーバー) | プロジェクト設定値が未設定の場合の既定値適用([TBL-004](../../02_basic_design/02_backend/04_database/TBL-004.md#TBL-004) `settings.headerColor`) |
| 起動エラー(O-07) | エラー種別を判別せず一律の接続不可メッセージへ集約 | 応答描画時(クライアント) | 公開キー不正・許可ドメイン外・サスペンション中を画面上は区別しない([SCR-030](../../02_basic_design/01_frontend/01_screens/SCR-030.md#SCR-030) EVT-02) |

## 7. 後続工程への引き継ぎ事項

テスト設計・DB 物理設計・詳細ロジック設計へ引き継ぐ観点を挙げる。許可ドメイン照合の判定ロジック・公開キーローテーションの詳細は別文書へ委ねる。

- 公開キー(I-01)の現行キー一致 / 旧キー猶予期限内 / 旧キー猶予期限超過 / いずれにも不一致の 4 パターン、およびオリジン(I-02)の許可ドメイン一致 / 不一致 / 未設定の 3 パターンはテスト設計でケース化する。
- 起動失敗時(O-07)の 3 要因(公開キー不正・許可ドメイン外・サスペンション中)は画面上は同一表示だが、ログ・監視上は要因を区別できることをテスト設計で確認する。
- 許可ドメイン照合の判定ロジックは [SYS-005](../../02_basic_design/02_backend/01_system/SYS-005.md#SYS-005)、公開キーローテーション・猶予期限の物理設計は [DBP-006](../07_db_physical/DBP-006.md#DBP-006) を参照する。
- 論理項目 ↔ 物理カラムの対応は [DBP-006](../07_db_physical/DBP-006.md#DBP-006) で確定する。
