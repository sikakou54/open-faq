# <span id="IO-018"></span>IO-018: 要対応の質問詳細 入出力

> **この入出力設計書は「要対応の質問詳細画面と未解決質問詳細・状況切替 API の入力/出力項目・入力チェック・編集加工」を論理項目単位で定義します。**

*種別 入出力設計 ・ ステータス ドラフト*

| 項目 | 値 |
|----|----|
| 入出力設計ID | IO-018 |
| 業務ユースケースID | [UC-030](../../01_requirements/04_business_usecases/UC-030.md#UC-030) ・ [UC-031](../../01_requirements/04_business_usecases/UC-031.md#UC-031) |

## 1. 目的

本入出力設計は、オーナー / メンバーが未解決質問の詳細を確認し対応状況を手動で切り替える一連の入力/出力項目を、画面と API を突き合わせて論理項目単位で確定する。橋渡しする画面は [SCR-007](../../02_basic_design/01_frontend/01_screens/SCR-007.md#SCR-007) の初期表示(EVT-01)・状況切替確認(EVT-02〜EVT-04)、API は [API-035](../../02_basic_design/02_backend/03_apis/API-035.md#API-035) の GET / PATCH である。

## 2. 対象範囲

対象とする画面・API・データと、対象外を明示する。未解決質問の登録経路・FAQ 化への遷移後の項目は別文書へ委ねる。

| 区分 | 対象 |
|----|----|
| 対象画面 | [SCR-007](../../02_basic_design/01_frontend/01_screens/SCR-007.md#SCR-007) 要対応の質問詳細(初期表示 EVT-01・状況切替確認 EVT-02〜EVT-04) |
| 対象API | [API-035](../../02_basic_design/02_backend/03_apis/API-035.md#API-035) 未解決質問詳細・状況切替(GET / PATCH) |
| 対象データ | 対象未解決質問の識別・状況(`open`/`closed`)切替、応答ログ・信頼度・未解決理由等の詳細表示項目 |
| 対象外 | 未解決質問の登録([IO-001](IO-001.md#IO-001))・FAQ 化への遷移([SCR-007](../../02_basic_design/01_frontend/01_screens/SCR-007.md#SCR-007) EVT-05・[SCR-008](../../02_basic_design/01_frontend/01_screens/SCR-008.md#SCR-008))・状態遷移そのものの契機とガード条件([STS-001](../01_state_transitions/STS-001.md#STS-001)) |

## 3. 入力項目一覧

質問詳細画面から受け取り API へ渡す入力項目を論理項目単位で定義する。

| 項目ID | 項目名 | データ型 | 必須 | 初期値 | 入力制約 | 入力例 | 備考 |
|----|----|----|----|----|----|----|----|
| I-01 | 対象未解決質問 ID | 文字列 | ◯ | — | 有効な未解決質問識別子 | — | 画面遷移時に確定(SCR-006 からの問い合わせ ID クリック等)。API パスパラメータ `id`。画面には表示しない |
| I-02 | 更新後状況 | 列挙値 | ◯(PATCH 時) | — | `open` / `closed`([状態モデル §4.2](../../02_basic_design/08_state-model.md#42-未解決質問状態)) | `closed` | SCR-007 EVT-03 の確認ダイアログ OK 押下で確定。API リクエストボディ `status` |
| I-03 | 利用者セッション | 文字列 | ◯ | — | 有効な利用者セッション | — | Cookie 認証。画面には表示しない([API-035](../../02_basic_design/02_backend/03_apis/API-035.md#API-035) 認証ヘッダ) |
| I-04 | CSRF トークン | 文字列 | ◯(PATCH 時) | — | 有効な CSRF トークン | — | PATCH のみ必須。画面には表示しない([API-035](../../02_basic_design/02_backend/03_apis/API-035.md#API-035) 認証ヘッダ) |
| I-05 | 冪等キー | 文字列 | — | — | PATCH 再送の重複防止用の一意識別子 | — | 任意。SCR-007 EVT-03 の PATCH 再送時の重複防止([API-035](../../02_basic_design/02_backend/03_apis/API-035.md#API-035) 認証ヘッダ `Idempotency-Key`) |

## 4. 出力項目一覧

API レスポンス・画面へ返す出力項目を論理項目単位で定義する。GET(初期表示)・PATCH(状況切替)いずれも同一の詳細項目を返す。

| 項目ID | 項目名 | データ型 | 表示形式 | 表示条件 | 備考 |
|----|----|----|----|----|----|
| O-01 | ページタイトル(質問内容) | 文字列 | ページタイトルに表示 | 常時 | SCR-007 §4 #2 に対応。API `question` |
| O-02 | 状況 | 列挙値 | 状況バッジ(SCR-007 §4 #6) | 常時 | `open`/`closed`。値の意味は §5 の列挙値表参照。API `status` |
| O-03 | 問い合わせ ID | 文字列 | パンくず・問い合わせ ID 欄に表示(SCR-007 §4 #1・#7) | 常時 | 利用者向け管理識別子([TBL-017](../../02_basic_design/02_backend/04_database/TBL-017.md#TBL-017) `inquiry_code` 採番体系)。API `inquiryCode` |
| O-04 | 未解決理由 | 文字列 | 未解決理由欄に表示(SCR-007 §4 #8) | 常時(未記録時は非表示) | コード→表示用ラベル変換結果。値の全集合は API-035 参照。API `reasonLabel` |
| O-05 | 発生日時 | 日時 | 発生日時欄に表示(SCR-007 §4 #9) | 常時 | 未解決質問の登録日時。API `createdAt` |
| O-06 | チャネル | 文字列 | チャネル欄に表示(SCR-007 §4 #10) | 常時 | 現 MVP は単一チャネル(ウィジェット)固定表示。API `channelLabel`・`projectName` |
| O-07 | 応答ログ | 配列 | 応答ログ欄に時系列表示(SCR-007 §4 #5) | 常時 | エンドユーザー質問・AI 応答を時系列表示。API `responseLog`(`role`/`message`/`postedAt`) |
| O-08 | 信頼度 | 数値 | 画面非表示 | 応答ログの AI 発話に付随するとき | AI 応答の信頼度(0.0〜1.0)。しきい値正本は [システム仕様書 §1](../../02_basic_design/07_system-spec.md#1-aiしきい値)。API `confidence`・`responseLog[].confidence` |
| O-09 | 対応済みにするボタン活性状態 | 真偽値 | ボタン表示/非表示(SCR-007 §4 #3) | 状況(O-02)が `open` のとき表示 | `closed` のとき非表示。SCR-007 EVT-01 の表示制御 |

## 5. 入力チェック仕様

各入力項目のチェック内容・実施主体(クライアント / サーバー)・エラー時参照を定義する。文面は本ページに再掲せず ID 参照する。

| 項目名 | チェック内容 | タイミング | エラー時メッセージ | 備考 |
|----|----|----|----|----|
| 対象未解決質問 ID(I-01) | 存在チェック・当該プロジェクトへの割当チェック(境界判定) | サーバー | [ERR-001](../../02_basic_design/05_errors/ERR-001.md#ERR-001) | 割当なし・部外者は境界判定により 404 偽装([API-035](../../02_basic_design/02_backend/03_apis/API-035.md#API-035) P-01・P-02) |
| 更新後状況(I-02) | 列挙値チェック(`open`/`closed` 以外を拒否) | サーバー | [ERR-001](../../02_basic_design/05_errors/ERR-001.md#ERR-001) | PATCH 時のみ。未指定時は更新しない([API-035](../../02_basic_design/02_backend/03_apis/API-035.md#API-035) `## リクエスト`) |
| 更新後状況(I-02) | 楽観的排他チェック(要求時点から対象が他操作で変化していないか) | サーバー | [SCR-007](../../02_basic_design/01_frontend/01_screens/SCR-007.md#SCR-007) EM-01 | 変化していた場合は更新を確定せず状況を変更前へ戻す([STS-001](../01_state_transitions/STS-001.md#STS-001) §5・[UC-031](../../01_requirements/04_business_usecases/UC-031.md#UC-031) 例外フロー) |
| 利用者セッション(I-03) | セッション有効性チェック | サーバー | [エラー設計](../../02_basic_design/05_errors/index.md) の標準エラー体系に従う([API-035](../../02_basic_design/02_backend/03_apis/API-035.md#API-035) に固有エラーコードなし) | GET / PATCH 共通([API-035](../../02_basic_design/02_backend/03_apis/API-035.md#API-035) P-01) |
| CSRF トークン(I-04) | CSRF トークン検証 | サーバー | [エラー設計](../../02_basic_design/05_errors/index.md) の標準エラー体系に従う([API-035](../../02_basic_design/02_backend/03_apis/API-035.md#API-035) に固有エラーコードなし) | PATCH のみ([API-035](../../02_basic_design/02_backend/03_apis/API-035.md#API-035) P-01) |

## 6. 編集・加工仕様

入力値の正規化・出力値の整形など編集加工を定義する。

| 項目名 | 加工内容 | 加工タイミング | 備考 |
|----|----|----|----|
| 未解決理由(O-04) | コード値から表示用ラベルへ変換 | 応答生成時(サーバー) | `reason` → `reasonLabel` に変換して返却([API-035](../../02_basic_design/02_backend/03_apis/API-035.md#API-035)) |
| チャネル(O-06) | チャネル種別コードから表示用ラベルへ変換し、プロジェクト名を付随 | 応答生成時(サーバー) | `channel` → `channelLabel` に変換し `projectName` を併せて返却([API-035](../../02_basic_design/02_backend/03_apis/API-035.md#API-035) P-03) |
| 対応済みにするボタン活性状態(O-09) | 状況(O-02)から表示/非表示を導出 | 画面描画時(クライアント) | `closed` のとき非表示に切替(SCR-007 EVT-01) |

## 7. 後続工程への引き継ぎ事項

テスト設計・DB 物理設計・詳細ロジック設計へ引き継ぐ観点を挙げる。状況切替の競合制御・トランザクション境界の実装方針は別文書へ委ねる。

- 状況(I-02/O-02)の `open ↔ closed` 双方向切替と、楽観的排他が成立する/しないケース(対象が他操作で変化している場合)はテスト設計でケース化する([STS-001](../01_state_transitions/STS-001.md#STS-001) §7)。
- 未解決理由(O-04)・信頼度(O-08)が未記録(null)の場合の表示分岐はテスト設計でケース化する。
- 楽観的排他の実装方式(要求時点の検出条件・Repository 更新の原子性)は詳細ロジック設計へ引き継ぐ。
- 論理項目 ↔ 物理カラムの対応は [DBP-001](../07_db_physical/DBP-001.md#DBP-001) で確定する。
