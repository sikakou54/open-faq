# <span id="IO-017"></span>IO-017: 要対応の質問一覧 入出力

> **この入出力設計書は「要対応の質問一覧画面と未解決質問一覧 API・未解決質問 CSV エクスポート API の入力/出力項目・入力チェック・編集加工」を論理項目単位で定義します。**

*種別 入出力設計 ・ ステータス ドラフト*

| 項目 | 値 |
|----|----|
| 入出力設計ID | IO-017 |

## 1. 目的

本入出力設計は、オーナー / メンバーが未解決質問を状況・期間・キーワードで絞り込んで一覧確認し、必要に応じて CSV 書き出しを行う一連の入力/出力項目を、画面と API を突き合わせて論理項目単位で確定する。橋渡しする画面は [SCR-006](../../02_basic_design/01_frontend/01_screens/SCR-006.md#SCR-006) の一覧表示・絞込・書き出し(EVT-01〜EVT-04・EVT-06・EVT-07)、API は [API-034](../../02_basic_design/02_backend/03_apis/API-034.md#API-034)(一覧取得)・[API-036](../../02_basic_design/02_backend/03_apis/API-036.md#API-036)(CSV エクスポート)である。

## 2. 対象範囲

対象とする画面・API・データと、対象外を明示する。未解決質問詳細・ウィジェット設定への遷移先の内容は別文書へ委ねる。

| 区分 | 対象 |
|----|----|
| 対象画面 | [SCR-006](../../02_basic_design/01_frontend/01_screens/SCR-006.md#SCR-006) 要対応の質問一覧(状況フィルタ・期間フィルタ・検索・CSV エクスポート・ページネーション・空状態 EVT-01〜EVT-04・EVT-06・EVT-07) |
| 対象API | [API-034](../../02_basic_design/02_backend/03_apis/API-034.md#API-034) 未解決質問一覧 ・ [API-036](../../02_basic_design/02_backend/03_apis/API-036.md#API-036) 未解決質問 CSV エクスポート |
| 対象データ | 状況・期間・フリーワードの絞込条件、未解決質問の一覧項目、CSV 書き出し結果 |
| 対象外 | 問い合わせ ID リンク押下後の詳細表示(SCR-006 EVT-05 / SCR-007)・空状態からのウィジェット設定遷移(SCR-006 EVT-08 / SCR-011)・未解決質問の状態遷移そのもの |

## 4. 出力項目一覧

API レスポンス・画面へ返す出力項目を論理項目単位で定義する。一覧表示と CSV エクスポートで出力の形式・列構成が異なる。

| 項目ID | 項目名 | データ型 | 表示形式 | 表示条件 | 備考 |
|----|----|----|----|----|----|
| O-01 | 問い合わせ ID | 文字列 | 一覧行のリンク | 1 件以上ある時 | 押下で SCR-007 へ遷移([SCR-006](../../02_basic_design/01_frontend/01_screens/SCR-006.md#SCR-006) §4 #7)。API `items[].inquiryCode` |
| O-02 | 状況 | 列挙値 | 一覧行のラベル | 1 件以上ある時 | 取りうる値と意味は §列挙値。SCR-006 §4 #8。API `items[].status` |
| O-03 | 質問 | 文字列 | 一覧行のラベル | 1 件以上ある時 | ウィジェット利用者が送信した質問内容。SCR-006 §4 #9。API `items[].question` |
| O-04 | 未解決理由 | 列挙値 | 一覧行のラベル | 1 件以上ある時 | 取りうる値と意味は §列挙値。信頼度しきい値未達の表示は信頼度(O-06)を伴う([SCR-006](../../02_basic_design/01_frontend/01_screens/SCR-006.md#SCR-006) データパターン #10)。SCR-006 §4 #10。API `items[].resultReason` |
| O-05 | 日時 | 文字列(日時) | 一覧行のラベル | 1 件以上ある時 | 未解決質問の登録日時。SCR-006 §4 #11。API `items[].createdAt` |
| O-06 | 信頼度 | 数値 | 未解決理由(O-04)のラベル内に付帯表示 | 未解決理由が信頼度しきい値未達の時 | しきい値は [システム仕様書 §1](../../02_basic_design/07_system-spec.md#1-aiしきい値)。API `items[].confidence` |
| O-07 | 件数表示 | 数値 | ラベル | 1 件以上ある時 | 取得件数の表示。SCR-006 §4 #12 |
| O-08 | 次ページカーソル | 文字列 | 画面非表示 | 常時 | ページネーション(SCR-006 §4 #13)の次ページ取得制御に使用。末尾は `null`。API `nextCursor` |
| O-09 | CSV ファイル | ファイル(text/csv) | ファイルダウンロード | CSV エクスポート成功時 | 列構成は問い合わせID・状況・質問・未解決理由・最終更新日時([API-036](../../02_basic_design/02_backend/03_apis/API-036.md#API-036) `## レスポンス`)。SCR-006 §4 #4 押下の結果 |

## 5. 入力チェック仕様

各入力項目のチェック内容・実施主体(クライアント / サーバー)・エラー時参照を定義する。本画面はサーバー側の絞込条件検証のみで、クライアント側の入力形式検証は行わない。

| 項目名 | チェック内容 | タイミング | エラー時メッセージ | 備考 |
|----|----|----|----|----|
| 期間フィルタ・開始日(I-02) / 終了日(I-03) | 日付形式チェック | サーバー | [ERR-001](../../02_basic_design/05_errors/ERR-001.md#ERR-001) | サーバーのみ判定。ISO 8601 形式でない場合は入力検証エラー |
| 対象プロジェクト(I-05) | 当該プロジェクトへのアクセス権検証 | サーバー | [ERR-001](../../02_basic_design/05_errors/ERR-001.md#ERR-001) | [API-034](../../02_basic_design/02_backend/03_apis/API-034.md#API-034) / [API-036](../../02_basic_design/02_backend/03_apis/API-036.md#API-036) P-01 |
| 利用者セッション(I-07) | セッショントークンの有効性検証 | サーバー | [ERR-001](../../02_basic_design/05_errors/ERR-001.md#ERR-001) | 認証失敗時はログイン画面へ誘導する標準エラー体系に従う |

## 列挙値

出力の状況(O-02)と未解決理由(O-04)の取りうる値と意味を示す。値の全集合の正本は [API-034](../../02_basic_design/02_backend/03_apis/API-034.md#API-034) `## レスポンス`、状況の意味は [状態モデル §4.2](../../02_basic_design/08_state-model.md#42-未解決質問状態)。

| 項目 | 取りうる値 | 意味 |
|----|----|----|
| 状況(O-02) | `open` | 対応中。[状態モデル §4.2](../../02_basic_design/08_state-model.md#42-未解決質問状態) |
| 状況(O-02) | `closed` | 対応済み。[状態モデル §4.2](../../02_basic_design/08_state-model.md#42-未解決質問状態) |
| 未解決理由(O-04) | `no_faq_match` | 該当 FAQ なし |
| 未解決理由(O-04) | `low_confidence` | 信頼度しきい値未達([システム仕様書 §1](../../02_basic_design/07_system-spec.md#1-aiしきい値)) |
| 未解決理由(O-04) | `contradiction` | FAQ 間で内容が矛盾 |
| 未解決理由(O-04) | `user_unresolved` | 利用者が回答後に未解決と回答 |

## 7. 後続工程への引き継ぎ事項

テスト設計・DB 物理設計・詳細ロジック設計へ引き継ぐ観点を挙げる。

- 状況フィルタ(I-01)の複数選択・未選択(両方対象)の組み合わせ、期間フィルタ(I-02・I-03)の開始日のみ/終了日のみ/両方指定/未指定の組み合わせはテスト設計でケース化する。
- 検索結果 0 件時の空状態表示(SCR-006 §4 #14・#15)と、未解決質問が一度も発生していない場合の空状態の表示条件はテスト設計で区別して確認する。
- CSV エクスポート(O-09)は現在の絞込条件(状況・期間・フリーワード)を引き継ぐこと、および失敗時に一覧表示自体は維持されることをテスト設計でケース化する。
- 未解決理由(O-04)・信頼度(O-06)の供給元である質問ログとの結合仕様は [API-034](../../02_basic_design/02_backend/03_apis/API-034.md#API-034) を正本とし、論理項目 ↔ 物理カラムの対応は DB 物理設計で確定する。
