> **この入出力設計書は「利用状況(オーナー全体)画面と利用量サマリ(オーナー)API・利用量サマリ(プロジェクト)API の入力/出力項目・入力チェック・編集加工」を論理項目単位で定義します。**

|----|----|

## 1. 目的

本入出力設計は、オーナーが自分の作成したプロジェクト全体の当月利用状況(全体サマリー・プロジェクト別内訳)を閲覧する一連の入力/出力項目を、画面と API を突き合わせて論理項目単位で確定する。橋渡しする画面は [SCR-021](../../02_basic_design/01_frontend/01_screens/SCR-021.md#SCR-021) の初期表示(EVT-01)、API は [API-042](../../02_basic_design/02_backend/03_apis/API-042.md#API-042)(オーナー全体サマリー)と [API-041](../../02_basic_design/02_backend/03_apis/API-041.md#API-041)(プロジェクト別内訳)である。

## 2. 対象範囲

本設計が対象とする画面・API・データと対象外を明示する。請求金額・支払方法・請求履歴、画面遷移先の詳細入出力は別文書へ委ねる。

| 区分 | 対象 |
|----|----|
| 対象画面 | [SCR-021](../../02_basic_design/01_frontend/01_screens/SCR-021.md#SCR-021) 利用状況(初期表示 EVT-01) |
| 対象API | [API-042](../../02_basic_design/02_backend/03_apis/API-042.md#API-042) 利用量サマリ(オーナー) ・ [API-041](../../02_basic_design/02_backend/03_apis/API-041.md#API-041) 利用量サマリ(プロジェクト) |
| 対象データ | 対象期間(当月固定)・全体サマリー(質問数 / 解決率 / アクティブプロジェクト数 / AI 推論コスト / 制限中プロジェクト数 / 前月実績)・プロジェクト別の質問数消化率内訳・支払い警告・空状態 |
| 対象外 | 請求金額・支払方法・請求履歴([SCR-028](../../02_basic_design/01_frontend/01_screens/SCR-028.md#SCR-028) の入出力)・「請求を確認」(EVT-02)/「プロジェクトへ」(EVT-03)押下後の遷移先画面の入出力・利用量集計ロジック自体([SYS-016](../../02_basic_design/02_backend/01_system/SYS-016.md#SYS-016)) |

## 3. 入力項目一覧

初期表示画面から API へ渡す入力項目を論理項目単位で定義する。本画面は利用者が値を入力する項目を持たず(SCR-021 §5)、初期表示時にサーバーへ送出する識別情報のみを扱う。

| 項目ID | 項目名 | データ型 | 必須 | 初期値 | 入力制約 | 入力例 | 備考 |
|----|----|----|----|----|----|----|----|
| I-01 | セッション認証(ログイン中オーナー) | — | ◯ | — | 有効なログインセッションであり、かつオーナー専有画面であること | — | 認証ヘッダで送信。画面には表示しない([API-042](../../02_basic_design/02_backend/03_apis/API-042.md#API-042) `## リクエスト` 認証ヘッダ P-01 / [API-041](../../02_basic_design/02_backend/03_apis/API-041.md#API-041) `## リクエスト` 認証ヘッダ P-01) |
| I-02 | 対象期間 | 列挙値 | — | `current_month`(当月固定) | 固定値 `current_month` のみ。画面に期間セレクトは設けない([API-042](../../02_basic_design/02_backend/03_apis/API-042.md#API-042) `## リクエスト` Query Parameters `period`) | `current_month` | API-042 `period`。画面からは送出せず既定値のまま呼び出す |
| I-03 | 対象プロジェクト識別子 | 文字列 | ◯(API-041 呼び出し時) | — | 自分が作成したプロジェクトの有効な識別子 | — | API-042 応答のプロジェクト別内訳(O-08)を基にプロジェクトごとに API-041 を呼び出す際の指定値([API-041](../../02_basic_design/02_backend/03_apis/API-041.md#API-041) `## リクエスト` Query Parameters `projectId`) |

## 4. 出力項目一覧

API レスポンス・画面へ返す出力項目を論理項目単位で定義する。API-042(オーナー全体サマリー)を主とし、支払い警告・空状態により表示項目が分岐する。

| 項目ID | 項目名 | データ型 | 表示形式 | 表示条件 | 備考 |
|----|----|----|----|----|----|
| O-01 | 対象期間ラベル | 文字列(ISO 8601 の起点・終点) | 対象期間ラベルに表示(SCR-021 §4 #1) | 通常時 | 当月固定。API-042 `period.from` / `period.to` |
| O-02 | 自分が作成したプロジェクト数 | 整数 | 画面非表示 | 常時 | 空状態判定(0 件)に用いる。API-042 `summary.projectCount` |
| O-03 | 当月の質問数(全体) | 整数 | 全体サマリーカードに表示(SCR-021 §4 #2) | 通常時 | API-042 `summary.questionCount` |
| O-04 | 解決率(全体) | 数値(%) | 全体サマリーカードに表示(SCR-021 §4 #2) | 通常時 | 算出方法は [API-042](../../02_basic_design/02_backend/03_apis/API-042.md#API-042) `## レスポンス` `summary.resolutionRate` を正本とする。API-042 `summary.resolutionRate` |
| O-05 | アクティブプロジェクト数 | 整数 | 全体サマリーカードに表示(SCR-021 §4 #2) | 通常時 | API-042 `summary.projectCount` |
| O-06 | 制限中プロジェクト数 | 整数 | アクティブプロジェクトカードの内訳に表示(SCR-021 §4 #2) | 通常時 | 当月質問数が月次上限到達(利用率 100% 以上)または課金停止中のプロジェクト件数([API-042](../../02_basic_design/02_backend/03_apis/API-042.md#API-042) `## レスポンス` `summary.limitedProjectCount`)。API-042 `summary.limitedProjectCount` |
| O-07 | 当月 AI 推論コスト(全体) | 整数(円) | 画面非表示 | 常時 | API-042 `summary.aiCostYen` |
| O-08 | プロジェクト別内訳 | 配列 | プロジェクト別の質問数消化率パネルに表示(SCR-021 §4 #3) | 通常時 | 要素ごとにプロジェクト名・質問数利用状況(使用数 / 上限有無 / 上限件数 / 利用率)・公開 FAQ 件数を保持。API-042 `projects[]` |
| O-09 | 前月実績(全体) | object | 全体サマリーカードの前月比に表示(SCR-021 §4 #2) | 通常時 | 前月比は本値と当月値(O-03〜O-04・O-07)から画面側で算出([API-042](../../02_basic_design/02_backend/03_apis/API-042.md#API-042) `## レスポンス` `summary.previousMonth`)。API-042 `summary.previousMonth` |
| O-10 | プロジェクト別の質問数利用状況(詳細) | object | 画面非表示 | 通常時 | API-041 `questions`(`used` / `limit` / `freeQuota` / `percentage` / `resetAt`)。O-08 のプロジェクト別内訳と同一値域を補足する詳細出力(API-041 側)。SCR-021 は API-042 の `projects[]` を表示に用いる |
| O-11 | 支払い警告状態 | 真偽値 | 支払い警告バナー(SCR-021 §4 #4 EM-01)・請求を確認ボタン(SCR-021 §4 #5)の表示要否を判定 | 支払方法未登録または支払い失敗時 | 専用フラグ項目はなく、画面側が課金アカウント状態([状態モデル §2](../../02_basic_design/08_state-model.md#2-課金アカウント状態) `suspended`)等の判定結果から確定する |
| O-12 | 空状態(作成したプロジェクトなし) | 真偽値 | 空状態案内(SCR-021 §4 #6)・プロジェクトへ CTA(SCR-021 §4 #7)の表示要否を判定 | 自分が作成したプロジェクトが 0 件のとき | O-02(`summary.projectCount` = 0)から画面側で判定 |
| O-13 | 集計値の最終更新日時 | 文字列(ISO 8601) | 画面非表示 | 常時 | API-042 `updatedAt` |

## 5. 入力チェック仕様

各入力項目のチェック内容・実施主体(クライアント / サーバー)・エラー時参照を定義する。本画面は利用者入力を持たないため(SCR-021 §5)、チェックはすべてサーバー側の認証・認可・パラメータ検証である。

| 項目名 | チェック内容 | タイミング | エラー時メッセージ | 備考 |
|----|----|----|----|----|
| セッション認証(I-01) | セッション有効性チェック | サーバー | [ERR-033](../../02_basic_design/05_errors/ERR-033.md#ERR-033) | 全認証保護 API 共通([エラー設計 §3](../../02_basic_design/05_errors/index.md#authz-map))。期限切れ時は再ログインへ誘導しサマリーを表示しない |
| セッション認証(I-01) | オーナー専有チェック(境界判定) | サーバー | [ERR-017](../../02_basic_design/05_errors/ERR-017.md#ERR-017) | オーナーとして作成したプロジェクトを持たない場合は境界判定で 404 偽装([エラー設計 §3](../../02_basic_design/05_errors/index.md#authz-map)) |
| 対象期間(I-02) | 固定値チェック(`current_month` 以外を許容しない) | サーバー | [ERR-001](../../02_basic_design/05_errors/ERR-001.md#ERR-001) | API-042 は当月固定集計のみを提供する([API-042](../../02_basic_design/02_backend/03_apis/API-042.md#API-042) `## リクエスト` Query Parameters) |
| 対象プロジェクト識別子(I-03) | 対象プロジェクトへのアクセス権チェック | サーバー | [ERR-017](../../02_basic_design/05_errors/ERR-017.md#ERR-017) | API-041 呼び出し時のみ実施([API-041](../../02_basic_design/02_backend/03_apis/API-041.md#API-041) P-01) |

## 6. 編集・加工仕様

入力値の正規化・出力値の整形など編集加工を定義する。前月比の算出は画面側で行う。

| 項目名 | 加工内容 | 加工タイミング | 備考 |
|----|----|----|----|
| 前月比(O-09 由来) | 当月値(O-03〜O-04・O-07)と前月実績(O-09)から前月比(質問数増減率・解決率ポイント差・AI 推論コスト増減率)を算出 | 画面表示時(クライアント) | API はいずれの値も百分率・差分値として加工せず個別に返す |
| プロジェクト別内訳(O-08) | 質問数利用率(%)に応じてプログレスバーの強調表示(上限到達時は警告配色)を切替 | 画面表示時(クライアント) | 100% 以上のときに強調配色へ切替(SCR-021 §4 #3 の表示形式) |
| 支払い警告状態(O-11) | サマリー取得結果と課金アカウント状態から支払い警告要否を判定 | 応答受信時(クライアント) | 判定条件は [状態モデル §2](../../02_basic_design/08_state-model.md#2-課金アカウント状態) を正本とする |

## 7. 後続工程への引き継ぎ事項

テスト設計・DB 物理設計・詳細ロジック設計へ引き継ぐ観点を挙げる。利用量集計ロジック自体の詳細は別文書へ委ねる。

- 通常時・支払い警告時・空状態(作成したプロジェクト 0 件)の 3 状態分岐(SCR-021 §7 EVT-01)をテスト設計でケース化する。
- プロジェクト別内訳(O-08)の件数(0 件 / 1 件 / 複数件)と利用率境界値(0% / 100% 未満 / 100% 到達)の表示分岐をテスト設計でケース化する。
- API-042 と API-041 を初期表示時にどう組み合わせて呼び出すか(呼び出し順・API-041 の呼び出し要否)の実装方式は詳細ロジック設計へ引き継ぐ。
- 論理項目 ↔ 物理カラムの対応は [DBP-001](../07_db_physical/DBP-001.md#DBP-001) で確定する。
