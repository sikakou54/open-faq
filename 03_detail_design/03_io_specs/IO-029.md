## 1. 目的

本入出力設計は、利用者が利用規約・プライバシーポリシーの最新版を閲覧し、ログイン済みの場合は自身の同意状態を併せて確認する一連の入力/出力項目を、画面と API を突き合わせて論理項目単位で確定する。橋渡しする画面は [SCR-015](../../02_basic_design/01_frontend/01_screens/SCR-015.md#SCR-015)(利用規約閲覧)・[SCR-025](../../02_basic_design/01_frontend/01_screens/SCR-025.md#SCR-025)(プライバシーポリシー閲覧)、API は [API-052](../../02_basic_design/02_backend/03_apis/API-052.md#API-052)(利用規約 最新版取得)・[API-053](../../02_basic_design/02_backend/03_apis/API-053.md#API-053)(プライバシーポリシー 最新版取得)である。

## 2. 対象範囲

本設計が対象とする画面・API・データと対象外を明示する。再同意の申請・記録は別文書へ委ねる。

| 区分 | 対象 |
|----|----|
| 対象画面 | [SCR-015](../../02_basic_design/01_frontend/01_screens/SCR-015.md#SCR-015) 利用規約閲覧(EVT-01)・[SCR-025](../../02_basic_design/01_frontend/01_screens/SCR-025.md#SCR-025) プライバシーポリシー閲覧(EVT-01) |
| 対象API | [API-052](../../02_basic_design/02_backend/03_apis/API-052.md#API-052) 利用規約 最新版取得・[API-053](../../02_basic_design/02_backend/03_apis/API-053.md#API-053) プライバシーポリシー 最新版取得 |
| 対象データ | 最新版の本文・発効日・版数・主な変更点・目次・同意状態バッジの表示制御 |
| 対象外 | 再同意の申請・記録([SCR-020](../../02_basic_design/01_frontend/01_screens/SCR-020.md#SCR-020)・API-054・API-055、別 IO で扱う)・過去バージョン履歴・差分表示 |

## 3. 入力項目一覧

両画面とも閲覧専用であり、利用者が入力する項目はない。画面初期表示時に画面遷移元から受け取る対象文書種別のみを入力項目として扱う。

| 項目ID | 項目名 | データ型 | 必須 | 初期値 | 入力制約 | 入力例 | 備考 |
|----|----|----|----|----|----|----|----|
| I-01 | 対象文書種別 | 列挙値 | ◯ | 画面固定([SCR-015](../../02_basic_design/01_frontend/01_screens/SCR-015.md#SCR-015) は `terms_of_service`・[SCR-025](../../02_basic_design/01_frontend/01_screens/SCR-025.md#SCR-025) は `privacy_policy`) | — | `terms_of_service` | 画面ごとに呼び出す API([API-052](../../02_basic_design/02_backend/03_apis/API-052.md#API-052) / [API-053](../../02_basic_design/02_backend/03_apis/API-053.md#API-053))が固定される。利用者は選択しない |
| I-02 | 認証情報(セッション) | 文字列 | — | — | ログイン済みの場合のみ有効な認証情報を送信 | — | 未ログイン時は送信しない。同意状態バッジ(O-05〜O-07)の表示可否を左右する([API-053](../../02_basic_design/02_backend/03_apis/API-053.md#API-053) P-04) |

| 項目ID | 項目名 | データ型 | 表示形式 | 表示条件 | 備考 |
|----|----|----|----|----|----|
| O-01 | 発効日・版数 | 文字列 | ラベル表示([SCR-015](../../02_basic_design/01_frontend/01_screens/SCR-015.md#SCR-015) §4 #1) | 常時 | API `effectiveDate` / `version` |
| O-02 | 最新版の全文 | 文字列(HTML) | 本文表示([SCR-015](../../02_basic_design/01_frontend/01_screens/SCR-015.md#SCR-015) §4 #4 / [SCR-025](../../02_basic_design/01_frontend/01_screens/SCR-025.md#SCR-025) §4 #5) | 常時 | API `bodyHtml` |
| O-03 | 主な変更点 | 文字列 | — | 画面には表示しない([SCR-015](../../02_basic_design/01_frontend/01_screens/SCR-015.md#SCR-015)・[SCR-025](../../02_basic_design/01_frontend/01_screens/SCR-025.md#SCR-025)いずれも§4に表示項目なし) | API `diffSummary`。NULL の場合は省略。改定時の主な変更点表示は SCR-020(再同意画面)で用いる |
| O-04 | 目次(章ナビ) | 配列 | 目次リンク表示([SCR-025](../../02_basic_design/01_frontend/01_screens/SCR-025.md#SCR-025) §4 #1) | [SCR-025](../../02_basic_design/01_frontend/01_screens/SCR-025.md#SCR-025) のみ・常時 | API `sections`(`anchorId` / `title` / `order`)。本文 HTML の見出しから構成([API-053](../../02_basic_design/02_backend/03_apis/API-053.md#API-053) P-03)。[SCR-015](../../02_basic_design/01_frontend/01_screens/SCR-015.md#SCR-015) には目次項目がなく非対応 |
| O-05 | 同意済みフラグ | 真偽値 | 同意状態バッジ表示([SCR-025](../../02_basic_design/01_frontend/01_screens/SCR-025.md#SCR-025) §4 #2/#3) | 認証済みリクエスト時のみ | API `consentStatus.agreed`。未認証時は `consentStatus` ごと省略しバッジ非表示 |

## 5. 入力チェック仕様

両画面とも閲覧専用で利用者入力欄を持たないため、入力チェックは対象文書種別の解決と認証状態の判定に限られる。エラー時メッセージは本ページに再掲せず ID 参照する。

| 項目名 | チェック内容 | タイミング | エラー時メッセージ | 備考 |
|----|----|----|----|----|
| 対象文書種別(I-01) | 呼び出し先 API の固定解決(画面遷移元の指定に依存しない) | クライアント | — | 誤った API を呼び出す実装不備を防止する設計上の固定制約であり、利用者入力に起因する検証エラーはない |
| 認証情報(I-02) | セッションの有効性検証(認証ヘッダを送信した場合のみ) | サーバー | — | 本 API は公開エンドポイントのため未認証でも取得は成立する([API-052](../../02_basic_design/02_backend/03_apis/API-052.md#API-052)・[API-053](../../02_basic_design/02_backend/03_apis/API-053.md#API-053) `## リクエスト` 認証ヘッダ)。認証ヘッダを送信したがセッションが無効な場合はエラーとせず未認証として扱い `consentStatus` を省略する |
| 最新版取得結果 | 取得失敗時の画面表示制御 | サーバー | — | 取得失敗時は全文([SCR-015](../../02_basic_design/01_frontend/01_screens/SCR-015.md#SCR-015) §4 #4 / [SCR-025](../../02_basic_design/01_frontend/01_screens/SCR-025.md#SCR-025) §4 #5)を表示せず内容を表示できない旨を示す([UC-011](../../01_requirements/04_business_usecases/UC-011.md#UC-011) 例外フロー / [UC-012](../../01_requirements/04_business_usecases/UC-012.md#UC-012) 例外フロー)。[API-052](../../02_basic_design/02_backend/03_apis/API-052.md#API-052)・[API-053](../../02_basic_design/02_backend/03_apis/API-053.md#API-053) の `## エラー` は本 API 固有のエラーコードなしと定義しており本ページで新たなエラーコードを割り当てない。[SCR-015](../../02_basic_design/01_frontend/01_screens/SCR-015.md#SCR-015) §8 の `EM-01` は再同意記録(API-054)失敗時の文面であり本チェックには用いない |

## 6. 編集・加工仕様

出力値の表示制御に関する編集加工を定義する。本文自体の整形・変換は行わない。

| 項目名 | 加工内容 | 加工タイミング | 備考 |
|----|----|----|----|
| 主な変更点(O-03) | 値が NULL の場合は出力項目自体を省略する | 応答生成時(サーバー) | [API-052](../../02_basic_design/02_backend/03_apis/API-052.md#API-052)・[API-053](../../02_basic_design/02_backend/03_apis/API-053.md#API-053) 共通 |
| 同意状態(O-05〜O-07) | 未認証時は `consentStatus` を丸ごと省略する | 応答生成時(サーバー) | 画面側は `consentStatus` の有無で同意状態バッジ全体の表示/非表示を切り替える |
| 目次(O-04) | 本文 HTML の見出しから章ナビ用の見出し構造を構成する | 応答生成時(サーバー) | [API-053](../../02_basic_design/02_backend/03_apis/API-053.md#API-053) のみ。[API-052](../../02_basic_design/02_backend/03_apis/API-052.md#API-052) は目次を構成しない([SCR-015](../../02_basic_design/01_frontend/01_screens/SCR-015.md#SCR-015) に目次項目なし) |

## 7. 後続工程への引き継ぎ事項

テスト設計・DB 物理設計・詳細ロジック設計へ引き継ぐ観点を挙げる。

- 認証済み/未認証、同意済み/未同意/再同意要の組み合わせによる同意状態バッジ(O-05〜O-07)の表示分岐はテスト設計でケース化する。
- 主な変更点(O-03)が NULL のときの省略表示、目次(O-04)の見出し 0 件時の表示はテスト設計でパターン化する。
- 最新版取得失敗時の画面挙動([UC-011](../../01_requirements/04_business_usecases/UC-011.md#UC-011)・[UC-012](../../01_requirements/04_business_usecases/UC-012.md#UC-012) 例外フロー)は詳細ロジック設計で確定する。
- 論理項目 ↔ 物理カラムの対応は [DBP-003](../07_db_physical/DBP-003.md#DBP-003)(`M_TERMS_VER` / `T_TERMS_AGREE` 物理設計)で確定する。
