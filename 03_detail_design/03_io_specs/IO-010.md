# <span id="IO-010"></span>IO-010: プロジェクト一覧 入出力

> **この入出力設計書は「プロジェクト一覧画面とプロジェクト一覧 API の入力/出力項目・入力チェック・編集加工」を論理項目単位で定義します。**

*種別 入出力設計 ・ ステータス ドラフト*

| 項目 | 値 |
|----|----|
| 入出力設計ID | IO-010 |
| 業務ユースケースID | [UC-014](../../01_requirements/04_business_usecases/UC-014.md#UC-014) ・ [UC-082](../../01_requirements/04_business_usecases/UC-082.md#UC-082) |

## 1. 目的

本入出力設計は、ユーザーが関与するプロジェクト(My / Join)を一覧表示するための入力/出力項目を、画面と API を突き合わせて論理項目単位で確定する。橋渡しする画面は [SCR-004](../../02_basic_design/01_frontend/01_screens/SCR-004.md#SCR-004) の初期表示・タブ切替(EVT-01・EVT-02)、API は [API-016](../../02_basic_design/02_backend/03_apis/API-016.md#API-016) である。

## 2. 対象範囲

本設計が対象とする画面・API・データと対象外を明示する。プロジェクトの新規作成・編集・削除、および概要画面への遷移後の詳細取得は別文書へ委ねる。

| 区分 | 対象 |
|----|----|
| 対象画面 | [SCR-004](../../02_basic_design/01_frontend/01_screens/SCR-004.md#SCR-004) プロジェクト(My / Join タブ切替・初期表示 EVT-01 / タブ切替 EVT-02) |
| 対象API | [API-016](../../02_basic_design/02_backend/03_apis/API-016.md#API-016) プロジェクト一覧 |
| 対象データ | 表示対象タブ(scope)・ページング条件、プロジェクトカード表示項目・空状態・立場別(オーナー / メンバー)操作可否 |
| 対象外 | プロジェクト新規作成・編集・削除([SCR-005](../../02_basic_design/01_frontend/01_screens/SCR-005.md#SCR-005))・プロジェクト概要の取得([SCR-012](../../02_basic_design/01_frontend/01_screens/SCR-012.md#SCR-012)) |

## 3. 入力項目一覧

一覧画面から受け取り API へ渡す入力項目を論理項目単位で定義する。本画面は入力フォームを持たず、タブ選択とページング操作のみが入力となる。

| 項目ID | 項目名 | データ型 | 必須 | 初期値 | 入力制約 | 入力例 | 備考 |
|----|----|----|----|----|----|----|----|
| I-01 | 表示対象タブ | 列挙値 | — | `my` | `my`(My プロジェクト)/ `join`(Join プロジェクト) | `my` | SCR-004 §4 #1 に対応。API リクエスト `scope`([API-016](../../02_basic_design/02_backend/03_apis/API-016.md#API-016) Query Parameters) |
| I-02 | 次ページ取得用カーソル | 文字列 | — | — | API-016 発行の有効なカーソル | — | 画面には表示しない。API リクエスト `cursor` |
| I-03 | 1 ページの取得件数 | 数値 | — | — | 共通ページング仕様([API-016](../../02_basic_design/02_backend/03_apis/API-016.md#API-016) Query Parameters) | — | 画面には表示しない。API リクエスト `limit` |
| I-04 | 利用者セッション | 文字列 | ◯ | — | 有効なセッションクッキー | — | 認証ヘッダで送信。画面には表示しない([API-016](../../02_basic_design/02_backend/03_apis/API-016.md#API-016) 認証ヘッダ) |

## 4. 出力項目一覧

API レスポンス・画面へ返す出力項目を論理項目単位で定義する。立場(myRole)により操作導線(新規作成・編集)の表示可否が分岐する。

| 項目ID | 項目名 | データ型 | 表示形式 | 表示条件 | 備考 |
|----|----|----|----|----|----|
| O-01 | プロジェクト識別子 | 文字列 | 画面非表示 | 1 件以上ある時 | プロジェクトカード(SCR-004 §4 #9)・プロジェクト名リンク(#10)・「概要を開く」導線(#2)の遷移先解決に用いる。API `items[].id` |
| O-02 | プロジェクト名 | 文字列 | プロジェクト名(SCR-004 §4 #10) | 1 件以上ある時 | My タブは編集導線のリンク表示、Join タブは非リンク表示([SCR-004](../../02_basic_design/01_frontend/01_screens/SCR-004.md#SCR-004) EVT-01)。API `items[].name` |
| O-03 | プロジェクト状態 | 列挙値 | 画面非表示 | 常時 | 一覧取得対象は `active` のみ。取りうる値と意味は [状態モデル §5](../../02_basic_design/08_state-model.md#5-プロジェクト状態)。API `items[].status` |
| O-04 | 稼働ステータス | 列挙値 | 稼働ステータス(SCR-004 §4 #12) | 1 件以上ある時 | `active`(稼働中)/ `limited`(制限中)。導出条件は [API-016](../../02_basic_design/02_backend/03_apis/API-016.md#API-016) レスポンス `operationalStatus`。表示形式は色分け+テキストラベル併記(SCR-004 §4 データパターン)。API `items[].operationalStatus` |
| O-05 | 立場 | 列挙値 | 画面非表示 | 常時 | `owner`(My)/ `member`(Join)。表示中タブに応じた操作導線の表示制御(§6)に用いる。API `items[].myRole` |
| O-06 | 作成日時 | 文字列(日時) | 画面非表示 | 常時 | API `items[].createdAt` |
| O-07 | 許可ドメイン | 配列 | 許可ドメイン(SCR-004 §4 #11) | 1 件以上ある時 | API `items[].allowedDomains` |
| O-08 | 質問数 | 数値 | 質問数(SCR-004 §4 #13) | 1 件以上ある時 | API `items[].questionCount` |
| O-09 | 未解決数 | 数値 | 未解決数(SCR-004 §4 #14) | 1 件以上ある時 | API `items[].unresolvedCount` |
| O-10 | メンバー数 | 数値 | メンバー数(SCR-004 §4 #15) | 1 件以上ある時 | API `items[].memberCount` |
| O-11 | 次ページ取得用カーソル | 文字列 | 画面非表示 | 常時 | 末尾ページは `null`。追加読込操作の要否判定に用いる。API `nextCursor` |

## 5. 入力チェック仕様

各入力項目のチェック内容・実施主体(クライアント / サーバー)・エラー時参照を定義する。本画面は入力フォームを持たないため検証対象はタブ選択とセッションのみである。

| 項目名 | チェック内容 | タイミング | エラー時メッセージ | 備考 |
|----|----|----|----|----|
| 表示対象タブ(I-01) | 許容値(`my` / `join`)チェック | クライアント | — | SCR-004 §4 #1 のタブ操作は許容値のみを送出するため画面側での不正値入力は発生しない |
| 利用者セッション(I-04) | セッション有効性チェック | サーバー | [ERR-033](../../02_basic_design/05_errors/ERR-033.md#ERR-033) | セッション期限切れ時はログインへ誘導([API-016](../../02_basic_design/02_backend/03_apis/API-016.md#API-016) P-01) |

## 6. 編集・加工仕様

入力値の正規化・出力値の整形など編集加工を定義する。立場(myRole)に応じて画面側の操作導線表示を切り替える。

| 項目名 | 加工内容 | 加工タイミング | 備考 |
|----|----|----|----|
| 表示対象タブ(I-01) | 未指定時は `my` を補う | 送信前(クライアント) | SCR-004 既定表示は My タブ([SCR-004](../../02_basic_design/01_frontend/01_screens/SCR-004.md#SCR-004) §1) |
| 立場(O-05) | `owner` のときのみ新規作成・編集導線(SCR-004 §4 #7・#10・#16)を活性表示、`member` のときは非表示 | 応答描画時(クライアント) | Join タブのカードは新規作成・編集・削除の導線を表示しない([SCR-004](../../02_basic_design/01_frontend/01_screens/SCR-004.md#SCR-004) EVT-01 / EVT-02) |
| プロジェクト一覧(O-01〜O-10) | 0 件時は空状態(SCR-004 §4 #17)へ切替、1 件以上時は件数表示(#8)・プロジェクトカード(#9〜#15)へ切替 | 応答描画時(クライアント) | My タブの空状態は新規作成導線を提示(SCR-004 EVT-06) |

## 7. 後続工程への引き継ぎ事項

テスト設計・DB 物理設計・詳細ロジック設計へ引き継ぐ観点を挙げる。稼働ステータスの導出ロジック・ページングの詳細は別文書へ委ねる。

- 表示対象タブ(I-01)の `my` / `join` それぞれについて、0 件(空状態)・1 件以上(一覧表示)・取得中(ローディング)・取得失敗の各分岐はテスト設計でケース化する。
- 立場(O-05)が `owner` / `member` のときの操作導線(新規作成・編集)の表示 / 非表示分岐はテスト設計でケース化する。
- 稼働ステータス(O-04)の導出条件(質問数上限到達・課金アカウント制限)は [API-016](../../02_basic_design/02_backend/03_apis/API-016.md#API-016) を参照し、詳細ロジックは IPO 側で確定する。
- 論理項目 ↔ 物理カラムの対応は [DBP-001](../07_db_physical/DBP-001.md#DBP-001) で確定する。
