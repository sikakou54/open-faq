# <span id="IO-001"></span>IO-001: ウィジェット質問送信 入出力

> **この入出力設計書は「FAQ ウィジェットの質問送信画面とウィジェット質問送信 API の入力/出力項目・入力チェック・編集加工」を論理項目単位で定義します。**

## 1. 目的

本入出力設計は、ウィジェット利用者が質問を送信して AI 回答または回答不可案内を受け取る一連の入力/出力項目を、画面と API を突き合わせて論理項目単位で確定する。橋渡しする画面は [SCR-030](../../02_basic_design/01_frontend/01_screens/SCR-030.md#SCR-030) の質問入力・送信(EVT-04)、API は [API-038](../../02_basic_design/02_backend/03_apis/API-038.md#API-038) である。

## 2. 対象範囲

本設計が対象とする画面・API・データと対象外を明示する。回答可否判定ロジックと未解決質問の生成範囲、AI 連携の詳細は別文書へ委ねる。

| 区分 | 対象 |
|----|----|
| 対象画面 | [SCR-030](../../02_basic_design/01_frontend/01_screens/SCR-030.md#SCR-030) FAQ ウィジェット(質問入力・送信 EVT-04) |
| 対象API | [API-038](../../02_basic_design/02_backend/03_apis/API-038.md#API-038) ウィジェット質問送信 |
| 対象データ | 質問文・対象プロジェクト・セッション認証、AI 回答応答(回答あり / 回答不可)の各出力項目 |
| 対象外 | 回答可否判定・スコア算出([IPO-001](../04_ipo/IPO-001.md#IPO-001))・未解決質問の状態遷移([STS-001](../01_state_transitions/STS-001.md#STS-001))・回答フィードバック送信([SCR-030](../../02_basic_design/01_frontend/01_screens/SCR-030.md#SCR-030) EVT-08 / API-068)・受付停止の契機判定 |

| I-02 | 対象プロジェクト | 文字列 | ◯ | — | 有効なプロジェクト識別子 | — | ウィジェット起動時に確定した対象プロジェクト。API リクエスト `projectId` |

## 4. 出力項目一覧

API レスポンス・画面へ返す出力項目を論理項目単位で定義する。回答種別により回答あり(`answered`)と回答不可(`unanswered`)で出力項目が分岐する。

| 項目ID | 項目名 | データ型 | 表示形式 | 表示条件 | 備考 |
|----|----|----|----|----|----|
| O-01 | 回答種別 | 列挙値 | — | 常時 | `answered`(回答あり)/ `unanswered`(回答不可)。§5 の分岐条件は列挙値表参照 |
| O-02 | 回答本文 | 文字列 | 会話欄に追加表示 | 回答種別が `answered` | AI が生成した回答本文。個人情報は種別ラベル付き伏字へマスキング済([RULE-024](../../01_requirements/01_business_requirement/08_rule.md#RULE-024))。SCR-030 §4 #6 に対応。API `answer` |
| O-03 | 信頼度 | 数値 | 画面非表示 | 回答種別が `answered` | 回答の信頼度(0〜1)。しきい値は [RULE-012](../../01_requirements/01_business_requirement/08_rule.md#RULE-012) / [システム仕様書 §1](../../02_basic_design/07_system-spec.md#1-aiしきい値)。API `confidence` |
| O-04 | 参照 FAQ 一覧 | 配列 | 会話欄に参照 FAQ として提示 | 回答種別が `answered` | 回答の根拠とした FAQ の識別子・質問・回答([FR-059](../../01_requirements/02_functional_requirement/02_faq-ai-fr.md#FR-059))。API `referencedFaqs` |
| O-05 | 関連質問サジェスト | 配列 | チップ表示(SCR-030 §4 #8) | 回答種別が `answered`(0 件のときは空配列) | 関連 FAQ の質問文を関連度順に最大 3 件(設計値)。API `suggestions` |
| O-06 | 質問ログ識別子 | 文字列 | 画面非表示 | 常時 | 記録された質問ログの識別子。冪等性の基準。API `questionLogId` |
| O-07 | 回答不可理由 | 列挙値 | 会話欄の案内文へ反映 | 回答種別が `unanswered` | 未回答 / 受付停止理由。取りうる値と意味は §列挙値。API `reason` |
| O-08 | 未解決質問識別子 | 文字列 | 画面非表示 | 回答種別が `unanswered`(回答不可理由 3 値のとき) | 同時生成された未解決質問の識別子。API `inquiryId` |
| O-09 | 問い合わせ ID | 文字列 | 画面非表示 | 回答種別が `unanswered`(回答不可理由 3 値のとき) | 未解決質問の管理用識別子。ウィジェットには描画しない([FR-062](../../01_requirements/02_functional_requirement/02_faq-ai-fr.md#FR-062))。API `inquiryCode` |
| O-10 | 連絡先メール | 文字列 | 連絡先メール案内に表示(SCR-030 §4 #11) | 回答種別が `unanswered` かつ連絡先設定済み | プロジェクトの問い合わせ先メール。未設定時は `null`。API `contactEmail` |

## 5. 入力チェック仕様

各入力項目のチェック内容・実施主体(クライアント / サーバー)・エラー時参照を定義する。クライアント検証違反時は送信を中止し、サーバー検証違反時は回答不可ではなく入力検証エラーとして扱う。文面は本ページに再掲せず ID 参照する。

| 項目名 | チェック内容 | タイミング | エラー時メッセージ | 備考 |
|----|----|----|----|----|
| 質問文(I-01) | 未入力チェック | クライアント・サーバー | [SCR-030](../../02_basic_design/01_frontend/01_screens/SCR-030.md#SCR-030) EM-01 / [ERR-001](../../02_basic_design/05_errors/ERR-001.md#ERR-001) | クライアントは送信時に判定([SCR-030](../../02_basic_design/01_frontend/01_screens/SCR-030.md#SCR-030) §5 #9)。違反時は送信を中止 |
| 質問文(I-01) | 最大長チェック | クライアント・サーバー | [SCR-030](../../02_basic_design/01_frontend/01_screens/SCR-030.md#SCR-030) EM-02 / [ERR-001](../../02_basic_design/05_errors/ERR-001.md#ERR-001) | 上限 1000 文字([SCR-030](../../02_basic_design/01_frontend/01_screens/SCR-030.md#SCR-030) §4 #9)。サーバーは編集加工(§6)後の文字数で判定 |
| 回答フィードバック未選択(送信可否) | 直前の AI 回答(回答可能)に回答フィードバックが未選択の場合は送信不可 | クライアント | [SCR-030](../../02_basic_design/01_frontend/01_screens/SCR-030.md#SCR-030) EM-04 | 送信ボタン押下時に判定([SCR-030](../../02_basic_design/01_frontend/01_screens/SCR-030.md#SCR-030) §5 #10 / [FR-196](../../01_requirements/02_functional_requirement/02_faq-ai-fr.md#FR-196))。違反時は送信を中止 |
| 対象プロジェクト(I-02) | セッションとオリジンの整合・許可ドメイン照合 | サーバー | [ERR-001](../../02_basic_design/05_errors/ERR-001.md#ERR-001) ・ [ERR-027](../../02_basic_design/05_errors/ERR-027.md#ERR-027) | サーバーのみ判定([API-038](../../02_basic_design/02_backend/03_apis/API-038.md#API-038) P-01) |
| ウィジェットセッショントークン(I-03) | セッショントークンの検証・レート制限判定 | サーバー | [ERR-039](../../02_basic_design/05_errors/ERR-039.md#ERR-039)(失効)/ [ERR-009](../../02_basic_design/05_errors/ERR-009.md#ERR-009)(レート制限) | 認証失敗・レート制限はサーバーのみ判定([API-038](../../02_basic_design/02_backend/03_apis/API-038.md#API-038) P-01 / P-02)。`wst_*` 失効時はウィジェットが [API-037](../../02_basic_design/02_backend/03_apis/API-037.md#API-037) で自動再取得して再送する(透過的再確立) |

## 6. 編集・加工仕様

入力値の正規化・出力値の整形など編集加工を定義する。最大長判定は加工後の値に対して行う。

| 項目名 | 加工内容 | 加工タイミング | 備考 |
|----|----|----|----|
| 質問文(I-01) | 前後空白の除去 | 検証前(サーバー) | 未入力・最大長チェックは加工後の値で判定する |
| 連絡先メール(O-10) | 未設定時は `null` を返す | 応答生成時(サーバー) | 連絡先設定済みのときのみ連絡先メール案内(SCR-030 §4 #11)を表示 |
| 問い合わせ ID(O-09) | ウィジェットには描画しない | 応答描画時(クライアント) | レスポンス互換性のため返却するが画面には表示しない([FR-062](../../01_requirements/02_functional_requirement/02_faq-ai-fr.md#FR-062)) |

## 列挙値

出力の回答種別(O-01)と回答不可理由(O-07)の取りうる値と意味を示す。回答不可理由は回答種別が `unanswered` のときのみ返す。値の全集合の正本は [API-038](../../02_basic_design/02_backend/03_apis/API-038.md#API-038) `## 列挙値`。

| 項目 | 取りうる値 | 意味 |
|----|----|----|
| 回答種別(O-01) | `answered` | 回答あり(AI が回答を生成) |
| 回答種別(O-01) | `unanswered` | 回答不可(未解決質問を生成し連絡先を案内、または受付停止案内) |
| 回答不可理由(O-07) | `low_confidence` | 信頼度がしきい値未満で回答を生成できなかった([RULE-012](../../01_requirements/01_business_requirement/08_rule.md#RULE-012)) |
| 回答不可理由(O-07) | `no_faq_match` | 該当する FAQ が無い |
| 回答不可理由(O-07) | `contradiction` | 候補 FAQ 間に矛盾があり断定回答しない([FR-060](../../01_requirements/02_functional_requirement/02_faq-ai-fr.md#FR-060) / [RULE-023](../../01_requirements/01_business_requirement/08_rule.md#RULE-023)) |
| 回答不可理由(O-07) | `usage_limit_reached` | 月次質問数上限の 100% に到達し新規質問受付を停止 |
| 回答不可理由(O-07) | `payment_required` | 支払方法未登録のまま無料枠を超過し新規質問受付を停止 |

> [!NOTE]
> **回答不可理由 3 値(`low_confidence` / `no_faq_match` / `contradiction`)のときのみ未解決質問識別子(O-08)・問い合わせ ID(O-09)を返す。** 受付停止 2 値(`usage_limit_reached` / `payment_required`)は 200 の `unanswered` として返し、未解決質問は生成しない。AI 推論タイムアウト / プロバイダエラーは処理エラーとして扱い未解決登録しない([FR-082](../../01_requirements/02_functional_requirement/02_faq-ai-fr.md#FR-082) / [ERR-036](../../02_basic_design/05_errors/ERR-036.md#ERR-036))。生成した回答本文(O-02)中の個人情報は [RULE-024](../../01_requirements/01_business_requirement/08_rule.md#RULE-024) により種別ラベル付き伏字へマスキングして `answered` で返すため、回答不可理由には現れない。

## 7. 後続工程への引き継ぎ事項

テスト設計・DB 物理設計・詳細ロジック設計へ引き継ぐ観点を挙げる。回答可否判定・未解決生成トランザクションの詳細は別文書へ委ねる。

- 質問文(I-01)の境界値(0 / 1 / 1000 / 1001 文字)、前後空白除去後の文字数判定はテスト設計でパターン化する。
- 回答種別(O-01)ごとの出力項目の分岐(回答あり / 回答不可理由 3 値 / 受付停止 2 値)と、連絡先メール(O-10)設定済み / 未設定の表示分岐はテスト設計でケース化する。
- 回答可否判定と関連質問サジェスト・信頼度の算出は [IPO-001](../04_ipo/IPO-001.md#IPO-001)、未解決質問の状態は [STS-001](../01_state_transitions/STS-001.md#STS-001)、AI 連携のタイムアウト / エラー応答は [EIF-001](../06_external_if/EIF-001.md#EIF-001) を参照する。
- 論理項目 ↔ 物理カラムの対応は [DBP-001](../07_db_physical/DBP-001.md#DBP-001) で確定する。
