## 1. 目的

本入出力設計は、アカウント利用者が自分宛のお知らせを一覧・詳細で閲覧し、個別または一括で既読化する一連の入力/出力項目を、画面と API を突き合わせて論理項目単位で確定する。橋渡しする画面は [SCR-016](../../02_basic_design/01_frontend/01_screens/SCR-016.md#SCR-016) の一覧表示・絞り込み・既読化(EVT-01〜EVT-11)と [SCR-017](../../02_basic_design/01_frontend/01_screens/SCR-017.md#SCR-017) の詳細表示・自動既読(EVT-01)、API は [API-048](../../02_basic_design/02_backend/03_apis/API-048.md#API-048)・[API-049](../../02_basic_design/02_backend/03_apis/API-049.md#API-049)・[API-050](../../02_basic_design/02_backend/03_apis/API-050.md#API-050)・[API-051](../../02_basic_design/02_backend/03_apis/API-051.md#API-051) である。

## 2. 対象範囲

対象とする画面・API・データと、対象外を明示する。お知らせの配信生成・配信対象の決定ロジックは対象外とする。

| 区分 | 対象 |
|----|----|
| 対象画面 | [SCR-016](../../02_basic_design/01_frontend/01_screens/SCR-016.md#SCR-016) お知らせ一覧(絞り込み・選択・一括既読化 EVT-01〜EVT-11)・[SCR-017](../../02_basic_design/01_frontend/01_screens/SCR-017.md#SCR-017) お知らせ詳細(表示・自動既読・前後移動 EVT-01〜EVT-04) |
| 対象API | [API-048](../../02_basic_design/02_backend/03_apis/API-048.md#API-048) お知らせ一覧・[API-049](../../02_basic_design/02_backend/03_apis/API-049.md#API-049) お知らせ個別既読・[API-050](../../02_basic_design/02_backend/03_apis/API-050.md#API-050) お知らせ一括既読・[API-051](../../02_basic_design/02_backend/03_apis/API-051.md#API-051) お知らせ未読件数 |
| 対象データ | 絞り込み条件(読了状態・期間・タイトル・種別)・選択お知らせ ID・お知らせのメタ情報(種別・重要度・タイトル・配信日時・既読日時・本文・行動導線)・未読件数 |
| 対象外 | お知らせの配信生成・配信対象ユーザーの決定・行動導線(CTA)遷移先の画面遷移仕様 |

## 4. 出力項目一覧

API レスポンス・画面へ返す出力項目を論理項目単位で定義する。お知らせ 1 件分の項目([O-01]〜[O-08])は一覧・詳細の双方で共通利用する。

| 項目ID | 項目名 | データ型 | 表示形式 | 表示条件 | 備考 |
|----|----|----|----|----|----|
| O-01 | お知らせ ID | 文字列 | リンク(SCR-016 §4 #10) | 常時 | 詳細画面(SCR-017)への遷移キー。API `id` |
| O-02 | タイトル | 文字列 | ラベル(SCR-016 §4 #11 / SCR-017 §4 #4) | 常時 | API `title` |
| O-03 | 種別 | 列挙値 | バッジ(SCR-016 §4 #7 / SCR-017 §4 #2) | 常時 | 取りうる値は §列挙値。API `category` |
| O-04 | 重要度 | 列挙値 | バッジ(SCR-016 §4 #8 / SCR-017 §4 #3) | 常時 | 取りうる値は §列挙値。API `priority` |
| O-05 | 本文(HTML) | 文字列 | 本文表示(SCR-017 §4 #7)。サニタイズして表示 | 常時(SCR-016 では非表示) | API `bodyHtml` |
| O-06 | 行動導線ラベル | 文字列 | ボタン文言(SCR-017 §4 #8) | 行動導線ありのお知らせ(非 `null`) | API `actionLabel` |
| O-07 | 行動導線遷移先 | 文字列 | ボタン押下時の遷移先(SCR-017 §4 #8) | 行動導線ありのお知らせ(非 `null`) | API `actionUrl` |
| O-08 | 既読日時 | 日時 | ラベル(SCR-017 §4 #6)・未読ドット非表示条件(SCR-016 §4 #9) | 既読時のみ表示・未読時は非表示 | 未読時は値が `null`。API `readAt` |
| O-09 | 配信日時 | 日時 | ラベル(SCR-016 §4 #12 / SCR-017 §4 #5) | 常時 | API `createdAt` |
| O-10 | 次ページカーソル | 文字列 | 画面非表示(次のページボタン #19 の表示制御に使用) | 末尾ページ以外 | 末尾ページは `null`。次のページボタン(SCR-016 §4 #19)は非 `null` 時のみ表示。API `nextCursor` |
| O-11 | 未読件数 | 数値 | 件数表示(SCR-016 §4 #6) | 常時 | API-051 `unreadsCount` |
| O-12 | 直近お知らせ一覧 | 配列 | 画面非表示(初期表示の一覧描画に使用) | 常時(0 件時は空配列) | 各要素は O-01〜O-09 と同形式。API-051 `recent` |
| O-13 | 個別既読結果 | 真偽値 | 画面非表示 | 個別既読 API 成功時 | 成功時に O-08(既読日時)を更新。API-049 応答 |
| O-14 | 一括既読件数 | 数値 | 画面非表示(一覧・未読件数の再取得契機) | 一括既読 API 成功時 | 今回既読にした件数。API-050 `readCount` |
| O-15 | 一括既読実行日時 | 日時 | 画面非表示 | 一括既読 API 成功時 | API-050 `readAt` |

## 7. 後続工程への引き継ぎ事項

テスト設計・DB 物理設計・詳細ロジック設計へ引き継ぐ観点を挙げる。

- 読了状態フィルタ(I-01)・種別フィルタ(I-02)・期間フィルタ(I-03・I-04)・タイトル検索(I-05)の組み合わせパターン、0 件時の空状態表示(SCR-016 §4 #20)はテスト設計でケース化する。
- 一括既読の指定パターン(選択 ID 指定 / 表示中の未読 / すべての未読、種別絞り込みあり / なし)と上限([RULE-019](../../01_requirements/01_business_requirement/08_rule.md#RULE-019))境界値はテスト設計でパターン化する。
- SCR-017 初期表示時の自動既読([API-049](../../02_basic_design/02_backend/03_apis/API-049.md#API-049))失敗時は一覧側 EM-02 相当の挙動を含め詳細画面での扱いを実装時に確認する。
- 論理項目 ↔ 物理カラムの対応は [DBP-012](../07_db_physical/DBP-012.md#DBP-012) で確定する。
