# <span id="IO-022"></span>IO-022: ウィジェット設定 入出力

> **この入出力設計書は「ウィジェット設定画面と公開キー再発行 API・AI しきい値設定取得・更新 API の入力/出力項目・入力チェック・編集加工」を論理項目単位で定義します。**

*種別 入出力設計 ・ ステータス ドラフト*

| 項目 | 値 |
|----|----|
| 入出力設計ID | IO-022 |
| 業務ユースケースID | [UC-038](../../01_requirements/04_business_usecases/UC-038.md#UC-038) ・ [UC-039](../../01_requirements/04_business_usecases/UC-039.md#UC-039) ・ [UC-075](../../01_requirements/04_business_usecases/UC-075.md#UC-075) |

## 1. 目的

本入出力設計は、オーナー・メンバーがウィジェット公開キーを再発行し、対象プロジェクトの AI しきい値(信頼度・関連度)を確認・保存する一連の入力/出力項目を、画面と API を突き合わせて論理項目単位で確定する。橋渡しする画面は [SCR-011](../../02_basic_design/01_frontend/01_screens/SCR-011.md#SCR-011) の公開キー再発行(EVT-09)・AI しきい値保存(EVT-12)、API は [API-019](../../02_basic_design/02_backend/03_apis/API-019.md#API-019)(鍵ローテーション)・[API-067](../../02_basic_design/02_backend/03_apis/API-067.md#API-067)(AI しきい値取得・更新)である。

## 2. 対象範囲

本設計が対象とする画面・API・データと対象外を明示する。ウィジェット見た目設定(主色・表示位置・見出し・初期メッセージ)の保存は別 IO(プロジェクト更新 API-018)で扱う。

| 区分 | 対象 |
|----|----|
| 対象画面 | [SCR-011](../../02_basic_design/01_frontend/01_screens/SCR-011.md#SCR-011) ウィジェット設定(公開キー再発行確認ダイアログ EVT-09・AI しきい値保存 EVT-12・初期表示 EVT-01 のうち公開キー / AI しきい値部分) |
| 対象API | [API-019](../../02_basic_design/02_backend/03_apis/API-019.md#API-019) ウィジェット鍵ローテーション・[API-067](../../02_basic_design/02_backend/03_apis/API-067.md#API-067) AI しきい値設定取得・更新(GET / PUT) |
| 対象データ | 再発行確認名称・再認証パスワード・新公開キー・旧キー失効予告、信頼度しきい値・関連度しきい値・適用元の各出力項目 |
| 対象外 | ウィジェット見た目設定(主色・表示位置・見出し・初期メッセージ)の保存(API-018・別 IO で扱う)・埋め込みコードのコピー([SCR-011](../../02_basic_design/01_frontend/01_screens/SCR-011.md#SCR-011) EVT-03)・再認証トークンの発行そのもの(API-005・別 IO で扱う)・AI しきい値キャッシュへの伝播処理([SYS-015](../../02_basic_design/02_backend/01_system/SYS-015.md#SYS-015)) |

## 4. 出力項目一覧

API レスポンス・画面へ返す出力項目を論理項目単位で定義する。公開キー再発行と AI しきい値取得・更新でそれぞれ出力項目が分岐する。

| 項目ID | 項目名 | データ型 | 表示形式 | 表示条件 | 備考 |
|----|----|----|----|----|----|
| O-01 | 新公開キー | 文字列 | 公開キー欄へ反映(#2) | 公開キー再発行成功時 | API-019 `newKey` |
| O-02 | 旧キー失効予告 | 文字列 | 保存完了通知内に表示 | 公開キー再発行成功時 | 旧キーの猶予失効予告メッセージ([システム仕様書 §4](../../02_basic_design/07_system-spec.md#4-データ保持期間削除猶予))。API-019 `deprecationWarning` |
| O-03 | 適用しきい値(信頼度) | 数値 | 入力欄へ反映(#21) | 常時(初期表示時 GET・保存成功時 PUT 応答) | 対象プロジェクトへ現在適用される信頼度しきい値。API-067 `effective.confidenceThreshold` |
| O-04 | 適用しきい値(関連度) | 数値 | 入力欄へ反映(#22) | 常時(初期表示時 GET・保存成功時 PUT 応答) | 対象プロジェクトへ現在適用される関連度しきい値。API-067 `effective.relevanceThreshold` |
| O-05 | AI しきい値 適用元 | 列挙値 | 適用元ラベル表示(#20) | 常時(初期表示時 GET・保存成功時 PUT 応答) | `project`(プロジェクト設定値)/ `global`(グローバル既定値)。取りうる値と意味は §列挙値。API-067 `effective.source` |
| O-06 | グローバル既定値(信頼度) | 数値 | 画面非表示 | 常時(初期表示時 GET) | [システム仕様書 §1](../../02_basic_design/07_system-spec.md#1-aiしきい値)の値。API-067 `layers.global.confidenceThreshold` |

## 5. 入力チェック仕様

各入力項目のチェック内容・実施主体(クライアント / サーバー)・エラー時参照を定義する。クライアント検証違反時は送信を中止または対象ボタンを非活性のまま維持し、サーバー検証違反時はエラーを表示する。文面は本ページに再掲せず ID 参照する。

| 項目名 | チェック内容 | タイミング | エラー時メッセージ | 備考 |
|----|----|----|----|----|
| 再発行確認 プロジェクト名入力(I-02) | 現プロジェクト名との完全一致チェック | クライアント | [SCR-011](../../02_basic_design/01_frontend/01_screens/SCR-011.md#SCR-011) §7 EVT-09 | 不一致の間は「再発行する」ボタン(#17)を非活性のまま維持し API 送信自体を行わない |
| 再発行確認 パスワード(I-03) | 再認証(パスワード再照合)の成否チェック | サーバー | [ERR-005](../../02_basic_design/05_errors/ERR-005.md#ERR-005) | 再認証 API 呼び出し時に判定([API-005](../../02_basic_design/02_backend/03_apis/API-005.md#API-005) P-02)。失敗時は再認証トークンを取得できず公開キー再発行は行われない |
| 対象プロジェクト識別子(I-01) | 再認証トークンの有効性チェック(公開キー再発行) | サーバー | [SCR-011](../../02_basic_design/01_frontend/01_screens/SCR-011.md#SCR-011) EM-05 / [ERR-013](../../02_basic_design/05_errors/ERR-013.md#ERR-013) | サーバーのみ判定([API-019](../../02_basic_design/02_backend/03_apis/API-019.md#API-019) P-00・P-01)。無効・未提示時は鍵ローテーションを実行しない |
| 対象プロジェクト識別子(I-01) | 権限チェック(当該プロジェクトのオーナー / メンバーであること) | サーバー | [SCR-011](../../02_basic_design/01_frontend/01_screens/SCR-011.md#SCR-011) EM-08 / [ERR-019](../../02_basic_design/05_errors/ERR-019.md#ERR-019) | サーバーのみ判定([API-067](../../02_basic_design/02_backend/03_apis/API-067.md#API-067) P-01) |

## 6. 編集・加工仕様

入力値の正規化・出力値の整形など編集加工を定義する。

| 項目名 | 加工内容 | 加工タイミング | 備考 |
|----|----|----|----|
| 信頼度・関連度しきい値(I-04・I-05) | 両方未入力時は `null` / `null` を送信しプロジェクト設定値を削除する | 保存時(クライアント) | [システム仕様書 §1](../../02_basic_design/07_system-spec.md#1-aiしきい値)。削除後は以後グローバル既定値を適用([API-067](../../02_basic_design/02_backend/03_apis/API-067.md#API-067) P-06) |
| 新公開キー(O-01) | 埋め込みコード全文中の公開キーを新キーへ再構成 | 保存成功後(クライアント) | 埋め込みコード(#11)は新キーで再表示。既存の埋め込みコードは旧キー失効後に動作しなくなる(#19) |
| AI しきい値 適用元(O-05) | `project` / `global` の表示ラベルへ変換 | 応答描画時(クライアント) | 取りうる値は §列挙値。SCR-011 #20 の表示名(プロジェクト設定値 / グローバル既定値)へ対応 |

## 7. 後続工程への引き継ぎ事項

テスト設計・DB 物理設計・詳細ロジック設計へ引き継ぐ観点を挙げる。再認証トークン発行そのものの詳細・AI しきい値キャッシュへの伝播処理は別文書へ委ねる。

- 再発行確認 プロジェクト名入力(I-02)の完全一致判定(大文字小文字・前後空白を含む差異)の境界はテスト設計でケース化する。
- 信頼度・関連度しきい値(I-04・I-05)の片側のみ指定・範囲外・両方未指定(削除)の組み合わせはテスト設計でケース化する。
- 公開キー再発行後の旧キー使用中バッジ(SCR-011 #4)表示・猶予期間中の新旧キー併用挙動は [システム仕様書 §4](../../02_basic_design/07_system-spec.md#4-データ保持期間削除猶予) を参照しテスト設計でケース化する。
- AI しきい値 適用元(O-05)の `project` / `global` 切り替えとキャッシュ伝播は [SYS-015](../../02_basic_design/02_backend/01_system/SYS-015.md#SYS-015) を参照する。
- 論理項目 ↔ 物理カラムの対応は、公開キー・鍵ローテーション関連は [DBP-006](../07_db_physical/DBP-006.md#DBP-006)、AI しきい値関連は [DBP-013](../07_db_physical/DBP-013.md#DBP-013) を参照する。
