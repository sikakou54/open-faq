# <span id="IO-004"></span>IO-004: パスワード再設定 入出力

> **この入出力設計書は「パスワード再設定画面と再設定要求 API・再設定確定 API の入力/出力項目・入力チェック・編集加工」を論理項目単位で定義します。**

*種別 入出力設計 ・ ステータス ドラフト*

| 項目 | 値 |
|----|----|
| 入出力設計ID | IO-004 |
| 業務ユースケースID | [UC-004](../../01_requirements/04_business_usecases/UC-004.md#UC-004) ・ [UC-005](../../01_requirements/04_business_usecases/UC-005.md#UC-005) |
| 関連画面 | [SCR-003](../../02_basic_design/01_frontend/01_screens/SCR-003.md#SCR-003) |
| 関連 API | [API-004](../../02_basic_design/02_backend/03_apis/API-004.md#API-004) ・ [API-010](../../02_basic_design/02_backend/03_apis/API-010.md#API-010) |

## 1. 目的

本入出力設計は、未認証ユーザーがメールアドレスを申告して再設定リンクを要求し(段階1)、受信したリンクから新しいパスワードを設定する(段階2)一連の入力/出力項目を、画面と各 API を突き合わせて論理項目単位で確定する。橋渡しする画面は [SCR-003](../../02_basic_design/01_frontend/01_screens/SCR-003.md#SCR-003)、API は要求側 [API-004](../../02_basic_design/02_backend/03_apis/API-004.md#API-004)、確定側 [API-010](../../02_basic_design/02_backend/03_apis/API-010.md#API-010) である。

## 2. 対象範囲

本設計が対象とする画面・API・データと対象外を明示する。

| 区分 | 対象 |
|----|----|
| 対象画面 | [SCR-003](../../02_basic_design/01_frontend/01_screens/SCR-003.md#SCR-003) パスワード再設定(段階1: 再設定リンク送信 SCR-003 EVT-02〜EVT-03 / 段階2: リンク有効性確認・パスワード設定 SCR-003 EVT-04〜EVT-06) |
| 対象API | [API-004](../../02_basic_design/02_backend/03_apis/API-004.md#API-004) パスワード再設定要求(段階1)・[API-010](../../02_basic_design/02_backend/03_apis/API-010.md#API-010) パスワード再設定確定(段階2) |
| 対象データ | 申告メールアドレス・再設定トークン・新しいパスワード / 確認用パスワード、受付結果・リンク有効性・確定結果の各出力項目 |
| 対象外 | 再設定メールの文面([MSG](../../02_basic_design/06_messages/index.md))・当該ユーザーの全セッション失効処理そのもの([API-010](../../02_basic_design/02_backend/03_apis/API-010.md#API-010) P-06)・ログイン([SCR-001](../../02_basic_design/01_frontend/01_screens/SCR-001.md#SCR-001) 側 IO で扱う) |

## 3. 入力項目一覧

段階1(再設定要求)・段階2(再設定確定)それぞれで画面から受け取り API へ渡す入力項目を論理項目単位で定義する。文字数上限などの具体値は基本設計・要件へ ID 参照する。

| 項目ID | 項目名 | データ型 | 必須 | 初期値 | 入力制約 | 入力例 | 備考 |
|----|----|----|----|----|----|----|----|
| I-01 | メールアドレス | 文字列 | ◯ | — | 254 文字以内・メールアドレス形式([SCR-003](../../02_basic_design/01_frontend/01_screens/SCR-003.md#SCR-003) §4 #2) | user@example.com | 段階1。SCR-003 §4 #2 に対応。API-004 リクエスト `email` |
| I-02 | 再設定トークン | 文字列 | ◯ | — | 有効な再設定トークン([システム仕様書 §4](../../02_basic_design/07_system-spec.md#4-データ保持期間削除猶予) 有効期限 1 時間) | reset_... | 段階2。再設定リンクの URL から取得し画面には表示しない。API-010 リクエスト `token` |
| I-03 | 新しいパスワード | 文字列 | ◯ | — | 12 文字以上・英大文字/小文字/数字/記号のうち 3 種類以上([FR-006](../../01_requirements/02_functional_requirement/01_account-fr.md#FR-006) / [RULE-003](../../01_requirements/01_business_requirement/08_rule.md#RULE-003))・128 文字以内([SCR-003](../../02_basic_design/01_frontend/01_screens/SCR-003.md#SCR-003) §4 #7) | — | 段階2。SCR-003 §4 #7 に対応。API-010 リクエスト `password` |
| I-04 | 新しいパスワード(確認) | 文字列 | ◯ | — | I-03 と一致・128 文字以内([SCR-003](../../02_basic_design/01_frontend/01_screens/SCR-003.md#SCR-003) §4 #8) | — | 段階2。SCR-003 §4 #8 に対応。API-010 リクエスト `passwordConfirm` |

## 4. 出力項目一覧

各 API レスポンス・画面へ返す出力項目を論理項目単位で定義する。

| 項目ID | 項目名 | データ型 | 表示形式 | 表示条件 | 備考 |
|----|----|----|----|----|----|
| O-01 | 再設定要求受付結果 | 真偽値 | メール送信済み案内(SCR-003 §4 #5)の表示切替 | 段階1 送信操作後 | アカウント存在有無に関わらず一律 `true`(列挙攻撃対策)。API-004 `ok` |
| O-02 | リンク有効性 | 真偽値 | 段階2 初期表示の分岐 | 段階2 初期表示時 | 有効時は新しいパスワード入力欄(#7・#8)、無効/期限切れ時はリンク無効エラー(#10)を表示。API-010 呼び出し前のトークン検証結果(SCR-003 EVT-04) |
| O-03 | 再設定確定結果 | 真偽値 | 完了案内(SCR-003 §4 #12)の表示切替 | 段階2 設定確定操作後・成功時 | 成功時、当該ユーザーの全セッションを失効させ完了案内とログインするボタン(#13)を表示。API-010 `ok` |

## 6. 編集・加工仕様

入力値の正規化・出力値の整形など編集加工を定義する。

| 項目名 | 加工内容 | 加工タイミング | 備考 |
|----|----|----|----|
| メールアドレス(I-01) | 前後空白の除去・小文字化 | 検証前(サーバー) | アカウント照合は加工後の値で行う。照合結果は O-01 に反映せず一律受付とする |
| 再設定要求受付結果(O-01) | アカウント存在有無を出力に含めない | 応答生成時(サーバー) | 列挙攻撃対策([API-004](../../02_basic_design/02_backend/03_apis/API-004.md#API-004) P-02) |
| 新しいパスワード(I-03) | 加工なし(ハッシュ化のみ実施) | 保存時(サーバー) | ハッシュ化アルゴリズムの詳細は本設計の対象外 |

## 7. 後続工程への引き継ぎ事項

テスト設計・DB 物理設計・詳細ロジック設計へ引き継ぐ観点を挙げる。

- 新しいパスワード(I-03)の境界値(11 / 12 文字、種類数 2 / 3)、確認用(I-04)との一致 / 不一致はテスト設計でパターン化する。
- 再設定トークン(I-02)の有効 / 期限切れ / 使用済み / 存在しない の 4 状態と、段階2初期表示(O-02)の表示分岐はテスト設計でケース化する。
- 再設定要求(段階1)の再送間隔制限([ERR-009](../../02_basic_design/05_errors/ERR-009.md#ERR-009))と再設定確定成功時の全セッション失効([API-010](../../02_basic_design/02_backend/03_apis/API-010.md#API-010) P-06)は詳細ロジック設計へ引き継ぐ。
- 論理項目 ↔ 物理カラムの対応は [T_ACCESS_TOKENS](../../02_basic_design/02_backend/04_database/TBL-014.md#TBL-014) を参照して DB 物理設計で確定する。
