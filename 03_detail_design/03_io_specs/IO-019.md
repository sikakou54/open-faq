# <span id="IO-019"></span>IO-019: FAQ一覧 入出力

> **この入出力設計書は「FAQ一覧画面と FAQ 一覧・一括状態変更・CSV エクスポート API の入力/出力項目・入力チェック・編集加工」を論理項目単位で定義します。**

*種別 入出力設計 ・ ステータス ドラフト*

| 項目 | 値 |
|----|----|
| 入出力設計ID | IO-019 |
| 業務ユースケースID | [UC-023](../../01_requirements/04_business_usecases/UC-023.md#UC-023) ・ [UC-026](../../01_requirements/04_business_usecases/UC-026.md#UC-026) ・ [UC-028](../../01_requirements/04_business_usecases/UC-028.md#UC-028) |
| 関連画面 | [SCR-008](../../02_basic_design/01_frontend/01_screens/SCR-008.md#SCR-008) |
| 関連 API | [API-025](../../02_basic_design/02_backend/03_apis/API-025.md#API-025) ・ [API-027](../../02_basic_design/02_backend/03_apis/API-027.md#API-027) ・ [API-030](../../02_basic_design/02_backend/03_apis/API-030.md#API-030) |

## 1. 目的

本入出力設計は、オーナー / メンバーが担当プロジェクトの FAQ を一覧・検索・絞り込み・並び替えで確認し、複数件を選択して公開状態を一括変更し、または一覧の絞り込み結果を CSV で取り出す一連の入力/出力項目を、画面と API を突き合わせて論理項目単位で確定する。橋渡しする画面は [SCR-008](../../02_basic_design/01_frontend/01_screens/SCR-008.md#SCR-008) の初期表示・絞り込み・並び替え(EVT-01〜EVT-04)、行選択・一括操作(EVT-05・EVT-08・EVT-09・EVT-11)、CSV エクスポート(EVT-13)、API は [API-025](../../02_basic_design/02_backend/03_apis/API-025.md#API-025)・[API-027](../../02_basic_design/02_backend/03_apis/API-027.md#API-027)・[API-030](../../02_basic_design/02_backend/03_apis/API-030.md#API-030) である。

## 2. 対象範囲

対象とする画面・API・データと、対象外を明示する。キーワード全文検索・個別 FAQ の作成/更新/削除・一括削除・CSV インポートは別文書へ委ねる。

| 区分 | 対象 |
|----|----|
| 対象画面 | [SCR-008](../../02_basic_design/01_frontend/01_screens/SCR-008.md#SCR-008) FAQ一覧(初期表示・絞り込み・並び替え・行選択・一括状態変更・CSV エクスポート) |
| 対象API | [API-025](../../02_basic_design/02_backend/03_apis/API-025.md#API-025) FAQ 一覧 ・ [API-027](../../02_basic_design/02_backend/03_apis/API-027.md#API-027) FAQ 一括状態変更 ・ [API-030](../../02_basic_design/02_backend/03_apis/API-030.md#API-030) FAQ CSV エクスポート |
| 対象データ | 検索・絞り込み・並び替え条件、一覧表示項目、行選択、一括状態変更の対象 ID と変更後状態、CSV エクスポート結果 |
| 対象外 | キーワード全文検索([API-031](../../02_basic_design/02_backend/03_apis/API-031.md#API-031)、別文書)・個別 FAQ の作成/更新/一括削除([API-026](../../02_basic_design/02_backend/03_apis/API-026.md#API-026)、別文書)・CSV インポート([SCR-010](../../02_basic_design/01_frontend/01_screens/SCR-010.md#SCR-010)、別文書) |

## 3. 入力項目一覧

一覧画面から受け取り各 API へ渡す入力項目を論理項目単位で定義する。文字数上限・件数上限などの具体値は基本設計・業務ルールへ ID 参照する。

| 項目ID | 項目名 | データ型 | 必須 | 初期値 | 入力制約 | 入力例 | 備考 |
|----|----|----|----|----|----|----|----|
| I-01 | 対象プロジェクト | 文字列 | ◯ | — | 有効なプロジェクト識別子 | — | 画面遷移時に確定。API-025/030 の `projectId`。API-027 は行単位(I-07)で判定 |
| I-02 | 状態フィルタ | 列挙値 | — | すべて | `draft` / `published` / `hidden`([SCR-008](../../02_basic_design/01_frontend/01_screens/SCR-008.md#SCR-008) §4 #2) | `published` | SCR-008 §4 #2 に対応。未指定時は全状態を対象。API-025 `status` |
| I-03 | カテゴリフィルタ | 文字列 | — | すべて | 対象プロジェクト内のカテゴリ値([API-025](../../02_basic_design/02_backend/03_apis/API-025.md#API-025) `category`) | 支払い | SCR-008 §4 #3 に対応。API-025 `category` |
| I-04 | 並び順 | 列挙値 | — | `relevance`(関連度) | `relevance` / `updated_at` / `created_at`([API-025](../../02_basic_design/02_backend/03_apis/API-025.md#API-025) `sort`) | `updated_at` | SCR-008 §4 #9 列ヘッダー押下(EVT-04)に対応。API-025 `sort` |
| I-05 | ページングカーソル | 文字列 | — | — | 共通ページング仕様(API-025 `cursor`) | — | 「もっと見る」(SCR-008 §4 #11)押下時に前回応答の `nextCursor` を送信。API-025 `cursor` |
| I-06 | 行選択チェック | 真偽値 | — | 未チェック | RULE-019 で定める上限まで選択可([RULE-019](../../01_requirements/01_business_requirement/08_rule.md#RULE-019)) | — | SCR-008 §4 #4 に対応。選択集合が I-07 の元になる |
| I-07 | 一括対象 FAQ ID リスト | 配列 | ◯(一括操作時) | — | 各要素は ULID・1 リクエストあたり RULE-019 で定める上限([RULE-019](../../01_requirements/01_business_requirement/08_rule.md#RULE-019) / [API-027](../../02_basic_design/02_backend/03_apis/API-027.md#API-027)) | — | I-06 のチェック結果から生成。API-027 `ids` |
| I-08 | 変更後状態 | 列挙値 | ◯(一括操作時) | — | `draft` / `published` / `hidden`([API-027](../../02_basic_design/02_backend/03_apis/API-027.md#API-027)) | `published` | 一括公開(SCR-008 §6 EVT-08)は `published`、一括非公開化(EVT-09)は `hidden` を固定送信 |
| I-09 | 利用者セッション | 文字列 | ◯ | — | 有効なセッション(Cookie) | — | 認証ヘッダで送信。画面には表示しない(API-025/027/030 共通 `Cookie: session=<token>`) |
| I-10 | CSRF トークン | 文字列 | ◯(一括状態変更時) | — | 有効な CSRF トークン | — | 状態変更系のみ必須([API-027](../../02_basic_design/02_backend/03_apis/API-027.md#API-027) `X-CSRF-Token`)。画面には表示しない |

## 4. 出力項目一覧

各 API レスポンス・画面へ返す出力項目を論理項目単位で定義する。API により出力項目が分岐する。

| 項目ID | 項目名 | データ型 | 表示形式 | 表示条件 | 備考 |
|----|----|----|----|----|----|
| O-01 | FAQ ID | 文字列 | ラベル(コピー可能) | 一覧 1 件以上時 | SCR-008 §4 #5 に対応。API-025 `items[].id` |
| O-02 | 質問 | 文字列 | リンク | 一覧 1 件以上時 | SCR-008 §4 #6 に対応。押下で SCR-009 編集モードへ遷移。API-025 `items[].question` |
| O-03 | カテゴリ | 文字列 | ラベル | 一覧 1 件以上時 | SCR-008 §4 #7 に対応。API-025 `items[].category` |
| O-04 | 状態バッジ | 列挙値 | バッジ(色 + テキストラベル併記) | 一覧 1 件以上時 | SCR-008 §4 #8 に対応。取りうる値は [状態モデル §4.1](../../02_basic_design/08_state-model.md#41-faq-状態)。API-025 `items[].status` |
| O-05 | 更新日時 | 文字列(ISO 8601) | ラベル(ローカライズ表示) | 一覧 1 件以上時 | SCR-008 §4 #9 に対応。API-025 `items[].updatedAt` |
| O-06 | 件数表示 | 数値 | ラベル | 常時 | SCR-008 §4 #10 に対応。取得済み件数を表示 |
| O-07 | 次ページカーソル | 文字列 | 画面非表示 | 続きの FAQ が存在する場合 | API-025 `nextCursor`。末尾は `null`。SCR-008 §4 #11「もっと見る」の表示条件に用いる |
| O-08 | 空状態 | 真偽値 | 空状態表示切替 | 一覧 0 件時 | SCR-008 §4 #20 に対応 |
| O-09 | 一括変更成功件数 | 数値 | トースト文言に反映 | 一括状態変更 実行時 | API-027 `succeeded`。SCR-008 §7 EVT-08/EVT-09 のトースト文言に用いる |
| O-10 | 一括変更失敗明細 | 配列 | トースト件数に反映(明細は画面非表示) | 一括状態変更 実行時かつ失敗 1 件以上 | API-027 `failed`(`id` / `code`)。SCR-008 §7 EVT-08/EVT-09 の「一部失敗」判定に用いる |
| O-11 | CSV ファイル本体 | 文字列(CSV/UTF-8) | ファイルダウンロード | CSV エクスポート成功時 | API-030 レスポンス本体(`text/csv`)。JSON は返らない |

## 5. 入力チェック仕様

各入力項目のチェック内容・実施主体(クライアント / サーバー)・エラー時参照を定義する。文面は本ページに再掲せず ID 参照する。

| 項目名 | チェック内容 | タイミング | エラー時メッセージ | 備考 |
|----|----|----|----|----|
| キーワード検索(SCR-008 §4 #1) | 最大長チェック(100 文字) | クライアント | [SCR-008](../../02_basic_design/01_frontend/01_screens/SCR-008.md#SCR-008) EM-01 | 検索実行自体は API-031(別文書)。本項目は一覧絞り込みの UI 制約として記載 |
| 対象プロジェクト(I-01) | セッションと当該プロジェクトの割当整合 | サーバー | [ERR-019](../../02_basic_design/05_errors/ERR-019.md#ERR-019) | サーバーのみ判定(API-025/027/030 共通 P-01) |
| 状態フィルタ(I-02) | 列挙値チェック | サーバー | [ERR-001](../../02_basic_design/05_errors/ERR-001.md#ERR-001) | `draft` / `published` / `hidden` 以外は拒否 |
| 一括対象 FAQ ID リスト(I-07) | 件数上限チェック([RULE-019](../../01_requirements/01_business_requirement/08_rule.md#RULE-019))・ID 形式(ULID)チェック | サーバー | [ERR-001](../../02_basic_design/05_errors/ERR-001.md#ERR-001) | 上限超過・形式不正は一括リクエスト全体を 400 で拒否([API-027](../../02_basic_design/02_backend/03_apis/API-027.md#API-027) P-02) |
| 一括対象 FAQ ID リスト(I-07) | 行単位の対象外判定(他プロジェクト / 論理削除済み) | サーバー | O-10(失敗明細の `code`) | 上限内の形式チェックを通過した後、行ごとに評価し当該行のみ失敗集計([API-027](../../02_basic_design/02_backend/03_apis/API-027.md#API-027) P-03) |
| 変更後状態(I-08) | 列挙値チェック | サーバー | [ERR-001](../../02_basic_design/05_errors/ERR-001.md#ERR-001) | `draft` / `published` / `hidden` 以外は拒否 |
| 利用者セッション(I-09) | セッション有効性チェック | サーバー | [ERR-019](../../02_basic_design/05_errors/ERR-019.md#ERR-019) | 認証失敗はサーバーのみ判定 |
| CSRF トークン(I-10) | CSRF トークン検証 | サーバー | [ERR-001](../../02_basic_design/05_errors/ERR-001.md#ERR-001) | 一括状態変更のみ必須([API-027](../../02_basic_design/02_backend/03_apis/API-027.md#API-027) 認証ヘッダ) |

## 6. 編集・加工仕様

入力値の正規化・出力値の整形など編集加工を定義する。

| 項目名 | 加工内容 | 加工タイミング | 備考 |
|----|----|----|----|
| 行選択チェック(I-06 → I-07) | 選択済み FAQ ID の集合を一括対象 FAQ ID リストへ変換 | 一括操作押下時(クライアント) | 「選択を解除」(SCR-008 §6 EVT-11)押下時は選択集合を空にする |
| 一括変更成功件数 / 失敗明細(O-09 / O-10) | 成功 / 失敗件数からトースト文言(成功のみ・一部失敗・全件失敗)を判定 | 応答受信時(クライアント) | 全件成功は一覧更新のみ、一部失敗は成功分のみ一覧へ反映、全件失敗は一覧を更新しない([SCR-008](../../02_basic_design/01_frontend/01_screens/SCR-008.md#SCR-008) §7 EVT-08/EVT-09) |
| 状態バッジ(O-04) | 状態値をバッジの色 + テキストラベルへ整形 | 表示時(クライアント) | 色のみに依存しない表示(SCR-008 §4 データパターン) |
| CSV ファイル本体(O-11) | 一覧と同一の絞り込み条件(状態 / プロジェクト / カテゴリ / キーワード)を適用して整形 | エクスポート実行時(サーバー) | キーワードは全文検索(API-031)と同条件を CSV エクスポート(API-030)へ渡す |

## 7. 後続工程への引き継ぎ事項

テスト設計・DB 物理設計・詳細ロジック設計へ引き継ぐ観点を挙げる。一括状態変更のトランザクション制御・CSV 整形アルゴリズムの詳細は別文書へ委ねる。

- 一括対象 FAQ ID リスト(I-07)の境界値(0 件 / RULE-019 上限件数 / 上限超過)、行単位の対象外判定(他プロジェクト / 論理削除済み)の組み合わせはテスト設計でケース化する。
- 一括変更結果(全件成功 / 一部失敗 / 全件失敗)ごとの一覧反映・選択解除・トースト表示分岐はテスト設計でケース化する。
- CSV エクスポート(O-11)は絞り込み結果 0 件時の挙動([UC-028](../../01_requirements/04_business_usecases/UC-028.md#UC-028) 例外フロー)をテスト設計でケース化する。
- 一括状態変更の行単位トランザクション制御は [STS-005](../01_state_transitions/STS-005.md#STS-005)、論理項目 ↔ 物理カラムの対応は [DBP-007](../07_db_physical/DBP-007.md#DBP-007) を参照する。
