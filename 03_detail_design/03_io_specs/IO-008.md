## 1. 目的

本入出力設計は、オーナー / メンバーがダッシュボードを開いた際にセットアップ進捗または KPI 指標を取得して表示する一連の入力/出力項目を、画面と API を突き合わせて論理項目単位で確定する。橋渡しする画面は [SCR-033](../../02_basic_design/01_frontend/01_screens/SCR-033.md#SCR-033) の初期表示・期間切替・プロジェクト絞り込み(EVT-01・EVT-04・EVT-05)、API は [API-063](../../02_basic_design/02_backend/03_apis/API-063.md#API-063)(セットアップ進捗取得)・[API-062](../../02_basic_design/02_backend/03_apis/API-062.md#API-062)(ダッシュボード集計取得)である。

## 2. 対象範囲

本設計が対象とする画面・API・データと対象外を明示する。Myプロジェクト / Joinプロジェクト一覧の取得・請求管理の集計・要対応の質問一覧・FAQ 一覧・質問ログの詳細は別文書へ委ねる。

| 区分 | 対象 |
|----|----|
| 対象画面 | [SCR-033](../../02_basic_design/01_frontend/01_screens/SCR-033.md#SCR-033) ダッシュボード(初期表示 EVT-01・期間切替 EVT-04・プロジェクト絞り込み EVT-05) |
| 対象API | [API-063](../../02_basic_design/02_backend/03_apis/API-063.md#API-063) セットアップ進捗取得・[API-062](../../02_basic_design/02_backend/03_apis/API-062.md#API-062) ダッシュボード集計取得 |
| 対象データ | セットアップ進捗(3 ステップの完了状態・全体完了状態)、KPI 集計期間・対象プロジェクト、質問数・未解決数・公開 FAQ 数・利用率・頻出質問リストの各出力項目 |
| 対象外 | Myプロジェクト / Joinプロジェクト一覧の取得表示(SCR-033 #2・#3)・請求管理の集計([UC-035](../../01_requirements/04_business_usecases/UC-035.md#UC-035) 該当画面)・要対応の質問一覧 / FAQ 一覧 / 質問ログへの遷移先詳細・頻出質問押下時の遷移先詳細 |

## 3. 入力項目一覧

ダッシュボード画面から受け取り API へ渡す入力項目を論理項目単位で定義する。文字数上限などの具体値は基本設計へ ID 参照する。

| 項目ID | 項目名 | データ型 | 必須 | 初期値 | 入力制約 | 入力例 | 備考 |
|----|----|----|----|----|----|----|----|
| I-01 | 利用者セッション | 文字列 | ◯ | — | 有効な利用者セッション(Cookie) | — | 認証ヘッダで送信。画面には表示しない([API-062](../../02_basic_design/02_backend/03_apis/API-062.md#API-062) / [API-063](../../02_basic_design/02_backend/03_apis/API-063.md#API-063) 認証ヘッダ) |
| I-02 | 集計期間区分 | 列挙値 | — | 当月 | `current_month` / `last_30d`(SCR-033 §4 #9) | current_month | 期間切替トグル(#9)に対応。API-062 リクエスト `period` |
| I-03 | 対象プロジェクト | 文字列 | 条件付き | オーナー: My全体(未指定) / メンバー: 参加プロジェクト | 所有プロジェクトを持たず参加のみのユーザーは必須(未指定不可) | prj_01H... | プロジェクト絞り込み(#10)に対応。API-062 リクエスト `projectId` |

## 4. 出力項目一覧

API レスポンス・画面へ返す出力項目を論理項目単位で定義する。セットアップ完了状態により表示パターン(セットアップ進捗 / KPI)が分岐する。

| 項目ID | 項目名 | データ型 | 表示形式 | 表示条件 | 備考 |
|----|----|----|----|----|----|
| O-01 | セットアップステップ配列 | 配列 | 各ステップラベル(SCR-033 §4 #5〜#7) | セットアップ未完了時 | 定義順 3 件(`project_created` / `faq_registered` / `widget_embedded`)。API-063 `steps` |
| O-02 | ステップ完了状態 | 真偽値 | 進捗バー(#4)・各ステップの完了表示 | セットアップ未完了時 | ステップごとの完了可否。API-063 `steps[].done` |
| O-03 | 次アクション CTA 対象ステップ | 列挙値 | 次アクション CTA(#8) | セットアップ未完了時(該当ステップが未完了のときのみ) | 未完了の先頭ステップに対応する CTA を表示。API-063 `steps[].key` から画面側で判定 |
| O-04 | 全体完了状態 | 真偽値 | 表示パターン切替(セットアップ進捗 / KPI) | 常時 | `true` で KPI 表示パターンへ切替。API-063 `completed` |
| O-05 | 集計対象期間 | オブジェクト | 期間切替トグル(#9)の選択状態に反映 | セットアップ完了時 | 起点・終点。API-062 `period.from` / `period.to` |
| O-06 | 質問数 | 数値 | 質問数カード(#11) | セットアップ完了時 | 期間内の総質問数。0 件時はカード非活性(SCR-033 EVT-07)。API-062 `questionCount` |
| O-07 | 未解決数 | 数値 | 未解決数カード(#12) | セットアップ完了時 | 期間内の未解決質問数。0 件時はカード非活性(SCR-033 EVT-08)。API-062 `unresolvedCount` |
| O-08 | 公開 FAQ 数 | 数値 | 公開 FAQ 数カード(#13) | セットアップ完了時 | 公開状態の FAQ 件数。0 件時はカード非活性(SCR-033 EVT-09)。API-062 `publishedFaqCount` |
| O-09 | 利用率 | 数値 | 利用率カード(#14) | セットアップ完了時 | 当月質問数を月次上限で割った比率(0〜1)。しきい値は [システム仕様書 §2](../../02_basic_design/07_system-spec.md#2-課金利用量上限)。過去期間選択時も当月基準である旨を付記(SCR-033 EVT-04)。対応する一覧を持たないため常にクリック不可。API-062 `usageRate` |
| O-10 | 頻出質問リスト | 配列 | 頻出質問リスト(#15・質問 / 出現回数) | セットアップ完了時(1 件以上あるとき) | 出現回数の多い順。押下時 SCR-032 質問ログへ遷移(SCR-033 EVT-06)。API-062 `topQuestions[].question` / `topQuestions[].count` |

## 5. 入力チェック仕様

各入力項目のチェック内容・実施主体(クライアント / サーバー)・エラー時参照を定義する。本画面は入力検証を持たず(SCR-033 §5)、チェックは API 側で実施する。

| 項目名 | チェック内容 | タイミング | エラー時メッセージ | 備考 |
|----|----|----|----|----|
| 利用者セッション(I-01) | セッションの有効性検証 | サーバー | 認可ミドルウェア共通(API-063 `## エラー`) | API-063 は固有エラーコードを持たない |
| 対象プロジェクト(I-03) | 所有プロジェクトを持たず参加のみのユーザーの必須チェック | サーバー | [ERR-001](../../02_basic_design/05_errors/ERR-001.md#ERR-001) | 未指定時 400([API-062](../../02_basic_design/02_backend/03_apis/API-062.md#API-062) `## バリデーション`) |
| 対象プロジェクト(I-03) | 当該プロジェクトへのアクセス権検証 | サーバー | [ERR-019](../../02_basic_design/05_errors/ERR-019.md#ERR-019) | 権限なし時 403(API-062 P-01) |

## 6. 編集・加工仕様

入力値の正規化・出力値の整形など編集加工を定義する。

| 項目名 | 加工内容 | 加工タイミング | 備考 |
|----|----|----|----|
| 集計対象期間(O-05) | 期間解決(`period` から `from` / `to` を確定) | 集計時(サーバー) | API-062 P-02 |
| 利用率(O-09) | 当月質問数 / 月次上限から算出 | 集計時(サーバー) | API-062 P-04。`period` が `last_30d` でも当月基準で算出(API-062 レスポンス項目説明) |
| 質問数カード(O-06)・未解決数カード(O-07)・公開 FAQ 数カード(O-08) | 0 件 / 集計中 / 取得失敗時はクリック不可(非活性)へ整形 | 描画時(クライアント) | SCR-033 EVT-07〜EVT-09。集計中・取得失敗は API 応答遅延・エラー時の画面側表示状態 |

## 7. 後続工程への引き継ぎ事項

テスト設計・DB 物理設計・詳細ロジック設計へ引き継ぐ観点を挙げる。

- セットアップ完了状態(O-04)の切替(未完了 → 完了)に伴う表示パターン遷移(セットアップ進捗 → KPI)はテスト設計でケース化する。
- KPI 各カード(O-06〜O-09)の 0 件 / 集計中 / 取得失敗時の非活性表示と、頻出質問リスト(O-10)の 0 件時非表示はテスト設計でケース化する。
- オーナー(My全体 / 各プロジェクト)とメンバー(参加プロジェクト必須)で異なる対象プロジェクト(I-03)必須条件の組み合わせはテスト設計でパターン化する。
- 集計ロジック(期間解決・利用率算出)の詳細は [IPO](../04_ipo/) 該当文書、論理項目 ↔ 物理カラムの対応は [DBP](../07_db_physical/) 該当文書へ引き継ぐ。
