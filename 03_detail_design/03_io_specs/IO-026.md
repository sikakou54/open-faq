# <span id="IO-026"></span>IO-026: 質問数上限設定 入出力

> **この入出力設計書は「質問数上限設定モーダルとプロジェクト上限・アラート取得 / 更新 API の入力/出力項目・入力チェック・編集加工」を論理項目単位で定義します。**

*種別 入出力設計 ・ ステータス ドラフト*

| 項目 | 値 |
|----|----|
| 入出力設計ID | IO-026 |
| 業務ユースケースID | [UC-034](../../01_requirements/04_business_usecases/UC-034.md#UC-034) |
| 関連画面 | [SCR-027](../../02_basic_design/01_frontend/01_screens/SCR-027.md#SCR-027) |
| 関連 API | [API-046](../../02_basic_design/02_backend/03_apis/API-046.md#API-046) ・ [API-047](../../02_basic_design/02_backend/03_apis/API-047.md#API-047) |

## 1. 目的

本入出力設計は、オーナー / メンバーが質問数の月次上限のオン / オフ・上限件数・アラート閾値を確認し、課金見込みを見ながら保存する一連の入力/出力項目を、画面と API を突き合わせて論理項目単位で確定する。橋渡しする画面は [SCR-027](../../02_basic_design/01_frontend/01_screens/SCR-027.md#SCR-027) の質問数上限設定モーダル(EVT-01〜EVT-06)、API は取得を担う [API-046](../../02_basic_design/02_backend/03_apis/API-046.md#API-046) と更新を担う [API-047](../../02_basic_design/02_backend/03_apis/API-047.md#API-047) である。

## 2. 対象範囲

本設計が対象とする画面・API・データと対象外を明示する。上限消化時の参考課金額の算出ロジックと再認証そのものの入出力は別文書へ委ねる。

| 区分 | 対象 |
|----|----|
| 対象画面 | [SCR-027](../../02_basic_design/01_frontend/01_screens/SCR-027.md#SCR-027) 質問数上限設定(モーダル)(EVT-01〜EVT-06) |
| 対象API | [API-046](../../02_basic_design/02_backend/03_apis/API-046.md#API-046) プロジェクト上限・アラート取得 ・ [API-047](../../02_basic_design/02_backend/03_apis/API-047.md#API-047) プロジェクト上限・アラート更新 |
| 対象データ | 月次上限のオン / オフ・上限件数・許容範囲(最小 / 最大件数)・アラート閾値・参考課金額 |
| 対象外 | 参考課金額の算出ロジック([API-046](../../02_basic_design/02_backend/03_apis/API-046.md#API-046))・再認証そのものの入出力(別 API-005)・アラート発火時の通知処理 |

## 3. 入力項目一覧

画面から受け取り API へ渡す入力項目を論理項目単位で定義する。文字数上限・範囲などの具体値は基本設計へ ID 参照する。

| 項目ID | 項目名 | データ型 | 必須 | 初期値 | 入力制約 | 入力例 | 備考 |
|----|----|----|----|----|----|----|----|
| I-01 | 上限設定(ON / OFF) | 真偽値 | ◯ | 現在値(未設定時 OFF) | ON / OFF の二値 | `true` | SCR-027 §4 #1 に対応。API リクエスト `limitEnabled` |
| I-02 | 今月の利用上限(件数) | 数値(整数) | I-01 が ON のとき必須 | 現在値 | 1 件刻み・[API-046](../../02_basic_design/02_backend/03_apis/API-046.md#API-046) `minLimit`〜`maxLimit` の範囲内([SCR-027](../../02_basic_design/01_frontend/01_screens/SCR-027.md#SCR-027) §5 EM-02 / EM-03) | `5000` | SCR-027 §4 #2 に対応。API リクエスト `limit`。I-01 が OFF のとき `null` |
| I-03 | アラート閾値(複数選択) | 配列(数値) | — | 現在値(未設定時 全未選択) | `25` / `50` / `80` / `90` / `100` の重複しない値のみ([SCR-027](../../02_basic_design/01_frontend/01_screens/SCR-027.md#SCR-027) §5 EM-04) | `[25, 50, 80, 90, 100]` | SCR-027 §4 #5 に対応。API リクエスト `alertThresholds`。空配列で通知なし |
| I-04 | 再認証パスワード | 文字列 | ◯ | — | 1〜128 文字 | — | SCR-027 §4 #9 に対応。画面入力値から再認証トークンを取得し API リクエスト `reauthToken` として送信([API-005](../../02_basic_design/02_backend/03_apis/API-005.md#API-005)) |

## 4. 出力項目一覧

API レスポンス・画面へ返す出力項目を論理項目単位で定義する。取得(API-046)・更新(API-047)のいずれの結果も同じ論理項目で画面へ反映する。

| 項目ID | 項目名 | データ型 | 表示形式 | 表示条件 | 備考 |
|----|----|----|----|----|----|
| O-01 | 上限設定(ON / OFF) | 真偽値 | ラジオボタンの選択状態(SCR-027 §4 #1) | 常時 | API-046 `questions.limitEnabled` / API-047 `limitEnabled` |
| O-02 | 今月の利用上限(件数) | 数値(整数) | 入力欄の初期値(SCR-027 §4 #2) | 常時(O-01 が OFF のとき `null`) | API-046 `questions.limit` / API-047 `limit` |
| O-03 | 許容最小件数 | 数値(整数) | 画面非表示(入力バリデーションの下限として利用) | O-01 が ON のとき | API-046 `questions.minLimit`。SCR-027 §5 EM-03 の判定に用いる |
| O-04 | 許容最大件数 | 数値(整数) | 入力欄の最大長ガード(SCR-027 §4 #2) | O-01 が ON のとき | API-046 `questions.maxLimit`。SCR-027 §5 EM-03 の判定に用いる |
| O-05 | 課金計算式の併記 | 文字列(整形表示) | 課金計算式ラベル(SCR-027 §4 #3) | O-01 が ON のとき | 「(上限件数) −(無料枠)=(課金対象件数)(参考額 / 月)」の形式で表示([API-046](../../02_basic_design/02_backend/03_apis/API-046.md#API-046))。無料枠・単価は本ページで再定義せず[システム仕様書 §2](../../02_basic_design/07_system-spec.md#2-課金利用量上限)を参照 |
| O-06 | 参考課金額 | 数値(整数) | O-05 の一部として表示 | O-01 が ON のとき | API-046 `questions.yenEquivalent` / API-047 `yenEquivalent`。O-01 が OFF のとき `null` |
| O-07 | アラート閾値(複数選択) | 配列(数値) | チェックボックスの選択状態(SCR-027 §4 #5) | 常時(O-01 が OFF のとき全未選択・入力不可) | API-046 `questions.alertThresholds` / API-047 `alertThresholds`(昇順) |
| O-08 | 更新日時 | 文字列(ISO 8601) | 画面非表示 | 保存成功時 | API-047 `updatedAt` |

## 5. 入力チェック仕様

各入力項目のチェック内容・実施主体(クライアント / サーバー)・エラー時参照を定義する。クライアント検証違反時は保存を中止し、サーバー検証違反時はエラーコードに応じた画面挙動を行う。文面は本ページに再掲せず ID 参照する。

| 項目名 | チェック内容 | タイミング | エラー時メッセージ | 備考 |
|----|----|----|----|----|
| 今月の利用上限(I-02) | 未入力チェック(I-01 が ON のとき) | クライアント・サーバー | [SCR-027](../../02_basic_design/01_frontend/01_screens/SCR-027.md#SCR-027) EM-01 / [ERR-001](../../02_basic_design/05_errors/ERR-001.md#ERR-001) | クライアントは入力時・保存時に判定([SCR-027](../../02_basic_design/01_frontend/01_screens/SCR-027.md#SCR-027) §5)。違反時は保存を中止 |
| 今月の利用上限(I-02) | 整数値チェック(1 件刻み) | クライアント・サーバー | [SCR-027](../../02_basic_design/01_frontend/01_screens/SCR-027.md#SCR-027) EM-02 / [ERR-001](../../02_basic_design/05_errors/ERR-001.md#ERR-001) | 違反時は件数エラー(SCR-027 §4 #4)を表示し保存不可 |
| 今月の利用上限(I-02) | 件数範囲チェック(O-03〜O-04 の範囲内) | クライアント・サーバー | [SCR-027](../../02_basic_design/01_frontend/01_screens/SCR-027.md#SCR-027) EM-03 / [ERR-001](../../02_basic_design/05_errors/ERR-001.md#ERR-001) | サーバーは KV 値([TBL-009](../../02_basic_design/02_backend/04_database/TBL-009.md#TBL-009) コード値・区分値)を正本に再判定。クライアントは取得済みの O-03〜O-04 を用いる |
| アラート閾値(I-03) | 許容値チェック(`25`/`50`/`80`/`90`/`100` のみ・重複不可) | クライアント・サーバー | [SCR-027](../../02_basic_design/01_frontend/01_screens/SCR-027.md#SCR-027) EM-04 / [ERR-001](../../02_basic_design/05_errors/ERR-001.md#ERR-001) | 保存時に判定([SCR-027](../../02_basic_design/01_frontend/01_screens/SCR-027.md#SCR-027) §5)。サーバーは重複除去せず不正値を検証で拒否 |
| 未サポート項目 | `freeQuota` / `alertEnabled` / `alertFrequency` を指定していないか | サーバー | [ERR-029](../../02_basic_design/05_errors/ERR-029.md#ERR-029) | 画面からは送信しない項目。API 単独呼び出し時の異常系 |
| 再認証パスワード(I-04) | 再認証トークンの検証 | サーバー | [SCR-027](../../02_basic_design/01_frontend/01_screens/SCR-027.md#SCR-027) EM-05 / [ERR-013](../../02_basic_design/05_errors/ERR-013.md#ERR-013) | 保存(EVT-05)時に判定。違反時は再認証モーダルにエラー表示し保存を中断 |
| 操作権限 | 当該プロジェクトへの割当チェック(オーナー / メンバー) | サーバー | [ERR-030](../../02_basic_design/05_errors/ERR-030.md#ERR-030) | 境界を通過した無関係ユーザーによる操作を拒否 |

## 6. 編集・加工仕様

入力値の正規化・出力値の整形など編集加工を定義する。

| 項目名 | 加工内容 | 加工タイミング | 備考 |
|----|----|----|----|
| 上限設定(I-01) | OFF 選択時は今月の利用上限(I-02)を `null`、アラート閾値(I-03)を空配列へ正規化 | 保存前(サーバー) | [API-047](../../02_basic_design/02_backend/03_apis/API-047.md#API-047) `limitEnabled=false` の入力仕様に合わせる |
| アラート閾値(I-03 / O-07) | 昇順への正規化 | 保存時(サーバー) | [API-047](../../02_basic_design/02_backend/03_apis/API-047.md#API-047)。画面は取得値をそのままチェック状態へ反映(SCR-027 §4 #5) |
| 課金計算式の併記(O-05) | 上限件数・無料枠・課金対象件数・参考額を整形表示 | 応答描画時(クライアント) | 数値は[システム仕様書 §2](../../02_basic_design/07_system-spec.md#2-課金利用量上限)の無料枠・単価を用いてサーバー側で算出済みの値をそのまま表示し、クライアントでは再計算しない |

## 7. 後続工程への引き継ぎ事項

テスト設計・DB 物理設計・詳細ロジック設計へ引き継ぐ観点を挙げる。参考課金額の算出ロジック・再認証の詳細は別文書へ委ねる。

- 今月の利用上限(I-02)の境界値(許容最小件数 O-03、許容最大件数 O-04 の前後)と、上限 ON / OFF 切替時の入力可否・値クリアはテスト設計でパターン化する。
- アラート閾値(I-03)の全未選択(通知なし)・全選択・重複指定・許容外値指定の組み合わせはテスト設計でケース化する。
- 再認証失敗時([SCR-027](../../02_basic_design/01_frontend/01_screens/SCR-027.md#SCR-027) EM-05)に画面側の入力値(I-01〜I-03)が保持されるかはテスト設計で確認する。
- 論理項目 ↔ 物理カラムの対応は [DBP-010](../07_db_physical/DBP-010.md#DBP-010) で確定する。
