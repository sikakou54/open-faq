# <span id="IO-006"></span>IO-006: メンバーアカウント有効化 入出力

> **この入出力設計書は「招待されたユーザーの参加受諾画面と招待トークン検証・招待受諾 API の入力/出力項目・入力チェック・編集加工」を論理項目単位で定義します。**

*種別 入出力設計 ・ ステータス ドラフト*

| 項目 | 値 |
|----|----|
| 入出力設計ID | IO-006 |
| 業務ユースケースID | [UC-006](../../01_requirements/04_business_usecases/UC-006.md#UC-006) |

| 関連 API | [API-007](../../02_basic_design/02_backend/03_apis/API-007.md#API-007) ・ [API-008](../../02_basic_design/02_backend/03_apis/API-008.md#API-008) |

## 1. 目的

本入出力設計は、招待された登録済みユーザー本人が招待内容を確認したうえで参加を受諾し、対象プロジェクトへの割当が有効になる一連の入力/出力項目を、画面と API を突き合わせて論理項目単位で確定する。橋渡しする画面は [SCR-023](../../02_basic_design/01_frontend/01_screens/SCR-023.md#SCR-023) の初期表示(EVT-01)・参加を受諾する押下(EVT-04)、API は [API-007](../../02_basic_design/02_backend/03_apis/API-007.md#API-007)(検証・プレビュー)と [API-008](../../02_basic_design/02_backend/03_apis/API-008.md#API-008)(受諾・割当有効化)である。

## 2. 対象範囲

本設計が対象とする画面・API・データと対象外を明示する。招待トークンの判定ロジック自体・アカウント新規作成・招待発行は別文書へ委ねる。

| 区分 | 対象 |
|----|----|
| 対象画面 | [SCR-023](../../02_basic_design/01_frontend/01_screens/SCR-023.md#SCR-023) メンバーアカウント有効化(招待受諾)(初期表示 EVT-01、参加を受諾する EVT-04) |
| 対象API | [API-007](../../02_basic_design/02_backend/03_apis/API-007.md#API-007) 招待トークン検証・プレビュー ・ [API-008](../../02_basic_design/02_backend/03_apis/API-008.md#API-008) 招待受諾(割当有効化) |
| 対象データ | 招待トークン・招待内容プレビュー(プロジェクト名 / 招待元オーナー名 / 宛先メール)・受諾結果(割当有効化・遷移先) |
| 対象外 | 招待トークンの状態判定・宛先一致判定ロジック([IPO-011](../04_ipo/IPO-011.md#IPO-011))・アカウント新規作成(独立サインアップ、[API-001](../../02_basic_design/02_backend/03_apis/API-001.md#API-001))・招待の発行([SCR-014](../../02_basic_design/01_frontend/01_screens/SCR-014.md#SCR-014)) |

## 3. 入力項目一覧

画面から受け取り API へ渡す入力項目を論理項目単位で定義する。本画面は表示名・パスワード・規約同意の入力は行わない(参加受諾のみを担う)。

| 項目ID | 項目名 | データ型 | 必須 | 初期値 | 入力制約 | 入力例 | 備考 |
|----|----|----|----|----|----|----|----|
| I-01 | 招待トークン | 文字列 | ◯ | — | 招待メールのリンクから URL パスで取得(画面入力なし) | — | [API-007](../../02_basic_design/02_backend/03_apis/API-007.md#API-007) / [API-008](../../02_basic_design/02_backend/03_apis/API-008.md#API-008) いずれもパスパラメータ `token` |
| I-02 | セッション認証(ログイン中ユーザー) | — | ◯(受諾時のみ) | — | 有効なログインセッションであること | — | プレビュー(API-007)は認証不要。受諾(API-008)は必須([API-008](../../02_basic_design/02_backend/03_apis/API-008.md#API-008) `## リクエスト` 認証ヘッダ) |
| I-03 | 参加を受諾するボタン押下 | — | — | — | 招待内容パネル表示中(トークン検証成功・本人一致)のみ押下可能 | — | SCR-023 §4 #4 に対応(EVT-04)。API へのリクエストボディはなし([API-008](../../02_basic_design/02_backend/03_apis/API-008.md#API-008) `## リクエスト`) |

## 4. 出力項目一覧

API レスポンス・画面へ返す出力項目を論理項目単位で定義する。API-007(プレビュー)と API-008(受諾)で出力項目が分かれる。

| 項目ID | 項目名 | データ型 | 表示形式 | 表示条件 | 備考 |
|----|----|----|----|----|----|
| O-01 | 招待先プロジェクト名 | 文字列 | 招待内容パネルに表示(SCR-023 §4 #1) | トークン検証成功・本人一致 | API-007 `projectName` |
| O-02 | 招待元オーナー名 | 文字列 | 招待内容パネルに表示(SCR-023 §4 #1) | トークン検証成功・本人一致 | API-007 `inviterOwnerName` |
| O-03 | ログイン中のメールアドレス | 文字列 | ログイン中のメールアドレス欄に表示(変更不可)(SCR-023 §4 #3) | トークン検証成功・本人一致 | API-007 `email` |
| O-04 | 招待トークン有効期限 | 文字列(ISO 8601) | 画面非表示 | トークン検証成功・本人一致 | 期限切れ判定は表示前にサーバー側で確定済みのため画面では利用しない。API-007 `expiresAt` |
| O-05 | 招待先プロジェクト識別子 | 文字列 | 画面非表示 | トークン検証成功・本人一致 | API-007 `projectId`。受諾押下時の対象プロジェクト特定に用いる |
| O-06 | 受諾結果(割当有効化済みプロジェクト識別子) | 文字列 | 画面非表示 | 受諾成功時 | API-008 `projectId` |
| O-07 | 受諾完了案内 | — | 受諾完了案内に表示(SCR-023 §4 #8)・ダッシュボードへ進むボタン表示(SCR-023 §4 #9) | 受諾成功時 | API-008 応答 200 を受けて画面側で確定(専用フラグ項目なし) |
| O-08 | 受諾後の遷移先 URL | 文字列 | 「ダッシュボードへ進む」押下時の遷移先(SCR-023 §4 #9、EVT-05) | 受諾成功時 | API-008 `redirectUrl` |

## 5. 入力チェック仕様

各入力項目のチェック内容・実施主体(クライアント / サーバー)・エラー時参照を定義する。本画面は入力項目を持たないため(SCR-023 §5)、チェックはすべてサーバー側のトークン・認証検証である。判定ロジックの詳細は [IPO-011](../04_ipo/IPO-011.md#IPO-011) を参照する。

| 項目名 | チェック内容 | タイミング | エラー時メッセージ | 備考 |
|----|----|----|----|----|
| 招待トークン(I-01) | 存在チェック | サーバー | [SCR-023](../../02_basic_design/01_frontend/01_screens/SCR-023.md#SCR-023) EM-01 / [ERR-008](../../02_basic_design/05_errors/ERR-008.md#ERR-008) | プレビュー(EVT-01)・受諾(EVT-04)いずれも実施([IPO-011](../04_ipo/IPO-011.md#IPO-011) 処理No.1) |
| 招待トークン(I-01) | 有効期限チェック | サーバー | [SCR-023](../../02_basic_design/01_frontend/01_screens/SCR-023.md#SCR-023) EM-01 / [ERR-006](../../02_basic_design/05_errors/ERR-006.md#ERR-006) | 有効期限は[システム仕様書 §4](../../02_basic_design/07_system-spec.md#4-データ保持期間削除猶予)を正本とする 7 日([IPO-011](../04_ipo/IPO-011.md#IPO-011) 処理No.2) |
| 招待トークン(I-01) | 使用済みチェック | サーバー | [SCR-023](../../02_basic_design/01_frontend/01_screens/SCR-023.md#SCR-023) EM-02 / [ERR-007](../../02_basic_design/05_errors/ERR-007.md#ERR-007) | 使用済み判定は期限切れ判定より優先して評価([IPO-011](../04_ipo/IPO-011.md#IPO-011) 処理No.2) |
| セッション認証(I-02) | ログイン中ユーザーと招待先メールの一致チェック(本人受諾) | サーバー | [SCR-023](../../02_basic_design/01_frontend/01_screens/SCR-023.md#SCR-023) EM-03 / [ERR-030](../../02_basic_design/05_errors/ERR-030.md#ERR-030) | 受諾(EVT-04)時のみ実施。プレビュー(EVT-01)は未ログイン時にログイン誘導のみ行い本判定は行わない([IPO-011](../04_ipo/IPO-011.md#IPO-011) 処理No.4) |

## 6. 編集・加工仕様

入力値の正規化・出力値の整形など編集加工を定義する。

| 項目名 | 加工内容 | 加工タイミング | 備考 |
|----|----|----|----|
| 招待トークン(I-01) | 平文トークンをハッシュ化して照会 | 検証前(サーバー) | 照会方法は [IPO-011](../04_ipo/IPO-011.md#IPO-011) 処理No.1。画面には元のトークン値を保持・表示しない |
| ログイン中のメールアドレス(O-03) | 加工なし(取得値をそのまま表示) | 画面表示時 | 変更不可欄として表示(SCR-023 §4 #3) |

## 7. 後続工程への引き継ぎ事項

テスト設計・DB 物理設計・詳細ロジック設計へ引き継ぐ観点を挙げる。招待トークンの判定ロジック自体の境界値検証は IPO-011 側の引き継ぎ事項を参照する。

- プレビュー(EVT-01・API-007)と受諾(EVT-04・API-008)で判定範囲が異なる(受諾のみ本人一致・重複割当確認を追加実施する)ことをテスト設計でケース化する。
- 招待内容パネル表示中に招待トークンが並行して失効(期限切れ / 他端末で先に受諾済みで使用済み化)した場合の、受諾(EVT-04)実行時のエラー分岐(EM-01 / EM-02)をテスト設計でケース化する。
- 招待トークン ↔ 物理カラムの対応は [DBP-002](../07_db_physical/DBP-002.md#DBP-002) で確定する。
- 招待トークンの状態判定順序・重複割当の整合異常時の挙動は [IPO-011](../04_ipo/IPO-011.md#IPO-011) §5 を参照する。
