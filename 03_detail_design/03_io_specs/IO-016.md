# <span id="IO-016"></span>IO-016: 概要(プロジェクトホーム) 入出力

> **この入出力設計書は「プロジェクト概要画面とダッシュボードサマリ API の入力/出力項目・入力チェック・編集加工」を論理項目単位で定義します。**

*種別 入出力設計 ・ ステータス ドラフト*

| 項目 | 値 |
|----|----|
| 入出力設計ID | IO-016 |
| 業務ユースケースID | [UC-001](../../01_requirements/04_business_usecases/UC-001.md#UC-001) ・ [UC-032](../../01_requirements/04_business_usecases/UC-032.md#UC-032) |
| 関連画面 | [SCR-012](../../02_basic_design/01_frontend/01_screens/SCR-012.md#SCR-012) |
| 関連 API | [API-040](../../02_basic_design/02_backend/03_apis/API-040.md#API-040) |

## 1. 目的

本入出力設計は、オーナー / メンバーが選択中プロジェクトの概要(質問数・未解決数・公開 FAQ 件数・質問数の推移・要対応の質問)を確認する一連の入力/出力項目を、画面と API を突き合わせて論理項目単位で確定する。橋渡しする画面は [SCR-012](../../02_basic_design/01_frontend/01_screens/SCR-012.md#SCR-012) の初期表示・期間選択(EVT-01・EVT-02)、API は [API-040](../../02_basic_design/02_backend/03_apis/API-040.md#API-040) である。

## 2. 対象範囲

本設計が対象とする画面・API・データと対象外を明示する。オーナー / メンバーで異なる権限別のバナー・アラート出し分けを対象範囲に含み、遷移先画面(要対応の質問一覧・FAQ 一覧・請求・利用量と上限)の詳細は別文書へ委ねる。

| 区分 | 対象 |
|----|----|
| 対象画面 | [SCR-012](../../02_basic_design/01_frontend/01_screens/SCR-012.md#SCR-012) 概要(初期表示 EVT-01・期間選択 EVT-02) |
| 対象API | [API-040](../../02_basic_design/02_backend/03_apis/API-040.md#API-040) ダッシュボードサマリ |
| 対象データ | 集計期間、質問数・未解決数・公開 FAQ 件数・質問数の推移・要対応の質問(最近)・集計完了時刻の各出力項目、権限(オーナー / メンバー)別のバナー・アラート表示制御 |
| 対象外 | 質問数カード / 未解決数カード押下時の要対応の質問一覧([SCR-006](../../02_basic_design/01_frontend/01_screens/SCR-006.md#SCR-006))・公開 FAQ 件数カード押下時の FAQ 一覧([SCR-008](../../02_basic_design/01_frontend/01_screens/SCR-008.md#SCR-008))・支払方法更新 / 登録時の請求画面([SCR-028](../../02_basic_design/01_frontend/01_screens/SCR-028.md#SCR-028))・利用量と上限画面([SCR-026](../../02_basic_design/01_frontend/01_screens/SCR-026.md#SCR-026))の各詳細、通知失敗 / バウンス集計(オーナーのみ返す `notification` は本画面が表示しないため対象外) |

## 3. 入力項目一覧

概要画面から受け取り API へ渡す入力項目を論理項目単位で定義する。文字数上限などの具体値は基本設計へ ID 参照する。

| 項目ID | 項目名 | データ型 | 必須 | 初期値 | 入力制約 | 入力例 | 備考 |
|----|----|----|----|----|----|----|----|
| I-01 | 利用者セッション | 文字列 | ◯ | — | 有効な利用者セッション(Cookie) | — | 認証ヘッダで送信。画面には表示しない([API-040](../../02_basic_design/02_backend/03_apis/API-040.md#API-040) 認証ヘッダ) |
| I-02 | 対象プロジェクト | 文字列 | ◯ | 選択中プロジェクト | 有効なプロジェクト識別子。メンバーは必須([API-040](../../02_basic_design/02_backend/03_apis/API-040.md#API-040) `## バリデーション`) | prj_01H... | 画面には表示せずプロジェクト選択状態から確定する。API-040 リクエスト `projectId` |
| I-03 | 期間選択 | 列挙値 | — | 当月 | `current_month` / `last_month` / `custom`(SCR-012 §4 #2) | current_month | SCR-012 §4 #2 に対応。API-040 リクエスト `period` |
| I-04 | 任意期間(起点・終点) | オブジェクト | 条件付き | — | 期間選択(I-03)が `custom` のとき必須。最大 13 ヶ月([API-040](../../02_basic_design/02_backend/03_apis/API-040.md#API-040) `## バリデーション`) | — | SCR-012 §4 #2 の任意期間指定に対応。API-040 リクエスト `from` / `to` |

## 4. 出力項目一覧

API レスポンス・画面へ返す出力項目を論理項目単位で定義する。権限(オーナー / メンバー)・プロジェクト課金状態・利用状況により表示するバナー・アラートが分岐する。

| 項目ID | 項目名 | データ型 | 表示形式 | 表示条件 | 備考 |
|----|----|----|----|----|----|
| O-01 | 集計対象期間 | オブジェクト | 期間選択(#2)の選択状態に反映 | 常時 | 起点・終点。API-040 `period.from` / `period.to` |
| O-02 | 質問数 | 数値 | 質問数カード(#5) | 常時 | 期間内の総質問数。0 件 / 集計中 / 取得失敗時はクリック不可(SCR-012 EVT-03)。API-040 `questions.total` |
| O-03 | 未解決数 | 数値 | 未解決数カード(#6) | 常時 | 期間内の未解決質問数。0 件 / 集計中 / 取得失敗時はクリック不可(SCR-012 EVT-04)。API-040 `questions.unresolved` |
| O-04 | 公開 FAQ 件数 | 数値 | 公開 FAQ 件数カード(#7) | 常時 | 公開状態の FAQ 件数。0 件 / 集計中 / 取得失敗時はクリック不可(SCR-012 EVT-05)。API-040 `publishedFaqCount` |
| O-05 | 質問数の推移 | 配列 | 質問数の推移チャート(#8) | 常時 | 日次の質問数推移(表示期間は[システム仕様書 §2](../../02_basic_design/07_system-spec.md#2-課金利用量上限))。API-040 `dailyTrend[].date` / `dailyTrend[].count` |
| O-06 | 要対応の質問(最近) | 配列 | 要対応の質問(最近)リスト(#9) | 常時(0 件時はリスト空表示) | 未解決の新着順(設計値 N 件は[システム仕様書 §2](../../02_basic_design/07_system-spec.md#2-課金利用量上限)の日次トレンド表示期間と同様に API-040 側の設計値)。押下時は SCR-012 EVT-09 で要対応の質問一覧へ遷移。API-040 `recentInquiries[].inquiryId` / `recentInquiries[].question` / `recentInquiries[].createdAt` |
| O-07 | 最終更新タイムスタンプ | 文字列 | 最終更新タイムスタンプ(#4) | 常時 | 集計完了時刻。現在時刻から 5 分以上前は「集計遅延」表記(API-040 レスポンス項目説明)。API-040 `lastAggregatedAt` |
| O-08 | 無料利用枠超過フラグ | 真偽値 | 無料利用枠超過バナー(#3) | プロジェクトが無料利用枠を超過した場合(オーナーのみ「支払い方法を登録」導線を表示) | しきい値は[システム仕様書 §2](../../02_basic_design/07_system-spec.md#2-課金利用量上限)。画面側で保有する権限別出し分け(SCR-012 EVT-07) |
| O-09 | サスペンション状態フラグ | 真偽値 | サスペンション中アラート(#11) | プロジェクト課金状態が停止中(サスペンション)の場合(オーナーのみ「支払方法を更新」導線を表示) | 状態の意味は[状態モデル §2](../../02_basic_design/08_state-model.md#2-課金アカウント状態)。画面側で保有する権限別出し分け(SCR-012 EVT-06) |
| O-10 | 質問数上限到達フラグ | 真偽値 | 質問数上限到達バナー(#12) | 質問数が月次上限に到達した場合 | しきい値は[システム仕様書 §2](../../02_basic_design/07_system-spec.md#2-課金利用量上限)。オーナー / メンバー共通表示(SCR-012 EVT-08) |

## 5. 入力チェック仕様

各入力項目のチェック内容・実施主体(クライアント / サーバー)・エラー時参照を定義する。本画面は集計値の閲覧のみで利用者が値を入力する項目を持たず(SCR-012 §5)、チェックは API 側で実施する。

| 項目名 | チェック内容 | タイミング | エラー時メッセージ | 備考 |
|----|----|----|----|----|
| 利用者セッション(I-01) | セッションの有効性検証 | サーバー | [ERR-001](../../02_basic_design/05_errors/ERR-001.md#ERR-001) | 認可ミドルウェア共通 |
| 対象プロジェクト(I-02) | メンバーの `projectId` 必須チェック | サーバー | [ERR-001](../../02_basic_design/05_errors/ERR-001.md#ERR-001) | 未指定時 400([API-040](../../02_basic_design/02_backend/03_apis/API-040.md#API-040) `## エラー`) |
| 対象プロジェクト(I-02) | 当該プロジェクトへのアクセス権検証 | サーバー | [ERR-019](../../02_basic_design/05_errors/ERR-019.md#ERR-019) | 権限なし時 403(API-040 P-01) |
| 任意期間(I-04) | 期間選択(I-03)が `custom` のときの起点・終点必須チェック | サーバー | [ERR-001](../../02_basic_design/05_errors/ERR-001.md#ERR-001) | 未指定時 400([API-040](../../02_basic_design/02_backend/03_apis/API-040.md#API-040) `## バリデーション`) |

## 6. 編集・加工仕様

入力値の正規化・出力値の整形など編集加工を定義する。

| 項目名 | 加工内容 | 加工タイミング | 備考 |
|----|----|----|----|
| 集計対象期間(O-01) | 期間解決(`period` と `from` / `to` は排他。双方指定時は `from` / `to` を優先) | 集計時(サーバー) | API-040 P-02。基準タイムゾーンは UTC(API-040 `## バリデーション`) |
| 最終更新タイムスタンプ(O-07) | 現在時刻との差分判定(5 分以上前は「集計遅延」表記へ整形) | 描画時(クライアント) | API-040 レスポンス項目説明 |
| 質問数カード(O-02)・未解決数カード(O-03)・公開 FAQ 件数カード(O-04) | 0 件 / 集計中 / 取得失敗時はクリック不可(非活性)へ整形 | 描画時(クライアント) | SCR-012 EVT-03〜EVT-05。集計中・取得失敗は API 応答遅延・エラー時の画面側表示状態 |
| 要対応の質問(最近)(O-06) の発生日時 | 相対表記(「3 分前」等)へ整形 | 描画時(クライアント) | API-040 `recentInquiries[].createdAt` の説明に基づく整形 |
| バナー・アラート(O-08〜O-10) | オーナー / メンバーの権限に応じて操作導線(「支払方法を更新」「支払い方法を登録」)の表示可否を出し分け | 描画時(クライアント) | メンバーは同一条件でバナー・アラート自体は表示するが操作導線は表示しない(SCR-012 EVT-06・EVT-07 はオーナー限定と明記) |

## 7. 後続工程への引き継ぎ事項

テスト設計・DB 物理設計・詳細ロジック設計へ引き継ぐ観点を挙げる。

- 権限(オーナー / メンバー)別のバナー・アラート表示と操作導線(「支払方法を更新」「支払い方法を登録」)の出し分けはテスト設計でケース化する。
- 質問数カード(O-02)・未解決数カード(O-03)・公開 FAQ 件数カード(O-04)の 0 件 / 集計中 / 取得失敗時の非活性表示と、要対応の質問(最近)(O-06)の 0 件時空表示はテスト設計でケース化する。
- 無料利用枠超過(O-08)・サスペンション(O-09)・質問数上限到達(O-10)の複数条件が同時に成立する場合のバナー・アラート同時表示可否はテスト設計でパターン化する。
- 集計ロジック(期間解決・日次推移集計・要対応の新着抽出)の詳細は [IPO](../04_ipo/) 該当文書、論理項目 ↔ 物理カラムの対応は [DBP](../07_db_physical/) 該当文書へ引き継ぐ。
