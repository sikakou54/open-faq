## 1. 目的

本入出力設計は、アカウント利用者が個人設定画面で自身のプロフィール・参加プロジェクトを確認し、表示名の保存とパスワードの変更を行う一連の入力/出力項目を、画面と API を突き合わせて論理項目単位で確定する。橋渡しする画面は [SCR-022](../../02_basic_design/01_frontend/01_screens/SCR-022.md#SCR-022) の初期表示(EVT-01)・プロフィール保存(EVT-03)・パスワード変更(EVT-04)と、重要操作の本人確認を担う [SCR-034](../../02_basic_design/01_frontend/01_screens/SCR-034.md#SCR-034)、API は [API-064](../../02_basic_design/02_backend/03_apis/API-064.md#API-064)・[API-012](../../02_basic_design/02_backend/03_apis/API-012.md#API-012)・[API-013](../../02_basic_design/02_backend/03_apis/API-013.md#API-013)である。

## 2. 対象範囲

本設計が対象とする画面・API・データと対象外を明示する。メールアドレス変更(表示名以外のプロフィール保存経路)は既に [IO-013](IO-013.md#IO-013) が対象としており、本設計では扱わない。

| 区分 | 対象 |
|----|----|
| 対象画面 | [SCR-022](../../02_basic_design/01_frontend/01_screens/SCR-022.md#SCR-022) 個人設定(初期表示 EVT-01・プロフィール保存 EVT-03 のうち表示名のみ変更する経路・パスワード変更 EVT-04)・[SCR-034](../../02_basic_design/01_frontend/01_screens/SCR-034.md#SCR-034) 再認証ダイアログ(EVT-01・EVT-02) |
| 対象API | [API-064](../../02_basic_design/02_backend/03_apis/API-064.md#API-064) 自己プロフィール取得 ・ [API-012](../../02_basic_design/02_backend/03_apis/API-012.md#API-012) 自己プロフィール更新 ・ [API-013](../../02_basic_design/02_backend/03_apis/API-013.md#API-013) 自己パスワード変更 |
| 対象データ | 表示名・連絡先メール・確認状態・参加プロジェクト一覧(プロジェクトごとの立場)の取得、表示名の入力・保存、新しいパスワード・確認用パスワード・再認証トークンの入力とパスワード更新結果 |
| 対象外 | メールアドレス変更を含むプロフィール保存経路([API-015](../../02_basic_design/02_backend/03_apis/API-015.md#API-015)・[IO-013](IO-013.md#IO-013)で扱う)・再認証トークンの発行そのもの([API-005](../../02_basic_design/02_backend/03_apis/API-005.md#API-005))・メールアドレス変更確認手続き([SCR-018](../../02_basic_design/01_frontend/01_screens/SCR-018.md#SCR-018))・参加プロジェクト名リンクからの遷移先([SCR-012](../../02_basic_design/01_frontend/01_screens/SCR-012.md#SCR-012)の内容)・支払方法登録変更と退会([SCR-029](../../02_basic_design/01_frontend/01_screens/SCR-029.md#SCR-029)) |

## 4. 出力項目一覧

API レスポンス・画面へ返す出力項目を論理項目単位で定義する。

| 項目ID | 項目名 | データ型 | 表示形式 | 表示条件 | 備考 |
|----|----|----|----|----|----|
| O-01 | 表示名(取得時) | 文字列 | 表示名欄(SCR-022 §4 #3)に初期値表示 | プロフィールタブ | API-064 レスポンス `displayName`。I-01 の初期値に対応 |
| O-02 | 連絡先メール(取得時) | 文字列 | メールアドレス欄(SCR-022 §4 #4)に初期値表示 | プロフィールタブ | API-064 レスポンス `email`。読み取り専用表示(編集経路は IO-013 が扱う) |
| O-03 | 連絡先メール確認状態 | 真偽値 | 画面非表示 | 常時 | API-064 レスポンス `emailVerified`。本画面での表示制御には用いない |
| O-04 | 参加プロジェクト一覧 | 配列 | 参加プロジェクト一覧(SCR-022 §4 #11)にプロジェクト名リンク(#12)として列挙 | 参加プロジェクトタブ(0 件時は一覧を空で表示) | API-064 レスポンス `projects`(`id` / `name` / `role`)。並び順は取得順 |
| O-05 | 作成したプロジェクト数 | 数値 | 作成したプロジェクト数欄(SCR-022 §4 #15)に表示(0 件時は「未使用」) | プロフィールタブ | O-04 のうち `role` が `owner` の件数(§6 参照) |
| O-06 | 表示名(更新後) | 文字列 | 表示名欄に反映 | プロフィール保存成功時(表示名のみ変更) | API-012 レスポンス `displayName` |
| O-07 | プロフィール保存結果 | 真偽値 | 画面非表示(成否分岐に使用) | プロフィール保存実行時 | API-012 レスポンス `ok` |
| O-08 | パスワード変更結果 | 真偽値 | 画面非表示(成否分岐に使用) | パスワード変更実行時 | API-013 レスポンス `ok` |

## 6. 編集・加工仕様

入力値の正規化・出力値の整形など編集加工を定義する。

| 項目名 | 加工内容 | 加工タイミング | 備考 |
|----|----|----|----|
| 表示名(I-01) | 前後空白の除去 | 保存前(サーバー) | 文字数チェックは加工後の値で判定する |
| 作成したプロジェクト数(O-05) | 参加プロジェクト一覧(O-04)のうち `role` が `owner` の件数を集計する | 表示前(クライアント) | API-064 は件数を直接返さないため画面側で集計する。0 件時は「未使用」表示(SCR-022 §4 #15) |
| 新しいパスワード(確認)(I-03) | 保存対象としては用いず一致検証にのみ使用する | 送信前(クライアント) | API-013 リクエストには `password` と共に送信するが、保存されるのは `password` のみ |

## 7. 後続工程への引き継ぎ事項

テスト設計・DB 物理設計・詳細ロジック設計へ引き継ぐ観点を挙げる。

- 表示名(I-01)の境界値(0 / 1 / 100 / 101 文字)、パスワード(I-02・I-03)の強度パターン([FR-006](../../01_requirements/02_functional_requirement/01_account-fr.md#FR-006))と不一致パターンはテスト設計でケース化する。
- 参加プロジェクト一覧(O-04)が 0 件のとき・複数プロジェクトで立場(`owner` / `member`)が混在するときの作成したプロジェクト数(O-05)集計はテスト設計で検証する。
- パスワード変更(EVT-04)における再認証トークン(I-04)の未取得・期限切れ([SCR-034](../../02_basic_design/01_frontend/01_screens/SCR-034.md#SCR-034)経由)と、[API-013](../../02_basic_design/02_backend/03_apis/API-013.md#API-013)側の再検証との整合はテスト設計でケース化する。
- 論理項目 ↔ 物理カラム(`M_USER.name` / `M_USER.password_hash` 等)の対応は [DBP-002](../07_db_physical/DBP-002.md#DBP-002) で確定する。
