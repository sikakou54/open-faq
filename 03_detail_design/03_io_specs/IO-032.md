# <span id="IO-032"></span>IO-032: 再認証ダイアログ 入出力

> **この入出力設計書は「再認証ダイアログと再認証 API の入力/出力項目・入力チェック・編集加工」を論理項目単位で定義します。**

*種別 入出力設計 ・ ステータス ドラフト*

| 項目 | 値 |
|----|----|
| 入出力設計ID | IO-032 |
| 業務ユースケースID | [UC-009](../../01_requirements/04_business_usecases/UC-009.md#UC-009) ・ [UC-010](../../01_requirements/04_business_usecases/UC-010.md#UC-010) ・ [UC-017](../../01_requirements/04_business_usecases/UC-017.md#UC-017) |
| 関連画面 | [SCR-034](../../02_basic_design/01_frontend/01_screens/SCR-034.md#SCR-034) |

## 1. 目的

本入出力設計は、重要操作の実行前に呼出元から割込みで開かれる再認証ダイアログが、入力したパスワードで再認証 API を呼び出し、成功時は再認証トークンを呼出元へ引き継ぎ、失敗時はダイアログ内にエラーを表示して再入力を促す一連の入力/出力項目を、画面と API を突き合わせて論理項目単位で確定する。橋渡しする画面は [SCR-034](../../02_basic_design/01_frontend/01_screens/SCR-034.md#SCR-034) の「確認」押下(EVT-02)、API は [API-005](../../02_basic_design/02_backend/03_apis/API-005.md#API-005) である。

## 2. 対象範囲

本設計が対象とする画面・API・データと対象外を明示する。再認証を要求する契機の判定と、発行された再認証トークンを用いる各重要操作本体の入出力は別文書へ委ねる。

| 区分 | 対象 |
|----|----|
| 対象画面 | [SCR-034](../../02_basic_design/01_frontend/01_screens/SCR-034.md#SCR-034) 再認証ダイアログ(確認 EVT-02・キャンセル EVT-03) |
| 対象API | [API-005](../../02_basic_design/02_backend/03_apis/API-005.md#API-005) 再認証 |
| 対象データ | 現在のパスワード(入力)、再認証の成否・再認証トークン(出力) |
| 対象外 | 重要操作(メールアドレス/パスワード変更・プロジェクト削除・メンバー招待・メンバー割当解除・月次上限件数変更等)本体の入出力([IO-013](IO-013.md#IO-013) / [IO-011](IO-011.md#IO-011) / [IO-024](IO-024.md#IO-024) / [IO-023](IO-023.md#IO-023) / [IO-026](IO-026.md#IO-026) ほか)・再認証要求の契機判定・再認証有効範囲(当該操作 1 回 / 15 分以内)の充足判定ロジック([IPO-013](../04_ipo/IPO-013.md#IPO-013))・呼出元画面への遷移そのもの |

## 3. 入力項目一覧

再認証ダイアログから受け取り API へ渡す入力項目を論理項目単位で定義する。

| 項目ID | 項目名 | データ型 | 必須 | 初期値 | 入力制約 | 入力例 | 備考 |
|----|----|----|----|----|----|----|----|
| I-01 | 現在のパスワード | 文字列 | ◯ | — | 最大 128 文字([SCR-034](../../02_basic_design/01_frontend/01_screens/SCR-034.md#SCR-034) §4 #3) | — | SCR-034 §4 #3 に対応。画面には平文表示しない(マスク入力)。API リクエスト `password` |

## 4. 出力項目一覧

API レスポンス・画面へ返す出力項目を論理項目単位で定義する。成否によって出力項目が分岐する。

| 項目ID | 項目名 | データ型 | 表示形式 | 表示条件 | 備考 |
|----|----|----|----|----|----|
| O-01 | 再認証結果 | 真偽値 | 画面非表示 | 常時 | API 応答 `ok`。真のとき再認証成功としてダイアログを閉じ、偽のときは §5 のエラー表示へ分岐する |
| O-02 | 再認証トークン | 文字列 | 画面非表示 | 再認証結果が真 | 機微操作(呼出元の重要操作)に用いるトークン。API 応答 `token`。呼出元の重要操作 API のリクエストへ引き継ぐ(例: [IO-013](IO-013.md#IO-013) I-04・[IO-011](IO-011.md#IO-011) I-08) |
| O-03 | エラーメッセージ表示 | 文字列 | メッセージ表示エリア(SCR-034 §4 #4)に表示 | 再認証結果が偽 | 表示文面は SCR-034 EM-02 を参照。文面はダイアログを開いたまま表示し再入力を促す |

## 5. 入力チェック仕様

各入力項目のチェック内容・実施主体(クライアント / サーバー)・エラー時参照を定義する。文面は本ページに再掲せず ID 参照する。

| 項目名 | チェック内容 | タイミング | エラー時メッセージ | 備考 |
|----|----|----|----|----|
| 現在のパスワード(I-01) | 未入力チェック | クライアント | [SCR-034](../../02_basic_design/01_frontend/01_screens/SCR-034.md#SCR-034) EM-01 | 「確認」(EVT-02)押下時に判定([SCR-034](../../02_basic_design/01_frontend/01_screens/SCR-034.md#SCR-034) §5 #3)。違反時は API 呼び出しを行わず送信を中止 |
| 現在のパスワード(I-01) | 再照合チェック | サーバー | [SCR-034](../../02_basic_design/01_frontend/01_screens/SCR-034.md#SCR-034) EM-02 / [ERR-005](../../02_basic_design/05_errors/ERR-005.md#ERR-005) | 不一致時は `401 INVALID_PASSWORD`([API-005](../../02_basic_design/02_backend/03_apis/API-005.md#API-005) P-02)。ダイアログを開いたまま再入力を促す(呼出元の重要操作は実行しない) |

## 6. 編集・加工仕様

入力値の正規化・出力値の整形など編集加工を定義する。

| 項目名 | 加工内容 | 加工タイミング | 備考 |
|----|----|----|----|
| 再認証トークン(O-02) | 画面には表示せず内部でのみ保持し呼出元の重要操作 API 呼び出し時に引き継ぐ | 再認証成功後(クライアント) | ダイアログを閉じて呼出元の重要操作を続行する際に O-02 を伴って呼出先 API を呼び出す |

## 7. 後続工程への引き継ぎ事項

テスト設計・DB 物理設計・詳細ロジック設計へ引き継ぐ観点を挙げる。再認証要求の契機判定・有効範囲の充足判定・トークンの保持先の詳細は別文書へ委ねる。

- 現在のパスワード(I-01)の境界値(0 / 1 / 128 / 129 文字)はテスト設計でパターン化する。
- 再認証失敗(EM-02・[ERR-005](../../02_basic_design/05_errors/ERR-005.md#ERR-005))時にダイアログを維持し再入力を促す挙動と、キャンセル(EVT-03)時に呼出元の重要操作を中断する挙動はテスト設計でケース化する。
- 再認証要求の契機(セッション失効・重要操作判定・有効範囲 15 分以内)は [IPO-013](../04_ipo/IPO-013.md#IPO-013)、再認証トークンの保持先・検証方式は同文書の課題候補を参照する。
- 発行された再認証トークン(O-02)を用いる各重要操作側の入力項目・有効性チェックは、呼出元ごとの入出力設計書([IO-013](IO-013.md#IO-013) / [IO-011](IO-011.md#IO-011) / [IO-024](IO-024.md#IO-024) / [IO-023](IO-023.md#IO-023) / [IO-026](IO-026.md#IO-026) ほか)へ引き継ぐ。
