## 1. 目的

本入出力設計は、オーナーが自分が作成した全プロジェクトの請求状況を確認し、支払方法の設定画面へ導線する一連の入力/出力項目を、画面と API を突き合わせて論理項目単位で確定する。橋渡しする画面は [SCR-028](../../02_basic_design/01_frontend/01_screens/SCR-028.md#SCR-028) の初期表示(EVT-01)・支払い方法の設定への導線(EVT-02・EVT-06)、API は [API-043](../../02_basic_design/02_backend/03_apis/API-043.md#API-043)・[API-044](../../02_basic_design/02_backend/03_apis/API-044.md#API-044)である。

## 2. 対象範囲

本設計が対象とする画面・API・データと対象外を明示する。支払方法の登録・変更操作自体の画面項目・入力チェックは [SCR-029](../../02_basic_design/01_frontend/01_screens/SCR-029.md#SCR-029) 側に属するため、本設計では請求画面(SCR-028)からの導線と、突き合わせ対象となる API 入出力項目のみを扱う。

| 区分 | 対象 |
|----|----|
| 対象画面 | [SCR-028](../../02_basic_design/01_frontend/01_screens/SCR-028.md#SCR-028) 請求管理(初期表示 EVT-01・支払い方法の設定への導線 EVT-02 / EVT-06・領収書取得 EVT-03) |
| 対象API | [API-043](../../02_basic_design/02_backend/03_apis/API-043.md#API-043) 請求サマリ・[API-044](../../02_basic_design/02_backend/03_apis/API-044.md#API-044) 請求書一覧 |
| 対象データ | 当月請求見込み・次回請求日・請求状態・支払方法登録状態・請求履歴・領収書 PDF の各出力項目 |
| 対象外 | 支払方法の取得・登録・更新操作([SCR-029](../../02_basic_design/01_frontend/01_screens/SCR-029.md#SCR-029) EVT-01・EVT-02・[API-045](../../02_basic_design/02_backend/03_apis/API-045.md#API-045) GET / PUT・再認証 [API-005](../../02_basic_design/02_backend/03_apis/API-005.md#API-005))・退会手続き([SCR-019](../../02_basic_design/01_frontend/01_screens/SCR-019.md#SCR-019))・利用量と上限の詳細([SCR-026](../../02_basic_design/01_frontend/01_screens/SCR-026.md#SCR-026)) |

## 3. 入力項目一覧

請求管理画面から API へ渡す入力項目を論理項目単位で定義する。本画面は照会・操作起点が中心のため、入力フォーム項目は持たず、請求書一覧の取得件数指定のみを扱う。

| 項目ID | 項目名 | データ型 | 必須 | 初期値 | 入力制約 | 入力例 | 備考 |
|----|----|----|----|----|----|----|----|
| I-01 | 請求履歴取得件数 | 数値 | — | 6 | 正の整数 | 6 | 画面には表示しない。API-044 リクエスト `limit`。初期表示時に画面側が固定値で指定する |
| I-02 | 領収書対象請求識別子 | 文字列 | ◯ | — | 請求履歴一覧に存在する請求識別子 | inv_... | SCR-028 §4 #7 の対象行選択に対応。API-044 レスポンス `items[].id` を再利用 |

## 4. 出力項目一覧

API レスポンス・画面へ返す出力項目を論理項目単位で定義する。当月請求見込み・支払方法・請求履歴でレスポンス元 API が異なる。

| 項目ID | 項目名 | データ型 | 表示形式 | 表示条件 | 備考 |
|----|----|----|----|----|----|
| O-01 | 当月請求見込み合計 | 数値 | 金額表示(通貨コード付き) | 常時 | SCR-028 §4 #3 に対応。API-043 `estimatedTotal` / `currency` |
| O-02 | 次回請求日 | 日付 | 日付表示 | 常時 | SCR-028 §4 #3 に対応。API-043 `nextBillingDate` |
| O-03 | 請求状態 | 列挙値 | バナー・案内文の出し分けに使用(画面非表示) | 常時 | `normal` / `payment_method_required` / `payment_failed`。取りうる値と意味は §列挙値。API-043 `billingStatus` |
| O-04 | プロジェクト別請求内訳 | 配列 | 内訳表示(プロジェクト名 / 質問数課金額 / FAQ 件数課金額 / 小計) | 常時(自分が作成したプロジェクトが 1 件以上) | SCR-028 §4 #3 に対応。API-043 `projects[]` |
| O-05 | 支払方法登録状態 | 真偽値 | バナー(#1)・支払方法表示(#4)の出し分けに使用 | 常時 | 未登録時は #4 に未登録を表示し #1・#2 を表示する。API-043 `paymentMethod.registered` |
| O-06 | 支払方法(ブランド・下 4 桁) | 文字列 | 支払方法表示(SCR-028 §4 #4) | 支払方法登録済み | API-043 `paymentMethod.brand` / `paymentMethod.last4` |
| O-07 | 請求履歴一覧 | 配列 | 表表示(SCR-028 §4 #6:請求日 / 内容 / 金額 / 状態 / 領収書) | 常時(請求履歴が 1 件以上) | API-044 `items[]`(`date` / `description` / `amount` / `currency` / `status`) |
| O-08 | 請求行状態 | 列挙値 | 請求履歴の状態列(SCR-028 §4 #6 データパターン) | 常時 | `paid`(支払済)/ `failed`(失敗)/ `draft`(下書き)。取りうる値と意味は §列挙値。API-044 `items[].status` |
| O-09 | 領収書 PDF 署名 URL | 文字列 | リンク(SCR-028 §4 #7) | 請求行が存在し領収書取得可能 | 有効期限は [システム仕様書 §4](../../02_basic_design/07_system-spec.md#4-データ保持期間削除猶予)。取得失敗時は EM-01 を表示。API-044 `items[].pdfUrl` |

## 5. 入力チェック仕様

各入力項目のチェック内容・実施主体(クライアント / サーバー)・エラー時参照を定義する。本画面は照会・操作起点が中心のため、サーバー側の認可・存在確認が中心となる。

| 項目名 | チェック内容 | タイミング | エラー時メッセージ | 備考 |
|----|----|----|----|----|
| 請求履歴取得件数(I-01) | 型・範囲チェック | サーバー | [ERR-001](../../02_basic_design/05_errors/ERR-001.md#ERR-001) | 画面固定値のため通常発生しない |
| 領収書対象請求識別子(I-02) | 存在チェック(自分の課金アカウントに紐づく請求か) | サーバー | SCR-028 EM-01 | 存在しない、または他アカウントの請求識別子のときは取得失敗として扱う |
| 請求情報全般(閲覧権限) | 認証・認可(自分が作成したプロジェクトのオーナーであること) | サーバー | [ERR-001](../../02_basic_design/05_errors/ERR-001.md#ERR-001) | API-043 P-01 / API-044 P-01。退会済み(`withdrawn`)は閲覧のみ許可([UC-081](../../01_requirements/04_business_usecases/UC-081.md#UC-081)) |

## 6. 編集・加工仕様

入力値の正規化・出力値の整形など編集加工を定義する。

| 項目名 | 加工内容 | 加工タイミング | 備考 |
|----|----|----|----|
| 請求行内容(O-07) | プラン名と請求年月から表示用文字列を導出 | レスポンス生成時(サーバー) | API-044 `items[].description`。物理項目からの導出であり本ページでは論理項目のみを扱う |
| 当月請求見込み合計(O-01) | 通貨コードに応じた金額表示への整形 | 表示時(クライアント) | API-043 `currency` に基づく |
| 請求状態(O-03) / 支払方法登録状態(O-05) | 状態値に基づくバナー(#1・#2)・案内文の表示要否判定 | 表示時(クライアント) | 退会済み時はバナー(#1・#2)・支払方法変更導線(#5)・退会導線(#8)を表示しない(SCR-028 EVT-01) |

## 列挙値

出力の請求状態(O-03)と請求行状態(O-08)の取りうる値と意味を示す。値の全集合の正本は [API-043](../../02_basic_design/02_backend/03_apis/API-043.md#API-043)・[API-044](../../02_basic_design/02_backend/03_apis/API-044.md#API-044) `## レスポンス`。

| 項目 | 取りうる値 | 意味 |
|----|----|----|
| 請求状態(O-03) | `normal` | 正常。支払方法登録済みで支払い失敗もない |
| 請求状態(O-03) | `payment_method_required` | 支払方法未登録([課金・請求設計書 §4](../../02_basic_design/05_billing-design.md#4-支払方法ゲート)) |
| 請求状態(O-03) | `payment_failed` | 直近の決済が失敗([課金・請求設計書 §5.1](../../02_basic_design/05_billing-design.md#51-決済失敗からサスペンションへ)) |
| 請求行状態(O-08) | `paid` | 支払済 |
| 請求行状態(O-08) | `failed` | 失敗 |
| 請求行状態(O-08) | `draft` | 下書き(確定前) |

> [!NOTE]
> **請求状態(O-03)と課金アカウント状態(`M_BILLING_ACCOUNT.status`)は別軸。** 請求状態は当月の請求見込みに対する支払い健全性を示す表示用の値であり、課金アカウントの状態遷移(`active` / `suspended` / `withdrawn` / `deleted`)は [状態モデル §2](../../02_basic_design/08_state-model.md#2-課金アカウント状態) を正本とする。退会済み(`withdrawn`)時は請求情報を閲覧専用で表示し、支払方法の設定導線(SCR-028 §4 #1・#2・#5)と退会導線(#8)を表示しない(SCR-028 EVT-01)。

## 7. 後続工程への引き継ぎ事項

テスト設計・DB 物理設計・詳細ロジック設計へ引き継ぐ観点を挙げる。支払方法の取得・登録・更新操作自体の入出力設計は別文書([SCR-029](../../02_basic_design/01_frontend/01_screens/SCR-029.md#SCR-029) 系の IO)へ委ねる。

- 請求状態(O-03)・支払方法登録状態(O-05)の組み合わせによるバナー(#1・#2)・案内文の表示分岐、退会済み時の閲覧専用表示への切り替えはテスト設計でケース化する。
- 請求履歴一覧(O-07)が 0 件のときの表示、領収書 PDF 署名 URL(O-09)の期限切れ時の再取得挙動はテスト設計でパターン化する。
- 支払方法の取得・登録・更新([API-045](../../02_basic_design/02_backend/03_apis/API-045.md#API-045) GET / PUT)・再認証([API-005](../../02_basic_design/02_backend/03_apis/API-005.md#API-005))の入出力項目・入力チェックは、SCR-029 を対象画面とする別の入出力設計書で確定する。
- 論理項目 ↔ 物理カラムの対応は DB 物理設計(DBP)で確定する。
