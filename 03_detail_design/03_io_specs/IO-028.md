# <span id="IO-028"></span>IO-028: 質問ログ 入出力

> **この入出力設計書は「質問ログ画面と質問ログ検索 API の入力/出力項目・入力チェック・編集加工」を論理項目単位で定義します。**

*種別 入出力設計 ・ ステータス ドラフト*

| 項目 | 値 |
|----|----|
| 入出力設計ID | IO-028 |
| 業務ユースケースID | [UC-076](../../01_requirements/04_business_usecases/UC-076.md#UC-076) |
| 関連画面 | [SCR-032](../../02_basic_design/01_frontend/01_screens/SCR-032.md#SCR-032) |
| 関連 API | [API-032](../../02_basic_design/02_backend/03_apis/API-032.md#API-032) |

## 1. 目的

本入出力設計は、オーナー / メンバーが現在開いているプロジェクトの質問ログをキーワード・対象期間・回答状態で絞り込んで検索・参照する一連の入力/出力項目を、画面と API を突き合わせて論理項目単位で確定する。橋渡しする画面は [SCR-032](../../02_basic_design/01_frontend/01_screens/SCR-032.md#SCR-032) の初期表示・各検索操作・追加読み込み(EVT-01〜EVT-06)、API は [API-032](../../02_basic_design/02_backend/03_apis/API-032.md#API-032) である。

## 2. 対象範囲

本設計が対象とする画面・API・データと対象外を明示する。FAQ 全文検索([API-031](../../02_basic_design/02_backend/03_apis/API-031.md#API-031))は別画面([SCR-008](../../02_basic_design/01_frontend/01_screens/SCR-008.md#SCR-008))が対象のため本設計の対象外とする。

| 区分 | 対象 |
|----|----|
| 対象画面 | [SCR-032](../../02_basic_design/01_frontend/01_screens/SCR-032.md#SCR-032) 質問ログ(初期表示・キーワード / 期間 / 回答状態の絞込・行選択・追加読み込み EVT-01〜EVT-06) |
| 対象API | [API-032](../../02_basic_design/02_backend/03_apis/API-032.md#API-032) 質問ログ検索 |
| 対象データ | 検索条件(キーワード・対象期間・回答状態)、質問ログ一覧(質問本文・回答有無・信頼度・問い合わせ ID・作成日時) |
| 対象外 | FAQ 全文検索([API-031](../../02_basic_design/02_backend/03_apis/API-031.md#API-031)・[SCR-008](../../02_basic_design/01_frontend/01_screens/SCR-008.md#SCR-008))・未解決質問一覧 / 詳細・状況切替([API-034](../../02_basic_design/02_backend/03_apis/API-034.md#API-034) / [API-035](../../02_basic_design/02_backend/03_apis/API-035.md#API-035)、[IO-017](IO-017.md#IO-017) / [IO-018](IO-018.md#IO-018)で扱う)・質問ログの記録(ウィジェット質問送信、[IO-001](IO-001.md#IO-001)で扱う) |

## 3. 入力項目一覧

質問ログ画面から受け取り API へ渡す入力項目を論理項目単位で定義する。文字数上限などの具体値は基本設計へ ID 参照する。

| 項目ID | 項目名 | データ型 | 必須 | 初期値 | 入力制約 | 入力例 | 備考 |
|----|----|----|----|----|----|----|----|
| I-01 | キーワード検索 | 文字列 | — | — | 最大 256 文字([SCR-032](../../02_basic_design/01_frontend/01_screens/SCR-032.md#SCR-032) §4 #1) | 返品 | SCR-032 §4 #1 に対応。API リクエスト `keyword` |
| I-02 | 対象期間(開始) | 文字列(日時) | — | — | ISO 8601([API-032](../../02_basic_design/02_backend/03_apis/API-032.md#API-032) `## リクエスト`) | 2026-06-01T00:00:00Z | SCR-032 §4 #2 に対応。API リクエスト `from` |
| I-03 | 対象期間(終了) | 文字列(日時) | — | — | ISO 8601([API-032](../../02_basic_design/02_backend/03_apis/API-032.md#API-032) `## リクエスト`) | 2026-06-23T23:59:59Z | SCR-032 §4 #3 に対応。API リクエスト `to` |
| I-04 | 回答状態フィルタ | 列挙値 | — | すべて | `すべて` / `回答あり` / `該当なし`(§列挙値) | 回答あり | SCR-032 §4 #4 に対応。API リクエスト `answerType`。「すべて」は無指定送信 |
| I-05 | 対象プロジェクト | 文字列 | ◯ | — | 有効なプロジェクト識別子 | — | 現在開いているプロジェクト。API パスパラメータ `id` |
| I-06 | 利用者セッション | 文字列 | ◯ | — | 有効なセッション(Cookie) | — | 認証情報。画面には表示しない([API-032](../../02_basic_design/02_backend/03_apis/API-032.md#API-032) 認証ヘッダ) |
| I-07 | 追加読み込みカーソル | 文字列 | — | — | 直前応答の次ページ取得用カーソル([API-032](../../02_basic_design/02_backend/03_apis/API-032.md#API-032) `## リクエスト`) | — | SCR-032 §4 #11「もっと見る」押下時のみ送信。画面には表示しない。API リクエスト `cursor` |

## 4. 出力項目一覧

API レスポンス・画面へ返す出力項目を論理項目単位で定義する。

| 項目ID | 項目名 | データ型 | 表示形式 | 表示条件 | 備考 |
|----|----|----|----|----|----|
| O-01 | 件数 | 数値 | 件数表示(SCR-032 §4 #5) | 常時 | 検索結果の総件数。API `items` の件数から画面側で算出 |
| O-02 | 質問本文 | 文字列 | 一覧行に表示(SCR-032 §4 #6) | 常時 | 一覧の行選択(EVT-05)で詳細表示にも用いる。API `items[].question` |
| O-03 | 回答有無バッジ | 列挙値 | 一覧行にバッジ表示(SCR-032 §4 #7) | 常時 | `answered`(回答あり・緑)/ `unanswered`(該当なし・赤)。取りうる値と意味は §列挙値。API `items[].answerType` |
| O-04 | 信頼度 | 数値 | 一覧行に表示(SCR-032 §4 #8) | 常時 | 回答の信頼度スコア(0〜1)。しきい値は [RULE-012](../../01_requirements/01_business_requirement/08_rule.md#RULE-012) / [システム仕様書 §1](../../02_basic_design/07_system-spec.md#1-aiしきい値)。API `items[].confidence` |
| O-05 | 問い合わせ ID | 文字列 | 一覧行に表示(SCR-032 §4 #9) | 関連する未解決質問がある行のみ | 未解決質問と関連付けがない行は非表示。API `items[].inquiryId`(無い場合は `null`) |
| O-06 | 作成日時 | 文字列(日時) | 一覧行に表示(SCR-032 §4 #10) | 常時 | 質問の作成日時。API `items[].createdAt` |
| O-07 | 次ページカーソル | 文字列 | 画面非表示 | 次ページがある場合のみ値を保持 | 「もっと見る」(SCR-032 §4 #11)の表示可否・追加読み込み(I-07)に用いる。末尾は `null`。API `nextCursor` |
| O-08 | 空状態 | — | 空状態メッセージ表示(SCR-032 §4 #12) | 検索結果 0 件時 | O-01 が 0 のとき画面側で判定して表示 |

## 5. 入力チェック仕様

各入力項目のチェック内容・実施主体(クライアント / サーバー)・エラー時参照を定義する。[SCR-032](../../02_basic_design/01_frontend/01_screens/SCR-032.md#SCR-032) §5 のとおり画面上の入力検証は無く、対象プロジェクトの境界判定・認証確認はサーバーのみで行う。

| 項目名 | チェック内容 | タイミング | エラー時メッセージ | 備考 |
|----|----|----|----|----|
| 対象プロジェクト(I-05) | 当該プロジェクトへの割当チェック(境界判定) | サーバー | [エラー設計](../../02_basic_design/05_errors/index.md) の標準エラー体系に従う([API-032](../../02_basic_design/02_backend/03_apis/API-032.md#API-032) に固有エラーコードなし) | 割当なし・部外者は境界判定の対象([API-032](../../02_basic_design/02_backend/03_apis/API-032.md#API-032) P-01) |
| 利用者セッション(I-06) | セッション有効性チェック | サーバー | [エラー設計](../../02_basic_design/05_errors/index.md) の標準エラー体系に従う([API-032](../../02_basic_design/02_backend/03_apis/API-032.md#API-032) に固有エラーコードなし) | [API-032](../../02_basic_design/02_backend/03_apis/API-032.md#API-032) P-01 |
| 対象期間(I-02・I-03) | 形式チェック(ISO 8601) | サーバー | [エラー設計](../../02_basic_design/05_errors/index.md) の標準エラー体系に従う([API-032](../../02_basic_design/02_backend/03_apis/API-032.md#API-032) に固有エラーコードなし) | 未入力は期間指定なしとして扱う |
| 回答状態フィルタ(I-04) | 列挙値チェック(§列挙値以外を拒否) | サーバー | [エラー設計](../../02_basic_design/05_errors/index.md) の標準エラー体系に従う([API-032](../../02_basic_design/02_backend/03_apis/API-032.md#API-032) に固有エラーコードなし) | 「すべて」選択時は API へ無指定で送信 |

## 6. 編集・加工仕様

入力値の正規化・出力値の整形など編集加工を定義する。

| 項目名 | 加工内容 | 加工タイミング | 備考 |
|----|----|----|----|
| キーワード検索(I-01) | 前後空白の除去 | 検索実行前(クライアント) | — |
| 回答状態フィルタ(I-04) | 「すべて」選択時は API リクエストの `answerType` を送信しない | 検索実行前(クライアント) | [API-032](../../02_basic_design/02_backend/03_apis/API-032.md#API-032) `## リクエスト` の無指定=絞り込みなしと整合 |
| 問い合わせ ID(O-05) | 未関連時は表示欄を非表示にする | 一覧描画時(クライアント) | API `inquiryId` が `null` のとき非表示([SCR-032](../../02_basic_design/01_frontend/01_screens/SCR-032.md#SCR-032) §4 #9) |
| 一覧(O-02〜O-06) | 追加読み込み(EVT-06)時は既存一覧の末尾へ追加する | 応答受信時(クライアント) | 置き換えではなく追記([SCR-032](../../02_basic_design/01_frontend/01_screens/SCR-032.md#SCR-032) §7 EVT-06) |

## 列挙値

出力の回答有無バッジ(O-03)と入力の回答状態フィルタ(I-04)の取りうる値と意味を示す。値の全集合の正本は [API-032](../../02_basic_design/02_backend/03_apis/API-032.md#API-032) `## リクエスト` / `## レスポンス`。

| 項目 | 取りうる値 | 意味 |
|----|----|----|
| 回答状態フィルタ(I-04) | すべて | 回答有無で絞り込まない(既定値。API へは無指定で送信) |
| 回答状態フィルタ(I-04) | 回答あり | `answered` の質問ログのみに絞り込む |
| 回答状態フィルタ(I-04) | 該当なし | `unanswered` の質問ログのみに絞り込む |
| 回答有無バッジ(O-03) | `answered` | 回答あり(緑バッジ) |
| 回答有無バッジ(O-03) | `unanswered` | 該当なし(赤バッジ) |

## 7. 後続工程への引き継ぎ事項

テスト設計・DB 物理設計・詳細ロジック設計へ引き継ぐ観点を挙げる。

- キーワード検索(I-01)の境界値(0 / 256 / 257 文字)、対象期間(I-02・I-03)の未指定 / 開始のみ / 終了のみ / 両方指定の組み合わせはテスト設計でパターン化する。
- 問い合わせ ID(O-05)の表示 / 非表示分岐(関連する未解決質問の有無)、空状態(O-08)の表示分岐(検索結果 0 件)はテスト設計でケース化する。
- 質問ログの検索対象範囲(テナント境界)・全文検索方式は [API-032](../../02_basic_design/02_backend/03_apis/API-032.md#API-032) を正本とし、論理項目 ↔ 物理カラムの対応は [DBP-001](../07_db_physical/DBP-001.md#DBP-001) で確定する。
