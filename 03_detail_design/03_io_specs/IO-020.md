# <span id="IO-020"></span>IO-020: FAQ編集 入出力

> **この入出力設計書は「FAQ 編集画面と FAQ 作成・更新・削除 API / FAQ 個別取得 API の入力/出力項目・入力チェック・編集加工」を論理項目単位で定義します。**

*種別 入出力設計 ・ ステータス ドラフト*

| 項目 | 値 |
|----|----|
| 入出力設計ID | IO-020 |
| 業務ユースケースID | [UC-024](../../01_requirements/04_business_usecases/UC-024.md#UC-024) ・ [UC-025](../../01_requirements/04_business_usecases/UC-025.md#UC-025) |
| 関連画面 | [SCR-009](../../02_basic_design/01_frontend/01_screens/SCR-009.md#SCR-009) |
| 関連 API | [API-026](../../02_basic_design/02_backend/03_apis/API-026.md#API-026) ・ [API-033](../../02_basic_design/02_backend/03_apis/API-033.md#API-033) |

## 1. 目的

本入出力設計は、オーナー・メンバーが FAQ を新規作成または編集し、自動保存・保存・削除を行う一連の入力/出力項目を、画面と API を突き合わせて論理項目単位で確定する。橋渡しする画面は [SCR-009](../../02_basic_design/01_frontend/01_screens/SCR-009.md#SCR-009) の初期表示(EVT-01)・自動保存(EVT-03)・保存(EVT-04)・削除(EVT-06)、API は [API-026](../../02_basic_design/02_backend/03_apis/API-026.md#API-026)(作成・更新・削除)と [API-033](../../02_basic_design/02_backend/03_apis/API-033.md#API-033)(個別取得)である。

## 2. 対象範囲

本設計が対象とする画面・API・データと対象外を明示する。楽観ロック衝突・件数上限超過時の状態遷移の正本、および登録元未解決質問の対応関係の詳細は別文書へ委ねる。

| 区分 | 対象 |
|----|----|
| 対象画面 | [SCR-009](../../02_basic_design/01_frontend/01_screens/SCR-009.md#SCR-009) FAQ 編集(初期表示 EVT-01・自動保存 EVT-03・保存 EVT-04・削除 EVT-06) |
| 対象API | [API-026](../../02_basic_design/02_backend/03_apis/API-026.md#API-026) FAQ 作成・更新・削除 ・ [API-033](../../02_basic_design/02_backend/03_apis/API-033.md#API-033) FAQ 個別取得 |
| 対象データ | 質問・回答・カテゴリ・状態・楽観ロックバージョン、登録元未解決質問の有無 |
| 対象外 | FAQ 一覧([SCR-008](../../02_basic_design/01_frontend/01_screens/SCR-008.md#SCR-008))・一括状態変更(API-027)・未解決質問詳細([SCR-007](../../02_basic_design/01_frontend/01_screens/SCR-007.md#SCR-007))・FAQ 状態遷移の判定条件([STS-005](../01_state_transitions/STS-005.md#STS-005)) |

## 3. 入力項目一覧

FAQ 編集画面から受け取り API へ渡す入力項目を論理項目単位で定義する。文字数上限などの具体値は基本設計へ ID 参照する。

| 項目ID | 項目名 | データ型 | 必須 | 初期値 | 入力制約 | 入力例 | 備考 |
|----|----|----|----|----|----|----|----|
| I-01 | 質問 | 文字列 | △ | 既存編集時は取得値・新規作成時は空(未解決質問起点は質問文を反映) | 1〜500 文字([SCR-009](../../02_basic_design/01_frontend/01_screens/SCR-009.md#SCR-009) §4 #3 / [API-026](../../02_basic_design/02_backend/03_apis/API-026.md#API-026) `question`)。公開保存時は必須・下書き保存時は任意([API-026](../../02_basic_design/02_backend/03_apis/API-026.md#API-026) P-02) | 返品の手続きは? | SCR-009 §4 #3 に対応。API リクエスト `question` |
| I-02 | 回答 | 文字列 | △ | 既存編集時は取得値・新規作成時は空 | 1〜5,000 文字([SCR-009](../../02_basic_design/01_frontend/01_screens/SCR-009.md#SCR-009) §4 #4 / [API-026](../../02_basic_design/02_backend/03_apis/API-026.md#API-026) `answer`)。公開保存時は必須・下書き保存時は任意([API-026](../../02_basic_design/02_backend/03_apis/API-026.md#API-026) P-02) | 商品到着後 7 日以内に... | SCR-009 §4 #4 に対応。API リクエスト `answer` |
| I-03 | カテゴリ | 文字列 | — | 既存編集時は取得値・新規作成時は未選択 | 0〜100 文字([SCR-009](../../02_basic_design/01_frontend/01_screens/SCR-009.md#SCR-009) §4 #5 / [API-026](../../02_basic_design/02_backend/03_apis/API-026.md#API-026) `category`) | 返品・返金 | SCR-009 §4 #5 に対応。既存カテゴリのサジェスト + 自由入力。API リクエスト `category` |
| I-04 | 状態 | 列挙値 | ◯ | 新規作成時は下書き・既存編集時は取得値 | `draft` / `published` / `hidden`([SCR-009](../../02_basic_design/01_frontend/01_screens/SCR-009.md#SCR-009) §4 #6・データパターン表 / [API-026](../../02_basic_design/02_backend/03_apis/API-026.md#API-026) `status`) | `published` | SCR-009 §4 #6 に対応。状態の意味は [状態モデル §4.1](../../02_basic_design/08_state-model.md#41-faq-状態)。API リクエスト `status` |
| I-05 | 楽観ロックバージョン | 数値 | △ | 既存編集時は取得値 | 更新・削除時は必須([API-026](../../02_basic_design/02_backend/03_apis/API-026.md#API-026) `version`) | 3 | 画面には表示しない。新規作成(POST)時は不要。API リクエスト `version` |
| I-06 | 対象 FAQ 識別子 | 文字列 | △ | 既存編集時は流入元(FAQ 一覧・未解決質問詳細)から確定 | 有効な FAQ 識別子([API-026](../../02_basic_design/02_backend/03_apis/API-026.md#API-026) / [API-033](../../02_basic_design/02_backend/03_apis/API-033.md#API-033) Path Parameters `id`) | — | 画面には表示しない。新規作成時は不要 |

## 4. 出力項目一覧

API レスポンス・画面へ返す出力項目を論理項目単位で定義する。

| 項目ID | 項目名 | データ型 | 表示形式 | 表示条件 | 備考 |
|----|----|----|----|----|----|
| O-01 | FAQ 識別子 | 文字列 | 画面非表示 | 常時 | 作成 / 更新した FAQ の識別子。以降の自動保存・保存・削除の対象特定に用いる。API `id` |
| O-02 | 質問 | 文字列 | #3 に反映 | 既存編集時(初期表示) | [API-033](../../02_basic_design/02_backend/03_apis/API-033.md#API-033) `question`。SCR-009 §4 #3 に対応 |
| O-03 | 回答 | 文字列 | #4 に反映 | 既存編集時(初期表示) | [API-033](../../02_basic_design/02_backend/03_apis/API-033.md#API-033) `answer`。SCR-009 §4 #4 に対応 |
| O-04 | カテゴリ | 文字列 | #5 に反映 | 既存編集時(初期表示) | [API-033](../../02_basic_design/02_backend/03_apis/API-033.md#API-033) `category`。SCR-009 §4 #5 に対応 |
| O-05 | 状態 | 列挙値 | #6 に反映 | 既存編集時(初期表示)・保存後 | `draft` / `published` / `hidden`。SCR-009 §4 #6 に対応。API `status` |
| O-06 | 楽観ロックバージョン | 数値 | 画面非表示 | 常時 | 次回更新 / 削除で送信する版番号(I-05)を更新する。API `version` |

## 6. 編集・加工仕様

入力値の正規化・出力値の整形など編集加工を定義する。最大長判定は加工後の値に対して行う。

| 項目名 | 加工内容 | 加工タイミング | 備考 |
|----|----|----|----|
| 質問(I-01) | 前後空白の除去 | 検証前(サーバー) | 未入力・最大長チェックは加工後の値で判定する |
| 回答(I-02) | 前後空白の除去 | 検証前(サーバー) | 未入力・最大長チェックは加工後の値で判定する |
| カテゴリ(I-03) | 前後空白の除去 | 検証前(サーバー) | 最大長チェックは加工後の値で判定する |
| 公開日時(O-07) | `published` 化時のみ記録し、未公開は `null` を返す | 応答生成時(サーバー) | [API-026](../../02_basic_design/02_backend/03_apis/API-026.md#API-026) P-05 |
| 自動保存結果(O-08) | API-026 のレスポンス可否を成功 / 失敗の 2 値へ変換 | 応答描画時(クライアント) | 版衝突時は失敗ではなく楽観ロック衝突として扱う([SCR-009](../../02_basic_design/01_frontend/01_screens/SCR-009.md#SCR-009) §7 EVT-03) |

## 7. 後続工程への引き継ぎ事項

テスト設計・DB 物理設計・詳細ロジック設計へ引き継ぐ観点を挙げる。状態遷移の判定条件・実行可能ロールの詳細は別文書へ委ねる。

- 質問(I-01)・回答(I-02)・カテゴリ(I-03)の境界値(0 / 1 / 上限 / 上限+1 文字)、前後空白除去後の文字数判定、下書き保存時の未入力許容と公開保存時の必須化の切替はテスト設計でパターン化する。
- 状態(I-04)の遷移可否・ガード条件・実行可能ロール・楽観ロック不一致時のエラー確定は [STS-005](../01_state_transitions/STS-005.md#STS-005) を参照する。
- 楽観ロックバージョン(I-05)不一致時の復旧ダイアログ(最新読込 / 編集継続)の選択結果反映と、自動保存(EVT-03)・保存(EVT-04)の再送時の冪等性の実装方針はテスト設計でケース化する。
- 論理項目 ↔ 物理カラムの対応は [DBP-007](../07_db_physical/DBP-007.md#DBP-007) で確定する。
