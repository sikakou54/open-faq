## <span id="1-移管方針"></span>1. 移管方針

個別チャット機能は MVP から除外し、Post-MVP の開発候補として本書へ移管する。MVP では AI が回答できなかった質問を管理用の問い合わせ ID 付きで `T_INQUIRIES` に登録し、ウィジェットへは確認済みプロジェクト連絡先メールを表示するところまでを提供する。問い合わせ ID はウィジェットに表示しない。

MVP では次の機能を実装しない。

- エンドユーザーと管理者ユーザー間のメッセージ送受信
- 個別チャット部屋・チャット ID の作成
- チャットへの担当者割当・デフォルト担当者設定
- エンドユーザー再入室トークン・再入室画面
- チャット投稿通知
- チャット部屋数の利用量計測・課金
- `chat_rooms` / `chat_messages` / チャット用 `end_users` の本番マイグレーション

## <span id="4-将来-apiデータ"></span>4. 将来 API・データ

| 区分 | 候補 |
|----|----|
| ウィジェット API | `POST /widget/v1/chat-rooms`, `POST /widget/v1/chat-rooms/{id}/messages`, `GET /widget/v1/reentry/{token}` |
| 管理 API | `GET /chat-rooms`, `GET/POST /chat-rooms/{id}/messages`, `PATCH /chat-rooms/{id}` |
| テーブル | `end_users`, `chat_rooms`, `chat_messages`, `T_ACCESS_TOKENS(purpose='reentry')` |
| 通知 | `NOTIF-CHAT_REPLY_TO_STAFF` |
| 課金 | 個別チャット部屋数の計測・無料枠・超過単価は導入時に再決定 |

これらは予約仕様であり、MVP の DDL・API ルーティング・画面ルーティング・課金計算には含めない。

## <span id="5-導入判断条件"></span>5. 導入判断条件

| 判断項目 | 導入条件 |
|----|----|
| 顧客需要 | 未解決質問への有人対応要望と外部ヘルプデスク連携要望を定量化できること |
| 運用体制 | 応答時間、担当割当、営業時間外対応、通報・削除手順を定義できること |
| セキュリティ | 本文暗号化、再入室トークン、権限境界、レート制限の設計レビュー完了 |
| コスト | 保存容量、通知、同時接続、有人対応コストを含む価格モデルが承認されること |
| 法務 | チャット本文の利用目的・保持期間・削除方針が規約とプライバシーポリシーへ反映されること |

## <span id="6-mvp-との境界"></span>6. MVP との境界

MVP の未解決時フローは次のとおりとする。

1.  質問ログを記録する。
2.  `T_INQUIRIES` に未解決質問を登録し、問い合わせ ID を発行する。
3.  ウィジェットに「回答を見つけられなかった」旨を表示し、管理用の問い合わせ ID は表示しない。
4.  確認済みプロジェクト連絡先メールがある場合は問い合わせ先として表示する。
5.  FAQ 質問入力は継続可能とし、有人チャットへの切替は行わない。
