## <span id="3-今回対象外との関係"></span>3. 今回対象外との関係

| 対象外内容 | 理由 | 将来対応候補 |
|----|----|----|
| 個別チャット機能全体 | MVPは未解決質問の管理用問い合わせID発行と連絡先案内までとし、問い合わせIDはウィジェットに表示せず、部屋・メッセージ・再入室・担当者・通知・課金データを持たない | FUT-06 |
| 利用者側 admin / operator MFA | MVP ではパスワード再入力、ロックアウト、Turnstile、操作通知、監査ログでリスク受容 | FUT-01 |
| SSO / ソーシャルログイン / パスキー | MVP は自前アカウント + パスワード + メール確認のみ | FUT-01 |
| Vectorize 連携(意味検索) | MVP は SQLite ベース全文検索のみ | FUT-02 |
| SSE / ストリーミング応答 | MVP は AI 回答をバッチ応答 | FUT-02 |
| PII 検出 NER 第 2 層 | MVP は第 1 層(正規表現)のみ | FUT-02 |
| 多言語 UI | MVP は日本語のみ | FUT-03 |
| マルチリージョン | MVP は日本リージョン(apac)のみ | FUT-03 |
| 管理者単位タイムゾーン上書き | MVP は表示 TZ JST 固定、データ層 UTC | FUT-03 |
| プラン別機能差別化 | MVP は単一プラン(従量課金 + 月次無料枠) | FUT-04 |
| ホワイトラベル化 | MVP は本サービス提供事業者ロゴ必須表示 | FUT-04 |
| 担当者ハブ(複数契約所属) | MVP は 1 アカウント = 1 オーナーアカウント固定 | FUT-04 |
| 請求書払い / KYC / 反社チェック | MVP はクレジットカード払い必須、B2B 低リスク前提 | FUT-04 |
| オーナー譲渡 | MVP は契約作成者がオーナー固定、緊急時のみサポート対応 | FUT-04 |
| カスタムロール / メンバー権限の細分化 | MVP は 2 種固定(オーナー / メンバー)で、プロジェクト割当にサブロールを持たない(`M_PRJ_USERS` に role 列なし) | FUT-04 |
| お知らせ新着取得のリアルタイム化 | MVP は 60 秒ポーリング | FUT-05 |
| 添付ファイル / 音声入出力 / 既読同期 SSE | 個別チャット本体の導入後に拡張として検討 | FUT-05 / FUT-06 |
| 多段承認ワークフロー / FAQ 自動翻訳 / 自動 FAQ 化 / カテゴリ階層 | MVP は管理者の明示操作で下書き・公開、日本語のみ | FUT-05 |
| 外部ヘルプデスク連携 / 既存 FAQ サービス移行支援 | MVP は CSV インポートのみ | FUT-05 |
| 契約全データエクスポート(契約内データの CSV 一括エクスポート、非同期ジョブ + 完了メール) | MVP は退会時のデータ削除のみ実施。退会前のデータ持ち出し / 移植性(FR-162 / FR-131 / NFR-065)は将来対応。UI 導線(旧 SCR-023 データ出力 / 旧 SCR-014 エクスポート CTA)は撤去済み | FUT-05 |
| 顧客向けスマホアプリ | MVP はレスポンシブ Web 管理画面 + 公開ウィジェット | FUT-05 |
| 電話 / SMS 通知、BIMI、契約独自送信ドメイン | MVP はメール + 受信箱、送信ドメインはサービス共通 | FUT-05 |

## <span id="4-改善候補"></span>4. 改善候補

### <span id="41-t2ベータ判定まで"></span>4.1 T2(ベータ判定まで)

| 項目 | 方針 | 期限 | 主管 |
|----|----|----|----|
| 月次無料枠の具体値 | MVP 初期値を質問 1,000 件/月、FAQ 100 件として開始し、原価実測で継続可否を判定 | 2026-08-31 | PO / 経営 |
| PII 検出 NER モデル名 | Cloudflare 内で完結する NER 候補を選定 | 2026-08-31 | AI 担当 / セキュリティ責任者 |
| 不正利用検知の自動ブロックしきい値 | 初期値を同一 IP 5 分 100 質問超、1 時間 5 回連続 CAPTCHA 失敗、PII 検出 10 件超/時間として調整判定 | 2026-08-31 | SRE / セキュリティ責任者 |
| 招待トークン有効期限 | MVP は 7 日、ベータで 72 時間短縮を判定 | 2026-08-31 | セキュリティ責任者 |
| 利用者側 admin / operator MFA | 限定公開で opt-in、ベータで契約単位強制可否を判定 | 2026-08-31 | セキュリティ責任者 |

### <span id="42-t3ga-判定まで"></span>4.2 T3(GA 判定まで)

| 項目 | 方針 | 期限 | 主管 |
|----|----|----|----|
| オンコール体制 | NFR-820 相当の 24/365 体制を確定 | 2026-12-31 | 運用責任者 |
| 個別チャットの導入可否 | FUT06の本人性・保持期間・暗号化・通知・課金仕様を確定して実装着手可否を判定 | 2026-12-31 | PO / セキュリティ責任者 |
| 利用規約・プライバシーポリシー本文整備 | 法務最終承認版を完成 | 2026-12-31 | 法務 |
| OSS ライセンス一覧・SBOM 整備 | SBOM を CI 生成し、ライセンス義務違反 0 件を確認 | 2026-12-31 | テックリード |
| プラン別機能・カスタマイズ拡張性 | 単一プラン継続またはプラン分割を判定 | 2026-12-31 | PO / 経営 |
| SLA(顧客契約値) | GA 後 6 ヶ月の実績をもとに契約 SLA を策定 | 2027-06-30 | PO / 経営 |
