共有概念対応表(メイン / 運営者 + 全体)

1. 本書の位置づけ

本書は 01_メインシステム/02_基本設計/ と 02_運営者システム/02_基本設計/ の両セット間で共有される概念について、どのドキュメントが正本で、どこから参照されるか を機械検査可能な形式で正本化する索引である。保守ルール「同期ルール」を表化し、99_script/check-spec-sync.sh の検査対象とする。

両セットで重複記載すべきではない概念について、本表で示された正本ファイル以外への再掲は欠陥として扱う。参照側ドキュメントでは「正本: [リンク]」の形でリンクのみ記載すること。

  • 対象ファイル群:
    • 01_メインシステム/02_基本設計/01_画面設計.html 〜 11_メール設計.html(計 11 ファイル)
    • 02_運営者システム/02_基本設計/01_画面設計.html 〜 11_メール設計.html(計 11 ファイル)

4. 機械検査ルール(99_script/check-spec-sync.sh 仕様)

99_script/check-spec-sync.sh は以下の検査を実装する。

検査 ID 検査内容 失敗時の意味
SC-001 各個別設計ファイルの必須ヘッダ存在(# 文書名, ## 1. ...) 構造不適合
SC-002 共有概念正本ファイルでの必須キーワード出現(本表の「検査キーワード」列) 正本ドキュメントから記載が抜けている
SC-003 参照側ドキュメントの相互リンク存在(共有概念正本以外の場所で再掲しないことを軽くチェック) 共有概念の二重正本化
SC-004 SCR ID クロスチェック(画面設計 ・ トレーサビリティマトリクス の SCR 集合一致) SCR 漏れ / 不一致
SC-005 メッセージ ID 体系 MSG-SCR-NNN-<種別>-NNN の出現 メッセージ一覧 06 の体系破綻
SC-006 エラー ID 体系 E-<分類>-NNN / E-OP-* の出現 エラー設計 05 の体系破綻
SC-007 contract_owners.contract_status 4 値が正本ドキュメントに出現 列挙値の取りこぼし
SC-008 通知重要度 4 値が正本ドキュメントに出現、critical 行に "強制送信" 文言を伴う critical 通知ルール違反
SC-009 連携 IF #1〜#12 すべてが メイン / 運営者 双方の API 設計に出現 IF 漏れ
SC-010 主要相対パスのリンク到達性確認 参照リンク切れ
SC-011 要件定義 FR グループファイル / 詳細設計 DD グループファイル / 将来対応 FUT ファイルが両システムで揃っているか 構成不適合
SC-012 FR / DD / FUT、SCR、テーブル、API 節、メッセージ節、メール節の連番と削除済み識別子の再混入防止 欠番 / 重複 / 削除済み識別子の再混入
