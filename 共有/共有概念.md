# 共有概念対応表(メイン / 運営者 + 全体)

## 1. 本書の位置づけ

本書は `01_メインシステム/個別設計書群/` と `02_運営者システム/個別設計書群/` の両セット間で共有される概念について、**どのドキュメントが正本で、どこから参照されるか** を機械検査可能な形式で正本化する索引である。CLAUDE.md「同期ルール」を表化し、`99_スクリプト/仕様同期検査.sh` の検査対象とする。

両セットで重複記載すべきではない概念について、本表で示された正本ファイル以外への再掲は欠陥として扱う。参照側ドキュメントでは「正本: [リンク]」の形でリンクのみ記載すること。

## 2. 適用範囲

- 対象ファイル群:
  - `01_メインシステム/個別設計書群/02_画面設計書.md` 〜 `11_課金請求設計書.md`(計 10 ファイル)
  - `02_運営者システム/個別設計書群/02_画面設計書.md` 〜 `11_課金請求設計書.md`(計 10 ファイル)
  - `01_メインシステム/02_基本設計書.md` / `02_運営者システム/02_基本設計書.md`
- 対象外:
  - 詳細設計書(`03_詳細設計書.md`)── 実装関連の詳細のみ扱う層
  - 要件定義書(`01_要件定義書.md`)── WHAT の正本
  - 画面遷移図(`画面遷移図.html`)
  - 04_将来拡張 / 03_運用 / 99_スクリプト

## 4. 機械検査ルール(`仕様同期検査.sh` 仕様)

`99_スクリプト/仕様同期検査.sh` は以下の検査を実装する。

| 検査 ID | 検査内容 | 失敗時の意味 |
|---|---|---|
| SC-001 | 各個別設計書ファイルの必須ヘッダ存在(`# 文書名`, `## 1. 文書概要`, `## 1.1 目的`, `## 1.4 関連ドキュメント` 表) | 11 ドキュメント体系の構造不適合 |
| SC-002 | 共有概念正本ファイルでの必須キーワード出現(本表の「検査キーワード」列) | 正本ドキュメントから記載が抜けている |
| SC-003 | 参照側ドキュメントでの再掲禁止(再掲とみなすパターンを正規表現で定義) | 共有概念の二重正本化 |
| SC-004 | SCR ID クロスチェック(画面設計書 02 ・ トレーサビリティマトリクス 08 ・ 画面遷移図 HTML の SCR 集合一致) | SCR 漏れ / 不一致 |
| SC-005 | メッセージ ID 体系 `MSG-SCR-NNN-<種別>-NNN` の重複・抜けチェック | メッセージ一覧 07 の体系破綻 |
| SC-006 | エラー ID 体系 `E-<分類>-NNN` / `E-OP-*` の重複チェック | エラー設計書 06 の体系破綻 |
| SC-007 | `accounts.contract_status` 4 値が正本ドキュメントに出現 | 列挙値の取りこぼし |
| SC-008 | 通知重要度 4 値が正本ドキュメントに出現、`critical` 行に "メール強制送信" 文言を伴う | critical 通知ルール違反 |
| SC-009 | 連携 IF #1〜#12 すべてが メイン / 運営者 双方の API 設計書 03 に出現 | IF 漏れ |
| SC-010 | 主要相対パスのリンク到達性確認 | 参照リンク切れ |
