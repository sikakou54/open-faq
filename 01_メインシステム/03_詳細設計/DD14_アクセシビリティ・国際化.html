このページは、メインシステム管理画面のアクセシビリティ(WCAG 2.2 AA 準拠)と国際化(i18n)基盤の実装方針を定義します。 キーボード操作・ARIA・コントラスト比などのアクセシビリティ実装規約、モーダルの focus trap、専用 403 / 401 画面のスクリーンリーダー対応、自動テスト(axe-core)、i18n キー命名規則とカタログ管理、JST 固定のタイムゾーン処理・日時表示・UTF-8 4 バイト対応を扱います(NFR-401 / WCAG 2.2 AA / i18n 基盤)。

3. 詳細設計本文

アクセシビリティ実装規約(§3.1〜3.4)と国際化基盤(§3.5〜3.10)の実装方針を定めます。基準・対応関係が主体のため、各規約は箇条書き・対応表で示し、実装コードは <pre><code> でそのまま保持します。

3.1 アクセシビリティ実装規約

WCAG 2.2 AA を満たすための画面共通の実装規約を定めます。次の各項目は、キーボード操作・ARIA・コントラスト比・ターゲットサイズなど、全画面で守るべき要件です。

  • キーボード操作: 全画面で Tab / Enter / Esc が動作。フォーカス可視（:focus-visible 2px outline）
  • ARIA: ランドマーク（role="main", role="navigation" 等）、aria-label, aria-describedby, aria-live
  • コントラスト比: 通常テキスト 4.5:1 以上、大テキスト 3:1 以上（Tailwind カラーパレットで担保）
  • 代替テキスト: <img alt> 必須、装飾画像は alt=""
  • フォームラベル: <label htmlFor> で関連付け、エラーは aria-describedby で連携
  • ターゲットサイズ: 主要なボタン、アイコンボタン、行アクションは 44×44 CSS px 以上。例外は文章中のインラインリンク
  • フォーカス非遮蔽: sticky header / footer でフォーカス対象を完全に隠さない。scroll-margin を共通指定
  • 状態表現: 色や絵文字だけで状態を伝えず、必ず可読テキストを併記。装飾絵文字は aria-hidden="true"
  • ドラッグ代替: 並べ替え等にドラッグを用いる場合、キーボード操作と上下移動ボタンを併設

3.2 モーダル focus trap 実装規約

モーダル内でフォーカスを循環させ、外部に抜けないようにする実装規約を定めます。axe-core ではキーボードトラップの一部ケース(テスト終了時のフォーカス検証)を自動検出できないため、専用ライブラリと回帰テストで担保します。

  • 採用ライブラリ: focus-trap-react（同等 OSS でも可。MVP 着手時の依存追加 PR で確定）
  • 対象モーダル (メイン側): ConfirmDialog / 再認証モーダル / 退会確認モーダル / お知らせ詳細モーダル
  • 挙動規約:
    • モーダル open 時、内部最初の フォーカス可能要素 にフォーカス
    • Tab / Shift+Tab で内部循環、外部へ抜けない
    • Esc キー: ConfirmDialog 系は閉じる（dismiss 扱い）。重要操作モーダル（退会・再認証）は Esc 無効、明示の × ボタンで閉じる
    • モーダル close 後、フォーカスを 発火元のボタン or 指定 data-focus-restore-target に明示復元

3.2.1 キーボードショートカット一覧

マウスを使わずに操作を完結できるよう、画面共通およびモーダル内のキー割当を定めます。次の表は、場面ごとのキーと動作の対応です。

場面 キー 動作
全画面共通 Tab / Shift+Tab フォーカス送り
全画面共通 Enter 主アクション（送信ボタン等）
モーダル内 Esc 閉じる（重要操作モーダルを除く）
SCR-006 FAQ 編集 Ctrl+S (Cmd+S) 下書き保存

3.2.2 回帰テスト

focus trap が継続して機能することを E2E テストで担保します。apps/admin/e2e/a11y/focus-trap.spec.ts で各モーダルにつき以下を検証します。

  • Tab × 50 回押下でフォーカスが循環し外部に抜けないこと
  • Esc 挙動が上表通り
  • close 後の document.activeElement が発火元一致

3.3 専用 403 / 401 画面のアクセシビリティ

アクセス拒否(403)・契約停止(401)の専用画面で、状態がスクリーンリーダー利用者に確実に伝わるための実装規約を定めます。次の各項目は、role="alert" と aria-live による状態通知、ステータスコードの可読テキスト併記などの要件です。

  • IP 許可リスト 403 画面（運営者ログイン前、operator_ip_allowlist） / 契約停止 401 画面:
    • ページ単位で role="alert" を main 領域に付与し、SR 利用者に状態が伝わるようにする
    • 文言は <h1> で「アクセスが拒否されました」、本文に原因と次のアクション（連絡先 / 管理者向けガイド URL）を aria-live="assertive" で読み上げ可
    • ステータスコード（403 / 401）は人間可読テキストとして本文に含める（数字単独は SR で意味が伝わらないため「アクセス権限がありません (403)」のように併記）
    • 連絡先メールアドレスは mailto: リンクで提供、aria-label="サポート窓口へメールで問い合わせる" を付与
    • メイン側 SCR-001 ログイン画面 でも、ログイン失敗時のエラーメッセージは role="alert" + aria-live="assertive" で読み上げ可

3.4 アクセシビリティ自動テスト

アクセシビリティ違反を CI で継続的に検出する方針を定めます。@axe-core/playwright を E2E に組み込み、Critical / Serious 違反は 0 件を維持します。focus trap / 専用 403 画面は axe-core で検出できないため、focus-trap.spec.ts / a11y-error-pages.spec.ts で個別検証します。

3.6 i18n キー命名規則とカタログ管理

i18n キーの命名規則とカタログの管理方法を定めます。キーは <scope>.<entity>.<state_or_action> の 3 階層を基本形とし、エラーメッセージは errors.<feature>.<code> を使います。次の表はカテゴリごとの命名パターンと例です。

カテゴリ 命名パターン 例
画面ラベル screen.<scr_id>.<element> screen.scr_012.faq_title
ボタン共通 common.button.<action> common.button.save / common.button.cancel
バリデーション errors.validation.<field>.<rule> errors.validation.email.required
ビジネスエラー errors.<feature>.<code> errors.faq.quota_exceeded / errors.search.range_too_wide
お知らせ種別 announcement.kind.<value> announcement.kind.critical

カタログ管理:

  • 場所: app/shared/src/i18n/ja.ts
  • 構造: ネストしたオブジェクト (TypeScript の型推論でキー補完を効かせる)
  • CI 検証: 未使用キー検出 + 翻訳ファイル間のキー欠落検出を i18n-lint 等で実施

3.7 タイムゾーン処理

MVP では全契約のタイムゾーンを JST に固定する方針を定めます。アプリ層の日時表示は formatJst に一本化し、API レスポンスは常に UTC ISO 8601 で受け渡します。次の各項目が MVP 仕様の要点です。

MVP 仕様:

  • 全契約 JST 固定。formatJst を アプリ層の唯一の日時表示関数 とし、new Date().toLocaleString() の直接呼出は ESLint ルール (no-restricted-syntax) で禁止。
  • API レスポンスの日時は 常に UTC ISO 8601 (Z 末尾)。クライアント側で formatJst 通過後に表示。
  • メール本文・PDF 帳票も JST 固定（テンプレート側で formatJst 呼出）。

回帰テスト: 既存契約全件が JST 固定で動作することを Integration テストで保証 (it-tz-default-001)。

3.8 日時表示

日時表示の唯一の関数 formatJst の実装を示します。DB は UTC ISO 8601 で保持し、表示時に JST へ変換します。次のコードがその実装です。

// app/admin/src/lib/format-date.ts
export function formatJst(iso: string): string {
  return new Date(iso).toLocaleString('ja-JP', {
    timeZone: 'Asia/Tokyo', year: 'numeric', month: '2-digit',
    day: '2-digit', hour: '2-digit', minute: '2-digit',
  });
}

DB は UTC ISO 8601、表示時 JST。

3.9 UTF-8 4 バイト対応

絵文字を含む 4 バイト UTF-8 文字を安全に格納・検索できることを定めます。SQLite はデフォルトで UTF-8 に対応し、4 バイト文字を含む文字列も格納可能で、インデックスは BINARY collation で問題ありません。

3.10 i18n 規約まとめ

国際化対応の最重要原則を 1 点に集約します。ユーザー向けテキストはすべて i18n キーで参照し、画面・コンポーネントへの直接文字列のハードコードは禁止します。
