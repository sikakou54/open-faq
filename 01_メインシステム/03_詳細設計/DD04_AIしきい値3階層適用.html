3. 詳細設計本文

しきい値の取得・更新・フォールバック監視・無効化を順に定義します。取得は 3 階層を上位優先で解決し、KV ヒットを最速経路、D1 永続キャッシュを次段、グローバル既定値を最終フォールバックとします。

3.1 取得フロー

1 リクエストごとのしきい値解決を、上位優先(プロジェクト > オーナー > グローバル)で行います。まず階層の優先順位を表で示し、続くシーケンス図で KV → D1 永続 → グローバル既定の段階的フォールバックと、KV ヒット / ミス・永続あり / なしの分岐を示します。最後に実装の getThreshold を擬似コード正本として保持します。

#階層(scope)解決キー意味
1projectproject_id最優先。当該プロジェクトに設定されたしきい値
2ownercontract_owner_user_idプロジェクト未設定時に適用する契約(オーナー)既定
3global—最終フォールバック。信頼度 0.60 / 関連度 0.50

scope 優先順位の実体D1 照会は ORDER BY CASE scope WHEN 'project' THEN 0 WHEN 'owner' THEN 1 ELSE 2 END LIMIT 1 で最優先 scope の 1 件を選びます。KV へ書き戻すことで次回以降 60 秒間は KV ヒット経路になります。
// app/workers/widget-api/src/domain/ai-threshold.ts
export async function getThreshold(env: Env, contractOwnerUserId: string, projectId: string): Promise<{
  confidenceThreshold: number;
  relevanceThreshold: number;
  source: 'kv' | 'persistent' | 'global_default';
}> {
  // 1) KV (TTL 60s)
  const kvKey = `ai_threshold:${contractOwnerUserId}:${projectId}`;
  const kvHit = await env.KV_CACHE.get<Threshold>(kvKey, 'json');
  if (kvHit) return { ...kvHit, source: 'kv' };

  // 2) 連携 IF #6 経由で受信したものが KV 反映されている前提。
  //    KV miss なら D1 永続キャッシュへフォールバック
  const persistent = await env.DB.prepare(`
    SELECT confidence_threshold, relevance_threshold FROM ai_threshold_persistent_cache
    WHERE (scope='project' AND project_id=?1)
       OR (scope='owner' AND contract_owner_user_id=?2)
       OR (scope='global')
    ORDER BY CASE scope WHEN 'project' THEN 0 WHEN 'owner' THEN 1 ELSE 2 END
    LIMIT 1
  `).bind(projectId, contractOwnerUserId).first<Threshold>();
  if (persistent) {
    await env.KV_CACHE.put(kvKey, JSON.stringify(persistent), { expirationTtl: 60 });
    return { ...persistent, source: 'persistent' };
  }

  // 3) グローバル既定値
  await enqueueServiceAlert(env, {
    level: 'normal', message: `AI しきい値フォールバック発動: owner=${contractOwnerUserId}`,
  });
  return { confidenceThreshold: 0.60, relevanceThreshold: 0.50, source: 'global_default' };
}

3.2 KV TTL 60s + 永続キャッシュ更新

運営者からのしきい値変更(IF #6)を受信したときのキャッシュ更新方針を定めます。KV の鮮度(TTL 60 秒)と D1 の永続性を両立させ、KV ミス時は D1 から再ロードして書き戻します。

契機更新対象内容
IF #6 受信KV + D1 両方D1 (ai_threshold_persistent_cache) を更新し、KV (expirationTtl: 60) も同時に更新する
KV ミスKV(書き戻し)D1 から再ロードし、KV へ TTL 60 秒で書き戻す(§3.1 取得フロー)
