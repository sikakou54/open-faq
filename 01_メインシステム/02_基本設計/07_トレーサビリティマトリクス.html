トレーサビリティマトリクス(メイン)

1. 文書概要

1.1 目的

メインシステムの要件 → 機能 → 画面 → API → DB → 権限 → エラー → メッセージ → テスト観点 の対応関係を一元化する。要件漏れ、設計漏れ、ID 不整合の機械検出を可能にする。

1.2 対象範囲

  • 対象: メインシステムの全要件(FR / NFR / BR / AC)、機能(F-)、画面(SCR-001〜027)、API(API-)、テーブル(31 個)、プロジェクト別ロール(admin / member)+ オーナー、エラー ID(E-*)、メッセージ ID(MSG-*)
  • 対象外: 運営者側要件(02_運営者システム/個別設計書群/08_トレーサビリティマトリクス.html 参照)
