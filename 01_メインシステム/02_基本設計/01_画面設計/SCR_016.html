SCR-016 利用状況

このページは、オーナーが契約全体と各プロジェクトの利用状況を当月固定の読み取り専用スナップショットで把握する起点画面 SCR-016 を定義します。 画面概要 / 画面遷移図 / 画面レイアウト / 画面項目定義 / 入出力一覧 / 画面イベント一覧 の 6 セクションで記述します。

1. 画面概要

契約ワークスペースのトップ画面で、契約全体の集計値とプロジェクト別の利用量を 1 画面に集約する読み取り専用画面です(オーナー専有)。請求金額・支払方法・請求履歴は SCR-022 へ分離します。

4. 画面項目定義

本画面の表示項目(全体サマリー・支払い警告・プロジェクト別利用状況・空状態)を定義します。項目の正本は本表です。各セルはリンクを持たない読み取り専用表示で、プロジェクトを開く・作成・編集する操作は本画面では提供しません。

項目 ID 項目 入出力 種類 備考
IT-01全体サマリー出力SummaryCard × 3利用中のプロジェクト数 / 質問数合計 / 公開 FAQ 数合計。各カードはクリック不可
IT-02支払い警告出力PaymentMethodBanner支払方法未登録または支払い失敗時のみ表示。「請求を確認」→ SCR-022
IT-03プロジェクト別利用状況出力テーブルプロジェクト名 / 質問数(利用 / 上限 + 利用率・状態を同一セルに併記)/ 公開 FAQ 数 の 3 列。利用率の高い順を既定、同率はプロジェクト名昇順。上限 OFF は「上限なし」と表示。各セルはリンクを持たない読み取り専用
IT-04空状態(プロジェクト 0 件)出力EmptyState「プロジェクトがまだありません」+「プロジェクトへ」CTA(→ SCR-004)

5. 入出力一覧

本画面が読み取るテーブルと、呼び出す API の一覧です。テーブルの正本は 03_テーブル設計、API の正本は 02_API設計 §5.7.1 です。

入出力名 種別 I/O アクセス種別(CRUD) 備考
C R U D
プロジェクトテーブル入力—◯——プロジェクト名・件数の取得。projects(テーブル設計 3.6)
利用量計測テーブル入力—◯——当月の質問数・利用率の取得。usage_metering(テーブル設計 3.22)
プロジェクト上限テーブル入力—◯——プロジェクト別の月次上限・上限 OFF 判定。project_quota_limits(テーブル設計 3.24)
契約サブスクリプションテーブル入力—◯——支払方法未登録・支払い失敗の判定。billing_subscriptions(テーブル設計 3.20)
利用状況取得API入力————本画面は契約全体サマリーと各プロジェクト別利用量の双方を表示する。契約横断の集計は viewMode=contract(または同等の集約パラメータ)を用い、プロジェクト別は viewMode=project&projectId={id} で取得する(SCR-021 の単一プロジェクト表示と区別する)。取得方式の正本は 02_API設計 §5.7.1 を正とする。物理名 GET /usage?period=current_month&viewMode={contract|project}
