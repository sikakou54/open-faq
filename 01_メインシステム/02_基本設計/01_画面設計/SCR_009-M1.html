1. 画面概要

SCR-009 から開く、メンバー招待(新規)とロール編集・招待再送・割当解除(既存)を全画面割込みモーダルで行う画面です。現在開いているプロジェクトの 1 件の割当のみを操作対象とします。

重要必要権限は当該プロジェクトに対する admin(オーナーは暗黙 admin として含む)です。招待モードでは氏名フィールドを表示しません(管理者は他人の氏名を事前入力できない。氏名は招待された本人が SCR-018 で入力。FR-016d 個人情報原則)。最後の有効割当を解除した場合のみアカウントを自動論理削除(users.valid=0)します。

4. 画面項目定義

本モーダルの表示・入力項目・操作ボタンと各バリデーション・ガードを定義します。項目の正本は本表です。招待モード / 編集モードでのみ表示する項目は備考に明記します。

項目 ID 項目 入出力 種類 備考
IT-01モード見出し出力見出し招待「{プロジェクト名} へメンバーを招待」/ 編集「{プロジェクト名} のメンバー編集 — {表示名}」
IT-02モーダル閉じる(×)入力アイコンボタン変更を破棄して閉じる。未保存変更があれば UnsavedChangesGuard で警告。Escape / オーバーレイクリックも同等
IT-03自己編集警告帯出力アラート帯編集モードかつ自分自身編集時のみ。「このプロジェクトのロールは変更できません / 自分のアカウントは削除できません」
IT-04メールアドレス入力メールアドレスボックス必須。招待モードでのみ編集可、編集モードでは表示のみ。同一オーナースコープに既存アカウントがある場合は名前解決し当該 PJ 分の project_users 行を追加(FR-021c)
IT-05表示名(氏名)出力テキスト表示(read-only)招待モードでは表示しない(個人情報原則)。編集モードは users.name を read-only 表示、NULL は「未設定(本人が SCR-018 で未入力)」
IT-06招待モード氏名注記出力helper招待モードのみ。「氏名(表示名)は招待されたご本人が SCR-018 で入力します」(FR-016d)
IT-07このプロジェクトでのロール入力ラジオボタン(2 値必須)「メンバー(member)/ プロジェクト管理者(admin)」。ラジオ末尾にロール説明のヘルプアイコン
IT-08招待状態出力バッジ編集モードかつ pending_activation のときのみ「招待中」を表示(残日数併記なし)。active では非表示
IT-09招待メールを再送入力ボタン(フッター、再認証)編集モードかつ pending_activation のみ。旧リンク失効・新リンク発行
IT-10プロジェクトから外す入力ボタン(Danger、L1 確認)編集モードのみ。当該 PJ の project_users 行を valid=0。最後の有効割当の場合は同一トランザクションで users.valid=0・全セッション失効・未使用招待トークン失効。自分・オーナーには非表示
IT-11招待メールを送信入力ボタン(Primary、再認証)招待モードのみ。access_tokens.purpose='activation'(7 日)を発行(meta に invitedProjectId / invitedRole)
IT-12変更を保存入力ボタン(Primary、再認証)編集モードのみ。当該 PJ の project_users.role をラジオ値で更新
IT-13最後の管理者保護ガード出力UI 制御 / バリデーション自分が唯一の admin の場合 member 切替を非活性 + tooltip。他メンバー降格で admin が 0 件になる場合は L2 確認 + E-AUTHZ-LAST-ADMIN-PROTECTED で拒否
注意バリデーション: メールアドレスは必須(招待モードのみ編集可)、ロールは member / admin の 2 値必須。「プロジェクトから外す」は L1 確認、最後の有効割当解除時は確認ダイアログに「このメンバーのアカウントも利用停止になります」を追記します。プロジェクト管理者降格は L2 確認 + 最後の管理者保護を行います。ロール定義は 04_権限設計 を正本とします(member = FAQ 管理 / 要対応質問の状況管理 / ログ参照、admin = それに加えメンバー招待・ロール変更・割当解除)。
