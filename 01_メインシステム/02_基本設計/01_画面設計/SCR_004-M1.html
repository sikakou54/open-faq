1. 画面概要

SCR-004 から開く、プロジェクトの新規作成・編集・削除を全画面割込みモーダルで行う画面です(オーナー専有)。呼出元で新規作成 / 編集モードを切り替えます。

重要本モーダルに「プロジェクト管理者を指定」セクションは設けません。オーナーは作成時に自動で当該プロジェクトの管理者となります(project_users(role='admin', valid=1) 自動 INSERT。FR-030a と整合)。他者をプロジェクト管理者として招待する操作はプロジェクト作成後に SCR-009 / SCR-009-M1 で行います。

4. 画面項目定義

本モーダルの入力項目・操作ボタン・DangerSection と各バリデーションを定義します。項目の正本は本表です。編集モードのみ表示する項目は備考に明記します。

項目 ID 項目 入出力 種類 備考
IT-01見出し出力見出し新規「新規プロジェクトを作成」/ 編集「プロジェクトを編集」
IT-02プロジェクト ID出力読み取り専用テキスト + コピーボタン編集モードのみ。クリック遷移には使わずコピー操作のみ提供
IT-03プロジェクト名入力テキストボックス必須。1〜100 文字、placeholder「例: サポートサイト」、文字数カウンタ。編集モードは現値を初期表示
IT-04許可ドメイン入力タグ入力(複数値)必須。Enter またはカンマで 1 件追加、即時検証。完全一致 + *.example.com 形式可。IP アドレス・プロトコル指定は不可。編集モードは現値を初期表示
IT-05許可ドメイン補足ヘルプ出力helper「ウィジェット埋め込みを許可するドメイン。サブドメインを許可するには *.example.com のように記載します」
IT-06プロジェクト連絡先メール入力メールアドレスボックス任意。確認メール送信 → リンククリックで確認完了。確認完了後はウィジェットの未解決時・制限中「お問い合わせ先」表示にのみ利用。編集モードは現値を初期表示
IT-07連絡先メール確認状態出力状態バッジ + 再送ボタン編集モードのみ。「確認済み」/「確認待ち」/「未設定」を表示。確認待ち時は確認メール再送(レート制限あり)を提供
IT-08キャンセル入力ボタン(Secondary)変更を破棄して閉じる。未保存変更があれば UnsavedChangesGuard で警告
IT-09プロジェクトを作成入力ボタン(Primary)新規モードのみ。バリデーション通過時に作成。ウィジェット公開キー初回発行 + オーナーを project_users(role='admin', valid=1) 自動 INSERT
IT-10保存入力ボタン(Primary)編集モードのみ。バリデーション通過時に PATCH /projects/{id} で更新。連絡先メール変更時は確認メールを再送信
IT-11プロジェクトを削除入力ボタン(Danger、L3)編集モードのみ(DangerSection)。プロジェクト名タイプ確認 + 再認証を経て DELETE /projects/{id}。論理削除(projects.valid=0)+ 関連 project_users 論理削除(FR-030b)。削除導線は本 DangerSection のみに集約
注意バリデーション: プロジェクト名は必須 1〜100 文字、許可ドメインは必須(URL またはワイルドカード形式、IP / プロトコル指定不可)、連絡先メールは任意(メール形式)。削除はプロジェクト名の完全一致タイプ確認 + 再認証(L3)を必須とします。
