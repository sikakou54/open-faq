4. 画面項目定義

本画面の入出力項目(公開キー管理・見た目・プレビュー・埋め込みコードの各セクション)を定義します。項目の正本は本表です。

項目 ID 項目 入出力 種類 表示条件 備考
IT-01スコープ注記出力注意文—「この公開キーは本プロジェクト専用です。他プロジェクトのウィジェットとは共有されません」
IT-02ウィジェット公開キー出力テキスト + コピーボタン—(コピーはメンバーも可)形式 pk_live_<32-char base62>(FR-194)。参照のみ。無期限のため有効期限 UI なし
IT-03旧キー使用中バッジ出力バッジ + 注意文ローテーション猶予 24 時間中に旧キー使用検知時のみ表示—
IT-04公開キーを再発行(ローテーション)入力ボタン(Danger / DangerSection / L3)admin 以上のみ(member は非活性)新キー発行と同時に旧キーは 24 時間猶予で失効予告。確認文に「既存の埋め込みコードは旧キー失効後に動作しなくなります」を必須表示(FR-031)
IT-05主色(プライマリカラー)入力カラーピッカー(HEX 指定)admin 以上のみ編集可(member は非活性)プレビューにリアルタイム反映(FR-152)
IT-06プレビュー出力プレビューウィンドウ—起動前の丸型ランチャーバッジと展開後のチャット UI を切り替えて確認。設定変更時にリアルタイム反映
IT-07「Powered by」表示出力注釈—本サービス運営者ロゴを必須表示
IT-08埋め込みコード出力コードブロック + コピーアイコン—(コピーはメンバーも可)コピーアイコンをコード右上にオーバーレイ。クリックで全文コピー
IT-09設定の保存入力ボタン(Primary)admin 以上のみ(member は非活性)設定更新 + KV キャッシュ無効化

5. 入出力一覧

本画面が読み書きするテーブルと、呼び出す API の一覧です。テーブルの正本は 03_テーブル設計、API の正本は 02_API設計 §5.3 です。ウィジェット公開キー・見た目設定は projects テーブルに保持します。

入出力名 種別 I/O アクセス種別(CRUD) 備考
C R U D
プロジェクトテーブル入力 / 出力—◯◯—公開キー・ウィジェット設定 JSON の取得・更新。projects(widget_public_key / ウィジェット設定 JSON)(テーブル設計 3.6)
レガシー公開キーテーブル出力◯———ローテーション時に旧キーを 24 時間保持。project_legacy_keys(テーブル設計 3.7)
プロジェクト更新API入力 / 出力————ウィジェット設定(主色等)の取得・保存。PATCH /projects/{id}(API 設計 5.3.3)
ウィジェット鍵ローテーションAPI出力————公開キー再発行(旧キー 24h 猶予)。POST /projects/{id}/widget-key/rotate(API 設計 5.3.4)
