SCR-017 個人設定

このページは、認証済みの利用者が自身のプロフィール・セキュリティ・参加プロジェクトを編集する画面 SCR-017 を定義します。 画面概要 / 画面遷移図 / 画面レイアウト / 画面項目定義 / 入出力一覧 / 画面イベント一覧 の 6 セクションで記述します。

版数 v1.0更新 2026-06-17承認済

1. 画面概要

ヘッダ右上のアカウントメニュー「個人設定」から開く全画面で、自分のプロフィール・セキュリティ・参加プロジェクトをタブで扱う画面です。契約連絡先と退会は SCR-023 設定へ分離します。

関連 内容
FR / BRFR-001, FR-005, FR-006, FR-340 / —
関連画面SCR-013 メールアドレス変更確認(確認メール再利用) / SCR-023 設定
ステークホルダ 対象
オーナー◯
プロジェクト管理者(admin)◯
メンバー(member)◯
補足認証済みであれば全ロールが利用でき、自分の情報のみ編集可能です。誤操作防止としてパスワード変更は再認証(現パスワード再入力)を要し、メールアドレス変更は再認証 + 新メールアドレスの確認メールを要します(再認証の正本は 08_認証・認可設計 §2)。個人ごとの通知受信オプトアウトは本画面では扱わず、プロジェクト関連通知は常時 ON 固定です。

4. 画面項目定義

本画面の入出力項目を、プロフィール・セキュリティ・参加プロジェクトの 3 タブに分けて定義します。項目の正本は本表です。

項目 ID 項目 入出力 種類 表示条件 備考
IT-01表示名入力テキストボックスプロフィールタブ必須。1〜100 文字
IT-02メールアドレス入力メールアドレスボックスプロフィールタブ必須。変更時は確認メール送信(SCR-013 再利用)
IT-03パスワードを変更する入力ボタンセキュリティタブ旧パスワード + 新パスワード + 確認。再認証(現パスワード再入力)を要する(08_認証・認可設計 §2)
IT-04参加プロジェクト一覧出力テーブル参加プロジェクトタブプロジェクト名 / 適用ロール。プロジェクト名から該当 SCR-008 へ遷移
補足アクティブセッション一覧の表示と自己セッション終了は MVP 対象外です(05_future、FR-332 改訂)。複数デバイス同時ログインは可能で、運営者による強制ログアウト(連携 IF #2)は MVP で提供します。

5. 入出力一覧

本画面が読み書きするテーブルと、呼び出す API の一覧です。テーブルの正本は 03_テーブル設計、メール確認 API の正本は 02_API設計 §5.1.6 です。

入出力名 種別 I/O アクセス種別(CRUD) 備考
C R U D
利用者テーブル入力 / 出力—◯◯—表示名・メールアドレス・パスワードの参照・更新。users(テーブル設計 3.1)
プロジェクト割当テーブル入力—◯——参加プロジェクト一覧・適用ロールの取得。project_users(テーブル設計 3.3)
メールアドレス変更確認API入力 / 出力————確認トークン検証 POST /auth/email-verifications/{token}(API 設計 5.1.6)
