メッセージ一覧(メイン)

このページは、メインシステムでアカウント利用者・ウィジェット利用者に表示するすべての文言と、各通知イベントの契機・配信先を定義します。 画面メッセージ ID(MSG-SCR-*)の文言全文と、通知の「いつ・誰に・どのチャネルで」を本書の正本とします。メール件名・本文の全文、エラー ID 定義、画面項目、配信信頼性の実装方式は他ドキュメントが正本のため、本書からはリンクで参照します。

版数 v2.0更新 2026-06-17承認済

対象範囲SCR-001〜023 + モーダルの全画面文言、エラー文言(05_エラー設計.html と対応)、通知契機・配信先(§6、メール本文は 11_メール設計.html 正本)、課金・契約関連メッセージ、サイドメニュー文言。対象外: コード内デバッグログ。

2. 共通基準

全文言は本節の共通基準(トーン・ボタン文言・確認ダイアログ・通知重要度・エラー文言・禁則語)に従う。各画面の具体文言は §4 以降で定義する。

2.1 文言トーン

全文言で守る語調・表記の基本ルールを定める。次の箇条書きは敬体・二人称・数値表記の方針を示す。

  • 丁寧体(です・ます調)、ユーザーを「あなた」と呼ばない(二人称省略)
  • 専門用語は説明を併記(初出時)
  • 数値は半角、単位は和(円 / 件 / 日)

2.2 ボタン文言ルール

ボタン文言の種別ごとの組み立て方を定める。次の表は主操作・取消・危険操作のテンプレートと例を示す。

種別 テンプレート 例
主操作 対象 + 具体動詞 + する 「削除する」「招待メールを送信する」「変更を保存する」
取消 文脈に合わせて「キャンセル」または「閉じる」 確認ダイアログ・編集モーダルでは「キャンセル」、情報表示のみのモーダルでは「閉じる」
危険操作 赤色 + 確認ダイアログ 「削除する」「退会する」

2.3 確認ダイアログルール

確認ダイアログのタイトル・本文の書き方を定める。次の表は要素ごとのテンプレートと例を示す。

要素 テンプレート 例
タイトル 「{動詞}しますか?」 「FAQ を公開しますか?」「メンバーを削除しますか?」
本文 操作結果 + 取消可否 「公開するとウィジェット利用者に表示されます。」「削除すると元に戻せません(復元には別途依頼が必要です)」

2.4 通知重要度ルール(共有概念正本)

通知重要度 4 値ごとの配信先とチャネルを定める。次の表のとおり critical はメール強制送信(オプトアウト不可)とする。本表は共有概念の正本である。

重要度 配信先 配信チャネル
low inbox のみ -
normal inbox + email(任意) inbox / email
high inbox + email inbox / email
critical inbox + email 強制送信 inbox + email

critical の送信先解決ルール: 同一オーナースコープで project_users.role='admin' AND valid=1 を 1 件でも保持する全ユーザー(オーナーは作成時に自動で admin 行を保持するため本クエリ単独でオーナー + 全プロジェクト管理者を網羅)。詳細クエリは 08_認証・認可設計.html の通知宛先解決を正本とし、テンプレート別の配信先は 11_メール設計.html §5.1 を参照。

2.5 エラーメッセージルール

エラー文言の作成時に守る原則を定める。次の箇条書きはエラー ID との対応付け・内部情報の秘匿・攻撃者へのヒント抑止の方針を示す。

  • 05_エラー設計.html の各エラー ID に必ず 1 つのメッセージ ID を割り当てる
  • 内部詳細(スタックトレース / DB エラー)はユーザーに見せない
  • 認可エラーは具体的な権限名を含めない(列挙攻撃対策)
  • 認証失敗時はメールアドレス / パスワードのどちらが誤りかを示さない(攻撃者にヒント与えない)
  • 可能な場合は、原因と次に取れる行動を 1 文または 2 文で示す

2.6 禁則語(単独使用禁止)

文言として単独使用してはならない曖昧語を列挙する。次の語は文脈を補わずに用いない。

「OK」「実行」「処理する」「確定」「送信」「こちら」「エラーが発生しました」「不正な値です」「失敗しました」

2.7 i18n キー化必須(基本設計書 §11.8 から移管)

将来の多言語化に備えた文言の持ち方を定める。次の箇条書きのとおり MVP は日本語のみだが全文言を i18n キー化する。

  • MVP は日本語のみだが、全メッセージは i18n キー化必須
  • メッセージID = i18n キーとして直接利用可能な形式
