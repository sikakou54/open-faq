## 2. ロール一覧

利用者側は 3 つのロール(オーナー / プロジェクト管理者 / メンバー)で構成する。プロジェクト管理者とメンバーは「プロジェクト単位」のロールであり、同一メンバーがプロジェクト A では `admin`、プロジェクト B では `member`、プロジェクト C には未割当、といった組合せをとり得る。

| ロールID | ロール名 | 概要 | DB 値 |
|---|---|---|---|
| OWNER | オーナーアカウント | 契約あたり 1 アカウント固定、全プロジェクトの暗黙の `admin` 相当 + オーナー専有機能。`account_project_grants` に行を持たない | `accounts.role='admin'` AND `is_owner=1` AND `owner_account_id=id` |
| PROJECT_ADMIN | プロジェクト管理者 | 該当プロジェクトに対する `admin` ロール保持メンバー。複数プロジェクト兼任可 | `accounts.is_owner=0` AND `account_project_grants.role='admin'`(プロジェクト単位)|
| MEMBER | メンバー | 該当プロジェクトに対する `member` ロール保持メンバー | `accounts.is_owner=0` AND `account_project_grants.role='member'`(プロジェクト単位)|
| END_USER | エンドユーザー | ウィジェット利用者(自分の問い合わせ ID のみ)| `accounts.role='end_user'` |
| SERVICE_OPERATOR | 運営者 | 本 SaaS 提供者(運営者側)| `accounts.role='service_operator'` |

**用語の衝突回避**: `accounts.role='admin'` は契約上の利用者管理者を表す DB 値で、すべての利用者アカウント(オーナー / メンバー双方)で `'admin'`(= `service_operator` / `end_user` ではない)。一方 `account_project_grants.role='admin'` はプロジェクト管理者ロールを示す。本書では文脈区別のため必ず `accounts.role` / `account_project_grants.role` のように修飾する。

## 3. ロール定義(`account_project_grants.role` 2 値 + オーナー)

### 3.1 ロール概要

| ロール | 表示名(UI)| 範囲 | できること |
|---|---|---|---|
| オーナー | オーナー | 契約全体 | プロジェクトの作成・編集・削除、IP 許可リスト、許可ドメイン等のプロジェクト設定全般、課金、退会、規約再同意、メンバー招待・編集・削除、全プロジェクトの FAQ / 個別チャット / ログ参照(暗黙の `admin` 相当) |
| `admin` | プロジェクト管理者 | 該当プロジェクト | FAQ 登録・編集・公開、未解決質問対応・チャット返信、ログ参照、データエクスポート、**該当プロジェクトへのメンバー招待・離脱・ロール変更**(操作者自身が `admin` のプロジェクト内に限る) |
| `member` | メンバー | 該当プロジェクト | FAQ 登録・編集・公開、未解決質問対応・チャット返信、ログ参照、データエクスポート(該当プロジェクト範囲) |

オーナーは `account_project_grants` に行を持たず、Principal 構築時に `is_owner=1` 判定で認可チェックを bypass する(詳細は [09_認証認可設計書.md §6.1](09_認証認可設計書.md))。

### 3.2 ロール × 主管 SCR / FR 対応

| ロール | 主な操作対象 SCR / FR |
|---|---|
| オーナー | 全 SCR、FR-015a / FR-015c / FR-030 / FR-030a / FR-030b / FR-035a / FR-115〜127(課金)/ FR-189(退会)/ FR-179 / FR-193(API キー)/ 規約 |
| `admin`(プロジェクト管理者)| SCR-011 / 012 / 013 / 015(該当プロジェクト範囲)/ SCR-017 / SCR-017-M1(操作者が `admin` のプロジェクト範囲)/ 選択中プロジェクト詳細 / FR-015 / FR-016 / FR-018a〜c / FR-040〜046 / FR-070〜079 / FR-086 / NFR-602 系列 |
| `member` | SCR-001(ダッシュボード)/ SCR-011 / 012 / 013 / 015(該当プロジェクト範囲)/ FR-040〜046 / FR-070〜079 / FR-086(返信のみ)|

### 3.3 オーナー専有機能(プロジェクト別ロールでは付与不可)

- **プロジェクトの作成・編集・削除**(SCR-010 / SCR-010-M1、許可ドメイン、IP 許可リスト等のプロジェクト設定全般を含む)
- **課金**(SCR-015 の月次予算変更 / 支払い方法変更)
- **退会**(SCR-024)
- **規約再同意**(SCR-025 の同意 / 不同意)
- **メンバーアカウント削除**(`DELETE /projects/{id}/members/{accountId}`、v1.7 でアカウント全体物理削除に統一)── オーナー操作時は全メンバー削除可、プロジェクト管理者(オーナー以外)操作時は当該プロジェクトの member のみ削除可(他 admin は member 在籍中削除不可)。詳細は FR-019 / FR-019a / FR-019b を正本とする
- **契約ホーム閲覧 / OnboardingChecklist 操作**(SCR-026 全体、FR-326)── プロジェクト管理者 / メンバーは契約ワークスペース自体が WorkspaceSwitcher に出現しないため到達不可。URL 直叩き(`/owner/*`)時は 403 + 既定ワークスペースへリダイレクト

UI 上は非表示 + API は 403(`E-AUTHZ-OWNER-ONLY`)。

## 5. 機能別権限

| 機能ID | 機能名 | 必要ロール | オーナー | 制限事項 |
|---|---|---|---|---|
| F-AUTH-* | 認証 | 不要 | ◯ | - |
| F-PRJ-* | プロジェクト管理(作成・編集・削除)| **オーナー専有** | ◯ | 削除は再認証必須 |
| F-FAQ-* | FAQ 管理 | `member`+(該当 PJ)| ◯ | 公開ガード(`draft → published` 直接遷移禁止)|
| F-INQ-* | 未解決質問 | `member`+(該当 PJ)| ◯ | `closed` 遷移は admin の明示操作のみ(FR-079)|
| F-CHAT-* | 個別チャット | `member`+(該当 PJ)| ◯ | 部屋クローズ / 再オープンは再認証必須 |
| F-WIDGET-* | ウィジェット設定(プロジェクト単位)| `admin`+(該当 PJ)| ◯ | 公開キー再発行は L3(再認証必須)。`member` は閲覧 + 埋め込みコードのコピーのみ可。配置はプロジェクトワークスペース(SCR-014)|
| F-BILL-* | 課金 | **オーナー専有** | ◯ | 月次予算変更は再認証必須 |
| F-USER-* | ユーザー管理(招待・ロール変更・アカウント削除)| `admin`+(該当 PJ)/ オーナー(暗黙 admin)| ◯ | 招待 / ロール変更 / アカウント削除 / 招待再送は再認証必須。削除はアカウント全体物理削除に統一(v1.7) |
| F-ACC-* | アカウント設定 / 退会 | **オーナー専有** | ◯ | 退会は再認証必須 |
| F-NOTIF-* | お知らせ | 全 admin | ◯ | 既読化のみ |
| F-LEGAL-* | 規約 / 同意 | **オーナー専有(同意)** / 全ユーザー(閲覧)| ◯ | - |

「`member`+(該当 PJ)」は該当プロジェクトに対するロールが `member` または `admin` であることを示す(オーナーは bypass)。「`admin`+(該当 PJ)」は該当プロジェクトに対するロールが `admin` のみを示す。
