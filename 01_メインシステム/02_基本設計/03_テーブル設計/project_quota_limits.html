TBL-024 project_quota_limits

質問数の月次上限件数・無料枠・アラート、および運営者が管理する FAQ 件数無料枠をプロジェクト単位で保持します。

本テーブルは表番号 #24(§2 全テーブル一覧)。掲載位置は他テーブルとの差分を最小化するため §3.23 直後に置く。

項目 内容

論理名 プロジェクト別利用上限・無料枠

カラム定義

No 論理名 物理名 データ型 桁数 NULL PK FK UNIQUE DEFAULT 説明

2 プロジェクト ID project_id TEXT - NO ○ ① projects(id) 参照

制約

No 制約名 種別 対象カラム 条件・内容
1 (暗黙) CHECK resource_kind resource_kind IN ('question_monthly_limit','faq_monthly_limit')
2 (暗黙) CHECK alert_thresholds json_valid(alert_thresholds) AND json_type(alert_thresholds) = 'array'。要素の許可値・重複排除・昇順化はAPI層で保証
3 (暗黙) CHECK source source IN ('owner','operator')
4 (暗黙) CHECK valid valid IN (0,1)
5 (暗黙) 複合CHECK 上限対象 (resource_kind = 'question_monthly_limit' AND (threshold IS NOT NULL OR alert_thresholds = '[]')) OR (resource_kind = 'faq_monthly_limit' AND threshold IS NULL AND alert_thresholds = '[]')
6 (暗黙) 複合CHECK 管理主体 resource_kind = 'question_monthly_limit' OR source = 'operator'。FAQに source='owner' の行を許可しない

既存データ移行時は alert_enabled=0 を []、alert_enabled=1 AND alert_frequency='first' を [80]、alert_enabled=1 AND alert_frequency='staged' を [80,100] へ変換する。移行後は alert_thresholds のみを参照する。

質問数の無料枠と月次上限は独立した境界として扱うため、free_quota <= threshold は制約としない。これにより、利用者は非表示の無料枠値に依存せず、無料枠より低い月次上限も設定できる。

コード値・区分値

カラム名 値 意味 デフォルト推奨値(プロジェクト単位)
resource_kind question_monthly_limit 質問数の月次上限件数 上限=KV usage-limit:default:question / 無料枠=KV usage-limit:free-default:question(初期 1,000)
resource_kind faq_monthly_limit FAQ 件数の無料枠(コード値は互換性のため維持、月次上限なし) 無料枠=KV usage-limit:free-default:faq(初期 100)

質問数上限の最小 / 最大件数は KV usage-limit:min / usage-limit:max を入力ガードに用いる。FAQには上限値を保持しない。KV ミラーは usage-limit:<contract_owner_user_id>:<project_id>(30s TTL)。
