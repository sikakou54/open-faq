TBL-024 project_quota_limits

質問数の月次上限件数・無料枠・アラート、および運営者が管理する FAQ 件数無料枠をプロジェクト単位で保持します。

本テーブルは表番号 #24(§2 全テーブル一覧)。掲載位置は他テーブルとの差分を最小化するため §3.23 直後に置く。

概要

項目 内容
テーブル名 project_quota_limits
論理名 プロジェクト別利用上限・無料枠
主キー id
外部キー
  • project_id → projects(id)
  • contract_owner_user_id → contract_owners(user_id)

カラム定義

No 論理名 物理名 データ型 桁数 NULL PK FK UNIQUE DEFAULT 制約
1 ID id TEXT - NO ○ （複合）(resource_kind = 'question_monthly_limit' AND (threshold IS NOT NULL OR alert_thresholds = '[]')) OR (resource_kind = 'faq_monthly_limit' AND threshold IS NULL AND alert_thresholds = '[]')
（複合）resource_kind = 'question_monthly_limit' OR source = 'operator'。FAQに source='owner' の行を許可しない

3 契約 owner ID contract_owner_user_id TEXT - NO contract_owners(user_id)

5 月次上限件数 threshold INTEGER - YES
6 月次無料枠 free_quota INTEGER - YES
7 アラート閾値 alert_thresholds TEXT - NO '[]' json_valid(alert_thresholds) AND json_type(alert_thresholds) = 'array'。要素の許可値・重複排除・昇順化はAPI層で保証
8 設定元 source TEXT - NO ① 'owner' source IN ('owner','operator')
9 理由 reason TEXT - YES
10 有効期限 valid_until TEXT - YES
11 有効フラグ valid INTEGER - NO 1 valid IN (0,1)
12 設定者 created_by TEXT - YES

14 更新日時 updated_at TEXT - NO

インデックス

No インデックス名 対象カラム UNIQUE 用途
1 uq_pql_project_kind_source (project_id, resource_kind, source) WHERE valid = 1 ○ プロジェクト × 課金対象 × 設定元で一意(有効行のみ)
2 idx_pql_project_kind (project_id, resource_kind) プロジェクト別利用設定解決

コード値・区分値

カラム名 値 意味 デフォルト推奨値(プロジェクト単位)
resource_kind question_monthly_limit 質問数の月次上限件数 上限=KV usage-limit:default:question / 無料枠=KV usage-limit:free-default:question(初期 1,000)
resource_kind faq_monthly_limit FAQ 件数の無料枠 無料枠=KV usage-limit:free-default:faq(初期 100)

質問数上限の最小 / 最大件数は KV usage-limit:min / usage-limit:max を入力ガードに用いる。FAQには上限値を保持しない。KV ミラーは usage-limit:<contract_owner_user_id>:<project_id>(30s TTL)。
