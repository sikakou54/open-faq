TBL-018 audit_logs

メイン管理者向けの API 操作ログを保持します。

概要

項目 内容
テーブル名 audit_logs
論理名 監査ログ(メイン側、API 操作ログ専用)
保持期間 / 保持起点 retention_class 別(general 1y / billing 7y / operator_high_priv 5y)/ 操作日時

カラム定義

No 論理名 物理名 データ型 桁数 NULL PK FK UNIQUE DEFAULT 説明
1 ID id TEXT - NO ○ ULID
2 契約 owner ID contract_owner_user_id TEXT - YES ○ contract_owners(user_id) 参照
3 操作者(管理者) actor_user_id TEXT - YES ○ users(id) 参照(actor_role='admin' 時)
4 操作者ロール actor_role TEXT - YES §3.18.7
5 アクション action TEXT - NO アクションコード
6 対象種別 target_type TEXT - YES -
7 対象 ID target_id TEXT - YES -
8 IP(マスク済) ip_address_masked TEXT - YES 末尾オクテットマスク
9 ユーザーエージェント user_agent TEXT - YES -
10 メタデータ metadata TEXT - YES JSON 文字列(差分・補助情報)
11 保持クラス retention_class TEXT - NO 'general' general / billing / operator_high_priv
12 作成日時 created_at TEXT - NO ISO 8601 UTC

主キー

項目 内容
主キー名 (暗黙、PRIMARY KEY 句で指定)
対象カラム id

外部キー

No 外部キー名 対象カラム 参照先テーブル 参照先カラム 更新時 削除時
1 (暗黙) contract_owner_user_id contract_owners user_id NO ACTION SET NULL
2 (暗黙) actor_user_id users id NO ACTION SET NULL

インデックス

No インデックス名 対象カラム UNIQUE 用途
1 idx_audit_owner_action_created (contract_owner_user_id, action, created_at DESC) オーナー × アクション × 新着
2 idx_audit_retention_created (retention_class, created_at) 保持期間バッチ用
3 idx_audit_actor_created (actor_user_id, created_at DESC) 管理者操作の操作者別新着

制約

No 制約名 種別 対象カラム 条件・内容
1 (暗黙) CHECK actor_role actor_role IS NULL OR actor_role IN ('admin','system','service_operator')
2 (暗黙) CHECK retention_class retention_class IN ('general','billing','operator_high_priv')

コード値・区分値

カラム名 値 意味
actor_role admin 利用者管理者
actor_role system システム(Cron / バッチ)
actor_role service_operator 運営者(IF 経由のメイン API 呼出)

ハッシュチェーン列(prev_hash / current_hash / tombstone)は持たない。チェーン正本は運営者側。詳細は §2.2 / 09_セキュリティ設計.html §7 参照。
