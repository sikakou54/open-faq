TBL-002 contract_owners

オーナー専有属性(contract_status / data_deletion_mode 等)を保持する契約者テーブルです。

概要

項目 内容
テーブル名 contract_owners
論理名 契約者ユーザー(オーナー専有属性)
保持期間 / 保持起点 永久(契約存続中)/ 作成日時
contract_owner_user_id 本テーブル自身が契約境界(user_id = contract_owner_user_id)
retention_class billing

カラム定義

No 論理名 物理名 データ型 桁数 NULL PK FK UNIQUE DEFAULT 説明
1 ユーザー ID user_id TEXT - NO ○ ○ users(id) への 1:1 参照
2 契約状態 contract_status TEXT - NO 'active' active / suspended / deleted_pending / deleted(共有概念正本)
3 データ削除モード data_deletion_mode TEXT - NO 'physical' physical / anonymize
4 有効フラグ valid INTEGER - NO 1 論理削除フラグ
5 作成日時 created_at TEXT - NO ISO 8601 UTC
6 更新日時 updated_at TEXT - NO ISO 8601 UTC

主キー

項目 内容
主キー名 (暗黙、PRIMARY KEY 句で指定)
対象カラム user_id

外部キー

No 外部キー名 対象カラム 参照先テーブル 参照先カラム 更新時 削除時
1 (暗黙) user_id users id NO ACTION CASCADE

インデックス

No インデックス名 対象カラム UNIQUE 用途
1 idx_contract_owners_status contract_status 状態別オーナー抽出
2 idx_contract_owners_valid valid WHERE valid = 0 論理削除対象抽出

制約

No 制約名 種別 対象カラム 条件・内容
1 (暗黙) CHECK contract_status contract_status IN ('active','suspended','deleted_pending','deleted')
2 (暗黙) CHECK data_deletion_mode data_deletion_mode IN ('physical','anonymize')
3 (暗黙) CHECK valid valid IN (0,1)
4 (暗黙) 複合CHECK contract_status × valid contract_status='deleted' のとき必ず valid=0(両者は同時セット)。(contract_status <> 'deleted' AND valid = 1) OR (contract_status = 'deleted' AND valid = 0)

コード値・区分値

カラム名 値 意味
contract_status active 有効(共有概念正本)
contract_status suspended サスペンション中
contract_status deleted_pending 退会申請中(90 日猶予)
contract_status deleted 削除済み(オーナー退会完了。valid=0 も同時セット = CHECK 制約で強制)
data_deletion_mode physical 物理削除(初期値)
data_deletion_mode anonymize 匿名化

詳細状態遷移は 10_課金・請求設計.html §4 を正本とする。contract_status × valid の整合は §4.5.5 参照。
