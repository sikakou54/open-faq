TBL-015 notification_logs

メール通知の送信履歴を保持します。

概要

項目 内容
テーブル名 notification_logs
論理名 通知ログ
保持期間 / 保持起点 1 年 / 送信日時

カラム定義

No 論理名 物理名 データ型 桁数 NULL PK FK UNIQUE DEFAULT 説明
1 ID id TEXT - NO ○ ULID
2 契約 owner ID contract_owner_user_id TEXT - NO ○ contract_owners(user_id) 参照
3 案件 ID inquiry_id TEXT - YES ○ inquiries(id) 参照(削除でも履歴維持)
4 受信者メール HMAC recipient_email_hmac TEXT - YES HMAC-SHA256
5 通知種別 notification_type TEXT - NO TPL-* テンプレ ID
6 配信状態 delivery_state TEXT - NO §3.15.7
7 メッセージ ID message_id TEXT - YES Resend Message-ID
8 試行回数 attempt_count INTEGER - NO 0 当方再送回数
9 送信日時 sent_at TEXT - YES -
10 配信日時 delivered_at TEXT - YES Webhook 経由
11 失敗日時 failed_at TEXT - YES -
12 失敗理由 fail_reason TEXT - YES -
13 作成日時 created_at TEXT - NO ISO 8601 UTC

主キー

項目 内容
主キー名 (暗黙、PRIMARY KEY 句で指定)
対象カラム id

外部キー

No 外部キー名 対象カラム 参照先テーブル 参照先カラム 更新時 削除時
1 (暗黙) contract_owner_user_id contract_owners user_id NO ACTION CASCADE
2 (暗黙) inquiry_id inquiries id NO ACTION SET NULL

インデックス

No インデックス名 対象カラム UNIQUE 用途
1 idx_nlog_owner_created (contract_owner_user_id, created_at DESC) オーナー別新着
2 idx_nlog_message_id message_id Webhook 紐付け
3 idx_nlog_state delivery_state 状態別集計

制約

No 制約名 種別 対象カラム 条件・内容
1 (暗黙) CHECK delivery_state delivery_state IN ('queued','sending','sent','delivered','failed','bounced','complained','suppressed')

コード値・区分値

カラム名 値 意味
delivery_state queued Queue 投入済み
delivery_state sending 送信中
delivery_state sent 送信成功
delivery_state delivered 配信成功(Resend Webhook 経由)
delivery_state failed 失敗(当方再送対象、最大 3 回)
delivery_state bounced バウンス(ハードまたはソフト 5 連続)
delivery_state complained スパム報告
delivery_state suppressed サプレスリスト追加済み(終了状態)

状態遷移は §4.5.4 参照。
