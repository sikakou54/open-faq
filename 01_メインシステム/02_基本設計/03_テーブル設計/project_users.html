TBL-003 project_users

オーナー / メンバーのプロジェクトへの割当とプロジェクト別ロール(admin / member)を保持します。

概要

項目 内容
テーブル名 project_users
論理名 プロジェクトユーザー割当 + プロジェクト別ロール
保持期間 / 保持起点 紐づくユーザーに従属 / -
contract_owner_user_id あり(契約境界クエリ高速化と意図明示)
retention_class general

カラム定義

No 論理名 物理名 データ型 桁数 NULL PK FK UNIQUE DEFAULT 説明
1 ユーザー ID user_id TEXT - NO ① ○ 複合 PK / users(id) 参照
2 プロジェクト ID project_id TEXT - NO ② ○ 複合 PK / projects(id) 参照
3 契約境界 owner contract_owner_user_id TEXT - NO ○ 認可境界クエリ高速化(projects.contract_owner_user_id と必ず一致、アプリ層担保)
4 ロール role TEXT - NO admin / member
5 有効フラグ valid INTEGER - NO 1 論理削除フラグ
6 付与日時 granted_at TEXT - NO ISO 8601 UTC
7 付与者 granted_by TEXT - YES ○ users(id) 参照(招待者削除でも整合維持)
8 更新日時 updated_at TEXT - NO ISO 8601 UTC

主キー

項目 内容
主キー名 (暗黙、PRIMARY KEY 句で指定)
対象カラム (user_id, project_id)(複合)

外部キー

No 外部キー名 対象カラム 参照先テーブル 参照先カラム 更新時 削除時
1 (暗黙) user_id users id NO ACTION CASCADE
2 (暗黙) project_id projects id NO ACTION CASCADE
3 (暗黙) contract_owner_user_id contract_owners user_id NO ACTION CASCADE
4 (暗黙) granted_by users id NO ACTION SET NULL

インデックス

No インデックス名 対象カラム UNIQUE 用途
1 idx_project_users_project project_id プロジェクト割当一覧
2 idx_project_users_owner contract_owner_user_id 契約境界クエリ
3 idx_project_users_role_admin user_id WHERE role = 'admin' AND valid = 1 プロジェクト管理者の高速検索(critical 通知宛先解決)
4 idx_project_users_valid valid WHERE valid = 0 論理削除対象抽出

制約

No 制約名 種別 対象カラム 条件・内容
1 (暗黙) CHECK role role IN ('admin','member')
2 (暗黙) CHECK valid valid IN (0,1)

コード値・区分値

カラム名 値 意味
role admin プロジェクト管理者(該当プロジェクトの FAQ 管理 / 要対応質問の状況管理 / ログ参照 / メンバー管理 = 招待・プロジェクト割当解除・ロール変更)
role member メンバー(該当プロジェクトの FAQ 管理 / 要対応質問の状況管理 / ログ参照)

詳細は 04_権限設計.html §3 を正本とする。
