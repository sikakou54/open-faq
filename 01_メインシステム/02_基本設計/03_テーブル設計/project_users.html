TBL-003 project_users

オーナー・メンバーのプロジェクトへの割当と、プロジェクトごとのロール(プロジェクト管理者 / メンバー)を保持します。

概要

項目 内容
テーブル名 project_users
論理名 プロジェクトユーザー割当 + プロジェクト別ロール
主キー (user_id, project_id)
外部キー user_id → users(id), project_id → projects(id), contract_owner_user_id → contract_owners(user_id), granted_by → users(id)

カラム定義

No 論理名 物理名 データ型 桁数 NULL PK FK UNIQUE DEFAULT 制約
1 ユーザー ID user_id TEXT - NO ① users(id)
2 プロジェクト ID project_id TEXT - NO ② projects(id)
3 契約境界 owner contract_owner_user_id TEXT - NO contract_owners(user_id)
4 ロール role TEXT - NO role IN ('admin','member')
5 有効フラグ valid INTEGER - NO 1 valid IN (0,1)
6 付与日時 granted_at TEXT - NO
7 付与者 granted_by TEXT - YES users(id)
8 更新日時 updated_at TEXT - NO

インデックス

No インデックス名 対象カラム UNIQUE 用途
1 idx_project_users_project project_id プロジェクト割当一覧
2 idx_project_users_owner contract_owner_user_id 契約境界クエリ
3 idx_project_users_role_admin user_id WHERE role = 'admin' AND valid = 1 プロジェクト管理者の高速検索(critical 通知宛先解決)
4 idx_project_users_valid valid WHERE valid = 0 論理削除対象抽出

コード値・区分値

カラム名 値 意味
role admin プロジェクト管理者(該当プロジェクトの FAQ 管理 / 要対応質問の状況管理 / ログ参照 / メンバー管理 = 招待・プロジェクト割当解除・ロール変更)
role member メンバー(該当プロジェクトの FAQ 管理 / 要対応質問の状況管理 / ログ参照)
