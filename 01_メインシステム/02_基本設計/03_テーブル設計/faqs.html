TBL-009 faqs

FAQ 本体(質問・回答・公開状態)を保持します。

概要

項目 内容
テーブル名 faqs
論理名 FAQ
説明 FAQ 本体
保持期間 / 保持起点 永久(FAQ 存続中) / 作成日時
contract_owner_user_id あり
retention_class general

カラム定義

No 論理名 物理名 データ型 桁数 NULL PK FK UNIQUE DEFAULT 説明
1 ID id TEXT - NO ○ ULID
2 契約 owner ID contract_owner_user_id TEXT - NO ○ contract_owners(user_id) 参照
3 プロジェクト ID project_id TEXT - NO ○ projects(id) 参照
4 質問(タイトル) title TEXT - NO FAQ 質問。1〜500 文字(FR-040)
5 回答(本文) body TEXT - NO FAQ 回答。1〜5000 文字(FR-040)
6 カテゴリ category TEXT - YES 任意
7 状態 status TEXT - NO 'draft' draft / published / hidden / deleted
8 バージョン version INTEGER - NO 1 改訂版数
9 タグ tags TEXT - YES JSON 配列(最大 10 要素、各 30 文字以下、API 層でバリデーション)
10 元未解決質問 ID source_unresolved_question_id TEXT - YES ○ inquiries(id) 参照
11 有効フラグ valid INTEGER - NO 1 論理削除フラグ
12 作成者 created_by TEXT - YES ○ users(id) 参照(削除でも整合維持)
13 更新者 updated_by TEXT - YES ○ users(id) 参照(同上)
14 作成日時 created_at TEXT - NO ISO 8601 UTC
15 更新日時 updated_at TEXT - NO ISO 8601 UTC

主キー

項目 内容
主キー名 (暗黙、PRIMARY KEY 句で指定)
対象カラム id

外部キー

No 外部キー名 対象カラム 参照先テーブル 参照先カラム 更新時 削除時
1 (暗黙) contract_owner_user_id contract_owners user_id NO ACTION CASCADE
2 (暗黙) project_id projects id NO ACTION CASCADE
3 (暗黙) source_unresolved_question_id inquiries id NO ACTION SET NULL
4 (暗黙) created_by users id NO ACTION SET NULL
5 (暗黙) updated_by users id NO ACTION SET NULL

インデックス

No インデックス名 対象カラム UNIQUE 用途
1 idx_faqs_owner_project (contract_owner_user_id, project_id) 契約境界 + プロジェクト
2 idx_faqs_project_status (project_id, status) プロジェクト × 状態
3 idx_faqs_source_inquiry source_unresolved_question_id 元未解決質問逆引き
4 idx_faqs_category (project_id, category) WHERE category IS NOT NULL カテゴリ別
5 idx_faqs_valid valid WHERE valid = 0 論理削除対象抽出

制約

No 制約名 種別 対象カラム 条件・内容
1 (暗黙) CHECK title length(title) BETWEEN 1 AND 500(FAQ 質問、FR-040)
2 (暗黙) CHECK body length(body) BETWEEN 1 AND 5000(FAQ 回答、FR-040)
3 (暗黙) CHECK status status IN ('draft','published','hidden','deleted')
4 (暗黙) CHECK valid valid IN (0,1)

コード値・区分値

カラム名 値 意味
status draft 下書き
status published 公開中
status hidden 非公開
status deleted 論理削除(valid=0 と同時セット。復元は 運営者のみ = IF #4)

状態遷移詳細は §4.5.1 参照。
