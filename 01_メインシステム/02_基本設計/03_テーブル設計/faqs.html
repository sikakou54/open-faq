TBL-009 faqs

FAQ 本体(質問・回答・公開状態)を保持します。

概要

項目 内容
テーブル名 faqs
論理名 FAQ
主キー id
外部キー
  • contract_owner_user_id → contract_owners(user_id)
  • project_id → projects(id)
  • source_unresolved_question_id → inquiries(id)
  • created_by → users(id)
  • updated_by → users(id)

カラム定義

No 論理名 物理名 データ型 桁数 NULL PK FK UNIQUE DEFAULT 制約
1 ID id TEXT - NO ○
2 契約 owner ID contract_owner_user_id TEXT - NO contract_owners(user_id)
3 プロジェクト ID project_id TEXT - NO projects(id)
4 質問(タイトル) title TEXT - NO length(title) BETWEEN 1 AND 500(FAQ 質問、FR-040)
5 回答(本文) body TEXT - NO length(body) BETWEEN 1 AND 5000(FAQ 回答、FR-040)
6 カテゴリ category TEXT - YES
7 状態 status TEXT - NO 'draft' status IN ('draft','published','hidden','deleted')
8 バージョン version INTEGER - NO 1
9 タグ tags TEXT - YES
10 元未解決質問 ID source_unresolved_question_id TEXT - YES inquiries(id)
11 有効フラグ valid INTEGER - NO 1 valid IN (0,1)
12 作成者 created_by TEXT - YES users(id)
13 更新者 updated_by TEXT - YES users(id)
14 作成日時 created_at TEXT - NO
15 更新日時 updated_at TEXT - NO

インデックス

No インデックス名 対象カラム UNIQUE 用途
1 idx_faqs_owner_project (contract_owner_user_id, project_id) 契約境界 + プロジェクト
2 idx_faqs_project_status (project_id, status) プロジェクト × 状態
3 idx_faqs_source_inquiry source_unresolved_question_id 元未解決質問逆引き
4 idx_faqs_category (project_id, category) WHERE category IS NOT NULL カテゴリ別
5 idx_faqs_valid valid WHERE valid = 0 論理削除対象抽出

コード値・区分値

カラム名 値 意味
status draft 下書き
status published 公開中
status hidden 非公開
status deleted 論理削除
