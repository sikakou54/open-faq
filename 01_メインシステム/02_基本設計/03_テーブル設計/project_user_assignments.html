TBL-003 project_user_assignments

プロジェクトユーザーのプロジェクトへの割当と、プロジェクトごとのロール(プロジェクト管理者 / メンバー)を保持します。割当対象は project_users マスタのユーザーです。

概要

項目 内容
テーブル名 project_user_assignments
論理名 プロジェクト割当 + プロジェクト別ロール
主キー
  • project_user_id
  • project_id
外部キー
  • project_user_id → project_users(id)
  • project_id → projects(id)
  • contract_owner_user_id → owners(id)
  • granted_by_type + granted_by_id → owners(id) / project_users(id)(論理参照)

カラム定義

No 論理名 物理名 データ型 桁数 NULL PK FK UNIQUE DEFAULT 制約
1 プロジェクトユーザー ID project_user_id TEXT - NO ① project_users(id)
2 プロジェクト ID project_id TEXT - NO ② projects(id)
3 契約境界 owner ID contract_owner_user_id TEXT - NO owners(id)
4 ロール role TEXT - NO role IN ('admin','member')
5 有効フラグ valid INTEGER - NO 1 valid IN (0,1)
6 付与日時 granted_at TEXT - NO
7 付与者種別 granted_by_type TEXT - YES granted_by_type IS NULL OR granted_by_type IN ('owner','project_user')
8 付与者 ID granted_by_id TEXT - YES granted_by_type に応じ owners(id) / project_users(id) を指す(論理参照)
9 更新日時 updated_at TEXT - NO

インデックス

No インデックス名 対象カラム UNIQUE 用途
1 idx_pua_project project_id プロジェクト割当一覧
2 idx_pua_owner contract_owner_user_id 契約境界クエリ
3 idx_pua_role_admin project_user_id WHERE role = 'admin' AND valid = 1 プロジェクト管理者の高速検索(critical 通知宛先解決)
4 idx_pua_valid valid WHERE valid = 0 論理削除対象抽出

コード値・区分値

カラム名 値 意味
role admin プロジェクト管理者
role member メンバー
granted_by_type owner 付与者がオーナー(owners(id) 参照)
granted_by_type project_user 付与者がプロジェクトユーザー(project_users(id) 参照)
