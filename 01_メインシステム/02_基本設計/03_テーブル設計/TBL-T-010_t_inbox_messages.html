TBL-T-010 T_inbox_messages

利用者(テナント側)が受け取るお知らせ・通知の既読状態を管理する受信箱です。

概要

項目 内容
テーブル名 T_inbox_messages
論理名 受信箱(Tenant Plane)
主キー id
外部キー
  • contract_owner_user_id → M_owners(id)
  • recipient_type + recipient_id → M_owners(id) / M_project_users(id)(論理参照)

カラム定義

No 論理名 物理名 データ型 桁数 NULL PK FK UNIQUE DEFAULT 制約
1 ID id TEXT - NO ○
2 契約 owner ID contract_owner_user_id TEXT - NO M_owners(id)
3 受信者種別 recipient_type TEXT - NO recipient_type IN ('owner','project_user')
4 受信者 ID recipient_id TEXT - NO recipient_type に応じ M_owners(id) / M_project_users(id) を指す(論理参照)
5 カテゴリ category TEXT - NO category IN ('billing','announcement','system')
6 優先度 priority TEXT - NO priority IN ('low','normal','high','critical')
7 タイトル title TEXT - NO length(title) BETWEEN 1 AND 200
8 本文(HTML) body_html TEXT - NO
9 由来種別 source_kind TEXT - YES
10 由来 ID source_id TEXT - YES
11 重複キー dedup_key TEXT - NO
12 既読日時 read_at TEXT - YES
13 非表示日時 hidden_at TEXT - YES
14 作成日時 created_at TEXT - NO

インデックス

No インデックス名 対象カラム UNIQUE 用途
1 idx_inbox_owner_user_unread (contract_owner_user_id, recipient_type, recipient_id, read_at, created_at DESC) 未読 + 新着
2 idx_inbox_dedup (contract_owner_user_id, dedup_key, created_at) 60 分集約判定

コード値・区分値

カラム名 値 意味
category billing 課金
category announcement お知らせ
category system システム
priority low 低
priority normal 通常
priority high 高
priority critical クリティカル
