TBL-001 users

管理者(オーナー / プロジェクト管理者 / メンバー)の認証情報(メール / パスワード / 氏名 / セッション状態)を一元保持するユーザーマスタです。

概要

項目 内容
テーブル名 users
論理名 ユーザーマスタ(管理者専用)
保持期間 / 保持起点 永久(契約存続中)/ 作成日時

カラム定義

No 論理名 物理名 データ型 桁数 NULL PK FK UNIQUE DEFAULT 説明
1 ID id TEXT - NO ○ ULID 主キー
2 メール(暗号化) email_encrypted TEXT - NO AES-256-GCM、オーナー派生鍵
3 メール HMAC email_hmac TEXT - NO 検索用 HMAC-SHA256
4 鍵バージョン email_key_version INTEGER - NO 1 dual-decrypt 期間中の鍵バージョン
5 パスワードハッシュ password_hash TEXT - YES bcrypt / Argon2id
6 氏名 name TEXT - YES 表示名。招待時は NULL で作成され、招待された本人が SCR-018 メンバーアカウント有効化ページで入力した時点で設定される(FR-016d / FR-016e、個人情報原則により管理者は他人の氏名を入力できない)。本人による変更は SCR-018 / SCR-017 個人設定で可能
7 状態 status TEXT - NO 'pending_verification' pending_verification / pending_activation / active
8 有効フラグ valid INTEGER - NO 1 論理削除フラグ(0=削除済 / 1=有効)
9 最終ログイン日時 last_login_at TEXT - YES ISO 8601 UTC
10 メール確認日時 email_verified_at TEXT - YES ISO 8601 UTC
11 作成日時 created_at TEXT - NO ISO 8601 UTC
12 更新日時 updated_at TEXT - NO ISO 8601 UTC(論理削除日時を代用)

主キー

項目 内容
主キー名 (暗黙、PRIMARY KEY 句で指定)
対象カラム id

外部キー

(なし)

インデックス

No インデックス名 対象カラム UNIQUE 用途
1 idx_users_email_hmac email_hmac メール検索
2 idx_users_valid valid WHERE valid = 0 論理削除対象抽出(物理削除バッチ)

制約

No 制約名 種別 対象カラム 条件・内容
1 (暗黙) CHECK status status IN ('pending_verification','pending_activation','active')
2 (暗黙) CHECK valid valid IN (0,1)

コード値・区分値

カラム名 値 意味
status pending_verification メール未確認
status pending_activation 招待中(SCR-018 メンバーアカウント有効化ページで氏名 + 初回パスワード + 規約同意 未完了)。users.name=NULL で作成され、SCR-018 完了時に users.name 設定 + status='active' に遷移する(FR-016e)
status active 有効
valid 1 有効
valid 0 論理削除済み(論理削除日時は updated_at で判別。updated_at < now() - 90d AND valid=0 の行は物理削除バッチで物理削除される)

ウィジェット利用者は本テーブルに含まれず、永続アカウントを作成しない。
