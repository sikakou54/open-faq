TBL-M-002 M_PRJ_USERS

プロジェクト管理者・メンバー(オーナー以外のログインユーザー)の認証情報(メール / パスワード / 氏名 / セッション状態)を保持するプロジェクトユーザーマスタです。オーナーは含まず、所属契約を contract_owner_user_id(M_OWNERS(id) 参照)で保持します。

概要

項目 内容
テーブル名 M_PRJ_USERS
論理名 プロジェクトユーザーマスタ
主キー id
外部キー
  • contract_owner_user_id → M_OWNERS(id)

カラム定義

No 論理名 物理名 データ型 桁数 NULL PK FK UNIQUE DEFAULT 制約
1 ID id TEXT - NO ○
2 契約境界 owner ID contract_owner_user_id TEXT - NO M_OWNERS(id)
3 メール(暗号化) email_encrypted TEXT - NO
4 メール HMAC email_hmac TEXT - NO
5 鍵バージョン email_key_version INTEGER - NO 1
6 パスワードハッシュ password_hash TEXT - YES
7 氏名 name TEXT - YES
8 状態 status TEXT - NO 'pending_verification' status IN ('pending_verification','pending_activation','active')
9 有効フラグ valid INTEGER - NO 1 valid IN (0,1)
10 最終ログイン日時 last_login_at TEXT - YES
11 メール確認日時 email_verified_at TEXT - YES
12 作成日時 created_at TEXT - NO
13 更新日時 updated_at TEXT - NO

インデックス

No インデックス名 対象カラム UNIQUE 用途
1 idx_project_users_email_hmac email_hmac メール検索
2 idx_project_users_owner contract_owner_user_id 契約境界クエリ
3 idx_project_users_valid valid WHERE valid = 0 論理削除対象抽出(物理削除バッチ)

コード値・区分値

カラム名 値 意味
status pending_verification メール未確認
status pending_activation 招待中。name=NULL で作成され、有効化完了時に name 設定 + status='active' に遷移する
status active 有効
valid 1 有効
valid 0 論理削除済み

オーナーは本テーブルに含まれず、M_OWNERS マスタで管理する。プロジェクトへの割当とロール(プロジェクト管理者 / メンバー)は M_PRJ_USER_ASGN で保持する。
