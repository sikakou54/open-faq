TBL-014 inquiries

FAQ 登録前の未解決質問を保持します。

概要

項目 内容
テーブル名 inquiries
論理名 未解決質問
主キー id
外部キー contract_owner_user_id, project_id, question_log_id

カラム定義

No 論理名 物理名 データ型 桁数 NULL PK FK UNIQUE DEFAULT 制約
1 ID id TEXT - NO ○
2 契約 owner ID contract_owner_user_id TEXT - NO ○
3 プロジェクト ID project_id TEXT - NO ○
4 問い合わせコード inquiry_code TEXT - NO ○
5 元質問ログ ID question_log_id TEXT - YES ○
6 質問本文 user_question TEXT - NO
7 状況 status TEXT - NO 'open' status IN ('open','closed')
8 有効フラグ valid INTEGER - NO 1 valid IN (0,1)
9 作成日時 created_at TEXT - NO
10 更新日時 updated_at TEXT - NO

インデックス

No インデックス名 対象カラム UNIQUE 用途
1 idx_inquiries_contract_status_created (contract_owner_user_id, status, created_at DESC) 契約境界 + 状況別新着
2 idx_inquiries_project_status (project_id, status) プロジェクト × 状況
3 uq_inquiries_code inquiry_code ○ 問い合わせコード一意
4 idx_inquiries_valid valid WHERE valid = 0 論理削除対象抽出

コード値・区分値

カラム名 値 意味
status open 対応必要(初期値、または closed から再オープン)
status closed 終了(手動操作のみ)

状態遷移詳細は §4.5.2 参照(共有概念正本)。
