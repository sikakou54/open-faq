TBL-M-001 M_OWNERS

オーナー(契約あたり 1 ユーザー)の認証情報(メール / パスワード / 氏名 / セッション状態)と、契約状態・データ削除モードなどの契約属性を一元保持するオーナーマスタです。id が契約境界キー本体であり、外部キーを持ちません。

概要

項目 内容
テーブル名 M_OWNERS
論理名 オーナーマスタ
主キー id
外部キー —

カラム定義

No 論理名 物理名 データ型 桁数 NULL PK FK UNIQUE DEFAULT 制約
1 ID(契約境界キー) id TEXT - NO ○
2 メール(暗号化) email_encrypted TEXT - NO
3 メール HMAC email_hmac TEXT - NO
4 鍵バージョン email_key_version INTEGER - NO 1
5 パスワードハッシュ password_hash TEXT - YES
6 氏名 name TEXT - YES
7 状態 status TEXT - NO 'pending_verification' status IN ('pending_verification','pending_activation','active')
8 契約状態 contract_status TEXT - NO 'active' contract_status IN ('active','suspended','deleted_pending','deleted')
9 データ削除モード data_deletion_mode TEXT - NO 'physical' data_deletion_mode IN ('physical','anonymize')
10 有効フラグ valid INTEGER - NO 1 valid IN (0,1)
11 最終ログイン日時 last_login_at TEXT - YES
12 メール確認日時 email_verified_at TEXT - YES
13 作成日時 created_at TEXT - NO
14 更新日時 updated_at TEXT - NO

インデックス

No インデックス名 対象カラム UNIQUE 用途
1 idx_owners_email_hmac email_hmac メール検索
2 idx_owners_contract_status contract_status 状態別オーナー抽出
3 idx_owners_valid valid WHERE valid = 0 論理削除対象抽出(物理削除バッチ)

コード値・区分値

カラム名 値 意味
status pending_verification メール未確認
status pending_activation 有効化待ち
status active 有効
contract_status active 有効
contract_status suspended サスペンション中
contract_status deleted_pending 退会申請中
contract_status deleted 削除済み
data_deletion_mode physical 物理削除
data_deletion_mode anonymize 匿名化
valid 1 有効
valid 0 論理削除済み

ウィジェット利用者は本テーブルに含まれず、永続アカウントを作成しない。プロジェクト管理者・メンバーは M_PRJ_USERS マスタに保持し、本テーブルには含まれない。
