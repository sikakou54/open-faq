API-MBR-003 メンバーロール変更

この API は、当該プロジェクトの project_users.role を上書きする(他プロジェクトに影響なし)。

版数 v1.6更新 2026-06-17承認済

基本情報

項目内容
API IDAPI-MBR-003
API 名プロジェクト単位のロール変更(SCR-009-001 編集モード正本)
エンドポイント/projects/{id}/members/{userId}
HTTP メソッドPATCH
概要当該プロジェクトの project_users.role を上書きする(他プロジェクトに影響なし)
利用画面SCR-009-001(編集モードの「変更を保存」)
認証Cookie + CSRF + 再認証必須
権限オーナー / 該当プロジェクトの admin ロール保持メンバー

リクエストパラメータ

Path Parameters: {id}(プロジェクト ID)、{userId}(対象ユーザー ID)。Request Body:

{
  "role": "admin" | "member"
}
  • 同時に表示名(displayName)を更新する場合は別途 PATCH /members/{id} を呼び出す(プロジェクト範囲外の属性のため)
  • 「該当プロジェクトの最後の admin を member へ降格」する結果になる場合は 403 LAST_ADMIN_PROTECTED(E-AUTHZ-LAST-ADMIN-PROTECTED)で拒否
  • 自分自身が当該プロジェクトの最後の admin で、自身を member へ降格する場合は 403 SELF_MUTATION_FORBIDDEN
  • ロール変更後も当該プロジェクトに project_users 行が維持される限り、normal / low 通知の配信対象に含まれる(配信先はトグル ON 時にプロジェクトの全メンバー)

レスポンス

(200):

{ "userId": "...", "projectId": "...", "role": "admin|member", "updatedAt": "..." }

エラー

HTTP ステータスエラーコード内容
404NOT_FOUND対象が存在しない
403PROJECT_ACCESS_DENIED当該プロジェクトへの権限なし
403LAST_ADMIN_PROTECTED最後の admin を降格しようとした
403SELF_MUTATION_FORBIDDEN自身を不正に降格しようとした
400VALIDATION_ERROR入力値の検証エラー
