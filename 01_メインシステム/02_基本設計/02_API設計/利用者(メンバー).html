基本情報

{}

{
  "items": [
    {
      "id": "01HZ...",
      "email": "member@...",

API-MBR-002 メンバー招待

基本情報

項目内容
API IDAPI-MBR-002
API 名プロジェクト単発招待(SCR-009-001 招待モード正本)
エンドポイント/projects/{id}/members
HTTP メソッドPOST
概要該当プロジェクト 1 件に対する単発招待を行う
利用画面SCR-009-001(プロジェクト管理者の招待操作)
認証Cookie + CSRF + 再認証必須
権限オーナー / 該当プロジェクトの admin ロール保持メンバー

既存メンバーアカウントの場合は project_users 行のみ追加(指定ロール)。新規メールアドレスの場合は users を name=NULL / status='pending_activation' で作成 + project_users を valid=0 で同時 INSERT + 招待メール送信(7 日有効、access_tokens.purpose='activation'、meta={"invitedProjectId": "...", "invitedRole": "..."} を JSON 保持)。氏名(displayName)は受け付けない(招待された本人が SCR-018 メンバーアカウント有効化ページで入力する。FR-016d / FR-016e)。

リクエストパラメータ

Query Parameters

{}

Path Parameters

{ "id": "プロジェクト ID" }

Request Body

{
  "email": "newmember@...",
  "role": "admin"
}

レスポンス(201)

{ "userId": "01HZ...", "status": "active|pending_activation", "expiresAt": "..." | null }

エラー

HTTP ステータスエラーコード内容
409ALREADY_EXISTS_IN_PROJECT既に該当プロジェクトに割当あり
400VALIDATION_ERROR入力値の検証エラー
403PROJECT_ACCESS_DENIED当該プロジェクトへの権限なし

API-MBR-003 メンバーロール変更

基本情報

項目内容
API IDAPI-MBR-003
API 名プロジェクト単位のロール変更(SCR-009-001 編集モード正本)
エンドポイント/projects/{id}/members/{userId}
HTTP メソッドPATCH
概要当該プロジェクトの project_users.role を上書きする(他プロジェクトに影響なし)
利用画面SCR-009-001(編集モードの「変更を保存」)
認証Cookie + CSRF + 再認証必須
権限オーナー / 該当プロジェクトの admin ロール保持メンバー

リクエストパラメータ

Query Parameters

{}

Path Parameters

{
  "id": "プロジェクト ID",
  "userId": "対象ユーザー ID"
}

Request Body

{
  "role": "admin" | "member"
}
  • 同時に表示名(displayName)を更新する場合は別途 PATCH /members/{id} を呼び出す(プロジェクト範囲外の属性のため)
  • 「該当プロジェクトの最後の admin を member へ降格」する結果になる場合は 403 LAST_ADMIN_PROTECTED(E-AUTHZ-LAST-ADMIN-PROTECTED)で拒否
  • 自分自身が当該プロジェクトの最後の admin で、自身を member へ降格する場合は 403 SELF_MUTATION_FORBIDDEN
  • ロール変更後も当該プロジェクトに project_users 行が維持される限り、normal / low 通知の配信対象に含まれる(配信先はトグル ON 時にプロジェクトの全メンバー)

レスポンス(200)

{ "userId": "...", "projectId": "...", "role": "admin|member", "updatedAt": "..." }

エラー

HTTP ステータスエラーコード内容
404NOT_FOUND対象が存在しない
403PROJECT_ACCESS_DENIED当該プロジェクトへの権限なし
403LAST_ADMIN_PROTECTED最後の admin を降格しようとした
403SELF_MUTATION_FORBIDDEN自身を不正に降格しようとした
400VALIDATION_ERROR入力値の検証エラー

API-MBR-004 プロジェクト割当解除

基本情報

項目内容
API IDAPI-MBR-004
API 名プロジェクト割当解除
エンドポイント/projects/{id}/members/{userId}
HTTP メソッドDELETE
概要対象プロジェクトの project_users.valid=0 を更新する
利用画面SCR-009-001「プロジェクトから外す」(L1)
認証Cookie + CSRF + 再認証必須
権限オーナー / 該当プロジェクトの admin ロール保持メンバー

対象が自分自身またはオーナーの場合は拒否する。

  • 対象ユーザーに他プロジェクトの有効割当が残る場合: users、他プロジェクトの割当、セッション、招待トークンは維持する。
  • 他プロジェクトの有効割当が 0 件になる場合: 同一トランザクションで users.valid=0、全セッション失効、未使用招待トークン失効を実行する。

リクエストパラメータ

Query Parameters

{}

Path Parameters

{
  "id": "プロジェクト ID",
  "userId": "対象ユーザー ID"
}

Request Body

{}

レスポンス(200)

{ "userId": "...", "projectId": "...", "accountDeactivated": true|false }

エラー

HTTP ステータスエラーコード内容
403E-AUTHZ-OWNER-PROTECTEDオーナーは解除不可
403E-AUTHZ-SELF-MUTATION自分自身は解除不可
403PROJECT_ACCESS_DENIED当該プロジェクトへの権限なし
404NOT_FOUND対象が存在しない

API-MBR-005 招待メール再送

基本情報

項目内容
API IDAPI-MBR-005
API 名招待メール再送
エンドポイント/members/{id}/resend-invitation
HTTP メソッドPOST
概要status='pending_activation' のメンバーへ招待メールを再送する
利用画面SCR-009-001(招待再送)
認証Cookie + CSRF + 再認証必須
権限オーナー / 該当プロジェクトの admin 保持メンバー

旧 access_tokens.purpose='activation' を失効させ新規トークン(7 日有効、meta を同一の invitedProjectId / invitedRole で再発行)を発行。

リクエストパラメータ

Query Parameters

{}

Path Parameters

{ "id": "メンバーユーザー ID" }

Request Body

{}

氏名 displayName を受け付けない(FR-016b / FR-016d)。

レスポンス(200)

再発行結果({ "ok": true, "expiresAt": "..." })。

エラー

HTTP ステータスエラーコード内容
403E-AUTHZ-OWNER-PROTECTEDオーナーは対象外
403E-AUTHZ-SELF-MUTATION自分自身は対象外
404NOT_FOUND対象が存在しない
