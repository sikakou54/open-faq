API-LNK-001 IF #1 契約停止イベント受信(顧管 → メ)

顧客管理システムからの契約停止イベントを受信し、契約状態とセッションを反映します。

POST /internal/admin-integration/v1/owner/suspend

認証: mTLS + JWT(aud=main、TTL 5min)。

処理概要

顧客管理システムからの契約停止イベントを受信し、契約状態とセッションを反映します。

処理 ID処理内容
P-01JWT と Idempotency-Key を検証する
P-02contract_owners.contract_status を更新する
P-03KV キャッシュ(セッション / KV / レート制限)を無効化する
P-04reason に応じてセッション処理(operator_manual / tos_violation は 5 秒以内に全セッション無効化)を行う
P-05受領応答を返す

API-LNK-004 IF #5 レート制限上書き受信

契約単位のレート制限上書きを受信し、即時反映します。

POST /internal/admin-integration/v1/rate-limit/override

処理概要

契約単位のレート制限上書きを受信し、即時反映します。

処理 ID処理内容
P-01JWT と Idempotency-Key を検証する
P-02owner_quota_overrides(レート種別のみ・契約単位)を更新する
P-03KV キー ratelimit:{contractOwnerUserId}:{kind} を即時反映する

補足本 IF はレート制限(契約単位)専用です。月次上限件数の運営者上書きはプロジェクト単位のため API-055 を用います。

API-LNK-005 IF #5(月次上限件数上書き受信 / プロジェクト単位)

プロジェクト単位の月次上限件数上書き(4-eyes 承認済み)を受信し、即時反映します。

POST /internal/admin-integration/v1/usage-limit/override

処理概要

プロジェクト単位の月次上限件数上書き(4-eyes 承認済み)を受信し、即時反映します。

処理 ID処理内容
P-01JWT と Idempotency-Key を検証する(4-eyes 承認済みであること)
P-02project_quota_limits(source='operator'、対象プロジェクト単位)を更新する
P-03KV キー usage-limit:{contractOwnerUserId}:{projectId} を即時反映する(30s TTL)
P-04受領応答を返す

kind は question_monthly_limit / faq_monthly_limit。質問数は limit と freeQuota、FAQ は運営者が管理する freeQuota だけを受け付ける。FAQ の limit は常に NULL とし、指定時は 422。質問数の優先順位は「プロジェクト設定 > 運営者上書き > デフォルト推奨値」、FAQ 無料枠は「運営者上書き > デフォルト値」。

API-LNK-007 IF #7 お知らせ生成受信

運営者が生成したお知らせを受信し、配信先を解決して inbox へ fan-out します。

POST /internal/admin-integration/v1/announcement/inbound

処理概要

運営者が生成したお知らせを受信し、配信先を解決して inbox へ fan-out します。

処理 ID処理内容
P-01JWT と Idempotency-Key を検証する
P-02service_announcements レコードを作成する
P-03配信先指定がある場合は announcement_audiences 行を登録する(中間テーブル化)
P-04宛先範囲(全契約 / 列挙 owner)に応じて announcement_recipients を生成する
P-05fan-out queue 経由で inbox_messages へ配信する
P-06受領応答を返す

API-LNK-010 IF #10 課金 Webhook 受信(顧管 → メ)

運営者が一次受信した Stripe 課金イベントを受信し、請求 / 契約状態へ反映します(FR-139)。

処理概要

運営者が一次受信した Stripe 課金イベントを受信し、請求 / 契約状態へ反映します(FR-139)。

処理 ID処理内容
P-01JWT を検証し event_id で冪等性を担保する
P-02Stripe イベント種別に応じてハンドラを分岐し請求 / 契約状態へ反映する(invoice.paid / invoice.payment_failed / customer.subscription.created / charge.refunded 等)

P-04受領応答を返す

Idempotency-Key: event_id(Stripe evt_*)。DLQ 滞留上限: 1000/24h。
