API-FAQ-002 FAQ 作成・更新・削除

基本情報

項目内容
API IDAPI-FAQ-002
API 名FAQ 作成・更新・削除
エンドポイント/faqs, /faqs/{id}
HTTP メソッドPOST / PATCH / DELETE
概要FAQ の CRUD と状態(draft / published / hidden)の相互遷移
利用画面SCR-006
認証Cookie + CSRF
権限オーナー / 該当 PJ の member+

楽観ロック: version 必須。CONFLICT 時 409。

FAQ の状態(draft / published / hidden)は status フィールドで指定し、PATCH /faqs/{id} の保存操作で相互に自由遷移できる(専用の公開 API・状態遷移ガードは持たない)。SCR-006 の「保存」ボタンは「状態」ラジオで選択した値をそのまま本 API に渡す。status='published' への変更は管理者(admin)が編集画面で内容を確認したうえで明示選択する操作であり、これが FR-045「FAQ の公開前に管理者が内容を確認できること」を満たす。published 設定時は publishedAt をサーバ側で記録する。

リクエストパラメータ

Path Parameters は PATCH / DELETE で {id}(FAQ ID)。Request Body は FAQ 内容(question / answer / category / status / version 等。POST は新規)。

レスポンス

: 作成 / 更新後の FAQ(publishedAt を含む)。

エラー

HTTP ステータスエラーコード内容
409CONFLICT楽観ロック競合(version 不一致)
400VALIDATION_ERROR質問 500 字 / 回答 5,000 字超過ほか(§3)

API-FAQ-003 FAQ 一括状態変更

基本情報

項目内容
API IDAPI-FAQ-003
API 名FAQ 一括状態変更
エンドポイント/faqs/bulk-status
HTTP メソッドPOST
概要選択した複数 FAQ の状態を一括で変更する(1 リクエスト上限 50 件)
利用画面SCR-006 一覧 BulkActionBar
認証Cookie + CSRF
権限オーナー / 該当 PJ の member+

SCR-006 一覧の BulkActionBar(チェックボックス選択)から呼び出し、選択した複数 FAQ の状態を一括で draft / published / hidden に変更する。1 リクエストの上限は 50 件(SCR-006 の BulkActionBar 上限、01_画面設計/index.html)。超過時は 400 VALIDATION_ERROR。処理は行単位で評価し、対象外(他契約 / 論理削除済み)の ID は当該行のみ失敗として集計する(オーナー境界違反は §2.2 に従い結果から除外し成功扱いにしない)。published へ変更した行は publishedAt をサーバ側で記録する。

リクエストパラメータ

{ "ids": ["faq_...", "faq_..."], "status": "published" }

レスポンス

(200):

{ "succeeded": 48, "failed": [ { "id": "faq_...", "code": "NOT_FOUND" } ] }

エラー

HTTP ステータスエラーコード内容
400VALIDATION_ERROR50 件超過 / 不正な status
403PROJECT_ACCESS_DENIED当該プロジェクトへの権限なし

API-FAQ-004 FAQ CSV インポート

基本情報

項目内容
API IDAPI-FAQ-004
API 名FAQ CSV インポート
エンドポイント/faqs/import
HTTP メソッドPOST
概要CSV による FAQ の一括取込(新規 / 上書き)
利用画面SCR-006-002 FAQ CSV インポートモーダル
認証Cookie + CSRF
権限オーナー / 該当 PJ の member+

リクエストパラメータ

multipart/form-data { file: <CSV> }。受理形式は CSV のみ(UTF-8、BOM 許容、ヘッダ行必須、1 ファイル最大 1000 件)。JSON 形式は受理しない(不正形式は 415 / 400、E-INPUT-CSV-INVALID)。CSV 列構成は FAQ ID, 質問, 回答, カテゴリ。各行の FAQ ID で 空欄 = 新規(status=draft)/ 既存 ID 一致 = 上書き(状態維持・faq_revisions に改訂追記)/ 当該契約に存在しない ID = 当該行を失敗(行単位エラー、E-INPUT-CSV-FAQID-NOTFOUND)と判定する。status 列は受理しない。

レスポンス

(202):

{ "jobId": "job_...", "status": "processing" }

エラー

HTTP ステータスエラーコード内容
415 / 400E-INPUT-CSV-INVALIDCSV 以外 / 不正形式
(行単位)E-INPUT-CSV-FAQID-NOTFOUND当該契約に存在しない FAQ ID

API-FAQ-007 FAQ 全文検索

基本情報

項目内容
API IDAPI-FAQ-007
API 名FAQ 全文検索(機能 ID: F-300 / FR-300)
エンドポイント/projects/{id}/faqs/search
HTTP メソッドGET
概要当該プロジェクトの FAQ をキーワードで全文検索する
利用画面SCR-006
認証Cookie
権限オーナー / 該当 PJ の member+

検索対象は現在開いているプロジェクトに属する FAQ に限る(§2.2 オーナー境界 + プロジェクト境界)。並び替え・ページングは共通仕様(§2.1)に従う。

リクエストパラメータ

Path Parameters: {id}(プロジェクト ID)。Query Parameters: keyword(必須)、status(任意 draft / published / hidden)、sort(relevance(既定)/ updated_at)、cursor / limit(50〜200)。全文検索方式・対象列は 03_テーブル設計.html(FTS インデックス)を正本とする。

レスポンス

(200):

{ "items": [ { "id": "faq_...", "question": "...", "answer": "...", "status": "published", "category": "...", "score": 0.82, "updatedAt": "..." } ], "nextCursor": null }

API-FAQ-008 質問ログ検索

基本情報

項目内容
API IDAPI-FAQ-008
API 名質問ログ検索(機能 ID: F-300 / FR-300)
エンドポイント/projects/{id}/question-logs/search
HTTP メソッドGET
概要当該プロジェクトの質問ログをキーワード・期間で検索する
利用画面SCR-005
認証Cookie
権限オーナー / 該当 PJ の member+

検索対象は現在開いているプロジェクトに属する質問ログに限る。

リクエストパラメータ

Path Parameters: {id}(プロジェクト ID)。Query Parameters: keyword(任意)、from / to(任意 ISO 8601、期間)、answerType(任意 answered / unanswered)、sort(created_at 降順既定)、cursor / limit(50〜200)。対象列・全文検索方式は 03_テーブル設計.html を正本とする。

レスポンス

(200):

{ "items": [ { "id": "qlog_...", "question": "...", "answerType": "unanswered", "confidence": 0.42, "inquiryId": "inq_..." | null, "createdAt": "..." } ], "nextCursor": null }
