ER 図の見方

  • 掲載カラム: MVPの主要キー(PK / FK)+ 業務キー(inquiry_code / widget_public_key 等)+ 状態カラム(inquiries.status / contract_status 等)+ 特徴的なカラム(actor_user_id 等)。
  • 省略カラム: created_at / updated_at / valid / *_encrypted / *_key_version / metadata 等の共通技術カラム。完全なカラム定義は 03_テーブル設計/index.html §3 を参照。
  • 関係性: 実線 = FK 制約あり / 注記 FK制約なし = 論理参照のみ(error_logs / announcement_recipients の一部カラム)。本図は FK 制約に基づく関係のみを線で描く。announcement_recipients → inbox_messages の受信箱 fan-out と billing_subscriptions → billing_invoices の請求書生成は FK 制約を持たないアプリ層の論理 fan-out のため線を引かない(inbox_messages / billing_invoices はいずれも contract_owners に FK で帰属)。
  • MVP主体モデル: users(管理者:オーナー / プロジェクト管理者 / メンバー)+ contract_owners(オーナー専有属性、行存在 = オーナー判定)+ project_users(プロジェクト割当 + 別ロール)。ウィジェット利用者は永続アカウントを持たない。
  • SaaS 境界キー: contract_owner_user_id(全契約スコープのテーブルに冗長保持、認可境界クエリ高速化)。
