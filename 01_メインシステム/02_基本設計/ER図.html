ER 図の見方

  • 掲載カラム: 主要キー(PK / FK)+ 業務キー(inquiry_code / widget_public_key 等)+ 状態カラム(case_status / contract_status 等)+ 特徴的なカラム(actor_user_id / actor_end_user_id 等)。
  • 省略カラム: created_at / updated_at / valid / *_encrypted / *_key_version / metadata 等の共通技術カラム。完全なカラム定義は 03_テーブル設計.md §3 を参照。
  • 関係性: 実線 = FK 制約あり / 注記 FK制約なし = 論理参照のみ(error_logs / announcement_recipients の一部カラム)。
  • 4 主体モデル: users(管理者:オーナー / プロジェクト管理者 / メンバー)+ end_users(エンドユーザー、プロジェクト単位スコープ、独立)+ contract_owners(オーナー専有属性、行存在 = オーナー判定)+ project_users(プロジェクト割当 + 別ロール)。
  • SaaS 境界キー: contract_owner_user_id(全契約スコープのテーブルに冗長保持、認可境界クエリ高速化)。
