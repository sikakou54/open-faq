# FR02: ユーザー管理(オーナー + メンバー)

## 1. 機能概要

| 項目 | 内容 |
|---|---|
| 概要 | オーナーアカウント(1 契約 = 1 アカウント固定)とメンバーアカウント(0..N、プロジェクト単位のロール `admin` / `member` で制御)のユーザーモデルを管理する。招待・アクティベーション・ロール変更・削除の各操作を扱う |
| 利用場面 | オーナーによる契約全体のメンバー管理、およびプロジェクト管理者ロール保持者による該当プロジェクト範囲のメンバー招待・離脱 |
| 優先度 | Must |

## 2. 収録 FR-NNN 一覧

| ID | 要件 | 優先度 |
|---|---|:---:|
| FR-015 | オーナーおよびプロジェクト管理者(`account_project_grants.role='admin'` 保持者)は、メールアドレス、氏名、対象プロジェクトとそのプロジェクトに対するロール(`admin` / `member`)を指定して自分(オーナー)に紐づくメンバーを招待できること。プロジェクト管理者が招待できる範囲は、操作者自身が `admin` ロールを持つプロジェクトに限る | P0 |
| FR-015a | オーナーはアカウント新規登録時に自動的に決定され、全プロジェクトに対する暗黙の全権(プロジェクト管理者ロール相当)とオーナー専有機能を恒久的に保持すること。オーナーはロールの編集対象外とし、降格・譲渡操作は提供しない(MVP)。1 アカウント = 1 オーナーとする | P0 |
| FR-015b | メンバーの権限は「プロジェクトごとに付与するロール」で表現し、ロールは `admin`(プロジェクト管理者)と `member`(メンバー)の 2 種とすること。`admin` は該当プロジェクトに対する FAQ 管理 + 個別チャット対応 + ログ参照 + ユーザー管理(該当プロジェクト範囲の招待・離脱)を、`member` は該当プロジェクトに対する FAQ 管理 + 個別チャット対応 + ログ参照のみを許可する。個別の権限フラグ(`faq:manage` 等の細粒度フラグ)、プリセットロール、権限テンプレートは提供しない | P0 |
| FR-015c | 課金情報の閲覧・変更、月次予算上限変更、支払方法更新、退会申請・解約、規約再同意の承諾、およびプロジェクトの作成・編集・削除(プロジェクト名 / 許可ドメイン / IP 許可リスト 等のプロジェクト設定全般)はオーナー専有機能とし、いずれのロールでも付与できないこと | P0 |
| FR-015d | メンバーは割当されたプロジェクトに帰属するデータ(FAQ・未解決質問・個別チャット 等)のみアクセス可能とすること。プロジェクト割当が 0 のメンバーはダッシュボードのみ利用可とする(プロジェクト非依存の集計やリストは表示しない)。オーナーは割当の有無に関わらず全プロジェクトを閲覧・操作できる | P0 |
| FR-015e | プロジェクト作成時には、当該プロジェクトに対するプロジェクト管理者(`admin` ロール保持者)を最低 1 名指定すること(FR-030a と整合)。指定方法は「オーナー自身を管理者にする」または「メールアドレスで他者をプロジェクト管理者として指定する」のいずれか(併用可)。SCR-010-M1 で指定する他者は `admin` ロール固定とし、プロジェクト作成成功時にプロジェクト管理者招待メールを送信する。一般メンバー追加・ロール選択はユーザー管理(SCR-017 / SCR-017-M1)で行う | P0 |
| FR-016 | 招待されたメンバーは招待リンクからアカウントを有効化(本人によるパスワード設定)できること。招待リンクの有効期限は **7 日**(FR-020 と整合)。期限切れの場合はオーナーまたは該当プロジェクトの管理者ロール保持者が再送信する | P0 |
| FR-016a | 招待時に指定されたロール割当(プロジェクト × `admin` / `member`)は、招待対象者がアクティベーション(パスワード設定)を完了した時点で有効化されること | P0 |
| FR-016b | 招待リンクの再送信は、オーナーまたは該当プロジェクト管理者のみが実行できること。再送信時は旧リンクを失効させ、新リンクを発行する。招待中メンバーをキャンセルしたい場合は招待取消ではなくメンバー削除(FR-019)で対応する | P0 |
| FR-016c | アクティベーション完了前のメンバーは、招待時に指定されたロール割当を予約状態として保持するが、ログインおよび認可チェックは通らないこと(アクティベーション完了時点で初めて有効化される) | P0 |
| FR-017 | 利用者(オーナー / メンバー)は同一契約内のオーナーおよびメンバーの一覧および詳細を参照できること。詳細にはロール種別(オーナー / プロジェクト管理者 / メンバー)、招待状態(招待中 / アクティベーション済み)、プロジェクト別ロール割当一覧、最終ログイン日時、招待有効期限を含めること。非オーナーが閲覧する場合、表示対象は操作者自身が `admin` を持つプロジェクトに割当のあるメンバーに限定する | P0 |
| FR-018 | オーナーおよびプロジェクト管理者は、自分(オーナー)に紐づくメンバーの氏名、メールアドレス、通知設定を更新できること。オーナーの氏名・メール・通知設定の更新はオーナー本人のみが行えること。プロジェクト管理者が更新できる対象は操作者自身が `admin` を持つプロジェクトに割当のあるメンバーに限る | P0 |
| FR-018a | オーナーは、メンバーのプロジェクト別ロール割当を全プロジェクト横断で一括変更できること(`POST /members/{id}/project-grants` で全体上書き)。プロジェクト管理者は該当プロジェクト範囲でのみメンバー招待・離脱・ロール変更ができ、他プロジェクトの割当には影響しない。自分自身が `admin` ロールを持つ最後のプロジェクトで自身を降格する操作は不可とし、別の `admin` 保持者または オーナー経由でのみ実行できること。他メンバーを `admin` から `member` へ降格する操作は誤操作防止レベル L2(対象表示名のタイプ確認 + 最後の管理者保護チェック)を必須とする | P0 |
| FR-018b | ロール割当の付与・剥奪時には、変更前後のプロジェクト × ロール差分を監査ログに記録し、当該メンバーへお知らせ受信箱とメールで通知できること(FR-145a と整合) | P0 |
| FR-018c | オーナーおよびプロジェクト管理者は、メンバーの割当プロジェクトを後から追加・削除できること。割当変更時は変更前後の差分を監査ログに記録し、当該メンバーへ通知できること。プロジェクト管理者が他メンバーへ割当できるプロジェクトは、操作者自身が `admin` ロールを持つプロジェクトに限る | P0 |
| FR-019 | オーナーは、同一契約内のメンバーアカウントを完全削除できること。完全削除操作は契約ワークスペース(オーナー専有)のメンバー画面からのみ実行可能とし、誤操作防止レベル L3(対象メールアドレスのタイプ確認 + 再認証)を必須とする。オーナーは削除の対象外とし、自分自身の削除も不可とする(FR-333 / FR-334 と整合)。プロジェクト管理者によるアカウント完全削除は不可とし、該当プロジェクトからの「離脱」(`account_project_grants` 行のみ削除、誤操作防止レベル L1)のみ可能とする。MVP ではメンバーの一時的な利用停止 / 再開機能(有効・無効切替や強制ログアウト UI 操作)は提供せず、運営者経由の連携 IF #2(強制ログアウト)で代替する | P0 |
| FR-020 | 招待リンクには有効期限を設けること。**有効期限: 7 日**(顧客管理システム FR-020 と整合)。期限切れ後は招待の再送信で新リンクが発行される | P0 |
| FR-021 | メンバー数に固定の上限を設けず、利用量しきい値に基づいて上限接近や急増を検知し、利用者(オーナー / メンバー)へアラート通知できること | P0 |
| FR-021a | メンバー数しきい値接近・超過時のアラート通知先は、オーナーおよび同一契約内で 1 件以上の `admin` ロールを保持する全プロジェクト管理者とすること(critical 通知の宛先と一致) | P1 |
| FR-021b | 契約にメンバーが 1 人も登録されていない状態でも、オーナーが単独で契約運営を継続できること(オーナーは全機能保持) | P0 |
| FR-021c | 同一オーナーに紐づく既存の有効・招待中アカウントと同一メールアドレスでメンバーを重複招待できないこと。別オーナー配下では別アカウントとして扱う(FR-335 と整合) | P0 |
