## 2. 収録 FR-NNN 一覧

### 8.11.1 課金単位・計測・月次起点(両書共通)

本サービスは **完全従量課金制 + 月次無料枠** を採用する(顧客管理システム FR-300 と整合)。要件定義では課金対象と MVP 初期値のみを定義し、計測タイミング、丸め、冪等性、月次確定処理、超過時アクションの方式は [基本設計書 §6.6](../02_基本設計/index.md) を正本とする。

| 課金対象 | MVP 無料枠初期値 | MVP 超過課金単価初期値 |
|---|---:|---:|
| 質問数 | 1,000 件 / 月 | 0.5 円 / 件 |
| FAQ 件数 | 100 件 | 5 円 / 件 / 月 |
| 個別チャット部屋数 | 30 部屋 / 月 | 30 円 / 部屋 |
| AI 利用コスト(原価) | サービス側吸収(MVP) | n/a |

上表の無料枠初期値は**プロジェクトごとのデフォルト値**であり、各プロジェクトの上限管理(SCR-036-M1)でプロジェクト単位に上書きできる。AI 利用コストは契約単位の社内可視化指標として扱う。

月次境界は JST の暦月とし、無料枠は毎月リセットする。新規登録完了時点で契約は `active` となり、月次無料枠の範囲は無料で利用できる(無料トライアル期間は設けない)。支払方法の登録は任意であり、未登録でも無料枠内の利用およびダッシュボード閲覧は可能。支払方法登録済の契約は無料枠超過分が事後課金される。**無料枠および月次上限件数はプロジェクト単位で判定する**(上記表の MVP 初期値はプロジェクトごとのデフォルト値であり、各プロジェクトの SCR-036-M1 で上書きできる)。**支払方法未登録の契約のいずれかのプロジェクトが当該プロジェクトの無料枠を超過した場合は支払方法ゲート(FR-136)により、契約は `active` のまま当該プロジェクトのウィジェットの新規質問受付のみを停止(429 + メール誘導)し、管理画面は全機能利用可能とする。支払方法登録で即時再開する**(支払方法は契約単位、無料枠判定はプロジェクト単位)。

### 8.11.2 機能要件

| ID | 要件 | 優先度 |
|---|---|:---:|
| FR-120 | 質問数、AI 回答数(参考値)、未解決質問数、個別チャット部屋数、AI 利用コスト(原価)、FAQ 件数を集計できること。集計粒度は §8.11.1 の表に従い、課金対象外フラグ(失敗時)も保持すること | P0 |
| FR-121 | **プロジェクトごとに**利用上限(質問数、FAQ 件数、個別チャット部屋数の月次無料枠および課金対象ごとの月次上限件数(円相当を併記))を任意に設定できること。設定したしきい値に基づき利用量の計測および制限をプロジェクト単位で行えること。設定の優先順位は「プロジェクト設定(オーナー / 当該プロジェクト管理者)> 運営者上書き(プロジェクト単位)> デフォルト推奨値」とすること(顧客管理システム FR-121 と整合) | P0 |
| FR-122 | §8.11.1 の「利用上限超過時の動作」表のとおり、**プロジェクトごとに**課金対象ごとの設定上限件数に対し 80% / 100% / 125% の三段階で警告・制限・追加制限を行えること(判定はプロジェクト単位に独立)。(a) 80% 到達で警告通知、(b) **質問数が当該プロジェクトの設定上限件数 100% に到達した場合、支払方法の有無に関わらず当該プロジェクトのウィジェットの新規質問受付を停止し**(契約は `active` のまま、応答 429 + Retry-After=当月末、エンドユーザーにはプロジェクト連絡先メールへの問い合わせを促す安全な定型文を表示)、オーナー / 当該プロジェクト管理者へ通知すること、(c) FAQ 件数 / 個別チャット部屋数は当該プロジェクトの 100% 到達で当該の新規作成を拒否すること、(d) 125% は集計遅延・誤差対策の最終ガード。停止からの復帰は翌月 1 日の無料枠リセット、またはオーナー / 当該プロジェクト管理者が上限件数を引き上げた時点(SCR-036-M1)で即時とすること。なお **支払方法未登録の契約のいずれかのプロジェクトが当該プロジェクトの無料枠を超過した場合も FR-136 の支払方法ゲートにより同様に当該プロジェクトのウィジェット質問受付を停止する**(契約は `active` のまま。本項の上限件数到達とは別契機だが挙動は同じウィジェット制限。復帰は支払方法登録)| P0 |
| FR-123 | 請求状態を管理画面で確認できること | P0 |
| FR-124 | 支払い失敗や停止時の利用制限を行えること(顧客管理システム FR-124 と整合)。具体的には: (a) 顧客管理システムから「決済失敗確定」イベント(連携 IF #1)を受信した時刻を起点として 7 日間の猶予期間を計測すること、(b) 猶予期間中も管理者ユーザーの管理画面ログイン・支払方法更新を許可すること、(c) 猶予期間中の再決済成功時(顧客管理システムからの「支払い成功」イベント受信時)は即時に猶予を解除すること、(d) 猶予期間経過後は契約状態をサスペンション中へ遷移させ、ウィジェット応答は機能停止の旨を示す応答に切り替えること、(e) サスペンション中も管理者ユーザーは管理画面の課金画面・データエクスポート画面・退会画面のみアクセス可能とし、それ以外の機能は利用不可とすること。応答コード等の実装仕様および詳細フローは §9.10 および基本設計書を参照 | P0 |
| FR-125 | **プロジェクトごとの**利用しきい値や上限件数アラートの設定・変更を、オーナー / 当該プロジェクト管理者が管理画面(SCR-036-M1)で行えること。利用契約の解約申請はオーナーが管理画面で行えること(解約はオーナー専有) | P0 |
| FR-126 | 利用量は上限件数判定(80% / 100% / 125%)についてはリクエスト到達時のリアルタイム集計、管理者ユーザー向けサマリ表示は **5 分以内の遅延**(顧客管理システム FR-306 と整合)で反映されること | P1 |
| FR-127 | **プロジェクトごとに**利用件数が設定したしきい値や月次上限件数を超過しそうな場合に、オーナー + 当該プロジェクト管理者へアラート通知できること(円相当を併記)| P0 |
| FR-128 | 各APIやチャット機能に対して契約・API種別別のレート制限を DDoS / Bot / 暴走対策(セキュリティ目的)として適用し、運営者が契約単位でしきい値を調整できること(具体しきい値は基本設計書 §11.7 リミット設計表を参照)。**本レート制限(FR-128)は引き続き契約単位で管理し、プロジェクト単位化(FR-121 / FR-122)の対象外とする**(月次上限件数・無料枠のみプロジェクト単位)| P0 |
| FR-136 | 支払い方式はクレジットカード払いとすること。新規登録時点では支払方法の登録は必須としない(月次無料枠内は支払方法未登録でも利用可能)。**無料枠判定はプロジェクト単位、支払方法は契約単位**とする。支払方法は **無料枠超過時に必須化**(支払方法ゲート)とし、具体的には: (a) いずれかのプロジェクトの無料枠 80% 到達時に支払方法登録が必要になる旨を SCR-015 ダッシュボードのバナーおよび受信箱 / メールで事前告知すること、(b) **支払方法未登録のままいずれかのプロジェクトが当該プロジェクトの月次無料枠を超過した場合、契約状態は `active` のまま、当該プロジェクトのウィジェットの新規質問受付のみを停止し**(応答 429、エンドユーザーにはプロジェクト連絡先メールへの問い合わせを促す安全な定型文を表示)、管理者ユーザーには支払方法登録を促すこと。**契約サスペンションは行わず、管理画面は全機能を引き続き利用可能とすること**、(c) 支払方法を登録した時点でウィジェットの質問受付を即時再開し、以降は無料枠超過分を事後課金とすること。なお FAQ 件数 / 個別チャット部屋数が当該プロジェクトの無料枠を超過した場合の新規作成は支払方法未登録時に拒否し支払方法登録へ誘導すること | P0 |
| FR-137 | 支払い失敗を検知した場合、猶予期間(7 日。§9.10 サスペンションフロー、§20.2 参照)を経過後に契約をサスペンション状態に移行すること | P0 |
| FR-138 | サスペンション中は管理者ログインと請求情報更新のみ可能とし、その他の機能および新規ウィジェット応答は機能停止の旨を示す応答に切り替えること。応答コード等の実装仕様は基本設計書を正本とする | P0 |
| FR-139 | 課金プロバイダの Webhook を受信し、送信元の正当性確認、重複処理防止、直近 30 日の受信履歴、運営者による再処理、処理失敗時の運営者通知を備えること。具体的なイベント種別、検証手順、再処理方式は基本設計で定義する(顧客管理システム FR-302 / NFR-808 と整合) | P0 |

## 3. 補足説明・統合参照

§8.11.1 課金単位・計測・月次起点は両書共通の正本。MVP 初期値の数値は本節を起点に基本設計・詳細設計で具体化する。サスペンションフローの全体像は §9.10 を参照。新規登録完了時点で契約は `active` となり、無料トライアル期間は設けない(支払方法の登録は任意)。支払方法未登録の契約が無料枠を超過した場合は支払方法ゲート(FR-136)により契約は `active` のままウィジェット質問受付のみ停止する(契約サスペンションではない)。
