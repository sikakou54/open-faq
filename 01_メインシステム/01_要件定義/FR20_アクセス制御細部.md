## 1. 機能概要

| 項目 | 内容 |
|---|---|
| 概要 | プロジェクト単位 IP 許可リスト(ウィジェットアクセス制限)、海外 IP 遮断(運営者画面のみデフォルト ON)、複数デバイス同時ログイン、オーナーの退会・停止・削除・降格・譲渡の不可制約、自己操作不可制約、メンバーの 1 オーナー固定、プロジェクト別ロール選択 UI、ダッシュボードの権限不問アクセス、ロール変更後のセッション扱い、未割当メンバーの 403 拒否を扱う |
| 利用場面 | プロジェクトごとのウィジェット IP 制限導入、運営者の海外 IP 制限、ユーザー管理画面でのプロジェクト別ロール変更、URL 直アクセス時の拒否 |
| 優先度 | Must |

## 2. 収録 FR-NNN 一覧

| ID | 要件 | 優先度 |
|---|---|:---:|
| FR-330 | IP 許可リスト(FAQ ウィジェット向け、プロジェクト単位)を設定できること。設定権限はオーナー専有(プロジェクト設定の一部)。入力形式、許容範囲、検証方式、評価対象スコープ(ウィジェット系エンドポイントのみ / 管理 API 系は対象外)は基本設計を正本とする | P1 |
| FR-331 | 海外 IP 遮断は **運営者画面のみデフォルト ON** とする。利用者側 API およびウィジェットは海外エンドユーザーから到達するため対象外 | P1 |
| FR-332 | 同一アカウントの **複数デバイス同時ログインは可能** とし、SCR-001 でアクティブセッション一覧を確認可能にすること。運営者は連携 IF #2 で強制ログアウトを実行できる | P0 |
| FR-333 | オーナーは **退会・停止・削除・降格・譲渡の不可制約** とすること。1 アカウント = 1 オーナーの自存により、orphan 状態は構造的に発生しない。オーナー譲渡機能は MVP 対象外とし、緊急時はサポート窓口経由の運営者対応とする | P0 |
| FR-334 | 利用者(オーナー / メンバー)は **自分自身の利用状態変更・削除・ロール降格不可** とすること。特に自分が当該プロジェクトで `admin` ロールを持つ最後の管理者である場合、自身を降格する操作(`admin` → `member`)は不可で、別の `admin` 保持者(またはオーナー)経由で実行する必要がある。自分自身のアカウント削除は常に不可とし、オーナーが他人を経由して削除する経路もない(オーナー自身の操作のみが他者を削除する。FR-019) | P0 |
| FR-335 | メンバーは 1 アカウント = 1 オーナー配下に固定し、同一メールアドレスでも別オーナー配下では別アカウントとして扱うこと | P1 |
| FR-336 | プロジェクト別ロールの指定は当該プロジェクトごとに「割当しない / メンバー / 管理者」の 3 値ラジオで行うこととし、契約横断のプリセットロールは MVP では持たないこと。プリセット導入は Future 検討事項とする | P0 |
| FR-337 | ダッシュボード(自分の通知・お知らせ受信箱)は、オーナーおよびメンバーが閲覧できること(アカウント削除はアカウントレベル論理削除のため、現存メンバーは常に 1 件以上の割当を保持する)。それ以外の画面はそれぞれのロールとプロジェクト境界を要求する | P0 |
| FR-338 | プロジェクト別ロールおよびプロジェクト割当の追加・剥奪はアクティブセッションを即時失効させず、次回認可チェック時点で反映することで足りること。反映上限時間とキャッシュ方式は基本設計を正本とする | P1 |
| FR-339 | メンバーアカウントの割当 0 件状態は構造的に発生させないこと。次の 2 経路のみが `project_users` 行の単独削除をもたらす: (a) プロジェクト削除起因で他プロジェクト割当が 0 件かつ `contract_owners` 行を持たない(非オーナー)メンバーは `users` ごと論理削除(FR-030b)、(b) アカウント削除(FR-019)はアカウント全体論理削除のため割当 0 件メンバーが残らない。SCR-029 割当ゼロホームは MVP では設けず、プロジェクト帰属データへの URL 直入力アクセスは 403 拒否(該当 `project_users` 行を持たない場合) | P0 |
| FR-340 | 管理コンソールの全認証後画面で、現在の操作者ロール(オーナー / プロジェクトワークスペース内では当該プロジェクトに対する `admin` / `member`)を画面右上のロールバッジで常時可視化すること。複合ロール(他プロジェクトでは `admin`、当該プロジェクトでは `member` 等)の場合は現在地のワークスペースで適用されるロールを表示する | P0 |
| FR-341 | 権限不足で操作不可なボタン・トグル等は、単純な非活性化のみではなく、ホバー / フォーカス時に「必要な権限」と「依頼先(同一プロジェクトの `admin` 保持者一覧 + オーナー)」を提示できること。403 を実際に踏んだ場合は画面上部に同等の説明バナーを表示すること(基本設計 §4 を正本とする) | P0 |
