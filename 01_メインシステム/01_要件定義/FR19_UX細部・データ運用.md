## 1. 機能概要

| 項目 | 内容 |
|---|---|
| 概要 | FAQ 編集の同時編集衝突検出、下書きの 30 秒自動保存、改訂履歴(直近 50 件)・差分表示・ロールバック、一括操作の上限値(お知らせ 100 件 / FAQ 50 件)、画面 URL のスコープ別 4 系列規約、初回セットアップガイド、契約共通項目の影響範囲表示といった UX 細部とデータ運用要件を扱う |
| 利用場面 | 複数管理者による FAQ 同時編集、長文編集中のデータ保護、改訂履歴を辿った修正、まとめ操作、契約ホーム初回着地時のオンボーディング、契約共通項目の影響把握 |
| 優先度 | Must |

## 2. 収録 FR-NNN 一覧

| ID | 要件 | 優先度 |
|---|---|:---:|
| FR-320 | FAQ 編集は同時編集時の衝突を検出できること。衝突時は SCR-012 で「最新版を確認してください」を表示し再読み込み導線を提供すること。衝突制御方式は基本設計を正本とする | P1 |
| FR-321 | FAQ 下書きは **30 秒ごと自動保存** し、未保存変更がある状態でページ離脱を試みた際は警告ダイアログを表示すること | P2 |
| FR-322 | FAQ 改訂履歴は本文変更および状態変更(下書き/公開/非公開)を保持すること。直近 **50 件** を保持し、差分表示および任意版へのロールバックを可能とする | P1 |
| FR-323 | お知らせの一括既読は **最大 100 件**、FAQ の一括状態変更は **最大 50 件** を 1 操作の上限とすること。複数タブ同時ログイン時の既読同期方式は基本設計を正本とする | P1 |
| FR-324 | 削除・公開キー再発行・退会・予算上限変更・規約再同意拒否などの不可逆または影響範囲の大きい操作は、誤操作防止レベル L1(確認ダイアログ)/ L2(L1 + 対象名のタイプ確認)/ L3(L2 + 再認証)の三段モデルで分類し、各操作にレベルを必ず割り当てること。レベルの正本(各操作と対応レベルのマトリクス)は基本設計 01_画面設計.md §4 に置く | P0 |
| FR-325 | 利用者向け管理コンソールの画面 URL はスコープ別に **4 系列**(契約ワークスペース = `/owner/*` / プロジェクトワークスペース = `/projects/:projectId/*` / 認証フロー = `/auth/*` / 共通領域 = `/account/*`)に分離し、URL から現在の管理スコープを判別可能とする。SCR ↔ URL の対応マスタは基本設計 01_画面設計.md §4 に置く | P1 |
| FR-326 | SCR-026 契約ホームに **初回セットアップガイド**(`(1) プロジェクトを作成` / `(2) メンバーを招待` / `(3) ウィジェットを公開` の 3 ステップ + 完了率バー)を表示すること。各ステップは完了判定ロジック(プロジェクト存在 / 非オーナーの `account_project_grants` 存在 / `widget_keys.published_at IS NOT NULL` のプロジェクト存在)に従い自動完了する。3 ステップ全てが完了した時点で当該セクションは非表示となる。表示対象はオーナーのみ | P1 |
| FR-327 | 契約スコープの参照表示(DomainBadge `契約共通` バッジが付与される項目。例: 月次予算、契約共通マスタ)には、影響範囲を示す **`Used by Projects: N`**(N = 当該項目を参照している自契約内プロジェクト件数)の補助表示を併記すること。N=0 の場合は「未使用」と表示する。Used by Projects のクリック導線は将来対応とする | P2 |
