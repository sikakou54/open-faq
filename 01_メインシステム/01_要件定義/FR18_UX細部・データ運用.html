1. 機能概要

項目 内容
概要 FAQ 編集の同時編集衝突検出、下書きの自動保存、改訂履歴(直近 50 件)・差分表示・ロールバック、一括操作の上限値(お知らせ 100 件 / FAQ 50 件)、画面のスコープ別分離、初回セットアップガイド、契約共通項目の影響範囲表示といった UX 細部とデータ運用要件を扱う
利用場面 複数管理者による FAQ 同時編集、長文編集中のデータ保護、改訂履歴を辿った修正、まとめ操作、契約ホーム初回着地時のオンボーディング、契約共通項目の影響把握
優先度 Must

2. 収録 FR-NNN 一覧

ID 要件 優先度
FR-320 FAQ 編集は同時編集時の衝突を検出できること。衝突時は最新版の確認を促し、再読み込みの導線を提供すること。衝突制御方式は基本設計を正本とする P1
FR-321 FAQ 下書きは自動保存し、未保存の変更がある状態でページ離脱を試みた際は警告すること P2
FR-322 FAQ 改訂履歴は本文変更および状態変更(下書き / 公開 / 非公開)を保持すること。直近 50 件 を保持し、差分表示および任意版へのロールバックを可能とする P1
FR-323 お知らせの一括既読は 最大 100 件、FAQ の一括状態変更は 最大 50 件 を 1 操作の上限とすること P1
FR-324 削除・公開キー再発行・退会・上限件数変更・規約再同意拒否などの不可逆または影響範囲の大きい操作は、影響度に応じて確認のレベル(確認ダイアログ / 対象名のタイプ確認 / 本人確認(再認証))を区分し、各操作に必ずレベルを割り当てること。各操作と確認レベルの対応の正本は基本設計の画面設計に置く P0
FR-325 利用者向け管理コンソールは、管理スコープ(契約範囲 / プロジェクト範囲 / 認証フロー / 共通領域)を画面から判別できるよう分離すること。スコープと各画面の対応の正本は基本設計の画面設計に置く P1
FR-326 利用状況でプロジェクトが 0 件の場合は、説明文と「新規プロジェクトを作成」する導線だけを持つ空状態を表示すること。段階型のセットアップチェックリストは表示しない P1
FR-327 契約スコープの参照表示には、影響範囲を示す「利用中のプロジェクト: N」を併記すること。N=0 の場合は「未使用」と表示する。月次利用上限・無料利用枠はプロジェクト単位のため対象外とする P2
FR-328 主ナビゲーションはユーザーの目的語を優先し、契約範囲では「利用状況 / プロジェクト / 請求 / 設定」の 4 項目だけを用いること。プロジェクト範囲では「概要 / 要対応の質問 / FAQ / ウィジェット / メンバー / 利用量と上限」を用いる。プロジェクトの作成・編集・削除は契約範囲の「プロジェクト」、通知はヘッダー、法務文書はヘルプまたはフッターへ配置する P0
FR-329 一覧では名称を主リンク、システム識別子をコピー可能な補助情報として表示すること。内部の英語値(状態コード等)はユーザー向け表示へ直接露出させず、日本語の業務語彙へ変換すること P0
FR-330 個人設定・契約設定・プロジェクト編集を分離し、設定対象と影響範囲を画面タイトル・説明から判別できること P0
FR-331 画面見出しは画面名だけを表示し、契約名・プロジェクト名・サイト名を連結しないこと。現在の管理範囲名は管理範囲の切替表示に集約すること。ただし、削除・公開キー再発行等の対象確認文やデータ名称そのものを扱う編集画面では、誤操作防止に必要な対象名表示を維持すること P1
