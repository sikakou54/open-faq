項目 内容
概要 FAQ 編集の同時編集衝突検出、下書きの 30 秒自動保存、改訂履歴(直近 50 件)・差分表示・ロールバック、一括操作の上限値(お知らせ 100 件 / FAQ 50 件)、画面 URL のスコープ別 4 系列規約、初回セットアップガイド、契約共通項目の影響範囲表示といった UX 細部とデータ運用要件を扱う

優先度 Must

2. 収録 FR-NNN 一覧

ID 要件 優先度
FR-320 FAQ 編集は同時編集時の衝突を検出できること。衝突時は SCR-006 で「最新版を確認してください」を表示し再読み込み導線を提供すること。衝突制御方式は基本設計を正本とする P1
FR-321 FAQ 下書きは 30 秒ごと自動保存 し、未保存変更がある状態でページ離脱を試みた際は警告ダイアログを表示すること P2
FR-322 FAQ 改訂履歴は本文変更および状態変更(下書き/公開/非公開)を保持すること。直近 50 件 を保持し、差分表示および任意版へのロールバックを可能とする P1
FR-323 お知らせの一括既読は 最大 100 件、FAQ の一括状態変更は 最大 50 件 を 1 操作の上限とすること。複数タブ同時ログイン時の既読同期方式は基本設計を正本とする P1
FR-324 削除・公開キー再発行・退会・上限件数変更・規約再同意拒否などの不可逆または影響範囲の大きい操作は、誤操作防止レベル L1(確認ダイアログ)/ L2(L1 + 対象名のタイプ確認)/ L3(L2 + 再認証)の三段モデルで分類し、各操作にレベルを必ず割り当てること。レベルの正本(各操作と対応レベルのマトリクス)は基本設計 01_画面設計.html §4 に置く P0
FR-325 利用者向け管理コンソールの画面 URL はスコープ別に 4 系列(契約ワークスペース = /owner/* / プロジェクトワークスペース = /projects/:projectId/* / 認証フロー = /auth/* / 共通領域 = /account/*)に分離し、URL から現在の管理スコープを判別可能とする。SCR ↔︎ URL の対応マスタは基本設計 01_画面設計.html §4 に置く P1
FR-326 SCR-016 利用状況でプロジェクトが0件の場合は、説明文と「新規プロジェクトを作成」CTAだけを持つ空状態を表示すること。段階型のセットアップチェックリストは表示しない P1
FR-327 契約スコープの参照表示には、影響範囲を示す「利用中のプロジェクト: N」を併記すること。N=0の場合は「未使用」と表示する。月次利用上限・無料利用枠はプロジェクト単位のため対象外とする P2
FR-328 主ナビゲーションはユーザーの目的語を優先し、契約範囲では「利用状況 / プロジェクト / 請求 / 設定」の4項目だけを用いること。プロジェクト範囲では「概要 / 要対応の質問 / FAQ / ウィジェット / メンバー / 利用量と上限」を用いる。プロジェクトの作成・編集・削除は契約サイドメニュー「プロジェクト」(SCR-004 / SCR-004-M1)、通知はヘッダー、法務文書はヘルプまたはフッターへ配置する P0
FR-329 一覧では名称を主リンク、システムIDをコピー可能な補助情報として表示すること。内部英語値(open / closed等)はユーザー向け表示へ直接露出させず、日本語の業務語彙へ変換すること P0
FR-330 個人設定(SCR-017)、契約設定(SCR-023)、プロジェクト編集(SCR-004-M1 編集モード)を分離し、設定対象と影響範囲を画面タイトル・説明・URLから判別できること P0
FR-331 PageHeaderおよび設定モーダルの見出しは画面名だけを表示し、契約名・プロジェクト名・サイト名を「—」等で連結しないこと。現在の管理範囲名は WorkspaceSwitcher / WorkspaceBar に集約すること。ただし、削除・公開キー再発行等の対象確認文やデータ名称そのものを扱う編集画面では、誤操作防止に必要な対象名表示を維持すること P1
