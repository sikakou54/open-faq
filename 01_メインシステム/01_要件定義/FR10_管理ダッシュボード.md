## 1. 機能概要

| 項目 | 内容 |
|---|---|
| 概要 | 管理範囲スイッチャーで契約とプロジェクトを切り替える。オーナー向け SCR-016 利用状況では契約全体と各プロジェクトの利用量を一覧し、プロジェクト概要では選択中プロジェクトの質問数・未解決数・公開FAQ件数を表示する。課金情報は SCR-022 請求へ分離する |
| 利用場面 | 管理者の日次運用、対応案件の俯瞰、FAQ 改善ターゲット候補の確認 |
| 優先度 | Must |

| FR-135b | SCR-016 利用状況は当月固定・読み取り専用のスナップショットとし、プロジェクト別一覧にリンクを設けない。プロジェクトを開く・作成・編集はサイドメニュー「プロジェクト」(SCR-004)に集約し、各プロジェクトの SCR-008 概要へは管理範囲スイッチャーから1操作で遷移できること。オーナーはスイッチャーから1操作で利用状況へ戻れること | P0 |
| FR-135c | SCR-008 / SCR-016 は画面冒頭に目的を1行で宣言すること。SCR-008 は選択中プロジェクトの利用状況、SCR-016 は契約全体と各プロジェクトの当月利用状況の把握を目的とする | P0 |
| FR-135d | 最終更新タイムスタンプを表示する画面では、KPI カード群の右上に「最終更新: YYYY-MM-DD HH:MM(JST)」を統一表示すること。集計バッチが遅延している場合は「⚠ 集計遅延(最終: YYYY-MM-DD HH:MM)」と警告色で表示し、ユーザーがリアルタイム値と勘違いするリスクを抑止する。最終更新時刻の取得元は集計バッチ最終実行時刻またはマテビュー最終更新時刻とする。なお SCR-016 利用状況は読み取り専用スナップショットのため最終更新タイムスタンプを表示しない | P0 |
| FR-135e | KPI カード(質問数 / 未解決数 / 公開 FAQ 件数等)はクリックで該当一覧画面に遷移できること。遷移先 SCR ID とフィルタ条件は基本設計 01_画面設計.md の項目表に明記する。0 件 / 集計中 / 取得失敗 状態のときはクリック不可とする | P1 |
| FR-135f | 比較値を表示する画面では「(前月比 +N.N%)」「(前週比 -M.M%)」形式に統一すること。SCR-008 の KPI カードは項目名と件数だけを表示し、前月比・状態説明等のコメントやゲージは表示しないこと | P1 |
| FR-135g | KPI / 一覧 / アラートの各表示要素は、0 件・null(未集計)・集計中・取得失敗・期間外 の 5 状態を区別して表示できること。状態ごとの文言と配色ルールは基本設計 01_画面設計.md の「§0 共通表示ルール」に正本化し、各画面はそれを参照する | P0 |
| FR-135h | SCR-016 利用状況でプロジェクトが0件の場合は、複雑なセットアップチェックリストを表示せず、「プロジェクトがまだありません」と「新規プロジェクトを作成」の単一CTAを持つ空状態を表示すること | P2 |

## 3. 補足説明・統合参照

### 3.1 画面目的の宣言(FR-135c 詳細)

| SCR ID | 画面目的(1 行宣言) |
|---|---|
| SCR-008 概要 | 当該プロジェクトの質問数・未解決数・公開 FAQ 件数を確認するための画面 |
| SCR-016 利用状況 | 契約全体と各プロジェクトの当月利用状況を比較・把握するための読み取り専用画面(プロジェクトを開く・作成・編集はサイドメニュー「プロジェクト」へ) |

### 3.2 表示ルールの正本化方針

数値・単位・期間・最終更新・色語彙・状態表現の各共通ルールは、本書では概要のみ示し、正本は基本設計 [01_画面設計.md §0 共通表示ルール](../02_基本設計/01_画面設計.md) に置く。SCR ごとに表記を発明せず、参照のみとする。
