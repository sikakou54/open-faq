> **v1.1 改訂(2026-06-09)**: FAQ インポート・全データエクスポートのファイル形式を **CSV に統一(JSON 形式を全廃)**。FAQ インポートは SCR-006-M1 FAQ CSV インポートモーダルに分離(FR-310 改訂)。
> **v1.2 改訂(2026-06-09)**: FAQ CSV インポートの新規/上書き判定を管理者の競合処理選択(スキップ / 上書き / 別件追加)から **CSV の FAQ ID カラムによる行単位判定** に一本化(FR-310 改訂)。CSV 列構成を `FAQ ID, 質問, 回答, カテゴリ` に確定。テンプレート CSV のダウンロードを追加。SCR-006-M1 をドラッグ&ドロップ対応に。

## 1. 機能概要

| 項目 | 内容 |
|---|---|
| 概要 | FAQ の CSV インポート(新規/上書きは CSV の FAQ ID カラムで行単位判定、部分失敗時の理由確認、上書き時は改訂履歴保持。SCR-006-M1 モーダルで実施)と、FAQ の CSV エクスポート(SCR-006 一覧)を扱う。**契約全データの CSV エクスポートは MVP 対象外・将来対応**([05_future](../05_future/index.md))|
| 利用場面 | 大量 FAQ の初期投入、運用中の FAQ 一括更新、FAQ データの定期エクスポート |
| 優先度 | Must |

## 2. 収録 FR-NNN 一覧

| ID | 要件 | 優先度 |
|---|---|:---:|
| FR-310 | FAQ インポートは CSV 形式(UTF-8、BOM 許容)を受け入れること(SCR-006-M1 FAQ CSV インポートモーダルで実施)。CSV は `FAQ ID, 質問, 回答, カテゴリ` の列構成とし、新規登録か既存上書きかは **CSV の FAQ ID カラムで行単位に判定**すること(**FAQ ID 空欄 = 新規登録〔状態は draft〕/ 既存 FAQ ID と一致 = 上書き〔状態は維持し改訂履歴に残す〕/ 当該契約に存在しない無効な FAQ ID = 当該行を失敗扱い**)。テンプレート CSV(ヘッダ行のみ)をダウンロードできること。部分失敗時は成功分を活かし、失敗分の理由を管理者が確認できること。大容量インポートのジョブ化、同一判定、ロールバック単位、エラーログ形式は基本設計 / 詳細設計を正本とする | P1 |
| FR-311 | 契約全データエクスポート(FR-162)は CSV 形式(UTF-8)で提供し、スキーマ定義書を別ファイルで提供すること。非同期ジョブ化し、完了時にメール通知(**MVP 対象外・将来対応**。退避先 [05_future](../05_future/index.md))。なお **FAQ の CSV エクスポート(SCR-006 一覧)は MVP 対象**として継続する | 将来 |
