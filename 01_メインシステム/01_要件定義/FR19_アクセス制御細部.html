1. 機能概要

項目 内容
概要 海外 IP 遮断(運営者画面のみデフォルト ON)、複数デバイス同時ログイン、オーナーの退会・停止・削除・降格・譲渡の不可制約、自己操作不可制約、メンバーの 1 オーナー固定、プロジェクト別ロール選択 UI、ダッシュボードの権限不問アクセス、ロール変更後のセッション扱い、未割当メンバーの 403 拒否を扱う
利用場面 運営者の海外 IP 制限、ユーザー管理画面でのプロジェクト別ロール変更、URL 直アクセス時の拒否
優先度 Must

2. 収録 FR-NNN 一覧

ID 要件 優先度
FR-331 海外 IP 遮断は 運営者画面のみデフォルト ON とする。利用者側 API およびウィジェットは海外エンドユーザーから到達するため対象外 P1
FR-332 同一アカウントの複数デバイス同時ログインは可能とすること。運営者は連携 IF #2 で強制ログアウトを実行できる。利用者自身によるアクティブセッション一覧の表示・自己セッション終了(SCR-017)は MVP 対象外・将来対応(05_future) P0
FR-333 オーナーは 退会・停止・削除・降格・譲渡の不可制約 とすること。1 アカウント = 1 オーナーの自存により、orphan 状態は構造的に発生しない。オーナー譲渡機能は MVP 対象外とし、緊急時はサポート窓口経由の運営者対応とする P0
FR-334 利用者(オーナー / メンバー)は 自分自身の利用状態変更・削除・ロール降格不可 とすること。特に自分が当該プロジェクトで admin ロールを持つ最後の管理者である場合、自身を降格する操作(admin → member)は不可で、別の admin 保持者(またはオーナー)経由で実行する必要がある。自分自身のアカウント削除は常に不可とし、オーナーが他人を経由して削除する経路もない(オーナー自身の操作のみが他者を削除する。FR-019) P0
FR-335 メンバーは 1 アカウント = 1 オーナー配下に固定し、同一メールアドレスでも別オーナー配下では別アカウントとして扱うこと P1
FR-336 プロジェクト別ロールの指定は当該プロジェクトごとに「割当しない / メンバー / 管理者」の 3 値ラジオで行うこととし、契約横断のプリセットロールは持たないこと P0
FR-337 ダッシュボード(自分の通知・お知らせ受信箱)は、オーナーおよびメンバーが閲覧できること(アカウント削除はアカウントレベル論理削除のため、現存メンバーは常に 1 件以上の割当を保持する)。それ以外の画面はそれぞれのロールとプロジェクト境界を要求する P0
FR-338 プロジェクト別ロールおよびプロジェクト割当の追加・剥奪はアクティブセッションを即時失効させず、次回認可チェック時点で反映することで足りること。反映上限時間とキャッシュ方式は基本設計を正本とする P1
FR-339 非オーナーの最後の有効なプロジェクト割当を解除した場合は、対象アカウントを自動無効化して全セッションと未使用招待トークンを失効すること。契約横断のメンバー管理画面は提供しない。割当のないプロジェクトURLへの直アクセスは403とする P0
FR-340 現在の管理範囲と適用ロールはヘッダーの「管理範囲を切り替え」およびアカウントメニューで確認できること。独立したロールバッジを全画面へ重複表示する必要はない。複合ロールの場合は現在のプロジェクトで適用されるロールを表示する P0
FR-341 権限外の主ナビゲーションと操作は原則として表示しないこと。共同作業上、存在を知らせる価値がある項目は読み取り専用で表示し、必要権限と依頼先を説明する。URL直アクセスの403では同等の説明と戻り先を提示する P0
