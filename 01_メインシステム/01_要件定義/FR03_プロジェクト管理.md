## 1. 機能概要

| 項目 | 内容 |
|---|---|
| 概要 | オーナー配下で FAQ・ウィジェット・ログを分けて管理する単位「プロジェクト」を作成・編集・削除する。許可ドメイン設定、ウィジェット設定、プロジェクト連絡先メール、プロジェクト単位 IP 許可リスト(FAQ ウィジェットアクセス制限)も含む |
| 利用場面 | 製品別 / サイト別にプロジェクトを分けて運用する場合、ウィジェット設置先サイトの追加、プロジェクト終了時の整理 |
| 優先度 | Must |

## 2. 収録 FR-NNN 一覧

| ID | 要件 | 優先度 |
|---|---|:---:|
| FR-030 | オーナーはプロジェクトを作成、編集、削除できること。プロジェクト作成・編集(名称・許可ドメイン・IP 許可リスト等の設定全般)・削除はオーナー専有とし、いずれのロール(プロジェクト管理者 / メンバー)にも付与できない。**プロジェクト削除動線は SCR-026 契約ホーム(オーナーダッシュボード)のみに集約**する(SCR-010 / SCR-010-M1 には削除動線を置かない)。削除操作は誤操作防止のため、プロジェクト名タイプ確認 + 再認証(レベル L3。基本設計 01_画面設計.md §4 の誤操作防止レベルを正本)を必須とする | P0 |
| FR-030a | プロジェクト作成時には、当該プロジェクトの作成者である**オーナー自身が常にプロジェクト管理者となる**(プロジェクト作成と同時に `project_users(user_id=オーナー, project_id=新プロジェクト, contract_owner_user_id=オーナー, role='admin', valid=1)` 行が自動で 1 行 INSERT される)。SCR-010-M1 では管理者を画面から個別指定するセクションは設けず、一般メンバー追加・他者をプロジェクト管理者として招待する操作はユーザー管理(SCR-017 / SCR-017-M1)で行う(FR-015e と整合) | P0 |
| FR-030b | プロジェクト削除時は当該プロジェクトの `project_users` を全行**論理削除**(`UPDATE … SET valid=0`)する。削除の結果、他プロジェクト割当(`valid=1` の `project_users` 行)が 0 件かつ `contract_owners` 行を持たない(=非オーナー)ユーザーは `users.valid=0` で**論理削除**する(孤立メンバー cleanup)。他プロジェクト割当があるメンバーは `users.valid=1` を維持し、当該プロジェクトに対する `project_users` 行のみ `valid=0` で論理削除する。オーナー(`contract_owners` 行を保持するユーザー)の `users` 行は本処理の対象外(オーナー自身が当該プロジェクトに対し保持していた `project_users(role='admin')` 行は `valid=0` に更新するが、オーナー `users` 行と `contract_owners` 行は維持)。論理削除データは `updated_at` から 90 日経過後に物理削除バッチで物理削除する | P0 |
| FR-031 | プロジェクトごとにウィジェット設定(公開キー・埋め込みコード・見た目・プレビュー)を管理できること。公開キーはプロジェクト単位で発行・ローテーションし、契約共通ではない。公開キーの再発行は当該プロジェクトの管理者(`project_users.role='admin'`)以上に許可し、誤操作防止レベル L3(再認証 + 「既存埋め込みコードが即座に無効化される」旨の確認)を必須とする。メンバー(`role='member'`)は閲覧と埋め込みコードのコピーのみ可。SCR-014 はプロジェクトワークスペース内に配置する(契約ワークスペースには配置しない) | P0 |
| FR-032 | プロジェクトごとにFAQ、質問ログ、未解決質問、個別チャットを分けて管理できること | P0 |
| FR-033 | 許可するWebサイトのドメインを複数設定できること | P0 |
| FR-033a | プロジェクトごとに連絡先メールアドレスを 1 件設定でき、確認メール(到達 + リンククリックによる所有権確認)を経て有効化できること。確認完了前のアドレスはウィジェット表示に用いない。メール送信時の Reply-To としては利用しない(エンドユーザー向けメールは常に共通ドメインの no-reply 送信で固定) | P0 |
| FR-033c | 公開ウィジェットのチャット画面において、確認完了済のプロジェクト連絡先メールを「チャットで解決できない場合のお問い合わせ先」として表示できること。未設定または未確認の場合は当該表示自体を行わないこと | P0 |
| FR-034 | プロジェクト削除時の関連データ(FAQ、ログ、チャット)の取扱いを利用者が確認・選択できること | P0 |
| FR-035 | プロジェクト数に固定の上限を設けず、利用状況を監視し急激な増加を検知してオーナーへ通知できること | P1 |
| FR-035a | プロジェクトごとに FAQ ウィジェットアクセスに対する IP 許可リストを設定できること(IPv4 / IPv6 の CIDR 複数行、空欄 = 制限なし)。設定権限はオーナー専有(プロジェクト設定の一部)。本 IP 許可リストは管理画面 API(ログイン・課金・ユーザー管理等)の評価対象外。詳細は FR-179 / FR-330 と基本設計を正本とする。全削除(空欄化 = 制限なしへの変更)時はレベル L2 確認(「制限なしになります」確認文 + 「実行」入力)を必須とする | P1 |
