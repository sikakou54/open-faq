## 1. 機能概要

| 項目 | 内容 |
|---|---|
| 概要 | オーナー配下で FAQ・ウィジェット・ログを分けて管理する単位「プロジェクト」を作成・編集・削除する。許可ドメイン設定、ウィジェット設定、プロジェクト連絡先メール、プロジェクト単位 IP 許可リスト(FAQ ウィジェットアクセス制限)も含む |
| 利用場面 | 製品別 / サイト別にプロジェクトを分けて運用する場合、ウィジェット設置先サイトの追加、プロジェクト終了時の整理 |
| 優先度 | Must |

## 2. 収録 FR-NNN 一覧

| ID | 要件 | 優先度 |
|---|---|:---:|
| FR-030 | オーナーはプロジェクトを作成、編集、削除できること。プロジェクト作成・編集(名称・許可ドメイン・IP 許可リスト等の設定全般)・削除はオーナー専有とし、いずれのロール(プロジェクト管理者 / メンバー)にも付与できない | P0 |
| FR-030a | プロジェクト作成時には、当該プロジェクトに対するプロジェクト管理者(`account_project_grants.role='admin'` 保持者)を最低 1 名指定すること。指定方法は (a) オーナー自身を管理者にする(SCR-010-M1 のチェックボックス、デフォルト ON) / (b) メールアドレスで他者を招待してロール `admin` を付与する(同モーダル内の招待者リスト、複数可)のいずれか(併用可)。(a) が OFF の場合は (b) の招待者の少なくとも 1 名を `admin` ロールとすること(FR-015e と整合) | P0 |
| FR-030b | プロジェクト削除時は当該プロジェクトの `account_project_grants` を全行削除すること。削除の結果、他プロジェクト割当が 0 件かつ `is_owner=0` となるメンバーは `accounts` ごと削除する(孤立メンバー cleanup)。他プロジェクト割当があるメンバーはアカウントを維持し、当該プロジェクトに対する `account_project_grants` 行のみ削除する。オーナーは対象外 | P0 |
| FR-031 | プロジェクトごとにウィジェット設定を管理できること | P0 |
| FR-032 | プロジェクトごとにFAQ、質問ログ、未解決質問、個別チャットを分けて管理できること | P0 |
| FR-033 | 許可するWebサイトのドメインを複数設定できること | P0 |
| FR-033a | プロジェクトごとに連絡先メールアドレスを 1 件設定でき、確認メール(到達 + リンククリックによる所有権確認)を経て有効化できること。確認完了前のアドレスはウィジェット表示に用いない。メール送信時の Reply-To としては利用しない(エンドユーザー向けメールは常に共通ドメインの no-reply 送信で固定) | P0 |
| FR-033c | 公開ウィジェットのチャット画面において、確認完了済のプロジェクト連絡先メールを「チャットで解決できない場合のお問い合わせ先」として表示できること。未設定または未確認の場合は当該表示自体を行わないこと | P0 |
| FR-034 | プロジェクト削除時の関連データ(FAQ、ログ、チャット)の取扱いを利用者が確認・選択できること | P0 |
| FR-035 | プロジェクト数に固定の上限を設けず、利用状況を監視し急激な増加を検知してオーナーへ通知できること | P1 |
| FR-035a | プロジェクトごとに FAQ ウィジェットアクセスに対する IP 許可リストを設定できること(IPv4 / IPv6 の CIDR 複数行、空欄 = 制限なし)。設定権限はオーナー専有(プロジェクト設定の一部)。本 IP 許可リストは管理画面 API(ログイン・課金・ユーザー管理等)の評価対象外。詳細は FR-179 / FR-330 と基本設計を正本とする | P1 |
