## 2. 収録 FR-NNN 一覧

### 8.11.1 課金単位・計測・月次起点(両書共通)

本サービスは **完全従量課金制 + 月次無料枠** を採用する(顧客管理システム FR-300 と整合)。要件定義では課金対象と MVP 初期値のみを定義し、計測タイミング、丸め、冪等性、月次確定処理、超過時アクションの方式は [基本設計書 §6.6](../02_基本設計/index.md) を正本とする。

| 課金対象 | MVP 無料枠初期値 | MVP 超過課金単価初期値 |
|---|---:|---:|
| 質問数 | 1,000 件 / 月 | 0.5 円 / 件 |
| FAQ 件数 | 100 件 | 5 円 / 件 / 月 |
| AI 利用コスト(原価) | サービス側吸収(MVP) | n/a |

上表の無料枠初期値は**プロジェクトごとのデフォルト値**とする。全課金対象の無料枠は料金設定として運営者が管理し、SCR-021 / SCR-021-M1 では表示・設定しない。月次上限件数を設定できる課金対象は質問数のみとし、FAQ 件数は「上限なし」で利用量計測と超過課金を継続する。AI 利用コストは契約単位の社内可視化指標として扱う。

月次境界は JST の暦月とし、無料枠は毎月リセットする。新規登録完了時点で契約は `active` となり、月次無料枠の範囲は無料で利用できる(無料トライアル期間は設けない)。支払方法の登録は任意であり、未登録でも無料枠内の利用およびダッシュボード閲覧は可能。支払方法登録済の契約は無料枠超過分が事後課金される。**無料枠は全課金対象でプロジェクト単位、月次上限件数は質問数のみプロジェクト単位で判定する**。**支払方法未登録の契約のいずれかのプロジェクトが質問数無料枠を超過した場合は支払方法ゲート(FR-136)により、契約は `active` のまま当該プロジェクトのウィジェットの新規質問受付のみを停止(429 + メール誘導)し、管理画面は全機能利用可能とする。支払方法登録で即時再開する**。FAQ 件数は月次件数を理由に新規作成を停止しない。

### 8.11.2 機能要件

| ID | 要件 | 優先度 |
|---|---|:---:|
| FR-120 | 質問数、AI 回答数(参考値)、未解決質問数、AI 利用コスト(原価)、FAQ 件数を集計できること。集計粒度は §8.11.1 の表に従い、課金対象外フラグ(失敗時)も保持すること | P0 |
| FR-121 | **プロジェクトごとに**質問数の月次上限件数と上限アラートを任意に設定できること。月次上限はON / OFFを切り替え可能とし、設定の優先順位は「プロジェクト設定(オーナー / 当該プロジェクト管理者)> 運営者上書き(プロジェクト単位)> デフォルト推奨値」とすること。SCR-021 は質問数だけを対象とし、「当月利用」「今月の利用上限」「利用量」だけを簡潔に表示すること。無料利用枠・アラート状態・設定元は同画面に表示しないこと。SCR-021-M1 は当月利用と無料利用枠を表示・更新せず、質問数上限とアラート設定だけを扱うこと。上限OFF時は「今月の利用上限」を「OFF」と表示し、「利用量」の割合・件数比・ゲージ・状態バッジを表示しないこと。FAQ 件数は SCR-021 / SCR-021-M1 に表示せず、管理者ユーザー向けの設定も提供しないこと。全課金対象の無料枠は運営者が料金設定として管理すること(顧客管理システム FR-121 と整合) | P0 |
| FR-122 | **プロジェクトごとの質問数上限がONの場合**、**質問数が当該プロジェクトの設定上限件数 100% に到達した時点で、支払方法の有無に関わらず当該プロジェクトのウィジェットの新規質問受付を停止すること**(契約は `active` のまま、応答 429 + Retry-After=当月末、エンドユーザーにはプロジェクト連絡先メールへの問い合わせを促す安全な定型文をチャット内のシステム返信として表示し、質問入力と送信を無効化する)。125% は集計遅延・誤差対策の最終ガードとして運営者へ追加通知すること。管理者向けメールはFR-127で選択された閾値に従い、100%が未選択の場合も停止処理自体は行う。上限OFF時は利用率の算出、上限アラート、上限到達による停止を行わないこと。停止からの復帰は翌月 1 日のリセット、オーナー / 当該プロジェクト管理者が質問数上限を引き上げた時点、または上限をOFFにした時点(SCR-021-M1)で即時とすること。FAQ 件数は本判定と月次件数による作成停止の対象外とする。なお **支払方法未登録の契約のいずれかのプロジェクトが質問数無料枠を超過した場合も FR-136 の支払方法ゲートにより同様に当該プロジェクトのウィジェット質問受付を停止する** | P0 |
| FR-123 | オーナーは SCR-022 請求で支払方法、請求状態、請求履歴、契約全体の請求見込み、プロジェクト別の課金内訳を確認できること。契約全体と各プロジェクトの利用量は SCR-016 利用状況、プロジェクト単位の上限詳細は SCR-021 利用量と上限で確認できること | P0 |
| FR-124 | 支払い失敗や停止時の利用制限を行えること(顧客管理システム FR-124 と整合)。具体的には: (a) 顧客管理システムから「決済失敗確定」イベント(連携 IF #1)を受信した時刻を起点として 7 日間の猶予期間を計測すること、(b) 猶予期間中も管理者ユーザーの管理画面ログイン・支払方法更新を許可すること、(c) 猶予期間中の再決済成功時(顧客管理システムからの「支払い成功」イベント受信時)は即時に猶予を解除すること、(d) 猶予期間経過後は契約状態をサスペンション中へ遷移させ、ウィジェット応答は機能停止の旨を示す応答に切り替えること、(e) サスペンション中も管理者ユーザーは管理画面の課金画面・退会画面のみアクセス可能とし、それ以外の機能は利用不可とすること。応答コード等の実装仕様および詳細フローは §9.10 および基本設計書を参照 | P0 |
| FR-125 | **プロジェクトごとの**質問数上限と上限アラートの設定変更を、オーナー / 当該プロジェクト管理者が質問数専用の管理画面(SCR-021-M1)で行えること。同画面には当月利用、無料利用枠、アラート送信先、FAQ 件数を表示しないこと。利用契約の解約申請はオーナーが管理画面で行えること(解約はオーナー専有) | P0 |
| FR-126 | 利用量は質問数の選択アラート閾値到達、100%停止判定、125%運営者通知についてリクエスト到達時のリアルタイム集計、管理者ユーザー向けサマリ表示は **5 分以内の遅延**(顧客管理システム FR-306 と整合)で反映されること | P1 |
| FR-127 | **プロジェクトごとに**質問数上限に対する 25% / 50% / 80% / 90% / 100% のアラート閾値を複数選択でき、選択した各閾値へ当月初回到達した時点で、オーナー + 当該プロジェクトの有効な管理者へメール送信できること(同一メールアドレスは重複排除)。送信先の表示・変更項目は設けず、全項目未選択はアラート通知なしとする。FAQ 件数は上限アラート対象外とする | P0 |
| FR-128 | 各APIに対して契約・API種別別のレート制限を DDoS / Bot / 暴走対策(セキュリティ目的)として適用し、運営者が契約単位でしきい値を調整できること(具体しきい値は基本設計書 §11.7 リミット設計表を参照)。**本レート制限(FR-128)は引き続き契約単位で管理し、プロジェクト単位化(FR-121 / FR-122)の対象外とする**(月次上限件数・無料枠のみプロジェクト単位)| P0 |
| FR-136 | 支払い方式はクレジットカード払いとすること。新規登録時点では支払方法の登録は必須としない(月次無料利用枠内は支払方法未登録でも利用可能)。無料利用枠判定はプロジェクト単位、支払方法は契約単位とする。(a) いずれかのプロジェクトが無料利用枠の80%に到達した場合、SCR-016 利用状況の該当プロジェクト行、お知らせ、メールで事前告知する、(b) 未登録のまま無料利用枠を超過した場合は当該プロジェクトのウィジェット新規質問受付のみを停止し、SCR-022 請求へ誘導する、(c) 支払方法登録後は即時再開する。契約サスペンションは行わず、管理画面は継続利用可能とする | P0 |
| FR-137 | 支払い失敗を検知した場合、猶予期間(7 日。§9.10 サスペンションフロー、§20.2 参照)を経過後に契約をサスペンション状態に移行すること | P0 |
| FR-138 | サスペンション中は管理者ログインと請求情報更新のみ可能とし、その他の機能および新規ウィジェット応答は機能停止の旨を示す応答に切り替えること。応答コード等の実装仕様は基本設計書を正本とする | P0 |
| FR-139 | 課金プロバイダの Webhook を受信し、送信元の正当性確認、重複処理防止、直近 30 日の受信履歴、運営者による再処理、処理失敗時の運営者通知を備えること。具体的なイベント種別、検証手順、再処理方式は基本設計で定義する(顧客管理システム FR-302 / NFR-808 と整合) | P0 |

## 3. 補足説明・統合参照

§8.11.1 課金単位・計測・月次起点は両書共通の正本。MVP 初期値の数値は本節を起点に基本設計・詳細設計で具体化する。サスペンションフローの全体像は §9.10 を参照。新規登録完了時点で契約は `active` となり、無料トライアル期間は設けない(支払方法の登録は任意)。支払方法未登録の契約が無料枠を超過した場合は支払方法ゲート(FR-136)により契約は `active` のままウィジェット質問受付のみ停止する(契約サスペンションではない)。
