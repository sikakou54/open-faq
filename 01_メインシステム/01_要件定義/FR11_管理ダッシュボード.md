## 1. 機能概要

| 項目 | 内容 |
|---|---|
| 概要 | 管理者の日次運用画面。ヘッダーのワークスペーススイッチャーで「契約ワークスペース」(オーナー専有・契約ホーム = 契約全体の集計)と「プロジェクトワークスペース」(プロジェクトホーム = 選択中プロジェクトのみの集計)を切替し、質問数 / 解決数 / 未解決数 / 個別チャットの未読・投稿状況 / FAQ 登録状況 / よく聞かれる質問 / 未解決傾向 / 通知失敗・バウンス件数を期間絞り込みで表示する |
| 利用場面 | 管理者の日次運用、対応案件の俯瞰、FAQ 改善ターゲット候補の確認 |
| 優先度 | Must |

## 2. 収録 FR-NNN 一覧

| ID | 要件 | 優先度 |
|---|---|:---:|
| FR-130 | 質問数、解決数、未解決数を確認できること | P0 |
| FR-131 | 個別チャットの未読メッセージ数と直近投稿状況を確認できること。未解決質問の状況とは合算しないこと | P0 |
| FR-132 | 未解決質問の状況(open / closed)を確認できること | P0 |
| FR-133 | よく聞かれる質問や未解決傾向を確認できること | P1 |
| FR-134 | 期間絞り込みができること | P0 |
| FR-135 | 通知失敗・バウンス件数を確認できること | P0 |
| FR-135a | オーナーはヘッダーのワークスペーススイッチャーから、契約全体を見るオーナー視点ダッシュボード(契約ワークスペース / 契約ホーム = SCR-026)と、選択中プロジェクトを見るプロジェクト視点ダッシュボード(プロジェクトワークスペース / プロジェクトホーム = SCR-015 プロジェクト表示)を切り替えられること。プロジェクト管理者・メンバーは契約ワークスペースに到達できず、プロジェクトワークスペースのプロジェクト視点ダッシュボードのみ利用できること。ワークスペースが現在どちらかは画面上部の領域バッジ(契約 / プロジェクト)で常時可視化する | P0 |
| FR-135b | 契約ワークスペースに居るオーナーは、契約ホームから各プロジェクトへのドリルダウン(プロジェクト行クリックで該当プロジェクトワークスペース / プロジェクトホームへ遷移)が 1 操作で行えること。逆にプロジェクトワークスペースに居るオーナー / プロジェクト管理者は、画面上部のリンク(「契約 WS へ」相当)で契約ワークスペースへ戻れること(オーナーのみ・プロジェクト管理者には表示しない) | P0 |
| FR-135c | 各ダッシュボード画面(SCR-015 / SCR-026)は、画面の冒頭に「画面目的」を 1 行で宣言すること。SCR-015 は「日常運用の状況把握とプロジェクト内の対応起点への着地」、SCR-026 は「契約全体の経営判断とプロジェクトへのドリルダウン起点」と定義する | P0 |
| FR-135d | KPI カード群の右上に「最終更新: YYYY-MM-DD HH:MM(JST)」を統一表示すること。集計バッチが遅延している場合は「⚠ 集計遅延(最終: YYYY-MM-DD HH:MM)」と警告色で表示し、ユーザーがリアルタイム値と勘違いするリスクを抑止する。最終更新時刻の取得元は集計バッチ最終実行時刻またはマテビュー最終更新時刻とする | P0 |
| FR-135e | KPI カード(質問数 / 解決数 / 未解決数 / 通知失敗 / 自分のタスク 等)はクリックで該当一覧画面に遷移できること。遷移先 SCR ID とフィルタ条件は基本設計 01_画面設計.md の項目表に明記する。0 件 / 集計中 / 取得失敗 状態のときはクリック不可とする | P1 |
| FR-135f | 比較値の表記は「(前月比 +N.N%)」「(前週比 -M.M%)」形式で統一し、各 KPI カードに比較粒度(前月比 / 前週比 / 前年比)のラベルを必ず付与すること。当月集計 KPI は前月比、週次 KPI は前週比を原則とする。増減の良し悪し(解決率増は緑、通知失敗率増は赤 等)は KPI ごとに方向性を定義する | P1 |
| FR-135g | KPI / 一覧 / アラートの各表示要素は、0 件・null(未集計)・集計中・取得失敗・期間外 の 5 状態を区別して表示できること。状態ごとの文言と配色ルールは基本設計 01_画面設計.md の「§0 共通表示ルール」に正本化し、各画面はそれを参照する | P0 |
| FR-135h | SCR-026 契約ホームの初回セットアップガイド(OnboardingChecklist)は、3 ステップ全完了で自動的に非表示となるが、ヘッダの「?」メニューから手動で再表示できること。退会・メンバー全員無効化など状態が逆戻りした場合の自動再表示は MVP では行わない | P2 |

## 3. 補足説明・統合参照

### 3.1 画面目的の宣言(FR-135c 詳細)

| SCR ID | 画面目的(1 行宣言) |
|---|---|
| SCR-015 プロジェクトホーム | プロジェクト管理者・メンバーが当該プロジェクトの日常運用状況を把握し、未解決質問・通知失敗・自分のタスクなど「次にやるべき対応」へ着地するための画面 |
| SCR-026 契約ホーム | オーナーが契約全体の経営状況(契約ステータス / AI コスト消化率 / 次回請求)と各プロジェクトの俯瞰を行い、各プロジェクトワークスペースへドリルダウンするための起点画面 |

### 3.2 表示ルールの正本化方針

数値・単位・期間・最終更新・色語彙・状態表現の各共通ルールは、本書では概要のみ示し、正本は基本設計 [01_画面設計.md §0 共通表示ルール](../02_基本設計/01_画面設計.md) に置く。SCR ごとに表記を発明せず、参照のみとする。
