2. 収録 FR-NNN 一覧

ID 要件 優先度
FR-310 FAQ インポートは CSV 形式(UTF-8、BOM 許容)を受け入れること(SCR-006-M1 FAQ CSV インポートモーダルで実施)。CSV は FAQ ID, 質問, 回答, カテゴリ の列構成とし、新規登録か既存上書きかは CSV の FAQ ID カラムで行単位に判定すること(FAQ ID 空欄 = 新規登録〔状態は draft〕/ 既存 FAQ ID と一致 = 上書き〔状態は維持し改訂履歴に残す〕/ 当該契約に存在しない無効な FAQ ID = 当該行を失敗扱い)。テンプレート CSV(ヘッダ行のみ)をダウンロードできること。部分失敗時は成功分を活かし、失敗分の理由を管理者が確認できること。大容量インポートのジョブ化、同一判定、ロールバック単位、エラーログ形式は基本設計 / 詳細設計を正本とする P1
FR-311 契約全データエクスポート(FR-162)は CSV 形式(UTF-8)で提供し、スキーマ定義書を別ファイルで提供すること。非同期ジョブ化し、完了時にメール通知(MVP 対象外・将来対応。退避先 05_future)。なお FAQ の CSV エクスポート(SCR-006 一覧)は MVP 対象として継続する 将来
