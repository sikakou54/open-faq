備考(v1.7 新規)
  • 未認証画面(連絡先確認トークン認証のみ、Cookie セッションなし)。URL = /auth/contact-verify?token=...
  • TPL-PROJECT_CONTACT_VERIFY 確認メールのリンクから到達(`access_tokens.purpose='contact_verify'`、24h、`meta.projectId` を JSON 保持)
  • 「閉じる」押下前にバックグラウンドで `POST /auth/contact-verifications/{token}` を実行 → 同一トランザクションで `UPDATE projects SET contact_email_verified_at=?` + `consumeToken('contact_verify')`(FR-033a)
  • 連絡先メール所有者はアカウントを持つ必要がない(第三者の共有メールアドレスでも本フローを進められる)。入力フォームを持たない確認専用ページ
  • サイドメニュー除外(未認証画面のため、01_画面設計.md §4.4.5)
