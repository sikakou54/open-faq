# メッセージ一覧(メイン)

## 1. 文書概要

### 1.1 目的

メインシステムでユーザーに表示するすべての文言(画面ラベル / プレースホルダー / ボタン / 説明文 / エラー / 成功 / 警告 / 確認 / 空状態 / トースト / 通知メール)を一元管理する。

### 1.2 対象範囲

- 対象: SCR-001〜027 の全画面文言、エラー文言(エラー設計書 06 と対応)、通知メールテンプレート(基本設計書 §9.1 / §9.4)、課金・契約関連メッセージ、サイドメニュー文言
- 対象外: 運営者側メッセージ(02_運営者システム/個別設計書群/07_メッセージ一覧.md)、コード内デバッグログ

## 2. 共通基準

### 2.1 文言トーン

- 丁寧体(です・ます調)、ユーザーを「あなた」と呼ばない(二人称省略)
- 専門用語は説明を併記(初出時)
- 数値は半角、単位は和(円 / 件 / 日)

### 2.2 ボタン文言ルール

| 種別 | テンプレート | 例 |
|---|---|---|
| 主操作 | 具体動詞 + する | 「削除する」「招待メールを送信」「変更を保存」 |
| 取消 | 「キャンセル」固定 | - |
| 危険操作 | 赤色 + 確認ダイアログ | 「削除する」「退会する」 |

### 2.3 確認ダイアログルール

| 要素 | テンプレート | 例 |
|---|---|---|
| タイトル | 「{動詞}しますか?」 | 「FAQ を公開しますか?」「メンバーを削除しますか?」 |
| 本文 | 操作結果 + 取消可否 | 「公開するとウィジェット利用者に表示されます。」「削除すると元に戻せません(運営者への復元依頼が必要です)」 |

### 2.4 通知重要度ルール(共有概念正本)

| 重要度 | 配信先 | 配信チャネル |
|---|---|---|
| `low` | inbox のみ | - |
| `normal` | inbox + email(任意)| inbox / email |
| `high` | inbox + email | inbox / email |
| **`critical`** | inbox + **email 強制送信** | inbox + email |

`critical` の送信先解決ルール: オーナーアカウント + 同一オーナースコープで `users:manage` フラグを保持する全メンバー。詳細クエリは [09_認証認可設計書.md §6.3](09_認証認可設計書.md) の正本を参照。

### 2.5 エラーメッセージルール

- エラー設計書(06)の各エラーID に必ず 1 つのメッセージID を割り当てる
- 内部詳細(スタックトレース / DB エラー)はユーザーに見せない
- 認可エラーは具体的な権限名を含めない(列挙攻撃対策)
- 認証失敗時はメールアドレス / パスワードのどちらが誤りかを示さない(攻撃者にヒント与えない)

### 2.6 禁則語(単独使用禁止)

「OK」「実行」「処理する」「確定」「送信」「こちら」「エラーが発生しました」「不正な値です」「失敗しました」

### 2.7 i18n キー化必須(基本設計書 §11.8 から移管)

- MVP は日本語のみだが、全メッセージは i18n キー化必須
- メッセージID = i18n キーとして直接利用可能な形式
- 多言語対応は Future(`04_将来拡張/01_将来要件定義書.md` 参照)

## 3. メッセージID 体系

`MSG-<対象ID>-<種別>-<連番>` 形式

| 種別コード | 種別 | 用途例 |
|---|---|---|
| `TITLE` | 画面タイトル | ページのヘッダ |
| `DESC` | 説明文 | 画面の説明 |
| `LBL` | ラベル | フォームの項目名 |
| `PH` | プレースホルダー | 入力欄のヒント |
| `BTN` | ボタン | アクション |
| `LINK` | リンク | テキストリンク |
| `STEP` | ステップ | Timeline 内のステップ |
| `HINT` | ヘルプ | ツールチップ |
| `TOOLTIP` | ツールチップ | hover ヒント |
| `WARN` | 警告 | 注意喚起 |
| `BANNER` | バナー | 強調帯 |
| `CHIP` | チップ | クイックフィルタ |
| `BADGE` | バッジ | ステータス表示 |
| `MSG` | 補足メッセージ | 注意書き |
| `SUCCESS` | 成功 | 完了表示 |
| `TOAST` | トースト | 完了通知バナー |
| `ERROR` | エラー | エラー表示 |
| `CONFIRM_TITLE` | 確認ダイアログタイトル | - |
| `CONFIRM_MSG` | 確認ダイアログ本文 | - |
| `CONFIRM_BTN` | 確認ダイアログ確定ボタン | - |
| `CONFIRM_LBL` | 確認ダイアログ内のラベル | - |
| `EMPTY` | 空状態 | データなし時 |
| `PERMISSION` | 権限不足表示 | - |
| `BTN_HINT` | ボタン横の補足 | - |
| `NAV` | ナビゲーション | 前後リンク等 |
| `NOTIF_SUBJECT` | 通知メール件名 | - |
| `NOTIF_BODY` | 通知メール本文 | - |

### 6.3 通知メール共通要件

- 送信元 From: `noreply@<service-domain>` 固定、Reply-To なし
- メール末尾に「このメールに直接返信せず、お手数ですが問い合わせ画面または各プロジェクトのお問い合わせ先からご返信ください」案内を含める
- エンドユーザー入力(質問本文等)は直接埋め込まない(FR-145)
- 引用時は HTML sanitize + 文字数制限 200 字 + サフィックスで明示
- HTML 版とテキスト版を両方生成
- 全テンプレートは i18n キー化、本文時刻表示は JST 固定(NFR-1103)
