## 1. 概要

| 項目 | 内容 |
|---|---|
| 概要 | MVP では 60 秒ポーリング / テキストチャットのみ / 管理者明示操作 / CSV インポート / レスポンシブ Web / メール + 受信箱に限定している通知・FAQ 運用・連携領域について、Post-MVP でリアルタイム化 / 添付・音声 / 多段承認・自動翻訳 / 外部ヘルプデスク連携 / スマホアプリ / 通知チャネル拡張を段階導入する。 |
| 背景 | エンドユーザー体験の即時性、FAQ 運用の自動化、既存ヘルプデスクからの移行支援、モバイル利用、到達性の高い通知チャネルが必要となる。 |
| 期待効果 | 解決率向上、FAQ 運用工数削減、既存サービスからの乗り換え促進、モバイル利用率向上、重要通知の到達率向上。 |
| 優先度(将来対応内) | 低 |
| 想定時期 | T3(GA 判定まで)〜 Post-MVP |

## 2. 関連要件・候補

## 4. 実施条件

| 項目 | 内容 |
|---|---|
| 実施判断条件 | リアルタイム化は接続維持コスト + 契約別新着件数 P95 + Cloudflare Workers 安定性、チャット拡張は保管容量 + マルウェア検査 + アクセシビリティ、FAQ 運用拡張は承認者数 + 翻訳品質 + 誤公開リスク、外部連携は対象サービス + データマッピング + サポート工数、スマホアプリはモバイル利用率 + 保守体制 + Push 要否、通知チャネル拡張は到達率 + コスト + ブランド要望 |
| 期限 | T3(お知らせリアルタイム化)、Post-MVP(その他全項目) |
| 依存関係 | 運営者システム側通知契機・テンプレート(FUT-02 運用・SLA・法務)との整合、共有概念の通知重要度 4 値(`low` / `normal` / `high` / `critical`)、IF #1〜#12 のメール送信(SendGrid 等)連携、4-eyes 承認基盤(FAQ 多段承認との整合) |
