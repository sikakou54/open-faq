1. 機能概要

項目 内容
概要 独立したチャット ID ごとの部屋でエンドユーザーと管理者ユーザーが匿名でメッセージをやり取りする。ブラウザへ直接発行する再入室トークン、投稿頻度制限、管理者向け通知を扱い、案件状態や FAQ 登録状態は持たない
利用場面 エンドユーザーと管理者ユーザーの個別メッセージ交換、ブラウザ保持トークンによる再入室、管理者向け通知連動
優先度 Post-MVP

2. 収録 FR-NNN 一覧

ID 将来要件 将来優先度
FR-080 未解決質問とは独立したチャット ID ごとに個別チャット部屋を作成できること。AI 未解決を契機に作成する場合は、メールアドレスを要求せず、ウィジェットへチャット ID と再入室トークンを返却すること P0
FR-081 AI 未解決時は画面遷移で会話履歴を失わせず、そのまま同じウィジェットの会話欄を個別チャットへ切り替え、「カスタマーサポートとのチャットに切り替わりました」と返信すること。チャット ID はヘッダー付近の常時表示領域と切替返信内の両方に表示すること P0
FR-082 サポートチャットはメールアドレス、氏名その他の連絡先登録を必須とせず、匿名で開始・投稿できること P0
FR-083 エンドユーザーはウィジェットのヘッダーメニューにある「チャットに再入室」または発行済み再入室 URL から、同じチャット部屋へ再入室できること。チャット ID 単独では本人認証に使用しないこと P0
FR-084 チャット部屋作成時に推測困難な再入室トークンを発行し、ウィジェットがブラウザへ保持できること。トークン有効期限は 30 日。期限切れ・紛失時はメールによる再発行を行わず、新しい問い合わせの開始を案内すること P0
FR-085 エンドユーザーはチャットにメッセージを投稿できること P0
FR-086 管理者ユーザーはチャットに返信できること P0
FR-086a 個別チャット部屋に担当管理者ユーザーを割り当てられること。担当者の指定は個別チャットのみの概念とし、未解決質問は担当者を持たない。担当者は 常に 1 名設定され、未割当(未設定)状態は許容しない。SCR-033 個別チャット部屋から、当該プロジェクトに割当のある管理者(オーナー / プロジェクト管理者 / メンバー)の誰でも担当者を変更できる P1
FR-086b エンドユーザーが新規チャット部屋を作成した時、当該プロジェクトの デフォルト担当者を自動で割り当てる こと。デフォルト担当者が未設定の場合は オーナーにフォールバック割当 する(オーナーは契約あたり常に 1 名存在し、全プロジェクトの管理者であるため担当者を必ず確定できる) P1
FR-086c プロジェクト単位で デフォルト担当者を 1 名設定できること。設定はチャット管理画面(SCR-034)から行い、操作はオーナー / プロジェクト管理者に限定する。デフォルト担当者には当該プロジェクトに割当のある管理者を指定する P1
FR-087 エンドユーザーがチャットへ投稿した場合、当該プロジェクトの管理者ユーザーへメールまたは受信箱で通知できること。管理者ユーザーからエンドユーザーへの返信はチャット内に表示し、エンドユーザーへのメール通知は行わないこと P0
FR-088 個別チャットは未解決質問・FAQ 登録状況と独立して管理し、チャット作成・投稿・既読・通知・再入室のいずれも未解決質問の状況を更新しないこと。また、個別チャット画面から FAQ 登録や未解決質問の状況変更を行えないこと P0
FR-089 個別チャットには業務上のオープン / クローズ状態、自動クローズ、手動クローズ、再オープン、利用者待ち判定を設けないこと。部屋とメッセージはデータ保持期間内で参照・投稿可能とし、再入室可否は FR-084 の本人確認で制御すること P1
FR-090 チャット投稿に上限文字数および投稿頻度制限を設定できること。MVP 初期値: 1 メッセージ文字数上限 = 2,000 文字、投稿頻度上限 = エンドユーザー 10 件/分・管理者ユーザー 無制限。契約別に運営者が調整可 P0
FR-091 機密情報(パスワード、決済情報等)入力を控えるよう注意喚起すること P0
FR-082a 不正利用防止のため、同一ウィジェットセッションが 24 時間に新規作成できるチャット部屋は 最大 5 件 とすること。超過時はブラウザに保持された既存チャットへの再入室を案内すること P1
FR-082b サポートチャットにはメールアドレス登録・変更 UI を提供せず、end_users のメール項目は未登録を許容すること P0
FR-082c エンドユーザーのメールアドレスを利用するチャット開始通知・管理者返信通知は送信しないこと P0

3. 補足説明・統合参照

8.8.1 未解決質問との機能境界

  • 未解決質問は FAQ 登録進捗を管理する。
  • 個別チャットはメッセージ交換を管理する。
  • 両機能は状態・操作・画面導線を相互依存させない。
  • 同一の質問ログを起点に各機能が作成された場合も、質問ログ ID を個別に保持するだけとし、相互の主従関係は作らない。
  • ウィジェット上では会話を連続表示するが、データ上は未解決質問 ID とチャット ID を別々に発行・保持する。
