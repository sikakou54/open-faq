# <span id="EVT-178"></span>EVT-178: 「パスワードを変更する」を押下

> **画面 SCR-022 のイベント「「パスワードを変更する」を押下」を定義します。** 対応画面・対応業務UC・対象項目・呼出API・遷移先と、処理内容を記述します。

*版数 v1.0 ・ 更新 2026-06-21 ・ 再構成 P3*

## 項目

| 項目 | 内容 |
|---|---|
| 画面イベントID | `EVT-178` |
| イベント名 | 「パスワードを変更する」を押下 |
| 対応画面ID | [SCR-022](../01_screens/SCR-022.md#SCR-022) |
| 対応業務UC | [UC-178](../../01_requirements/02_business_usecases/UC-178.md#UC-178) |
| 対象項目ID | [IT-03](../01_screens/SCR-022.md#IT-03) |
| 呼出API | [再認証(API-AUTH-005)](../03_apis/API-auth.md#API-AUTH-005) ・ [自己パスワード変更(API-AUTH-013)](../03_apis/API-auth.md#API-AUTH-013) |
| 遷移先 | — |

## 処理

<ul>
<li><a href="../03_apis/API-auth.md#API-AUTH-005">再認証(API-AUTH-005)</a> モーダルを表示し、現パスワードの入力を求める(<a href="../../01_requirements/01_specifications/FR-005.md#FR-005">FR-005</a>)</li>
<li>再認証成功後: 新パスワード・確認パスワードの入力フォームを表示し、送信で <a href="../03_apis/API-auth.md#API-AUTH-013">自己パスワード変更(API-AUTH-013)</a> を呼び出して新パスワードへ更新する(<a href="../../01_requirements/01_specifications/FR-006.md#FR-006">FR-006</a> — 12 文字以上・3 種以上)</li>
<li>失敗時(再認証失敗): モーダルにエラーを表示し変更しない</li>
<li>失敗時(パスワード複雑性不足): 新パスワード欄下部にエラーを表示し変更しない</li>
</ul>

## 備考

再構成 P3 で旧画面イベント `SCR-017/EV-06`(現 SCR-022 §6 の `EV-06`)から導出。処理内容の正本は SCR-022 §6 と本ページ。
