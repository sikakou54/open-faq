# <span id="EVT-130"></span>EVT-130: 割当解除の確認ダイアログで「外す」を押下

> **画面 SCR-014 のイベント「割当解除の確認ダイアログで「外す」を押下」を定義します。** 対応画面・対応業務UC・対象項目・呼出API・遷移先と、処理内容を記述します。

*版数 v1.0 ・ 更新 2026-06-21 ・ 再構成 P3*

## 項目

| 項目 | 内容 |
|---|---|
| 画面イベントID | `EVT-130` |
| イベント名 | 割当解除の確認ダイアログで「外す」を押下 |
| 対応画面ID | [SCR-014](../01_screens/SCR-014.md#SCR-014) |
| 対応業務UC | [UC-130](../../01_requirements/02_business_usecases/UC-130.md#UC-130) |
| 対象項目ID | [IT-13](../01_screens/SCR-014.md#IT-13) |
| 呼出API | [プロジェクト割当解除](../03_apis/API-member.md#API-MBR-004) |
| 遷移先 | [SCR-013](../01_screens/SCR-013.md#SCR-013) |

## 処理

<ul>
<li>成功時: <a href="../03_apis/API-member.md#API-MBR-004">プロジェクト割当解除</a> API で当該 PJ の割当を解除する(<code>valid=0</code>)。変更を監査記録し、当該メンバーへ通知する(FR-027)</li>
<li>最後の有効割当の場合: アカウントを論理削除し、全ログインセッションと未使用招待を無効化する(FR-031)</li>
<li>モーダルを閉じ SCR-013 の一覧を更新する</li>
<li>失敗時: エラートーストを表示し、モーダルを開いたままにする</li>
</ul>

## 備考

再構成 P3 で旧画面イベント `SCR-009-001/EV-08`(現 SCR-014 §6 の `EV-08`)から導出。処理内容の正本は SCR-014 §6 と本ページ。
