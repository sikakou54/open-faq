# <span id="EVT-128"></span>EVT-128: 「変更を保存する」を押下

> **画面 SCR-014 のイベント「「変更を保存する」を押下」を定義します。** 対応画面・対応業務UC・対象項目・呼出API・遷移先と、処理内容を記述します。

*版数 v1.0 ・ 更新 2026-06-21 ・ 再構成 P3*

## 項目

| 項目 | 内容 |
|---|---|
| 画面イベントID | `EVT-128` |
| イベント名 | 「変更を保存する」を押下 |
| 対応画面ID | [SCR-014](../01_screens/SCR-014.md#SCR-014) |
| 対応業務UC | [UC-128](../../01_requirements/02_business_usecases/UC-128.md#UC-128) |
| 対象項目ID | [IT-11](../01_screens/SCR-014.md#IT-11) |
| 呼出API | [メンバー情報更新](../03_apis/API-member.md#API-MBR-003) |
| 遷移先 | [SCR-013](../01_screens/SCR-013.md#SCR-013) |

## 処理

<ul>
<li>入力バリデーションを実行し、エラーがあればインラインエラーを表示して処理を中止する</li>
<li>成功時: <a href="../03_apis/API-member.md#API-MBR-003">メンバー情報更新</a> API で対象メンバーのメールアドレスを更新する。変更を監査記録し、当該メンバーへ通知する(FR-027)。モーダルを閉じ SCR-013 の一覧を更新する</li>
<li>失敗時: エラートーストを表示し、入力内容を保持する</li>
</ul>

## 備考

再構成 P3 で旧画面イベント `SCR-009-001/EV-06`(現 SCR-014 §6 の `EV-06`)から導出。処理内容の正本は SCR-014 §6 と本ページ。
