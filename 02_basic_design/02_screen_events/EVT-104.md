# <span id="EVT-104"></span>EVT-104: 「公開キーを再発行する」を押下

> **画面 SCR-011 のイベント「「公開キーを再発行する」を押下」を定義します。** 対応画面・対応業務UC・対象項目・呼出API・遷移先と、処理内容を記述します。

*版数 v1.0 ・ 更新 2026-06-21 ・ 再構成 P3*

## 項目

| 項目 | 内容 |
|---|---|
| 画面イベントID | `EVT-104` |
| イベント名 | 「公開キーを再発行する」を押下 |
| 対応画面ID | [SCR-011](../01_screens/SCR-011.md#SCR-011) |
| 対応業務UC | [UC-104](../../01_requirements/02_business_usecases/UC-104.md#UC-104) |
| 対象項目ID | [IT-04](../01_screens/SCR-011.md#IT-04) |
| 呼出API | [ウィジェット鍵ローテーション](../03_apis/API-project.md#API-PRJ-004) |
| 遷移先 | — |

## 処理

<ul>
<li>L3 確認(プロジェクト名タイプ + 再認証)ダイアログを表示する</li>
<li>確認完了後: <a href="../03_apis/API-project.md#API-PRJ-004">ウィジェット鍵ローテーション</a> API を呼び出し、新キーを発行して旧キーを 24 時間猶予で失効予告する。画面を再読み込みして新キーを表示する</li>
<li>確認キャンセル時: ダイアログを閉じ、何もしない</li>
<li>API エラー時: エラートーストを表示し、キーは変更しない</li>
</ul>

## 備考

再構成 P3 で旧画面イベント `SCR-007/EV-09`(現 SCR-011 §6 の `EV-09`)から導出。処理内容の正本は SCR-011 §6 と本ページ。
