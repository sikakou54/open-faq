# <span id="EVT-159"></span>EVT-159: 確認ダイアログの「OK」を押下

> **画面 SCR-019 のイベント「確認ダイアログの「OK」を押下」を定義します。** 対応画面・対応業務UC・対象項目・呼出API・遷移先と、処理内容を記述します。

*版数 v1.0 ・ 更新 2026-06-21 ・ 再構成 P3*

## 項目

| 項目 | 内容 |
|---|---|
| 画面イベントID | `EVT-159` |
| イベント名 | 確認ダイアログの「OK」を押下 |
| 対応画面ID | [SCR-019](../01_screens/SCR-019.md#SCR-019) |
| 対応業務UC | [UC-159](../../01_requirements/02_business_usecases/UC-159.md#UC-159) |
| 対象項目ID | — |
| 呼出API | [再認証](../03_apis/API-auth.md#API-AUTH-005) ・ [退会申請](../03_apis/API-terms.md#API-TRM-005) |
| 遷移先 | [SCR-022](../01_screens/SCR-022.md#SCR-022) |

## 処理

<ul>
<li><a href="../03_apis/API-auth.md#API-AUTH-005">再認証</a> API を呼び出し、本人確認を行う</li>
<li>再認証成功時: <a href="../03_apis/API-terms.md#API-TRM-005">退会申請</a> API を実行し、退会申請を登録して契約状態を <code>deleted_pending</code> へ更新する。完了後は SCR-022 個人設定へ遷移する</li>
<li>再認証失敗時: エラーメッセージを表示し、確認ダイアログへ戻る</li>
<li>退会申請 API エラー時: エラーメッセージを表示し、確認ダイアログへ戻る</li>
</ul>

## 備考

再構成 P3 で旧画面イベント `SCR-014/EV-04`(現 SCR-019 §6 の `EV-04`)から導出。処理内容の正本は SCR-019 §6 と本ページ。
