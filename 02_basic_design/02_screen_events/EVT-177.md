# <span id="EVT-177"></span>EVT-177: 「保存する」を押下(プロフィール)

> **画面 SCR-022 のイベント「「保存する」を押下(プロフィール)」を定義します。** 対応画面・対応業務UC・対象項目・呼出API・遷移先と、処理内容を記述します。

*版数 v1.0 ・ 更新 2026-06-21 ・ 再構成 P3*

## 項目

| 項目 | 内容 |
|---|---|
| 画面イベントID | `EVT-177` |
| イベント名 | 「保存する」を押下(プロフィール) |
| 対応画面ID | [SCR-022](../01_screens/SCR-022.md#SCR-022) |
| 対応業務UC | [UC-177](../../01_requirements/02_business_usecases/UC-177.md#UC-177) |
| 対象項目ID | [IT-06](../01_screens/SCR-022.md#IT-06) |
| 呼出API | [自己プロフィール更新(API-AUTH-012)](../03_apis/API-auth.md#API-AUTH-012) ・ [再認証(API-AUTH-005)](../03_apis/API-auth.md#API-AUTH-005) |
| 遷移先 | [SCR-018](../01_screens/SCR-018.md#SCR-018) |

## 処理

<ul>
<li>IT-01・IT-02 の値をサーバー側で検証し、<a href="../03_apis/API-auth.md#API-AUTH-012">自己プロフィール更新(API-AUTH-012)</a> を呼び出して自身のプロフィールを更新する</li>
<li>成功時(メールアドレス未変更): 保存完了メッセージを表示する</li>
<li>成功時(メールアドレス変更): <a href="../03_apis/API-auth.md#API-AUTH-005">再認証(API-AUTH-005)</a> で現パスワード再入力を求め、通過後に新メールアドレスへ確認メールを送信し、<a href="../01_screens/SCR-018.md">SCR-018 メールアドレス変更確認</a>フローへ引き渡す(<a href="../../01_requirements/01_specifications/FR-005.md#FR-005">FR-005</a>)</li>
<li>失敗時(バリデーションエラー): エラー内容を入力欄付近に表示し保存しない</li>
<li>失敗時(再認証失敗): 再認証モーダルにエラーを表示し保存しない</li>
</ul>

## 備考

再構成 P3 で旧画面イベント `SCR-017/EV-05`(現 SCR-022 §6 の `EV-05`)から導出。処理内容の正本は SCR-022 §6 と本ページ。
