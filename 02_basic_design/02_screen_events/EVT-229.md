# <span id="EVT-229"></span>EVT-229: 処理エラーを受信

> **画面 SCR-030 のイベント「処理エラーを受信」を定義します。** 対応画面・対応業務UC・対象項目・呼出API・遷移先と、処理内容を記述します。

*版数 v1.0 ・ 更新 2026-06-21 ・ 再構成 P3*

## 項目

| 項目 | 内容 |
|---|---|
| 画面イベントID | `EVT-229` |
| イベント名 | 処理エラーを受信 |
| 対応画面ID | [SCR-030](../01_screens/SCR-030.md#SCR-030) |
| 対応業務UC | [UC-229](../../01_requirements/02_business_usecases/UC-229.md#UC-229) |
| 対象項目ID | [IT-10](../01_screens/SCR-030.md#IT-10) |
| 呼出API | [ウィジェット起動](../03_apis/API-widget.md#API-WGT-001) ・ [ウィジェット質問送信](../03_apis/API-widget.md#API-WGT-002) |
| 遷移先 | — |

## 処理

<ul>
<li><a href="../03_apis/API-widget.md#API-WGT-001">ウィジェット起動</a> または <a href="../03_apis/API-widget.md#API-WGT-002">ウィジェット質問送信</a> API の処理エラー(通信障害・上流障害・認可エラー等)が発生した場合に発生する</li>
<li>エラーメッセージを会話欄または UI 内に表示する(IT-10)</li>
<li>再試行が妥当な場合は再試行案内を表示する(連打防止付き)</li>
<li>処理エラーは未解決質問として自動登録しない(未解決登録分岐とは区別する)</li>
</ul>

## 備考

再構成 P3 で旧画面イベント `SCR-WIDGET/EV-08`(現 SCR-030 §6 の `EV-08`)から導出。処理内容の正本は SCR-030 §6 と本ページ。
