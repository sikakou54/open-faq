## 概要

選択した複数の FAQ を一括で公開状態へ変更する。サーバーは行単位で変更し、成功件数と失敗明細を返したのち画面が一覧を再取得し、選択を解除する。

## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  actor U as 利用者
  participant Screen as FAQ 一覧
  participant Server as サーバー
  U->>Screen: 「公開する」を押下
  Screen->>Server: 選択 FAQ の一括公開要求
  Server->>Server: 認証・認可・入力検証
  alt 件数上限超過 / 不正な状態値
    Server-->>Screen: 入力エラー
    Screen-->>U: エラーを表示
  else 権限なし
    Server-->>Screen: 権限エラー
    Screen-->>U: 権限エラーを表示
  else 検証成功
    Server->>Server: 各 FAQ を公開へ変更・対象外は失敗集計
    Server-->>Screen: 成功件数 + 失敗明細
    Screen->>Server: 一覧の再取得要求
    Server->>Server: 認証・データ取得
    Server-->>Screen: 最新一覧
    alt 失敗あり
      Screen-->>U: 失敗件数をトースト表示・一覧更新・選択解除
    else 全件成功
      Screen-->>U: 一覧更新・選択解除
    end
  end
```

## 例外フロー

- 1 リクエストの件数上限を超過、または不正な状態値が指定された場合は入力エラーを返す。
- 当該プロジェクトへの権限がない場合は権限エラーを返す。
- 指定 FAQ が対象外（他契約 / 論理削除済み）の行は当該行のみ失敗として集計し、成功件数とともに失敗明細を返す。
