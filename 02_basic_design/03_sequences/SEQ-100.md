## 概要

連続ログイン失敗による総当たり攻撃を [システム仕様書 §3](../07_system-spec.md#3-タイムアウトセッション認証) のロックアウト条件で抑止し、ロック中の到達は認証せず拒否する。ロックは時間経過の自動解除または権限者の手動解除で解け、解除後は失敗回数を初期化して試行を再受付する。

## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  actor U as アカウント利用者
  actor M as 権限者(オーナー / メンバー)
  participant Server as サーバー
  participant DB as DB
  participant SCH as スケジューラ
  U->>Server: ログイン要求(認証情報)
  Server->>DB: 認証情報・データを取得(ロック状態の判定)
  DB-->>Server: 取得結果
  alt ロック中
    Server-->>U: 認証せず拒否(ロック中)
  else 未ロック・認証成功
    Server->>DB: 業務データを更新(失敗回数を初期化)
    DB-->>Server: 更新完了
    Server-->>U: ログイン成立
  else 未ロック・認証不一致
    Server->>DB: 業務データを更新(失敗回数を加算)
    DB-->>Server: 更新完了
    alt ロックアウト条件到達
      Server->>DB: 業務データを更新(ロックアウトを発動)
      DB-->>Server: 更新完了
      Server-->>U: ロック発動を通知(救済導線)
      Server-->>M: 必要な権限者へ通知
    else 上限未満
      Server-->>U: 認証情報が不正
    end
  end
  alt ロック解除条件を満たす(システム仕様書 §3)
    SCH->>Server: 期限経過の判定を起動
    Server->>DB: 業務データを更新(ロック解除・失敗回数を初期化)
    DB-->>Server: 更新完了
  else 権限者による手動解除
    M->>Server: ロック解除を実行
    Server->>DB: 業務データを更新(ロック解除・失敗回数を初期化)
    DB-->>Server: 更新完了
    Server-->>M: 解除完了を通知
  end
```

## 例外フロー

- ロック中に追加のログイン試行が到達しても認証は行わず、一律にロック中として拒否する。
- 解除直後に再び ロックアウト条件到達した場合は、改めて ロックアウトを発動する。
