## 概要

オーナーが請求画面を開いたときに、当月請求見込み・次回請求日・請求状態・プロジェクト別内訳・支払方法と請求履歴を取得して表示する。支払い失敗または支払方法未登録のときは復旧バナーを併せて表示する。

## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  actor U as オーナー
  participant Screen as 請求
  participant Server as サーバー
  U->>Screen: 画面を開く(初期表示)
  Screen->>Server: 請求サマリ・請求履歴取得要求
  Server->>Server: 認証・データ取得
  Server-->>Screen: 請求サマリ・請求履歴・請求状態
  alt 支払い失敗 / 支払方法未登録
    Screen-->>U: サマリ・履歴・復旧バナーを表示
  else 正常
    Screen-->>U: サマリ・履歴を表示
  end
```

## 備考

- 本図は基本設計レベルの抽象度(ユーザー / 画面 / サーバー、システム起点は外部システム・スケジューラ・バッチを加える)で記述する。DB 操作はサーバー自己メッセージで表し、テーブル別 CRUD は本図に書かず 関連テーブル 欄で示す。
- 図の出典は業務ユースケース [UC-037](../../01_requirements/04_business_usecases/UC-037.md#UC-037)。画面イベントとの対応は UC-037 を参照。
