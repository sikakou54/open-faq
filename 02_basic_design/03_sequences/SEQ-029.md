# <span id="SEQ-029"></span>SEQ-029: 「非公開化する」を押下

## 概要

FAQ 一覧で選択した複数の FAQ を、一括操作バーの「非公開化する」押下により一括で非公開状態へ変更する。サーバーで状態を更新後、一覧を再取得し選択を解除する。

## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  actor U as 利用者
  participant Screen as FAQ 一覧
  participant Server as サーバー
  participant DB as DB
  U->>Screen: 「非公開化する」を押下
  Screen->>Screen: 入力検証
  Screen->>Server: FAQ 一括非公開化を要求
  Server->>Server: 入力検証
  Server->>DB: 業務データを更新
  DB-->>Server: 更新完了
  alt 権限不足
    Server-->>Screen: 権限なしを応答
    Screen-->>U: 権限エラーを表示
  else 件数上限超過・不正な状態値
    Server-->>Screen: 検証エラーを応答
    Screen-->>U: 入力エラーを表示
  else 一部失敗あり
    Server-->>Screen: 成功件数と失敗明細を応答
    Screen->>Server: FAQ 一覧の再取得を要求
    Server->>DB: 認証情報・データを取得
    DB-->>Server: 取得結果
    Server-->>Screen: 最新の一覧を応答
    Screen-->>U: 失敗件数を表示・一覧更新・選択解除
  else 全件成功
    Server-->>Screen: 成功件数を応答
    Screen->>Server: FAQ 一覧の再取得を要求
    Server->>DB: 認証情報・データを取得
    DB-->>Server: 取得結果
    Server-->>Screen: 最新の一覧を応答
    Screen-->>U: 一覧更新・選択解除
  end
```

## 例外フロー

- 当該プロジェクトへの権限がない場合は権限エラーを表示し、状態を変更しない。
- 1 リクエストの件数上限を超過、または不正な状態値の場合は入力エラーを表示する。
- 対象外の FAQ（他プロジェクト / 論理削除済み）は当該行のみ失敗として集計し、失敗件数を一覧上で通知する。
