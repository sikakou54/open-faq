## 概要

お知らせ一覧で「表示中の未読を既読化」を押下し、現在のフィルタ条件で表示中の未読を一括既読にする。完了後に一覧を再取得し、未読件数を更新する。

## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  actor U as オーナー / メンバー
  participant Screen as お知らせ一覧
  participant Server as サーバー
  U->>Screen: 「表示中の未読を既読化」を押下
  Screen->>Server: 表示中の未読を一括既読にする要求
  Server->>Server: 認証・認可を検証
  Server->>Server: 件数上限を検証
  alt 件数上限超過
    Server-->>Screen: 入力エラー応答
    Screen-->>U: エラーをトースト表示
  else 検証成功
    Server->>Server: 業務処理・DB更新(一括既読・監査記録)
    Server-->>Screen: 既読件数を応答
    Screen->>Server: お知らせ一覧の再取得要求
    Server->>Server: 認証・データ取得
    Server-->>Screen: 最新の一覧を応答
    Screen-->>U: 一覧と未読件数を更新
  end
```

## 例外フロー

- 件数上限(1 リクエスト 100 件)を超過した場合は入力エラーを返し、画面はエラーをトースト表示する。
- 既読済みのお知らせは再度既読にしても結果が変わらない（冪等）。
