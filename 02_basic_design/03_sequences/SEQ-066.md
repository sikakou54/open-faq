## 概要

オーナーが退会内容の確認ダイアログで「OK」を押下し、再認証を経て退会申請を登録する。成功時は契約状態を退会保留へ更新し個人設定画面へ遷移、失敗時は確認ダイアログへ戻る。

## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  actor U as オーナー
  participant Screen as 退会申請
  participant Server as サーバー
  participant DB as DB
  U->>Screen: 確認ダイアログの「OK」を押下
  Screen->>Screen: 入力検証
  Screen->>Server: 再認証要求(パスワード)
  Server->>DB: 認証情報・データを取得
  DB-->>Server: 取得結果
  alt 再認証失敗
    Server-->>Screen: 認証エラー
    Screen-->>U: エラー表示・確認ダイアログへ戻る
  else 再認証成功
    Server-->>Screen: 再認証トークンを発行
    Screen->>Server: 退会申請要求
    Server->>Server: 認証・認可(オーナー専有)を検証
    alt 申請成功
      Server->>DB: 業務データを更新(退会申請登録・契約状態を退会保留へ更新)
      DB-->>Server: 更新完了
      Server-->>Screen: 申請完了
      Screen-->>U: 個人設定へ遷移
    else 権限不足 / 申請エラー
      Server-->>Screen: エラー
      Screen-->>U: エラー表示・確認ダイアログへ戻る
    end
  end
```

## 例外フロー

- 再認証でパスワードが不一致の場合は認証エラーを表示し、確認ダイアログへ戻る。
- オーナー以外(メンバー)が申請を試みた場合は権限不足として拒否し、エラーを表示する。
