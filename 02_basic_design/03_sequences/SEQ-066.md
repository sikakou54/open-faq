## 概要

未同意の規約改定がある状態で割込み表示された規約再同意割込み画面が、利用規約・プライバシーポリシーの最新版と差分サマリをサーバーから取得する。改定対象文書の主な変更点と同意チェックを全画面モーダルで表示し、改定対象外の文書の同意チェックは非表示にする。

## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  actor U as 利用者
  participant Screen as 規約再同意割込み
  participant Server as サーバー
  participant DB as DB
  U->>Screen: 割込み表示(初期表示)
  Screen->>Server: 利用規約 最新版取得要求
  Server->>DB: 認証情報・データを取得
  DB-->>Server: 取得結果
  Server-->>Screen: 利用規約(最新版・差分サマリ)
  Screen->>Server: プライバシーポリシー 最新版取得要求
  Server->>DB: 認証情報・データを取得
  DB-->>Server: 取得結果
  Server-->>Screen: プライバシーポリシー(最新版・差分サマリ)
  alt 取得成功
    Screen-->>U: 改定対象の変更点と同意チェックを表示(改定対象外は非表示)
  else 取得失敗
    Screen-->>U: 変更点を表示せずエラーを表示(モーダルは維持)
  end
```

## 例外フロー

- 利用規約またはプライバシーポリシーの取得に失敗した場合、変更点を表示せずエラーを表示する(モーダルは維持する)。
