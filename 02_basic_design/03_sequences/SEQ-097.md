## 概要

ウィジェット利用者の質問送信検証時に、当該プロジェクトの月次質問数が上限件数の 100% に到達しているかをサーバーが判定する。到達済みなら新規受付を停止して安全な定型文を返し入力・送信を無効化する一方、課金アカウント状態は `active` を維持し、未到達・上限オフなら通常どおり受け付ける。

## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  actor U as ウィジェット利用者
  participant Screen as エンドユーザー向け FAQ ウィジェット
  participant Server as サーバー
  participant DB as DB
  U->>Screen: 質問を送信
  Screen->>Screen: 入力検証
  Screen->>Server: 受付可否判定要求
  Server->>DB: 認証情報・データを取得
  DB-->>Server: 取得結果
  Server->>DB: 上限件数と当月利用量を照合
  DB-->>Server: 照合結果
  alt 100% 到達（上限オン）
    Server-->>Screen: 受付停止（安全な定型文・送信無効化）
    Screen-->>U: 連絡先案内の定型文を表示し入力・送信を無効化
  else 未到達 / 上限オフ
    Server-->>Screen: 受付可
    Screen-->>U: 通常どおり質問を受け付ける
  end
```

## 例外フロー

- **上限オフ / 未到達**: 上限がオフ、または 100% 未到達のプロジェクトは受付を停止せず通常受付する。
- **集計遅延（100% 超検知）**: 集計遅延・誤差で 100% 超を検知した場合も受付は停止のままとし、最終ガードの追加通知契機へ引き渡す。
