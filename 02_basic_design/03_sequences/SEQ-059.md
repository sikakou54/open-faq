## 概要

利用者が選択した複数のお知らせを「既読化する」操作で一括既読にし、一覧を再取得して未読件数を更新し、選択を解除して一括操作バーを非表示にする。

## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  actor U as 利用者
  participant Screen as お知らせ一覧
  participant Server as サーバー
  participant DB as DB
  U->>Screen: 選択行で「既読化する」を押下
  Screen->>Server: 一括既読要求(選択行)
  Server->>Server: 認証・認可・入力検証
  alt 件数上限超過
    Server-->>Screen: 入力エラー
    Screen-->>U: エラーをトースト表示
  else 正常
    Server->>DB: 業務データを更新
    DB-->>Server: 更新完了
    Server-->>Screen: 既読化完了
    Screen->>Server: 一覧再取得要求
    Server->>DB: 認証情報・データを取得
    DB-->>Server: 取得結果
    Server-->>Screen: 最新一覧
    Screen-->>U: 一覧・未読件数を更新し選択解除・一括操作バー非表示
  end
```

## 例外フロー

- 一括既読の件数が上限(1 リクエスト 100 件)を超える場合は入力エラーを返し、画面はトーストでエラーを表示する。
- 認証・認可に失敗した場合は処理を中断し、画面はエラーを表示する。
