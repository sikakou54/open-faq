## 概要

各機能へのリクエストが発生すると、サーバーがゲートウェイ層でリクエスト元の契約と機能種別を識別し、当該契約・種別に適用するレート制限の上限を契約単位で評価する。上限の範囲内であればリクエストの処理を許可して後段の業務処理へ渡し、上限を超過していれば当該リクエストを抑制して後段へ渡さない。いずれの場合もアクセス状況を記録し、超過抑制を濫用検知につなげて、DDoS・Bot・暴走による過負荷からサービスを防御する。本処理は全 API 横断のため特定の機能 API には結線しない。

## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  participant Caller as リクエスト元(全API横断)
  participant Server as サーバー
  Caller->>Server: 機能へのリクエスト
  Server->>Server: リクエスト元の契約と機能種別を識別する
  Server->>Server: 当該契約・種別の上限を取得する
  Server->>Server: 契約単位で適用状況を評価する
  alt 上限の範囲内
    Server->>Server: リクエストの処理を許可し後段へ渡す
    Server->>Server: アクセス状況を記録する
    Server-->>Caller: 処理を継続する
  else 上限超過
    Server->>Server: 当該リクエストを抑制し後段へ渡さない
    Server->>Server: 超過抑制をアクセス状況として記録する
    Server-->>Caller: リクエストを抑制した旨を返す
  end
```

## 備考
