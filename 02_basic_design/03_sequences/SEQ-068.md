## 概要

規約再同意割込み画面で「同意して続行する」を押下すると、改定対象の文書のみを対象にサーバーへ同意を記録し、割込み前の操作へ戻る。失敗時はモーダルを維持しエラーを表示する。

## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  actor U as 利用者
  participant Screen as 規約再同意割込み
  participant Server as サーバー
  participant DB as DB
  U->>Screen: 同意して続行を押下
  opt 利用規約が改定対象
    Screen->>Server: 利用規約への同意を記録要求
    Server->>Server: 認証・入力検証
    Server->>DB: 業務データを更新
    DB-->>Server: 更新完了
    Server-->>Screen: 記録結果
  end
  opt プライバシーポリシーが改定対象
    Screen->>Server: プライバシーポリシーへの同意を記録要求
    Server->>Server: 認証・入力検証
    Server->>DB: 業務データを更新
    DB-->>Server: 更新完了
    Server-->>Screen: 記録結果
  end
  alt 全て成功
    Screen-->>U: 割込み前の操作へ戻る
  else 失敗
    Screen-->>U: エラーを表示しモーダルを維持
  end
```

## 例外フロー

- 同意の記録に失敗した場合、エラーメッセージを表示し、モーダルはそのまま維持する。
