## 概要

招待メンバーが表示名・初回パスワード・規約同意を入力して登録を完了すると、サーバーが全項目を再検証し、予約利用者への表示名・パスワード設定、割当の有効化、規約同意記録、招待トークンの消費を同一トランザクションで実行する。成功時は完了画面を表示し、失敗時は内容を確定しない。

## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  actor U as 利用者
  participant Screen as メンバーアカウント有効化
  participant Server as サーバー
  participant DB as DB
  U->>Screen: 登録を完了するを押下
  Screen->>Screen: 入力検証
  Screen->>Server: 有効化要求(表示名・パスワード・規約同意)
  Server->>Server: 入力検証
  alt 検証 OK ・ トークン有効
    Server->>DB: 業務データを更新(表示名・パスワード設定・有効化・割当有効化・規約同意記録・トークン消費)
    DB-->>Server: 更新完了
    Server-->>Screen: 有効化完了
    Screen-->>U: 完了画面を表示
  else 入力エラー
    Server-->>Screen: フィールドエラー
    Screen-->>U: フィールドエラー表示(フォーム維持)
  else トークン無効 / 期限切れ
    Server-->>Screen: トークン無効
    Screen-->>U: エラー画面を表示
  end
```

## 例外フロー

- 入力エラー: 表示名長・パスワード強度・規約同意未チェックなどの検証に失敗した場合、フィールド単位のエラーを表示し、入力フォームを操作可能なまま維持する。
- トークン無効 / 期限切れ: 招待トークンが期限切れ・使用済みの場合、エラー画面を表示する。

## 詳細設計への移管候補

| 内容 | 移管先候補 | 理由 |
|---|---|---|
| 複数テーブル更新の同一トランザクション内の順序・ロールバック | 詳細設計 | 基本設計では 1 段に集約し、トランザクション内の実装順序は展開しない |
