## 概要

新パスワードの強度と一致を検証し、サーバーがパスワードハッシュを更新して当該ユーザーの全セッションを失効させ、完了画面を表示する。検証失敗やトークン無効時は確定を行わずエラーを表示する。

## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  actor U as 利用者
  participant Screen as パスワード再設定
  participant Server as サーバー
  participant DB as DB
  U->>Screen: 「新しいパスワードを設定する」を押下
  Screen->>Screen: 入力検証
  alt 入力検証成功
    Screen->>Server: パスワード再設定確定要求(トークン + 新パスワード)
    Server->>Server: 入力検証
    alt 確定成功
      Server->>DB: 業務データを更新
      DB-->>Server: 更新完了
      Server-->>Screen: 設定完了
      Screen-->>U: 完了画面を表示
    else トークン無効 / 期限切れ
      Server-->>Screen: トークンエラー
      Screen-->>U: トークンエラーと再送導線を表示
    end
  else 入力検証失敗
    Screen-->>U: 不足要件 / 不一致を表示
  end
```

## 例外フロー

- 強度不足 / 不一致: 不足要件メッセージ / 不一致エラーを表示してリクエストを中断する。
- トークン無効 / 期限切れ: 確定を行わず、エラーと再送導線を表示する。
- 確定失敗: 完了画面へ遷移せず、エラーを表示する。
