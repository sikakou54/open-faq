# <span id="SEQ-127"></span>SEQ-127: AIしきい値設定

> **このページは、AIしきい値設定のシーケンス図を定義します。** 回答可否の信頼度・関連度しきい値を 3 階層で取得・更新し、更新後はキャッシュへ伝播する。

| メッセージ (MSG) | — |

## 概要

回答可否の信頼度・関連度しきい値を 3 階層(グローバル / オーナー / プロジェクト)で取得・更新する。より具体的な階層を優先適用し、未設定の階層は上位を継承する。更新後はシステム処理 SYS-017 が AI しきい値キャッシュへ伝播する。割当のないプロジェクトへのアクセス・入力値不正・存在しないプロジェクトはエラーを返す。

## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  actor U as 認証済みユーザー
  participant Screen as ウィジェット設定
  participant Server as サーバー
  participant DB as DB
  U->>Screen: 画面を開く
  Screen->>Server: AI しきい値取得要求
  Server->>Server: 当該プロジェクトへの割当を判定
  Server->>DB: 3 階層のしきい値を取得
  DB-->>Server: 取得結果
  Server-->>Screen: 継承解決後の有効値・階層別しきい値
  Screen-->>U: 現在のしきい値を表示
  U->>Screen: しきい値を変更し保存
  Screen->>Server: AI しきい値更新要求
  Server->>Server: 入力値を検証
  alt 入力値が妥当
    Server->>DB: プロジェクト階層のしきい値を更新・キャッシュへ書き込み
    DB-->>Server: 更新完了
    Server-->>Screen: 更新結果
    Screen-->>U: 保存完了を表示
    Note over Server,DB: 更新後 SYS-017 がしきい値をキャッシュへ伝播
  else 入力値が不正
    Server-->>Screen: 検証エラー
    Screen-->>U: 入力エラーを表示
  end
```

## 例外フロー

- 当該プロジェクトに割当のないユーザーのアクセスはアクセス拒否エラーを返す。
- 入力値が不正な場合は検証エラーを返す。
- 対象プロジェクトが存在しない場合は未検出エラーを返す。

## 備考

- 本図は基本設計レベルの抽象度(ユーザー / 画面 / サーバー / DB)で記述する。DB 操作は DB アクターへのメッセージで表し、テーブル別 CRUD は本図に書かず 関連テーブル 欄で示す。
- しきい値更新後は [SYS-017](../02_backend/01_system/SYS-017.md#SYS-017) が `TP_AI_THRESH_CACHE` へ伝播し、AI 推論側はキャッシュをフォールバックで参照する。
