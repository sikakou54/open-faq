## 概要

スケジューラが日次で整合性検証バッチを起動し、監査ログを全件走査して改ざん検知情報の整合性を検証する。整合性違反を検出した対象はアラート通知し、違反が無ければ正常終了する。

## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  participant SCH as スケジューラ
  participant B as 整合性検証バッチ
  participant DB as DB
  participant OP as 運用担当
  SCH->>B: 日次起動
  B->>DB: 監査ログを全件走査
  DB-->>B: 走査結果
  B->>B: 改ざん検知情報の整合性を検証
  alt 整合性違反あり
    B-->>OP: 違反対象をアラート通知(走査は継続)
  else 違反なし
    B-->>SCH: 正常終了(違反なし)
  end
```

## 例外フロー

- 整合性違反の検出: 改ざん・欠落の疑いを検出した対象を特定し、アラート通知する。検証は中断せず残りの全件走査を継続する。
- 対象なし: 検証対象の監査ログが無い場合は検証を行わず、正常終了する。
