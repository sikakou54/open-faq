## 概要

システムは、通知送信の発生または定期監視を契機として、配信事業者からの配信結果をもとに契約・プロジェクト単位の送信品質を評価する。直近の送信レート・バウンス率・苦情率を集計して許容水準と照合し、いずれかの指標が許容水準を超えて悪化した契約・プロジェクト宛の通知送信を抑制状態にする。抑制の有無と対象を記録し、必要に応じて管理者へ知らせる。評価に足る実績が無い契約は判定対象から除外し、集計が未完了の場合は当該回の評価を見送って次の監視機会に再評価する。

## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  participant ESP as 配信事業者(外部)
  participant SCH as スケジューラ
  participant Server as サーバー
  ESP->>Server: 配信結果の通知(送信・バウンス・苦情)
  Server->>Server: 配信結果を契約・プロジェクト単位の送信実績へ反映する
  SCH->>Server: 定期の品質監視を起動する
  Note over Server: 通知送信の発生時にも評価を起動する
  loop 契約・プロジェクトごと
    alt 評価に足る送信実績がある
      alt 送信実績の集計が完了している
        Server->>Server: 直近の送信レート・バウンス率・苦情率を集計する
        Server->>Server: 各指標を許容水準と照合する
        alt いずれかの指標が許容水準を超えて悪化している
          Server->>Server: 当該契約・プロジェクト宛の通知送信を抑制状態にする
          Server->>Server: 抑制の有無と対象を記録する
          Server->>ESP: 抑制状態に基づき以後の通知送信を抑制する
          ESP-->>Server: 送信抑制の反映を受け付ける
          opt 管理者への通知が必要な場合
            Server->>ESP: 抑制を行った旨と対象を管理者へ通知する
            ESP-->>Server: 通知メールの受付を返す
          end
        else すべての指標が許容水準内
          Server->>Server: 送信を継続できる状態を維持する
        end
      else 集計が未完了
        Server->>Server: 当該回の評価を見送り次の監視機会に再評価する
      end
    else 評価に足る送信実績が無い
      Server->>Server: 当該契約を判定対象から除外する
    end
  end
```
