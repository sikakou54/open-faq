## 概要

サーバーは、お知らせ配信時に対象となるアカウント利用者ごとに受信者と受信箱を生成し、各受信箱の閲覧を本人に限定して本人専用の受信状態として保持する。アカウント利用者の退会・無効化が確定すると、サーバーは確定したアカウント情報をもとに対象利用者を特定し、当該利用者の受信箱と受信者を削除して、退会後に契約の機微情報が残らない状態へ収束させる。削除が一時的な障害で完了しなかった場合は、再度削除を試みて受信箱を残さない状態にする。

## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  participant DLV as お知らせ配信契機(外部)
  participant ACC as 退会・無効化の確定契機(外部)
  participant Server as サーバー

  DLV->>Server: お知らせ配信の発生(配信対象を含む)
  Server->>Server: 配信対象となる利用者を特定する
  loop 対象となる利用者ごと
    Server->>Server: 利用者ごとの受信者と受信箱を生成・DB更新
    Server->>Server: 受信箱の閲覧を本人に限定して保持する
  end
  Server-->>DLV: 利用者別の受信箱保持を完了

  ACC->>Server: 退会・無効化の確定(対象アカウントを含む)
  Server->>Server: 確定したアカウント情報をもとに削除対象の利用者を特定する
  Server->>Server: 当該利用者の受信箱・受信者を削除・DB更新
  alt 削除が正常に完了
    Server->>Server: 当該利用者のお知らせが残らない状態を確認する
  else 一時的な障害で未完了
    Server->>Server: 再度削除を試みて受信箱を残さない状態にする
  end
  Server-->>ACC: 受信箱の削除完了
```
