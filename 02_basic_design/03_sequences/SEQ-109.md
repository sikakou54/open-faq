## 概要

プロジェクトの新規作成または定期監視を契機に、サーバーが対象契約のプロジェクト数を集計し、想定外の急激な増加に該当するかを判定する。急増に該当する場合はオーナー向けの通知内容を生成して届け、結果を記録する。増加が緩やかで基準に達しない場合は通知せず終了し、通知の生成または送信に失敗した場合は再送対象として記録する。

## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  participant SCH as スケジューラ
  participant Server as サーバー
  Note over SCH: プロジェクト作成 または 定期監視を契機に起動
  SCH->>Server: 急増評価を起動
  Server->>Server: 対象契約のプロジェクト数を集計
  Server->>Server: 想定外の急増に該当するかを判定
  alt 急増に該当しない
    Server-->>SCH: 通知せず終了
  else 急増に該当する
    Server->>Server: オーナー向け通知内容を生成
    Server->>Server: オーナーへ通知を登録・配信
    alt 通知に成功
      Server->>Server: 通知結果を記録
      Server-->>SCH: 完了(通知済み)
    else 通知に失敗
      Server->>Server: 再送対象として記録
      Server-->>SCH: 完了(再送待ち)
    end
  end
```

## 例外フロー

- **通知の生成または送信失敗**: サーバーは当該通知を再送対象として記録し、後続の監視・再送で改めて通知できるようにする。1 件の失敗は他契約の評価・通知に影響しない。

## 備考
