## 概要

アカウント利用者が管理ダッシュボードへ遷移すると、システムは当人宛の未読お知らせ件数を取得して表示へ反映する。画面に滞在している間も、過大な負荷を生まない適度な間隔で未読件数を取得し直して最新化する。未読が 1 件もない場合は未読なしとして反映し、取得に一時的に失敗した場合は直前に表示していた件数を維持して次の取得タイミングで再取得する。

## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  participant TR as ダッシュボード遷移契機
  participant Server as サーバー
  participant DB as DB
  Note over Server: トリガ(管理ダッシュボードへの遷移)
  TR->>Server: 当人宛の未読お知らせ件数の取得を要求
  Server->>DB: 当人宛の未読状況を集計
  DB-->>Server: 集計結果
  alt 取得に成功
    alt 未読が 1 件以上ある
      Server-->>TR: 取得した未読件数を応答(表示へ反映)
    else 未読なし
      Server-->>TR: 未読なしを応答(表示へ反映)
    end
  else 取得に一時的に失敗
    Server-->>TR: 取得失敗を応答(直前の件数を維持)
  end
  loop 画面滞在中(過負荷を避ける適度な間隔)
    Note over Server: トリガ(定期取得タイミング)
    Server->>DB: 当人宛の未読状況を再集計
    DB-->>Server: 集計結果
    Server-->>TR: 最新の未読件数を応答(失敗時は直前の件数を維持)
  end
```
