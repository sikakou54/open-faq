## 概要

システムは、通知送信の発生または定期監視を契機として、配信事業者からの配信結果をもとにプロジェクト単位の送信品質を評価する。直近の送信レート・バウンス率・苦情率を集計して許容水準と照合し、いずれかの指標が許容水準を超えて悪化したプロジェクト宛の通知送信を抑制状態にする。抑制の有無と対象を記録し、必要に応じて運営へ知らせる。評価に足る実績が無いプロジェクトは判定対象から除外し、集計が未完了の場合は当該回の評価を見送って次の監視機会に再評価する。

## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  participant ESP as 配信事業者(外部)
  participant SCH as スケジューラ
  participant Server as サーバー
  participant DB as DB
  ESP->>Server: 配信結果の通知(送信・バウンス・苦情)
  Server->>DB: 配信結果をプロジェクト単位の送信実績へ反映する
  DB-->>Server: 反映完了
  SCH->>Server: 定期の品質監視を起動する
  Note over Server: 通知送信の発生時にも評価を起動する
  loop プロジェクトごと
    alt 評価に足る送信実績がある
      alt 送信実績の集計が完了している
        Server->>DB: 直近の送信レート・バウンス率・苦情率を集計する
        DB-->>Server: 集計結果
        Server->>Server: 各指標を許容水準と照合する
        alt いずれかの指標が許容水準を超えて悪化している
          Server->>DB: 当該プロジェクト宛の通知送信を抑制状態にする
          DB-->>Server: 更新完了
          Server->>DB: 抑制の有無と対象を記録する
          DB-->>Server: 記録完了
          Server->>ESP: 抑制状態に基づき以後の通知送信を抑制する
          ESP-->>Server: 送信抑制の反映を受け付ける
          opt 運営への通知が必要な場合
            Server->>DB: 抑制を行った旨と対象を運営向け通知として記録・生成する
            DB-->>Server: 記録完了
            Note over Server: 運営向けに抑制の実施と対象を知らせる通知を生成する
          end
        else すべての指標が許容水準内
          Server->>Server: 送信を継続できる状態を維持する
        end
      else 集計が未完了
        Server->>Server: 当該回の評価を見送り次の監視機会に再評価する
      end
    else 評価に足る送信実績が無い
      Server->>Server: 当該プロジェクトを判定対象から除外する
    end
  end
```
