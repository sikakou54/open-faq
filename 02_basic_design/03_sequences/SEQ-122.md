# <span id="SEQ-122"></span>SEQ-122: AIしきい値設定

> **このページは、AIしきい値設定のシーケンス図を定義します。** 回答可否の信頼度・関連度しきい値をプロジェクト単位で取得・更新し、更新後はキャッシュへ伝播する。

|----|----|----|----|
| SEQ-122 | [UC-075](../../01_requirements/04_business_usecases/UC-075.md#UC-075) | — | [TBL-004](../02_backend/04_database/TBL-004.md#TBL-004) ・ [TBL-031](../02_backend/04_database/TBL-031.md#TBL-031) |

## 概要

回答可否の信頼度・関連度しきい値をプロジェクト単位で取得・更新する。対象プロジェクトの設定値が登録されている場合はその値を使用し、未登録または取得失敗の場合はグローバル既定値を使用する。更新後はシステム処理 SYS-015 が AI しきい値キャッシュへ伝播する。割当のないプロジェクトへのアクセス・入力値不正・存在しないプロジェクトはエラーを返す。

## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  actor U as 認証済みユーザー
  participant Screen as ウィジェット設定
  participant Server as サーバー
  participant DB as DB
  U->>Screen: 画面を開く
  Screen->>Server: AI しきい値取得要求
  Server->>Server: 当該プロジェクトへの割当を判定
  Server->>DB: プロジェクト設定値を取得
  DB-->>Server: 取得結果
  Server-->>Screen: 適用値・layers(global/project)
  Screen-->>U: 現在のしきい値を表示
  U->>Screen: しきい値を変更し保存
  Screen->>Server: AI しきい値更新要求
  Server->>Server: 入力値を検証
  alt 入力値が妥当
    Server->>DB: プロジェクト設定値を更新・キャッシュへ書き込み
    DB-->>Server: 更新完了
    Server-->>Screen: 更新結果
    Screen-->>U: 保存完了を表示
    Note over Server,DB: 更新後 SYS-015 がしきい値をキャッシュへ伝播
  else 入力値が不正
    Server-->>Screen: 検証エラー
    Screen-->>U: 入力エラーを表示
  end
```

## 例外フロー

- 当該プロジェクトに割当のないユーザーのアクセスはアクセス拒否エラーを返す。
- 入力値が不正な場合は検証エラーを返す。
- 対象プロジェクトが存在しない場合は未検出エラーを返す。

## 備考

- 本図は基本設計レベルの抽象度(ユーザー / 画面 / サーバー / DB)で記述する。DB 操作は DB アクターへのメッセージで表し、テーブル別 CRUD は本図に書かず 関連テーブル 欄で示す。
- しきい値更新後は [SYS-015](../02_backend/01_system/SYS-015.md#SYS-015) が `TP_AI_THRESH_CACHE` へプロジェクト設定値を伝播し、AI 推論側は取得不能時にグローバル既定値へ直接フォールバックする。
