## 概要

アカウント利用者が FAQ 編集画面を開いたとき、起動モード（既存編集 / 未解決質問起点 / 手動新規）に応じたフォームを表示する。既存編集では指定 FAQ の現値を取得して各欄へ展開し、権限がない場合は権限不足メッセージを表示して操作不可とする。

## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  actor U as アカウント利用者
  participant Screen as FAQ 編集
  participant Server as サーバー
  participant DB as DB
  U->>Screen: 画面を開く（起動モード判定）
  alt 既存編集
    Screen->>Server: FAQ 個別取得を要求
    Server->>Server: 認証・認可検証
    alt 権限あり・対象あり
      Server->>DB: 認証情報・データを取得
      DB-->>Server: 取得結果
      Server-->>Screen: FAQ 現値
      Screen-->>U: 現値を各欄へ展開・表示
    else 権限不足
      Server-->>Screen: 権限不足
      Screen-->>U: 権限不足メッセージ（操作不可）
    else 対象なし
      Server-->>Screen: 対象なし
      Screen-->>U: 対象が存在しない旨を表示
    end
  else 未解決質問起点 / 手動新規
    Screen-->>U: 空フォーム（起点時は質問内容をプリフィル）
  else 権限なし
    Screen-->>U: 権限不足メッセージ（操作不可）
  end
```

## 例外フロー

- 当該プロジェクトへの権限がない場合は、権限不足メッセージを表示して操作不可とする（[ERR-019](../05_errors/ERR-019.md#ERR-019)）。
- 指定した FAQ が存在しない / 論理削除済みの場合は、対象が存在しない旨を表示する（[ERR-017](../05_errors/ERR-017.md#ERR-017)）。
