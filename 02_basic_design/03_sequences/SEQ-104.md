## 概要

アカウントの停止（手動停止 / 規約違反停止）イベントを契機に、対象アカウント利用者の全セッションを速やかに一斉無効化する。進行中の長時間処理は中断せず完了まで継続させ、停止後の次回操作で再認証を要求し停止時アクセス制限を適用する。

## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  participant OP as アカウント管理(システム)
  participant Server as サーバー
  participant DB as DB
  actor U as 対象アカウント利用者
  OP->>Server: 停止 / 規約違反停止イベントを連携
  Server->>DB: 認証情報・データを取得
  DB-->>Server: 取得結果
  alt 停止状態
    Server->>DB: 対象アカウントの全セッションを一斉無効化
    DB-->>Server: 更新完了
    Note over Server: 進行中の長時間処理は完了まで継続(中断しない)
    Server-->>OP: 無効化完了
  else 停止対象外
    Server-->>OP: 対象なしで終了
  end
  U->>Server: 次回操作
  Server->>DB: 認証情報・データを取得
  DB-->>Server: 取得結果
  Server-->>U: 再認証を要求 / 停止時アクセス制限を適用
```

## 例外フロー

- 一斉無効化の進行中に対象アカウント利用者の操作が到達しても、無効化済みのセッションでは再認証が要求される。
- 無効化時点で進行中の長時間処理は中断せず、完了後の次回操作で再認証を要求する。
