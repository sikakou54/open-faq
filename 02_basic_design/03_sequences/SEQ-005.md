## 概要

レート制限の解除後に利用者がパスワード再設定要求を再発行し、対象アカウントが存在する場合のみ再設定メールを再送するシーケンス。応答受取後はカウントダウンをリセットし、再送ボタンを再び非活性にする。

## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  actor U as 利用者
  participant Screen as パスワード再設定
  participant Server as サーバー
  participant DB as DB
  participant MAIL as メール配信IF
  U->>Screen: カウントダウン完了後に再送を押下
  Screen->>Server: パスワード再設定要求の再発行
  Server->>Server: 入力検証
  Server->>DB: 認証情報・データを取得(対象アカウント照合)
  DB-->>Server: 取得結果
  alt 対象アカウントあり
    Server->>DB: 業務データを更新(再設定トークン発行)
    DB-->>Server: 更新完了
    Server->>MAIL: 再設定リンクメールの送信を依頼
    MAIL-->>Server: 受理
  else 対象アカウントなし
    Server->>Server: 送信を行わず処理を継続
  end
  Server-->>Screen: 一律の受付応答(存在有無を含めない)
  Screen-->>U: カウントダウンをリセットし再送ボタンを非活性化
```
