## 概要

運用イベント等を契機に受信箱お知らせの生成要求が発生したとき、同一プロジェクト・同一イベント種別で 60 分以内に連続発火したお知らせを 1 件に集約し、重複したお知らせの生成を抑える。60 分窓を外れた発火や異なるイベント種別は別お知らせとして生成される。

## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  participant G as お知らせ生成要求
  participant Server as サーバー
  participant DB as DB
  G->>Server: 受信箱お知らせの生成を要求(プロジェクト / イベント種別)
  Server->>DB: 同一プロジェクト・同一種別の 60 分以内を照会
  DB-->>Server: 取得結果
  alt 60 分以内に既存あり
    Server->>DB: 既存お知らせを更新(1 件へ集約)
    DB-->>Server: 更新完了
  else 既存なし
    Server->>DB: 新規お知らせを生成
    DB-->>Server: 生成完了
  end
  Server-->>G: 反映結果
```
