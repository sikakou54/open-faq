## 概要

未解決質問の登録契機が発生したとき、サーバーが質問内容・未解決理由・発生日時を未解決質問として記録し、関連プロジェクトと元となった質問のやり取りを紐づけ、追跡用識別子を付与する。元の質問のやり取りを特定できない場合は、当該項目を空のまま記録する。

## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  participant T as 登録契機(外部)
  participant Server as サーバー
  participant DB as DB
  T->>Server: 未解決質問の登録契機(質問内容・未解決理由)
  Server->>Server: 登録対象の質問内容と未解決理由を受け取る
  Server->>DB: 質問内容・未解決理由・発生日時を記録
  DB-->>Server: 記録完了
  Server->>DB: 関連プロジェクトを紐づけ
  DB-->>Server: 更新完了
  Server->>DB: 追跡用識別子を付与
  DB-->>Server: 更新完了
  alt 元の質問のやり取りを特定できる
    Server->>DB: 元の質問のやり取りを紐づけ
    DB-->>Server: 更新完了
  else 特定できない
    Server->>DB: 当該項目を空のまま記録
    DB-->>Server: 記録完了
  end
  Server->>DB: FAQ 改善追跡の起点として履歴を記録
  DB-->>Server: 記録完了
  Server-->>T: 未解決質問の記録完了
```
