## 概要

認証済みの利用者が個人設定画面でパスワード変更を要求し、再認証ダイアログ(SCR-034)で再認証を通過したうえで新しいパスワードと確認入力を送信して自身のパスワードを更新する。

## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  actor U as 認証済みの利用者
  participant Screen as 個人設定
  participant Dialog as 再認証ダイアログ
  participant Server as サーバー
  participant DB as DB
  U->>Screen: 「パスワードを変更する」を押下
  Screen->>Dialog: 再認証を要求(割込み)
  U->>Dialog: 現在のパスワードを入力 → 確認
  Dialog->>Server: 再認証要求
  Server->>DB: 認証情報・データを取得
  DB-->>Server: 取得結果
  alt 再認証成功
    Server-->>Dialog: 再認証トークンを発行
    Dialog-->>Screen: 再認証トークンで変更を続行
    U->>Screen: 新パスワード・確認を入力 → 送信
    Screen->>Screen: 入力検証
    Screen->>Server: パスワード変更要求(再認証トークン)
    Server->>Server: 入力検証・再認証トークン検証
    alt 複雑性 OK ・ 確認一致
      Server->>DB: 業務データを更新
      DB-->>Server: 更新完了
      Server-->>Screen: 変更完了
      Screen-->>U: 変更完了を表示
    else 複雑性不足 ・ 確認不一致
      Server-->>Screen: 検証エラー
      Screen-->>U: 新パスワード欄にエラー表示
    end
  else 再認証失敗
    Server-->>Dialog: 再認証エラー
    Dialog-->>U: ダイアログにエラー表示・変更しない
  end
```

## 例外フロー

- 再認証で現在のパスワードが一致しない場合、再認証トークンを発行せず再認証ダイアログにエラーを表示し、パスワードを変更しない。
- 新パスワードが強度要件を満たさない、または確認入力と一致しない場合、検証エラーを返し新パスワード欄にエラーを表示する。
