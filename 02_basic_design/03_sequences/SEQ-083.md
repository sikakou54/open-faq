## 概要

利用者が設定画面を開くと、サーバーが認可を検証したうえでプロフィール名・アカウントのメールを取得し、各項目へ表示する。

## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  actor U as 利用者
  participant Screen as 個人設定
  participant Server as サーバー
  participant DB as DB
  U->>Screen: 画面を開く（初期表示）
  Screen->>Server: 設定（プロフィール）取得要求
  Server->>Server: 認証・認可を検証
  alt 認証済み
    Server->>DB: 認証情報・データを取得
    DB-->>Server: 取得結果
    Server-->>Screen: プロフィール名・アカウントのメール
    Screen-->>U: 各項目へ表示
  else 未認証
    Server-->>Screen: 権限不足
    Screen-->>U: 権限不足を表示し本画面を描画しない
  end
```

## 例外フロー

- 未認証の利用者が本画面へアクセスした場合は権限不足を表示し、本画面を描画しない。
