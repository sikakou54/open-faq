## 概要

CSV による FAQ の一括取込を非同期で受け付け、ジョブの進捗を逐次表示する。全件成功時はモーダルを閉じて FAQ 一覧を再読み込みし、部分失敗時は成功分を反映してエラー一覧に失敗行を表示する。

## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  actor U as 利用者
  participant Screen as FAQ 一覧
  participant Server as サーバー
  participant J as CSV 取込ジョブ
  U->>Screen: 「読み込みを開始」を押下
  Screen->>Screen: 入力検証
  Screen->>Server: CSV を送信して取込を要求
  Server->>Server: 認証・入力検証
  alt 受理形式が不正
    Server-->>Screen: 形式エラーを返す
    Screen-->>U: エラーメッセージを表示
  else 受理
    Server->>J: 取込ジョブを受け付け
    Server-->>Screen: 受付(処理中)
    Screen-->>U: 進捗バーを表示
    J->>J: 各行を新規 / 上書き / 失敗で判定し取り込む
    J->>J: 結果を集計
    alt 全件成功
      J-->>Screen: 完了通知
      Screen-->>U: 成功メッセージを表示・モーダルを閉じ FAQ 一覧を再読み込み
    else 部分失敗
      J-->>Screen: 完了通知(失敗行を含む)
      Screen-->>U: 成功分を反映・エラー一覧に失敗行を表示
    end
  end
```

## 例外フロー

- 受理形式が不正(CSV 以外 / 文字コード不正 / 上限超過)の場合は取込を受け付けず、形式エラーを表示する。
- 行内の `FAQ ID` が当該契約に存在しない場合は当該行を失敗とし、エラー一覧に失敗行を表示する。
