# <span id="SEQ-050"></span>SEQ-050: 割当解除の確認ダイアログで「外す」を押下

## 概要

確認ダイアログで「外す」を押下すると、当該プロジェクトの割当を解除し、変更を監査記録して当該メンバーへ通知する。最後の有効割当ならアカウントを論理削除し全セッションと未使用招待を無効化したうえで、モーダルを閉じて一覧を更新する。失敗時はモーダルを保持してエラーを表示する。

## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  actor U as オーナー / メンバー
  participant Screen as メンバー招待 / 編集モーダル(プロジェクト単位)
  participant Server as サーバー
  participant DB as DB
  U->>Screen: 確認ダイアログで「外す」を押下
  Screen->>Server: 割当解除を要求
  Server->>Server: 認証・認可・再認証を検証
  alt オーナー / 自分自身は解除不可
    Server-->>Screen: 権限不足エラー
    Screen-->>U: エラーを表示(モーダル保持)
  else 対象が存在しない
    Server-->>Screen: 対象なしエラー
    Screen-->>U: エラーを表示(モーダル保持)
  else 解除可能
    Server->>DB: 業務データを更新(当該割当を論理削除・監査記録)
    DB-->>Server: 更新完了
    opt 最後の有効割当
      Server->>DB: 業務データを更新(アカウント論理削除・全セッション失効・未使用招待失効)
      DB-->>Server: 更新完了
    end
    Server-->>Screen: 解除結果(アカウント無効化の有無)
    Screen-->>U: モーダルを閉じ一覧を更新
  end
```

## 例外フロー

- 解除対象がオーナーの場合は解除を拒否し、モーダルを保持してエラーを表示する。
- 解除対象が自分自身の場合は解除を拒否し、モーダルを保持してエラーを表示する。
- 当該プロジェクトへの権限がない場合は解除を拒否し、モーダルを保持してエラーを表示する。
- 対象の割当が存在しない場合は対象なしとしてエラーを表示する。
