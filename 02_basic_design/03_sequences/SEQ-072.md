## 概要

認証済みの利用者が個人設定画面でパスワード変更を要求し、再認証を通過したうえで新しいパスワードと確認入力を送信して自身のパスワードを更新する。

## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  actor U as 認証済みの利用者
  participant Screen as 個人設定
  participant Server as サーバー
  U->>Screen: 「パスワードを変更する」を押下
  Screen->>U: 再認証(現在のパスワード)を要求
  U->>Screen: 現在のパスワードを入力 → 送信
  Screen->>Server: 再認証要求
  Server->>Server: 認証・データ取得
  alt 再認証成功
    Server-->>Screen: 再認証トークンを発行
    U->>Screen: 新パスワード・確認を入力 → 送信
    Screen->>Screen: 入力検証
    Screen->>Server: パスワード変更要求
    Server->>Server: 入力検証
    alt 複雑性 OK ・ 確認一致
      Server->>Server: 業務処理・DB更新
      Server-->>Screen: 変更完了
      Screen-->>U: 変更完了を表示
    else 複雑性不足 ・ 確認不一致
      Server-->>Screen: 検証エラー
      Screen-->>U: 新パスワード欄にエラー表示
    end
  else 再認証失敗
    Server-->>Screen: 再認証エラー
    Screen-->>U: モーダルにエラー表示・変更しない
  end
```

## 例外フロー

- 再認証で現在のパスワードが一致しない場合、再認証トークンを発行せずモーダルにエラーを表示し、パスワードを変更しない。
- 新パスワードが強度要件を満たさない、または確認入力と一致しない場合、検証エラーを返し新パスワード欄にエラーを表示する。
