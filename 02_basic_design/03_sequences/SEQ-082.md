## 概要

オーナーが請求画面で支払方法の変更を要求し、再認証に成功した場合のみ支払方法を登録・更新する。完了後は画面の支払方法表示を最新化し、再認証失敗時は操作を中断する。

## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  actor U as オーナー
  participant Screen as 請求
  participant Server as サーバー
  U->>Screen: 「支払方法を変更」を押下
  Screen->>Server: 再認証要求(パスワード)
  Server->>Server: 入力検証
  Server->>Server: 認証・データ取得
  alt 再認証失敗
    Server-->>Screen: 認証エラー
    Screen-->>U: エラー表示・操作中断
  else 再認証成功
    Server-->>Screen: 再認証トークンを発行
    Screen-->>U: 支払方法入力を表示
    U->>Screen: 支払方法を入力・確定
    Screen->>Screen: 入力検証
    Screen->>Server: 支払方法の登録・更新要求
    Server->>Server: 認証・認可(オーナー専有)
    alt カードが拒否された
      Server-->>Screen: 決済エラー
      Screen-->>U: エラー表示
    else 登録・更新成功
      Server->>Server: 業務処理・DB更新
      Server-->>Screen: 更新完了(ブランド・下4桁)
      Screen-->>U: 支払方法表示を最新化
    end
  end
```

## 例外フロー

- 再認証でパスワードが不正な場合はエラーを表示し、支払方法の変更を中断する。
- オーナー以外が操作した場合は権限不足として拒否する。
- 入力値の検証に失敗した場合はエラーを表示する。
- 課金プロバイダがカードを拒否した場合は決済エラーを表示する。
