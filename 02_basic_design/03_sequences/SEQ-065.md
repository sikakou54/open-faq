# <span id="SEQ-065"></span>SEQ-065: 確認ダイアログの「退会を確定する」を押下

## 概要

オーナーが退会内容の確認ダイアログで「退会を確定する」を押下し、再認証を経て退会を即時実行する。成功時はアカウント状態を退会済みへ即時更新し、自分が作成したプロジェクトの停止（ウィジェット停止）と参加プロジェクトからの離脱を起動したうえで個人設定画面へ遷移、失敗時は確認ダイアログへ戻る。

## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  actor U as オーナー
  participant Screen as 退会
  participant Server as サーバー
  participant DB as DB
  U->>Screen: 確認ダイアログの「退会を確定する」を押下
  Screen->>Screen: 入力検証
  Screen->>Server: 再認証要求(パスワード)
  Server->>DB: 認証情報・データを取得
  DB-->>Server: 取得結果
  alt 再認証失敗
    Server-->>Screen: 認証エラー
    Screen-->>U: エラー表示・確認ダイアログへ戻る
  else 再認証成功
    Server-->>Screen: 再認証トークンを発行
    Screen->>Server: 退会実行要求
    Server->>Server: 認証・認可(オーナー専有)を検証
    alt 退会成功
      Server->>DB: 業務データを更新(アカウント状態を退会済みへ即時更新・作成PJの停止と参加PJ離脱を起動)
      DB-->>Server: 更新完了
      Server-->>Screen: 退会完了
      Screen-->>U: 個人設定へ遷移
    else 権限不足 / 退会エラー
      Server-->>Screen: エラー
      Screen-->>U: エラー表示・確認ダイアログへ戻る
    end
  end
```

## 例外フロー

- 再認証でパスワードが不一致の場合は認証エラーを表示し、確認ダイアログへ戻る。
- オーナー以外(メンバー)が退会を試みた場合は権限不足として拒否し、エラーを表示する。
