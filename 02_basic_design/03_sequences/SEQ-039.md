## 概要

オーナーがウィジェット設定画面で公開キーの再発行を要求し、強い本人確認の完了後に新しい公開キーを発行する。旧キーは 24 時間猶予で失効予告し、画面を再読み込みして新キーを表示する。確認キャンセル時・エラー時はキーを変更しない。

## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  actor U as 利用者
  participant Screen as ウィジェット設定
  participant Server as サーバー
  participant DB as DB
  U->>Screen: 「公開キーを再発行する」を押下
  Screen-->>U: 確認(プロジェクト名入力 + 再認証)を要求
  alt 確認をキャンセル
    U->>Screen: キャンセル
    Screen-->>U: キーを変更せず終了
  else 確認を完了
    U->>Screen: 確認を完了
    Screen->>Server: 公開キー再発行を要求
    Server->>Server: 認証・認可・再認証を検証
    alt 検証成功
      Server->>DB: 業務データを更新(新キー発行 / 旧キー24h失効予約)
      DB-->>Server: 更新完了
      Server-->>Screen: 新キーと失効予告を返す
      Screen-->>U: 画面を再読み込みし新キーを表示
    else 再認証失敗 / 権限不足 / エラー
      Server-->>Screen: エラーを返す
      Screen-->>U: エラーを表示(キー変更なし)
    end
  end
```

## 例外フロー

- 再認証に失敗した場合は公開キーを発行せず、画面はエラーを表示してキーを変更しない。
- オーナー以外（権限不足）の要求は拒否し、キーを変更しない。
- 発行処理がエラーで完了しなかった場合はキーを変更せず、画面はエラーを表示する。
