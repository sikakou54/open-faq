## 概要

メールアドレス確認・パスワード再設定・ロックアウト通知などアカウント認証に関わる通知の契機が発生すると、サーバーは当該通知を認証関連通知として識別し、受信拒否(オプトアウト)の対象外と判定する。利用者が受信拒否を設定していても送信対象から外さず、配信用の連携先を介して対象本人へ確実に送信し、本人の受信箱へ配置したうえで送信結果(成否)を記録する。一時的な送信障害で届かなかった場合は再送して確実な到達を図り、なりすましや乗っ取りのリスクから利用者を守る。

## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  participant TRG as 認証関連通知の契機(外部)
  participant Server as サーバー
  participant Email as メール配信IF(外部)
  TRG->>Server: 認証関連通知の契機(メール確認・パスワード再設定・ロックアウト等)
  Server->>Server: 当該通知をアカウント認証関連通知として識別する
  alt 認証関連通知である
    Server->>Server: 受信拒否(オプトアウト)の対象外と判定する
    Server->>Server: 受信拒否設定にかかわらず送信先のアカウント利用者を送信対象とする
    Server->>Email: 対象のアカウント利用者へ通知の配信を依頼する
    Email-->>Server: 配信の受付結果を返す
    Server->>Server: 本人の受信箱へ通知を配置する
    alt 送信に成功した
      Server->>Server: 送信結果(成功)を記録する
    else 一時的な送信障害が発生した
      Server->>Email: 再送して確実な到達を図る
      Email-->>Server: 配信の受付結果を返す
      Server->>Server: 送信結果(成否)を記録する
    end
  else 認証関連通知でない
    Server->>Server: 本処理の対象外とする
  end
```
