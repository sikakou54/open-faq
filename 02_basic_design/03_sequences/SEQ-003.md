## 概要

利用者が登録ボタンを押下すると、全項目を再検証し Turnstile トークンを添えて新規登録要求を送る。サーバーは検証・重複確認のうえ利用者と契約を作成して確認メールを送信し、成功時はメール確認画面へ遷移する。

## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  actor U as 利用者
  participant Screen as アカウント登録
  participant Server as サーバー
  participant MAIL as メール配信IF
  U->>Screen: 登録を押下
  Screen->>Screen: 入力検証
  Screen->>Server: 新規登録要求(Turnstile トークン同梱)
  Server->>Server: 入力検証
  alt 検証 OK ・ メール重複なし
    Server->>Server: 業務処理・DB更新
    Server->>MAIL: 確認メール送信依頼
    MAIL-->>Server: 受理
    Server-->>Screen: 登録受付
    Screen-->>U: メール確認画面へ遷移
  else メール重複 / Turnstile 失敗 / 入力エラー
    Server-->>Screen: エラー応答
    Screen-->>U: 重複エラー / Turnstile リセット / 共通エラーを表示
  end
```

## 例外フロー

- 入力検証エラー: 送信を中止し、該当フィールド直下にエラーを表示する。
- メール重複: 該当フィールドにエラーメッセージを表示し、アカウントを作成しない。
- Turnstile 検証失敗: フォーム上部にエラーを表示し、Turnstile ウィジェットをリセットする。
- その他失敗: フォーム上部に共通エラーメッセージを表示する。
