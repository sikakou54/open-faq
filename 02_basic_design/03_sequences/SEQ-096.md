## 概要

メンバーのプロジェクト別役割割当が追加・変更・剥奪されたことを契機に、当該メンバーへお知らせ受信箱(「運営お知らせ」種別・「通常」重要度)とメールで変更を通知する。宛先が解決できない場合は受信箱のみ生成し、メール送信は行わない。

## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  participant E as 割当変更イベント発生元
  participant Server as サーバー
  participant MAIL as メール配信IF
  actor M as メンバー
  E->>Server: 割当の追加 / 変更 / 剥奪イベント
  Server->>Server: 対象メンバー・変更内容を特定
  Server->>Server: 業務処理・DB更新(受信箱お知らせ生成)
  alt 宛先解決可
    Server->>MAIL: 割当変更通知メールを送信
    MAIL-->>M: 割当変更通知メール
    Server->>Server: 業務処理・DB更新(配信ログ記録)
  else 宛先解決不可
    Server-->>E: 受信箱のみ生成(メールなし)
  end
```

## 例外フロー

- **宛先解決不可**: 対象メンバーの通知宛先が解決できない場合は受信箱お知らせのみ生成し、メール送信は行わない。
- **メール配信失敗**: 受信箱お知らせは生成済みとし、メール送信失敗は配信ログに失敗として記録する。再送は通知再送ユースケースが扱う。
