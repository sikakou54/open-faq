## 概要

ウィジェット利用者の質問送信到達を契機に、サーバーが利用量をリアルタイムに計測し、上限オンのプロジェクトでは利用率に応じてアラート閾値・100% 停止・125% の各契機を判定する。集計結果は管理画面向け利用量サマリへ 5 分以内に反映される。

## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  actor U as ウィジェット利用者
  participant Screen as エンドユーザー向け FAQ ウィジェット
  participant Server as サーバー
  participant DB as DB
  U->>Screen: 質問を送信
  Screen->>Server: 質問リクエスト
  Server->>DB: 業務データを更新（当月質問数を計測。推論失敗は課金対象外として区別）
  DB-->>Server: 更新完了
  Server->>DB: 認証情報・データを取得（上限・選択閾値を参照）
  DB-->>Server: 取得結果
  alt 上限オン
    Server->>Server: 利用率を算出し閾値 / 100% / 125% を判定
    Server->>Server: 各契機を担当ユースケースへ引き渡し
  else 上限オフ
    Server->>Server: 計測のみ（判定なし）
  end
  Server->>DB: 利用量サマリへ反映（5 分以内）
  DB-->>Server: 反映完了
  Server-->>Screen: 応答を返却
  Screen-->>U: 回答を表示
```

## 例外フロー

- **入力不正**: 必須パラメータ不足など要求が不正な場合はエラーを返す（[ERR-001](../05_errors/ERR-001.md#ERR-001)）。
- **権限なし**: 管理画面サマリ参照時に当該プロジェクトへの権限がない場合は拒否する（[ERR-019](../05_errors/ERR-019.md#ERR-019)）。
- **上限オフ**: 質問数上限がオフのプロジェクトは利用率算出・閾値判定・停止を行わず、計測のみ行う。
- **推論失敗**: 推論失敗の質問はカウントするが課金対象外として区別して保持し、請求集計から除外する。
