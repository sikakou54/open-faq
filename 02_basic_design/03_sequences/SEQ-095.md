## 概要

運用イベント(利用上限接近・AI 利用上限到達・通知失敗急増・サスペンション・復元・規約改定・価格改定 等)の発生を契機に、対象アカウント利用者の受信箱へ「システム通知」のお知らせを自動生成し、メール対象の契機ではシステム通知メールも送信して配信ログを記録する。

## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  participant E as 運用イベント発生元
  participant Server as サーバー
  participant MAIL as メール配信IF
  actor U as アカウント利用者
  E->>Server: 運用イベント発生を通知
  Server->>Server: サブ契機判定・宛先解決
  alt 宛先あり
    Server->>Server: システム通知を受信箱へ生成
    alt メール対象契機
      Server->>MAIL: システム通知メールを送信
      MAIL-->>U: システム通知メール
      Server->>Server: 配信ログを記録
    else メール対象外契機
      Server-->>U: 受信箱のみで通知
    end
  else 宛先なし
    Server-->>E: 正常終了(通知なし)
  end
```

## 例外フロー

- **宛先なし**: 対象アカウント利用者が解決できない場合は通知を生成せず、正常終了する。
- **メール配信失敗**: 受信箱お知らせは生成済みとし、メール送信失敗は配信ログに失敗として記録する(再送は通知再送ユースケースが扱う)。
