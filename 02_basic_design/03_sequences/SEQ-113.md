## 概要

課金プロバイダから届く決済・契約状態の通知を受信し、送信元の正当性を検証する。検証を通過した通知のみを重複排除して受信履歴に記録し、契約・サブスクリプション・請求書へ反映する。送信元検証に失敗した通知は不正な受信として記録して取り込まず、重複した通知は取込をスキップし受信記録のみ残す。反映に失敗した通知は失敗として記録・通知し、再処理の対象とする。

## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  participant EXT as 課金プロバイダ(外部)
  participant Server as サーバー

  EXT->>Server: 決済・契約状態の通知を送信
  Server->>Server: 送信元の正当性を検証
  alt 検証に失敗
    Server->>Server: 不正な受信として記録
    Server-->>EXT: 受信応答(取込せず)
  else 検証を通過
    Server->>Server: 過去に受信済みか照合
    alt 重複した通知
      Server->>Server: 取込をスキップし受信記録のみ保持
      Server-->>EXT: 受信応答(処理済み)
    else 新規の通知
      Server->>Server: 受信履歴・取込前段として記録
      Server->>Server: 通知内容に応じて契約・決済状態へ反映
      alt 反映に成功
        Server->>Server: 取込完了として記録
        Server-->>EXT: 受信応答(取込完了)
      else 反映に失敗
        Server->>Server: 失敗として記録し再処理対象化
        Note over Server: 取込失敗を運用者へ通知
        Server-->>EXT: 受信応答(後続で再処理)
      end
    end
  end
```
