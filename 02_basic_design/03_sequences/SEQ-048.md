# <span id="SEQ-048"></span>SEQ-048: 「招待メールを再送する」を押下

> **このページは、業務ユースケース UC-019（「招待メールを再送する」を押下）のシーケンス図を定義します。**

## 概要

オーナーまたは当該プロジェクトのメンバーが招待中の対象者に対して招待メールを再送する。成功時は旧リンクを失効させ新トークン（7 日）を発行して再送し完了トーストを表示し、失敗時はエラートーストを表示する。

## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  actor U as 利用者
  participant Screen as メンバー招待 / 編集モーダル(プロジェクト単位)
  participant Server as サーバー
  participant DB as DB
  participant MAIL as メール配信IF
  U->>Screen: 「招待メールを再送する」を押下
  Screen->>Server: 招待メール再送要求
  Server->>Server: 認証・認可・再認証を検証
  alt オーナー / 自分自身が対象
    Server-->>Screen: 権限エラー応答
    Screen-->>U: エラートーストを表示
  else 対象が存在しない
    Server-->>Screen: 対象なし応答
    Screen-->>U: エラートーストを表示
  else 招待中のメンバー
    Server->>DB: 業務データを更新(旧トークン失効・新トークン発行)
    DB-->>Server: 更新完了
    Server->>MAIL: 招待メール再送を依頼
    MAIL-->>Server: 受付
    Server-->>Screen: 再送結果(有効期限を含む)
    Screen-->>U: 完了トーストを表示
  end
```

## 例外フロー

- オーナーまたは自分自身を対象に指定した場合、権限エラー（[ERR-021](../05_errors/ERR-021.md#ERR-021) / [ERR-022](../05_errors/ERR-022.md#ERR-022)）を返しエラートーストを表示する。
- 対象メンバーが存在しない場合、対象なしエラー（[ERR-017](../05_errors/ERR-017.md#ERR-017)）を返しエラートーストを表示する。
