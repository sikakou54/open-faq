## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  actor U as アカウント利用者
  participant API as API /v1
  participant DB as DB (D1)
  U->>API: 操作リクエスト
  API->>DB: T_SESSIONS(R) 有効性を照会(優先順位で判定)
  DB-->>API: 判定結果(有効 / 無操作30分 / 絶対12時間)
  alt 失効(タイムアウト)
    API->>DB: T_SESSIONS(D) 無効化
    DB-->>API: 反映結果
    API-->>U: 明示メッセージ(再ログインへ誘導)
  else 有効
    opt 重要操作で再認証未充足
      API-->>U: 再認証を要求(当該操作 1 回・15 分以内)
    end
    API-->>U: 操作を継続
  end
```

## 備考

- 本図は基本設計レベルの抽象度（利用者 / 画面 / API / DB / 外部・バッチ・通知、`テーブル名(CRUD)` 表記）で記述する。
- 図の出典は業務ユースケース [UC-242](../../01_requirements/02_business_usecases/UC-242.md#UC-242)。画面イベントとの対応は UC-242 を参照。
