## 概要

アカウント利用者の操作リクエストを契機に、失効の優先順位とタイムアウト（無操作・絶対）に従ってセッションの有効性を判定する。失効済みは無効化して再ログインへ誘導し、有効でも重要操作で再認証が未充足なら再認証を要求する。

## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  actor U as アカウント利用者
  participant Server as サーバー
  U->>Server: 操作リクエスト
  Server->>Server: 認証・データ取得
  Server->>Server: セッション有効性を判定(失効優先順位・タイムアウト)
  alt 失効(タイムアウト)
    Server->>Server: 業務処理・DB更新(セッション無効化)
    Server-->>U: 明示メッセージ(再ログインへ誘導)
  else 有効
    opt 重要操作で再認証未充足
      Server-->>U: 再認証を要求(当該操作 1 回・15 分以内)
    end
    Server-->>U: 操作を継続
  end
```

## 例外フロー

- **同時失効**: 操作実行中にセッションが失効した場合は当該操作を許可せず、明示メッセージで再ログインへ誘導する。
- **再認証期限切れ**: 再認証が 15 分を超過、または別操作で消費済みの場合は再認証未充足として要求する。

## 備考

- 本図は基本設計レベルの抽象度(ユーザー / 画面 / サーバー、システム起点は外部システム・スケジューラ・バッチを加える)で記述する。DB 操作はサーバー自己メッセージで表し、テーブル別 CRUD は本図に書かず 関連テーブル 欄で示す。
