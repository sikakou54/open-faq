## 概要

アカウント利用者が FAQ の保存を実行し、入力検証後に FAQ を新規作成または更新して FAQ 一覧へ遷移する。版衝突・件数上限超過時は遷移せずエラーを表示する。

## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  actor U as アカウント利用者
  participant Screen as FAQ 一覧
  participant Server as サーバー
  U->>Screen: 保存を押下
  Screen->>Screen: 入力検証
  alt 入力に不備あり
    Screen-->>U: エラー箇所を表示(保存しない)
  else 検証 OK
    Screen->>Server: FAQ 保存要求(新規作成 / 更新)
    Server->>Server: 入力検証
    Server->>Server: 業務処理・DB更新
    alt 保存成功
      Server-->>Screen: 保存結果
      Screen-->>U: FAQ 一覧へ遷移
    else 版衝突(楽観ロック)
      Server-->>Screen: 版不一致
      Screen-->>U: 再読み込みを促す
    else 件数上限超過
      Server-->>Screen: 上限超過
      Screen-->>U: エラーを表示(保存しない)
    end
  end
```

## 例外フロー

- 入力検証エラー: 質問・回答の必須範囲を満たさない場合は保存せずエラー箇所を表示する。
- 版衝突: 他者の更新と版が一致しない場合は保存せず、再読み込みを促す。
- 件数上限超過: FAQ 件数が上限を超える場合は保存せずエラーを表示する。
