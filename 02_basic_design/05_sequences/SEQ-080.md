# <span id="SEQ-080"></span>SEQ-080: 「保存」を押下

## シーケンス図

```mermaid
sequenceDiagram
  actor U as 利用者
  participant S as モーダル SCR-027
  participant API as API
  participant DB as DB
  U->>S: 「保存」を押下(IT-07)
  S->>S: バリデーション
  alt 検証エラー
    S-->>U: エラー表示(中断)
  else 検証 OK
    S-->>U: L3 再認証(パスワード再入力)を要求
    alt 再認証失敗
      U->>S: 再認証失敗
      S-->>U: E-AUTH-REAUTH-FAILED(保存中断)
    else 再認証成功
      U->>S: 再認証成功
      S->>API: PATCH /projects/{id}/quota-limits/questions(API-BIL-007)
      API->>DB: M_PRJ_QUOTA_LIMITS(U)
      alt 成功
        DB-->>API: 更新結果
        API-->>S: 200
        S-->>U: トースト表示・モーダルを閉じる
      else 失敗 / 403
        API-->>S: 4xx
        S-->>U: エラー表示(保存しない)
      end
    end
  end
```

## 備考

- 本図は基本設計レベルの抽象度（利用者 / 画面 / API / DB / 外部・バッチ・通知、`テーブル名(CRUD)` 表記）で記述する。
- 図の出典は業務ユースケース [UC-206](../../01_requirements/04_business_usecases/UC-206.md#UC-206)。画面イベントとの対応は UC-206 を参照。
