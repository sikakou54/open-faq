## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  participant SCH as スケジューラ
  participant B as 再送バッチ
  participant DB as DB (D1)
  participant MAIL as メール配信 IF
  SCH->>B: 定期起動
  B->>DB: H_NOTIF_LOGS(R) 再送対象を検出
  DB-->>B: 失敗・未達(上限未到達)の一覧
  alt 再送可(抑制外 かつ 上限未満)
    B->>MAIL: 通知メールを再送
    MAIL-->>B: 再送結果
    B->>DB: H_NOTIF_LOGS(U) 配信状態・再送回数を更新
  else 上限到達 / 抑制該当
    B->>DB: H_NOTIF_LOGS(U) 確定失敗として記録
  end
```

## 備考

- 本図は基本設計レベルの抽象度（利用者 / 画面 / API / DB / 外部・バッチ・通知、`テーブル名(CRUD)` 表記）で記述する。
