## 概要

お知らせ一覧で「すべての未読を既読化」を押下し、確認後にフィルタを無視して全未読を一括既読化する。完了後は一覧を再取得し、未読件数を更新する。

## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  actor U as 利用者
  participant Screen as お知らせ一覧
  participant Server as サーバー
  U->>Screen: 「すべての未読を既読化」を押下
  Screen-->>U: 確認ダイアログを表示
  alt キャンセル
    U->>Screen: キャンセル
    Screen-->>U: 状態不変(処理中断)
  else 確認 OK
    U->>Screen: 既読化を確認
    Screen->>Server: 全未読の一括既読要求
    Server->>Server: 認証・認可・入力検証
    alt 成功
      Server->>Server: 業務処理・DB更新(全未読を既読化・監査記録)
      Server-->>Screen: 既読化完了
      Screen->>Server: 一覧の再取得要求
      Server->>Server: 認証・データ取得
      Server-->>Screen: 最新一覧
      Screen-->>U: 一覧・未読件数を更新
    else 失敗
      Server-->>Screen: エラー
      Screen-->>U: エラーをトースト表示
    end
  end
```

## 例外フロー

- 一括既読の更新がサーバーで失敗した場合、状態を変えずエラーをトースト表示する。
- 既読済みのお知らせは冪等に扱い、二重既読でも状態は変わらない。
