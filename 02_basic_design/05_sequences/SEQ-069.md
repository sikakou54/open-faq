## シーケンス図

```mermaid
sequenceDiagram
  actor U as 利用者
  participant S as 画面 SCR-026 利用状況
  participant API as API
  participant DB as DB
  U->>S: 画面を開く(初期表示)
  S->>API: GET 利用量サマリ(契約)(API-BIL-002)
  API->>DB: M_PROJECTS(R) ・ T_USAGE_METER(R) ・ T_BILL_SUBS(R)
  DB-->>API: 契約全体サマリー
  API-->>S: 200 契約サマリー
  S->>API: GET 利用量サマリ(プロジェクト)(API-BIL-001)
  API->>DB: T_USAGE_METER(R) ・ M_PRJ_QUOTA_LIMITS(R)
  DB-->>API: プロジェクト別利用量
  API-->>S: 200 プロジェクト別
  alt プロジェクト 1 件以上
    S-->>U: 全体サマリー・プロジェクト別を表示(必要時 支払い警告)
  else 0 件
    S-->>U: 空状態(IT-04)を表示
  end
```

## 備考

- 本図は基本設計レベルの抽象度（利用者 / 画面 / API / DB / 外部・バッチ・通知、`テーブル名(CRUD)` 表記）で記述する。
- 図の出典は業務ユースケース [UC-170](../../01_requirements/04_business_usecases/UC-170.md#UC-170)。画面イベントとの対応は UC-170 を参照。
