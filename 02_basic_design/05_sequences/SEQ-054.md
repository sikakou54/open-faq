## シーケンス図

```mermaid
sequenceDiagram
  actor U as 利用者
  participant S as 画面 SCR-026 お知らせ一覧
  participant API as API
  participant DB as DB
  U->>S: 画面を開く(初期表示)
  S->>API: GET /me/announcements(未読のみ)(API-ANN-001)
  API->>DB: M_SERVICE_ANNOUNCE(R) ・ T_ANNOUNCE_RCPT(R)
  DB-->>API: 一覧データ
  API-->>S: 200 一覧(0 件もありうる)
  S->>API: GET /me/announcements/unread-summary(API-ANN-004)
  API->>DB: T_ANNOUNCE_RCPT(R)
  DB-->>API: 未読件数
  API-->>S: 200 未読件数
  alt 1 件以上
    S-->>U: 一覧・件数(IT-04)を表示
  else 0 件
    S-->>U: 空状態(IT-17)を表示
  end
```

## 備考

- 本図は基本設計レベルの抽象度（利用者 / 画面 / API / DB / 外部・バッチ・通知、`テーブル名(CRUD)` 表記）で記述する。
- 図の出典は業務ユースケース [UC-136](../../01_requirements/04_business_usecases/UC-136.md#UC-136)。画面イベントとの対応は UC-136 を参照。
