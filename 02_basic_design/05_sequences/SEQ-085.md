# <span id="SEQ-085"></span>SEQ-085: 初期表示

## 概要

オーナーが設定画面を開くと、サーバーがオーナー専有の認可を検証したうえで契約名・連絡先メール・タイムゾーンを取得し、各項目へ表示する。オーナー以外には権限不足を表示し本画面を描画しない。

## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  actor U as オーナー
  participant Screen as 設定
  participant Server as サーバー
  U->>Screen: 画面を開く（初期表示）
  Screen->>Server: 契約設定取得要求
  Server->>Server: 認証・認可（オーナー専有）を検証
  alt オーナー
    Server->>Server: 認証・データ取得
    Server-->>Screen: 契約名・連絡先メール・タイムゾーン
    Screen-->>U: 各項目へ表示
  else オーナー以外
    Server-->>Screen: 権限不足
    Screen-->>U: 権限不足を表示し本画面を描画しない
  end
```

## 例外フロー

- オーナー以外が本画面へアクセスした場合は権限不足を表示し、本画面を描画しない。

## 備考

- 本図は基本設計レベルの抽象度(ユーザー / 画面 / サーバー、システム起点は外部システム・スケジューラ・バッチを加える)で記述する。DB 操作はサーバー自己メッセージで表し、テーブル別 CRUD は本図に書かず 関連テーブル 欄で示す。
