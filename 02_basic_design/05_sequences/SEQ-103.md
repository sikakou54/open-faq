## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  actor U as アカウント利用者
  participant API as API /v1
  participant DB as DB (D1)
  participant M as 権限者(オーナー / メンバー)
  U->>API: POST /auth/login(認証情報)
  API->>DB: ロック状態 / 認証情報を照会
  DB-->>API: 照合結果(不一致 / ロック中)
  alt ロック中
    API-->>U: E-AUTH-LOCKED(認証せず拒否)
  else 不一致
    API->>DB: 失敗回数を加算
    alt 5 回連続失敗
      API->>DB: 15 分ロックを発動
      API-->>U: ロック発動を通知(救済導線)
      API-->>M: 必要な権限者へ通知
    end
  end
  alt 15 分経過
    API->>DB: ロックを自動解除(失敗回数を初期化)
  else 手動解除
    M->>API: ロック解除を実行
    API->>DB: ロックを解除(失敗回数を初期化)
  end
```

## 備考

- 本図は基本設計レベルの抽象度（利用者 / 画面 / API / DB / 外部・バッチ・通知、`テーブル名(CRUD)` 表記）で記述する。
- 図の出典は業務ユースケース [UC-243](../../01_requirements/02_business_usecases/UC-243.md#UC-243)。画面イベントとの対応は UC-243 を参照。
