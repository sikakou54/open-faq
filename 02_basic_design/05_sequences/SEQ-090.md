## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  actor U as アカウント利用者
  participant S as 画面 SCR-025
  participant API as API /v1
  participant J as インポートジョブ
  participant DB as DB (D1)
  U->>S: CSV をアップロード
  S->>API: POST /faqs/import (CSV)
  API->>API: 認証・認可 / 受理形式を検証
  alt 形式不正
    API-->>S: 415 / 400 (E-INPUT-CSV-INVALID)
    S-->>U: 受付エラーを表示
  else 受理
    API->>J: 取込ジョブを受け付け
    API-->>S: 202 + jobId (processing)
    S-->>U: 進捗を表示
    loop 各行(行単位)
      J->>DB: M_FAQS(CRU) 新規 / 上書き / 行失敗
      DB-->>J: 反映結果(行ごと独立)
    end
    J->>J: 成功 / 失敗件数と失敗明細を集計
    J-->>U: 完了通知(成功 N / 失敗 M ・ 失敗明細)
  end
```

## 備考

- 本図は基本設計レベルの抽象度（利用者 / 画面 / API / DB / 外部・バッチ・通知、`テーブル名(CRUD)` 表記）で記述する。
- 図の出典は業務ユースケース [UC-230](../../01_requirements/04_business_usecases/UC-230.md#UC-230)。画面イベントとの対応は UC-230 を参照。
