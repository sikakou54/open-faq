## 概要

お知らせ詳細を開いたときに、本文とメタ情報(種別・重要度・配信日時・既読日時)を表示する。初回表示(未読)のときは自動で既読化し、既読日時を更新表示する。

## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  actor U as 利用者
  participant Screen as お知らせ詳細
  participant Server as サーバー
  U->>Screen: 画面を開く(初期表示)
  Screen->>Server: お知らせ本文・メタ情報の取得要求
  Server->>Server: 認証・データ取得
  Server-->>Screen: 本文・メタ情報・既読状態
  alt 未読(初回表示)
    Screen->>Server: 既読化要求
    Server->>Server: 業務処理・DB更新
    Server-->>Screen: 既読日時
    Screen-->>U: 本文・メタ情報を表示し既読日時を更新
  else 既読済
    Screen-->>U: 本文・メタ情報・既読日時を表示
  end
```

## 備考

- 本図は基本設計レベルの抽象度(ユーザー / 画面 / サーバー、システム起点は外部システム・スケジューラ・バッチを加える)で記述する。DB 操作はサーバー自己メッセージで表し、テーブル別 CRUD は本図に書かず 関連テーブル 欄で示す。
