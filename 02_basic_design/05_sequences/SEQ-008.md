## シーケンス図

```mermaid
sequenceDiagram
  actor U as 利用者
  participant S as 画面 SCR-003
  participant API as API
  participant DB as DB
  U->>S: 設定を押下(IT-08)
  S->>S: IT-06 強度 ・ IT-07 一致を検証
  alt 検証成功
    S->>API: POST /auth/password-reset(トークン + 新パスワード)(API-AUTH-010)
    API->>DB: M_CONTRACT(U) / M_PRJ_USERS(U) パスワードハッシュ更新
    API->>DB: T_SESSIONS(U 全セッション失効)
    DB-->>API: 更新結果
    API-->>S: 200 設定完了
    S-->>U: IT-10 完了画面を表示
  else 検証失敗 / トークン無効
    S-->>U: 不足要件 / 不一致 / トークンエラーを表示
  end
```

## 備考

- 本図は基本設計レベルの抽象度（利用者 / 画面 / API / DB / 外部・バッチ・通知、`テーブル名(CRUD)` 表記）で記述する。
- 図の出典は業務ユースケース [UC-025](../../01_requirements/04_business_usecases/UC-025.md#UC-025)。画面イベントとの対応は UC-025 を参照。
