## 概要

当該プロジェクトへの割当を持たないログイン済みユーザーがメンバー画面の URL へ直接アクセスした際の認可判定フローを定義する。割当が無い場合は権限不足ガードを表示し、ダッシュボードへ戻る導線を提示する。

## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  actor U as 利用者
  participant Screen as メンバー(プロジェクト)
  participant Server as サーバー
  U->>Screen: URL へ直接アクセス
  Screen->>Server: メンバー一覧取得要求
  Server->>Server: 認証・認可判定
  alt 当該プロジェクトに割当あり
    Server->>Server: 認証・データ取得
    Server-->>Screen: メンバー一覧
    Screen-->>U: メンバー一覧を表示
  else 割当なし(権限不足)
    Server-->>Screen: 権限不足(アクセス不可)
    Screen-->>U: 権限不足ガード・戻る導線を表示
  end
```

## 例外フロー

- 当該プロジェクトへの割当が無い場合、サーバーはアクセスを拒否し、画面は権限不足ガードと「ダッシュボードへ戻る」導線を表示する。
