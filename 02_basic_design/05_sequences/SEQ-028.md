## シーケンス図

```mermaid
sequenceDiagram
  actor U as 利用者
  participant S as 画面 SCR-028 FAQ 一覧
  participant API as API
  participant DB as DB
  U->>S: 「公開する」を押下(IT-12)
  S->>API: POST /faqs/bulk-status(status=published)(API-FAQ-003)
  API->>DB: M_FAQS(RU)
  DB-->>API: 更新結果
  API-->>S: 200 成功件数 + 失敗明細
  S->>API: GET /faqs(再取得)(API-FAQ-001)
  API->>DB: M_FAQS(R)
  DB-->>API: 最新一覧
  API-->>S: 200 一覧
  alt 失敗あり
    S-->>U: 失敗件数をトースト表示・一覧更新・選択解除
  else 全件成功
    S-->>U: 一覧更新・選択解除
  end
```

## 備考

- 本図は基本設計レベルの抽象度（利用者 / 画面 / API / DB / 外部・バッチ・通知、`テーブル名(CRUD)` 表記）で記述する。
- 図の出典は業務ユースケース [UC-069](../../01_requirements/04_business_usecases/UC-069.md#UC-069)。画面イベントとの対応は UC-069 を参照。
