## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  actor U as 利用者
  participant S as 画面 SCR-001
  participant API as API /v1
  participant DB as DB (D1)
  U->>S: SCR-001 を開く
  S->>API: セッション検証要求
  API->>DB: T_SESSIONS(R) ・ M_CONTRACT(R) ・ M_PRJ_USERS(R)
  DB-->>API: セッション・利用者種別
  API-->>S: 認証状態(有効 / 無効)
  alt セッション有効
    S-->>U: 管理範囲に応じた初期画面へ遷移
  else 未認証 / 無効
    S-->>U: ログインフォームを表示
  end
```

## 備考

- 本図は基本設計レベルの抽象度（利用者 / 画面 / API / DB / 外部・バッチ・通知、`テーブル名(CRUD)` 表記）で記述する。
- 図の出典は業務ユースケース [UC-001](../../01_requirements/04_business_usecases/UC-001.md#UC-001)。画面イベントとの対応は UC-001 を参照。
