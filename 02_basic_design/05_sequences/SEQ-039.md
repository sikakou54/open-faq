## シーケンス図

```mermaid
sequenceDiagram
  actor U as 利用者
  participant S as 画面 SCR-026 ウィジェット設定
  participant API as API
  participant DB as DB
  U->>S: 「公開キーを再発行する」を押下(IT-04)
  S-->>U: L3 確認(プロジェクト名タイプ + 再認証)
  alt 確認キャンセル
    U->>S: キャンセル
    S-->>U: キー変更なし
  else 確認完了
    U->>S: L3 確認を完了
    S->>API: POST /projects/{id}/widget-key/rotate(API-PRJ-004)
    API->>DB: M_PROJECTS(U 新キー) ・ T_PRJ_LEGACY_KEYS(C 旧キー 24h)
    alt 成功
      DB-->>API: ローテーション結果
      API-->>S: 200 新キー
      S-->>U: 画面を再読み込みし新キーを表示
    else 失敗(再認証失敗 / 403 / エラー)
      API-->>S: 4xx
      S-->>U: エラートースト(キー変更なし)
    end
  end
```

- 本図は基本設計レベルの抽象度（利用者 / 画面 / API / DB / 外部・バッチ・通知、`テーブル名(CRUD)` 表記）で記述する。
