## シーケンス図

```mermaid
sequenceDiagram
  actor U as 利用者
  participant S as 画面 SCR-028 個人設定
  participant API as API
  participant DB as DB
  U->>S: 「パスワードを変更する」を押下(IT-03)
  S->>API: POST 再認証(API-AUTH-005)
  API->>DB: M_CONTRACT / M_PRJ_USERS(R)
  DB-->>API: 再認証結果
  alt 再認証成功
    API-->>S: 再認証トークン
    U->>S: 新パスワード・確認を入力 → 送信
    S->>API: 自己パスワード変更(API-AUTH-013)
    API->>DB: M_CONTRACT / M_PRJ_USERS(U)
    DB-->>API: 更新結果
    alt 複雑性 OK
      API-->>S: 200 変更完了
      S-->>U: 変更完了を表示
    else 複雑性不足
      API-->>S: 検証エラー
      S-->>U: 新パスワード欄にエラー表示
    end
  else 再認証失敗
    API-->>S: 再認証エラー
    S-->>U: モーダルにエラー表示・変更しない
  end
```

## 備考

- 本図は基本設計レベルの抽象度（利用者 / 画面 / API / DB / 外部・バッチ・通知、`テーブル名(CRUD)` 表記）で記述する。
- 図の出典は業務ユースケース [UC-178](../../01_requirements/02_business_usecases/UC-178.md#UC-178)。画面イベントとの対応は UC-178 を参照。
