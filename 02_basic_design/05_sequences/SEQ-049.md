# <span id="SEQ-049"></span>SEQ-049: 「招待メールを再送する」を押下

## 概要

オーナーまたは当該プロジェクトのメンバーが招待中の対象者に対して招待メールを再送する。成功時は旧リンクを失効させ新トークン（7 日）を発行して再送し完了トーストを表示し、失敗時はエラートーストを表示する。

## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  actor U as 利用者
  participant Screen as メンバー招待 / 編集モーダル(プロジェクト単位)
  participant Server as サーバー
  participant MAIL as メール配信IF
  U->>Screen: 「招待メールを再送する」を押下
  Screen->>Server: 招待メール再送要求
  Server->>Server: 認証・認可・再認証を検証
  alt オーナー / 自分自身が対象
    Server-->>Screen: 権限エラー応答
    Screen-->>U: エラートーストを表示
  else 対象が存在しない
    Server-->>Screen: 対象なし応答
    Screen-->>U: エラートーストを表示
  else 招待中のメンバー
    Server->>Server: 業務処理・DB更新(旧トークン失効・新トークン発行)
    Server->>MAIL: 招待メール再送を依頼
    MAIL-->>Server: 受付
    Server-->>Screen: 再送結果(有効期限を含む)
    Screen-->>U: 完了トーストを表示
  end
```

## 例外フロー

- オーナーまたは自分自身を対象に指定した場合、権限エラー（[ERR-023](../07_errors/ERR-023.md#ERR-023) / [ERR-024](../07_errors/ERR-024.md#ERR-024)）を返しエラートーストを表示する。
- 対象メンバーが存在しない場合、対象なしエラー（[ERR-019](../07_errors/ERR-019.md#ERR-019)）を返しエラートーストを表示する。
