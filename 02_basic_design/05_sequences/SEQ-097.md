## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  participant E as 上限到達イベント発生元
  participant A as アラート通知処理
  participant DB as DB (D1)
  participant MAIL as メール配信 IF
  actor R as オーナー / メンバー
  E->>A: 選択閾値への到達を検知
  A->>DB: M_PRJ_QUOTA_LIMITS(R) / T_USAGE_METER(R) 参照
  DB-->>A: 上限・選択閾値・当月質問数
  alt 当月初回到達 かつ 閾値選択あり
    A->>A: 宛先(オーナー + 有効メンバー)を解決・重複排除
    A->>DB: T_INBOX_MSG(C) 上限アラートお知らせ生成
    A->>MAIL: 上限アラートメールを送信
    MAIL-->>R: 上限アラートメール
    A->>DB: H_NOTIF_LOGS(C) 配信ログ記録
  else 再到達 / 未選択
    A-->>E: 通知なし(冪等スキップ)
  end
```
