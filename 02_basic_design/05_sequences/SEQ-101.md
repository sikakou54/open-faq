## シーケンス図

```mermaid
sequenceDiagram
  autonumber
  participant P as 課金プロバイダ(外部)
  participant API as API /v1
  participant SCH as スケジューラ
  participant B as サスペンション判定バッチ
  participant DB as DB (D1)
  participant O as オーナー
  P->>API: 決済失敗確定 Webhook(署名付き)
  API->>API: 署名を検証
  alt 署名不正 / 重複受信
    API-->>P: 拒否 / 冪等にスキップ(状態変更なし)
  else 新規イベント
    API->>DB: 受信時刻を起点に 7 日猶予を開始(M_CONTRACT status=active 維持)
    DB-->>API: 反映結果
    API->>O: critical 通知(管理画面ログイン・支払方法更新は許可)
  end
  SCH->>B: 定期起動(猶予経過を判定)
  B->>DB: 猶予中契約の経過時間を判定
  DB-->>B: 7 日経過 / 未経過
  alt 7 日経過・再決済未成立
    B->>DB: M_CONTRACT(U) status=suspended
    DB-->>B: 反映結果
  end
  P->>API: 再決済成功 / 解除 Webhook
  API->>DB: M_CONTRACT(U) status=active(即時復帰)
  DB-->>API: 反映結果
```

## 備考

- 本図は基本設計レベルの抽象度（利用者 / 画面 / API / DB / 外部・バッチ・通知、`テーブル名(CRUD)` 表記）で記述する。
- 図の出典は業務ユースケース [UC-241](../../01_requirements/02_business_usecases/UC-241.md#UC-241)。画面イベントとの対応は UC-241 を参照。
