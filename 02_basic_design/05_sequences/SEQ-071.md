## シーケンス図

```mermaid
sequenceDiagram
  actor U as 利用者
  participant S as 画面 SCR-028 個人設定
  participant API as API
  participant DB as DB
  U->>S: 「保存する」を押下(IT-06)
  S->>S: IT-01・IT-02 を検証
  alt メールアドレス変更あり
    S->>API: POST 再認証(API-AUTH-005)
    API->>DB: M_CONTRACT / M_PRJ_USERS(R)
    DB-->>API: 再認証結果
    API-->>S: 再認証トークン / エラー
  end
  S->>API: 自己プロフィール更新(API-AUTH-012)
  API->>DB: M_CONTRACT / M_PRJ_USERS(U)
  DB-->>API: 更新結果
  alt 更新成功(メール未変更)
    API-->>S: 200 更新完了
    S-->>U: 保存完了を表示
  else 更新成功(メール変更)
    API-->>S: 200 + 確認メール送信
    S-->>U: SCR-029 確認フローへ引き渡し
  else 失敗
    API-->>S: 検証エラー / 再認証失敗
    S-->>U: エラー表示・保存しない
  end
```

## 備考

- 本図は基本設計レベルの抽象度（利用者 / 画面 / API / DB / 外部・バッチ・通知、`テーブル名(CRUD)` 表記）で記述する。
- 図の出典は業務ユースケース [UC-177](../../01_requirements/04_business_usecases/UC-177.md#UC-177)。画面イベントとの対応は UC-177 を参照。
