## シーケンス図

```mermaid
sequenceDiagram
  actor U as 利用者
  participant S as 画面 SCR-028 請求
  participant API as API
  participant DB as DB
  U->>S: 画面を開く(初期表示)
  S->>API: GET /billing/summary(API-BIL-003)
  API->>DB: T_BILL_SUBS(R) ・ T_USAGE_METER(R)
  DB-->>API: 請求サマリ・内訳
  API-->>S: 200 サマリ
  S->>API: GET /billing/invoices(API-BIL-004)
  API->>DB: T_BILL_INVOICES(R)
  DB-->>API: 請求履歴
  API-->>S: 200 履歴
  alt 支払い失敗 / 未登録
    S-->>U: サマリ・履歴・復旧バナー(IT-09)を表示
  else 正常
    S-->>U: サマリ・履歴を表示
  end
```

## 備考

- 本図は基本設計レベルの抽象度（利用者 / 画面 / API / DB / 外部・バッチ・通知、`テーブル名(CRUD)` 表記）で記述する。
- 図の出典は業務ユースケース [UC-208](../../01_requirements/02_business_usecases/UC-208.md#UC-208)。画面イベントとの対応は UC-208 を参照。
