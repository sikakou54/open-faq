# <span id="API-024"></span>API-024: 招待メール再送

> **このページは 招待メール再送 API の契約を定義します。** `status='pending_activation'` のメンバーへ招待メールを再送する。

*版数 v2.0 ・ 更新 2026-06-21 ・ 再構成 P4*

## 項目

| 項目 | 内容 |
|---|---|
| API ID | `API-024` |
| API名 | 招待メール再送 |
| 対応業務UC | [UC-019](../../01_requirements/04_business_usecases/UC-019.md#UC-019) |
| 対応画面ID | [SCR-014](../01_screens/SCR-014.md#SCR-014) |
| 対応画面イベントID | [EVT-127](../02_screen_events/EVT-127.md#EVT-127) |
| エンドポイント | `/members/{id}/resend-invitation` |
| HTTPメソッド | POST |
| 認証 | Cookie + CSRF + 再認証必須 |
| 認可 | オーナー / 当該プロジェクトのメンバー |

## 処理概要

`status='pending_activation'` のメンバーへ招待メールを再送する。

| 処理 ID | 処理内容 |
|----|----|
| P-01 | 認証・認可(オーナー / 当該プロジェクトのメンバー)と再認証を検証する |
| P-02 | 対象が未有効化(`pending_activation`)のメンバーであることを確認する(オーナー / 自身は対象外) |
| P-03 | 旧有効化トークンを失効させ、同一の招待先情報で新規トークンを再発行する |
| P-04 | 招待メールを再送し再発行結果を返す |

## リクエスト

#### Query Parameters

```json
{}
```

#### Path Parameters

```json
{ "id": "メンバーユーザー ID" }
```

| 項目 | 型 | 説明 |
|----|----|----|
| `id` | string | 招待メールを再送する対象メンバーの利用者 ID(必須・パス内で指定) |

#### Request Body

```json
{}
```

氏名 `displayName` を受け付けない(FR-020 / FR-022)。

## レスポンス

**レスポンス(200)**

再発行結果(`{ "ok": true, "expiresAt": "..." }`)。

| 項目        | 型                | 説明                             |
|-------------|-------------------|----------------------------------|
| `ok`        | boolean           | 招待メール再送の成否             |
| `expiresAt` | string (ISO 8601) | 再発行した招待トークンの有効期限 |

## バリデーション

入力値の検証規則は `## リクエスト` の各パラメータ説明を参照する。
