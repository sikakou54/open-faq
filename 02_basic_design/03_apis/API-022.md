# <span id="API-022"></span>API-022: メンバー情報更新

> **このページは メンバー情報更新 API の契約を定義します。** 対象メンバーのメールアドレスを更新する。

*版数 v2.0 ・ 更新 2026-06-21 ・ 再構成 P4*

## 項目

| 項目 | 内容 |
|---|---|
| API ID | `API-022` |
| API名 | メンバー情報更新 |
| 対応業務UC | [UC-128](../../01_requirements/04_business_usecases/UC-128.md#UC-128) |
| 対応画面ID | [SCR-014](../01_screens/SCR-014.md#SCR-014) |
| 対応画面イベントID | [EVT-128](../02_screen_events/EVT-128.md#EVT-128) |
| エンドポイント | `/projects/{id}/members/{userId}` |
| HTTPメソッド | PATCH |
| 認証 | Cookie + CSRF + 再認証必須 |
| 認可 | オーナー / 当該プロジェクトのメンバー |

## 処理概要

対象メンバーのメールアドレスを更新する。

| 処理 ID | 処理内容                                                         |
|---------|------------------------------------------------------------------|
| P-01    | 認証・認可(オーナー / 当該プロジェクトのメンバー)と再認証を検証する  |
| P-02    | 対象割当の存在と入力値(メール形式)を検証する                    |
| P-03    | メールアドレスの重複を判定し該当時は拒否する                     |
| P-04    | 対象メンバーのメールアドレスを更新する                           |
| P-05    | 更新結果を返す                                                   |

## リクエスト

#### Query Parameters

```json
{}
```

#### Path Parameters

```json
{
  "id": "プロジェクト ID",
  "userId": "対象ユーザー ID"
}
```

| 項目     | 型     | 説明                                                |
|----------|--------|-----------------------------------------------------|
| `id`     | string | 対象プロジェクトの ID(必須・パス内で指定)           |
| `userId` | string | 情報を更新する対象利用者の ID(必須・パス内で指定)   |

#### Request Body

```json
{
  "email": "member@..."
}
```

| 項目    | 型     | 説明                               |
|---------|--------|------------------------------------|
| `email` | string | 更新後のメールアドレス(必須)       |

- 表示名(`displayName`)はメンバー本人が SCR-023 有効化ページで入力するため、本 API では受け付けない
- メンバーは当該プロジェクト内で同一権限を持ち、役割の区別は存在しない

## レスポンス

**レスポンス(200)**

```json
{ "userId": "...", "projectId": "...", "email": "member@...", "updatedAt": "..." }
```

| 項目        | 型                | 説明                               |
|-------------|-------------------|------------------------------------|
| `userId`    | string            | 情報を更新した対象利用者の ID      |
| `projectId` | string            | 対象プロジェクトの ID              |
| `email`     | string            | 更新後のメールアドレス             |
| `updatedAt` | string (ISO 8601) | 更新日時                           |

## バリデーション

入力値の検証規則は `## リクエスト` の各パラメータ説明を参照する。

## エラー

| HTTP ステータス | エラーコード            | 内容                         |
|-----------------|-------------------------|------------------------------|
| 404 | `NOT_FOUND` → [ERR-019](../07_errors/ERR-019.md#ERR-019) | 対象が存在しない |
| 403 | `PROJECT_ACCESS_DENIED` → [ERR-021](../07_errors/ERR-021.md#ERR-021) | 当該プロジェクトへの権限なし |
| 409 | `EMAIL_ALREADY_EXISTS` → [ERR-022](../07_errors/ERR-022.md#ERR-022) | メールアドレスの重複 |
| 400 | `VALIDATION_ERROR` → [ERR-001](../07_errors/ERR-001.md#ERR-001) | 入力値の検証エラー |
