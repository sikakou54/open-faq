# 利用者(メンバー)API

> **このページは、プロジェクトのメンバー招待・メンバー情報更新・割当解除など利用者管理 API の契約を定義します。**
>
> **このページの API**
>
> - API-MBR-001 メンバー一覧
> - API-MBR-002 メンバー招待
> - API-MBR-003 メンバー情報更新
> - API-MBR-004 プロジェクト割当解除
> - API-MBR-005 招待メール再送

*版数 v1.7 ・ 更新 2026-06-20 ・ 承認済*

## <span id="API-MBR-001"></span>API-MBR-001 メンバー一覧

<span id="521-get-projectsidmembers"></span>

### 基本情報

| 項目           | 内容                              |
|----------------|-----------------------------------|
| API ID         | API-MBR-001                       |
| API 名         | メンバー一覧                      |
| エンドポイント | `/projects/{id}/members`          |
| HTTP メソッド  | GET                               |
| 認証           | Cookie                            |
| 権限           | オーナー / 当該プロジェクトのメンバー |

### 処理概要

指定プロジェクトのメンバー割当一覧を返す。

| 処理 ID | 処理内容                                                |
|---------|---------------------------------------------------------|
| P-01    | 認証・認可(オーナー / 当該プロジェクトのメンバー)を検証する |
| P-02    | 指定プロジェクトの有効な割当を取得する                  |
| P-03    | ページング(カーソル)で整形して返す                      |

### I/O

| テーブル      | C   | R   | U   | D   |
|---------------|-----|-----|-----|-----|
| `M_PRJ_USERS` | —   | ◯   | —   | —   |
| `M_PRJ_USERS` | —   | ◯   | —   | —   |

### リクエストパラメータ

#### Query Parameters

```json
{
  "cursor": "(共通仕様)",
  "limit": "(共通仕様)"
}
```

| 項目     | 型      | 説明                                             |
|----------|---------|--------------------------------------------------|
| `cursor` | string  | 次ページ取得用カーソル(任意・共通ページング仕様) |
| `limit`  | integer | 1 ページの取得件数(任意・共通ページング仕様)     |

#### Path Parameters

```json
{ "id": "プロジェクト ID" }
```

| 項目 | 型     | 説明                                      |
|------|--------|-------------------------------------------|
| `id` | string | 対象プロジェクトの ID(必須・パス内で指定) |

#### Request Body

```json
{}
```

### レスポンス(200)

```json
{
  "items": [
    {
      "id": "01HZ...",
      "email": "member@...",
      "status": "active|pending_activation",
      "isOwner": false,
      "invitationExpiresAt": "2026-05-20T..."
    }
  ],
  "nextCursor": "01HZ..." | null
}
```

| 項目 | 型 | 説明 |
|----|----|----|
| `items` | array\<object\> | 指定プロジェクトのメンバー割当一覧 |
| `　└ id` | string | メンバー(利用者)の ID |
| `　└ email` | string | メンバーのメールアドレス |
| `　└ status` | enum | アカウント状態(`active` / `pending_activation`) |
| `　└ isOwner` | boolean | オーナーかどうか |
| `　└ invitationExpiresAt` | string (ISO 8601) | 未有効化時の招待トークン有効期限 |
| `nextCursor` | string | 次ページ取得用カーソル(末尾は `null`) |

> [!NOTE]
> **補足** `items[]` は URL で指定したプロジェクトの割当だけを返す。最終ログイン日時と他プロジェクトの割当は返さない。

## <span id="API-MBR-002"></span>API-MBR-002 メンバー招待

<span id="522-post-projectsidmembers"></span>

### 基本情報

| 項目           | 内容                                                     |
|----------------|----------------------------------------------------------|
| API ID         | API-MBR-002                                              |
| API 名         | プロジェクト単発招待(SCR-014 招待モード正本)         |
| エンドポイント | `/projects/{id}/members`                                 |
| HTTP メソッド  | POST                                                     |
| 認証           | Cookie + CSRF + 再認証必須                               |
| 権限           | オーナー / 当該プロジェクトのメンバー                    |

既存のメンバーアカウントの場合は `M_PRJ_USERS` 行のみ追加。新規メールアドレスの場合は予約 `M_USER`(**`name`/`password_hash`=NULL** / **`status='pending_activation'`** / `valid=1`)を作成 + `M_PRJ_USERS` を `valid=0` で同時 INSERT + 招待メール送信(7 日有効、`T_ACCESS_TOKENS.purpose='activation'`、`meta={"invitedProjectId": "..."}` を JSON 保持)。氏名・パスワードは未設定で、招待された本人が SCR-023 メンバーアカウント有効化ページで設定し有効化する(API-AUTH-008)。**氏名(`displayName`)は受け付けない**(FR-022 / FR-023)。

### 処理概要

該当プロジェクト 1 件に対する単発招待を行う。

| 処理 ID | 処理内容 |
|----|----|
| P-01 | 認証・認可(オーナー / 当該プロジェクトのメンバー)と再認証を検証する |
| P-02 | 入力値(メール形式)を検証し、当該プロジェクトへの既存割当(重複)を判定する |
| P-03 | 既存アカウントは当該プロジェクトの割当を追加し、新規メールは予約 `M_USER`(`name`/`password_hash`=NULL, `status='pending_activation'`)と無効状態の割当(`M_PRJ_USERS.valid=0`)を作成する |
| P-04 | 新規メールには有効化トークンを発行し招待メールを送信する |
| P-05 | 招待結果を返す |

### I/O

| テーブル          | C   | R   | U   | D   |
|-------------------|-----|-----|-----|-----|
| `M_USER`          | ◯   | ◯   | —   | —   |
| `M_PRJ_USERS`     | ◯   | ◯   | —   | —   |
| `T_ACCESS_TOKENS` | ◯   | —   | —   | —   |
| `H_NOTIF_LOGS`    | ◯   | —   | —   | —   |

### リクエストパラメータ

#### Query Parameters

```json
{}
```

#### Path Parameters

```json
{ "id": "プロジェクト ID" }
```

| 項目 | 型     | 説明                                        |
|------|--------|---------------------------------------------|
| `id` | string | 招待先プロジェクトの ID(必須・パス内で指定) |

#### Request Body

```json
{
  "email": "newmember@..."
}
```

| 項目    | 型     | 説明                       |
|---------|--------|----------------------------|
| `email` | string | 招待先メールアドレス(必須) |

### レスポンス(201)

```json
{ "userId": "01HZ...", "status": "active|pending_activation", "expiresAt": "..." | null }
```

| 項目 | 型 | 説明 |
|----|----|----|
| `userId` | string | 招待した(または既存の)メンバーの ID |
| `status` | enum | アカウント状態(`active` / `pending_activation`) |
| `expiresAt` | string (ISO 8601) | 新規招待時のトークン有効期限(既存アカウントは `null`) |

### エラー

| HTTP ステータス | エラーコード                | 内容                           |
|-----------------|-----------------------------|--------------------------------|
| 409             | `ALREADY_EXISTS_IN_PROJECT` | 既に該当プロジェクトに割当あり |
| 400             | `VALIDATION_ERROR`          | 入力値の検証エラー             |
| 403             | `PROJECT_ACCESS_DENIED`     | 当該プロジェクトへの権限なし   |

## <span id="API-MBR-003"></span>API-MBR-003 メンバー情報更新

<span id="523-patch-projectsidmembersuserid"></span>

### 基本情報

| 項目           | 内容                                                     |
|----------------|----------------------------------------------------------|
| API ID         | API-MBR-003                                              |
| API 名         | メンバー情報更新(SCR-014 編集モード正本)             |
| エンドポイント | `/projects/{id}/members/{userId}`                        |
| HTTP メソッド  | PATCH                                                    |
| 認証           | Cookie + CSRF + 再認証必須                               |
| 権限           | オーナー / 当該プロジェクトのメンバー                    |

メンバーの属性(メールアドレス)を更新する。プロジェクト内の役割差は持たないためロール項目は扱わない。

### 処理概要

対象メンバーのメールアドレスを更新する。

| 処理 ID | 処理内容                                                         |
|---------|------------------------------------------------------------------|
| P-01    | 認証・認可(オーナー / 当該プロジェクトのメンバー)と再認証を検証する  |
| P-02    | 対象割当の存在と入力値(メール形式)を検証する                    |
| P-03    | メールアドレスの重複を判定し該当時は拒否する                     |
| P-04    | 対象メンバーのメールアドレスを更新する                           |
| P-05    | 更新結果を返す                                                   |

### I/O

| テーブル      | C   | R   | U   | D   |
|---------------|-----|-----|-----|-----|
| `M_PRJ_USERS` | —   | ◯   | ◯   | —   |

### リクエストパラメータ

#### Query Parameters

```json
{}
```

#### Path Parameters

```json
{
  "id": "プロジェクト ID",
  "userId": "対象ユーザー ID"
}
```

| 項目     | 型     | 説明                                                |
|----------|--------|-----------------------------------------------------|
| `id`     | string | 対象プロジェクトの ID(必須・パス内で指定)           |
| `userId` | string | 情報を更新する対象利用者の ID(必須・パス内で指定)   |

#### Request Body

```json
{
  "email": "member@..."
}
```

| 項目    | 型     | 説明                               |
|---------|--------|------------------------------------|
| `email` | string | 更新後のメールアドレス(必須)       |

- 表示名(`displayName`)はメンバー本人が SCR-023 有効化ページで入力するため、本 API では受け付けない
- メンバーは当該プロジェクト内で同一権限を持ち、役割の区別は存在しない

### レスポンス(200)

```json
{ "userId": "...", "projectId": "...", "email": "member@...", "updatedAt": "..." }
```

| 項目        | 型                | 説明                               |
|-------------|-------------------|------------------------------------|
| `userId`    | string            | 情報を更新した対象利用者の ID      |
| `projectId` | string            | 対象プロジェクトの ID              |
| `email`     | string            | 更新後のメールアドレス             |
| `updatedAt` | string (ISO 8601) | 更新日時                           |

### エラー

| HTTP ステータス | エラーコード            | 内容                         |
|-----------------|-------------------------|------------------------------|
| 404             | `NOT_FOUND`             | 対象が存在しない             |
| 403             | `PROJECT_ACCESS_DENIED` | 当該プロジェクトへの権限なし |
| 409             | `EMAIL_ALREADY_EXISTS`  | メールアドレスの重複         |
| 400             | `VALIDATION_ERROR`      | 入力値の検証エラー           |

## <span id="API-MBR-004"></span>API-MBR-004 プロジェクト割当解除

<span id="524-delete-projectsidmembersuserid"></span>

### 基本情報

| 項目           | 内容                                                     |
|----------------|----------------------------------------------------------|
| API ID         | API-MBR-004                                              |
| API 名         | プロジェクト割当解除                                     |
| エンドポイント | `/projects/{id}/members/{userId}`                        |
| HTTP メソッド  | DELETE                                                   |
| 認証           | Cookie + CSRF + 再認証必須                               |
| 権限           | オーナー / 当該プロジェクトのメンバー                    |

対象が自分自身またはオーナーの場合は拒否する。

- 対象ユーザーに他プロジェクトの有効割当が残る場合: `M_PRJ_USERS`、他プロジェクトの割当(`M_PRJ_USERS`)、セッション、招待トークンは維持する。
- 他プロジェクトの有効割当が 0 件になる場合: 同一トランザクションで `M_PRJ_USERS.valid=0`、全セッション失効、未使用招待トークン失効を実行する。

### 処理概要

対象プロジェクトの `M_PRJ_USERS.valid=0` を更新する。

| 処理 ID | 処理内容 |
|----|----|
| P-01 | 認証・認可(オーナー / 当該プロジェクトのメンバー)と再認証を検証する |
| P-02 | 対象が自分自身またはオーナーの場合は拒否する |
| P-03 | 当該プロジェクトの割当を論理削除する |
| P-04 | 他プロジェクトの有効割当が 0 件になる場合はアカウントを論理削除し、全セッション失効と未使用招待トークン失効を行う |
| P-05 | 解除結果(アカウント無効化の有無)を返す |

### I/O

| テーブル          | C   | R   | U   | D   |
|-------------------|-----|-----|-----|-----|
| `M_PRJ_USERS`     | —   | ◯   | ◯   | —   |
| `M_PRJ_USERS`     | —   | —   | ◯   | —   |
| `T_SESSIONS`      | —   | —   | ◯   | —   |
| `T_ACCESS_TOKENS` | —   | —   | ◯   | —   |

### リクエストパラメータ

#### Query Parameters

```json
{}
```

#### Path Parameters

```json
{
  "id": "プロジェクト ID",
  "userId": "対象ユーザー ID"
}
```

| 項目     | 型     | 説明                                                |
|----------|--------|-----------------------------------------------------|
| `id`     | string | 割当を解除するプロジェクトの ID(必須・パス内で指定) |
| `userId` | string | 割当を解除する対象利用者の ID(必須・パス内で指定)   |

#### Request Body

```json
{}
```

### レスポンス(200)

```json
{ "userId": "...", "projectId": "...", "accountDeactivated": true|false }
```

| 項目 | 型 | 説明 |
|----|----|----|
| `userId` | string | 割当を解除した対象利用者の ID |
| `projectId` | string | 割当を解除したプロジェクトの ID |
| `accountDeactivated` | boolean | 他プロジェクトの有効割当が 0 件となりアカウントを論理削除したかどうか |

### エラー

| HTTP ステータス | エラーコード              | 内容                         |
|-----------------|---------------------------|------------------------------|
| 403             | `E-AUTHZ-OWNER-PROTECTED` | オーナーは解除不可           |
| 403             | `E-AUTHZ-SELF-MUTATION`   | 自分自身は解除不可           |
| 403             | `PROJECT_ACCESS_DENIED`   | 当該プロジェクトへの権限なし |
| 404             | `NOT_FOUND`               | 対象が存在しない             |

## <span id="API-MBR-005"></span>API-MBR-005 招待メール再送

<span id="525-post-membersidresend-invitation"></span>

### 基本情報

| 項目           | 内容                                               |
|----------------|----------------------------------------------------|
| API ID         | API-MBR-005                                        |
| API 名         | 招待メール再送                                     |
| エンドポイント | `/members/{id}/resend-invitation`                  |
| HTTP メソッド  | POST                                               |
| 認証           | Cookie + CSRF + 再認証必須                         |
| 権限           | オーナー / 当該プロジェクトのメンバー              |

旧 `T_ACCESS_TOKENS.purpose='activation'` を失効させ新規トークン(7 日有効、`meta` を同一の `invitedProjectId` で再発行)を発行。

### 処理概要

`status='pending_activation'` のメンバーへ招待メールを再送する。

| 処理 ID | 処理内容 |
|----|----|
| P-01 | 認証・認可(オーナー / 当該プロジェクトのメンバー)と再認証を検証する |
| P-02 | 対象が未有効化(`pending_activation`)のメンバーであることを確認する(オーナー / 自身は対象外) |
| P-03 | 旧有効化トークンを失効させ、同一の招待先情報で新規トークンを再発行する |
| P-04 | 招待メールを再送し再発行結果を返す |

### I/O

| テーブル          | C   | R   | U   | D   |
|-------------------|-----|-----|-----|-----|
| `M_PRJ_USERS`     | —   | ◯   | —   | —   |
| `M_PRJ_USERS`     | —   | ◯   | —   | —   |
| `T_ACCESS_TOKENS` | ◯   | —   | ◯   | —   |
| `H_NOTIF_LOGS`    | ◯   | —   | —   | —   |

### リクエストパラメータ

#### Query Parameters

```json
{}
```

#### Path Parameters

```json
{ "id": "メンバーユーザー ID" }
```

| 項目 | 型 | 説明 |
|----|----|----|
| `id` | string | 招待メールを再送する対象メンバーの利用者 ID(必須・パス内で指定) |

#### Request Body

```json
{}
```

氏名 `displayName` を受け付けない(FR-020 / FR-022)。

### レスポンス(200)

再発行結果(`{ "ok": true, "expiresAt": "..." }`)。

| 項目        | 型                | 説明                             |
|-------------|-------------------|----------------------------------|
| `ok`        | boolean           | 招待メール再送の成否             |
| `expiresAt` | string (ISO 8601) | 再発行した招待トークンの有効期限 |

### エラー

| HTTP ステータス | エラーコード              | 内容             |
|-----------------|---------------------------|------------------|
| 403             | `E-AUTHZ-OWNER-PROTECTED` | オーナーは対象外 |
| 403             | `E-AUTHZ-SELF-MUTATION`   | 自分自身は対象外 |
| 404             | `NOT_FOUND`               | 対象が存在しない |
