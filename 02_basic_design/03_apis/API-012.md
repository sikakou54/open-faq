# <span id="API-012"></span>API-012: 自己プロフィール更新

> **このページは 自己プロフィール更新 API の契約を定義します。** 認証済み利用者が自身の表示名・メールアドレスを更新する。メールアドレス変更時は再認証トークンを要求し、新アドレスへ確認メールを送信する。

*版数 v2.0 ・ 更新 2026-06-21 ・ 再構成 P4*

## 項目

| 項目 | 内容 |
|---|---|
| API ID | `API-012` |
| API名 | 自己プロフィール更新 |
| 対応業務UC | [UC-177](../../01_requirements/02_business_usecases/UC-177.md#UC-177) |
| 対応画面ID | [SCR-022](../01_screens/SCR-022.md#SCR-022) |
| 対応画面イベントID | [EVT-177](../02_screen_events/EVT-177.md#EVT-177) |
| エンドポイント | `/me/profile` |
| HTTPメソッド | PATCH |
| 認証 | 要(Cookie + CSRF) |
| 認可 | 認証済み |

## 処理概要

認証済み利用者が自身の表示名・メールアドレスを更新する。メールアドレス変更時は再認証トークンを要求し、新アドレスへ確認メールを送信する。

| 処理 ID | 処理内容 |
|---------|----------|
| P-01 | セッションから認証済み利用者を特定する |
| P-02 | リクエストボディの入力値(表示名長・メールアドレス形式)を検証する |
| P-03 | メールアドレス変更時は再認証トークン(`X-Reauth-Token` ヘッダ)を検証する |
| P-04 | 表示名・メールアドレスを対応マスタ(`M_CONTRACT` または `M_PRJ_USERS`)に更新する |
| P-05 | メールアドレス変更時はメール確認トークンを発行し新アドレスへ確認メールを送信する |
| P-06 | 更新結果を返す |

## リクエスト

#### Query Parameters

```json
{}
```

#### Path Parameters

```json
{}
```

#### Request Body

```json
{
  "displayName": "山田 太郎",
  "email": "new@example.com",
  "reauthToken": "reauth_..."
}
```

| 項目           | 型     | 説明                                                       |
|----------------|--------|------------------------------------------------------------|
| `displayName`  | string | 新しい表示名(任意・1〜100 文字)                            |
| `email`        | string | 新しいメールアドレス(任意・変更時は `reauthToken` 必須)    |
| `reauthToken`  | string | 再認証トークン(メールアドレス変更時必須・API-005 で取得) |

## レスポンス

**レスポンス(200)**

```json
{ "ok": true, "emailVerificationPending": false }
```

| 項目                       | 型      | 説明                                                      |
|----------------------------|---------|-----------------------------------------------------------|
| `ok`                       | boolean | 更新の成否                                                |
| `emailVerificationPending` | boolean | 確認メールを送信して検証待ちの場合 `true`(メールアドレス変更時) |

## バリデーション

入力値の検証規則は `## リクエスト` の各パラメータ説明を参照する。

## エラー

| HTTP ステータス | エラーコード                   | 内容                         |
|-----------------|--------------------------------|------------------------------|
| 400             | `VALIDATION_ERROR`(E-INPUT-\*) | 入力値の検証エラー           |
| 401             | `REAUTH_REQUIRED`              | 再認証トークンが無効または未提示 |
| 409             | `EMAIL_ALREADY_USED`           | 入力メールアドレスが既に使用中 |
