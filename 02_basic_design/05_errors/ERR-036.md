# <span id="ERR-036"></span>ERR-036: AI_UNAVAILABLE

> **このページは API レスポンスで返すエラー `AI_UNAVAILABLE` の HTTP ステータス・分類・メッセージ・業務ユースケースID・API ID を定義します。**

| ID | 業務ユースケースID | API ID |
|----|----|----|
| ERR-036 | [UC-042](../../01_requirements/04_business_usecases/UC-042.md#UC-042) | [API-038](../02_backend/03_apis/API-038.md#API-038) |

| エラーコード | HTTPステータス | エラー分類 |
|----|----|----|
| `AI_UNAVAILABLE` | 503 | システム |
*種別 エラー定義 ・ 分類 システム ・ ステータス ドラフト*

## <span id="def"></span>1. 定義

| 項目 | 値 |
|----|----|
| エラーコード | `AI_UNAVAILABLE` |
| 分類コード(taxonomy) | `E-AI-UNAVAILABLE` |
| HTTP ステータス | 503 |
| エラー分類 | システム |

## <span id="msg"></span>2. メッセージ

AI 推論タイムアウトまたは AI プロバイダエラーにより、回答生成を完了できない。

## <span id="cond"></span>3. 発生条件・表示先・復旧手段

| 観点 | 内容 |
|----|----|
| 発生条件 | [API-038](../02_backend/03_apis/API-038.md#API-038) で AI 推論 IF がタイムアウト、またはプロバイダエラーを返したとき。 |
| 表示先 | [SCR-030](../01_frontend/01_screens/SCR-030.md#SCR-030) EVT-07 の処理エラー表示(EM-03) |
| 復旧手段 | 時間をおいて再試行する。継続する場合は運用者が AI プロバイダ・モデルの稼働状況を確認する |

> [!NOTE]
> 本エラーは処理エラーであり、未解決質問として自動登録しない([FR-082](../../01_requirements/02_functional_requirement/02_faq-ai-fr.md#FR-082))。質問ログを残す場合は `H_QUESTION_LOGS.result_type=error`、`result_reason_code=ai_unavailable` とする。
