# <span id="ERR-038"></span>ERR-038: PAYMENT_METHOD_REQUIRED

> **このページは API レスポンスで返すエラー `PAYMENT_METHOD_REQUIRED` の HTTP ステータス・分類・メッセージ・業務ユースケースID・API ID を定義します。**

| ID | 業務ユースケースID | API ID |
|----|----|----|
| ERR-038 | [UC-034](../../01_requirements/04_business_usecases/UC-034.md#UC-034) | [API-047](../02_backend/03_apis/API-047.md#API-047) |

| エラーコード | HTTPステータス | エラー分類 |
|----|----|----|
| `PAYMENT_METHOD_REQUIRED` | 402 | 業務(課金) |
*種別 エラー定義 ・ 分類 業務(課金) ・ ステータス ドラフト*

## <span id="def"></span>1. 定義

| 項目 | 値 |
|----|----|
| エラーコード | `PAYMENT_METHOD_REQUIRED` |
| 分類コード(taxonomy) | `E-BILL-PAYMENT-METHOD-REQUIRED` |
| HTTP ステータス | 402 |
| エラー分類 | 業務(課金) |

## <span id="msg"></span>2. メッセージ

利用上限を設定するには支払方法の登録が必要です

## <span id="cond"></span>3. 発生条件・表示先・復旧手段

| 観点 | 内容 |
|----|----|
| 発生条件 | 課金アカウントに支払方法が未登録([API-045](../02_backend/03_apis/API-045.md#API-045) の `registered=false` 相当)のまま、質問数の月次上限を有効化(`limitEnabled=true`)しようとしたとき。支払方法未登録では無料枠超過時に支払方法ゲート([課金・請求設計 §4](../05_billing-design.md#4-支払方法ゲート))で受付停止となるため、利用上限の設定は支払方法登録を前提とする。 |
| 表示先 | 画面トースト([SCR-027](../01_frontend/01_screens/SCR-027.md#SCR-027)。上限設定操作の結果) |
| 復旧手段 | 支払方法を登録([SCR-028](../01_frontend/01_screens/SCR-028.md#SCR-028) / [API-045](../02_backend/03_apis/API-045.md#API-045))したうえで利用上限を設定する。 |
