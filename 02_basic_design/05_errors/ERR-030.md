> **このページは API レスポンスで返すエラー `PERMISSION_DENIED` の HTTP ステータス・分類・メッセージ・業務ユースケースID・API ID を定義します。**

| ID | 業務ユースケースID | API ID |
|----|----|----|

| エラーコード | HTTPステータス | エラー分類 |
|----|----|----|
| `PERMISSION_DENIED` | 403 | 認可 |
*種別 エラー定義 ・ 分類 認可 ・ ステータス ドラフト*

## <span id="def"></span>1. 定義

| 項目 | 値 |
|----|----|
| エラーコード | `PERMISSION_DENIED` |
| 分類コード(taxonomy) | `E-AUTHZ-FORBIDDEN` |
| HTTP ステータス | 403 |
| エラー分類 | 認可 |

## <span id="msg"></span>2. メッセージ

- 当該プロジェクトに割当のないユーザー(対応 API: API-047 / API-061)
- メンバーは申請不可(対応 API: API-056)

## <span id="cond"></span>3. 発生条件・表示先・復旧手段

| 観点 | 内容 |
|----|----|
| 発生条件 | 境界を通過した関係者だが、メンバーが専有操作を要求した・本人以外が招待を受諾した等、操作権限が不足するとき。 |
| 表示先 | 画面トースト |
| 復旧手段 | 権限を持つオーナーに操作を依頼する。本人確認が必要な操作は本人のアカウントで行う。 |
