# 基本設計書 再構成結果

> **このページは、基本設計書を「1 識別子 = 1 ファイル」のフラット個別ページ構成へ再構成した結果を、基本設計書の視点でまとめます。** 目的 / 新構成 / ID リナンバ / フェーズ実績 / トレーサビリティ状況 / 残課題を要約します。

*種別 再構成結果サマリ ・ 更新 2026-06-21 ・ ステータス ドラフト*

## 1. 目的と読み順

旧来のグループ別大型ファイル(画面 / API / DB / ユースケース / 認証 / メール等)を解体し、識別子 1 件 = 1 ファイルの**フラット個別ページ**へ再構成した。画面(SCR)・画面イベント(EVT)・API・テーブル(TBL)・シーケンス(SEQ)・権限(PERM)・エラー(ERR)・メッセージ(MSG)をそれぞれ独立ページ化し、層間を双方向に結線した。

読み順(全体)は **要件定義 ＞ 業務ユースケース ＞ 画面設計 ＞ 画面イベント ＞ API設計 ＞ DB設計 ＞ シーケンス ＞ 権限 / エラー / メッセージ**。

## 5. トレーサビリティ状況

要件 → 業務UC → 画面 → 画面イベント → API → テーブル の一気通貫トレースを確立し、UC 単位 **247 行**のマトリクスを `99_management/02_traceability_matrix.md` に生成した。基本設計層の主な逆引き結線実績: EVT→API 96 / UC→API 133 / API→TBL 117 / UC→TBL 178。

> [!WARNING]
> **被覆ギャップ(後続課題)** 上位の要件被覆ギャップ(**132 FR + 91 BR が UC 未対応 / 128 UC が要件未対応**)に加え、基本設計層には次の逆引き欠落がある。**要件被覆の精査が後続課題**として残る。
> - **API**: `API-031`(FAQ 全文検索)・`API-032`(質問ログ検索)は EVT / UC から逆引き不可(検索イベントの呼出API 結線欠落)。`API-003/057/058/059` は内部 / IF 系で EVT 無し(設計どおり)。
> - **TBL**: `TBL-008/011/016/028/029/031` は UC 逆引き無し、`TBL-008/010/011/012/015/016/021/028/029/031` は API 逆引き無し(中間 / ログ / キャッシュ / レート上書き系が中心。お知らせ・規約系は API `## 利用テーブル` 追記で補完余地)。
> - **ERR**: 認可ミドルウェア由来のエラー(`E-AUTH-*` / `E-AUTHZ-*` 等)は API の `## エラー` 表未記載のため未採番。
>
> 詳細件数・対象 ID は `99_management/p7_gaps.json` を参照。

## 6. 既知の残課題

基本設計由来の残課題は `99_management/06_remaining_issues.md` に集約している(GitHub 再接続後に `[設計再構成][区分]` で起票予定)。主なもの:

- **design-gap** — 上記の API / TBL / ERR 逆引き欠落の結線補完。
- **needs-review** — EVT イベント名 / 遷移先の命名統一、TBL 論理名のカタログ整合、ERR 採番粒度(非エラー応答の扱い)、契約停止エラーの接頭辞統一(`E-BIZ` / `E-BILL`)。
- **cleanup** — EVT `## 処理` と SCR §6 の二重管理の正本一元化、MSG index 節番号の振り直し、TBL セクションアンカーの統一。

GitHub Issue #3〜#12(エピック #3 + 各フェーズ)および P7 検出ギャップが対象。`99_management/` は再構成期間の作業ディレクトリで、最終的に内容を正式設計書 / GitHub Issues へ移管して削除する。
