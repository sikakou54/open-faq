# バックエンド設計

> **このセクションは、サーバー側の基本設計をまとめます。** システム設計(`SYS-*`)・システムイベント設計(`SEV-*`)・API設計(`API-*`)・DB設計(`TBL-*`)で構成します。

- [システム設計](01_system/index.md) — 無人処理(バッチ / Webhook / 非同期 / 監視 / 通知)`SYS-*`
- [システムイベント設計](02_system_events/index.md) — システム処理上のイベント `SEV-*`(システムイベント → 業務UC は多:1)
- [API設計](03_apis/index.md) — エンドポイント `API-*`(1 エンドポイント = 1 ファイル)
- [DB設計](04_database/index.md) — テーブル `TBL-*`(1 テーブル = 1 ファイル)

> [!NOTE]
> **運用・インフラ前提(NFR-066 鍵管理)** バックアップは暗号化して保管し、その暗号鍵は本番データの暗号化に用いるマスター鍵とは分離して管理する([NFR-066](../../01_requirements/03_non_functional_requirement/07_nfr.md#NFR-066))。鍵の保管場所・鍵管理サービス連携・バックアップの取得方式といった実現手段はインフラ/詳細設計の領域であり、本基本設計(画面 / API / テーブル / システム処理)では「暗号化済みバックアップ」「鍵のマスター鍵分離」を前提方針として記すに留める。
