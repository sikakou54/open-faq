# <span id="API-013"></span>API-013: 自己パスワード変更

> **このページは 自己パスワード変更 API の契約を定義します。** 認証済み利用者が再認証トークンを提示して自身のパスワードを変更する。

## 項目

| 項目 | 内容 |
|---|---|
| API ID | `API-013` |
| API名 | 自己パスワード変更 |
| トレーサビリティID | [TR-010](../../00_traceability/index.md#TR-010) |
| エンドポイント | `/me/password` |
| HTTPメソッド | PATCH |
| 認証 | 要(Cookie + CSRF) |
| 認可 | 認証済み |

## 処理概要

認証済み利用者が再認証トークンを提示して自身のパスワードを変更する。

| 処理 ID | 処理内容 |
|---------|----------|
| P-01 | セッションから認証済み利用者を特定する |
| P-02 | 再認証トークン(リクエストボディ `reauthToken`)を検証する |
| P-03 | 新パスワードの強度要件(FR-006: 12 文字以上・英大小文字 / 数字 / 記号 3 種類以上)を検証する |
| P-04 | `passwordConfirm` が `password` と一致することを確認する |
| P-05 | 対象アカウントのパスワードハッシュを更新する |
| P-06 | 更新結果を返す |

## リクエスト

#### 認証ヘッダ

本 API は利用者セッション(Cookie)と CSRF トークンを必要とする状態変更系である。

| ヘッダ | 要否 | 説明 |
|---|---|---|
| `Cookie: session=<token>` | 必須 | 利用者セッション |
| `X-CSRF-Token: <token>` | 必須 | CSRF 対策(状態変更系) |
| `Idempotency-Key: <ULID>` | 任意 | 冪等性(書き込み再送の重複防止) |

#### Query Parameters

```json
{}
```

#### Path Parameters

```json
{}
```

#### Request Body

```json
{
  "reauthToken": "reauth_...",
  "password": "P@ssw0rd-new",
  "passwordConfirm": "P@ssw0rd-new"
}
```

| 項目              | 型     | 説明                                                                         |
|-------------------|--------|------------------------------------------------------------------------------|
| `reauthToken`     | string | 再認証トークン(必須・API-005 で取得)                                    |
| `password`        | string | 新しいパスワード(必須・12 文字以上、英大小文字 / 数字 / 記号のうち 3 種類以上) |
| `passwordConfirm` | string | 新しいパスワード確認用(必須・`password` と一致)                              |

## レスポンス

**レスポンス(200)**

```json
{ "ok": true }
```

| 項目 | 型      | 説明               |
|------|---------|--------------------|
| `ok` | boolean | パスワード変更の成否 |

## バリデーション

入力値の検証規則は `## リクエスト` の各パラメータ説明を参照する。

## エラー

| 400 | `VALIDATION_ERROR` → [ERR-001](../../05_errors/ERR-001.md#ERR-001) | パスワード強度不足 / 不一致 |
