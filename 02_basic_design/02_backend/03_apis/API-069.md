# <span id="API-069"></span>API-069: ウィジェット回答フィードバック送信

> **このページは ウィジェット回答フィードバック送信 API の仕様を定義します。** ウィジェット利用者が AI 回答に対して送る「役に立った(解決)/ 役に立たなかった(未解決)」のフィードバックを記録する。未解決フィードバック時は当該質問を未解決質問として登録する。

| ID | 業務ユースケースID | イベント(画面ID EVT-NN) | テーブルID |
|----|----|----|----|
| API-069 | [UC-083](../../../01_requirements/04_business_usecases/UC-083.md#UC-083) | SCR-030 EVT-08 | [TBL-025](../04_database/TBL-025.md#TBL-025) ・ [TBL-017](../04_database/TBL-017.md#TBL-017) ・ [TBL-004](../04_database/TBL-004.md#TBL-004) |

| API名 | エンドポイント | HTTPメソッド |
|----|----|----|
| ウィジェット回答フィードバック送信 | `/widget/v1/feedback` | POST |

## 処理概要

ウィジェット利用者が AI 回答(回答可能)に対して送る解決 / 未解決のフィードバックを記録する。「役に立った」は解決として記録し、「役に立たなかった」は当該質問を未解決質問として登録する(理由コード `user_unresolved`)。フィードバックは次の質問送信の前提として必須であり、本 API はその選択結果を受け取る。

| 処理 ID | 処理内容 |
|---------|----------|
| P-01    | ウィジェットセッショントークンを検証する |
| P-02    | 冪等キーで重複登録を排除する |
| P-03    | 対象質問ログにフィードバック結果を記録する(`helpful` = 解決 / `unhelpful` = 未解決) |
| P-04    | `unhelpful` の場合は当該質問を未解決質問として登録する(結果理由コード `user_unresolved`)。`helpful` の場合は未解決質問を登録しない |
| P-05    | 記録結果(解決 / 未解決・未解決質問 ID)と、未解決時の連絡先案内要否(`showContactGuide`)・連絡先メール(`contactEmail`)を返却する。`showContactGuide` は `unresolved` かつプロジェクトの連絡先メールアドレスが設定済み、かつ確認完了済みの場合のみ `true` とする([FR-044](../../../01_requirements/02_functional_requirement/01_account-fr.md#FR-044)) |

## リクエスト

#### 認証ヘッダ

本 API はウィジェットセッショントークン(Bearer)による認証を必要とする。

| ヘッダ | 要否 | 説明 |
|---|---|---|
| `Authorization: Bearer <wst_*>` | 必須 | ウィジェットセッション(起動 API で発行) |
| `Idempotency-Key: <ULID>` | 任意 | 冪等性(再送の重複防止) |

#### Query Parameters

```json
{}
```

#### Path Parameters

```json
{}
```

#### Request Body

```json
{
  "questionId": "qlog_...",
  "feedback": "unhelpful",
  "idempotencyKey": "<UUIDv4>"
}
```

| 項目             | 型     | 説明                                                    |
|------------------|--------|---------------------------------------------------------|
| `questionId`     | string | フィードバック対象の AI 回答に対応する質問ログの ID。必須 |
| `feedback`       | enum   | フィードバック結果。`helpful`(役に立った)/ `unhelpful`(役に立たなかった)。必須 |
| `idempotencyKey` | string | 重複登録を排除する冪等キー(UUIDv4)。必須 |

## レスポンス

**レスポンス(200)**

```json
{
  "recorded": true,
  "result": "unresolved",
  "inquiryId": "inq_...",
  "contactEmail": "support@example.com",
  "showContactGuide": true
}
```

| 項目 | 型 | 説明 |
|---|---|---|
| `recorded` | boolean | フィードバックを記録したか |
| `result` | enum | 記録結果。`resolved`(役に立った)/ `unresolved`(役に立たなかった) |
| `inquiryId` | string \| null | `unresolved` 時に登録した未解決質問の ID。`resolved` 時は `null` |
| `contactEmail` | string \| null | 確認完了済みのプロジェクト問い合わせ先メールアドレス。連絡先未設定・未確認・`resolved` 時は `null` |
| `showContactGuide` | boolean | 連絡先案内を表示すべきか。`result=unresolved` かつプロジェクトの連絡先メールアドレスが設定済み、かつ確認完了済みの場合のみ `true`([FR-044](../../../01_requirements/02_functional_requirement/01_account-fr.md#FR-044)) |

## 列挙値

リクエスト / レスポンスで用いる enum の取りうる値と意味を示す。

| 項目 | 取りうる値 | 意味 |
|----|----|----|
| `feedback` | `helpful` / `unhelpful` | 役に立った(解決)/ 役に立たなかった(未解決) |
| `result` | `resolved` / `unresolved` | 解決として記録 / 未解決質問として登録 |

## バリデーション

入力値の検証規則は `## リクエスト` の各パラメータ説明を参照する。主な規則は次のとおり。

- `questionId`: 必須。対象の質問ログが存在しないフィードバックは受け付けない。
- `feedback`: 必須・`helpful` / `unhelpful` のいずれか。
- `idempotencyKey`: 必須(UUIDv4)。

## エラー

| HTTP ステータス | エラーコード | 内容 |
