## 処理概要

機微操作の前にパスワードを再確認する。

| 処理 ID | 処理内容                                   |
|---------|--------------------------------------------|
| P-01    | セッションから認証済み利用者を特定する     |
| P-02    | パスワードを再照合する(不一致は 401)       |
| P-03    | 機微操作に使う再認証トークンを `purpose='reauth'` で発行して返す(有効期限 15 分・対象操作 1 回限りで消費) |

## リクエスト

#### 認証ヘッダ

本 API は利用者セッション(Cookie)と CSRF トークンを必要とする状態変更系である。

| ヘッダ | 要否 | 説明 |
|---|---|---|
| `Cookie: session=<token>` | 必須 | 利用者セッション |
| `X-CSRF-Token: <token>` | 必須 | CSRF 対策(状態変更系) |
| `Idempotency-Key: <ULID>` | 任意 | 冪等性(書き込み再送の重複防止) |

#### Query Parameters

```json
{}
```

#### Path Parameters

```json
{}
```

#### Request Body

```json
{ "password": "..." }
```

| 項目       | 型     | 説明                       |
|------------|--------|----------------------------|
| `password` | string | 再確認するパスワード(必須) |

## レスポンス

**レスポンス(200)**

```json
{ "ok": true, "token": "reauth_..." }
```

| 項目    | 型      | 説明                         |
|---------|---------|------------------------------|
| `ok`    | boolean | 再認証の成否                 |
| `token` | string  | 機微操作に使う再認証トークン |

## バリデーション
