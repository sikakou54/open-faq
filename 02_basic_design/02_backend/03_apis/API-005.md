# <span id="API-005"></span>API-005: 再認証

> **このページは 再認証 API の仕様を定義します。** 機微操作の前にパスワードを再確認する。

## 項目

| 項目 | 内容 |
|---|---|
| API ID | `API-005` |
| API名 | 再認証 |
| トレーサビリティID | [TR-009](../../00_traceability/index.md#TR-009) ・ [TR-010](../../00_traceability/index.md#TR-010) ・ [TR-017](../../00_traceability/index.md#TR-017) ・ [TR-023](../../00_traceability/index.md#TR-023) ・ [TR-038](../../00_traceability/index.md#TR-038) |
| エンドポイント | `/auth/re-auth` |
| HTTPメソッド | POST |
| 認証 | 要(Cookie) |
| 認可 | 認証済み |

## 処理概要

機微操作の前にパスワードを再確認する。

| 処理 ID | 処理内容                                   |
|---------|--------------------------------------------|
| P-01    | セッションから認証済み利用者を特定する     |
| P-02    | パスワードを再照合する(不一致は 401)       |
| P-03    | 機微操作に使う再認証トークンを発行して返す |

## リクエスト

#### 認証ヘッダ

本 API は利用者セッション(Cookie)と CSRF トークンを必要とする状態変更系である。

| ヘッダ | 要否 | 説明 |
|---|---|---|
| `Cookie: session=<token>` | 必須 | 利用者セッション |
| `X-CSRF-Token: <token>` | 必須 | CSRF 対策(状態変更系) |
| `Idempotency-Key: <ULID>` | 任意 | 冪等性(書き込み再送の重複防止) |

#### Query Parameters

```json
{}
```

#### Path Parameters

```json
{}
```

#### Request Body

```json
{ "password": "..." }
```

| 項目       | 型     | 説明                       |
|------------|--------|----------------------------|
| `password` | string | 再確認するパスワード(必須) |

## レスポンス

**レスポンス(200)**

```json
{ "ok": true, "token": "reauth_..." }
```

| 項目    | 型      | 説明                         |
|---------|---------|------------------------------|
| `ok`    | boolean | 再認証の成否                 |
| `token` | string  | 機微操作に使う再認証トークン |

## バリデーション

入力値の検証規則は `## リクエスト` の各パラメータ説明を参照する。
