## 処理概要

オーナー単位(自分が作成したプロジェクト全体)の当月請求見込み合計・次回請求日・請求状態・プロジェクト別内訳・支払方法登録状態を返す。請求はオーナー(課金アカウント)単位で1通に集約し、各プロジェクトの内訳明細を含む。

| 処理 ID | 処理内容                                                       |
|---------|----------------------------------------------------------------|
| P-01    | 認証・認可(自分が作成したプロジェクトのオーナー)を検証する     |
| P-02    | 当月の請求見込み合計・プロジェクト別内訳を算出する             |
| P-03    | 次回請求日・請求状態・支払方法登録状態を解決する               |
| P-04    | 整形して返却する                                               |

## リクエスト

本 API は利用者セッション(Cookie)による認証を必要とする。

## レスポンス

**レスポンス(200)**

```json
{
  "estimatedTotal": 12500,
  "currency": "JPY",
  "nextBillingDate": "2026-07-01",
  "billingStatus": "normal|payment_method_required|payment_failed",
  "paymentMethod": { "registered": true, "brand": "visa", "last4": "4242" },
  "projects": [
    {
      "projectId": "prj_...",
      "projectName": "サポートサイト",
      "questionCharge": 3500,
      "faqCharge": 1000,
      "subtotal": 4500
    }
  ]
}
```

| 項目 | 型 | 説明 |
|----|----|----|
| `estimatedTotal` | integer | 当月請求見込み合計 |
| `currency` | string | 通貨コード |
| `nextBillingDate` | string (ISO 8601) | 次回請求日 |
| `billingStatus` | enum | 請求状態。`normal` / `payment_method_required` / `payment_failed` |
| `paymentMethod` | object | 支払方法の登録状態 |
| `　└ registered` | boolean | 支払方法が登録済みか |
| `　└ brand` | string | カードブランド |
| `　└ last4` | string | カード番号下 4 桁 |
| `projects` | array\<object\> | プロジェクト別の請求内訳 |
| `　└ projectId` | string | プロジェクトの ID |
| `　└ projectName` | string | プロジェクト名 |
| `　└ questionCharge` | integer | 質問数に対する課金額 |
| `　└ faqCharge` | integer | FAQ 件数に対する課金額 |
| `　└ subtotal` | integer | プロジェクトの小計 |

## 利用テーブル

本 API が参照・更新するテーブルと CRUD 区分。

| テーブル | CRUD |
|---|---|
| [`M_BILLING_ACCOUNT`](../04_database/TBL-002.md#TBL-002) | `-R--` |
| [`M_PROJECTS`](../04_database/TBL-004.md#TBL-004) | `-R--` |
| [`T_BILL_SUBS`](../04_database/TBL-018.md#TBL-018) | `-R--` |
| [`T_USAGE_METER`](../04_database/TBL-020.md#TBL-020) | `-R--` |

`M_BILLING_ACCOUNT`(`user_id` = 認証ユーザー)から請求状態・支払方法登録状態を、`M_PROJECTS`(`owner_user_id` = 認証ユーザー)から集計対象プロジェクトの一覧・名称を、`T_BILL_SUBS`(`billing_account_id`)・`T_USAGE_METER`(`project_id`)から課金・利用量を解決する。
