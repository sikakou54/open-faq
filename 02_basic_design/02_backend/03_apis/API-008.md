## 処理概要

招待された登録済みユーザー本人が招待を受諾し、対象プロジェクトへの自分の割当を有効化する。アカウントは独立サインアップで事前作成済みのため、本 API では氏名・初回パスワード・規約同意の登録は行わない。本処理は 1 つのトランザクションで実行する。

| 処理 ID | 処理内容 |
|----|----|
| P-01 | 招待トークンの有効性(期限・使用済み)を検証する |
| P-02 | トークンから招待先プロジェクトと招待先メールを取得する |
| P-03 | 招待先メールがログイン中ユーザーのメールと一致することを確認する(本人受諾) |
| P-04 | 対象プロジェクトへの当該ユーザーの割当を有効化する(`valid=1`) |
| P-05 | 招待トークンを使用済みにする |
| P-06 | 監査ログ(招待受諾・割当有効化完了)を記録する |

## リクエスト

#### 認証ヘッダ

本 API はログイン中の招待された本人のセッション認証を必要とし、検証用トークンを URL パスで受け取って検証する。

| ヘッダ | 要否 | 説明 |
|---|---|---|
| `Authorization` / セッション | 必須 | 招待された登録済みユーザー本人のセッション |
| `Idempotency-Key: <ULID>` | 任意 | 冪等性(再送の重複防止) |

#### Query Parameters

```json
{}
```

#### Path Parameters

```json
{ "token": "招待トークン" }
```

| 項目    | 型     | 説明                             |
|---------|--------|----------------------------------|
| `token` | string | 招待トークン(必須・パス内で指定) |

#### Request Body

```json
{}
```

本 API はリクエストボディを取らない(受諾対象は URL パスのトークン、受諾者はセッションのログインユーザーで一意に定まる)。

## レスポンス

**レスポンス(200)**

```json
{ "projectId": "01HZ...", "userId": "01HZ...", "redirectUrl": "/dashboard" }
```

| 項目          | 型     | 説明                                   |
|---------------|--------|----------------------------------------|
| `projectId`   | string | 割当を有効化したプロジェクトの ID       |
| `userId`      | string | 割当を有効化した利用者(本人)の ID      |
| `redirectUrl` | string | 受諾完了後の遷移先 URL(ダッシュボード) |

## バリデーション

- セッションが有効であること(ログイン中の招待された本人)
- `token`: 必須(パス内で指定)、有効(期限内・未使用)であること
- 招待先メールがログイン中ユーザーのメールと一致すること(本人受諾)

## エラー

| HTTP ステータス | エラーコード | 内容 |
|----|----|----|
| 410 | `TOKEN_EXPIRED` → [ERR-006](../../05_errors/ERR-006.md#ERR-006) | トークン期限切れ |
| 410 | `TOKEN_USED` → [ERR-007](../../05_errors/ERR-007.md#ERR-007) | トークン使用済み |
| 403 | `FORBIDDEN` → [ERR-009](../../05_errors/ERR-009.md#ERR-009) | 招待先メールとログインユーザーが不一致(本人以外の受諾) |
