## 処理概要

プロジェクトの連絡先メールアドレスへ確認メールを再送する。レート制限により一定時間内の再送回数を制限する。

| 処理 ID | 処理内容 |
|---------|----------|
| P-01 | セッションから認証済みユーザーを特定する |
| P-02 | 対象プロジェクトの `owner_user_id` が認証ユーザーと一致(=作成者=オーナー)することを検証する |
| P-03 | 連絡先メールが設定済みかつ未確認状態であることを確認する |
| P-04 | レート制限(直近 X 分以内の再送上限)を確認し、超過している場合は 429 を返す |
| P-05 | 新しい連絡先確認トークンを発行し確認メールを再送する |
| P-06 | 再送結果を返す |

## リクエスト

#### 認証ヘッダ

本 API は利用者セッション(Cookie)と CSRF トークンを必要とする状態変更系である。

| ヘッダ | 要否 | 説明 |
|---|---|---|
| `Cookie: session=<token>` | 必須 | 利用者セッション |
| `X-CSRF-Token: <token>` | 必須 | CSRF 対策(状態変更系) |
| `Idempotency-Key: <ULID>` | 任意 | 冪等性(書き込み再送の重複防止) |

#### Query Parameters

```json
{}
```

#### Path Parameters

```json
{}
```

#### Request Body

```json
{ "projectId": "proj_01HMK4Z8Q..." }
```

| 項目        | 型     | 説明                               |
|-------------|--------|------------------------------------|
| `projectId` | string | 確認メールを再送する対象プロジェクト ID(必須) |

## レスポンス

**レスポンス(202)**

```json
{ "ok": true, "retryAfter": 300 }
```

| 項目          | 型      | 説明                                                     |
|---------------|---------|----------------------------------------------------------|
| `ok`          | boolean | 再送受付の成否                                           |
| `retryAfter`  | integer | 次回再送が可能になるまでの秒数(レート制限内の場合は `0`) |

## バリデーション

入力値の検証規則は `## リクエスト` の各パラメータ説明を参照する。

## エラー

| HTTP ステータス | エラーコード                        | 内容                                             |
|-----------------|-------------------------------------|--------------------------------------------------|
| 429 | `RATE_LIMITED` → [ERR-009](../../05_errors/ERR-009.md#ERR-009) | レート制限超過(再送間隔未到達) |
| 409 | `ALREADY_VERIFIED` → [ERR-010](../../05_errors/ERR-010.md#ERR-010) | 連絡先メールが既に確認済み |
| 404 | `PROJECT_NOT_FOUND` → [ERR-011](../../05_errors/ERR-011.md#ERR-011) | 対象プロジェクトが存在しない |
| 400 | `CONTACT_EMAIL_NOT_SET` → [ERR-012](../../05_errors/ERR-012.md#ERR-012) | 連絡先メールが未設定 |
