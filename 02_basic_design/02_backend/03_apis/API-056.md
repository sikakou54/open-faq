## 処理概要

アカウント保有者本人が自身のアカウントを即時に退会する(猶予なし)。退会の確定にはアカウントのメールアドレス一致確認と再認証を要する。退会と同時に、本人がメンバーとして参加しているプロジェクトから離脱(割当解除)し、本人がオーナー(作成者)であるプロジェクトを削除(運用停止)し、課金アカウントを `withdrawn`(退会済み)へ遷移させ、退会日時(`withdrawn_at`)を起算点として運用データの削除を起動し、請求データの [システム仕様書 §4](../../07_system-spec.md#4-データ保持期間削除猶予) に従って保持と保持期間経過後の削除予定を記録する。物理削除はせず、退会後も請求情報の閲覧のみ可能とする。

| 処理 ID | 処理内容                                                                                       |
|---------|------------------------------------------------------------------------------------------------|
| P-00    | 再認証トークン(リクエストボディ `reauthToken`)を検証する                                      |
| P-01    | 認証・認可(アカウント保有者本人、再認証必須)を検証する                                       |
| P-02    | 退会確認入力(リクエストボディ `confirmEmail`)がアカウントのメールアドレス(`M_USER.email`)と一致することを検証し、不一致のときは `400` を返す |
| P-03    | 課金アカウント状態を確認し、既に `withdrawn` / `deleted` のときは重複退会として `409` を返す   |
| P-04    | 本人がメンバーとして参加しているプロジェクトの割当(`M_PRJ_USERS`)を解除し離脱する             |
| P-05    | 本人がオーナー(`M_PROJECTS.owner_user_id` 一致)であるプロジェクトを削除(運用停止)へ遷移する  |
| P-06    | 課金アカウントを `withdrawn` へ即時遷移し、退会日時(`withdrawn_at`)を記録する                |
| P-07    | 運用データの削除を起動する(物理削除は [SYS-027](../01_system/SYS-027.md#SYS-027) が実施)     |
| P-08    | 退会記録を登録し、請求データの削除予定日([システム仕様書 §4](../../07_system-spec.md#4-データ保持期間削除猶予) に基づく削除予定日)を記録する                            |
| P-09    | 退会結果を返却する                                                                              |

## リクエスト

#### 認証ヘッダ

本 API は利用者セッション(Cookie)と CSRF トークンを必要とする状態変更系である。

| ヘッダ | 要否 | 説明 |
|---|---|---|
| `Cookie: session=<token>` | 必須 | 利用者セッション |
| `X-CSRF-Token: <token>` | 必須 | CSRF 対策(状態変更系) |
| `Idempotency-Key: <ULID>` | 任意 | 冪等性(書き込み再送の重複防止) |

> 本操作は再認証必須。直近の再認証([API-005](API-005.md#API-005))で得た再認証状態を伴う必要がある。

#### Query Parameters

```json
{}
```

#### Path Parameters

```json
{}
```

#### Request Body

```json
{ "reauthToken": "reauth_...", "confirmEmail": "owner@example.com", "reason": "退会理由(任意)" }
```

| 項目     | 型     | 説明                                         |
|----------|--------|----------------------------------------------|
| `reauthToken` | string | 再認証トークン(必須・API-005 で取得) |
| `confirmEmail` | string | 退会確認用メールアドレス(必須)。アカウントのメールアドレスと完全一致する必要がある |
| `reason` | string | 退会理由(任意・最大 500 文字) |

## レスポンス

**レスポンス(2xx)**

退会の実行結果(退会済みに遷移した課金アカウント状態・退会日時を含む)。

## バリデーション

入力値の検証規則は `## リクエスト` の各パラメータ説明を参照する。

| パラメータ | 規則 |
|---|---|
| `confirmEmail` | 必須。アカウントのメールアドレス(`M_USER.email`)と完全一致すること。不一致・未入力のときは `400 VALIDATION_ERROR` を返す |
| `reason` | 任意。指定する場合は最大 500 文字。超過時は `400 VALIDATION_ERROR` を返す |

## エラー

| HTTP ステータス | エラーコード        | 内容               |
|-----------------|---------------------|--------------------|
| 401 | `REAUTH_REQUIRED` → [ERR-013](../../05_errors/ERR-013.md#ERR-013) | 再認証トークンが無効または未提示 |
| 400 | `VALIDATION_ERROR` → [ERR-001](../../05_errors/ERR-001.md#ERR-001) | 退会確認メールアドレス(`confirmEmail`)がアカウントのメールアドレスと不一致・未入力、または退会理由(`reason`)が最大文字数(500)を超過 |
| 409 | `CONFLICT` → [ERR-023](../../05_errors/ERR-023.md#ERR-023) | 既に退会済み(`withdrawn`)または削除済み(`deleted`)で重複退会 |
