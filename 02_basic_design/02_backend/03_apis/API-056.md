# <span id="API-056"></span>API-056: 契約退会(即時)

> **このページは 契約退会(即時)API の契約を定義します。** オーナーが契約を即時に退会する(猶予なし)。

## 項目

| 項目 | 内容 |
|---|---|
| API ID | `API-056` |
| API名 | 契約退会(即時) |
| トレーサビリティID | [TR-023](../../00_traceability/index.md#TR-023) ・ [TR-085](../../00_traceability/index.md#TR-085) |
| エンドポイント | `/withdrawals` |
| HTTPメソッド | POST |
| 認証 | オーナー専有、再認証必須 |
| 認可 | オーナー専有 |

## 処理概要

オーナーが契約を即時に退会する(猶予なし)。退会と同時に契約を `withdrawn`(退会済み)へ遷移させ、運用データの削除を起動し、契約・請求データの 7 年保持と保持期間経過後の削除予定を記録する。

| 処理 ID | 処理内容                                                                                       |
|---------|------------------------------------------------------------------------------------------------|
| P-00    | 再認証トークン(リクエストボディ `reauthToken`)を検証する                                      |
| P-01    | 認証・認可(オーナー専有、再認証必須)を検証する                                               |
| P-02    | 契約状態を確認し、既に `withdrawn` / `deleted` のときは重複退会として `409` を返す             |
| P-03    | 契約を `withdrawn` へ即時遷移し、退会日時(`withdrawn_at`)を記録する                          |
| P-04    | 運用データの削除を起動する(物理削除は [SYS-029](../01_system/SYS-029.md#SYS-029) が実施)     |
| P-05    | 退会記録を登録し、契約・請求データの削除予定日(退会日 + 7 年)を記録する                      |
| P-06    | 退会結果を返却する                                                                              |

## リクエスト

#### 認証ヘッダ

本 API は利用者セッション(Cookie)と CSRF トークンを必要とする状態変更系である。

| ヘッダ | 要否 | 説明 |
|---|---|---|
| `Cookie: session=<token>` | 必須 | 利用者セッション |
| `X-CSRF-Token: <token>` | 必須 | CSRF 対策(状態変更系) |
| `Idempotency-Key: <ULID>` | 任意 | 冪等性(書き込み再送の重複防止) |

> 本操作は再認証必須。直近の再認証([API-005](API-005.md#API-005))で得た再認証状態を伴う必要がある。

#### Query Parameters

```json
{}
```

#### Path Parameters

```json
{}
```

#### Request Body

```json
{ "reauthToken": "reauth_...", "reason": "退会理由(任意)" }
```

| 項目     | 型     | 説明                                         |
|----------|--------|----------------------------------------------|
| `reauthToken` | string | 再認証トークン(必須・API-005 で取得) |
| `reason` | string | 退会理由(任意・最大 500 文字) |

## レスポンス

**レスポンス(2xx)**

退会の実行結果(退会済みに遷移した契約状態・退会日時を含む)。

## バリデーション

入力値の検証規則は `## リクエスト` の各パラメータ説明を参照する。

| パラメータ | 規則 |
|---|---|
| `reason` | 任意。指定する場合は最大 500 文字。超過時は `400 VALIDATION_ERROR` を返す |

## エラー

| HTTP ステータス | エラーコード        | 内容               |
|-----------------|---------------------|--------------------|
| 401 | `REAUTH_REQUIRED` → [ERR-013](../../05_errors/ERR-013.md#ERR-013) | 再認証トークンが無効または未提示 |
| 400 | `VALIDATION_ERROR` → [ERR-001](../../05_errors/ERR-001.md#ERR-001) | 退会理由(`reason`)が最大文字数(500)を超過 |
| 403 | `PERMISSION_DENIED` → [ERR-030](../../05_errors/ERR-030.md#ERR-030) | メンバーは退会不可(オーナー専有) |
| 409 | `CONFLICT` → [ERR-023](../../05_errors/ERR-023.md#ERR-023) | 既に退会済み(`withdrawn`)または削除済み(`deleted`)で重複退会 |

## 備考

- 退会は即時実行で猶予はない。退会申請の保留・退会撤回は廃止した。
- 運用データは退会と同時に削除を起動し、契約・請求データのみ退会日から 7 年保持する。保持期間経過後の物理削除は [SYS-036](../01_system/SYS-036.md#SYS-036) が実施する。
- 退会後のオーナーは保持期間中に限りログインでき、請求情報の閲覧([API-043](API-043.md#API-043) / [API-044](API-044.md#API-044))のみ可能となる(他操作は 403)。詳細は [PERM-009](../../04_permissions/PERM-009.md#PERM-009) を参照。
- 退会時のメンバーのアクセスは、契約状態(`withdrawn`)による認可ゲート([PERM-009](../../04_permissions/PERM-009.md#PERM-009)、[ERR-034](../../05_errors/ERR-034.md#ERR-034))で次回操作時に即時拒否される。本 API は同期的なセッション無効化・メンバー割当削除は行わず、メンバー割当・利用量等の運用データは [SYS-029](../01_system/SYS-029.md#SYS-029) が物理削除する(オーナーは請求閲覧のため保持期間中もログイン可)。
