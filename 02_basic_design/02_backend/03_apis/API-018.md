# <span id="API-018"></span>API-018: プロジェクト更新・削除

> **このページは プロジェクト更新・削除 API の契約を定義します。** プロジェクトの部分更新 / 論理削除(FR-039)。

## 項目

| 項目 | 内容 |
|---|---|
| API ID | `API-018` |
| API名 | プロジェクト更新・削除 |
| トレーサビリティID | [TR-016](../../00_traceability/index.md#TR-016) ・ [TR-017](../../00_traceability/index.md#TR-017) ・ [TR-039](../../00_traceability/index.md#TR-039) ・ [TR-077](../../00_traceability/index.md#TR-077) |
| エンドポイント | `/projects/{id}` |
| HTTPメソッド | PATCH / DELETE |
| 認証 | Cookie + CSRF(削除は再認証必須、L3 = プロジェクト名タイプ確認 + パスワード再認証) |
| 認可 | **オーナー専有** |

## 処理概要

プロジェクトの部分更新 / 論理削除(FR-039)。

| 処理 ID | 処理内容 |
|----|----|
| P-01 | 当該プロジェクトの全メンバー割当を論理削除する(オーナー自身の自動メンバー割当を含む) |
| P-02 | 他プロジェクトの有効な割当が 0 件かつオーナーでない利用者は、アカウントを論理削除し、全セッション失効と未使用招待トークンの失効を行う |
| P-03 | プロジェクトを削除状態(`status='deleted'`・論理削除)にする |
| P-04 | 関連データ(許可ドメイン / FAQ / 未解決質問 / 質問ログ)へ論理削除を伝播する。匿名化モードの即時匿名化は行わず、論理削除後の物理削除はシステム処理 SYS-029(猶予 90 日)で実施する |
| P-05 | 監査ログ(プロジェクト論理削除・`retention_class='general'`)を記録する |

上記 1-5 はアプリ層のトランザクション内で実装する。論理削除データは `updated_at` 基準で 90 日経過後にシステム処理 [SYS-029](../01_system/SYS-029.md#SYS-029) で物理削除される。物理削除時は `ON DELETE CASCADE` で関連行が連鎖物理削除される。

## リクエスト

#### 認証ヘッダ

本 API は利用者セッション(Cookie)と CSRF トークンを必要とする状態変更系である。

| ヘッダ | 要否 | 説明 |
|---|---|---|
| `Cookie: session=<token>` | 必須 | 利用者セッション |
| `X-CSRF-Token: <token>` | 必須 | CSRF 対策(PATCH / DELETE) |
| `Idempotency-Key: <ULID>` | 任意 | 冪等性(書き込み再送の重複防止) |

> DELETE(プロジェクト削除)は再認証必須。直近の再認証([API-005](API-005.md#API-005))で得た再認証状態を伴う必要がある。

#### Query Parameters

```json
{}
```

#### Path Parameters

```json
{ "id": "プロジェクト ID" }
```

| 項目 | 型     | 説明                                                   |
|------|--------|--------------------------------------------------------|
| `id` | string | 更新 / 削除の対象プロジェクトの ID(必須・パス内で指定) |

#### Request Body

```json
{
  "name": "...",
  "allowedDomains": ["..."],
  "contactEmail": "...",
  "settings": {
    "headerColor": "#2563EB",
    "placement": "bottom-right",
    "headerTitle": "...",
    "welcomeMessage": "..."
  }
}
```

| 項目 | 型 | 説明 |
|----|----|----|
| `name` | string | 更新後のプロジェクト名(PATCH 時・任意の部分更新項目) |
| `allowedDomains` | array | 更新後の許可ドメインのリスト(PATCH 時・任意の部分更新項目) |
| `contactEmail` | string | 更新後の連絡先メールアドレス(PATCH 時・任意の部分更新項目) |
| `settings` | object | 更新後のウィジェット見た目設定(PATCH 時・任意の部分更新項目)。供給元は [`M_PROJECTS`](../04_database/TBL-004.md#TBL-004) の `settings`(JSON)で、指定したキーのみを更新する |
| `settings.headerColor` | string | ウィジェットの主色(HEX。`#` + 6 桁、`^#[0-9A-Fa-f]{6}$`) |
| `settings.placement` | enum | ウィジェットの表示位置(`bottom-left` = 左下 / `bottom-right` = 右下) |
| `settings.headerTitle` | string | ウィジェットの見出し(0〜60 文字) |
| `settings.welcomeMessage` | string | ウィジェットの初期メッセージ(0〜200 文字) |

上記 Request Body は PATCH(部分更新)時のもの。DELETE 時はボディなし。`DELETE /projects/{id}` 実行時の挙動(FR-039、論理削除)。

## レスポンス

**レスポンス(200)**

PATCH は更新後のプロジェクト、DELETE は `{ "ok": true, "status": "deleted" }`。

**PATCH(更新後のプロジェクト)**

| 項目             | 型     | 説明                         |
|------------------|--------|------------------------------|
| `id`             | string | 更新したプロジェクトの ID    |
| `name`           | string | 更新後のプロジェクト名       |
| `allowedDomains` | array  | 更新後の許可ドメインのリスト |
| `contactEmail`   | string | 更新後の連絡先メールアドレス |
| `settings`       | object | 更新後のウィジェット見た目設定。供給元は [`M_PROJECTS`](../04_database/TBL-004.md#TBL-004) の `settings`(JSON) |
| `settings.headerColor`    | string | ウィジェットの主色(HEX。`#` + 6 桁)             |
| `settings.placement`      | enum   | ウィジェットの表示位置(`bottom-left` / `bottom-right`) |
| `settings.headerTitle`    | string | ウィジェットの見出し(0〜60 文字)                 |
| `settings.welcomeMessage` | string | ウィジェットの初期メッセージ(0〜200 文字)         |

**DELETE(論理削除結果)**

| 項目     | 型      | 説明                                |
|----------|---------|-------------------------------------|
| `ok`     | boolean | 論理削除の成否                      |
| `status` | enum    | 削除後のプロジェクト状態(`deleted`) |
