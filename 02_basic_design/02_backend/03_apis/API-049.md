# <span id="API-049"></span>API-049: お知らせ個別既読

> **このページは お知らせ個別既読 API の契約を定義します。** お知らせ 1 件を既読にする。

## 項目

| 項目 | 内容 |
|---|---|
| API ID | `API-049` |
| API名 | お知らせ個別既読 |
| 対応業務UC | [UC-046](../../../01_requirements/04_business_usecases/UC-046.md#UC-046) ・ [UC-045](../../../01_requirements/04_business_usecases/UC-045.md#UC-045) |
| 対応画面ID | [SCR-016](../../01_frontend/01_screens/SCR-016.md#SCR-016) ・ [SCR-017](../../01_frontend/01_screens/SCR-017.md#SCR-017) |
| 対応画面イベントID | EVT-141 ・ EVT-147 |
| エンドポイント | `/me/announcements/{id}/read` |
| HTTPメソッド | POST |
| 認証 | Cookie + CSRF |
| 認可 | オーナー / メンバー |

## 処理概要

お知らせ 1 件を既読にする。

| 処理 ID | 処理内容                                          |
|---------|---------------------------------------------------|
| P-01    | 認証・認可(オーナー / メンバー、CSRF)を検証する   |
| P-02    | 対象お知らせ 1 件を既読に更新する(既読済みは冪等) |
| P-03    | 既読時刻を返却する                                |

## リクエスト

#### 認証ヘッダ

本 API は利用者セッション(Cookie)と CSRF トークンを必要とする状態変更系である。

| ヘッダ | 要否 | 説明 |
|---|---|---|
| `Cookie: session=<token>` | 必須 | 利用者セッション |
| `X-CSRF-Token: <token>` | 必須 | CSRF 対策(状態変更系) |
| `Idempotency-Key: <ULID>` | 任意 | 冪等性(書き込み再送の重複防止) |

#### Query Parameters

```json
{}
```

#### Path Parameters

```json
{ "id": "お知らせ ID" }
```

| 項目 | 型     | 説明                          |
|------|--------|-------------------------------|
| `id` | string | 既読にするお知らせの ID。必須 |

#### Request Body

```json
{}
```

## レスポンス

**レスポンス(200)**

```json
{ "id": "...", "readAt": "..." }
```

| 項目     | 型                | 説明                    |
|----------|-------------------|-------------------------|
| `id`     | string            | 既読にしたお知らせの ID |
| `readAt` | string (ISO 8601) | 既読日時                |
