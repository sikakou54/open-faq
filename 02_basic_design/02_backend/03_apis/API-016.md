# <span id="API-016"></span>API-016: プロジェクト一覧

> **このページは プロジェクト一覧 API の仕様を定義します。** 認証ユーザーが関与するプロジェクト一覧を返す。`scope` で「自分が作成したプロジェクト(オーナー)」と「参加プロジェクト(メンバー)」を切り替える。

## 項目

| 項目 | 内容 |
|---|---|
| API ID | `API-016` |
| API名 | プロジェクト一覧 |
| トレーサビリティID | [TR-014](../../00_traceability/index.md#TR-014) ・ [TR-090](../../00_traceability/index.md#TR-090) |
| エンドポイント | `/projects` |
| HTTPメソッド | GET |
| 認証 | Cookie |
| 認可 | **認証ユーザー**(取得対象は `scope` と本人の立場で絞り込む) |

## 処理概要

認証ユーザーが関与するプロジェクト一覧を返す。`scope=my` は自分が作成した(オーナーの)プロジェクト、`scope=join` は他者作成で自分が割り当てられている(メンバーの)プロジェクトを返す。

| 処理 ID | 処理内容 |
|----|----|
| P-01 | 認証(Cookie セッション)を検証する |
| P-02 | `scope` に応じて対象プロジェクトを絞り込む。`my` は M_PROJECTS の `owner_user_id` が認証ユーザーと一致するプロジェクト、`join` は M_PRJ_USERS に有効割当(`valid = 1`)があり、かつ `owner_user_id` が認証ユーザーと一致しない(非オーナーの)プロジェクトを取得する。各プロジェクトについて認証ユーザーの立場(`owner_user_id` 一致なら `owner`、それ以外で割当ありなら `member`)を判定して付帯する |
| P-03 | 有効なプロジェクトを取得する(FAQ 件数・許可ドメイン等を付帯)。あわせて質問数(H_QUESTION_LOGS を `project_id` 別に集計)・未解決質問件数(T_INQUIRIES `status = 'open' AND valid = 1`)・メンバー割当件数(M_PRJ_USERS `valid = 1`)を付帯し、稼働ステータス(当月質問数が M_PRJ_QUOTA_LIMITS の `q_monthly_limit` に到達、またはオーナーの課金アカウント M_BILLING_ACCOUNT が `suspended` 等で制限されている場合は `limited`)を判定して付帯する |
| P-04 | ページング(カーソル)で整形して返す |

## リクエスト

#### 認証ヘッダ

本 API は利用者セッション(Cookie)による認証を必要とする。

| ヘッダ | 要否 | 説明 |
|---|---|---|
| `Cookie: session=<token>` | 必須 | 利用者セッション |

#### Query Parameters

```json
{
  "scope": "my|join",
  "cursor": "(共通仕様)",
  "limit": "(共通仕様)"
}
```

| 項目     | 型      | 説明                                             |
|----------|---------|--------------------------------------------------|
| `scope`  | enum    | 取得対象の絞り込み(任意・既定 `my`)。`my`=自分が作成したプロジェクト(オーナー: `owner_user_id` が認証ユーザーと一致)、`join`=参加プロジェクト(メンバー: M_PRJ_USERS に有効割当があり、かつ非オーナー) |
| `cursor` | string  | 次ページ取得用カーソル(任意・共通ページング仕様) |
| `limit`  | integer | 1 ページの取得件数(任意・共通ページング仕様)     |

```json
{}
```

```json
{}
```

## レスポンス

**レスポンス(200)**

```json
{
  "items": [
    {
      "id": "01HZ...",
      "name": "...",
      "status": "active|deleted",
      "operationalStatus": "active|limited",
      "myRole": "owner|member",
      "createdAt": "...",
      "faqCount": 42,
      "questionCount": 1247,
      "unresolvedCount": 12,
      "memberCount": 6,
      "widgetKey": "pk_live_...",
      "allowedDomains": ["example.com", "*.example.com"],
      "contactEmail": "contact@...",
      "contactEmailVerifiedAt": "..." | null
    }
  ],
  "nextCursor": null
}
```

| 項目 | 型 | 説明 |
|----|----|----|
| `items` | array\<object\> | 認証ユーザーが関与するプロジェクト一覧(`scope` と本人の立場で絞り込み) |
| `　└ id` | string | プロジェクト ID |
| `　└ name` | string | プロジェクト名 |
| `　└ status` | enum | プロジェクト状態(永続状態。`active` / `deleted`) |
| `　└ operationalStatus` | enum | 稼働ステータス(導出値。`active`=稼働中 / `limited`=制限中)。当月質問数が上限([`M_PRJ_QUOTA_LIMITS`](../04_database/TBL-009.md#TBL-009) の `q_monthly_limit`)に到達、またはオーナーの課金アカウント([`M_BILLING_ACCOUNT`](../04_database/TBL-002.md#TBL-002))が `suspended` 等で制限されている場合に `limited` を返す。`status` とは別の派生フィールド |
| `　└ createdAt` | string (ISO 8601) | 作成日時 |
| `　└ faqCount` | integer | 当該プロジェクトの FAQ 件数 |
| `　└ questionCount` | integer | 当該プロジェクトの質問数([`H_QUESTION_LOGS`](../04_database/TBL-025.md#TBL-025) を `project_id` 別に集計) |
| `　└ unresolvedCount` | integer | 未解決質問件数([`T_INQUIRIES`](../04_database/TBL-017.md#TBL-017) を `project_id` 別に `status = 'open' AND valid = 1` で集計) |
| `　└ memberCount` | integer | プロジェクトに割り当てられたメンバー数([`M_PRJ_USERS`](../04_database/TBL-003.md#TBL-003) を `project_id` 別に `valid = 1` で集計) |
| `　└ widgetKey` | string | ウィジェット公開キー |
| `　└ allowedDomains` | array | 許可ドメインのリスト |
| `　└ contactEmail` | string | プロジェクト連絡先メールアドレス |
| `　└ contactEmailVerifiedAt` | string (ISO 8601) | 連絡先メール確認日時(未確認は `null`) |
| `nextCursor` | string | 次ページ取得用カーソル(末尾は `null`) |
