# <span id="API-001"></span>API-001: 新規登録

> **このページは 新規登録 API の仕様を定義します。** 利用者アカウントを新規登録(独立サインアップ)し、メール確認を要求する。

| ID | 業務ユースケースID | イベント(画面ID EVT-NN) | テーブルID |
|----|----|----|----|
| API-001 | [UC-002](../../../01_requirements/04_business_usecases/UC-002.md#UC-002) ・ [UC-003](../../../01_requirements/04_business_usecases/UC-003.md#UC-003) | SCR-002 EVT-05 ・ SCR-018 EVT-02 | [TBL-001](../04_database/TBL-001.md#TBL-001) ・ [TBL-012](../04_database/TBL-012.md#TBL-012) ・ [TBL-014](../04_database/TBL-014.md#TBL-014) ・ [TBL-024](../04_database/TBL-024.md#TBL-024) ・ [TBL-027](../04_database/TBL-027.md#TBL-027) |

| API名 | エンドポイント | HTTPメソッド |
|----|----|----|
| 新規登録 | `/auth/signup` | POST |
| 確認メール再送 | `/auth/signup/email-verification` | POST |

## 処理概要

利用者アカウントを新規登録(独立サインアップ)し、メール確認を要求する。利用者本体として `M_USER` を作成し、登録時の利用規約・プライバシーポリシー同意を `T_TERMS_AGREE` に記録し、本人確認用の短期トークンを `T_ACCESS_TOKENS` に発行する。課金アカウントやプロジェクトは作成しない(作成直後はどのプロジェクトのオーナーでもメンバーでもない)。

| 処理 ID | 処理内容 |
|----|----|
| P-01 | 入力値(表示名・メール形式・パスワード強度・規約 / プライバシー同意)を検証する |
| P-02 | `M_USER`(利用者本体)を作成する |
| P-03 | 利用規約・プライバシーポリシーへの同意を記録する |
| P-04 | メール確認トークンを発行し確認メールを送信する |
| P-05 | 受付結果を返す |
| P-06 | 確認メール再送要求では、未確認アカウントに対して既存の未使用確認トークンを失効し、新しい確認トークンを発行して確認メールを再送する |

## リクエスト

#### 認証ヘッダ

本 API は公開エンドポイントで、認証セッションを必要としない(資格情報・入力はリクエストボディで送出)。

| ヘッダ | 要否 | 説明 |
|---|---|---|
| `Idempotency-Key: <ULID>` | 任意 | 冪等性(再送の重複防止) |

#### Query Parameters

```json
{}
```

#### Path Parameters

```json
{}
```

#### Request Body: 新規登録

```json
{
  "name": "山田 太郎",
  "email": "user@example.com",
  "password": "P@ssw0rd123!",
  "agreedTerms": true,
  "agreedPrivacy": true
}
```

| 項目              | 型      | 説明                                       |
|-------------------|---------|--------------------------------------------|
| `name`            | string  | 表示名(プロフィール名。必須・最大 100 文字。`M_USER.name` へ保存) |
| `email`           | string  | 登録するメールアドレス(必須・RFC 5322 準拠のメール形式・最大 254 文字) |
| `password`        | string  | 初回パスワード(必須・12 文字以上、英大文字 / 小文字 / 数字 / 記号のうち 3 種類以上 = FR-006) |
| `agreedTerms`     | boolean | 利用規約への同意(必須・`true`)             |
| `agreedPrivacy`   | boolean | プライバシーポリシーへの同意(必須・`true`) |

#### Request Body: 確認メール再送

```json
{
  "email": "user@example.com"
}
```

| 項目 | 型 | 説明 |
|---|---|---|
| `email` | string | 再送対象のメールアドレス(必須・RFC 5322 準拠・最大 254 文字) |

## レスポンス

**レスポンス(202)**

```json
{ "ok": true }
```

| 項目 | 型      | 説明                                                  |
|------|---------|-------------------------------------------------------|
| `ok` | boolean | 新規登録受付または確認メール再送受付を受理した場合 `true` |

## バリデーション

入力値の検証規則は `## リクエスト` の各パラメータ説明を参照する。主な規則は次のとおり。

- `name`: 必須・最大 100 文字(空文字不可)。
- `email`: 必須・RFC 5322 準拠のメール形式・最大 254 文字。
- `password`: 必須・12 文字以上、英大文字 / 小文字 / 数字 / 記号のうち 3 種類以上(FR-006)。
- `agreedTerms` / `agreedPrivacy`: 必須・`true`。
- 既存メール重複(処理概要 P-02 のチェック失敗)は 409 `EMAIL_ALREADY_USED` を返す。

## エラー

| HTTP ステータス | エラーコード                   | 内容               |
|-----------------|--------------------------------|--------------------|
| 400 | `VALIDATION_ERROR` → [ERR-001](../../05_errors/ERR-001.md#ERR-001) | 入力値の検証エラー(メール形式 / パスワード強度) |
| 409 | `EMAIL_ALREADY_USED` → [ERR-014](../../05_errors/ERR-014.md#ERR-014) | メールアドレス重複(既存登録あり。P-02 のチェック失敗時) |
| 429 | `RATE_LIMITED` → [ERR-009](../../05_errors/ERR-009.md#ERR-009) | 確認メール再送の間隔未到達 |
