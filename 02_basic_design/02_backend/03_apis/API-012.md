# <span id="API-012"></span>API-012: 自己プロフィール更新

> **このページは 自己プロフィール更新 API の仕様を定義します。** 認証済み利用者が自身の表示名・メールアドレスを更新する。メールアドレス変更時は再認証トークンを要求し、新アドレスへ確認メールを送信する。

## 項目

| 項目 | 内容 |
|---|---|
| API ID | `API-012` |
| API名 | 自己プロフィール更新 |
| トレーサビリティID | [TR-009](../../00_traceability/index.md#TR-009) |
| エンドポイント | `/me/profile` |
| HTTPメソッド | PATCH |
| 認証 | 要(Cookie + CSRF) |
| 認可 | 認証済み |

## 処理概要

認証済み利用者が自身の表示名・メールアドレスを更新する。メールアドレス変更時は再認証トークンを要求し、新アドレスへ確認メールを送信する。

| 処理 ID | 処理内容 |
|---------|----------|
| P-01 | セッションから認証済み利用者を特定する |
| P-02 | リクエストボディの入力値(表示名長・メールアドレス形式)を検証する |
| P-03 | メールアドレス変更時は再認証トークン(リクエストボディ `reauthToken`)を検証する |
| P-04 | 表示名(`displayName`→`M_USER.name`)・メールアドレス(`email`→`M_USER.email`)を `M_USER` に更新する |
| P-05 | メールアドレス変更時はメール確認トークンを発行し新アドレスへ確認メールを送信する |
| P-06 | 更新結果を返す |

## リクエスト

#### 認証ヘッダ

本 API は利用者セッション(Cookie)と CSRF トークンを必要とする状態変更系である。

| ヘッダ | 要否 | 説明 |
|---|---|---|
| `Cookie: session=<token>` | 必須 | 利用者セッション |
| `X-CSRF-Token: <token>` | 必須 | CSRF 対策(状態変更系) |
| `Idempotency-Key: <ULID>` | 任意 | 冪等性(書き込み再送の重複防止) |

#### Query Parameters

```json
{}
```

#### Path Parameters

```json
{}
```

#### Request Body

```json
{
  "displayName": "山田 太郎",
  "email": "new@example.com",
  "reauthToken": "reauth_..."
}
```

| 項目           | 型     | 説明                                                       |
|----------------|--------|------------------------------------------------------------|
| `displayName`  | string | 新しい表示名(任意・1〜100 文字)                            |
| `email`        | string | 新しいメールアドレス(任意・変更時は `reauthToken` 必須)    |
| `reauthToken`  | string | 再認証トークン(メールアドレス変更時必須・API-005 で取得) |

## レスポンス

**レスポンス(200)**

```json
{ "ok": true, "emailVerificationPending": false }
```

| 項目                       | 型      | 説明                                                      |
|----------------------------|---------|-----------------------------------------------------------|
| `ok`                       | boolean | 更新の成否                                                |
| `emailVerificationPending` | boolean | 確認メールを送信して検証待ちの場合 `true`(メールアドレス変更時) |

## バリデーション

入力値の検証規則は `## リクエスト` の各パラメータ説明を参照する。

## エラー

| HTTP ステータス | エラーコード                   | 内容                         |
|-----------------|--------------------------------|------------------------------|
| 400 | `VALIDATION_ERROR` → [ERR-001](../../05_errors/ERR-001.md#ERR-001) | 入力値の検証エラー |
| 401 | `REAUTH_REQUIRED` → [ERR-013](../../05_errors/ERR-013.md#ERR-013) | 再認証トークンが無効または未提示 |
| 409 | `EMAIL_ALREADY_USED` → [ERR-014](../../05_errors/ERR-014.md#ERR-014) | 入力メールアドレスが既に使用中 |

## 備考

表示名・メールアドレスは全ユーザー共通の `M_USER` に保持する(`displayName`→`M_USER.name` / `email`→`M_USER.email`)。オーナー / メンバーの区別なく本人の `M_USER` 行を更新する。
