# <span id="API-012"></span>API-012: 自己プロフィール更新

> **このページは 自己プロフィール更新 API の仕様を定義します。** 認証済み利用者が自身の表示名(プロフィール名)を更新する。メールアドレス・パスワード等のセキュリティ設定は [API-015](API-015.md#API-015) が担う(本 API は表示名のみを扱う)。

| ID | 業務ユースケースID | イベント(画面ID EVT-NN) | テーブルID |
|----|----|----|----|
| API-012 | [UC-009](../../../01_requirements/04_business_usecases/UC-009.md#UC-009) | SCR-022 EVT-03 | [TBL-001](../04_database/TBL-001.md#TBL-001) ・ [TBL-002](../04_database/TBL-002.md#TBL-002) ・ [TBL-003](../04_database/TBL-003.md#TBL-003) ・ [TBL-014](../04_database/TBL-014.md#TBL-014) |

| API名 | エンドポイント | HTTPメソッド |
|----|----|----|
| 自己プロフィール更新 | `/me/profile` | PATCH |

## 処理概要

認証済み利用者が自身の表示名(プロフィール名)を更新する。メールアドレス・パスワードのセキュリティ設定は本 API では扱わない([API-015](API-015.md#API-015) が担う)。

| 処理 ID | 処理内容 |
|---------|----------|
| P-01 | セッションから認証済み利用者を特定する |
| P-02 | リクエストボディの入力値(表示名の必須・文字数 1〜100)を検証する |
| P-03 | 表示名(`displayName`→`M_USER.name`)を `M_USER` に更新する |
| P-04 | 更新結果を返す |

## リクエスト

#### 認証ヘッダ

本 API は利用者セッション(Cookie)と CSRF トークンを必要とする状態変更系である。

| ヘッダ | 要否 | 説明 |
|---|---|---|
| `Cookie: session=<token>` | 必須 | 利用者セッション |
| `X-CSRF-Token: <token>` | 必須 | CSRF 対策(状態変更系) |
| `Idempotency-Key: <ULID>` | 任意 | 冪等性(書き込み再送の重複防止) |

#### Query Parameters

```json
{}
```

#### Path Parameters

```json
{}
```

#### Request Body

```json
{
  "displayName": "山田 太郎"
}
```

| 項目           | 型     | 説明                                                       |
|----------------|--------|------------------------------------------------------------|
| `displayName`  | string | 新しい表示名(必須・空文字不可・1〜100 文字)                |

## レスポンス

**レスポンス(200)**

```json
{ "ok": true, "displayName": "山田 太郎" }
```

| 項目          | 型      | 説明                       |
|---------------|---------|----------------------------|
| `ok`          | boolean | 更新の成否                 |
| `displayName` | string  | 更新後の表示名(`M_USER.name`) |
