## 処理概要

過去請求書を一覧取得する。

| 処理 ID | 処理内容                                      |
|---------|-----------------------------------------------|
| P-01    | 認証・認可(オーナー専有)を検証する            |
| P-02    | ログイン中ユーザーの課金アカウント(`billing_account_id`)に紐づく過去請求書を新しい順に取得する。プラン名と請求年月(`billing_ym`)から請求内容の表示文字列(`description`)を導出して付与する |
| P-03    | PDF 署名 URL(有効期限 5 分)を付与して返却する |

## リクエスト

#### 認証ヘッダ

本 API は利用者セッション(Cookie)による認証を必要とする。

| ヘッダ | 要否 | 説明 |
|---|---|---|
| `Cookie: session=<token>` | 必須 | 利用者セッション |

#### Query Parameters

```json
{ "limit": "6(等)" }
```

| 項目    | 型      | 説明                                |
|---------|---------|-------------------------------------|
| `limit` | integer | 取得する請求書の件数。任意(例: `6`) |

## レスポンス

**レスポンス(200)**

```json
{
  "items": [
    {
      "id": "inv_...",
      "date": "2026-05-01T...",
      "description": "スタンダードプラン(5 月分)",
      "amount": 5000,
      "currency": "JPY",
      "status": "paid|failed|draft",
      "pdfUrl": "https://..."
    }
  ]
}
```

| 項目           | 型                | 説明                                      |
|----------------|-------------------|-------------------------------------------|
| `items`        | array\<object\>   | 請求書の配列(新しい順)                    |
| `　└ id`          | string            | 請求書の ID                               |
| `　└ date`        | string (ISO 8601) | 請求日                                    |
| `　└ description` | string            | 請求内容の表示文字列(例: `スタンダードプラン(5 月分)`)。プラン名と請求年月(`T_BILL_INVOICES.billing_ym`)から導出して返す |
| `　└ amount`      | integer           | 請求金額                                  |
| `　└ currency` | string            | 通貨コード                                |
| `　└ status`   | enum              | 請求書の状態。`paid` / `failed` / `draft` |
| `　└ pdfUrl`   | string            | 請求書 PDF の署名 URL(有効期限 5 分)      |

`pdfUrl` は R2 署名 URL(有効期限 5 分)。
