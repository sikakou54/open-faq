# <span id="API-002"></span>API-002: ログイン

> **このページは ログイン API の仕様を定義します。** メール / パスワードを認証しセッションを発行する。

## 項目

| 項目 | 内容 |
|---|---|
| API ID | `API-002` |
| API名 | ログイン |
| トレーサビリティID | [TR-001](../../00_traceability/index.md#TR-001) ・ [TR-071](../../00_traceability/index.md#TR-071) ・ [TR-072](../../00_traceability/index.md#TR-072) |
| エンドポイント | `/auth/login` |
| HTTPメソッド | POST |
| 認証 | 不要 |
| 認可 | —(公開) |

## 処理概要

メール / パスワードを認証しセッションを発行する。

| 処理 ID | 処理内容 |
|----|----|
| P-01 | メール / パスワードの資格情報を照合する(不一致は失敗試行として計上) |
| P-02 | アカウントロック状態(5 回連続失敗で 15 分ロック = RULE-001)・アカウント停止(サスペンション)状態を判定する |
| P-03 | セッションを発行し、再規約同意要否と有効セッション一覧を返す |

## リクエスト

#### 認証ヘッダ

本 API は公開エンドポイントで、認証セッションを必要としない(資格情報・入力はリクエストボディで送出)。

| ヘッダ | 要否 | 説明 |
|---|---|---|
| `Idempotency-Key: <ULID>` | 任意 | 冪等性(再送の重複防止) |

#### Query Parameters

```json
{}
```

#### Path Parameters

```json
{}
```

#### Request Body

```json
{
  "email": "admin@example.com",
  "password": "..."
}
```

| 項目 | 型 | 説明 |
|----|----|----|
| `email` | string | ログインするメールアドレス(必須) |
| `password` | string | パスワード(必須) |

## レスポンス

**レスポンス(200)**

```json
{
  "userId": "01HZ...",
  "actorType": "owner",
  "requireTermsAgreement": false,
  "activeSessions": [
    { "id": "01HZ...", "ipAddress": "203.0.113.x", "userAgent": "Mozilla/5.0 ...", "createdAt": "2026-06-24T09:00:00Z" }
  ]
}
```

| 項目                    | 型                | 説明                           |
|-------------------------|-------------------|--------------------------------|
| `userId`                | string            | 認証された利用者の ID          |
| `actorType`             | enum              | 利用者の種別(`owner` = オーナー / `member` = メンバー) |
| `requireTermsAgreement` | boolean           | 再規約同意が必要かどうか       |
| `activeSessions`        | array\<object\>   | 現在有効なセッションの一覧     |
| `　└ id`                | string (ULID)     | セッション ID                  |
| `　└ ipAddress`         | string            | セッション発行元の IP アドレス |
| `　└ userAgent`         | string            | セッション発行元の User-Agent  |
| `　└ createdAt`         | string (ISO 8601) | セッション作成日時             |

## バリデーション

入力値の検証規則は `## リクエスト` の各パラメータ説明を参照する。主な規則は次のとおり。

- `email` / `password`: 必須。
- 連続失敗のロックアウト: 同一アカウントで 5 回連続失敗すると 15 分間ロックする(RULE-001)。ロック中の試行は 423 `LOCKED_OUT` を返す。

## エラー

| HTTP ステータス | エラーコード | 内容 |
|----|----|----|
| 401 | `INVALID_CREDENTIALS` → [ERR-002](../../05_errors/ERR-002.md#ERR-002) | 認証情報が不正 |
| 423 | `LOCKED_OUT` → [ERR-003](../../05_errors/ERR-003.md#ERR-003) | アカウントロック中(5 回連続失敗で 15 分ロック = RULE-001) |
| 403 | `CONTRACT_SUSPENDED` → [ERR-004](../../05_errors/ERR-004.md#ERR-004) | アカウント停止中(サスペンション) |
