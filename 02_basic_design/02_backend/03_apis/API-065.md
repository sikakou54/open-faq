# <span id="API-065"></span>API-065: プロジェクト範囲データ概要取得

> **このページは プロジェクト範囲データ概要取得 API の契約を定義します。** 当該プロジェクトに属する FAQ・質問ログ・未解決質問の件数と概要を返し、プロジェクト単位のデータ分離を担保する参照系 API。

## 項目

| 項目 | 内容 |
|---|---|
| API ID | `API-065` |
| API名 | プロジェクト範囲データ概要取得 |
| トレーサビリティID | [TR-078](../../00_traceability/index.md#TR-078) |
| エンドポイント | `/projects/{id}/overview` |
| HTTPメソッド | GET |
| 認証 | セッション(Cookie) |
| 認可 | オーナー / 当該プロジェクトのメンバー |

## 処理概要

当該プロジェクトに属する FAQ・質問ログ・未解決質問の件数と概要を返す。集計は常に当該プロジェクトのデータに限定し、他プロジェクトのデータを混在させないこと(プロジェクト単位のデータ分離)を担保する([SCR-006](../../01_frontend/01_screens/SCR-006.md#SCR-006) / [SCR-007](../../01_frontend/01_screens/SCR-007.md#SCR-007) / [SCR-008](../../01_frontend/01_screens/SCR-008.md#SCR-008))。

| 処理 ID | 処理内容 |
|---------|----------|
| P-01 | リクエスト元が当該プロジェクトに割当のあるオーナー / メンバーかを判定する(不一致は 403) |
| P-02 | 対象プロジェクトの存在を確認する(存在しなければ 404) |
| P-03 | 当該プロジェクトに範囲を限定して FAQ・質問ログ・未解決質問を集計する |
| P-04 | 件数と概要をまとめて返す |

## リクエスト

#### 認証ヘッダ

本 API は利用者セッション(Cookie)による認証を必要とする参照系であり、CSRF トークンは不要である。

| ヘッダ | 要否 | 説明 |
|---|---|---|
| `Cookie: session=<token>` | 必須 | 利用者セッション |

#### Query Parameters

```json
{}
```

#### Path Parameters

```json
{
  "id": "対象プロジェクト ID"
}
```

| 項目 | 型 | 説明 |
|---|---|---|
| `id` | string | 対象プロジェクト ID(必須) |

## レスポンス

当該プロジェクトに範囲を限定した FAQ・質問ログ・未解決質問の件数と概要を返す。

**レスポンス(200)**

```json
{
  "projectId": "prj_01",
  "faqCount": 124,
  "questionLogCount": 3820,
  "openInquiryCount": 17
}
```

| 項目 | 型 | 説明 |
|---|---|---|
| `projectId` | string | 対象プロジェクト ID |
| `faqCount` | number | 当該プロジェクトの FAQ 件数 |
| `questionLogCount` | number | 当該プロジェクトの質問ログ件数 |
| `openInquiryCount` | number | 当該プロジェクトの未解決質問件数 |

## バリデーション

入力値の検証規則は `## リクエスト` の各パラメータ説明を参照する。

## エラー

| HTTP ステータス | エラーコード | 内容 |
|-----------------|---------------------|--------------------------------|
| 403 | `PROJECT_ACCESS_DENIED` → [ERR-021](../../05_errors/ERR-021.md#ERR-021) | 当該プロジェクトに割当のないユーザーのアクセス |
| 404 | `PROJECT_NOT_FOUND` → [ERR-013](../../05_errors/ERR-013.md#ERR-013) | 対象プロジェクトが存在しない |

## 備考

集計は常に当該プロジェクト(`{id}`)に範囲を限定し、他プロジェクトのデータを混在させない。プロジェクト単位のデータ分離を実現する参照系 API であり、各データの更新は対応する個別 API が担う。
