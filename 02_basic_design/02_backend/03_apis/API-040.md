# <span id="API-040"></span>API-040: ダッシュボードサマリ

> **このページは ダッシュボードサマリ API の契約を定義します。** 質問数・未解決・要対応状況・通知失敗 / バウンスをまとめて返す。

## 項目

| 項目 | 内容 |
|---|---|
| API ID | `API-040` |
| API名 | ダッシュボードサマリ |
| トレーサビリティID | [TR-033](../../00_traceability/index.md#TR-033) ・ [TR-055](../../00_traceability/index.md#TR-055) ・ [TR-066](../../00_traceability/index.md#TR-066) |
| エンドポイント | `/dashboard/summary` |
| HTTPメソッド | GET |
| 認証 | Cookie |
| 認可 | オーナー / 当該プロジェクトのメンバー(`notification` ブロックの配信失敗・バウンス件数はオーナー専有) |

## 処理概要

質問数・未解決・要対応状況・通知失敗 / バウンスをまとめて返す。

| 処理 ID | 処理内容 |
|----|----|
| P-01 | 認証・認可(対象プロジェクトへのアクセス権、メンバーは `projectId` 必須)を検証する |
| P-02 | 期間・プロジェクトの質問数・未解決数・要対応状況を集計する(集計の基準タイムゾーンは UTC = 設計値。`current_month` は UTC 当月初日 00:00 起算)。`period` と `from` / `to` は排他で、双方指定時は `from` / `to` を優先する |
| P-03 | オーナー向けに通知失敗・バウンス件数を集計する(メンバー指定時は `null`) |
| P-04 | 整形して返却する |

## リクエスト

#### 認証ヘッダ

本 API は利用者セッション(Cookie)による認証を必要とする。

| ヘッダ | 要否 | 説明 |
|---|---|---|
| `Cookie: session=<token>` | 必須 | 利用者セッション |

#### Query Parameters

```json
{ "period": "current_month", "projectId": "{id}(メンバーは必須)", "from": "任意(period の代わりに範囲指定)", "to": "任意(period の代わりに範囲指定)" }
```

| 項目 | 型 | 説明 |
|----|----|----|
| `period` | enum | 集計期間。任意。`current_month`(当月)/ `last_30d`(直近 30 日)。`from` / `to` を指定する場合は不要。基準タイムゾーンは UTC(設計値。契約はタイムゾーン設定を持たない)。`current_month` は UTC の当月初日 00:00:00 起算、`last_30d` は集計時点から遡る 30 日間 |
| `projectId` | string | 対象プロジェクトの ID。オーナーは任意(未指定で契約全体を集約)、メンバーは必須 |
| `from` | string (ISO 8601) | 集計期間の起点。任意(`period` の代わりに範囲指定する場合に使用) |
| `to` | string (ISO 8601) | 集計期間の終点。任意(`period` の代わりに範囲指定する場合に使用) |

#### Path Parameters

```json
{}
```

```json
{}
```

## レスポンス

**レスポンス(200)**

```json
{
  "period": { "from": "2026-06-01T00:00:00Z", "to": "2026-06-30T23:59:59Z" },
  "questions": { "total": 2450, "unresolved": 120 },
  "inquiryStatus": { "open": 35, "closed": 85 },
  "topQuestions": [ { "question": "返品方法は?", "count": 42 } ],
  "notification": { "failedCount": 3, "bouncedCount": 1 }
}
```

| 項目 | 型 | 説明 |
|----|----|----|
| `period` | object | 集計対象期間 |
| `　└ from` | string (ISO 8601) | 期間の起点 |
| `　└ to` | string (ISO 8601) | 期間の終点 |
| `questions` | object | 質問数の集計 |
| `　└ total` | integer | 期間内の総質問数 |
| `　└ unresolved` | integer | 未解決の質問数 |
| `inquiryStatus` | object | 未解決質問の状況別件数 |
| `　└ open` | integer | 対応中(`open`)の件数 |
| `　└ closed` | integer | 対応済み(`closed`)の件数 |
| `topQuestions` | array\<object\> | 頻出質問の配列 |
| `　└ question` | string | 質問内容 |
| `　└ count` | integer | 当該質問の出現回数 |
| `notification` | object | 通知の配信失敗・バウンス集計(メンバーが `projectId` 指定時は `null`) |
| `　└ failedCount` | integer | 配信失敗件数 |
| `　└ bouncedCount` | integer | バウンス件数 |

`notification.failedCount` / `bouncedCount` は `H_NOTIF_LOGS` の配信失敗・バウンス集計(列定義は [データベース設計.md](../04_database/index.md))。メンバーが `projectId` 指定で呼んだ場合、本ブロックは `null` を返す。

## バリデーション

入力値の検証規則は `## リクエスト` の各パラメータ説明を参照する。主な規則は次のとおり。

- `period`: 任意・`current_month` / `last_30d`。集計の基準タイムゾーンは UTC(設計値。契約はタイムゾーン設定を持たないため、保存値の UTC をそのまま基準とする)。`current_month` は UTC 当月初日 00:00:00 起算。
- `from` / `to`: 任意・ISO 8601。`period` と排他(双方指定時は `from` / `to` を優先)。
- `projectId`: メンバーは必須(未指定は 400 `VALIDATION_ERROR`)。オーナーは任意。

## エラー

| HTTP ステータス | エラーコード            | 内容                          |
|-----------------|-------------------------|-------------------------------|
