## 処理概要

質問数・未解決・要対応状況・通知失敗 / バウンスをまとめて返す。

| 処理 ID | 処理内容 |
|----|----|
| P-01 | 認証・認可(対象プロジェクトへのアクセス権、メンバーは `projectId` 必須)を検証する |
| P-02 | 期間・プロジェクトの質問数・未解決数・要対応状況に加え、日次の質問数推移(`H_QUESTION_LOGS` を日単位 GROUP BY)・要対応(未解決)の新着 N 件(設計値 N=5。`T_INQUIRIES` の `status = 'open'` 新着順)・公開 FAQ 件数(`M_FAQS` の `status = 'published' AND valid = 1`)を集計する(集計の基準タイムゾーンは UTC = 設計値。`current_month` は UTC 当月初日 00:00 起算、`last_month` は UTC 前月初日〜前月末日、`custom` は `from` / `to` の範囲。日次推移の期間は解決後の `from` / `to` に従う)。`period` と `from` / `to` は排他で、双方指定時は `from` / `to` を優先する。集計完了時刻を `lastAggregatedAt` として算出する |
| P-03 | オーナー向けに通知失敗・バウンス件数を集計する(メンバー指定時は `null`) |
| P-04 | 整形して返却する |

## リクエスト

#### 認証ヘッダ

本 API は利用者セッション(Cookie)による認証を必要とする。

| ヘッダ | 要否 | 説明 |
|---|---|---|
| `Cookie: session=<token>` | 必須 | 利用者セッション |

#### Query Parameters

```json
{ "period": "current_month", "projectId": "{id}(メンバーは必須)", "from": "任意(period=custom の範囲指定。custom 指定時は必須)", "to": "任意(period=custom の範囲指定。custom 指定時は必須)" }
```

| 項目 | 型 | 説明 |
|----|----|----|
| `period` | enum | 集計期間。任意。`current_month`(当月)/ `last_month`(前月)/ `custom`(任意期間。`from` / `to` で指定、最大 13 ヶ月)。`custom` 指定時は `from` / `to` が必須。基準タイムゾーンは UTC(設計値。プロジェクトはタイムゾーン設定を持たない)。`current_month` は UTC の当月初日 00:00:00 起算、`last_month` は UTC の前月初日 00:00:00 〜 前月末日 23:59:59 |
| `projectId` | string | 対象プロジェクトの ID。オーナーは任意(未指定で自分が作成した全プロジェクトを集約)、メンバーは必須 |
| `from` | string (ISO 8601) | 集計期間の起点。任意(`period=custom` の範囲指定に使用。`custom` 指定時は必須) |
| `to` | string (ISO 8601) | 集計期間の終点。任意(`period=custom` の範囲指定に使用。`custom` 指定時は必須) |

## レスポンス

**レスポンス(200)**

```json
{
  "period": { "from": "2026-06-01T00:00:00Z", "to": "2026-06-30T23:59:59Z" },
  "questions": { "total": 2450, "unresolved": 120 },
  "inquiryStatus": { "open": 35, "closed": 85 },
  "topQuestions": [ { "question": "返品方法は?", "count": 42 } ],
  "dailyTrend": [ { "date": "2026-06-19", "count": 84 } ],
  "recentInquiries": [ { "inquiryId": "inq_01H...", "question": "料金プランの変更方法について", "createdAt": "2026-06-19T09:09:00Z" } ],
  "publishedFaqCount": 156,
  "lastAggregatedAt": "2026-06-19T09:12:00Z",
  "notification": { "failedCount": 3, "bouncedCount": 1 }
}
```

| 項目 | 型 | 説明 |
|----|----|----|
| `period` | object | 集計対象期間 |
| `　└ from` | string (ISO 8601) | 期間の起点 |
| `　└ to` | string (ISO 8601) | 期間の終点 |
| `questions` | object | 質問数の集計 |
| `　└ total` | integer | 期間内の総質問数 |
| `　└ unresolved` | integer | 未解決の質問数 |
| `inquiryStatus` | object | 未解決質問の状況別件数 |
| `　└ open` | integer | 対応中(`open`)の件数 |
| `　└ closed` | integer | 対応済み(`closed`)の件数 |
| `topQuestions` | array\<object\> | 頻出質問の配列 |
| `　└ question` | string | 質問内容 |
| `　└ count` | integer | 当該質問の出現回数 |
| `dailyTrend` | array\<object\> | 日次の質問数推移(`period` で解決した `from` / `to` に従う。画面は過去 14 日を日次バーで表示) |
| `　└ date` | string (ISO 8601 日付) | 集計日 |
| `　└ count` | integer | 当日の質問数 |
| `recentInquiries` | array\<object\> | 要対応(未解決)の質問を新着順に N 件(設計値 N=5)。`status = 'open'` の新着順 |
| `　└ inquiryId` | string | 遷移先となる未解決質問の識別子 |
| `　└ question` | string | 質問本文 |
| `　└ createdAt` | string (ISO 8601) | 発生日時(画面で『3 分前』等の相対表記に変換) |
| `publishedFaqCount` | integer | 公開状態の FAQ 件数 |
| `lastAggregatedAt` | string (ISO 8601) | 集計が完了した時刻(サーバ算出値)。画面はこの値が現在時刻から 5 分以上前なら『集計遅延』黄表記とする |
| `notification` | object | 通知の配信失敗・バウンス集計(メンバーが `projectId` 指定時は `null`) |
| `　└ failedCount` | integer | 配信失敗件数 |
| `　└ bouncedCount` | integer | バウンス件数 |

`notification.failedCount` / `bouncedCount` は `H_NOTIF_LOGS` の配信失敗・バウンス集計(列定義は [データベース設計.md](../04_database/index.md))。メンバーが `projectId` 指定で呼んだ場合、本ブロックは `null` を返す。

## バリデーション

入力値の検証規則は `## リクエスト` の各パラメータ説明を参照する。主な規則は次のとおり。

- `period`: 任意・`current_month` / `last_month` / `custom`。集計の基準タイムゾーンは UTC(設計値。プロジェクトはタイムゾーン設定を持たないため、保存値の UTC をそのまま基準とする)。`current_month` は UTC 当月初日 00:00:00 起算、`last_month` は UTC 前月初日 00:00:00 〜 前月末日 23:59:59。`period=custom` 指定時は `from` / `to` が必須(未指定は 400 `VALIDATION_ERROR`)。
- `from` / `to`: 任意・ISO 8601(`period=custom` 指定時は必須)。`period` と排他(双方指定時は `from` / `to` を優先)。期間は最大 13 ヶ月。
- `projectId`: メンバーは必須(未指定は 400 `VALIDATION_ERROR`)。オーナーは任意。
