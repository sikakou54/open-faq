# <span id="API-027"></span>API-027: FAQ 一括状態変更

> **このページは FAQ 一括状態変更 API の契約を定義します。** 選択した複数 FAQ の状態を一括で変更する(1 リクエスト上限 50 件)。

## 項目

| 項目 | 内容 |
|---|---|
| API ID | `API-027` |
| API名 | FAQ 一括状態変更 |
| トレーサビリティID | [TR-027](../../00_traceability/index.md#TR-027) |
| エンドポイント | `/faqs/bulk-status` |
| HTTPメソッド | POST |
| 認証 | Cookie + CSRF |
| 認可 | オーナー / 当該プロジェクトのメンバー |

## 処理概要

選択した複数 FAQ の状態を一括で変更する(1 リクエスト上限 50 件)。

| 処理 ID | 処理内容 |
|----|----|
| P-01 | 認証・認可(オーナー / 該当プロジェクトの編集権限)を検証する |
| P-02 | 件数上限(最大 50 件 = RULE-019)・各 ID 形式(ULID)・状態値を検証する(超過 / 不正は 400 `VALIDATION_ERROR`) |
| P-03 | 各 ID を行単位で評価し、対象外(他契約 / 論理削除済み)は当該行のみ失敗として集計する |
| P-04 | 対象 FAQ の状態を一括変更し、`published` 化した行は `publishedAt` を記録する |
| P-05 | 成功件数と失敗明細を返す |

## リクエスト

#### 認証ヘッダ

本 API は利用者セッション(Cookie)と CSRF トークンを必要とする状態変更系である。

| ヘッダ | 要否 | 説明 |
|---|---|---|
| `Cookie: session=<token>` | 必須 | 利用者セッション |
| `X-CSRF-Token: <token>` | 必須 | CSRF 対策(状態変更系) |
| `Idempotency-Key: <ULID>` | 任意 | 冪等性(書き込み再送の重複防止) |

#### Query Parameters

```json
{}
```

#### Path Parameters

```json
{}
```

#### Request Body

```json
{ "ids": ["faq_...", "faq_..."], "status": "published" }
```

| 項目 | 型 | 説明 |
|----|----|----|
| `ids` | array\<string\> | 状態を変更する FAQ の ID リスト(必須・各要素は ULID・1 リクエスト最大 50 件 = RULE-019)。50 件超過は 400 `VALIDATION_ERROR` |
| `status` | enum | 変更後の状態(必須・`draft` / `published` / `hidden`) |

## レスポンス

**レスポンス(200)**

```json
{ "succeeded": 48, "failed": [ { "id": "faq_...", "code": "NOT_FOUND" } ] }
```

| 項目        | 型              | 説明                                   |
|-------------|-----------------|----------------------------------------|
| `succeeded` | integer         | 状態変更に成功した件数                 |
| `failed`    | array\<object\> | 失敗した行の明細                       |
| `　└ id`    | string          | 失敗した FAQ の ID                     |
| `　└ code`  | string          | 失敗理由のエラーコード(例 `NOT_FOUND`) |

## バリデーション

入力値の検証規則は `## リクエスト` の各パラメータ説明を参照する。主な規則は次のとおり。

- `ids`: 必須・`array<string>`・各要素は ULID・最大 50 件(RULE-019)。50 件超過は 400 `VALIDATION_ERROR`。
- `status`: 必須・`draft` / `published` / `hidden` のいずれか。

## エラー

| HTTP ステータス | エラーコード            | 内容                         |
|-----------------|-------------------------|------------------------------|
| 400 | `VALIDATION_ERROR` → [ERR-001](../../05_errors/ERR-001.md#ERR-001) | 50 件超過(RULE-019) / `ids` 形式不正 / 不正な `status` |
| 403 | `PROJECT_ACCESS_DENIED` → [ERR-021](../../05_errors/ERR-021.md#ERR-021) | 当該プロジェクトへの権限なし |
