# <span id="API-019"></span>API-019: ウィジェット鍵ローテーション

> **このページは ウィジェット鍵ローテーション API の契約を定義します。** 公開キーを再発行する(旧キーは 24 時間猶予で失効)。

## 項目

| 項目 | 内容 |
|---|---|
| API ID | `API-019` |
| API名 | ウィジェット鍵ローテーション |
| トレーサビリティID | [TR-040](../../00_traceability/index.md#TR-040) |
| エンドポイント | `/projects/{id}/widget-key/rotate` |
| HTTPメソッド | POST |
| 認証 | 再認証必須 |
| 認可 | オーナー専有 |

## 処理概要

公開キーを再発行する(旧キーは 24 時間猶予で失効)。

| 処理 ID | 処理内容                                       |
|---------|------------------------------------------------|
| P-01    | 認証・認可(オーナー専有)と再認証を検証する     |
| P-02    | 新しいウィジェット公開キーを発行して有効化する |
| P-03    | 旧キーを 24 時間猶予で失効予約する             |
| P-04    | 新キーと失効予告を返す                         |

## リクエスト

#### 認証ヘッダ

本 API は利用者セッション(Cookie)と CSRF トークンを必要とする状態変更系である。

| ヘッダ | 要否 | 説明 |
|---|---|---|
| `Cookie: session=<token>` | 必須 | 利用者セッション |
| `X-CSRF-Token: <token>` | 必須 | CSRF 対策(状態変更系) |
| `Idempotency-Key: <ULID>` | 任意 | 冪等性(書き込み再送の重複防止) |

> 本操作は再認証必須。直近の再認証([API-005](API-005.md#API-005))で得た再認証状態を伴う必要がある。

#### Query Parameters

```json
{}
```

#### Path Parameters

```json
{ "id": "プロジェクト ID" }
```

| 項目 | 型 | 説明 |
|----|----|----|
| `id` | string | 鍵をローテーションする対象プロジェクトの ID(必須・パス内で指定) |

#### Request Body

```json
{}
```

公開キーは無期限のため有効期限指定パラメータは持たない。

## レスポンス

**レスポンス(201)**

```json
{ "newKey": "pk_live_xyz789...", "deprecationWarning": "..." }
```

| 項目                 | 型     | 説明                                   |
|----------------------|--------|----------------------------------------|
| `newKey`             | string | 再発行したウィジェット公開キー         |
| `deprecationWarning` | string | 旧キーの 24 時間猶予失効予告メッセージ |

(`deprecationWarning` は旧キーの 24 時間猶予失効予告)
