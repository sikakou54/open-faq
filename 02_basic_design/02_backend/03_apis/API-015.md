# <span id="API-015"></span>API-015: プロフィール・セキュリティ設定更新

> **このページは プロフィール・セキュリティ設定更新 API の仕様を定義します。** 認証済みのアカウント保有者が設定画面から自身の氏名(プロフィール名)・メールアドレス・パスワードを更新する。メールアドレス・パスワードの変更時は再認証トークンを要求する。メールアドレス変更時は新アドレスへ確認メールを送信する。

## 項目

| 項目 | 内容 |
|---|---|
| API ID | `API-015` |
| API名 | プロフィール・セキュリティ設定更新 |
| トレーサビリティID | [TR-022](../../00_traceability/index.md#TR-022) |
| エンドポイント | `/me/settings` |
| HTTPメソッド | PATCH |
| 認証 | 要(Cookie + CSRF) |
| 認可 | 認証済み |

## 処理概要

認証済みのアカウント保有者が設定画面から自身の氏名(プロフィール名)・メールアドレス・パスワードを更新する。氏名・メールアドレスはアカウントの `M_USER` に保持し、メールアドレス・パスワード変更時は再認証トークンを検証する。メールアドレス変更時は新アドレスへ確認メールを送信する。

| 処理 ID | 処理内容                                                       |
|---------|----------------------------------------------------------------|
| P-01    | セッションから認証済みのアカウント保有者を特定する                |
| P-02    | 入力値(氏名の文字数・メールアドレス形式・パスワード強度等)を検証する |
| P-03    | メールアドレス・パスワード変更時は再認証トークン(リクエストボディ `reauthToken`)を検証する |
| P-04    | 氏名(`displayName`→`M_USER.name`)・メールアドレス(`email`→`M_USER.email`)・パスワードハッシュを `M_USER` に更新する |
| P-05    | メールアドレス変更時はメール確認トークンを発行し新アドレスへ確認メールを送信する |
| P-06    | 更新後の値を返却する                                           |

## リクエスト

#### 認証ヘッダ

本 API は利用者セッション(Cookie)と CSRF トークンを必要とする状態変更系である。

| ヘッダ | 要否 | 説明 |
|---|---|---|
| `Cookie: session=<token>` | 必須 | 利用者セッション |
| `X-CSRF-Token: <token>` | 必須 | CSRF 対策(状態変更系) |
| `Idempotency-Key: <ULID>` | 任意 | 冪等性(書き込み再送の重複防止) |

#### Query Parameters

```json
{}
```

#### Path Parameters

```json
{}
```

#### Request Body

```json
{
  "displayName": "山田 太郎",
  "email": "new@example.com",
  "password": "P@ssw0rd-new",
  "passwordConfirm": "P@ssw0rd-new",
  "reauthToken": "reauth_..."
}
```

| 項目               | 型     | 説明                                                       |
|--------------------|--------|------------------------------------------------------------|
| `displayName`      | string | 新しい氏名(プロフィール名。任意・1〜100 文字)。`M_USER.name` に保存する |
| `email`            | string | 新しいメールアドレス(任意・変更時は `reauthToken` 必須)。`M_USER.email` に保存する |
| `password`         | string | 新しいパスワード(任意・変更時は `passwordConfirm` と `reauthToken` 必須。12 文字以上、英大小文字 / 数字 / 記号のうち 3 種類以上) |
| `passwordConfirm`  | string | 新しいパスワード確認用(パスワード変更時必須・`password` と一致) |
| `reauthToken`      | string | 再認証トークン(メールアドレス・パスワード変更時必須・API-005 で取得) |

## レスポンス

**レスポンス(200)**

```json
{
  "displayName": "山田 太郎",
  "email": "owner@example.com",
  "emailVerificationPending": false
}
```

| 項目                       | 型      | 説明                                                      |
|----------------------------|---------|-----------------------------------------------------------|
| `displayName`              | string  | 更新後の氏名(`M_USER.name`)                              |
| `email`                    | string  | 現在のメールアドレス(`M_USER.email`)                     |
| `emailVerificationPending` | boolean | 確認メールを送信して検証待ちの場合 `true`(メールアドレス変更時) |

## バリデーション

入力値の検証規則は `## リクエスト` の各パラメータ説明を参照する。

## エラー

| HTTP ステータス | エラーコード                   | 内容                           |
|-----------------|--------------------------------|--------------------------------|
| 400 | `VALIDATION_ERROR` → [ERR-001](../../05_errors/ERR-001.md#ERR-001) | 氏名の文字数超過 / パスワード強度不足・不一致等の入力検証エラー |
| 401 | `REAUTH_REQUIRED` → [ERR-013](../../05_errors/ERR-013.md#ERR-013) | 再認証トークンが無効または未提示(メール・パスワード変更時) |
| 409 | `EMAIL_ALREADY_USED` → [ERR-014](../../05_errors/ERR-014.md#ERR-014) | 入力メールアドレスが既に使用中 |

## 備考

氏名・メールアドレス・パスワードは全ユーザー共通の `M_USER` に保持する(`displayName`→`M_USER.name` / `email`→`M_USER.email`)。オーナー / メンバーの区別なく本人の `M_USER` 行を更新する。組織名は廃止しており、アカウント保有者の氏名(プロフィール名)で代替する。
