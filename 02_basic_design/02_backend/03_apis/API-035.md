| API名 | エンドポイント | HTTPメソッド |
|----|----|----|
| 未解決質問詳細・状況切替 | `/inquiries/{id}` | GET / PATCH |

## 処理概要

詳細取得と状況の手動切替(担当者概念は持たない)。

| 処理 ID | 処理内容 |
|----|----|
| P-01 | 認証・認可(PATCH は CSRF も)を検証する |
| P-02 | 対象未解決質問を特定する |
| P-03 | GET は詳細を取得する。元質問ログ(`question_log_id` → H_QUESTION_LOGS)を結合し、応答ログ(エンドユーザー質問・AI 応答・投稿時刻)・信頼度・未解決理由を付帯する。チャネルは現 MVP では定数(ウィジェット)とし、付随するプロジェクト名(M_PROJECTS.name)を併せて取得する |
| P-04 | PATCH は状況(`open` ↔ `closed`)を更新する(担当者概念・履歴永続化なし) |
| P-05 | 結果を返却する |

## リクエスト

#### 認証ヘッダ

本 API は利用者セッション(Cookie)による認証を必要とし、更新(PATCH)時は CSRF トークンを併用する。

| ヘッダ | 要否 | 説明 |
|---|---|---|
| `Cookie: session=<token>` | 必須 | 利用者セッション |
| `X-CSRF-Token: <token>` | PATCH 時必須 | CSRF 対策(状況切替の状態変更時) |
| `Idempotency-Key: <ULID>` | 任意 | 冪等性(PATCH 再送の重複防止) |

#### Query Parameters

```json
{}
```

#### Path Parameters

```json
{ "id": "未解決質問 ID" }
```

| 項目 | 型     | 説明                        |
|------|--------|-----------------------------|
| `id` | string | 対象の未解決質問の ID。必須 |

#### Request Body

```json
{
  "status": "open" | "closed"
}
```

| 項目 | 型 | 説明 |
|----|----|----|
| `status` | enum | 切り替える状況。`open` / `closed`。PATCH 時のみ。任意(指定したものだけ更新) |

Request Body は PATCH 時のもの。各フィールドはオプション(指定したものだけ更新)。状況は `open` ↔ `closed` の双方向遷移を許容(再オープン制限なし)。FAQ 下書き保存・FAQ 公開は本フィールドを変更しない(連動ロジックなし)。状態変更履歴の永続化は行わない。

## レスポンス

**レスポンス(200)**

GET は未解決質問の詳細、PATCH は更新後の状況。

```json
{
  "id": "未解決質問 ID",
  "inquiryCode": "INQ-...",
  "question": "ウィジェット利用者の質問内容",
  "status": "open",
  "projectId": "プロジェクト ID",
  "channel": "widget",
  "channelLabel": "ウィジェット",
  "projectName": "サポートサイト",
  "confidence": 0.38,
  "reason": "no_faq_match",
  "reasonLabel": "該当 FAQ なし",
  "responseLog": [
    { "role": "enduser", "message": "利用者の質問本文", "postedAt": "2026-06-19T09:09:00Z" },
    { "role": "ai", "message": "AI の応答本文", "postedAt": "2026-06-19T09:09:03Z", "confidence": 0.38 }
  ],
  "createdAt": "2026-06-19T09:12:00Z",
  "updatedAt": "2026-06-19T09:30:00Z"
}
```

| 項目 | 型 | 説明 |
|----|----|----|
| `id` | string | 未解決質問の ID |
| `inquiryCode` | string | 未解決質問の管理用識別子 |
| `question` | string | ウィジェット利用者が送信した質問内容 |
| `status` | enum | 状況。`open` / `closed`(PATCH 時は更新後の値) |
| `projectId` | string | 当該未解決質問が属するプロジェクトの ID |
| `channel` | string | 受信チャネル種別。現 MVP は単一チャネル(ウィジェット)前提のため定数 `widget` 相当 |
| `channelLabel` | string | チャネルの表示名(例 `ウィジェット`) |
| `projectName` | string | チャネルに付随表示するプロジェクト名(例 `サポートサイト`)。M_PROJECTS.name 由来 |
| `confidence` | number \| null | AI 応答の信頼度(0.0〜1.0)。算出不能・未記録時は null。H_QUESTION_LOGS.confidence_score 由来 |
| `reason` | enum \| null | 未解決理由コード。`answered` / `no_faq_match` / `low_confidence` / `pii_detected` / `ai_unavailable` / `rate_limited`。H_QUESTION_LOGS.result_reason_code 由来。未記録時は null |
| `reasonLabel` | string \| null | 未解決理由の表示用ラベル(例 `no_faq_match`=該当 FAQ なし)。コード→ラベル変換結果 |
| `responseLog` | array<object> | 応答ログ(エンドユーザー質問・AI 応答の時系列)。元質問ログ(H_QUESTION_LOGS)から取得 |
| `└ role` | enum | 発話者。`enduser`(ウィジェット利用者発話)/ `ai`(AI 発話) |
| `└ message` | string | 発話本文。`enduser` は H_QUESTION_LOGS.user_question、`ai` は H_QUESTION_LOGS.ai_response 由来 |
| `└ postedAt` | string (ISO 8601) | 投稿時刻。H_QUESTION_LOGS.created_at 由来 |
| `└ confidence` | number \| null | `ai` 要素の信頼度(0.0〜1.0、null 許容)。H_QUESTION_LOGS.confidence_score 由来 |
| `createdAt` | string (ISO 8601) | 未解決質問の登録日時 |
| `updatedAt` | string (ISO 8601) | 状況の最終更新日時 |
