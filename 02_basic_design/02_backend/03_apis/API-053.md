# <span id="API-053"></span>API-053: プライバシーポリシー 最新版取得

> **このページは プライバシーポリシー 最新版取得 API の契約を定義します。** プライバシーポリシーの最新版を返す。

## 項目

| 項目 | 内容 |
|---|---|
| API ID | `API-053` |
| API名 | プライバシーポリシー 最新版取得 |
| トレーサビリティID | [TR-012](../../00_traceability/index.md#TR-012) ・ [TR-013](../../00_traceability/index.md#TR-013) |
| エンドポイント | `/privacy/current` |
| HTTPメソッド | GET |
| 認証 | 不要 |
| 認可 | —(公開) |

## 処理概要

プライバシーポリシーの最新版を返す。

| 処理 ID | 処理内容                                   |
|---------|--------------------------------------------|
| P-01    | 最新版のプライバシーポリシーを取得する     |
| P-02    | 本文・施行日・主な変更点を整形して返却する |
| P-03    | 本文 HTML の見出しから目次(章ナビ)用の見出し構造を構成して返却する(`M_TERMS_VER.body_html`) |
| P-04    | 認証済みリクエスト時のみ、利用者の最新版同意状態(同意有無・同意日・再同意要否)を `T_TERMS_AGREE` × `M_TERMS_VER`(最新版)から照合して返却する(未認証時は省略) |

## レスポンス

**レスポンス(200)**

```json
{ "docType": "privacy_policy", "version": "...", "effectiveDate": "...", "bodyHtml": "...", "diffSummary": "...", "sections": [ { "anchorId": "sec-1", "title": "第 1 条 収集する情報", "order": 1 } ], "consentStatus": { "agreed": true, "agreedAt": "...", "requiresReconsent": false } }
```

| 項目 | 型 | 説明 |
|----|----|----|
| `docType` | string | 文書種別(プライバシーポリシーは `privacy_policy` 固定) |
| `version` | string | 返却したプライバシーポリシーのバージョン |
| `effectiveDate` | string (ISO 8601) | 当該バージョンの施行日 |
| `bodyHtml` | string | プライバシーポリシーの本文 HTML |
| `diffSummary` | string | 主な変更点(`M_TERMS_VER.diff_summary`。NULL の場合は省略) |
| `sections` | array&lt;object&gt; | 目次(章ナビ)用の見出し構造。本文 HTML の見出しから構成する(SCR-025 の左 aside「目次」のジャンプリンクに用いる) |
| `└ anchorId` | string | 本文 HTML 内の見出しに対応するアンカー(`id` 属性)。ジャンプ先 |
| `└ title` | string | 見出し文言(例「第 1 条 収集する情報」) |
| `└ order` | integer | 文書内の見出し出現順 |
| `consentStatus` | object | 認証済みリクエスト時のみ設定する利用者の同意状態。未認証(認証ヘッダ無し)時は本オブジェクトを省略する |
| `└ agreed` | boolean | 利用者が最新版に同意済みか |
| `└ agreedAt` | string (ISO 8601) \| null | 同意日(`T_TERMS_AGREE.agreed_at`)。未同意のときは null |
| `└ requiresReconsent` | boolean | 再同意要否(現行最新版に対する同意が無い場合 true) |

(最新版のみ。`diffSummary` は `M_TERMS_VER.diff_summary`。SCR-020 の「主な変更点」表示に用いる。NULL の場合は省略)

`sections` は `M_TERMS_VER.body_html` の見出しから構成する(DB カラム追加は不要)。`consentStatus` は認証済みリクエスト時のみ返し、`T_TERMS_AGREE`(`user_id` × `doc_type = 'privacy_policy'` × `terms_version`)と `M_TERMS_VER`(最新 `version`)を照合して算出する(DB カラム追加は不要)。SCR-025 の目次(章ナビ)・同意状態バッジ表示に用いる。

## 利用テーブル

本 API が参照・更新するテーブルと CRUD 区分。

| テーブル | CRUD |
|---|---|
| [`M_TERMS_VER`](../04_database/TBL-012.md#TBL-012) | `-R--` |
| [`T_TERMS_AGREE`](../04_database/TBL-024.md#TBL-024) | `-R--` |

`M_TERMS_VER` は最新版の本文・施行日・差分サマリ・目次(`sections`)の供給元。`T_TERMS_AGREE` は認証済みリクエスト時の同意状態(`consentStatus`)照合に用いる(未認証時は参照しない)。いずれも DB カラム追加は不要。
