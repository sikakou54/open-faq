# <span id="API-021"></span>API-021: メンバー招待

> **このページは メンバー招待 API の仕様を定義します。** 該当プロジェクト 1 件に対し、登録済みユーザーを 1 名招待する。

## 項目

| 項目 | 内容 |
|---|---|
| API ID | `API-021` |
| API名 | メンバー招待 |
| トレーサビリティID | [TR-019](../../00_traceability/index.md#TR-019) |
| エンドポイント | `/projects/{id}/members` |
| HTTPメソッド | POST |
| 認証 | Cookie + CSRF + 再認証必須 |
| 認可 | 対象プロジェクトのオーナー(`M_PROJECTS.owner_user_id` 一致)/ 当該プロジェクトのメンバー |

## 処理概要

該当プロジェクト 1 件に対し、メールで指定した登録済みユーザーを 1 名招待する。指定メールが未登録(アカウントなし)の場合は招待せず拒否し、先に独立サインアップ([API-001](API-001.md#API-001))での登録を促す。本 API はアカウントの新規作成を行わない。

| 処理 ID | 処理内容 |
|----|----|
| P-00 | 再認証トークン(リクエストボディ `reauthToken`)を検証する |
| P-01 | 認証・認可(対象プロジェクトのオーナー(`M_PROJECTS.owner_user_id` 一致)/ 当該プロジェクトのメンバー)と再認証を検証する |
| P-02 | 入力値(メール形式)を検証し、指定メールに一致する登録済みユーザー(`M_USER`)を検索する。一致が無ければ招待先未登録(ERR-035 相当)で拒否し、先にアカウント登録([API-001](API-001.md#API-001))を促す(本 API はアカウント新規作成を行わない) |
| P-03 | 当該プロジェクトへの既存割当(重複)を判定する。重複があれば拒否する |
| P-04 | 一致した登録済みユーザーに対し、当該プロジェクトの割当(`M_PRJ_USERS`)を作成し、招待トークンを発行して招待メールを送信する(招待先は[招待プレビュー](API-007.md#API-007)から参加先プロジェクトを確認できる) |
| P-05 | 招待結果を返す |

## リクエスト

#### 認証ヘッダ

本 API は利用者セッション(Cookie)と CSRF トークンを必要とする状態変更系である。

| ヘッダ | 要否 | 説明 |
|---|---|---|
| `Cookie: session=<token>` | 必須 | 利用者セッション |
| `X-CSRF-Token: <token>` | 必須 | CSRF 対策(状態変更系) |
| `Idempotency-Key: <ULID>` | 任意 | 冪等性(書き込み再送の重複防止) |

> 本操作は再認証必須。直近の再認証([API-005](API-005.md#API-005))で得た再認証状態を伴う必要がある。

#### Query Parameters

```json
{}
```

#### Path Parameters

```json
{ "id": "プロジェクト ID" }
```

| 項目 | 型     | 説明                                        |
|------|--------|---------------------------------------------|
| `id` | string | 招待先プロジェクトの ID(必須・パス内で指定) |

#### Request Body

```json
{
  "reauthToken": "reauth_...",
  "email": "newmember@..."
}
```

| 項目    | 型     | 説明                       |
|---------|--------|----------------------------|
| `reauthToken` | string | 再認証トークン(必須・API-005 で取得) |
| `email` | string | 招待先メールアドレス(必須) |

## レスポンス

**レスポンス(201)**

```json
{ "userId": "01HZ...", "status": "active", "expiresAt": "..." }
```

| 項目 | 型 | 説明 |
|----|----|----|
| `userId` | string | 招待した登録済みユーザーの ID |
| `status` | enum | アカウント状態(招待先は登録済みのため常に `active`) |
| `expiresAt` | string (ISO 8601) | 発行した招待トークンの有効期限 |

## バリデーション

入力値の検証規則は `## リクエスト` の各パラメータ説明を参照する。

## エラー

| HTTP ステータス | エラーコード                | 内容                           |
|-----------------|-----------------------------|--------------------------------|
| 401 | `REAUTH_REQUIRED` → [ERR-013](../../05_errors/ERR-013.md#ERR-013) | 再認証トークンが無効または未提示 |
| 404 | `INVITE_TARGET_NOT_REGISTERED` → [ERR-035](../../05_errors/ERR-035.md#ERR-035) | 招待先メールが未登録(先にアカウント登録が必要) |
| 409 | `ALREADY_EXISTS_IN_PROJECT` → [ERR-018](../../05_errors/ERR-018.md#ERR-018) | 既に該当プロジェクトに割当あり |
| 400 | `VALIDATION_ERROR` → [ERR-001](../../05_errors/ERR-001.md#ERR-001) | 入力値の検証エラー |
| 403 | `PROJECT_ACCESS_DENIED` → [ERR-019](../../05_errors/ERR-019.md#ERR-019) | 当該プロジェクトへの権限なし |
