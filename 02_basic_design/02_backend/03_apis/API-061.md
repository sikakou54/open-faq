# <span id="API-061"></span>API-061: 通知配信状態サマリ

> **このページは 通知配信状態サマリ API の仕様を定義します。** 当該プロジェクトの通知配信実績を集計し、配信状態の内訳と失敗・バウンス件数を返す。

| ID | 業務ユースケースID | イベント(画面ID EVT-NN) | テーブルID |
|----|----|----|----|
| API-061 | [UC-077](../../../01_requirements/04_business_usecases/UC-077.md#UC-077) | — | [TBL-007](../04_database/TBL-007.md#TBL-007) ・ [TBL-026](../04_database/TBL-026.md#TBL-026) |

| API名 | エンドポイント | HTTPメソッド |
|----|----|----|
| 通知配信状態サマリ | `/notifications/delivery-status` | GET |

## 処理概要

当該プロジェクトの通知配信実績を集計し、配信状態の内訳(送信待ち・送信済み・配信済み・失敗・バウンス・苦情・送信停止)と失敗件数・バウンス件数を返す。

| 処理 ID | 処理内容 |
|----|----|
| P-01 | リクエスト元が当該プロジェクトに割当のあるオーナー / メンバーかを判定する(不一致は 403) |
| P-02 | 当該プロジェクトの通知ログを集計対象期間で集計する |
| P-03 | 送信停止(抑制)対象を加味し、配信状態を通知種別ごとに区分集計する |
| P-04 | 件数サマリー(送信済み・配信済み・失敗・バウンス)と内訳を返す |

## リクエスト

#### 認証ヘッダ

本 API は利用者セッション(Cookie)による認証を必要とする。

| ヘッダ | 要否 | 説明 |
|---|---|---|
| `Cookie: session=<token>` | 必須 | 利用者セッション |

#### Query Parameters

```json
{
  "projectId": "対象プロジェクト ID",
  "period": "current_month"
}
```

| 項目 | 型 | 説明 |
|----|----|----|
| `projectId` | string | 対象プロジェクト ID(必須) |
| `period` | enum | 集計対象期間(既定 `current_month`) |

#### Path Parameters

```json
{}
```

## レスポンス

本 API の戻り値は本文の型定義を参照する。件数サマリーと通知種別別の配信状態内訳を返す。

## バリデーション

入力値の検証規則は `## リクエスト` の各パラメータ説明を参照する。

## エラー

| HTTP ステータス | エラーコード | 内容 |
|-----------------|---------------------|--------------------------------|
| 403 | `PERMISSION_DENIED` → [ERR-030](../../05_errors/ERR-030.md#ERR-030) | 当該プロジェクトに割当のないユーザーのアクセス |
| 404 | `PROJECT_NOT_FOUND` → [ERR-011](../../05_errors/ERR-011.md#ERR-011) | 対象プロジェクトが存在しない |
