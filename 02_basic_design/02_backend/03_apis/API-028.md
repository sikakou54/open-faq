# <span id="API-028"></span>API-028: FAQ CSV インポート

> **このページは FAQ CSV インポート API の仕様を定義します。** CSV による FAQ の一括取込(新規 / 上書き)。

| ID | 業務ユースケースID | イベント(画面ID EVT-NN) | テーブルID |
|----|----|----|----|
| API-028 | [UC-027](../../../01_requirements/04_business_usecases/UC-027.md#UC-027) ・ [UC-046](../../../01_requirements/04_business_usecases/UC-046.md#UC-046) | SCR-010 EVT-04 | [TBL-006](../04_database/TBL-006.md#TBL-006) ・ [TBL-033](../04_database/TBL-033.md#TBL-033) |

| API名 | エンドポイント | HTTPメソッド |
|----|----|----|
| FAQ CSV インポート | `/faqs/import` | POST |

## 処理概要

CSV による FAQ の一括取込(新規 / 上書き)。

| 処理 ID | 処理内容 |
|----|----|
| P-01 | 認証・認可(オーナー / 該当プロジェクトの編集権限)を検証する |
| P-02 | 受理形式(CSV・UTF-8・ヘッダ行・最大 1000 件・最大 10 MB = 設計値)と各行の文字数(質問 500 文字 / 回答 5,000 文字 = RULE-011)を検証する(形式不正は 415 / 400 `CSV_INVALID`) |
| P-03 | 取込ジョブを受け付け、非同期処理として開始する |
| P-04 | 各行の `FAQ ID` で 新規 / 上書き / 当該プロジェクトに存在しない ID(行失敗)を判定し取り込む |
| P-05 | ジョブ ID と受付状態を返す |

## リクエスト

#### 認証ヘッダ

本 API は利用者セッション(Cookie)と CSRF トークンを必要とする状態変更系である。

| ヘッダ | 要否 | 説明 |
|---|---|---|
| `Cookie: session=<token>` | 必須 | 利用者セッション |
| `X-CSRF-Token: <token>` | 必須 | CSRF 対策(状態変更系) |
| `Idempotency-Key: <ULID>` | 任意 | 冪等性(書き込み再送の重複防止) |

#### Query Parameters

```json
{}
```

#### Path Parameters

```json
{}
```

#### Request Body

```
multipart/form-data { file: <CSV> }
```

| 項目 | 型 | 説明 |
|----|----|----|
| `file` | string | 取込対象の CSV ファイル(必須・`multipart/form-data`、UTF-8、ヘッダ行必須、1 ファイル最大 1000 件・最大 10 MB = 設計値) |

受理形式は **CSV のみ**(UTF-8、BOM 許容、ヘッダ行必須、1 ファイル最大 1000 件、最大 10 MB = 設計値)。**JSON 形式は受理しない**(不正形式・サイズ超過は 415 / 400、E-INPUT-CSV-INVALID)。CSV 列構成は `FAQ ID, 質問, 回答, カテゴリ`。CSV のエスケープは **RFC 4180 準拠**とする(改行・カンマ・ダブルクォートを含む値はダブルクォートで囲み、値中のダブルクォートは 2 連続(`""`)でエスケープする)。RFC 4180 に従わず引用・エスケープが破綻した CSV は形式不正(`CSV_INVALID`)として扱う。各行の文字数は **質問 500 文字以内 / 回答 5,000 文字以内**(RULE-011)とし、超過行は形式不正(`CSV_INVALID`)として扱う。各行の `FAQ ID` で **空欄 = 新規(`status=draft`)/ 既存 ID 一致 = 上書き(状態維持)/ 当該プロジェクトに存在しない ID = 当該行を失敗**(行単位エラー、E-INPUT-CSV-FAQID-NOTFOUND)と判定する。`status` 列は受理しない。

## レスポンス

**レスポンス(202)**

```json
{ "jobId": "job_...", "status": "processing" }
```

| 項目     | 型     | 説明                           |
|----------|--------|--------------------------------|
| `jobId`  | string | 受け付けた取込ジョブの ID      |
| `status` | enum   | ジョブの受付状態(`processing`) |

取込状況は [API-068](API-068.md#API-068) `GET /faqs/import/{jobId}` でポーリングする。

## バリデーション

入力値の検証規則は `## リクエスト` の各パラメータ説明を参照する。主な規則は次のとおり。

- `file`: 必須・CSV(UTF-8・ヘッダ行必須)。1 ファイル最大 1000 件・最大 10 MB(設計値)。
- 各行の文字数: 質問 500 文字以内 / 回答 5,000 文字以内(RULE-011)。
- 形式不正(CSV 以外 / サイズ超過 / 文字数超過)は 415 / 400 `CSV_INVALID`(ERR-024)。

## エラー

| HTTP ステータス | エラーコード               | 内容                        |
|-----------------|----------------------------|-----------------------------|
| 415 / 400 | E-INPUT-CSV-INVALID → [ERR-024](../../05_errors/ERR-024.md#ERR-024) | CSV 以外 / 不正形式 / サイズ超過(最大 10 MB = 設計値) / 文字数超過(質問 500・回答 5,000 = RULE-011) |
| (行単位) | E-INPUT-CSV-FAQID-NOTFOUND → [ERR-025](../../05_errors/ERR-025.md#ERR-025) | 当該プロジェクトに存在しない FAQ ID |
