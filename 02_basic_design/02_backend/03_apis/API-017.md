# <span id="API-017"></span>API-017: プロジェクト新規作成

> **このページは プロジェクト新規作成 API の契約を定義します。** プロジェクトを作成し公開キーを発行する(オーナーは isOwner で当該プロジェクトの全権を持つ)。

## 項目

| 項目 | 内容 |
|---|---|
| API ID | `API-017` |
| API名 | プロジェクト新規作成 |
| トレーサビリティID | [TR-015](../../00_traceability/index.md#TR-015) |
| エンドポイント | `/projects` |
| HTTPメソッド | POST |
| 認証 | Cookie + CSRF |
| 認可 | **オーナー専有**(プロジェクト設定全般を含めオーナーのみ実行可) |

## 処理概要

プロジェクトを作成し公開キーを発行する(オーナーは isOwner で当該プロジェクトの全権を持つ)。

| 処理 ID | 処理内容                                                         |
|---------|------------------------------------------------------------------|
| P-01    | 認証・認可(オーナー専有)を検証する                               |
| P-02    | 入力値(プロジェクト名・許可ドメイン)を検証し、名称重複を判定する |
| P-03    | プロジェクトを作成し(連絡先メール `contactEmail` を `M_PROJECTS.contact_email` に保存)ウィジェット公開キーを発行する。連絡先メールは未確認(`contact_verified_at=NULL`)で保存し、確認は既存の連絡先メール確認フロー(API-009 / API-011)に従う |
| P-04    | 作成時にオーナーを当該プロジェクトのメンバーとして `M_PRJ_USERS`(`valid=1`)に登録する(一覧表示・担当割当・通知宛先の網羅用。認可権威は引き続き isOwner) |
| P-05    | 作成結果(ID・公開キー)を返す                                     |

## リクエスト

#### 認証ヘッダ

本 API は利用者セッション(Cookie)と CSRF トークンを必要とする状態変更系である。

| ヘッダ | 要否 | 説明 |
|---|---|---|
| `Cookie: session=<token>` | 必須 | 利用者セッション |
| `X-CSRF-Token: <token>` | 必須 | CSRF 対策(状態変更系) |
| `Idempotency-Key: <ULID>` | 任意 | 冪等性(書き込み再送の重複防止) |

#### Query Parameters

```json
{}
```

#### Path Parameters

```json
{}
```

#### Request Body

```json
{
  "name": "新規プロジェクト",
  "description": "説明文",
  "allowedDomains": ["example.com"],
  "contactEmail": "support@example.com"
}
```

| 項目             | 型              | 説明                                   |
|------------------|-----------------|----------------------------------------|
| `name`           | string          | プロジェクト名(必須・1〜100 文字)       |
| `description`    | string          | プロジェクトの説明文(任意)             |
| `allowedDomains` | array\<string\> | ウィジェット許可ドメインのリスト(必須)。各要素は完全一致ドメイン(例 `example.com`)または `*.example.com` 形式のワイルドカードサブドメイン。IP アドレス・プロトコル(`https://` 等)・ポート・パスは指定不可。最大 20 件(設計値) |
| `contactEmail`   | string | プロジェクト連絡先メール(必須・メール形式)。`M_PROJECTS.contact_email` に保存する |

オーナー固定化(FR-038 / FR-017):

- リクエストにメンバー指定フィールドは持たない。オーナーは `isOwner` bypass で当該プロジェクトの全権を持つため、メンバー指定は不要。
- サーバー側処理: プロジェクト作成は `M_PROJECTS` 行と公開キーを作成し、加えてオーナーを当該プロジェクトのメンバーとして `M_PRJ_USERS`(`valid=1`)へ自動登録する(一覧表示・担当割当・通知宛先の網羅用。専有判定には用いず、認可権威は引き続き `isOwner=true` bypass)。critical 通知の宛先解決はオーナー(`M_CONTRACT`)+ 当該スコープの有効メンバー(`M_PRJ_USERS.valid=1`)を両マスタから合算し、正規化メールで重複排除して行う([権限設計](../../04_permissions/index.md) 正本)。
- 他のメンバーをプロジェクトへ招待する操作は、プロジェクト作成後に当該プロジェクトの SCR-013 / SCR-014(プロジェクト WS)から `POST /projects/{id}/members`(API-011)経由で行う。

## レスポンス

**レスポンス(201)**

```json
{
  "id": "01HZ...",
  "widgetKey": "pk_live_abc123..."
}
```

| 項目        | 型     | 説明                         |
|-------------|--------|------------------------------|
| `id`        | string | 作成したプロジェクトの ID    |
| `widgetKey` | string | 発行したウィジェット公開キー |

## バリデーション

入力値の検証規則は `## リクエスト` の各パラメータ説明を参照する。主な規則は次のとおり。

- `name`: 必須・1〜100 文字。
- `allowedDomains`: 必須・`array<string>`・最大 20 件(設計値)。各要素は完全一致ドメインまたは `*.example.com` 形式のワイルドカードサブドメインのみ。IP アドレス・プロトコル・ポート・パスを含む値は不正(400)。
- `contactEmail`: 必須・メール形式。

## エラー

| HTTP ステータス | エラーコード         | 内容                               |
|-----------------|----------------------|------------------------------------|
| 400 | `VALIDATION_ERROR` → [ERR-001](../../05_errors/ERR-001.md#ERR-001) | `name` / `allowedDomains` 必須違反・`name` 文字数超過・許可ドメイン形式不正・許可ドメイン 20 件超過 |
| 409 | `DUPLICATE_NAME` → [ERR-018](../../05_errors/ERR-018.md#ERR-018) | プロジェクト名重複 |
| 403 | `OWNER_ONLY` → [ERR-017](../../05_errors/ERR-017.md#ERR-017) | オーナー以外は不可 |
