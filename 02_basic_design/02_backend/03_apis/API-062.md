# <span id="API-062"></span>API-062: ダッシュボード集計取得

> **このページは ダッシュボード集計取得 API の契約を定義します。** 契約全体(またはプロジェクト)の質問数・未解決数・公開 FAQ 数・利用率をまとめて返す。

## 項目

| 項目 | 内容 |
|---|---|
| API ID | `API-062` |
| API名 | ダッシュボード集計取得 |
| トレーサビリティID | [TR-033](../../00_traceability/index.md#TR-033) ・ [TR-036](../../00_traceability/index.md#TR-036) |
| エンドポイント | `/dashboard/overview` |
| HTTPメソッド | GET |
| 認証 | Cookie |
| 認可 | オーナー / メンバー(メンバーは `projectId` 必須) |

## 処理概要

契約全体(オーナーが `projectId` 未指定)または特定プロジェクトの主要指標(質問数・未解決数・公開 FAQ 数・利用率)と頻出質問を集計して返す。

| 処理 ID | 処理内容 |
|----|----|
| P-01 | 認証・認可(対象プロジェクトへのアクセス権、メンバーは `projectId` 必須)を検証する |
| P-02 | 期間(`period`)を解決し、集計対象の `from` / `to` を確定する |
| P-03 | 期間・プロジェクトの質問数・未解決数・公開 FAQ 数を集計する |
| P-04 | 当月質問数 / 月次上限から利用率(0〜1)を算出する |
| P-05 | 整形して返却する |

## リクエスト

#### 認証ヘッダ

本 API は利用者セッション(Cookie)による認証を必要とする。

| ヘッダ | 要否 | 説明 |
|---|---|---|
| `Cookie: session=<token>` | 必須 | 利用者セッション |

#### Query Parameters

```json
{ "period": "current_month", "projectId": "{id}(メンバーは必須)" }
```

| 項目 | 型 | 説明 |
|----|----|----|
| `period` | enum | 集計期間。任意。`current_month` / `last_30d`。未指定時は `current_month` |
| `projectId` | string | 対象プロジェクトの ID。オーナーは任意(未指定で契約全体を集約)、メンバーは必須 |

#### Path Parameters

```json
{}
```

#### Request Body

```json
{}
```

## レスポンス

**レスポンス(200)**

```json
{
  "period": { "from": "2026-06-01T00:00:00Z", "to": "2026-06-30T23:59:59Z" },
  "questionCount": 2450,
  "unresolvedCount": 120,
  "publishedFaqCount": 86,
  "usageRate": 0.62,
  "topQuestions": [
    { "question": "返品方法を教えてください", "count": 42 }
  ]
}
```

| 項目 | 型 | 説明 |
|----|----|----|
| `period` | object | 集計対象期間 |
| `　└ from` | string (ISO 8601) | 期間の起点 |
| `　└ to` | string (ISO 8601) | 期間の終点 |
| `questionCount` | integer | 期間内の総質問数 |
| `unresolvedCount` | integer | 未解決の質問数 |
| `publishedFaqCount` | integer | 公開状態の FAQ 件数 |
| `usageRate` | number | 利用率。当月質問数を月次上限で割った比率(0〜1)。`period` が `last_30d` でも当月基準で算出する |
| `topQuestions` | array&lt;object&gt; | 期間内の頻出質問リスト(出現回数の多い順) |
| `　└ question` | string | 質問本文 |
| `　└ count` | integer | 当該質問の出現回数 |

## バリデーション

入力値の検証規則は `## リクエスト` の各パラメータ説明を参照する。メンバーは `projectId` を必須とし、未指定の場合は `400 VALIDATION_ERROR` を返す。

## エラー

| HTTP ステータス | エラーコード | 内容 |
|-----------------|-------------------------|-------------------------------|
| 400 | `VALIDATION_ERROR` → [ERR-001](../../05_errors/ERR-001.md#ERR-001) | メンバーが `projectId` 未指定 |
| 403 | `PROJECT_ACCESS_DENIED` → [ERR-019](../../05_errors/ERR-019.md#ERR-019) | 当該プロジェクトへの権限なし |
