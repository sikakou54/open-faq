# <span id="API-067"></span>API-067: AIしきい値設定取得・更新

> **このページは AIしきい値設定取得・更新 API の仕様を定義します。** 回答可否の信頼度・関連度しきい値をプロジェクト単位で取得・更新する。しきい値はプロジェクト作成時に必ず設定されており、適用元は常にプロジェクト設定値でグローバル既定値などのフォールバック先は持たない。

|----|----|----|----|
| API-067 | [UC-075](../../../01_requirements/04_business_usecases/UC-075.md#UC-075) | — | [TBL-004](../04_database/TBL-004.md#TBL-004) ・ [TBL-031](../04_database/TBL-031.md#TBL-031) |

| API名 | エンドポイント | HTTPメソッド |
|----|----|----|
| AIしきい値設定取得・更新 | `/projects/{id}/ai-thresholds` | GET / PUT |

## 処理概要

回答可否の信頼度・関連度しきい値をプロジェクト単位で取得・更新する。設定単位はプロジェクト単位のみとし、しきい値はプロジェクト作成時に必ず設定されているため適用元は常にプロジェクト設定値とする(グローバル既定値などのフォールバック先は持たない)。更新(PUT)後はシステム処理 [SYS-015](../01_system/SYS-015.md#SYS-015) が AI しきい値キャッシュへ伝播する([SCR-011](../../01_frontend/01_screens/SCR-011.md#SCR-011))。

| 処理 ID | 処理内容 |
|---------|----------|
| P-01 | リクエスト元が当該プロジェクトに割当のあるオーナー / メンバーかを判定する(不一致は 403) |
| P-02 | 対象プロジェクトの存在を確認する(存在しなければ 404) |
| P-03 | GET: 当該プロジェクトの設定値を取得して適用値として返す。取得できない場合はフォールバック先が存在しないため取得失敗エラーを返す |
| P-04 | PUT: 入力値(信頼度・関連度しきい値)を検証する(不正は 400)。信頼度・関連度は 1 セットで必須 |
| P-05 | PUT: プロジェクト設定値を更新し、更新値を AI しきい値キャッシュへ書き込む |

## リクエスト

#### 認証ヘッダ

GET は利用者セッション(Cookie)のみを必要とする参照系で CSRF 不要、PUT は状態変更系のため Cookie と CSRF トークンを必要とする。

| ヘッダ | 要否 | 説明 |
|---|---|---|
| `Cookie: session=<token>` | 必須 | 利用者セッション |
| `X-CSRF-Token: <token>` | PUT のみ必須 | CSRF 対策(状態変更系) |
| `Idempotency-Key: <ULID>` | 任意 | 冪等性(PUT 再送の重複防止) |

#### Query Parameters

```json
{}
```

#### Path Parameters

```json
{
  "id": "対象プロジェクト ID"
}
```

| 項目 | 型 | 説明 |
|---|---|---|
| `id` | string | 対象プロジェクト ID(必須) |

#### Request Body(PUT)

```json
{
  "confidenceThreshold": 0.7,
  "relevanceThreshold": 0.6
}
```

| 項目 | 型 | 説明 |
|---|---|---|
| `confidenceThreshold` | number | 回答可否の信頼度しきい値(0〜1) |
| `relevanceThreshold` | number | 関連度しきい値(0〜1) |

## レスポンス

対象プロジェクトに適用されるしきい値を返す。しきい値は作成時に必ず設定されているため適用元は常に `project` であり、グローバル既定値などのソース切替は存在しない。

**レスポンス(200)**

```json
{
  "effective": {
    "source": "project",
    "confidenceThreshold": 0.7,
    "relevanceThreshold": 0.6
  }
}
```

| 項目 | 型 | 説明 |
|---|---|---|
| `effective` | object | 対象プロジェクトに適用されるしきい値 |
| `effective.source` | enum | 適用元。常に `project`(フォールバック先がないため固定値) |
| `effective.confidenceThreshold` | number | 当該プロジェクトの信頼度しきい値 |
| `effective.relevanceThreshold` | number | 当該プロジェクトの関連度しきい値 |

## バリデーション

入力値の検証規則は `## リクエスト` の各パラメータ説明を参照する。信頼度・関連度しきい値は 0〜1 の範囲で検証する。PUT では `confidenceThreshold` と `relevanceThreshold` の両方を必須とし、片方だけの更新は許容しない。

## エラー

| HTTP ステータス | エラーコード | 内容 |
|-----------------|---------------------|--------------------------------|
| 400 | `VALIDATION_ERROR` → [ERR-001](../../05_errors/ERR-001.md#ERR-001) | 入力値の検証エラー |
| 403 | `PROJECT_ACCESS_DENIED` → [ERR-019](../../05_errors/ERR-019.md#ERR-019) | 当該プロジェクトに割当のないユーザーのアクセス |
| 404 | `PROJECT_NOT_FOUND` → [ERR-011](../../05_errors/ERR-011.md#ERR-011) | 対象プロジェクトが存在しない |
