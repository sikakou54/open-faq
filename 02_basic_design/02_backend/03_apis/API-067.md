# <span id="API-067"></span>API-067: AIしきい値設定取得・更新

> **このページは AIしきい値設定取得・更新 API の仕様を定義します。** 回答可否の信頼度・関連度しきい値をプロジェクト単位で取得・更新・リセットし、未登録または取得失敗時はグローバル既定値を返す。

| ID | 業務ユースケースID | イベント(画面ID EVT-NN) | テーブルID |
|----|----|----|----|
| API-067 | [UC-075](../../../01_requirements/04_business_usecases/UC-075.md#UC-075) | — | [TBL-004](../04_database/TBL-004.md#TBL-004) ・ [TBL-031](../04_database/TBL-031.md#TBL-031) |

| API名 | エンドポイント | HTTPメソッド |
|----|----|----|
| AIしきい値設定取得・更新 | `/projects/{id}/ai-thresholds` | GET / PUT |

## 処理概要

回答可否の信頼度・関連度しきい値をプロジェクト単位で取得・更新・リセットする。適用順序はプロジェクト設定値、グローバル既定値の 2 段階のみとする。対象プロジェクトの設定値が登録されている場合はその値を使用し、未登録または取得失敗の場合はグローバル既定値を使用する。設定単位はプロジェクト単位のみとし、プロジェクト設定値は信頼度・関連度を 1 セットとして保存または削除する。更新(PUT)後はシステム処理 [SYS-015](../01_system/SYS-015.md#SYS-015) が AI しきい値キャッシュへ伝播する([SCR-011](../../01_frontend/01_screens/SCR-011.md#SCR-011))。

| 処理 ID | 処理内容 |
|---------|----------|
| P-01 | リクエスト元が当該プロジェクトに割当のあるオーナー / メンバーかを判定する(不一致は 403) |
| P-02 | 対象プロジェクトの存在を確認する(存在しなければ 404) |
| P-03 | GET: プロジェクト設定値を取得する。未登録または取得失敗時はグローバル既定値を対象プロジェクトの適用値として返す |
| P-04 | PUT: 入力値(信頼度・関連度しきい値)を検証する(不正は 400)。信頼度・関連度は 1 セットで扱い、両方 number または両方 null のみ許容する |
| P-05 | PUT: 両方 number の場合はプロジェクト設定値を更新し、更新値を AI しきい値キャッシュへ書き込む |
| P-06 | PUT: 両方 null の場合は当該プロジェクト固有の設定を削除し、以後はグローバル既定値へ復帰させる |

## リクエスト

#### 認証ヘッダ

GET は利用者セッション(Cookie)のみを必要とする参照系で CSRF 不要、PUT は状態変更系のため Cookie と CSRF トークンを必要とする。

| ヘッダ | 要否 | 説明 |
|---|---|---|
| `Cookie: session=<token>` | 必須 | 利用者セッション |
| `X-CSRF-Token: <token>` | PUT のみ必須 | CSRF 対策(状態変更系) |
| `Idempotency-Key: <ULID>` | 任意 | 冪等性(PUT 再送の重複防止) |

#### Query Parameters

```json
{}
```

#### Path Parameters

```json
{
  "id": "対象プロジェクト ID"
}
```

| 項目 | 型 | 説明 |
|---|---|---|
| `id` | string | 対象プロジェクト ID(必須) |

#### Request Body(PUT)

```json
{
  "confidenceThreshold": 0.7,
  "relevanceThreshold": 0.6
}
```

リセット(グローバル既定値へ復帰):

```json
{
  "confidenceThreshold": null,
  "relevanceThreshold": null
}
```

| 項目 | 型 | 説明 |
|---|---|---|
| `confidenceThreshold` | number \| null | 回答可否の信頼度しきい値(0〜1)。`null` はプロジェクト設定値削除時のみ指定可 |
| `relevanceThreshold` | number \| null | 関連度しきい値(0〜1)。`null` はプロジェクト設定値削除時のみ指定可 |

## レスポンス

対象プロジェクトに適用されるしきい値と、設定ソース別の値を返す。`layers` は `global` と `project` の 2 要素のみを持つ。

**レスポンス(200)**

```json
{
  "effective": {
    "source": "project",
    "confidenceThreshold": 0.7,
    "relevanceThreshold": 0.6
  },
  "layers": {
    "global": { "confidenceThreshold": 0.6, "relevanceThreshold": 0.5 },
    "project": { "confidenceThreshold": 0.7, "relevanceThreshold": 0.6 }
  }
}
```

リセット後(プロジェクト設定値削除後):

```json
{
  "effective": {
    "source": "global",
    "confidenceThreshold": 0.6,
    "relevanceThreshold": 0.5
  },
  "layers": {
    "global": { "confidenceThreshold": 0.6, "relevanceThreshold": 0.5 },
    "project": null
  }
}
```

| 項目 | 型 | 説明 |
|---|---|---|
| `effective` | object | 対象プロジェクトに適用されるしきい値 |
| `effective.source` | enum | 適用元。`project` または `global` |
| `layers.global` | object | グローバル既定値。[システム仕様書 §1](../../07_system-spec.md#1-aiしきい値) の値を返す |
| `layers.project` | object / null | プロジェクト設定値。未登録または取得失敗時は `null` |

## バリデーション

入力値の検証規則は `## リクエスト` の各パラメータ説明を参照する。信頼度・関連度しきい値は 0〜1 の範囲で検証する。PUT では `confidenceThreshold` と `relevanceThreshold` の両フィールドを必須とし、両方 number または両方 null のみ許容する。片方だけの更新、片方だけ null、両方省略は許容しない。両方 null の場合は当該プロジェクト設定値を削除し、レスポンスは `effective.source=global`、`layers.project=null` を返す。

## エラー

| HTTP ステータス | エラーコード | 内容 |
|-----------------|---------------------|--------------------------------|
| 400 | `VALIDATION_ERROR` → [ERR-001](../../05_errors/ERR-001.md#ERR-001) | 入力値の検証エラー |
| 403 | `PROJECT_ACCESS_DENIED` → [ERR-019](../../05_errors/ERR-019.md#ERR-019) | 当該プロジェクトに割当のないユーザーのアクセス |
| 404 | `PROJECT_NOT_FOUND` → [ERR-011](../../05_errors/ERR-011.md#ERR-011) | 対象プロジェクトが存在しない |
