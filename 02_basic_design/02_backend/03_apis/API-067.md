# <span id="API-067"></span>API-067: AIしきい値設定取得・更新

> **このページは AIしきい値設定取得・更新 API の契約を定義します。** 回答可否の信頼度・関連度しきい値を 3 階層(グローバル / 契約 / プロジェクト)で取得・更新する。

## 項目

| 項目 | 内容 |
|---|---|
| API ID | `API-067` |
| API名 | AIしきい値設定取得・更新 |
| トレーサビリティID | [TR-080](../../00_traceability/index.md#TR-080) |
| エンドポイント | `/projects/{id}/ai-thresholds` |
| HTTPメソッド | GET / PUT |
| 認証 | セッション(Cookie)、PUT は CSRF 併用 |
| 認可 | オーナー / 当該プロジェクトのメンバー |

## 処理概要

回答可否の信頼度・関連度しきい値を 3 階層(グローバル / 契約 / プロジェクト)で取得・更新する。より具体的な階層を優先適用し、未設定の階層は上位を継承する。更新(PUT)後はシステム処理 [SYS-017](../01_system/SYS-017.md#SYS-017) が AI しきい値キャッシュへ伝播する([SCR-011](../../01_frontend/01_screens/SCR-011.md#SCR-011))。

| 処理 ID | 処理内容 |
|---------|----------|
| P-01 | リクエスト元が当該プロジェクトに割当のあるオーナー / メンバーかを判定する(不一致は 403) |
| P-02 | 対象プロジェクトの存在を確認する(存在しなければ 404) |
| P-03 | GET: グローバル / 契約 / プロジェクトの 3 階層のしきい値を取得し、有効値を継承解決して返す |
| P-04 | PUT: 入力値(信頼度・関連度しきい値)を検証する(不正は 400) |
| P-05 | PUT: プロジェクト階層のしきい値を更新し、更新値を AI しきい値キャッシュへ書き込む |

## リクエスト

#### 認証ヘッダ

GET は利用者セッション(Cookie)のみを必要とする参照系で CSRF 不要、PUT は状態変更系のため Cookie と CSRF トークンを必要とする。

| ヘッダ | 要否 | 説明 |
|---|---|---|
| `Cookie: session=<token>` | 必須 | 利用者セッション |
| `X-CSRF-Token: <token>` | PUT のみ必須 | CSRF 対策(状態変更系) |
| `Idempotency-Key: <ULID>` | 任意 | 冪等性(PUT 再送の重複防止) |

#### Query Parameters

```json
{}
```

#### Path Parameters

```json
{
  "id": "対象プロジェクト ID"
}
```

| 項目 | 型 | 説明 |
|---|---|---|
| `id` | string | 対象プロジェクト ID(必須) |

#### Request Body(PUT)

```json
{
  "confidenceThreshold": 0.7,
  "relevanceThreshold": 0.6
}
```

| 項目 | 型 | 説明 |
|---|---|---|
| `confidenceThreshold` | number | 回答可否の信頼度しきい値(0〜1) |
| `relevanceThreshold` | number | 関連度しきい値(0〜1) |

## レスポンス

3 階層のしきい値と継承解決後の有効値を返す。

**レスポンス(200)**

```json
{
  "effective": { "confidenceThreshold": 0.7, "relevanceThreshold": 0.6 },
  "layers": {
    "global": { "confidenceThreshold": 0.5, "relevanceThreshold": 0.5 },
    "contract": { "confidenceThreshold": 0.6, "relevanceThreshold": null },
    "project": { "confidenceThreshold": 0.7, "relevanceThreshold": 0.6 }
  }
}
```

| 項目 | 型 | 説明 |
|---|---|---|
| `effective` | object | 継承解決後の有効しきい値 |
| `layers.global` | object | グローバル階層のしきい値 |
| `layers.contract` | object | 契約階層のしきい値(未設定は `null`) |
| `layers.project` | object | プロジェクト階層のしきい値(未設定は `null`) |

## バリデーション

入力値の検証規則は `## リクエスト` の各パラメータ説明を参照する。信頼度・関連度しきい値は 0〜1 の範囲で検証する。

## エラー

| HTTP ステータス | エラーコード | 内容 |
|-----------------|---------------------|--------------------------------|
| 400 | `VALIDATION_ERROR` → [ERR-001](../../05_errors/ERR-001.md#ERR-001) | 入力値の検証エラー |
| 403 | `PROJECT_ACCESS_DENIED` → [ERR-019](../../05_errors/ERR-019.md#ERR-019) | 当該プロジェクトに割当のないユーザーのアクセス |
| 404 | `PROJECT_NOT_FOUND` → [ERR-011](../../05_errors/ERR-011.md#ERR-011) | 対象プロジェクトが存在しない |

## 備考

しきい値は 3 階層(グローバル / 契約 / プロジェクト)で保持し、より具体的な階層を優先適用、未設定の階層は上位を継承する。しきい値更新は [SYS-017](../01_system/SYS-017.md#SYS-017) が `TP_AI_THRESH_CACHE` へ伝播し、AI 推論側はキャッシュをフォールバックで参照する。
