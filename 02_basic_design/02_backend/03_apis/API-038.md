## 処理概要

AI 回答を生成し質問ログを記録する。未回答時は未解決質問を同時生成する。

| 処理 ID | 処理内容                                             |
|---------|------------------------------------------------------|
| P-01    | セッショントークンとオリジンを検証する               |
| P-02    | オーナー単位のレート制限とプロジェクト月次上限を判定する |
| P-03    | AI 推論 IF で候補 FAQ から回答を生成し、候補 FAQ 照合結果から関連質問サジェスト(関連 FAQ の質問文・関連度順に最大 3 件・設計値)を併せて求める。所定時間（[システム仕様書 §3](../../07_system-spec.md#3-タイムアウトセッション認証)）内に応答しない、または AI プロバイダがエラーを返した場合は、未解決質問として扱うフォールバックへ遷移する |
| P-04    | 質問ログを記録する(AI タイムアウト / プロバイダエラー時は未回答理由を記録) |
| P-05    | 未回答時(低信頼度・該当 FAQ なし・個人情報検出・AI タイムアウト / プロバイダエラー)は未解決質問を同一トランザクションで生成する |
| P-06    | 回答または未回答理由・連絡先を返却する。回答時は関連質問サジェストを併せて返す。AI タイムアウト / プロバイダエラー時は定型の案内文(未解決・制限・エラー案内 UC に準拠)を返す |

## リクエスト

#### 認証ヘッダ

本 API はウィジェットセッショントークン(Bearer)による認証を必要とする。

| ヘッダ | 要否 | 説明 |
|---|---|---|
| `Authorization: Bearer <wst_*>` | 必須 | ウィジェットセッション(起動 API で発行) |
| `Idempotency-Key: <ULID>` | 任意 | 冪等性(再送の重複防止) |

## レスポンス

**レスポンス(200)**

answered:

```json
{
  "type": "answered",
  "answer": "返品は商品到着後 7 日以内に...",
  "confidence": 0.78,
  "referencedFaqs": [
    { "id": "faq_...", "question": "返品ポリシー", "answer": "..." }
  ],
  "suggestions": [
    { "faqId": "faq_...", "label": "支払い方法を変更したい" },
    { "faqId": "faq_...", "label": "領収書がほしい" }
  ],
  "questionLogId": "qlog_..."
}
```

| 項目             | 型              | 説明                            |
|------------------|-----------------|---------------------------------|
| `type`           | enum            | 回答種別。`answered`(回答あり)  |
| `answer`         | string          | AI が生成した回答本文           |
| `confidence`     | number          | 回答の信頼度(0〜1)              |
| `referencedFaqs` | array\<object\> | 回答の根拠とした参照 FAQ の配列 |
| `　└ id`         | string          | 参照 FAQ の ID                  |
| `　└ question`   | string          | 参照 FAQ の質問                 |
| `　└ answer`     | string          | 参照 FAQ の回答                 |
| `suggestions`    | array\<object\> | 関連質問サジェスト。候補 FAQ 照合結果から関連 FAQ の質問文を関連度順に最大 3 件(設計値)返す(ウィジェットはチップ表示) |
| `　└ faqId`      | string          | サジェスト元 FAQ の ID          |
| `　└ label`      | string          | チップ表示文(関連 FAQ の質問文) |
| `questionLogId`  | string          | 記録された質問ログの ID         |

**レスポンス(200)**

unanswered:

```json
{
  "type": "unanswered",
  "reason": "low_confidence" | "no_faq_match" | "pii_detected" | "ai_unavailable" | "contradiction" | "usage_limit_reached" | "payment_required",
  "questionLogId": "qlog_...",
  "inquiryId": "inq_...",
  "inquiryCode": "INQ-20260513-XYZ123",
  "contactEmail": "support@example.com"
}
```

| 項目 | 型 | 説明 |
|----|----|----|
| `type` | enum | 回答種別。`unanswered`(回答不可) |
| `reason` | enum | 未回答 / 受付停止理由。`low_confidence`(信頼度不足)/ `no_faq_match`(該当 FAQ なし)/ `pii_detected`(個人情報検出)/ `ai_unavailable`(AI 推論タイムアウト（[システム仕様書 §3](../../07_system-spec.md#3-タイムアウトセッション認証)） / プロバイダエラー)/ `contradiction`(候補 FAQ 間の矛盾)/ `usage_limit_reached`(質問数上限 100% 到達による受付停止)/ `payment_required`(支払方法ゲートによる受付停止) |
| `questionLogId` | string | 記録された質問ログの ID |
| `inquiryId` | string | 同時生成された未解決質問の ID |
| `inquiryCode` | string | 未解決質問の管理用識別子(ウィジェットには描画しない) |
| `contactEmail` | string | プロジェクトの問い合わせ先メールアドレス(未設定時は `null`) |

`unanswered` では質問ログと未解決質問を同一トランザクションで作成する。`inquiryCode` は管理用識別子としてレスポンス互換性のため返却するが、ウィジェットには描画しない。ウィジェットは回答できなかった旨と確認済みプロジェクト連絡先メールを表示し、FAQ 質問入力は継続可能とする。連絡先未設定時は `contactEmail=null` を返す。冪等性は `questionLogId` を基準に担保する。

AI 推論が所定時間（[システム仕様書 §3](../../07_system-spec.md#3-タイムアウトセッション認証)）内に応答しない、または AI プロバイダがエラーを返した場合も、エラー(5xx)を返さず未解決質問として記録し、利用者へは定型の「回答を生成できませんでした。お問い合わせをご利用ください」案内(未解決・制限・エラー案内 UC に準拠)を返す。この場合の `unanswered` レスポンスでは `reason` に AI 推論失敗を示す値(`ai_unavailable`)を設定し、`contactEmail` を併せて返す。

`suggestions`(関連質問サジェスト)は候補 FAQ 照合結果から関連 FAQ の質問文を関連度順に最大 3 件(設計値。0 件のときは空配列)返す。

## 列挙値

リクエスト / レスポンスで用いる enum 項目の取りうる値の全集合と意味を示す。レスポンスは回答可否で `type` が分岐し、未回答時のみ `reason` を返す。

| 項目 | 取りうる値 | 意味 |
|----|----|----|
| `type` | `answered` | 回答あり(AI が回答を生成) |
| `type` | `unanswered` | 回答不可(未解決質問を生成し連絡先を案内) |
| `reason` | `low_confidence` | 信頼度がしきい値未満(回答を生成できなかった) |
| `reason` | `no_faq_match` | 該当する FAQ が無い |
| `reason` | `pii_detected` | **質問文**に個人情報を検出し回答を行わなかった(未解決扱い)。一方、生成した**回答文**中の個人情報は [RULE-024](../../../01_requirements/01_business_requirement/08_rule.md#RULE-024) によりマスキングして `answered` で返すため本理由には該当しない |
| `reason` | `ai_unavailable` | AI 推論タイムアウト（[システム仕様書 §3](../../07_system-spec.md#3-タイムアウトセッション認証)）/ AI プロバイダエラー |
| `reason` | `contradiction` | 候補 FAQ 間に矛盾があり断定回答しない([FR-060](../../../01_requirements/02_functional_requirement/02_faq-ai-fr.md#FR-060) / [RULE-023](../../../01_requirements/01_business_requirement/08_rule.md#RULE-023)) |
| `reason` | `usage_limit_reached` | 当該プロジェクトの質問数が月次上限の 100% に到達し新規質問受付を停止(受付停止) |
| `reason` | `payment_required` | 支払方法未登録のまま無料枠を超過し新規質問受付を停止(支払方法ゲート停止) |

> [!NOTE]
> **`reason` は `type=unanswered` のときのみ返す。** 質問ログ側のコード値([`H_QUESTION_LOGS.result_reason_code`](../04_database/TBL-025.md#TBL-025))は回答成功を示す `answered`・レート制限を示す `rate_limited` も保持するが、本 API のレスポンス `reason` には上記 7 値(未回答理由 4 値 + 受付停止 3 値)が現れる(レート制限は本 API では 429 [ERR-009](../../05_errors/ERR-009.md#ERR-009) で返す)。

> [!IMPORTANT]
> **受付停止は 200(graceful)で返す。** `usage_limit_reached`(質問数上限 100% 到達)・`payment_required`(支払方法ゲート)による新規質問受付停止は、エラー(4xx/5xx)ではなく 200 の `type=unanswered`(対応する `reason`)として返す。ウィジェットは送信を無効化し、停止理由に応じた定型文を表示する。停止の契機・課金アカウント状態・解除条件の正本は [課金・請求設計 §2.1](../../05_billing-design.md#21-停止状態とウィジェット応答一覧) / [§4](../../05_billing-design.md#4-支払方法ゲート)。`contradiction` は候補 FAQ 間の矛盾により断定回答を避け未解決として扱うもので、未解決質問を生成し連絡先を案内する([FR-060](../../../01_requirements/02_functional_requirement/02_faq-ai-fr.md#FR-060) / [RULE-023](../../../01_requirements/01_business_requirement/08_rule.md#RULE-023))。なお課金アカウントが `suspended` の場合は受付停止ではなく別途エラー応答(`BILLING_ACCOUNT_SUSPENDED`)で扱う(詳細設計で確定)。

## バリデーション

入力値の検証規則は `## リクエスト` の各パラメータ説明を参照する。`question` は 1〜1000 文字とする(画面 SCR-030 の最大長 1000 文字と一致)。

## エラー

| HTTP ステータス | エラーコード         | 内容               |
|-----------------|----------------------|--------------------|
| 429 | `RATE_LIMITED` → [ERR-009](../../05_errors/ERR-009.md#ERR-009) | レート制限超過 |
| 403 | `DOMAIN_NOT_ALLOWED` → [ERR-027](../../05_errors/ERR-027.md#ERR-027) | 許可ドメイン外 |

> [!NOTE]
> **AI 推論のタイムアウト（[システム仕様書 §3](../../07_system-spec.md#3-タイムアウトセッション認証)）/ プロバイダエラーはエラー応答にしない。** これらは 200 の `unanswered`(`reason=ai_unavailable`)として処理し、未解決質問を記録のうえ定型の「回答を生成できませんでした。お問い合わせをご利用ください」案内を返す(未解決・制限・エラー案内 UC に準拠)。
