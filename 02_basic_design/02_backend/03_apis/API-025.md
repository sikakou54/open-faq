## 処理概要

状態 / プロジェクトで FAQ を一覧取得する(キーワード全文検索は API-031)。

| 処理 ID | 処理内容                                                      |
|---------|---------------------------------------------------------------|
| P-01    | 認証・認可(オーナー / 当該プロジェクトの利用者)を検証する     |
| P-02    | 状態・プロジェクト・カテゴリのフィルタ条件で FAQ を取得し、指定の並び順(`relevance` / `updated_at` / `created_at`)で整列する |
| P-03    | ページング(カーソル)で整形して返す                            |

## リクエスト

#### 認証ヘッダ

本 API は利用者セッション(Cookie)による認証を必要とする。

| ヘッダ | 要否 | 説明 |
|---|---|---|
| `Cookie: session=<token>` | 必須 | 利用者セッション |

#### Query Parameters

```json
{
  "status": "draft(任意)",
  "projectId": "(必須)",
  "category": "(任意)",
  "sort": "relevance(任意)",
  "cursor": "(共通仕様)",
  "limit": "(共通仕様)"
}
```

| 項目 | 型 | 説明 |
|----|----|----|
| `status` | enum | 状態での絞り込み(任意・`draft` / `published` / `hidden`) |
| `projectId` | string | 対象プロジェクトの ID(必須) |
| `category` | string | カテゴリでの絞り込み(任意・対象プロジェクト内のカテゴリ値で絞り込み・未指定=すべて) |
| `sort` | enum | 並び順(任意・`relevance` / `updated_at` / `created_at`・既定は `relevance`=関連度) |
| `cursor` | string | 次ページ取得用カーソル(任意・共通ページング仕様) |
| `limit` | integer | 1 ページの取得件数(任意・共通ページング仕様) |

## レスポンス

**レスポンス(200)**

FAQ の配列 + `nextCursor`(構造は API-031 のレスポンスに準ずる)。

| 項目 | 型 | 説明 |
|----|----|----|
| `items` | array\<object\> | フィルタ条件に一致した FAQ の一覧(各要素の構造は API-031 全文検索のレスポンスに準ずる) |
| `nextCursor` | string | 次ページ取得用カーソル(末尾は `null`) |
