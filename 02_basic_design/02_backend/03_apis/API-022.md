# <span id="API-022"></span>API-022: メンバー情報更新

## 処理概要

対象メンバーのメールアドレスを更新する。

| 処理 ID | 処理内容                                                         |
|---------|------------------------------------------------------------------|
| P-00    | 再認証トークン(リクエストボディ `reauthToken`)を検証する          |
| P-01    | 認証・認可(オーナー / 当該プロジェクトのメンバー)と再認証を検証する  |
| P-02    | 対象割当の存在と入力値(メール形式)を検証する                    |
| P-03    | メールアドレスの重複を判定し該当時は拒否する                     |
| P-04    | 対象メンバーのメールアドレスを更新する                           |
| P-05    | 更新結果を返す                                                   |

## リクエスト

#### 認証ヘッダ

本 API は利用者セッション(Cookie)と CSRF トークンを必要とする状態変更系である。

| ヘッダ | 要否 | 説明 |
|---|---|---|
| `Cookie: session=<token>` | 必須 | 利用者セッション |
| `X-CSRF-Token: <token>` | 必須 | CSRF 対策(状態変更系) |
| `Idempotency-Key: <ULID>` | 任意 | 冪等性(書き込み再送の重複防止) |

> 本操作は再認証必須。直近の再認証([API-005](API-005.md#API-005))で得た再認証状態を伴う必要がある。

#### Query Parameters

```json
{}
```

#### Path Parameters

```json
{
  "id": "プロジェクト ID",
  "userId": "対象ユーザー ID"
}
```

| 項目     | 型     | 説明                                                |
|----------|--------|-----------------------------------------------------|
| `id`     | string | 対象プロジェクトの ID(必須・パス内で指定)           |
| `userId` | string | 情報を更新する対象利用者の ID(必須・パス内で指定)   |

#### Request Body

```json
{
  "reauthToken": "reauth_...",
  "email": "member@..."
}
```

| 項目    | 型     | 説明                               |
|---------|--------|------------------------------------|
| `reauthToken` | string | 再認証トークン(必須・API-005 で取得) |
| `email` | string | 更新後のメールアドレス(必須)       |

- 表示名(`displayName`)はメンバー本人が SCR-023 有効化ページで入力するため、本 API では受け付けない
- メンバーは当該プロジェクト内で同一権限を持ち、役割の区別は存在しない

## レスポンス

**レスポンス(200)**

```json
{ "userId": "...", "projectId": "...", "email": "member@...", "updatedAt": "..." }
```

| 項目        | 型                | 説明                               |
|-------------|-------------------|------------------------------------|
| `userId`    | string            | 情報を更新した対象利用者の ID      |
| `projectId` | string            | 対象プロジェクトの ID              |
| `email`     | string            | 更新後のメールアドレス             |
| `updatedAt` | string (ISO 8601) | 更新日時                           |

## バリデーション

入力値の検証規則は `## リクエスト` の各パラメータ説明を参照する。

## エラー

| HTTP ステータス | エラーコード            | 内容                         |
|-----------------|-------------------------|------------------------------|
| 401 | `REAUTH_REQUIRED` → [ERR-013](../../05_errors/ERR-013.md#ERR-013) | 再認証トークンが無効または未提示 |
| 404 | `NOT_FOUND` → [ERR-017](../../05_errors/ERR-017.md#ERR-017) | 対象が存在しない |
| 403 | `PROJECT_ACCESS_DENIED` → [ERR-019](../../05_errors/ERR-019.md#ERR-019) | 当該プロジェクトへの権限なし |
| 409 | `EMAIL_ALREADY_EXISTS` → [ERR-020](../../05_errors/ERR-020.md#ERR-020) | メールアドレスの重複 |
| 400 | `VALIDATION_ERROR` → [ERR-001](../../05_errors/ERR-001.md#ERR-001) | 入力値の検証エラー |
