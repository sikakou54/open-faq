# <span id="API-060"></span>API-060: 課金プロバイダ Webhook 受信

> **このページは 課金プロバイダ Webhook 受信 API の仕様を定義します。** 課金プロバイダからの決済・課金状態通知を受信し、署名検証・重複排除のうえ課金アカウントの状態へ取り込む。

| ID | 業務ユースケースID | イベント(画面ID EVT-NN) | テーブルID |
|----|----|----|----|
| API-060 | [UC-056](../../../01_requirements/04_business_usecases/UC-056.md#UC-056) | — | [TBL-002](../04_database/TBL-002.md#TBL-002) ・ [TBL-018](../04_database/TBL-018.md#TBL-018) ・ [TBL-019](../04_database/TBL-019.md#TBL-019) ・ [TBL-027](../04_database/TBL-027.md#TBL-027) ・ [TBL-032](../04_database/TBL-032.md#TBL-032) |

| API名 | エンドポイント | HTTPメソッド |
|----|----|----|
| 課金プロバイダ Webhook 受信 | `/webhooks/billing` | POST |

## 処理概要

課金プロバイダからの決済・課金状態通知を受信し、署名検証・重複排除のうえ課金アカウントの状態へ取り込む。取込失敗は再処理対象として記録する。

| 処理 ID | 処理内容 |
|----|----|
| P-01 | 送信元署名(HMAC-SHA256)を検証する(失敗時は 401・不正受信として記録) |
| P-02 | 受信済みイベント ID と照合し重複受信を冪等に扱う |
| P-03 | 受信内容を受信ログへ記録する |
| P-04 | 通知内容を課金アカウント・サブスクリプション・請求書の状態へ反映する |
| P-05 | 取込失敗を記録・通知し再処理対象とする |
| P-06 | 受領応答を返す |

## リクエスト

#### 認証ヘッダ

本 API は課金プロバイダ(Stripe)が付与する署名(HMAC-SHA256)を検証して受信元を認証する(認可はなし)。

| ヘッダ | 要否 | 説明 |
|---|---|---|
| `Stripe-Signature: <署名>` | 必須 | Stripe 署名。HMAC-SHA256 で検証(失敗時 401) |

```json
{}
```

#### Path Parameters

```json
{}
```

#### Request Body

(課金プロバイダ仕様準拠):

```json
{
  "type": "payment.succeeded|payment.failed|subscription.updated|invoice.created",
  "data": {
    "event_id": "...",
    "billing_account_ref": "...",
    "timestamp": "..."
  }
}
```

| 項目 | 型 | 説明 |
|----|----|----|
| `type` | enum | 通知イベント種別。取りうる値の全集合は `## 列挙値` を参照 |
| `data` | object | イベント詳細データ |
| 　└ `event_id` | string | プロバイダ側イベント ID(重複排除キー) |
| 　└ `billing_account_ref` | string | 対象課金アカウントの参照キー |
| 　└ `timestamp` | string (ISO 8601) | イベント発生時刻 |

## 列挙値

`type`(通知イベント種別)の取りうる値の全集合と、各値が反映する状態の対応を示す(課金プロバイダ = Stripe 準拠)。取込時に各 type を [SYS-004](../01_system/SYS-004.md#SYS-004) が課金アカウント・サブスクリプション・請求書の状態へ反映する。受信値は [`T_BILLING_WEBHOOK_LOG.event_type`](../04_database/TBL-032.md#TBL-032) に保存する。本表に無い type は受信ログへ記録のうえ状態反映なしで冪等に応答する。

| 項目 | 取りうる値 | 反映対象 | 意味 / 反映する状態 |
|----|----|----|----|
| `type` | `payment.succeeded` | 課金アカウント / 請求書 | 決済成功。猶予中であれば課金アカウントを `active` へ復帰し、対象請求書を `paid` にする |
| `type` | `payment.failed` | 課金アカウント / 請求書 | 決済失敗。失敗確定通知として決済失敗の猶予(7 日)を起算し、対象請求書を `past_due` にする |
| `type` | `subscription.updated` | サブスクリプション | サブスク状態の更新(`active` / `past_due` / `unpaid` / `incomplete` への遷移を反映) |
| `type` | `subscription.deleted` | サブスクリプション | サブスク解約。サブスクを `canceled` にする |
| `type` | `invoice.created` | 請求書 | 請求書の発行。請求書を `issued` で取り込む |
| `type` | `invoice.paid` | 請求書 / 課金アカウント | 請求書の支払完了。請求書を `paid` にする |
| `type` | `invoice.payment_failed` | 請求書 / 課金アカウント | 請求書の支払失敗。請求書を `past_due` にし、決済失敗の猶予判定へ連動する |
| `type` | `invoice.voided` | 請求書 | 請求書の無効化。請求書を `void` にする |

> [!NOTE]
> **課金アカウント状態(`active` / `suspended` / `withdrawn` / `deleted`)・サブスクリプション状態・請求書状態の定義そのものは再掲しない。** 課金アカウント状態は [状態モデル §2](../../08_state-model.md#2-課金アカウント状態)、サブスクリプション状態は [`T_BILL_SUBS`](../04_database/TBL-018.md#TBL-018)、請求書状態は [`T_BILL_INVOICES`](../04_database/TBL-019.md#TBL-019) を正本とする。決済失敗の猶予(7 日)経過によるサスペンション遷移は [課金・請求設計 §5](../../05_billing-design.md#5-課金アカウント状態ライフサイクル) が正本。

## レスポンス

本 API の戻り値は本文の型定義を参照する。受領を示す応答を返す。

## バリデーション

入力値の検証規則は `## リクエスト` の各パラメータ説明を参照する。署名検証を通過しない通知は取り込まない。

## エラー

| HTTP ステータス | エラーコード | 内容 |
|-----------------|---------------------|--------------------------------|
| 401 | `SIGNATURE_INVALID` → [ERR-031](../../05_errors/ERR-031.md#ERR-031) | 署名検証失敗(不正受信) |
| 200 | `IDEMPOTENT_REPLAY` → [ERR-032](../../05_errors/ERR-032.md#ERR-032) | 重複受信を冪等に処理(状態変更なし) |
