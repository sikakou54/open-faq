# <span id="API-070"></span>API-070: ログイン失敗ロック解除

> **このページは ログイン失敗ロック解除 API の仕様を定義します。** 連続ログイン失敗でロック中の対象利用者のロックを、権限者の操作で解除する。

| ID | 業務ユースケースID | イベント(画面ID EVT-NN) | テーブルID |
|----|----|----|----|
| API-070 | [UC-068](../../../01_requirements/04_business_usecases/UC-068.md#UC-068) | SCR-014 EVT-10 | [TBL-001](../04_database/TBL-001.md#TBL-001) |

| API名 | エンドポイント | HTTPメソッド |
|----|----|----|
| ログイン失敗ロック解除 | `/projects/{id}/members/{userId}/unlock` | POST |

## 処理概要

連続ログイン失敗でロック中の対象利用者について、`login_failed_count` を初期化し `locked_until` を解除する([RULE-001](../../../01_requirements/01_business_requirement/08_rule.md#RULE-001)・[PERM-007](../../04_permissions/PERM-007.md#PERM-007))。対象がロック中でない場合も同じ完了応答を返す(冪等)。

| 処理 ID | 処理内容 |
|----|----|
| P-01 | 認証・認可(オーナー / 当該プロジェクトの有効メンバー、[PERM-007](../../04_permissions/PERM-007.md#PERM-007))を検証する |
| P-02 | 対象利用者が当該プロジェクトの割当を持つことを確認する |
| P-03 | 対象がロック中(`locked_until` が現在時刻より未来)かどうかを判定する |
| P-04 | ロック中の場合、`locked_until` を解除し `login_failed_count` を初期化する |
| P-05 | ロック中でない場合は状態を変更せず、解除済みと同じ結果を返す(冪等) |
| P-06 | 解除結果(解除の要否・実施有無)を返す |

## リクエスト

#### 認証ヘッダ

本 API は利用者セッション(Cookie)と CSRF トークンを必要とする状態変更系である。

| ヘッダ | 要否 | 説明 |
|---|---|---|
| `Cookie: session=<token>` | 必須 | 利用者セッション |
| `X-CSRF-Token: <token>` | 必須 | CSRF 対策(状態変更系) |
| `Idempotency-Key: <ULID>` | 任意 | 冪等性(書き込み再送の重複防止) |

#### Query Parameters

```json
{}
```

#### Path Parameters

```json
{
  "id": "プロジェクト ID",
  "userId": "対象ユーザー ID"
}
```

| 項目 | 型 | 説明 |
|----|----|----|
| `id` | string | 対象プロジェクトの ID(必須・パス内で指定) |
| `userId` | string | ロックを解除する対象利用者の ID(必須・パス内で指定) |

#### Request Body

```json
{}
```

## レスポンス

**レスポンス(200)**

```json
{ "userId": "...", "projectId": "...", "wasLocked": true, "unlockedAt": "..." }
```

| 項目 | 型 | 説明 |
|----|----|----|
| `userId` | string | ロックを解除した対象利用者の ID |
| `projectId` | string | 対象プロジェクトの ID |
| `wasLocked` | boolean | 呼び出し時点でロック中だったか(`false` は既に解除済みだったことを示す) |
| `unlockedAt` | string (ISO 8601) | 応答時点の解除確認日時(`wasLocked=false` でも現在時刻を返す) |

|----|----|----|

## 備考

対象がロック中でない場合も 200 で同一形状の応答を返し、実施有無は `wasLocked` で示す(再実行しても副作用を生まない冪等応答)。
