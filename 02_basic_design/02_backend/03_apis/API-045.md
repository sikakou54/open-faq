## 処理概要

支払方法の取得と登録 / 更新(クレジットカードのみ)。支払方法はユーザー(アカウント)単位で、認証ユーザー本人の課金アカウント(`M_BILLING_ACCOUNT`、`user_id` UNIQUE)に 1 つ紐づき、本人が作成した(オーナーである)全プロジェクトの請求に用いられる。

| 処理 ID | 処理内容 |
|----|----|
| P-00 | 再認証トークン(リクエストボディ `reauthToken`)を検証する(PUT 時必須) |
| P-01 | 認証・認可(本人(課金アカウント保有者)、PUT は再認証必須)を検証する。認証ユーザーの課金アカウント(`M_BILLING_ACCOUNT.user_id` が認証ユーザーと一致)を特定する |
| P-02 | GET は本人の課金アカウントに登録済みの支払方法(ブランド・下 4 桁等)を取得する |
| P-03 | PUT は課金プロバイダのトークンを検証し、本人の課金アカウントへ支払方法を登録 / 更新する(生データは保持しない) |
| P-04 | 結果を返却する |

## リクエスト

#### 認証ヘッダ

本 API は利用者セッション(Cookie)による認証を必要とし、登録・更新(PUT)時は CSRF トークンを併用する。

| ヘッダ | 要否 | 説明 |
|---|---|---|
| `Cookie: session=<token>` | 必須 | 利用者セッション |
| `X-CSRF-Token: <token>` | PUT 時必須 | CSRF 対策(支払方法の登録・更新時) |
| `Idempotency-Key: <ULID>` | 任意 | 冪等性(PUT 再送の重複防止) |

> PUT(支払方法の登録・更新)は再認証必須。直近の再認証([API-005](API-005.md#API-005))で得た再認証状態を伴う必要がある。

#### Query Parameters

```json
{}
```

#### Path Parameters

```json
{}
```

#### Request Body

```json
{ "reauthToken": "reauth_...", "paymentToken": "tok_..." }
```

| 項目 | 型 | 説明 |
|----|----|----|
| `reauthToken` | string | 再認証トークン(PUT 時のみ必須・API-005 で取得) |
| `paymentToken` | string | 課金プロバイダが発行した支払方法トークン。PUT 時のみ必須(GET ではボディなし) |

(GET: なし。PUT のみ。`paymentToken` は課金プロバイダ発行トークン)

## レスポンス

**レスポンス(200)**

GET: `{ "registered": true, "brand": "visa", "last4": "4242", "expMonth": 12, "expYear": 2028 }`(未登録時は `{ "registered": false }`)。PUT: `{ "registered": true, "brand": "visa", "last4": "4242" }`。

**GET レスポンス**

| 項目 | 型 | 説明 |
|----|----|----|
| `registered` | boolean | 支払方法が登録済みか(未登録時は本項目のみ `false` を返す) |
| `brand` | string | カードブランド |
| `last4` | string | カード番号下 4 桁 |
| `expMonth` | integer | カード有効期限(月) |
| `expYear` | integer | カード有効期限(年) |

**PUT レスポンス**

| 項目         | 型      | 説明                             |
|--------------|---------|----------------------------------|
| `registered` | boolean | 支払方法が登録済みか             |
| `brand`      | string  | 登録 / 更新したカードブランド    |
| `last4`      | string  | 登録 / 更新したカード番号下 4 桁 |
