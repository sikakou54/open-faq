## 処理概要

質問数の月次上限 ON/OFF・件数・アラート閾値を更新する。

| 処理 ID | 処理内容                                                        |
|---------|-----------------------------------------------------------------|
| P-00    | 再認証トークン(リクエストボディ `reauthToken`)を検証する         |
| P-01    | 認証・認可(オーナー / 当該プロジェクトのメンバー、再認証必須)を検証する |
| P-02    | 入力(上限 ON/OFF・件数・アラート閾値)を検証し正規化する         |
| P-03    | 質問数月次上限(`source='owner'`)を作成 / 更新する               |
| P-04    | 参考課金額を再算出し更新後の値を返却する                        |

## リクエスト

#### 認証ヘッダ

本 API は利用者セッション(Cookie)と CSRF トークンを必要とする状態変更系である。

| ヘッダ | 要否 | 説明 |
|---|---|---|
| `Cookie: session=<token>` | 必須 | 利用者セッション |
| `X-CSRF-Token: <token>` | 必須 | CSRF 対策(状態変更系) |
| `Idempotency-Key: <ULID>` | 任意 | 冪等性(書き込み再送の重複防止) |

> 本操作は再認証必須。直近の再認証([API-005](API-005.md#API-005))で得た再認証状態を伴う必要がある。

#### Query Parameters

```json
{}
```

#### Path Parameters

```json
{ "id": "プロジェクト ID" }
```

| 項目 | 型     | 説明                        |
|------|--------|-----------------------------|
| `id` | string | 対象プロジェクトの ID。必須 |

#### Request Body

```json
{ "reauthToken": "reauth_...", "limitEnabled": true, "limit": 5000, "alertThresholds": [25, 50, 80, 90, 100] }
```

| 項目 | 型 | 説明 |
|----|----|----|
| `reauthToken` | string | 再認証トークン(必須・API-005 で取得) |
| `limitEnabled` | boolean | 月次上限の有効 / 無効。必須 |
| `limit` | integer | 月次上限件数。`limitEnabled=true` の場合は必須(範囲内・1 件刻み)、`false` の場合は `null` |
| `alertThresholds` | array | アラート発火の閾値(%)の配列。`25` / `50` / `80` / `90` / `100` の重複しない値。空配列で通知なし |

例(ON)。例(OFF): `{ "limitEnabled": false, "limit": null, "alertThresholds": [] }`。

バリデーション: `limitEnabled=true` の場合、`limit` は必須かつ 1 件刻みで最小件数・最大件数の範囲内。`alertThresholds` は `25` / `50` / `80` / `90` / `100` の重複しない配列とし、サーバ側で昇順へ正規化する。空配列はアラート通知なし。`limitEnabled=false` の場合は `limit=null`、`alertThresholds=[]` とする。`freeQuota` / `alertEnabled` / `alertFrequency` を指定した場合は未サポート項目として 422。

## レスポンス

**レスポンス(200)**

```json
{ "projectId": "prj_...", "limitEnabled": true, "limit": 5000, "alertThresholds": [25, 50, 80, 90, 100], "yenEquivalent": 2000, "source": "owner", "updatedAt": "..." }
```

| 項目 | 型 | 説明 |
|----|----|----|
| `projectId` | string | 対象プロジェクトの ID |
| `limitEnabled` | boolean | 更新後の月次上限の有効 / 無効 |
| `limit` | integer | 更新後の月次上限件数(OFF 時は `null`) |
| `alertThresholds` | array | 更新後のアラート閾値(%、昇順)の配列 |
| `yenEquivalent` | integer | 上限消化時の参考課金額(OFF 時は `null`) |
| `source` | enum | 上限設定元。`owner` |
| `updatedAt` | string (ISO 8601) | 更新日時 |

## バリデーション

入力値の検証規則は `## リクエスト` の各パラメータ説明を参照する。

`limit` の最小件数 / 最大件数は KV `usage-limit:min` / `usage-limit:max`([`M_PRJ_QUOTA_LIMITS`](../04_database/TBL-009.md#TBL-009) のコード値・区分値節で定義)を正本とし、サーバ側で算出してこの範囲内かを検証する。画面([SCR-027](../../01_frontend/01_screens/SCR-027.md#SCR-027) §4 #2)の入力桁数はこの最大件数を超えない範囲のガード値(暫定桁数)であり、最終的な範囲判定はサーバ側の KV 値に従う。

## エラー

| HTTP ステータス | エラーコード | 内容 |
|----|----|----|
| 401 | `REAUTH_REQUIRED` → [ERR-013](../../05_errors/ERR-013.md#ERR-013) | 再認証トークンが無効または未提示 |
| 422 | (未サポート項目) → [ERR-029](../../05_errors/ERR-029.md#ERR-029) | `freeQuota` / `alertEnabled` / `alertFrequency` を指定 |
| 403 | `PERMISSION_DENIED` → [ERR-030](../../05_errors/ERR-030.md#ERR-030) | 当該プロジェクトに割当のないユーザー |
