# <span id="API-026"></span>API-026: FAQ 作成・更新・削除

> **このページは FAQ 作成・更新・削除 API の仕様を定義します。** FAQ の CRUD と状態(`draft` / `published` / `hidden`)の相互遷移。

| ID | 業務ユースケースID | イベント(画面ID EVT-NN) | テーブルID |
|----|----|----|----|
| API-026 | [UC-024](../../../01_requirements/04_business_usecases/UC-024.md#UC-024) ・ [UC-025](../../../01_requirements/04_business_usecases/UC-025.md#UC-025) | SCR-008 EVT-10 ・ SCR-009 EVT-03 ・ SCR-009 EVT-04 ・ SCR-009 EVT-06 | [TBL-006](../04_database/TBL-006.md#TBL-006) ・ [TBL-017](../04_database/TBL-017.md#TBL-017) ・ [TBL-029](../04_database/TBL-029.md#TBL-029) |

| API名 | エンドポイント | HTTPメソッド |
|----|----|----|
| FAQ 作成・更新・削除 | `/faqs`, `/faqs/{id}` | POST / PATCH / DELETE |

## 処理概要

FAQ の CRUD と状態(`draft` / `published` / `hidden`)の相互遷移。

| 処理 ID | 処理内容 |
|----|----|
| P-01 | 認証・認可(オーナー / 該当プロジェクトの編集権限)を検証する |
| P-02 | 入力値(質問 500 字 / 回答 5,000 字ほか)を検証する。`draft`(下書き)への保存時は `question` / `answer` を任意(空でも保存可)とし、`published`(公開)への遷移時は `question` / `answer` を必須とする |
| P-03 | 更新 / 削除時は `version` による楽観ロックを判定する(不一致は 409) |
| P-04 | FAQ を作成 / 更新 / 論理削除し、指定状態へ遷移する |
| P-05 | `published` へ変更した場合は `publishedAt` を記録する |
| P-06 | 新規作成(POST)時に起点未解決質問(`sourceInquiryId`)が指定された場合は、FAQ 作成に合わせて [H_INQUIRY_FAQ(TBL-029)](../04_database/TBL-029.md#TBL-029) へ移行履歴(元未解決質問 ID・作成 FAQ ID・質問スナップショット・実行者・移行日時)を記録する |
| P-07 | 作成 / 更新後の FAQ を返す |

## リクエスト

#### 認証ヘッダ

本 API は利用者セッション(Cookie)と CSRF トークンを必要とする状態変更系である。

| ヘッダ | 要否 | 説明 |
|---|---|---|
| `Cookie: session=<token>` | 必須 | 利用者セッション |
| `X-CSRF-Token: <token>` | 必須 | CSRF 対策(状態変更系) |
| `Idempotency-Key: <ULID>` | 任意 | 冪等性(書き込み再送の重複防止) |

#### Query Parameters

```json
{}
```

#### Path Parameters

```json
{ "id": "FAQ ID(PATCH / DELETE 時)" }
```

| 項目 | 型 | 説明 |
|----|----|----|
| `id` | string | 対象 FAQ の ID(PATCH / DELETE 時は必須・パス内で指定。POST 時は不要) |

#### Request Body

```json
{
  "question": "...",
  "answer": "...",
  "category": "...",
  "status": "draft | published | hidden",
  "version": 1,
  "sourceInquiryId": "..."
}
```

| 項目 | 型 | 説明 |
|----|----|----|
| `question` | string | 質問文(最大 500 字。`draft` 保存時は任意・空可 / `published` 遷移時は必須) |
| `answer` | string | 回答文(最大 5,000 字。`draft` 保存時は任意・空可 / `published` 遷移時は必須) |
| `category` | string | カテゴリ(任意) |
| `status` | enum | FAQ 状態(必須・`draft` / `published` / `hidden`) |
| `version` | integer | 楽観ロック用バージョン番号(更新 / 削除時は必須・不一致は 409) |
| `sourceInquiryId` | string | 起点未解決質問 ID(任意・POST 時のみ有効。指定時は当該未解決質問からの FAQ 化として移行履歴を記録する) |

Request Body は FAQ 内容(POST は新規)。

## レスポンス

**レスポンス(200 / 201)**

作成 / 更新後の FAQ(`publishedAt` を含む)。

| 項目 | 型 | 説明 |
|----|----|----|
| `id` | string | 作成 / 更新した FAQ の ID |
| `question` | string | 質問文 |
| `answer` | string | 回答文 |
| `category` | string | カテゴリ |
| `status` | enum | FAQ 状態(`draft` / `published` / `hidden`) |
| `version` | integer | 楽観ロック用バージョン番号 |
| `publishedAt` | string (ISO 8601) | 公開日時(`published` 化時に記録、未公開は `null`) |

## バリデーション

入力値の検証規則は `## リクエスト` の各パラメータ説明を参照する。

## エラー

| HTTP ステータス | エラーコード       | 内容                                    |
|-----------------|--------------------|-----------------------------------------|
| 409 | `CONFLICT` → [ERR-023](../../05_errors/ERR-023.md#ERR-023) | 楽観ロック競合(`version` 不一致) |
| 400 | `VALIDATION_ERROR` → [ERR-001](../../05_errors/ERR-001.md#ERR-001) | 質問 500 字 / 回答 5,000 字超過ほか |
