# <span id="API-026"></span>API-026: FAQ 作成・更新・削除

> **このページは FAQ 作成・更新・削除 API の契約を定義します。** FAQ の CRUD と状態(`draft` / `published` / `hidden`)の相互遷移。

## 項目

| 項目 | 内容 |
|---|---|
| API ID | `API-026` |
| API名 | FAQ 作成・更新・削除 |
| トレーサビリティID | [TR-025](../../00_traceability/index.md#TR-025) ・ [TR-026](../../00_traceability/index.md#TR-026) |
| エンドポイント | `/faqs`, `/faqs/{id}` |
| HTTPメソッド | POST / PATCH / DELETE |
| 認証 | Cookie + CSRF |
| 認可 | オーナー / 当該プロジェクトのメンバー |

## 処理概要

FAQ の CRUD と状態(`draft` / `published` / `hidden`)の相互遷移。

| 処理 ID | 処理内容 |
|----|----|
| P-01 | 認証・認可(オーナー / 該当プロジェクトの編集権限)を検証する |
| P-02 | 入力値(質問 500 字 / 回答 5,000 字ほか)を検証する |
| P-03 | 更新 / 削除時は `version` による楽観ロックを判定する(不一致は 409) |
| P-04 | FAQ を作成 / 更新 / 論理削除し、指定状態へ遷移する(`sortOrder` 指定時は表示順を保存する) |
| P-05 | `published` へ変更した場合は `publishedAt` を記録する |
| P-06 | 作成 / 更新後の FAQ を返す |

## リクエスト

#### 認証ヘッダ

本 API は利用者セッション(Cookie)と CSRF トークンを必要とする状態変更系である。

| ヘッダ | 要否 | 説明 |
|---|---|---|
| `Cookie: session=<token>` | 必須 | 利用者セッション |
| `X-CSRF-Token: <token>` | 必須 | CSRF 対策(状態変更系) |
| `Idempotency-Key: <ULID>` | 任意 | 冪等性(書き込み再送の重複防止) |

#### Query Parameters

```json
{}
```

#### Path Parameters

```json
{ "id": "FAQ ID(PATCH / DELETE 時)" }
```

| 項目 | 型 | 説明 |
|----|----|----|
| `id` | string | 対象 FAQ の ID(PATCH / DELETE 時は必須・パス内で指定。POST 時は不要) |

#### Request Body

```json
{
  "question": "...",
  "answer": "...",
  "category": "...",
  "status": "draft | published | hidden",
  "sortOrder": 3,
  "version": 1
}
```

| 項目 | 型 | 説明 |
|----|----|----|
| `question` | string | 質問文(必須・最大 500 字) |
| `answer` | string | 回答文(必須・最大 5,000 字) |
| `category` | string | カテゴリ(任意) |
| `status` | enum | FAQ 状態(必須・`draft` / `published` / `hidden`) |
| `sortOrder` | integer | プロジェクト内での表示順(任意・保存時に `M_FAQS.sort_order` へ反映) |
| `version` | integer | 楽観ロック用バージョン番号(更新 / 削除時は必須・不一致は 409) |

Request Body は FAQ 内容(POST は新規)。

## レスポンス

**レスポンス(200 / 201)**

作成 / 更新後の FAQ(`publishedAt` を含む)。

| 項目 | 型 | 説明 |
|----|----|----|
| `id` | string | 作成 / 更新した FAQ の ID |
| `question` | string | 質問文 |
| `answer` | string | 回答文 |
| `category` | string | カテゴリ |
| `status` | enum | FAQ 状態(`draft` / `published` / `hidden`) |
| `sortOrder` | integer | プロジェクト内での表示順(`M_FAQS.sort_order` 由来) |
| `version` | integer | 楽観ロック用バージョン番号 |
| `publishedAt` | string (ISO 8601) | 公開日時(`published` 化時に記録、未公開は `null`) |

## バリデーション

入力値の検証規則は `## リクエスト` の各パラメータ説明を参照する。

## エラー

| HTTP ステータス | エラーコード       | 内容                                    |
|-----------------|--------------------|-----------------------------------------|
| 409 | `CONFLICT` → [ERR-025](../../05_errors/ERR-025.md#ERR-025) | 楽観ロック競合(`version` 不一致) |
| 400 | `VALIDATION_ERROR` → [ERR-001](../../05_errors/ERR-001.md#ERR-001) | 質問 500 字 / 回答 5,000 字超過ほか |
