### <span id="コード値区分値"></span>コード値・区分値

| カラム名  | 値               | 意味                         |
|-----------|------------------|------------------------------|
| `purpose` | `email_verify`   | メール確認                   |
| `purpose` | `password_reset` | パスワード再設定             |
| `purpose` | `activation`     | メンバーアカウント有効化     |
| `purpose` | `contact_verify` | プロジェクト連絡先メール確認 |

### <span id="meta"></span>`meta` JSON 構造(用途別)

`meta`(JSON 文字列)は `purpose` ごとに必要な対象情報を保持する。主体ユーザーが存在する用途は `user_id` で本人を特定し、`meta` には付随情報のみ保持する。`contact_verify` は主体ユーザーが無く(`user_id` は NULL)、対象プロジェクト・対象メールアドレスを `meta` で保持する。

| `purpose` | `meta` 構造(キー: 型 / 説明) | 主な発行元 |
|----|----|----|
| `email_verify` | `{}` または `{ "email": string }`(確認対象メール。本人は `user_id` で特定。設計値) | アカウント登録・メール確認 |
| `password_reset` | `{}`(本人は `user_id` で特定。付随情報なし。設計値) | パスワード再設定要求 |
| `activation` | `{ "projectId": string(ULID 26), "invitedEmail": string(最大254) }`(招待先プロジェクトと招待メール。被招待ユーザーは `user_id` で特定) | メンバー招待 [API-021](../03_apis/API-021.md#API-021) / 招待再送 [API-024](../03_apis/API-024.md#API-024) |
| `contact_verify` | `{ "projectId": string(ULID 26), "contactEmail": string(最大254) }`(連絡先確認の対象プロジェクトと確認対象メール。`user_id` は NULL) | 連絡先確認再送 [API-011](../03_apis/API-011.md#API-011) |

> [!NOTE]
> `meta` の各キー名・有無は用途ごとの設計値であり、詳細設計で確定する。共通として ULID は 26 文字、メールアドレスは最大 254 文字(§6.1・RFC 準拠の設計値)とする。トークン本体は `token_hash`(ハッシュ済)で保持し、`meta` に平文トークンを保持しない。
