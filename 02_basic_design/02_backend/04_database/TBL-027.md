### <span id="metadata"></span>`metadata` JSON 構造(監査付随情報)

`metadata`(JSON 文字列)は、操作の追跡に必要な付随情報を保持する。操作対象は `target_type` / `target_id` カラムで特定し、`metadata` には変更内容の要約・補足のみを保持する(以下は設計値であり、詳細設計で `action` ごとに確定する)。

| キー | 型 | 必須 | 説明 |
|----|----|----|----|
| `changes` | object | 任意 | 変更されたフィールドの変更前後要約。`{ "<field>": { "before": any, "after": any } }` 形式 |
| `reason` | string | 任意 | 操作理由・補足メモ |
| `request_id` | string | 任意 | 相関のためのリクエスト ID |
| `extra` | object | 任意 | `action` 固有の追加情報(例: 対象件数・関連 ID) |

> [!NOTE]
> 個人情報・秘匿値(平文パスワード・トークン・メール平文等)は `metadata` に保持しない。変更前後の要約に機微値を含める場合はマスク済みの値とする(設計値)。`changes` のキー名・粒度は `action` ごとに詳細設計で確定する。
