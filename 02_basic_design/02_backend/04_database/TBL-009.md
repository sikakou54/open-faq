### <span id="コード値区分値"></span>コード値・区分値

| カラム名 | 値 | 意味 | デフォルト推奨値(プロジェクト単位) |
|----|----|----|----|
| `resource_kind` | `q_monthly_limit` | 質問数の月次上限件数 | 上限=KV `usage-limit:default:question` / 無料枠=KV `usage-limit:free-default:question`(初期 1,000) |
| `resource_kind` | `faq_monthly_limit` | FAQ 件数の無料枠 | 無料枠=KV `usage-limit:free-default:faq`(初期 100) |

質問数上限の最小 / 最大件数は KV `usage-limit:min` / `usage-limit:max` を入力ガードに用いる。FAQには上限値を保持しない。KV ミラーは `usage-limit:<contract_id>:<project_id>`(30s TTL)。

### <span id="alert_thresholds"></span>`alert_thresholds` JSON 構造(アラート閾値)

`alert_thresholds`(JSON 文字列)は、質問数上限に対する到達率(パーセント)のアラート閾値を保持する。[RULE-014 質問数アラート閾値](../../../01_requirements/01_business_requirement/08_rule.md#RULE-014) に従い、複数選択した閾値の当月初回到達時に通知する。

| 項目 | 規約 |
|----|----|
| 全体の型 | JSON 配列。例 `[50, 80, 100]` |
| 要素の型 | 整数(INTEGER パーセント)。小数・文字列は不可 |
| 要素の許可値 | `25` / `50` / `80` / `90` / `100`(RULE-014 の閾値集合) |
| 要素の重複 | 不可(重複排除して保持) |
| 並び | 昇順(API 層で正規化) |
| 最大要素数 | 5(許可値が 5 種のため上限 5。設計値) |
| 既定値 | `[]`(アラートなし) |

> [!NOTE]
> 100% 到達時は [RULE-014](../../../01_requirements/01_business_requirement/08_rule.md#RULE-014) と別に新規質問受付を停止する。`alert_thresholds` は通知の発火点のみを表し、受付停止判定(100%)・追加アラート(125%)とは独立に保持する。要素の許可値・重複排除・昇順化は API 層で保証する。
