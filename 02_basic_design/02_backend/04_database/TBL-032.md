# <span id="TBL-032"></span>T_BILLING_WEBHOOK_LOG(課金Webhook受信ログ)

課金プロバイダから受信した決済・課金アカウントの課金状態通知を、検証結果・取込状態とともに記録します。重複受信の検出と取込失敗の再処理に用います。

| ID |
|----|
| TBL-032 |

| No | 論理名 | 物理名 | データ型 | 桁数 | NULL | PK | FK | UNIQUE | DEFAULT | 制約 |
|---:|----|----|----|---:|----|----|----|----|----|----|
| 1 | ID | `id` | TEXT | 26 | NO | ○ |  |  |  |  |
| 2 | 課金プロバイダ | `provider` | TEXT | 20 | NO |  |  |  | `'stripe'` | `provider IN ('stripe')`。取りうる値は「コード値・区分値」を参照 |
| 3 | プロバイダ側イベント ID | `event_id` | TEXT | 255 | NO |  |  | ○ |  | 重複排除キー(プロバイダ採番。Stripe 例 `evt_...`。設計値) |

### <span id="コード値区分値"></span>コード値・区分値

| カラム名 | 値 | 意味 |
|----------|------|------|
| `status` | `received` | [状態モデル §8.3](../../08_state-model.md#83-webhook取込状態) |
| `status` | `processed` | [状態モデル §8.3](../../08_state-model.md#83-webhook取込状態) |
| `status` | `failed` | [状態モデル §8.3](../../08_state-model.md#83-webhook取込状態) |
| `status` | `skipped` | [状態モデル §8.3](../../08_state-model.md#83-webhook取込状態) |
| `signature_valid` | `1` | 署名検証成功 |
| `signature_valid` | `0` | 署名検証失敗(不正受信) |
| `provider` | `stripe` | Stripe(課金プロバイダ。現状唯一) |

> [!NOTE]
> 課金プロバイダは現状 Stripe のみのため `provider` の許可値は `stripe` のみとする(設計値)。将来プロバイダを追加する場合は許可値を拡張する。
