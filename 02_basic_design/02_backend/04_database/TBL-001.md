# <span id="TBL-001"></span>M_USER(ユーザーマスタ)

オーナー・メンバーを含む**すべてのユーザー**の認証情報(メール / パスワード / 氏名)を一元保持するユーザーマスタです。オーナーかどうかの判定は、対象プロジェクトの `M_PROJECTS.owner_user_id` 参照で行い(立場はプロジェクトごとに決まる)、本テーブルには所有・課金属性を持ちません。新規サインアップ直後は `status='pending_activation'`(メール確認待ち)とし、メール確認の完了で `active` に確定します。



| ID | 物理名 | 論理名 | 区分 |
|----|----|----|----|
| TBL-001 | `M_USER` | ユーザーマスタ | マスタ |

### <span id="カラム定義"></span>カラム定義

| No | 論理名 | 物理名 | データ型 | 桁数 | NULL | PK | FK | UNIQUE | DEFAULT | 制約 |
|----|----|----|----|----|----|----|----|----|----|----|
| 1 | ユーザーID | `id` | TEXT | 26 | NOT NULL | ○ |  |  |  |  |
| 2 | メールアドレス | `email` | TEXT | 254 | NOT NULL |  |  | ○ |  | `length(email) <= 254`(RFC 5321 準拠の設計値) |
| 3 | パスワードハッシュ | `password_hash` | TEXT | 255 | NULL可 |  |  |  |  | Argon2id のエンコード文字列(最大 255 文字。§6.1 共通データ規約準拠)。平文は保持しない。招待直後(`pending_activation`)は未設定 |
| 4 | 氏名 | `name` | TEXT | 100 | NULL可 |  |  |  |  | `name IS NULL OR length(name) BETWEEN 1 AND 100`(設計値)。招待直後は未設定 |
| 5 | メール確認日時 | `email_verified_at` | TEXT |  | NULL可 |  |  |  |  |  |
| 6 | 状態 | `status` | TEXT |  | NOT NULL |  |  |  | `'active'` | `status IN ('pending_activation','active','suspended','deleted')` |
| 7 | 有効フラグ | `valid` | INTEGER |  | NOT NULL |  |  |  | `1` | `valid IN (0,1)` |
| 8 | 作成日時 | `created_at` | TEXT |  | NOT NULL |  |  |  |  |  |
| 9 | 更新日時 | `updated_at` | TEXT |  | NOT NULL |  |  |  |  |  |

### <span id="インデックス"></span>インデックス

### <span id="コード値"></span>コード値・区分値

| カラム   | 値                   | 意味                       |
|----------|----------------------|----------------------------|

| `status` | `active`             | 有効                       |
| `status` | `suspended`          | 停止                       |
| `status` | `deleted`            | 論理削除                   |
