# <span id="TBL-002"></span>M_BILLING_ACCOUNT(課金アカウントマスタ)

**ユーザー単位の課金アカウントを管理する課金アカウントマスタ**です。`user_id` ごとに 1 件(UNIQUE)で、当該ユーザーが**作成した(オーナーである)プロジェクト**の課金・サブスクリプション・支払方法・請求・課金状態を束ねます。請求は本アカウント単位(オーナー宛)に集約し、各プロジェクトの内訳は明細で持ちます。**所有・ロール判定には用いません**(プロジェクトの所有者は `M_PROJECTS.owner_user_id`、メンバーは `M_PRJ_USERS` で表す)。支払方法はユーザー単位で本アカウントに紐づけ、連絡先・重要通知の宛先はオーナーの `M_USER.email` を参照します。組織名は保持しません。



| ID | 物理名 | 論理名 | 区分 |
|----|----|----|----|
| TBL-002 | `M_BILLING_ACCOUNT` | 課金アカウントマスタ | マスタ |

### <span id="コード値"></span>コード値・区分値

| カラム   | 値                | 意味                 |
|----------|-------------------|----------------------|
| `status` | `active`          | 有効                 |
| `status` | `suspended`       | 停止(決済失敗等)。当該ユーザーが作成したプロジェクトのウィジェット応答を停止 |
| `status` | `withdrawn`       | 退会済み(アカウント退会。運用データは削除済み、請求データを 7 年保持。本人は請求閲覧のみ可) |
| `status` | `deleted`         | 削除済(7 年の保持期間経過後の最終状態=トムストーン。請求等の従属データを物理削除し、識別子は再利用しない(NFR-051)・ログイン不可)。運用データの論理削除(`deleted_at` 起点・猶予 90 日で物理削除)を表す [TBL-004](TBL-004.md#TBL-004) / [TBL-006](TBL-006.md#TBL-006) の `deleted` とは意味・保持期間が異なる |
| `data_deletion_mode` | `NULL` | 未退会(通常運用中)。退会していないアカウントは設定しない |
| `data_deletion_mode` | `immediate` | 即時退会時のデータ削除モード(設計値)。退会成立と同時に運用データを速やかに削除し、アカウント・請求データは退会日(`withdrawn_at`)起点に 7 年保持後に物理削除する。退会記録は [`T_WITHDRAW_REQ`](TBL-023.md#TBL-023) を正本とする |

> [!NOTE]
> **`data_deletion_mode` は設計値。** 現行設計の退会は「即時退会・運用データ速やか削除・請求データ 7 年保持」の単一モードのみのため、取りうる値は `immediate`(退会時)/ `NULL`(未退会)とする。退会の遷移条件は [課金・請求設計 §5.2](../../05_billing-design.md#52-退会から削除へ)、アカウント状態は [状態モデル §2](../../08_state-model.md#2-課金アカウント状態) を正本とする。
