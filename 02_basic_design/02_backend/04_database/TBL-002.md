# <span id="TBL-002"></span>M_BILLING_ACCOUNT(課金アカウントマスタ)

**ユーザー単位の課金アカウントを管理する課金アカウントマスタ**です。`user_id` ごとに 1 件(UNIQUE)で、当該ユーザーが**作成した(オーナーである)プロジェクト**の課金・サブスクリプション・支払方法・請求・課金状態を束ねます。請求は本アカウント単位(オーナー宛)に集約し、各プロジェクトの内訳は明細で持ちます。**所有・ロール判定には用いません**(プロジェクトの所有者は `M_PROJECTS.owner_user_id`、メンバーは `M_PRJ_USERS` で表す)。支払方法はユーザー単位で本アカウントに紐づけ、連絡先・重要通知の宛先はオーナーの `M_USER.email` を参照します。組織名は保持しません。

### <span id="コード値"></span>コード値・区分値

| カラム   | 値                | 意味 |
|----------|-------------------|------|
| `status` | `active`          | [状態モデル §2](../../08_state-model.md#2-課金アカウント状態) |
| `status` | `suspended`       | [状態モデル §2](../../08_state-model.md#2-課金アカウント状態) |
| `status` | `withdrawn`       | [状態モデル §2](../../08_state-model.md#2-課金アカウント状態) |

| `data_deletion_mode` | `NULL` | 未退会(通常運用中)。退会していないアカウントは設定しない |
| `data_deletion_mode` | `immediate` | 即時退会時のデータ削除モード。保持期間・削除猶予の具体値は [システム仕様書 §4](../../07_system-spec.md#4-データ保持期間削除猶予) を参照し、退会記録は [`T_WITHDRAW_REQ`](TBL-023.md#TBL-023) を正本とする |

> [!NOTE]
> **`data_deletion_mode` は設計値。** 現行設計の退会は「即時退会・運用データ速やか削除」の単一モードのみのため、取りうる値は `immediate`(退会時)/ `NULL`(未退会)とする。保持期間の具体値は [システム仕様書 §4](../../07_system-spec.md#4-データ保持期間削除猶予) を参照する。退会の遷移条件は [課金・請求設計 §5.2](../../05_billing-design.md#52-退会から削除へ)、アカウント状態は [状態モデル §2](../../08_state-model.md#2-課金アカウント状態) を正本とする。
