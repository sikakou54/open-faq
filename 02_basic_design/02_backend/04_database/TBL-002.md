# <span id="TBL-002"></span>M_CONTRACT(契約マスタ)

マスタ 新規

**契約を管理する契約マスタ**です。`id` が契約境界キー本体であり、`M_PROJECTS` など契約スコープのテーブルはこの契約を親に持ちます。**オーナー判定**は `user_id` と一致するユーザーを当該契約のオーナーとみなします(1 契約 = 1 オーナー)。組織名(`organization`)を保持し、連絡先メール・重要通知の宛先はオーナーの `M_USER.email` を参照します(タイムゾーン設定は持ちません)。

### <span id="インデックス"></span>インデックス

| No | インデックス名 | 対象カラム | UNIQUE | 用途 |
|----|----|----|----|----|
| 1 | `idx_contract_owner` | `user_id` |  | オーナーユーザーからの契約逆引き(オーナー判定) |
| 2 | `idx_status` | `status` |  | 契約状態での抽出(サスペンション / 退会済み・保持期間経過の削除バッチ) |

### <span id="コード値"></span>コード値・区分値

| カラム   | 値                | 意味                 |
|----------|-------------------|----------------------|
| `status` | `active`          | 有効                 |
| `status` | `suspended`       | 停止(決済失敗等)     |
| `status` | `withdrawn`       | 退会済み(即時退会。運用データは削除済み、契約・請求データを 7 年保持。オーナーは請求閲覧のみ可) |
| `status` | `deleted`         | 削除済(7 年の保持期間経過後。請求・利用者等の従属データを物理削除し、契約識別子は再利用しない(NFR-051)・ログイン不可) |
