# <span id="SEV-028"></span>SEV-028: 滞在中の定期的な未読件数最新化

> **システム処理 SYS-015 のイベント「滞在中の定期的な未読件数最新化」を定義します。** 対応システム・対応業務UC・トリガー・呼出API・連携先と処理内容を記述します。

*種別 システムイベント設計 ・ ステータス ドラフト*

## 項目

| 項目 | 内容 |
|---|---|
| システムイベントID | `SEV-028` |
| イベント名 | 滞在中の定期的な未読件数最新化 |
| 対応システムID | [SYS-015](../01_system/SYS-015.md#SYS-015) |
| 対応業務UC | [UC-088](../../../01_requirements/04_business_usecases/UC-088.md#UC-088) |
| トリガー | 画面滞在中の定期取得タイミング |
| 呼出API | [API-051](../03_apis/API-051.md#API-051) |
| 連携先 | 未読件数の表示反映、取得失敗時の直前件数維持と次回再取得 |

## 処理

処理内容の正本は対応システム [SYS-015](../01_system/SYS-015.md#SYS-015) の §6 システムイベント一覧 を参照(二重管理を避けるため本ページには再掲しない)。本イベントは、画面滞在中も過大な負荷を生まない適度な間隔で未読お知らせ件数を取得し直して最新化し、取得に失敗した場合は直前の件数を維持して次の取得タイミングで再取得する役割を担う。
