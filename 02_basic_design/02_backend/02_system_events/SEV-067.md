# <span id="SEV-067"></span>SEV-067: 取込失敗通知の再処理

> **システム処理 SYS-035 のイベント「取込失敗通知の再処理」を定義します。** 対応システム・対応業務UC・トリガー・呼出API・連携先と処理内容を記述します。

*種別 システムイベント設計 ・ ステータス ドラフト*

## 項目

| 項目 | 内容 |
|---|---|
| システムイベントID | `SEV-067` |
| イベント名 | 取込失敗通知の再処理 |
| 対応システムID | [SYS-035](../01_system/SYS-035.md#SYS-035) |
| 対応業務UC | [UC-061](../../../01_requirements/04_business_usecases/UC-061.md#UC-061) |
| トリガー | 定期スケジュールによる再処理バッチの起動 |
| 呼出API | — |
| 連携先 | 後段イベント(再処理上限到達のエスカレーション) |

## 処理

処理内容の正本は対応システム [SYS-035](../01_system/SYS-035.md#SYS-035) の §6 システムイベント一覧 を参照(二重管理を避けるため本ページには再掲しない)。本イベントは取込失敗として記録された通知を再処理対象として抽出し、再取込して契約・決済状態へ再反映し、成功分を取込完了として更新する役割を担う。
