# <span id="SEV-004"></span>SEV-004: オーナー通知

> **システム処理 SYS-002 のイベント「オーナー通知」を定義します。** 対応システム・対応業務UC・トリガー・呼出API・連携先と処理内容を記述します。

*種別 システムイベント設計 ・ ステータス ドラフト*

## 項目

| 項目 | 内容 |
|---|---|
| システムイベントID | `SEV-004` |
| イベント名 | オーナー通知 |
| 対応システムID | [SYS-002](../01_system/SYS-002.md#SYS-002) |
| 対応業務UC | [UC-050](../../../01_requirements/04_business_usecases/UC-050.md#UC-050) |
| トリガー | 前段イベント([SEV-003](SEV-003.md#SEV-003) で急増を検知) |
| 呼出API | — |
| 連携先 | オーナーへの受信箱通知 / 通知ログ / 再送対象記録(例外時) |

## 処理

処理内容の正本は対応システム [SYS-002](../01_system/SYS-002.md#SYS-002) の §6 システムイベント一覧 を参照(二重管理を避けるため本ページには再掲しない)。本イベントは急増に該当する契約のオーナーへ通知を届けて結果を記録し、通知の生成または送信に失敗した場合は再送対象として記録する役割を担う。
