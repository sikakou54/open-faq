# <span id="SEV-018"></span>SEV-018: 通知送信の抑制

> **システム処理 SYS-009 のイベント「通知送信の抑制」を定義します。** 対応システム・対応業務UC・トリガー・呼出API・連携先と処理内容を記述します。

*種別 システムイベント設計 ・ ステータス ドラフト*

## 項目

| 項目 | 内容 |
|---|---|
| システムイベントID | `SEV-018` |
| イベント名 | 通知送信の抑制 |
| 対応システムID | [SYS-009](../01_system/SYS-009.md#SYS-009) |
| 対応業務UC | [UC-070](../../../01_requirements/04_business_usecases/UC-070.md#UC-070) |
| トリガー | 送信品質の評価で、いずれかの指標が許容水準を超えて悪化したと判定されたこと |
| 呼出API | [API-058](../03_apis/API-058.md#API-058) |
| 連携先 | メール配信IF(以後の送信抑制)、管理者への通知 |

## 処理

処理内容の正本は対応システム [SYS-009](../01_system/SYS-009.md#SYS-009) の §6 システムイベント一覧 を参照(二重管理を避けるため本ページには再掲しない)。本イベントは、許容水準を超えて悪化した契約・プロジェクト宛の通知送信を抑制状態にし、抑制の有無と対象を記録したうえで、必要に応じて管理者へ知らせる役割を担う。
