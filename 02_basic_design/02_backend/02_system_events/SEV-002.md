# <span id="SEV-002"></span>SEV-002: アラート通知生成・記録

> **システム処理 SYS-001 のイベント「アラート通知生成・記録」を定義します。** 対応システム・対応業務UC・トリガー・呼出API・連携先と処理内容を記述します。

*種別 システムイベント設計 ・ ステータス ドラフト*

## 項目

| 項目 | 内容 |
|---|---|
| システムイベントID | `SEV-002` |
| イベント名 | アラート通知生成・記録 |
| 対応システムID | [SYS-001](../01_system/SYS-001.md#SYS-001) |
| 対応業務UC | [UC-049](../../../01_requirements/04_business_usecases/UC-049.md#UC-049) |
| トリガー | 前段イベント(メンバー数集計・しきい値評価)での接近・急増の検知 |
| 呼出API | — |
| 連携先 | 受信箱アラート通知 / 通知ログ(再送対象を含む) |

## 処理

処理内容の正本は対応システム [SYS-001](../01_system/SYS-001.md#SYS-001) の §6 システムイベント一覧 を参照(二重管理を避けるため本ページには再掲しない)。本イベントは検知した契約のアカウント利用者(オーナー・関係するメンバー)を特定して受信箱アラートを生成し、検知・通知の結果を運用記録として残す(生成・送信に失敗した場合は再送対象として記録する)役割を担う。
