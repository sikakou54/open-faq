# <span id="SEV-017"></span>SEV-017: 送信品質の評価

> **システム処理 SYS-009 のイベント「送信品質の評価」を定義します。** 対応システム・対応業務UC・トリガー・呼出API・連携先と処理内容を記述します。

*種別 システムイベント設計 ・ ステータス ドラフト*

## 項目

| 項目 | 内容 |
|---|---|
| システムイベントID | `SEV-017` |
| イベント名 | 送信品質の評価 |
| 対応システムID | [SYS-009](../01_system/SYS-009.md#SYS-009) |
| 対応業務UC | [UC-070](../../../01_requirements/04_business_usecases/UC-070.md#UC-070) |
| トリガー | 通知送信の発生、または定期的な品質監視のタイミング |
| 呼出API | [API-059](../03_apis/API-059.md#API-059) |
| 連携先 | 後段処理(送信品質の悪化判定) |

## 処理

処理内容の正本は対応システム [SYS-009](../01_system/SYS-009.md#SYS-009) の §6 システムイベント一覧 を参照(二重管理を避けるため本ページには再掲しない)。本イベントは、配信事業者からの配信結果をもとに契約・プロジェクト単位で直近の送信レート・バウンス率・苦情率を集計し、各指標を許容水準と照合して品質の悪化を判定する役割を担う。
