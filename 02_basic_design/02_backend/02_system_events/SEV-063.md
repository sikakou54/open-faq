# <span id="SEV-063"></span>SEV-063: 監査ログ整合性検証

> **システム処理 SYS-033 のイベント「監査ログ整合性検証」を定義します。** 対応システム・対応業務UC・トリガー・呼出API・連携先と処理内容を記述します。

*種別 システムイベント設計 ・ ステータス ドラフト*

## 項目

| 項目 | 内容 |
|---|---|
| システムイベントID | `SEV-063` |
| イベント名 | 監査ログ整合性検証 |
| 対応システムID | [SYS-033](../01_system/SYS-033.md#SYS-033) |
| 対応業務UC | [UC-075](../../../01_requirements/04_business_usecases/UC-075.md#UC-075) |
| トリガー | 日次の定期処理(スケジューラ起動) |
| 呼出API | — |
| 連携先 | 後段イベント(整合性違反通知) |

## 処理

処理内容の正本は対応システム [SYS-033](../01_system/SYS-033.md#SYS-033) の §6 システムイベント一覧 を参照(二重管理を避けるため本ページには再掲しない)。本イベントは保持されている監査ログを全件走査し、各記録の改ざん検知情報の整合性(改ざん・欠落の疑い)を検証して、違反を検出した対象を後段の通知イベントへ引き渡す役割を担う。
