# <span id="SEV-020"></span>SEV-020: 上限超過リクエストの抑制

> **システム処理 SYS-010 のイベント「上限超過リクエストの抑制」を定義します。** 対応システム・対応業務UC・トリガー・呼出API・連携先と処理内容を記述します。

*種別 システムイベント設計 ・ ステータス ドラフト*

## 項目

| 項目 | 内容 |
|---|---|
| システムイベントID | `SEV-020` |
| イベント名 | 上限超過リクエストの抑制 |
| 対応システムID | [SYS-010](../01_system/SYS-010.md#SYS-010) |
| 対応業務UC | [UC-076](../../../01_requirements/04_business_usecases/UC-076.md#UC-076) |
| トリガー | 契約単位レート制限の評価で上限超過と判定された場合 |
| 呼出API | — |
| 連携先 | アクセス状況の記録(濫用検知への連携) |

## 処理

処理内容の正本は対応システム [SYS-010](../01_system/SYS-010.md#SYS-010) の §6 システムイベント一覧 を参照(二重管理を避けるため本ページには再掲しない)。本イベントは、上限を超過したリクエストを抑制して後段の業務処理へ渡さず、超過抑制をアクセス状況として記録し、サービスの過負荷を防いで濫用検知につなげる役割を担う。
