# <span id="SEV-010"></span>SEV-010: 受信検証

> **システム処理 SYS-006 のイベント「受信検証」を定義します。** 対応システム・対応業務UC・トリガー・呼出API・連携先と処理内容を記述します。

*種別 システムイベント設計 ・ ステータス ドラフト*

## 項目

| 項目 | 内容 |
|---|---|
| システムイベントID | `SEV-010` |
| イベント名 | 受信検証 |
| 対応システムID | [SYS-006](../01_system/SYS-006.md#SYS-006) |
| 対応業務UC | [UC-061](../../../01_requirements/04_business_usecases/UC-061.md#UC-061) |
| トリガー | 課金プロバイダからの決済・契約状態通知の受信 |
| 呼出API | [API-060](../03_apis/API-060.md#API-060) |
| 連携先 | 受信ログ(取込前段) |

## 処理

処理内容の正本は対応システム [SYS-006](../01_system/SYS-006.md#SYS-006) の §6 システムイベント一覧 を参照(二重管理を避けるため本ページには再掲しない)。本イベントは、受信した通知の送信元の正当性を検証し、なりすましの通知を取り込まないための入口判定を担う。
