# <span id="SEV-051"></span>SEV-051: 通知再送

> **システム処理 SYS-027 のイベント「通知再送」を定義します。** 対応システム・対応業務UC・トリガー・呼出API・連携先と処理内容を記述します。

*種別 システムイベント設計 ・ ステータス ドラフト*

## 項目

| 項目 | 内容 |
|---|---|
| システムイベントID | `SEV-051` |
| イベント名 | 通知再送 |
| 対応システムID | [SYS-027](../01_system/SYS-027.md#SYS-027) |
| 対応業務UC | [UC-067](../../../01_requirements/04_business_usecases/UC-067.md#UC-067) |
| トリガー | 定期起動の再送処理で再送可(抑制外 かつ 上限未満)と判定された対象 |
| 呼出API | [API-040](../03_apis/API-040.md#API-040) ・ [API-058](../03_apis/API-058.md#API-058) |
| 連携先 | 後段イベント(配信状態更新) |

## 処理

処理内容の正本は対応システム [SYS-027](../01_system/SYS-027.md#SYS-027) の §6 システムイベント一覧 を参照(二重管理を避けるため本ページには再掲しない)。本イベントは抑制リスト該当外かつ再送回数が上限未満の配信失敗通知を再送し、再送結果を後段の配信状態更新イベントへ引き渡す役割を担う。
