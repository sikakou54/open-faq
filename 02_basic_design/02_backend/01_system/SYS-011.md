# <span id="SYS-011"></span>SYS-011: 外部露出箇所の入力サニタイズ

> **このページは、ウィジェット利用者の入力を含む通知を生成する際、外部に露出する箇所(件名・送信元情報など)へ入力文字列をそのまま使わず安全な値で構成する横断的なシステム処理 SYS-011 を定義します。** 処理概要 / 処理フロー図 / 入出力 / 処理項目定義 / 入出力一覧 / システムイベント一覧 の 6 セクションで記述します。

*種別 システム設計 ・ 優先度 P0 ・ ステータス ドラフト*

## 1. 処理概要

ウィジェット利用者の入力を含む通知を組み立てる際、外部に露出する箇所(メール件名・送信元情報など)には入力文字列をそのまま流さず、安全な固定値またはサニタイズ済みの値で構成する。これにより、入力に紛れ込ませた不正な文字列によるスパム埋め込みやなりすましを防ぐ。特定の機能ではなく通知生成全体に適用される横断的なガード処理である。本処理は、ウィジェット利用者の入力を含む通知生成時に適用する(お知らせ本文にエンドユーザー入力を含めない方針 [NFR-018](../../../01_requirements/03_non_functional_requirement/07_nfr.md#NFR-018) と整合)。

「不正・危険」とみなして除去・無害化する対象は次のとおり(いずれも設計値・詳細は詳細設計で確定。具体のコードポイント範囲・許容文字集合・各箇所の長さ上限値などの確定は詳細設計に委ねる):

- 制御文字(C0 制御文字 `U+0000`–`U+001F` / C1 制御文字 `U+0080`–`U+009F`。改行など正当に必要な箇所を除く)の除去。
- HTML 特殊文字(`<` `>` `&` `"` `'`)のエスケープ(HTML 文脈で露出する箇所)。
- `<script>` などの能動コンテンツ(スクリプト・イベントハンドラ属性・`javascript:` 等のスキーム)の除去。
- 長さ上限を超過した入力の切り詰め(露出箇所ごとに定める上限。例: 件名はメールヘッダの実務上限に収める)。
- 件名・送信元などヘッダ相当の箇所では、改行(`CR` / `LF`)注入によるヘッダインジェクションを除去する。

| `SYS-011` | 外部露出箇所の入力サニタイズ | guard | ウィジェット利用者の入力を含む通知の生成時 | 外部露出箇所を特定し、入力文字列を安全な固定値・サニタイズ済みの値へ置き換えて通知を構成する |

## 2. 処理フロー図

```mermaid
flowchart TB
  A[通知の生成を開始] --> B[外部に露出する箇所を特定]
  B --> C{入力文字列を<br/>外部露出箇所で使うか}
  C -- 使わない --> D[安全な固定値で構成]
  C -- 使う --> E{不正・危険な<br/>内容を含むか}
  E -- 含む --> F[除去・無害化]
  E -- 含まない --> G[サニタイズ済みの値に整形]

  D --> H[サニタイズ済みの値で通知を構成]
  G --> H
  H --> I[通知を生成し配信へ引き渡す]
```

| メール配信IF | 安全な値で構成した通知の配信を委ねる | API | 出力 | — | [API-058](../03_apis/API-058.md#API-058) |

## 詳細設計への移管候補

- 露出箇所ごとの長さ上限の確定値と、出力エンコードの具体方式(露出先のメディアタイプ別)。基本設計では除去・無害化の対象(制御文字 C0/C1 除去・HTML 特殊文字エスケープ・能動コンテンツ除去・長さ上限切り詰め・改行注入除去。設計値)を定める。
- 露出箇所別(件名・送信元情報など)の固定値の採用範囲と、サニタイズ済み値の許容文字集合の細目。
