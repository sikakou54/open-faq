# <span id="SYS-006"></span>SYS-006: 課金プロバイダ通知の受信・検証・取込

> **このページは、課金プロバイダから届く決済・契約状態の通知を、なりすましや重複を防ぎながら受信して取り込み、失敗時に再処理できるよう記録・通知するシステム処理 SYS-006 を定義します。** 処理概要 / 処理フロー図 / 入出力 / 処理項目定義 / 入出力一覧 / システムイベント一覧 の 6 セクションで記述します。

*種別 システム設計 ・ 優先度 P0 ・ ステータス ドラフト*

## 1. 処理概要

課金プロバイダから送られる決済・契約状態の通知を受信し、送信元の正当性を検証したうえで取り込む。検証を通過した通知のみを受信履歴として記録し、契約状態・サブスクリプション・請求書へ反映する。重複した通知は二重処理を防いで受信記録のみ残し、取込に失敗した通知は失敗として記録・通知し再処理の対象とする。

| システム ID | 処理名 | 種別 | トリガー / スケジュール | 機能概要 |
|---|---|---|---|---|
| `SYS-006` | 課金プロバイダ通知の受信・検証・取込 | webhook | 課金プロバイダからの HTTP 通知受信時 | 通知を受信・署名検証・重複排除し、受信履歴を残して契約・サブスク・請求書へ反映し、失敗は再処理対象として記録・通知する |

## 2. 処理フロー図

```mermaid
flowchart TB
  A[課金プロバイダ通知を受信] --> B{送信元の正当性が確認できるか}
  B -- いいえ --> C[不正な受信として記録]
  C --> Z[終了]
  B -- はい --> D{過去に受信済みの通知か}
  D -- はい --> E[取込をスキップし受信記録のみ保持]
  E --> Z
  D -- いいえ --> F[受信履歴として記録]
  F --> G[通知内容に応じて契約・決済状態へ反映]
  G --> H{取込に成功したか}
  H -- はい --> I[取込完了として記録]
  I --> Z
  H -- いいえ --> J[失敗として記録し通知・再処理対象化]
  J --> Z
```

## 3. 入出力

| 区分 | 内容 |
|---|---|
| 入力ソース | 課金プロバイダからの決済・契約状態通知(外部 HTTP 通知) |
| 出力先 | 受信履歴の記録、契約状態・サブスクリプション・請求書への反映、失敗時の運用通知 |

## 4. 処理項目定義

| 項目 ID | ステップ | 説明 | 種別 | 実行条件 |
|---|---|---|---|---|
| <span id="PR-01"></span>`PR-01` | 受信・署名検証 | 課金プロバイダからの通知を受信し、送信元の正当性を検証する。検証に失敗した通知は不正な受信として記録し取り込まない | 判定 | 通知受信時 |
| <span id="PR-02"></span>`PR-02` | 重複排除 | 過去に受信済みの通知でないかを照合し、重複した通知の二重処理を防ぐ | 判定 | 署名検証を通過 |
| <span id="PR-03"></span>`PR-03` | 受信履歴の記録 | 受信した通知を一定期間の受信履歴として記録する | 記録 | 重複でない通知 |
| <span id="PR-04"></span>`PR-04` | 契約・決済状態への反映 | 通知内容に応じて決済状態・契約状態を取り込み、サービス内の契約・サブスクリプション・請求書へ反映する | 更新 | 受信履歴の記録後 |
| <span id="PR-05"></span>`PR-05` | 取込完了の記録 | 反映に成功した通知を取込完了として記録する | 記録 | 反映に成功 |
| <span id="PR-06"></span>`PR-06` | 失敗記録・通知・再処理対象化 | 反映に失敗した通知を失敗として記録・通知し、後から再処理できる状態にする | 例外 | 反映に失敗 |
