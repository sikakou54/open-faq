# <span id="SYS-002"></span>SYS-002: プロジェクト数急増検知通知

> **このページは、契約ごとのプロジェクト数を監視し、想定外の急増を検知してオーナーへ通知するシステム処理 SYS-002 を定義します。** 処理概要 / 処理フロー図 / 入出力 / 処理項目定義 / 入出力一覧 / システムイベント一覧 の 6 セクションで記述します。

*種別 システム設計 ・ 優先度 P0 ・ ステータス ドラフト*

## 1. 処理概要

プロジェクト数に固定上限を設けず、契約ごとのプロジェクト数を監視する処理である。プロジェクトの新規作成や定期的な監視を契機に対象契約のプロジェクト数を集計し、想定外の急激な増加に該当するかを評価して、該当時にオーナーへ通知する。

| システム ID | 処理名 | 種別 | トリガー / スケジュール | 機能概要 |
|---|---|---|---|---|
| `SYS-002` | プロジェクト数急増検知通知 | monitor | プロジェクト作成時 + 定期監視 | 契約ごとのプロジェクト数を集計し、急増を検知してオーナーへ通知する |

| 関連 | 内容 |
|---|---|

## 2. 処理フロー図

```mermaid
flowchart TB
  start([プロジェクト作成 または 定期監視]) --> agg[対象契約のプロジェクト数を集計]
  agg --> judge{想定外の急増に該当するか}
  judge -- 該当しない --> done([通知せず終了])
  judge -- 該当する --> compose[オーナー向け通知内容を生成]
  compose --> notify[オーナーへ通知を届ける]
  notify --> result{通知に成功したか}
  result -- 成功 --> log[通知結果を記録]
  log --> done
  result -- 失敗 --> retry[再送対象として記録]
  retry --> done
```

## 3. 入出力

| 区分 | 内容 |
|---|---|
| 入力ソース | プロジェクトの新規作成イベント / 定期監視スケジュール / 対象契約のプロジェクト数 |
| 出力先 | オーナーへの受信箱通知 / 通知ログ / 再送対象記録(例外時) |

## 4. 処理項目定義

| 項目 ID | ステップ | 説明 | 種別 | 実行条件 |
|---|---|---|---|---|
| <span id="PR-01"></span>`PR-01` | プロジェクト数集計 | 対象契約のプロジェクト数を集計する | 集計 | プロジェクト作成 または 定期監視 |
| <span id="PR-02"></span>`PR-02` | 急増判定 | 集計結果をもとに想定外の急激な増加に該当するかを判定する | 判定 | 集計完了後 |
| <span id="PR-03"></span>`PR-03` | 通知内容生成 | 急増に該当する契約のオーナー向け通知内容を生成する | 通知 | 急増に該当する場合 |
| <span id="PR-04"></span>`PR-04` | オーナー通知 | 生成した通知をオーナーへ届け、通知結果を記録する | 通知 | 通知内容生成後 |
| <span id="PR-05"></span>`PR-05` | 再送対象記録 | 通知の生成または送信に失敗した場合、再送対象として記録する | 例外 | 通知の生成または送信に失敗した場合 |

集計入力はプロジェクトのマスタ、出力は受信箱通知・通知ログ・例外記録である。本処理は無人監視のため画面 API を経由せず、外部 API も呼び出さない。

| 入出力 | 説明 | 種別 | I/O | CRUD | 参照 |
|---|---|---|---|---|---|

| 受信箱通知 | オーナーへ届ける通知を登録する | テーブル | 出力 | `C - - -` | [TBL-022](../04_database/TBL-022.md#TBL-022) |
