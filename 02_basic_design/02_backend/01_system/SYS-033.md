# <span id="SYS-033"></span>SYS-033: 監査ログ整合性検証(日次)

> **このページは、日次の定期処理として監査ログを全件走査し、改ざん検知情報の整合性を検証して違反を検出した対象を担当者へ通知するシステム処理 SYS-033 を定義します。** 処理概要 / 処理フロー図 / 入出力 / 処理項目定義 / 入出力一覧 / システムイベント一覧 の 6 セクションで記述します。

*種別 システム設計 ・ 優先度 P0 ・ ステータス ドラフト*

## 1. 処理概要

スケジューラが日次で整合性検証バッチを起動し、保持されている監査ログを全件走査して改ざん検知情報の整合性を検証する。整合性違反(改ざん・欠落の疑い)を検出した対象はアラート通知し、違反が無ければ正常終了する。検証は読み取りのみで監査ログ自体は変更しない。

| システム ID | 処理名 | 種別 | トリガー / スケジュール | 機能概要 |
|---|---|---|---|---|
| `SYS-033` | 監査ログ整合性検証(日次) | batch | 日次の定期処理(1 日 1 回・スケジューラ起動) | 監査ログを全件走査し改ざん検知情報の整合性を検証、違反対象をアラート通知する |

## 2. 処理フロー図

```mermaid
flowchart TB
  S([日次スケジュールで起動]) --> A[監査ログを全件走査]
  A --> B{検証対象が<br/>存在するか}
  B -- 存在しない --> Z([検証せず正常終了])
  B -- 存在する --> C[各記録の改ざん検知情報の整合性を検証]
  C --> D{整合性違反を<br/>検出したか}
  D -- 違反あり --> E[違反対象を特定し担当者へアラート通知]
  E --> F[走査を継続]
  D -- 違反なし --> F
  F --> G{全件の走査が<br/>完了したか}
  G -- 未完了 --> C
  G -- 完了 --> H([正常終了])
```

## 3. 入出力

| 区分 | 内容 |
|---|---|
| 入力ソース | 日次スケジューラの起動契機、保持されている監査ログ全件(改ざん検知情報を含む) |
| 出力先 | 整合性違反を検出した対象の担当者向けアラート通知 |

## 4. 処理項目定義

| 項目 ID | ステップ | 説明 | 種別 | 実行条件 |
|---|---|---|---|---|
| <span id="PR-01"></span>`PR-01` | 全件走査 | 保持されている監査ログを全件読み取り、点検対象とする | 取得 | — |
| <span id="PR-02"></span>`PR-02` | 整合性検証 | 各記録について改ざん検知情報の整合性(改ざん・欠落の疑い)を順に検証する | 判定 | 検証対象が存在するとき |
| <span id="PR-03"></span>`PR-03` | 違反通知 | 整合性違反を検出した対象を特定し担当者へアラート通知する(走査は中断せず継続) | 通知 | 整合性違反を検出したとき |
| <span id="PR-04"></span>`PR-04` | 正常終了 | 全件走査の完了後、または検証対象が存在しない場合に正常終了する | 完了 | — |
