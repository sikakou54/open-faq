# <span id="SYS-020"></span>SYS-020: 決済失敗猶予・サスペンション移行

> **このページは、決済失敗確定の通知を起点に猶予期間を監視し、猶予中に支払いが回復しなければ課金アカウントをサスペンションへ移行し、回復すれば通常状態へ復帰させるシステム処理 SYS-020 を定義します。**

*種別 システム設計 ・ 優先度 P0 ・ ステータス ドラフト*

| ID | 業務ユースケースID | API ID | テーブルID |
|----|----|----|----|
| SYS-020 | [UC-055](../../../01_requirements/04_business_usecases/UC-055.md#UC-055) | [API-045](../03_apis/API-045.md#API-045) | [TBL-002](../04_database/TBL-002.md#TBL-002) ・ [TBL-018](../04_database/TBL-018.md#TBL-018) |

| 処理名 | 種別 | トリガー / スケジュール |
|----|----|----|
| 決済失敗猶予・サスペンション移行 | monitor | 決済失敗確定 / 再決済成功 / 解除の通知受信、および猶予経過を判定する定期起動 |

## 1. 処理概要

- 課金プロバイダからの決済失敗確定通知を起点に猶予期間の監視を開始し、課金アカウントは通常利用可能な状態のまま維持する。
- 定期処理が猶予の経過を判定し、猶予期間を過ぎても再決済が成立しない課金アカウントをサスペンションへ移行する。
- 猶予中・サスペンション中のいずれであっても、再決済成功または解除の通知を受けた時点で課金アカウントを通常状態へ即時復帰させる。
- 通知の真正性が確認できない場合は処理せず拒否し、同一通知の重複受信は冪等に扱う。

## 2. 処理フロー図

```mermaid
flowchart TB
  S([決済失敗確定の通知受信を契機に起動]) --> A[通知の真正性を確認]
  A --> B{真正性を<br/>確認できたか}
  B -- できない / 重複受信 --> X[処理せず拒否・冪等にスキップ]
  X --> Z([完了])
  B -- 新規かつ真正 --> C[猶予期間の監視を開始・課金アカウントは有効を維持]
  C --> D[オーナーへ支払いに関する重要通知]
  D --> Z
  SC([定期処理の起動を契機に判定]) --> E[猶予中の課金アカウントの経過状況を判定]
  E --> F{猶予経過かつ<br/>再決済未成立か}
  F -- はい --> G[課金アカウントをサスペンションへ移行]
  F -- いいえ --> H[移行せず正常終了]
  G --> Z
  H --> Z
  RC([再決済成功 / 解除の通知受信を契機に起動]) --> I[課金アカウントを通常状態へ即時復帰]
  I --> Z
```

## 3. 入出力

| 区分 | 内容 |
|---|---|
| 入力ソース | 課金プロバイダからの決済失敗確定 / 再決済成功 / 解除の通知、および猶予経過を判定する定期処理の起動 |
| 出力先 | 課金アカウント状態・猶予期間の記録、オーナーへの支払いに関する重要通知 |

## 4. 処理項目定義

| 項目 ID | ステップ | 説明 | 種別 | 実行条件 |
|---|---|---|---|---|
| <span id="PR-01"></span>`PR-01` | 通知真正性確認 | 受信した通知の真正性を確認し、不正・重複受信は処理せず冪等に扱う | 判定 | 通知受信時 |
| <span id="PR-02"></span>`PR-02` | 猶予監視開始 | 受信時点を起点に猶予期間の監視を開始し、課金アカウントを有効のまま維持する | 記録 | 真正な新規の決済失敗確定通知のとき |
| <span id="PR-03"></span>`PR-03` | 支払い重要通知 | オーナーへ支払いに関する重要通知を行う | 通知 | 猶予監視を開始したとき |
| <span id="PR-04"></span>`PR-04` | 猶予経過判定 | 定期処理が猶予中の各課金アカウントについて経過状況を判定する | 判定 | 定期起動時 |
| <span id="PR-05"></span>`PR-05` | サスペンション移行 | 猶予経過かつ再決済未成立の課金アカウントをサスペンションへ移行する | 記録 | 猶予経過かつ再決済未成立のとき |
| <span id="PR-06"></span>`PR-06` | 通常状態復帰 | 再決済成功または解除の通知を受けた課金アカウントを通常状態へ即時復帰させる | 記録 | 再決済成功 / 解除の通知を受けたとき |
