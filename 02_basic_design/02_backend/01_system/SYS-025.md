# <span id="SYS-025"></span>SYS-025: 配信失敗通知の再送

> **このページは、配信に失敗した通知を検出し、再送上限の範囲内で自動的に再送するシステム処理 SYS-025 を定義します。**

| 処理名 | 種別 | トリガー / スケジュール |
|----|----|----|
| 配信失敗通知の再送 | batch | 定期起動の再送処理(失敗分の検出) |

## 1. 処理概要

- スケジューラが定期起動する再送処理が、配信に失敗した通知を検出し、抑制リスト該当外かつ再送上限未満の対象を上限の範囲内で再送する。
- 再送結果に応じて配信状態と再送回数を更新し、再送上限到達分・抑制該当分は確定失敗として記録する。
- 再送上限は 3 回(設計値)とする。
- 再送間隔は指数バックオフ([システム仕様書 §3](../../07_system-spec.md#3-タイムアウトセッション認証) の通知再送バックオフ)で、直近の配信失敗時刻からの経過時間が次回間隔に達した対象のみを各バッチで再送する。
- 再送回数が上限(3 回)に到達した時点で再送を打ち切り、確定失敗として記録する。

## 2. 処理フロー図

```mermaid
flowchart TB
  S([スケジューラが再送処理を定期起動]) --> A[認証・データ取得]
  A --> B{再送対象あり}
  B -- なし --> Z([正常終了])
  B -- あり --> C[再送可否を判定<br/>(抑制リスト・再送上限)]
  C --> D{再送可<br/>(抑制外 かつ 上限未満)}
  D -- 可 --> E[通知を再送]
  E --> F[再送結果を取得]
  F --> G[配信状態・再送回数を更新]
  D -- 上限到達 / 抑制該当 --> H[確定失敗として記録]
  G --> X([完了])
  H --> X
```

## 3. 入出力

| 区分 | 内容 |
|---|---|
| 入力ソース | 定期起動の再送処理(配信失敗通知・配信状態・再送回数・抑制リスト・再送上限) |
| 出力先 | 通知の再送、配信状態・再送回数の更新、再送上限到達分・抑制該当分の確定失敗記録 |

## 4. 処理項目定義

| 項目 ID | ステップ | 説明 | 種別 | 実行条件 |
|---|---|---|---|---|
| <span id="PR-01"></span>`PR-01` | 再送処理起動 | スケジューラが定期的に再送処理を起動し、認証・データ取得を行う | トリガー | 定期スケジュール到来時 |
| <span id="PR-02"></span>`PR-02` | 再送対象抽出 | 配信に失敗し、まだ再送上限に達していない通知を再送対象として抽出する | 抽出 | — |
| <span id="PR-03"></span>`PR-03` | 再送可否判定 | 各対象について抑制リスト該当外であり、再送回数とバックオフが [システム仕様書 §3](../../07_system-spec.md#3-タイムアウトセッション認証) の通知再送条件を満たすことを判定する | 判定 | 再送対象があるとき |
| <span id="PR-04"></span>`PR-04` | 通知再送 | 条件を満たす対象の通知を再送する | 通知 | 抑制外 かつ 上限未満のとき |
| <span id="PR-05"></span>`PR-05` | 配信状態更新 | 再送結果に応じて配信状態と再送回数を更新する | 記録 | 再送を実施したとき |
| <span id="PR-06"></span>`PR-06` | 確定失敗記録 | 再送回数が上限(3 回。設計値)に到達した分・抑制該当分は再送せず打ち切り、確定失敗として記録する | 例外 | 上限到達 または 抑制該当のとき |

| 入出力 | 説明 | 種別 | I/O | CRUD | 参照 |
|---|---|---|---|---|---|
| 通知メール再送 | 配信失敗通知を再送する配信 IF(メール配信 IF) | API | 出力 | — | [API-058](../03_apis/API-058.md#API-058) |
| 通知配信ログ | 配信状態・再送回数を参照し、再送結果・確定失敗を更新する | テーブル | 入出力 | `- R U -` | [TBL-026](../04_database/TBL-026.md#TBL-026) |
