# <span id="SYS-001"></span>SYS-001: メンバー数の上限接近・急増の検知と通知

> **このページは、プロジェクト内のメンバー数を集計し上限への接近・急増を検知してアカウント利用者へアラート通知を生成するシステム処理 SYS-001 を定義します。** 処理概要 / 処理フロー図 / 入出力 / 処理項目定義 / 入出力一覧 / システムイベント一覧 の 6 セクションで記述します。

*種別 システム設計 ・ 優先度 P0 ・ ステータス ドラフト*

## 1. 処理概要

プロジェクトごとに最新のメンバー数を集計し、設定済みのしきい値への接近と短期間での急増を評価する監視処理である。検知したプロジェクトについてアカウント利用者(オーナーおよび関係するメンバー)を特定し、受信箱へのアラート通知を生成するとともに、検知・通知の結果を運用記録として残す。しきい値の具体値は機能要件・業務ルールに委ね、急増判定のしきい値は [RULE-023](../../../01_requirements/01_business_requirement/08_rule.md#RULE-023) に従う。

トリガーは次の二段構成(設計値)とする。

1. **即時評価**: メンバー割当の追加・解除が確定した時点(メンバー招待 [API-021](../03_apis/API-021.md#API-021)・プロジェクト割当解除 [API-023](../03_apis/API-023.md#API-023) 等の確定をフック)で、当該プロジェクトのメンバー数を再集計してしきい値接近・急増を評価する。割当が変動したプロジェクトのみを対象とし、急増の検知に間に合わせる。
2. **日次バッチ**: 即時評価の取りこぼし(フック失敗・整合のずれ)を補うため、日次の定期集計で全プロジェクトを走査して再評価する。

両者は同じ評価ロジックを用い、二重通知を避けるため検知・通知記録で重複発火を抑止する(抑止方式は詳細設計)。

| システム ID | 処理名 | 種別 | トリガー / スケジュール | 機能概要 |
|---|---|---|---|---|
| `SYS-001` | メンバー数の上限接近・急増の検知と通知 | monitor | メンバー割当の追加/解除確定時の即時評価(API-021 / API-023 等のフック)+ 定期集計(日次) | プロジェクトごとにメンバー数を集計し、しきい値接近・急増を検知してアカウント利用者へアラート通知を生成し、検知・通知結果を記録する |

## 2. 処理フロー図

```mermaid
flowchart TB
  start([メンバー数の変動 / 定期集計を契機に起動]) --> agg[プロジェクトごとに最新のメンバー数を集計]
  agg --> eval[しきい値への接近・短期間の急増を評価]
  eval --> judge{接近または急増を検知?}
  judge -- いいえ --> watch[アラートを生成せず通常の監視を継続]
  watch --> done([終了])
  judge -- はい --> target[対象のアカウント利用者を特定]
  target --> notify[アラート通知を生成し受信箱へ届ける]
  notify --> sent{通知の生成・送信に成功?}
  sent -- はい --> record[検知・通知の結果を運用記録として残す]
  sent -- いいえ --> retry[再送対象として記録し後続の再試行に委ねる]
  retry --> record
  record --> done
```

## 3. 入出力

| 区分 | 内容 |
|---|---|
| 入力ソース | メンバー数の変動契機 / 定期集計スケジュール(日次)、プロジェクトごとのメンバー数 |
| 出力先 | アカウント利用者向けの受信箱アラート通知、検知・通知の運用記録 |

## 4. 処理項目定義

| 項目 ID | ステップ | 説明 | 種別 | 実行条件 |
|---|---|---|---|---|
| <span id="PR-01"></span>`PR-01` | メンバー数集計 | メンバー数の変動または定期集計を契機に、プロジェクトごとの最新のメンバー数を把握する | 集計 | 変動契機 / 定期集計タイミング |
| <span id="PR-02"></span>`PR-02` | 接近・急増の評価 | 設定済みのしきい値に対する接近の有無と、短期間での急増の有無を評価する | 判定 | PR-01 完了後 |
| <span id="PR-03"></span>`PR-03` | 通常監視の継続 | しきい値に達しておらず急増も認められない場合、アラートを生成せず通常の監視を継続する | 例外 | 接近・急増を検知しない場合 |
| <span id="PR-04"></span>`PR-04` | 通知対象の特定 | 検知したプロジェクトについて、通知すべきアカウント利用者(オーナー・関係するメンバー)を特定する | 判定 | 接近または急増を検知した場合 |
| <span id="PR-05"></span>`PR-05` | アラート通知の生成 | 検知内容を知らせるアラート通知を生成し、対象の受信箱へ届ける | 通知 | PR-04 完了後 |
| <span id="PR-06"></span>`PR-06` | 検知・通知の記録 | 検知と通知の結果を運用記録として残す。通知の生成・送信に失敗した場合は再送対象として記録する | 記録 | PR-05 後 |
