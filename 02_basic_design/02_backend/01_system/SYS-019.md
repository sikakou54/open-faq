# <span id="SYS-019"></span>SYS-019: 質問数上限アラート通知

> **このページは、当月質問数が設定済みのアラート閾値へ当月初めて到達したとき、オーナーと当該プロジェクトの有効なメンバーへ上限アラートを通知するシステム処理 SYS-019 を定義します。** 処理概要 / 処理フロー図 / 入出力 / 処理項目定義 / 入出力一覧 / システムイベント一覧 の 6 セクションで記述します。

*種別 システム設計 ・ 優先度 P0 ・ ステータス ドラフト*

## 1. 処理概要

プロジェクトの当月質問数が、設定済みのアラート閾値へ当月初めて到達したことを契機に、上限到達前の気づきを促す通知を生成し、オーナーと当該プロジェクトの有効なメンバーへ届ける。同一プロジェクト・同一請求月・同一閾値の通知は受信者ごとに一度だけ行い、当月内の再到達では重ねて通知しない。

| システム ID | 処理名 | 種別 | トリガー / スケジュール | 機能概要 |
|---|---|---|---|---|
| `SYS-019` | 質問数上限アラート通知 | async | 当月質問数が設定済みアラート閾値へ到達したことを検知したとき | 当月初回到達を判定し、オーナーと有効メンバーへ上限アラートのお知らせ生成とメール送信を行う |

## 2. 処理フロー図

```mermaid
flowchart TB
  S([アラート閾値への到達を検知して起動]) --> A[上限・閾値設定と当月質問数を照合]
  A --> B{当月初回到達<br/>かつ閾値選択あり}
  B -- いいえ --> X([通知なしで終了])
  B -- はい --> C[宛先(オーナー + 有効メンバー)を特定し重複を排除]
  C --> D[各宛先の受信箱に上限アラートのお知らせを生成]
  D --> E[各宛先へ上限アラートメールを送信]
  E --> F[送信結果を記録]
  F --> H([完了])
```

## 3. 入出力

| 区分 | 内容 |
|---|---|
| 入力ソース | 当月質問数のアラート閾値到達検知(当該プロジェクトの上限・閾値設定、当月の質問数) |
| 出力先 | 各宛先の受信箱お知らせ生成、上限アラートメール送信、送信結果の記録 |

## 4. 処理項目定義

| 項目 ID | ステップ | 説明 | 種別 | 実行条件 |
|---|---|---|---|---|
| <span id="PR-01"></span>`PR-01` | 設定照合 | 当該プロジェクトの上限・閾値設定と当月の質問数を照合する | 判定 | — |
| <span id="PR-02"></span>`PR-02` | 初回到達判定 | その閾値への到達が当月で初めてかを判定する | 判定 | — |
| <span id="PR-03"></span>`PR-03` | 宛先解決 | オーナーと当該プロジェクトの有効なメンバーを特定し、宛先の重複を取り除く | 処理 | 当月初回到達かつ閾値選択ありのとき |
| <span id="PR-04"></span>`PR-04` | お知らせ生成 | 各宛先の受信箱に上限アラートのお知らせを生成する | 記録 | 当月初回到達かつ閾値選択ありのとき |
| <span id="PR-05"></span>`PR-05` | メール送信 | 各宛先へ上限アラートのメールを送信する | 通知 | 当月初回到達かつ閾値選択ありのとき |
| <span id="PR-06"></span>`PR-06` | 送信結果記録 | メールの送信結果を記録し、失敗時は後続の通知再送の対象とする | 記録 | メール送信後 |
| <span id="PR-07"></span>`PR-07` | 冪等スキップ | 当月内の再到達、または閾値が一つも選択されていない場合は通知せず処理を終える | 例外 | 再到達 / 閾値未選択のとき |
