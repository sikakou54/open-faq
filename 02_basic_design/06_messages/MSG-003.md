# <span id="MSG-003"></span>MSG-003: TPL-MEMBER_INVITATION

> **このページは、メールテンプレート `TPL-MEMBER_INVITATION` の送信契機・件名・本文・テンプレート変数を定義する正本です。**

*種別 メッセージ(メール)・ ステータス ドラフト*

## <span id="meta"></span>1. テンプレートメタ

配信先・重要度・強制送信可否・リンク有効期限の索引です。共通基準は [メッセージ設計 index §共通基準](index.md#common) が正本です。

| 項目 | 値 |
|----|----|
| テンプレートID | `TPL-MEMBER_INVITATION` |
| 通知ID | NOTIF-MEMBER_INVITATION |
| 重要度 | critical |
| 配信先 | 招待対象(登録済みユーザー) |
| 強制送信 | ◯ |
| リンク有効期限 | 7d |
| 件名 | `open-faq のプロジェクトへの招待` |

## <span id="body"></span>3. テンプレート定義

発火条件・テンプレート変数・件名・本文(テキスト / HTML 版)です。本文の件名・テキストは送信される文言そのものの正本です。

メンバー招待および招待再送で共通利用する `critical` メールです。**招待は登録済みユーザーのみを対象**とし、プロジェクトへのメンバー招待はすべて本テンプレートに一本化する。招待受諾 URL(7d 有効)と招待内容(プロジェクト・招待元)を提示し、着地ページで参加を受諾させる。**既にアカウントを持つ利用者が対象のため、氏名・パスワードの設定や規約同意は求めない**(アカウント新規作成は独立サインアップで別途行う)。

**発火条件**: SCR-014 メンバー招待モーダルの「招待メールを送信する」/「招待メールを再送する」操作時、招待受諾用トークンの発行時。

**テンプレート変数**:

| 変数 | 説明 |
|----|----|
| `{recipient_name}` | 招待対象(登録済みユーザー)の表示名 |
| `{recipient_email}` | 招待対象のメールアドレス |
| `{inviter_owner_name}` | 招待元オーナーの表示名 |
| `{inviter_admin_name}` | 招待操作者の表示名(オーナー以外のメンバーの場合) |
| `{project_name}` | 招待対象プロジェクト名 |
| `{accept_url}` | 招待受諾 URL(7d 有効、SCR-023 招待受諾ページに着地) |
| `{expire_at_jst}` | 失効日時(JST) |

**件名**: `open-faq のプロジェクトへの招待`

**本文(テキスト版)**:

```json
{recipient_name} 様

{inviter_owner_name} 様の open-faq アカウントから、{project_name} のメン
バーとして招待を受けました。以下のリンクから参加を受諾してください。
すでに open-faq のアカウントをお持ちのため、氏名・パスワードの設定や
規約同意は不要です。

▶ 招待を受諾する
{accept_url}

このリンクは {expire_at_jst}(JST) まで有効です(発行から 7 日)。
失効後に参加する場合は、招待元のメンバーへ再送をご依頼ください。

招待内容:
  - プロジェクト: {project_name}
  - 招待元オーナー: {inviter_owner_name}
  - 招待操作者: {inviter_admin_name}

心当たりがない場合は本メールを破棄してください。リンクをクリックしない限
り、プロジェクトへの参加は確定しません。

──────────────────────────────────────────────
このメールに直接返信しないでください。
open-faq <noreply@<service-domain>>
```

**本文(HTML 版)**: 招待内容を背景色 `#eef6ff` のパネルで表示 + プライマリボタン「招待を受諾する」+ 失効日時を強調表示。本文には「既にアカウントをお持ちのため、氏名・パスワード設定や規約同意は不要」である旨を含める。

**着地ページ**: SCR-023 招待受諾([画面設計書](../01_frontend/01_screens/index.md))。

**備考**: 再送時は **既存の招待を無効化し新しい招待を発行** したうえで本テンプレートで再送信する。本文は新規招待時と同一で、差出人による文脈は本文中の「招待操作者」フィールドで補足する。未登録(アカウントのない)メールアドレスは招待できず、招待操作時に SCR-014 でエラー([ERR-035](../05_errors/ERR-035.md#ERR-035))となるため、本メールは登録済みユーザーにのみ送信される。
