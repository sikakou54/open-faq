## <span id="body"></span>3. テンプレート定義

発火条件・テンプレート変数・件名・本文(テキスト / HTML 版)です。本文の件名・テキストは送信される文言そのものの正本です。

退会の即時成立(`withdrawn` への移行)を契機に、退会受付・サービス停止・運用データ削除・請求情報の保持期間中閲覧可・問い合わせ先を一度だけ通知する `high` メールです。退会は猶予なく即時成立するため、事前リマインド・退会取消は提供しない。

**発火条件**: アカウントの退会が即時成立した時に 1 回送信する(アカウント状態が `withdrawn` へ移行した契機)。退会撤回・90 日猶予・退会発効日(当月末)は持たない。退会時に運用データ(参加プロジェクトからの離脱・作成プロジェクトの削除を含む)は速やかに削除し、アカウント・請求データは退会時刻を起点に 7 年保持する(RULE-022)。保持期間中は本人がログインして請求情報を閲覧でき、7 年経過後に物理削除される。

**テンプレート変数**:

| 変数 | 説明 |
|----|----|
| `{recipient_display_name}` | 退会したアカウント本人の表示名 |
| `{withdrawn_at_jst}` | 退会成立日時(`M_BILLING_ACCOUNT.withdrawn_at`、JST) |
| `{billing_retention_until_jst}` | 請求情報の保持期限(退会成立から 7 年、JST) |

**件名**:

- `退会手続きが完了しました`

**本文(テキスト版)**:

```json
{recipient_display_name} 様

open-faq のご利用、誠にありがとうございました。
{withdrawn_at_jst}(JST) をもって退会手続きが完了し、サービスのご利用を停止しました。

退会に伴い、運用データ(FAQ・問い合わせ・利用状況等)は速やかに削除されます。
請求に関する情報は、法令に基づき {billing_retention_until_jst}(JST) まで保持します。
保持期間中は、ログインのうえ請求情報の閲覧のみ行えます。

ご不明な点は、お問い合わせ画面または各プロジェクトのお問い合わせ先からご連絡ください。

──────────────────────────────────────────────
このメールに直接返信しないでください。
open-faq <noreply@<service-domain>>
```

**本文(HTML 版)**: テキスト版と同一内容を HTML で構成する(リンクは持たない)。ヘッダ色は完了通知として中立色(青系)とする。

**備考**: 退会の即時成立を契機に 1 回のみ送信する。事前リマインド(発効前)・退会取消リンク・フェーズ出し分けは持たない。
