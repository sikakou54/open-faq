# 画面設計(SCR)テンプレート

> **本テンプレートは画面設計(SCR)の記載骨格(6 セクション固定)を定義します。**

運用ルールの正本は [../CLAUDE.md](../CLAUDE.md)。共通記載スタイルは [共通記載スタイル](00_common-style.md) を参照する。

- **配置先**: `02_basic_design/01_frontend/01_screens/`(画面モックは同階層配下の `mocks/`)
- **採番**: `SCR-001`〜(1 画面 = 1 ファイル(フラット。`-NNN` 従属は廃止)・ゼロ詰め 3 桁・欠番なし)

## 骨格(6 セクション固定)

**1 画面 = 1 ファイル(フラット `SCR-NNN`)**。6 セクション固定:

1. **画面概要**(画面 ID 表 + `関連`(FR/BR)+ `対応業務UC` 行)
2. **画面遷移図**(mermaid flowchart)
3. **画面レイアウト**(モック = **PNG 画像のみ**。HTML は `mocks/` に外出し)
4. **画面項目定義**(項目 ID `IT-01`…)
5. **入出力一覧**(CRUD マトリクス)
6. **画面イベント一覧**(列に **EVT 列**(`EVT-NNN` へリンク)/ イベント ID `EV-01`… / 項目 ID(§4 の `IT-` に対応、無ければ `—`)/ イベント / 処理)

## 記載ルール

- 各 `##`/`###` 直後に 1〜2 文のリード文を置いてから表へ入る(表をいきなり置かない)。
- §3 画面レイアウトは **PNG 画像で表示**する(MD は画像 1 行のみ)。HTML ソースは `mocks/<画面>-<n>.html` に外出しし、各 SCR からは `mocks/<画面>-<n>.png`(子階層相対・`../` なし)で参照する。画面モックの画像化手順は [../CLAUDE.md](../CLAUDE.md) の「画面モックの画像化」を参照。
- §1 の `対応業務UC` 行が UC ↔ 画面の逆引き正本。UC・API 等へのリンクは [共通記載スタイル](00_common-style.md) の相互参照アンカー例に従う。
