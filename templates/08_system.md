# システム設計(SYS)テンプレート

> **本テンプレートはシステム設計(SYS)の記載骨格(6 セクション固定)を定義します。**

運用ルールの正本は [../CLAUDE.md](../CLAUDE.md)。共通記載スタイルは [共通記載スタイル](00_common-style.md) を参照する。

- **配置先**: `02_basic_design/02_backend/01_system/`
- **採番**: `SYS-001`〜(1 システム処理 = 1 ファイル(フラット)・ゼロ詰め 3 桁・欠番なし)

## 骨格(6 セクション固定)

**1 システム処理 = 1 ファイル(フラット `SYS-NNN`)**。システム起点(バッチ / Webhook / 非同期 / 監視 / 通知の無人処理)の業務UCを実現する、画面設計(SCR)の backend 版。**モックは持たない**。6 セクション固定:

1. **処理概要**(処理 ID 表 = システム ID / 処理名 / 種別(`batch`/`cron`/`webhook`/`async`/`monitor`/`guard` 等)/ トリガー・スケジュール / 機能概要 + 関連表(業務ユースケースID / 関連システム `SYS` / API ID / テーブルID))
2. **処理フロー図**(mermaid flowchart)
3. **入出力**(入力ソース / 出力先)
4. **処理項目定義**(項目 ID `PR-01`…)
5. **入出力一覧**(テーブル/呼出API/外部IF を CRUD。横断的で結線が無い箇所は `—`)
6. **システムイベント一覧**(列に **SEV 列**(`<span id="SEV-NNN"></span>SEV-NNN` の非リンクアンカー。SEV は独立ページ・フォルダを持たず、この §6 にアンカーとして保持する)/ イベント ID `SE-01`… / 項目 ID(§4 の `PR-` に対応、無ければ `—`)/ イベント / 処理)

全層の厳密な紐付けはトレーサビリティ一覧表 [`00_traceability/index.md`](../../00_traceability/index.md) に一元管理し、SYS 本文に `TR-NNN` は記載しない。

## 記載ルール

- 各 `##`/`###` 直後に 1〜2 文のリード文を置いてから表へ入る(表をいきなり置かない)。
- システム → API は同グルーピング内の兄弟参照、システム → エラーはグルーピング/グループをまたぐ参照。[共通記載スタイル](00_common-style.md) の相互参照アンカー例に従う(システム → API `[API-060](../03_apis/API-060.md#API-060)` / システム → エラー `[ERR-001](../../05_errors/ERR-001.md#ERR-001)`)。
