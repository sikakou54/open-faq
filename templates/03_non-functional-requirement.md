# 非機能要件(NFR)テンプレート

> **本テンプレートは非機能要件(NFR)の記載骨格(分類節・通し連番)を定義します。**

運用ルールの正本は [../CLAUDE.md](../CLAUDE.md)。共通記載スタイルは [共通記載スタイル](00_common-style.md) を参照する。要件仕様の 3 フォルダ構成・6 カテゴリの共通説明は [テンプレート索引](README.md) を参照する。

- **配置先**: `01_requirements/03_non_functional_requirement/07_nfr.md`(`## <分類>` 節別に統合)
- **採番**: `NFR-001`〜(**分類別番号帯は使わず通し連番**・ゼロ詰め 3 桁・欠番なし)

## 骨格

NFR は `07_nfr.md` 内で**分類節へ配置**し、各要件を節として保持する。

- 各要件節の骨格: `## <span id="ID"></span>ID: 名称` → 要約 → メタ → `### 要件`。**リード文は置かない。** ID アンカーは要件見出しに付け、節見出し(要件 等)には付けない(ファイル内で重複するため)。
- **非機能要件 `NFR-*`**: メタに `分類`。`07_nfr.md` 内で分類節へ配置。**分類別番号帯は使わず通し連番**。
- 要件は後工程へのリンク・参照を持たない(書式上の指示)。詳細ルールは [../CLAUDE.md](../CLAUDE.md) の粒度分離ルールを参照。

## 追加方法

NFR は分類節へ `## <span id="ID"></span>…` を追加し、`分類` で振り分ける。通し連番で採番する。

## 記載例(要件節骨格)

```markdown
## <span id="NFR-001"></span>NFR-001: 名称

> **本要件は「…」を定義します。**

*種別 非機能要件 ・ 分類 性能 ・ ステータス ドラフト*

### 要件

…(非機能の要求を記述)
```
