# 業務要件(BR)テンプレート

> **本テンプレートは業務要件(BR)の記載骨格(カテゴリ別ファイル内の HTML テーブル行)を定義します。**

運用ルールの正本は [../CLAUDE.md](../CLAUDE.md)。共通記載スタイルは [共通記載スタイル](00_common-style.md) を参照する。要件仕様の 3 フォルダ構成・6 カテゴリの共通説明は [テンプレート索引](README.md) を参照する。

- **配置先**: `01_requirements/01_business_requirement/`(`01_account-br.md` 〜 `06_security-br.md` の 6 カテゴリ別ファイル)
- **採番**: `BR-001`〜(フラット連番・ゼロ詰め 3 桁・欠番なし)

## 骨格

BR は**カテゴリ別ファイル内の HTML テーブル(1 行 = 1 要件)**で保持する。BR ファイルは単一種別なので種別節は持たず、要件を HTML テーブルで直接並べる。

- **BR ファイルの表骨格**: ファイル先頭の `# 見出し` + 要約 + メタ(掲載要件は全件 優先度 P0 ・ ステータス ドラフト と 1 行で明示)に続けて HTML テーブル 1 つ。列は `ID / 名称 / 機能グループ / 要件`(列順は識別子→名称→値→説明)。**ID セル先頭に `<span id="BR-NNN"></span>BR-NNN`** を置きアンカーを保持する(他層からの参照先)。`要件` セルで役割別の箇条書きが要るときは `<ul><li>` を使う(GFM ではなく生 HTML テーブル)。**リード文は置かない。**
- **業務要件 `BR-*`**: 表の 1 行(`ID / 名称 / 機能グループ / 要件`)。関連列は持たない。`要件` セルは役割別の箇条書き可(粒度に従い実装詳細は持たない)。優先度は全件 P0 のためファイル先頭メタに 1 回記載し、列には持たない。
- **`### 関連` セクションを持たない**(対応業務UC・関連 NFR・関連業務要件は記載しない。書式上の指示)。
- 粒度・逆参照(要件は後工程へのリンクを持たない/BR と FR の重複を作らない)のルールは [../CLAUDE.md](../CLAUDE.md) の粒度分離ルールを参照。

## 追加方法

要件を追加するときは、**BR は当該カテゴリファイルの表へ `<tr>` 行を 1 つ追加**(ID セルに `<span id="BR-NNN"></span>`、`機能グループ` 列で振り分け)。`01_business_requirement/index.md` の業務要件はカテゴリ単位の集約テーブル(`BR-ACCOUNT` 等のカテゴリ集約コードで 1 カテゴリ = 1 行)で索引し、件数列も更新する。

## 記載例(ファイル骨格)

```markdown
# アカウント(業務要件)

> **本ページは「アカウント・ユーザー・アクセス制御」カテゴリの業務要件を定義します。**

*種別 業務要件 ・ 掲載要件は全件 優先度 P0 ・ ステータス ドラフト*

<table>
  <thead>
    <tr><th>ID</th><th>名称</th><th>機能グループ</th><th>要件</th></tr>
  </thead>
  <tbody>
    <tr>
      <td><span id="BR-001"></span>BR-001</td>
      <td>名称</td>
      <td>機能グループ</td>
      <td>業務意図(誰が・何のために・何を可能にするか + 業務上の制約/リスク/方針)。役割別が要るときは <ul><li>役割A: …</li><li>役割B: …</li></ul></td>
    </tr>
  </tbody>
</table>
```
