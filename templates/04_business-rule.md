# 業務ルール(RULE)テンプレート

> **本テンプレートは業務ルール(RULE)の記載骨格(`### ルール`)を定義します。**

運用ルールの正本は [../CLAUDE.md](../CLAUDE.md)。共通記載スタイルは [共通記載スタイル](00_common-style.md) を参照する。要件仕様の 3 フォルダ構成・6 カテゴリの共通説明は [テンプレート索引](README.md) を参照する。

- **配置先**: `01_requirements/01_business_requirement/08_rule.md`(`## <分類>` ではなく RULE 節で統合)
- **採番**: `RULE-001`〜(フラット連番・ゼロ詰め 3 桁・欠番なし)

## 骨格

RULE は **FR / BR 文中の定量しきい値・ポリシーを抽出**して `08_rule.md` に RULE 節として保持する。

- 各要件節の骨格: `## <span id="ID"></span>ID: 名称` → 要約 → メタ → `### ルール`。**リード文は置かない。** ID アンカーは要件見出しに付け、節見出し(ルール)には付けない(ファイル内で重複するため)。
- **業務ルール `RULE-*`**: FR / BR 文中の定量しきい値・ポリシーを抽出し、`### ルール` のみを持つ。
- **下流へのリンクを持たない(逆参照禁止)**: RULE は業務UC・基本設計など後工程へのリンク(`### 適用UC` 等)を持たない。どの業務UC・設計で使われるかはトレーサビリティ一覧表(`02_basic_design/00_traceability/index.md`)で一元管理し、要件側からは辿らない(下流 → 上流の一方向のみ)。詳細は [../CLAUDE.md](../CLAUDE.md) の「ドキュメントの粒度分離ルール」逆参照禁止を参照。

## 追加方法

RULE は種別節へ `## <span id="ID"></span>…` を追加する。

## 記載例(要件節骨格)

```markdown
## <span id="RULE-001"></span>RULE-001: 名称

> **本ルールは「…」を定義します。**

*種別 業務ルール ・ ステータス ドラフト*

### ルール

…(定量しきい値・ポリシーを記述)
```
