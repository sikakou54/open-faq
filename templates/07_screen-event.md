# 画面イベント設計(EVT)テンプレート

> **本テンプレートは画面イベント設計(EVT)の記載骨格を定義します。**

運用ルールの正本は [../CLAUDE.md](../CLAUDE.md)。共通記載スタイルは [共通記載スタイル](00_common-style.md) を参照する。

- **配置先**: `02_basic_design/01_frontend/02_screen_events/`
- **採番**: `EVT-001`〜(1 イベント = 1 ファイル・ゼロ詰め 3 桁・欠番なし)

## 骨格

**1 イベント = 1 ファイル(`EVT-NNN`)**。`## 項目`(画面イベントID / イベント名 / 対応画面ID `SCR` / 対応業務UC `UC` / 対象項目ID `IT` / 呼出API `API` / 遷移先 `SCR`)+ `## 処理`(正本は SCR §6)+ 備考。

## 記載ルール

- 複数 EVT が 1 つの業務UCを実現する(EVT→UC は多:1)。`対応業務UC` は当該イベントが属する業務処理単位(統合後 UC)を指す。
- `## 処理` の正本は SCR §6(画面イベント一覧)。
- 画面・UC・API への参照は [共通記載スタイル](00_common-style.md) の相互参照アンカー例に従う(画面イベント → 画面 `[SCR-001](../01_screens/SCR-001.md#SCR-001)`)。
