# 機能要件(FR)テンプレート

> **本テンプレートは機能要件(FR)の記載骨格(`### 要件` + `### シーケンス`)を定義します。**

運用ルールの正本は [../CLAUDE.md](../CLAUDE.md)。共通記載スタイルは [共通記載スタイル](00_common-style.md) を参照する。要件仕様の 3 フォルダ構成・6 カテゴリの共通説明は [テンプレート索引](README.md) を参照する。

- **配置先**: `01_requirements/02_functional_requirement/`(`01_account-fr.md` 〜 `06_security-fr.md` の 6 カテゴリ別ファイル)
- **採番**: `FR-001`〜(フラット連番・ゼロ詰め 3 桁・欠番なし)

## 骨格

FR は**カテゴリ別ファイル内に各要件を節として保持**する。FR ファイルは単一種別なので種別節は持たず、要件節を直接並べる。

- 各要件節の骨格: `## <span id="ID"></span>ID: 名称` → 要約 → メタ → `### 要件` → **`### シーケンス`(FR のみ)**。**リード文は置かない。** ID アンカーは要件見出しに付け、節見出し(要件 / シーケンス 等)には付けない(ファイル内で重複するため)。
- **機能要件 `FR-*`**: `### 要件` + `### シーケンス`(関連セクションなし)。優先度 `P0`/`P1`/`P2` とカテゴリ振り分け根拠の `機能グループ` はメタに記載。
- **`### 関連` セクションを持たない**(書式上の指示)。粒度・逆参照(要件は後工程へのリンクを持たない/FR から BR の業務観点を再掲しない)のルールは [../CLAUDE.md](../CLAUDE.md) の粒度分離ルールを参照。

## `### シーケンス`(FR のみ)

- アクターは固定3者 `ユーザー / UI / サーバー`(`actor U as ユーザー` / `participant UI as UI` / `participant S as サーバー`)の mermaid `sequenceDiagram`。
- `autonumber` を付け、要求に応答線を対で描く。分岐は `alt/else/end`。
- 数値しきい値・状態名・テーブル/API ID・SQL は書かず相互作用の流れに限定し、無人処理は `Note over S: トリガ` で S 中心に。
- 図中に Markdown リンク・`id=` を書かない。
- UC 単位の詳細シーケンスは `02_basic_design/03_sequences/`(`SEQ-*`)が正本。

## 追加方法

FR は種別節へ `## <span id="ID"></span>…` を追加し、`機能グループ` で振り分ける(FR には `### シーケンス` も付ける)。

## 記載例(要件節骨格)

```markdown
## <span id="FR-001"></span>FR-001: 管理者はアカウント登録

> **本機能は「…」を定義します。**

*種別 機能要件 ・ 優先度 P0 ・ 機能グループ アカウント ・ ステータス ドラフト*

### 要件

…(システム機能の細粒度を記述)

### シーケンス

\`\`\`mermaid
sequenceDiagram
    autonumber
    actor U as ユーザー
    participant UI as UI
    participant S as サーバー
    U->>UI: 操作
    UI->>S: 要求
    S-->>UI: 応答
    UI-->>U: 結果表示
    alt 条件
        S-->>UI: …
    else
        S-->>UI: …
    end
\`\`\`
```
