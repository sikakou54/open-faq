# DB設計(TBL)テンプレート

> **本テンプレートは DB設計(TBL)の記載骨格(1 テーブル = 1 ファイル)を定義します。**

運用ルールの正本は [../CLAUDE.md](../CLAUDE.md)。共通記載スタイルは [共通記載スタイル](00_common-style.md) を参照する。

- **配置先**: `02_basic_design/02_backend/04_database/`
- **採番**: `TBL-001`〜(1 テーブル = 1 ファイル・通し連番・ゼロ詰め 3 桁・欠番なし)

## 骨格

**1 テーブル = 1 ファイル(`TBL-NNN`)**。物理名に分類接頭辞 `M_`(マスタ)/ `T_`(トランザクション)/ `H_`(履歴)/ `TP_`(ワーク・一時)を残す。骨格:

- H1 アンカー `# <span id="TBL-NNN"></span><物理名>(<論理名>)`
- リード文
- `### 項目`(テーブルID / 物理名 / 論理名 / 概要 / 主キー(PK)/ 論理削除 / 監査項目 / 対応業務UC(逆引き)/ 利用API(逆引き))
- 概要
- カラム定義(PK / FK / UK / index / NULL / DEFAULT / 制約)
- コード値

データモデルの正本は `02_backend/04_database/index.md` および各 `TBL-*`。

## 記載ルール

- 各 `##`/`###` 直後に 1〜2 文のリード文を置いてから表へ入る(表をいきなり置かない)。
- DDL 等は ` ```sql ` コードフェンス。
- `対応業務UC` / `利用API` は逆引きとして保持する。[共通記載スタイル](00_common-style.md) の相互参照アンカー例に従う。
